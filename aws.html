CLI v2

Setup

Install AWS CLI v2, configure credentials, and verify you can hit your account. Five minutes.

Install

macOS
# pkg installer
curl "https://awscli.amazonaws.com/AWSCLIV2.pkg" -o "AWSCLIV2.pkg"
sudo installer -pkg AWSCLIV2.pkg -target /

# or via Homebrew
brew install awscli

aws --version
# aws-cli/2.15.x Python/3.x Darwin/24.x
Linux
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install

# Apple Silicon / ARM
curl "https://awscli.amazonaws.com/awscli-exe-linux-aarch64.zip" -o "awscliv2.zip"
Windows
# MSI installer
msiexec.exe /i https://awscli.amazonaws.com/AWSCLIV2.msi

# or in Powershell:
winget install -e --id Amazon.AWSCLI

aws configure (default profile)
aws configure

# AWS Secret Access Key: ...

Named profiles
Most teams have multiple accounts (dev, staging, prod). Use a profile per account.
aws configure --profile prod
# every command:
aws s3 ls --profile prod
# or via env var:
export AWS_PROFILE=prod

SSO (recommended for humans)
No long-lived keys on your laptop. Federated, time-limited credentials.
aws configure sso
aws sso login --profile prod

# in ~/.aws/config:
[profile prod]
sso_session = my-sso
sso_account_id = 123456789012
sso_role_name = PowerUser
region = us-west-2
Env vars (CI / scripts)
export AWS_ACCESS_KEY_ID=AKIA...
export AWS_SECRET_ACCESS_KEY=...
export AWS_SESSION_TOKEN=...        # for STS
export AWS_REGION=us-east-1
export AWS_PROFILE=prod              # OR use a profile
Verify
aws sts get-caller-identity
# {
#   "UserId": "AIDA...",
#   "Account": "123456789012",
#   "Arn": "arn:aws:iam::123...:user/jane"
# }
Don't: commit ~/.aws/credentials, paste keys into Slack, bake them into Docker images, or check them into Lambda env vars. Prefer IAM roles (EC2 instance role, Lambda execution role, EKS IRSA) — they hand out short-lived credentials automatically.

IAM Basics

Identity (who) + Policy (what) + Resource (where). Every API call is authorized against a policy.

Concepts

ThingWhat it is
UserLong-lived identity tied to a person (or service). Can have access keys, console password, MFA.
RoleAn identity that can be assumed. Returns short-lived credentials (STS). Used by EC2, Lambda, EKS, cross-account access, federation.
PolicyJSON document. Identity policy attaches to user/role. Resource policy attaches to a resource (e.g., S3 bucket policy).
GroupBag of users that share policies. No nesting.
STSSecurity Token Service. Issues short-lived creds via AssumeRole, AssumeRoleWithWebIdentity, etc.

Minimal S3 policy

Read-only on one bucket
{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Action": ["s3:GetObject", "s3:ListBucket"],
    "Resource": [
      "arn:aws:s3:::my-bucket",
      "arn:aws:s3:::my-bucket/*"
    ]
  }]
}
Write to one prefix
{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Action": ["s3:PutObject", "s3:DeleteObject"],
    "Resource": "arn:aws:s3:::my-bucket/uploads/*"
  }]
}
Trust policy (role assumed by EC2)
Attached to a role; says who can assume me.
{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Principal": {"Service": "ec2.amazonaws.com"},
    "Action": "sts:AssumeRole"
  }]
}

S3 Essentials

  
A bucket is a flat namespace inside a region. Keys look like paths but aren't folders. Below are the commands you'll run weekly.


  
Buckets

  

    

      
Create / list / delete

      
# create — bucket names are GLOBAL
aws s3 mb s3://my-bucket --region us-east-1

aws s3 ls
aws s3 ls s3://my-bucket
aws s3 ls s3://my-bucket --recursive --human-readable --summarize

# delete (must be empty)
aws s3 rb s3://my-bucket
# or force-empty + delete:
aws s3 rb s3://my-bucket --force

    

    

      
cp — single file

      
aws s3 cp ./report.csv s3://my-bucket/reports/

# download
aws s3 cp s3://my-bucket/reports/report.csv ./

# cross-bucket
aws s3 cp s3://src/key s3://dst/key

# with metadata + storage class
aws s3 cp ./big.tgz s3://my-bucket/ \
  --storage-class STANDARD_IA \
  --metadata "team=ml,owner=jane" \
  --content-type application/gzip

    

    

      
sync — directory

      
Copies only files that differ. Default does not delete remote files missing locally — pass --delete for that.

      
aws s3 sync ./build/ s3://my-bucket/v1/ \
  --exclude "*.tmp" \
  --exclude ".git/*" \
  --include "*.html" \
  --delete \
  --dryrun

    

    

      
rm

      
aws s3 rm s3://my-bucket/old.csv

# entire prefix (recursive)
aws s3 rm s3://my-bucket/tmp/ --recursive

# with filter
aws s3 rm s3://my-bucket/ --recursive \
  --exclude "*" --include "*.log"

    

    

      
Presigned URL

      
Time-limited signed URL anyone can use without AWS creds. Default expiry is 3600s.

      
aws s3 presign s3://my-bucket/report.csv \
  --expires-in 900

# for upload (PUT) — needs s3api:
aws s3 presign s3://my-bucket/upload.bin \
  --expires-in 600
# Note: aws s3 presign issues GET URLs.
# For PUT presigns use the SDK or s3api command.

--storage-class  STANDARD | STANDARD_IA | INTELLIGENT_TIERING |
                 ONEZONE_IA | GLACIER_IR | GLACIER | DEEP_ARCHIVE
--exclude PATTERN     # can repeat
--include PATTERN     # overrides exclude
--dryrun              # show what would happen

--no-progress

S3 Advanced

  
Versioning, encryption, lifecycle, hosting. The features that turn a bucket from a dumping ground into infrastructure.


  
Versioning

  

    

      
Enable / list / delete

      
aws s3api put-bucket-versioning \
  --bucket my-bucket \
  --versioning-configuration Status=Enabled

# list versions
aws s3api list-object-versions --bucket my-bucket --prefix reports/

# delete a specific version
aws s3api delete-object \
  --bucket my-bucket --key reports/q1.csv \
  --version-id 3HL4kqtJlcpXroDTDmJ+rmSpXd3dIbrHY+MTRCxf3vjVBH40Nrjfkd

    

    

      
MFA Delete

      
Forces an MFA token to delete an object version or disable versioning. Root account only.

      
aws s3api put-bucket-versioning \
  --bucket my-bucket \
  --versioning-configuration Status=Enabled,MFADelete=Enabled \
  --mfa "arn:aws:iam::123456789012:mfa/root 123456"

    

  


  
Encryption

  

    

      
Default bucket encryption

      
# SSE-S3 (AES256, AWS-managed)
aws s3api put-bucket-encryption \
  --bucket my-bucket \
  --server-side-encryption-configuration '{
    "Rules":[{"ApplyServerSideEncryptionByDefault":{
      "SSEAlgorithm":"AES256"}}]}'

# SSE-KMS (customer-managed key)
aws s3api put-bucket-encryption \
  --bucket my-bucket \
  --server-side-encryption-configuration '{
    "Rules":[{"ApplyServerSideEncryptionByDefault":{
      "SSEAlgorithm":"aws:kms",
      "KMSMasterKeyID":"alias/my-key"}}]}'

    

    

      
SSE-S3 vs SSE-KMS

      
SSE-S3: zero-config, AWS owns the key. Free. Use unless you need audit/IAM control over key usage.
SSE-KMS: per-key audit logs, key rotation, cross-account grants. Costs $0.03/10k requests + KMS key fee.

    

    

      
Per-object encryption

      
aws s3 cp file.txt s3://my-bucket/ \
  --sse aws:kms \
  --sse-kms-key-id alias/my-key

Hello World Workflows

  
Three end-to-end recipes you can run today.


  
① Host a static site on S3 + CloudFront

  

    

      
1. Make the bucket

      
BUCKET=my-site-$(uuidgen | tr A-Z a-z)
REGION=us-east-1

aws s3 mb s3://$BUCKET --region $REGION

# keep public access blocked — CloudFront will serve the content
aws s3api put-public-access-block \
  --bucket $BUCKET \
  --public-access-block-configuration \
   "BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true"

    

    

      
2. Sync the site

      
aws s3 sync ./site/ s3://$BUCKET/ \
  --delete \
  --cache-control "public,max-age=3600" \
  --exclude "*.html" \
  --exclude "service-worker.js"

# short-cache HTML so deploys are visible quickly
aws s3 sync ./site/ s3://$BUCKET/ \
  --exclude "*" --include "*.html" \
  --cache-control "public,max-age=60"

    

    

      
3. CloudFront in front

      
aws cloudfront create-distribution \
  --origin-domain-name $BUCKET.s3.$REGION.amazonaws.com
# In the console: turn on OAC, attach a bucket policy that
# allows the CloudFront SourceArn (see "Bucket policy" card).

# invalidate after deploy
aws cloudfront create-invalidation \
  --distribution-id E1ABC... --paths "/*"

③ S3 → Lambda trigger

  

    

      
Trust + execution role

      
aws iam create-role \
  --role-name lambda-thumb \
  --assume-role-policy-document '{
    "Version":"2012-10-17",
    "Statement":[{"Effect":"Allow",
      "Principal":{"Service":"lambda.amazonaws.com"},
      "Action":"sts:AssumeRole"}]}'

aws iam attach-role-policy \
  --role-name lambda-thumb \
  --policy-arn arn:aws:iam::aws:policy/service-role/AWSLambdaBasicExecutionRole

aws iam put-role-policy --role-name lambda-thumb \
  --policy-name s3rw --policy-document '{
    "Version":"2012-10-17",
    "Statement":[{"Effect":"Allow",
      "Action":["s3:GetObject","s3:PutObject"],
      "Resource":"arn:aws:s3:::my-bucket/*"}]}'

    

    

      
Create the function

      
zip thumb.zip lambda_function.py

aws lambda create-function \
  --function-name thumb \
  --runtime python3.12 \
  --handler lambda_function.handler \
  --role arn:aws:iam::123:role/lambda-thumb \
  --zip-file fileb://thumb.zip

    

    

      
Wire S3 → Lambda

      
aws lambda add-permission \
  --function-name thumb --statement-id s3 \
  --action lambda:InvokeFunction \
  --principal s3.amazonaws.com \
  --source-arn arn:aws:s3:::my-bucket

aws s3api put-bucket-notification-configuration \
  --bucket my-bucket \
  --notification-configuration '{
    "LambdaFunctionConfigurations":[{
      "LambdaFunctionArn":"arn:aws:lambda:us-east-1:123:function:thumb",
      "Events":["s3:ObjectCreated:*"],
      "Filter":{"Key":{"FilterRules":[
        {"Name":"prefix","Value":"uploads/"},
        {"Name":"suffix","Value":".jpg"}]}}
    }]}'

CloudWatch Logs

  

    

      
Tail / filter

aws logs tail /aws/lambda/thumb --follow --since 10m

# filter
aws logs tail /aws/lambda/thumb --since 1h \
  --filter-pattern '"ERROR" -InvalidArgument'

  --log-group-name /aws/lambda/thumb \

  --end-time   $(date -u +%s) \
  --query-string 'fields @timestamp, @message | filter @message like /ERROR/ | limit 50'

Troubleshooting

  
If your CLI call is unhappy, read this page first.


  
Common errors

  

    
SymptomLikely cause & fix

    
      
An error occurred (AccessDenied)IAM policy doesn't grant the action. Check both the identity policy and the bucket/resource policy. aws sts get-caller-identity first to confirm which identity. Use the IAM Policy Simulator.

      
NoSuchBucketBucket name typo, or you're in the wrong region/account. Bucket names are global; aws s3 ls only lists buckets your account owns.

      
The bucket you are attempting to access must be addressed using the specified endpointRegion mismatch. Pass --region us-west-2 or set AWS_REGION.

      
ExpiredToken / InvalidTokenSSO/STS creds expired. Run aws sso login --profile foo or re-assume the role.

      
Could not connect to the endpoint URLWrong region (no S3 in that one), VPC endpoint policy, or proxy. Try --debug.

      
RequestTimeTooSkewedYour clock drifted. Sync time (NTP).

      
An error occurred (SlowDown) when calling …S3 is rate-limiting per-prefix. Spread keys across more prefixes; back off and retry. Per-prefix limit: 3500 PUT/COPY/POST/DELETE and 5500 GET req/s.

      
InvalidAccessKeyIdWrong key, deactivated key, or the key belongs to another account. Re-check ~/.aws/credentials and $AWS_PROFILE.

      
EntityAlreadyExists on bucket createBucket names are global. Pick another.

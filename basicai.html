What is a tensor?
A tensor is a multi-dimensional array — a number, vector, matrix, or higher. Everything inside a neural network is tensors: inputs, weights, gradients, outputs. PyTorch's torch.Tensor is just like NumPy's ndarray, plus it can run on GPU and remember how it was made (autograd).
What is a model?
A function with learnable parameters. Inputs flow through layers (matrix multiplies + nonlinearities) and produce outputs. We adjust the parameters by comparing outputs to truth, computing gradients, and nudging weights downhill.
Training
Show the model many input/target pairs. For each one: forward pass → compute loss → backward pass (gradients) → optimizer step (update weights). Repeat thousands of times. Loss curve goes down. That's it. That's deep learning.
Inference
Once trained, freeze weights and just run forward passes on new inputs. Faster (no gradient bookkeeping) and deterministic. Use model.eval() and torch.no_grad().

Build a Model — your first nn.Module

Every PyTorch model is a subclass of nn.Module. You define layers in __init__ and the forward pass in forward. That's the whole API.

a 2-layer MLP for 10-class classification (full)
import torch
import torch.nn as nn

class Net(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1  = nn.Linear(784, 128)
        self.relu = nn.ReLU()
        self.fc2  = nn.Linear(128, 10)

    def forward(self, x):
        x = self.fc1(x)
        x = self.relu(x)
        x = self.fc2(x)
        return x

model = Net()

Loss & Optimizer

The loss is a number that says "how wrong was the prediction" — smaller is better. The optimizer uses gradients to nudge weights so the loss decreases. Two losses cover 90% of cases; two optimizers cover 95% of cases.

losses
regression nn.MSELoss — mean squared error
classification nn.CrossEntropyLoss — softmax + log + nll baked in
binary nn.BCEWithLogitsLoss
optimizers
simple optim.SGD(params, lr=0.01)
default-good optim.Adam(params, lr=1e-3)
modern optim.AdamW — Adam + decoupled weight decay
putting it together
import torch.nn as nn
import torch.optim as optim

model     = Net()
loss_fn   = nn.CrossEntropyLoss()
optimizer = optim.Adam(model.parameters(), lr=1e-3)

the canonical PyTorch training loop
model.train()
for epoch in range(num_epochs):
    for x, y in dataloader:
        pred = model(x)              # 1. forward
        loss = loss_fn(pred, y)      # 2. loss
        optimizer.zero_grad()        # 3. clear old gradients
        loss.backward()              # 4. backprop
        optimizer.step()             # 5. update weights

Common bug: if you forget optimizer.zero_grad(), gradients accumulate across batches and your model trains as if every batch were the sum of all previous batches. Always zero them.

Inference — using a trained model

After training, you want to run the model on new inputs without computing gradients (faster, less memory) and without dropout / batchnorm acting up. Two switches do this.

inference template
model.eval()                          # dropout off, BN uses running stats
with torch.no_grad():                 # no gradient tracking
    logits = model(x_new)
    probs  = torch.softmax(logits, dim=-1)
    pred   = probs.argmax(dim=-1)        # class id

Save / load: torch.save(model.state_dict(), 'm.pt') to save, model.load_state_dict(torch.load('m.pt')) to load. Always save the state_dict, not the whole model.

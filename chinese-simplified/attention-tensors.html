01 Attention Is All You Need · 张量形状

一个张量,一路走到 logits

Attention Is All You Need 里的 Transformer 就是一套配方:拿一个源 token ID 张量, 送过编码器(encoder),把目标 token 送过一个对编码器做交叉注意力(cross-attention)的 解码器(decoder),投影到词表,最后吐出下一个 token 的 logits。 本页追踪形状——从输入 ID 到最终 [B, S_tgt, V] 概率的每一个阶段,按颜色编码,让你读论文时永远不会跟丢这个张量。

h (heads) = 8 # 注意力头数
d_k = d_v = 64 # d_model / h
d_ff = 2048# FFN 内部维度(4·d_model)

02 输入与嵌入

从 token ID 到一个准备好上下文的张量

在做任何注意力之前,输入必须先变成一个向量张量。三个快速阶段: 输入整数 ID,输出实数向量,并通过相加把位置信息烘焙进去。

阶段 1 · token ID
分词器的输出。一个批里有两条样本,每条 8 个 token。
$\text{ids}\in\mathbb{Z}^{B\times S}$
形状 [B=2, S=8] · dtype: int32 
# 两个句子,各填充/截断至 S=8 个 token
ids = [[ 101, 2057, 2293, 3019, 2025, 2748, 102, 0 ],
       [ 101, 7592, 2088,  102,    0,    0,   0, 0 ]]
# 形状:[B=2, S=8]
阶段 2 · 嵌入查表

权重矩阵 $E\in\mathbb{R}^{V\times d_{model}}$ 为词表里每个条目保存一行。 按 ID 查表。张量多出一维:每个整数被一个大小为 d_model 的实数向量替换。

$X = E[\text{ids}] \;\in\; \mathbb{R}^{B\times S\times d_{model}}$
[2, 10] → [2, 10, 512] 
import torch.nn as nn
embed = nn.Embedding(num_embeddings=30522, embedding_dim=512)
X = embed(ids)            # [B, S] -> [B, S, d_model]
print(X.shape)            # torch.Size([2, 10, 512])

参数量:$V \cdot d_{model} = 30522 \cdot 512 \approx 15.6\text{M}$——光这张嵌入表 就比任何单层的其余部分都大。

阶段 3 · + 位置编码

自注意力对位置是等变的(permutation-equivariant)——没有位置信息时,"the cat sat" 和 "sat cat the" 会产生相同的输出。论文加上了一个与 $X$ 同形状的固定正弦模式。

$\mathrm{PE}_{(pos, 2i)} = \sin\!\big(pos / 10000^{2i/d_{model}}\big)$
$\mathrm{PE}_{(pos, 2i+1)} = \cos\!\big(pos / 10000^{2i/d_{model}}\big)$
[2, 10, 512] + [10, 512] → [2, 10, 512] 
# PE 形状为 [S, d_model],沿 B 广播
X = X + PE[:S]            # [B, S, d_model] + [S, d_model] -> [B, S, d_model]

进去什么形状,出来还是什么形状。信息进的是数值,不是维度。

阶段 3 结束: 现在我们有一个形状为 [B, S, d_model] 的张量,它既关于这些词,也关于它们在哪里。之后的每个阶段 都是在这个布局上做变换。

03 多头注意力

阶段 4–11 · 注意力,逐维拆解

这篇论文存在的全部理由。每个 token 都变成一个查询(query)、键(key)和值(value); 计算相似度、缩放、做 softmax,再用来混合这些值。我们还会 reshape 两次: 一次拆分多头,一次再把它们拼回去。

为什么现在每个形状都一目了然: 这里 h = 8 个头、 S = 10 个序列位置、d_k = 64——每一维都有各自独特的数字, 所以你总能分清哪个轴是哪个。(早期版本用序列长度 8,正好和 8 个头撞上: [2, 8, 8, 64] 根本看不清。换成 S=10 后变成 [2, 8, 10, 64]——清清楚楚。)

阶段 4 · Q、K、V 投影

三个独立的线性映射,每个都是 $\mathbb{R}^{d_{model}}\to\mathbb{R}^{d_{model}}$:

$Q = X W^Q,\quad K = X W^K,\quad V = X W^V$
X [2, 10, 512] Wᵠ [512, 512] → Q [2, 10, 512] 
# 三个线性层,论文中无偏置,实际使用时加偏置也可以
Wq = nn.Linear(d_model, d_model)   # 512*512 = 262 144 个参数
Wk = nn.Linear(d_model, d_model)
Wv = nn.Linear(d_model, d_model)

Q = Wq(X)                              # [B, S, d_model]
K = Wk(X)                              # [B, S, d_model]
V = Wv(X)                              # [B, S, d_model]

三个形状为 $[d_{model}, d_{model}]$ 的矩阵——这里的总参数量: $3\cdot 512\cdot 512 = 786\,432$。某些实现会把这三个融合成一个 $[d_{model}, 3\cdot d_{model}]$ 的矩阵。

阶段 5 · 为 h 个头做 reshape 与 transpose

把最后一维切成 $h$ 块,每块 $d_k = d_{model}/h$,然后 transpose,让头变成更外层的一维。 没有新参数——只是 view + permute。

[2, 10, 512] view→ [2, 10, 8, 64] transpose(1,2)→ [2, 8, 10, 64] 
B, S, d_model = Q.shape
h, d_k = 8, d_model // h         # 8, 64

# 将最后一维拆成 (h, d_k),然后把头移到紧邻批次的位置
Q = Q.view(B, S, h, d_k).transpose(1, 2)   # [B, h, S, d_k]
K = K.view(B, S, h, d_k).transpose(1, 2)   # [B, h, S, d_k]
V = V.view(B, S, h, d_k).transpose(1, 2)   # [B, h, S, d_k]
为什么头很重要: 有了 $h=8$ 个独立的 $[S, d_k=64]$ 子空间,每个头 都能专门负责一种不同的关系模式。计算量不变:8 个小的 $64$ 维注意力,而不是一个 $512$ 维的。
阶段 6 · Q @ Kᵀ —— 两两相似度

对每个头,每个查询都关注每个键。这次矩阵乘会收缩 $d_k$ 轴, 为每个头、每个 batch 产生一个 $S\times S$ 的分数矩阵。

$\text{scores} = Q K^\top$
Q [2, 8, 10, 64] Kᵀ [2, 8, 64, 10] → scores [2, 8, 10, 10] 
scores = Q @ K.transpose(-2, -1)   # [B, h, S, S]
# 元素 (b, head, i, j) = token i 对 token j 的关注程度(在该头中)
阶段 7 · 按 1/√d_k 缩放

不缩放的话,$d_k$ 维向量的点积会以 $\sqrt{d_k}$ 的量级增长——过大的值会把 softmax 推入饱和区,梯度随之消失。把每个分数都除以 $\sqrt{d_k}$。

$\text{scaled} = \text{scores} / \sqrt{d_k} = \text{scores} / 8$
[2, 8, 10, 10] ÷ √64 [2, 8, 10, 10] 形状不变 
import math
scores = scores / math.sqrt(d_k)   # 1/√64 = 1/8 = 0.125
阶段 8 · 沿最后一维做 softmax

把原始分数变成每个查询的一个概率分布。$[S,S]$ 块的每一行 都和为 1——这些就是注意力权重。

$A = \mathrm{softmax}(\text{scaled},\ \text{dim}=-1)$
[2, 8, 10, 10] → [2, 8, 10, 10] 每行和为 1 
import torch.nn.functional as F
A = F.softmax(scores, dim=-1)    # [B, h, S, S]

对解码器自注意力,会在 softmax 之前加上一个 $-\infty$ 加性掩码来强制因果性。

阶段 9 · attention @ V

用注意力权重去混合值向量。每个查询的输出都是全部 $S$ 个值向量的加权和。 被收缩的是内侧的 $S$ 轴。

A [2, 8, 10, 10] V [2, 8, 10, 64] → out [2, 8, 10, 64] 
head_out = A @ V                  # [B, h, S, d_k]
阶段 10 · 拼接多头(transpose + reshape)

把多个头粘回成一个大的最后一维。transpose(1,2) 把头移到紧挨 $d_k$ 的位置, 然后 reshape 把它们合并。无参数。

[2, 8, 10, 64] transpose→ [2, 10, 8, 64] reshape→ [2, 10, 512] 
out = head_out.transpose(1, 2).contiguous()   # [B, S, h, d_k] —— 此时各头相邻
out = out.reshape(B, S, d_model)              # reshape 把 8 个头首尾相接地合并 = Concat(head_0..head_7)
这个 reshape 就是拼接(concat)。 论文写的是 $\mathrm{MHA}(Q,K,V) = \mathrm{Concat}(\text{head}_1,\dots,\text{head}_h)\,W^O$,但这里 并没有真正的 torch.cat。做完 transpose(1,2) 后,对每一个 S = 10 的 token 位置, h = 8 个头在最后两维里彼此相邻: [2, 10, 8, 64]。 reshape 随后把这最后两维拍平 ($h\cdot d_k = 8\cdot 64 = $ 512),把各头沿特征维首尾相接地排开。 这种在连续内存上的并排布局正是拼接——所以 concat 并没有被省略,它恰恰就是这个 reshape 在做的事 (.contiguous() 只是让 transpose 之后的 reshape 合法)。每个 token 的 512 维向量是:
[512] = [head₀ 64]‖[head₁ 64]‖ … ‖[head₇ 64] 等价于 torch.cat([head_0, …, head_7], dim=-1)
阶段 11 · 输出投影 W_O

再来一个 $\mathbb{R}^{d_{model}\to d_{model}}$ 的线性映射。形状不变,但让模型 在拼接之后能跨头混合信息。

$\text{MHA}(X) = \mathrm{Concat}(\text{head}_1, \dots, \text{head}_h)\,W^O$
[2, 10, 512] ·W_O [2, 10, 512] 
Wo = nn.Linear(d_model, d_model)   # 又 262 144 个参数
mha_out = Wo(out)                       # [B, S, d_model]

MHA 总参数量: $4 \cdot d_{model}^2 = 4\cdot 512^2 = 1{,}048{,}576$ (Q、K、V、O——各 $d_{model}\times d_{model}$)。

注意力结束: 我们带着形状 [B, S, d_model] 离开——和进来时的形状一模一样。注意力是一个保持形状的算子。

04 FFN 与编码器块

LayerNorm、FFN,以及残差高速公路

注意力之后,编码器块用两条残差连接、两个 LayerNorm,以及一个逐位置(position-wise)的 FFN 把一切缝合起来。FFN 是大部分参数所在之处——也是很多人认为事实性知识被存储的地方。

阶段 12 · 残差 + LayerNorm

把原始的 $X$ 加回来(跳跃连接),然后对每个 token 沿 $d_{model}$ 做归一化。

$X' = \mathrm{LayerNorm}(X + \mathrm{MHA}(X))$
[2, 10, 512] + [2, 10, 512] → LN → [2, 10, 512] 
ln1 = nn.LayerNorm(d_model)      # 2 * d_model = 1024 个参数(γ, β)
X_post = ln1(X + mha_out)         # [B, S, d_model]

LN 沿最后一维归一化:每个 $[d_{model}]$ 向量被变成均值 0、方差 1,再由可学习的 $\gamma, \beta\in\mathbb{R}^{d_{model}}$ 重新缩放。批次和 token 位置都保持互相独立。

阶段 13–14 · FFN 线性层 1 + ReLU

一个逐位置的前馈网络:同一个 MLP 独立地作用到每个 token 上。 第一个线性层把 $d_{model} \to d_{ff}$ 扩展(通常是 $4\cdot d_{model}$)。

$H = \mathrm{ReLU}(X' W_1 + b_1)$
[2, 10, 512] ·W₁ [2, 10, 2048] → ReLU → [2, 10, 2048] 
W1 = nn.Linear(d_model, d_ff)       # 512 * 2048 = 1 048 576 个参数
H = F.relu(W1(X_post))            # [B, S, d_ff]
阶段 15 · FFN 线性层 2

第二个线性层投影回去:$d_{ff} \to d_{model}$。恢复成标准形状,好让下一个块从头开始。

$Y = H W_2 + b_2$
[2, 10, 2048] ·W₂ [2, 10, 512] 
W2 = nn.Linear(d_ff, d_model)       # 2048 * 512 = 又 1 048 576 个参数
ffn_out = W2(H)                       # [B, S, d_model]

FFN 总参数量: $2\cdot d_{model}\cdot d_{ff} = 2\cdot 512\cdot 2048 = 2{,}097{,}152$—— 大约是全部四个注意力投影加起来的两倍。大模型的重量就压在这里。

阶段 16 · 残差 + LayerNorm(再来一次)

和阶段 12 一样的套路。加上 FFN 之前的张量,做 layer-norm,编码器块就完成了。

$Y' = \mathrm{LayerNorm}(X' + Y)$
[2, 10, 512] → [2, 10, 512] 块输出形状 = 块输入形状 
ln2 = nn.LayerNorm(d_model)
out = ln2(X_post + ffn_out)       # [B, S, d_model]
堆叠并重复: 论文堆了 $N=6$ 个相同的编码器块。每一个的输入形状 都等于它的输出形状,这正是它们能干净地串接起来的原因。 编码器总参数量(单块,base):大约 $4\cdot 512^2 + 2\cdot 512\cdot 2048 + 4\cdot 512 \approx 3.15$M。

14 行写完整个编码器块

EncoderBlock —— 最小化 PyTorch
class EncoderBlock(nn.Module):
    def __init__(self, d_model=512, h=8, d_ff=2048):
        super().__init__()
        self.attn = nn.MultiheadAttention(d_model, h, batch_first=True)
        self.ffn  = nn.Sequential(
            nn.Linear(d_model, d_ff), nn.ReLU(), nn.Linear(d_ff, d_model)
        )
        self.ln1 = nn.LayerNorm(d_model)
        self.ln2 = nn.LayerNorm(d_model)

    def forward(self, x):
        # x: [B, S, d_model]
        a, _ = self.attn(x, x, x)
        x = self.ln1(x + a)
        x = self.ln2(x + self.ffn(x))
        return x                       # [B, S, d_model] —— 块输出

05 解码器块

解码器块 —— 先带掩码自注意力,再交叉注意力

解码器接收目标 token(右移一位),产生一个同时了解两件事的张量:(a) 它目前为止看过的目标 token ——通过带掩码的自注意力——以及 (b) 源端——通过交叉注意力, 其中 Q 来自解码器,K、V 来自编码器输出(即 "memory")。与编码器相同的嵌入维度、相同的残差 + LN 规范。每层三个子块,论文里六层。

论文配置(仍然)
B = 2, S_src = 10, S_tgt = 7
d_model = 512, h = 8, d_k = 64
# S_tgt 与 S_src 互相独立。
# 交叉注意力是它们唯一相遇的地方。
为什么要"右移"?

训练时,解码器看到的是前缀加了 <sos> 的标准目标,并预测以 <eos> 结尾的同一个目标。

# target  : "die katze sitzt"           ← 真值
# dec_in  : "<sos> die katze sitzt"     ← 右移(输入)
# labels  : "die katze sitzt <eos>"     ← 我们要预测的

阶段 17 · 解码器输入 → 嵌入

[B, S_tgt] → 嵌入查表 → [B, S_tgt, d_model] = [2, 7, 512]

阶段 19 · 带掩码的自注意力 (因果掩码)

和编码器自注意力完全相同——只是分数矩阵会在对角线以上的每个位置盖上 −∞,这样位置 i 永远看不到 i+1, i+2, ...。经过 softmax 后这些位置恰好变成 0。

# scores: [B, h, S_tgt, S_tgt] = [2, 8, 7, 7]
mask = torch.triu(torch.ones(S_tgt, S_tgt), diagonal=1).bool()
scores = scores.masked_fill(mask, float('-inf'))
A = F.softmax(scores, dim=-1)        # 下三角,每行和为 1
[B, h, S_tgt, S_tgt] = [2, 8, 7, 7] 上三角在 softmax 后 = 0

A 掩码 — 交互

为什么解码器不能偷看未来

训练时解码器会一次性看到整个目标(teacher forcing)。为了仍然学会"预测下一个 token", 我们禁止位置 i 关注任何更靠后的位置 j > i——这些单元格在 softmax 之前被设为 −∞,所以 softmax 之后它们恰好为 0。切换掩码、滑动查询位置来感受一下。

因果掩码:开启
查询 i →
键 j → (S_tgt = 7)
允许 (j ≤ i) 屏蔽 (−∞)

掩码关闭时,每个查询都能看到每个键——这就是编码器自注意力。掩码开启时, 网格变成下三角:位置 0 只看到自己,位置 6 看到全部 7 个。正是这一点让模型既能在所有位置上 并行训练,又能在推理时从左到右逐步生成。

Padding 掩码

另外,S_src = 10 的源序列末尾有 padding 零(第 0 行:9 个真实 token + 1 个 pad;第 1 行:3 个真实 + 7 个 pad)。 这些位置也会被掩成 −∞,这样注意力永远不会流进 padding。点击某个键位置把它切换为 padding, 看它整列变黑。

阶段 20 · 交叉注意力 (Q ← 解码器,K、V ← 编码器)

这里编码器终于和解码器对上话了。解码器发出查询;编码器输出提供键和值。

# dec_x: [B, S_tgt, d_model] = [2, 7, 512]   ← 来自带掩码的自注意力
# memory: [B, S_src, d_model] = [2, 10, 512]  ← 编码器输出(阶段 16)
Q = W_q(dec_x).view(B, S_tgt, h, d_k).transpose(1, 2)   # [B, h, S_tgt, d_k]
K = W_k(memory).view(B, S_src, h, d_k).transpose(1, 2)   # [B, h, S_src, d_k]
V = W_v(memory).view(B, S_src, h, d_k).transpose(1, 2)   # [B, h, S_src, d_k]
scores = Q @ K.transpose(-2, -1) / math.sqrt(d_k)         # [B, h, S_tgt, S_src] = [2, 8, 7, 10]
A = F.softmax(scores, dim=-1)                              # 最后一维是 S_src
out = (A @ V).transpose(1, 2).reshape(B, S_tgt, d_model) # [2, 7, 512]

关键点:这个分数矩阵不是方阵。它是 [B, h, S_tgt, S_src]——每个目标 token 都关注每个源 token。

16 行写完解码器块

DecoderBlock —— 最小化 PyTorch
class DecoderBlock(nn.Module):
    def __init__(self, d=512, h=8, d_ff=2048):
        super().__init__()
        self.self_attn  = nn.MultiheadAttention(d, h, batch_first=True)
        self.cross_attn = nn.MultiheadAttention(d, h, batch_first=True)
        self.ffn = nn.Sequential(nn.Linear(d, d_ff), nn.ReLU(), nn.Linear(d_ff, d))
        self.ln1, self.ln2, self.ln3 = nn.LayerNorm(d), nn.LayerNorm(d), nn.LayerNorm(d)

    def forward(self, x, memory, causal_mask):
        # x:      [B, S_tgt, d]      memory: [B, S_src, d]
        a, _ = self.self_attn(x, x, x, attn_mask=causal_mask)
        x = self.ln1(x + a)                 # [B, S_tgt, d]
        c, _ = self.cross_attn(x, memory, memory)
        x = self.ln2(x + c)                 # [B, S_tgt, d]
        x = self.ln3(x + self.ffn(x))
        return x                            # [B, S_tgt, d]

解码器堆栈的输出(经过 6 个这样的块之后): [B, S_tgt, d_model] = [2, 7, 512]。下一站:投影到词表。

06 输出 → logits

从隐藏状态到 logits

一个 nn.Linear(d_model, V) 就把每个 token 的隐藏向量变成词表里每个词的分数。 沿最后一维做 softmax,把这些分数变成概率。这就是整个输出头。在论文里,W_lm 与输入嵌入矩阵绑定——同一份参数,转置而已。

阶段 22 · 线性投影到词表

[B, S_tgt, d_model] @ W_lm.T → [B, S_tgt, V] = [2, 7, 37000] 
# dec_out: [B, S_tgt, d_model]
# W_lm:    [V, d_model]   ← 常与 embed.weight 绑定
logits = dec_out @ W_lm.T           # [B, S_tgt, V]
# 或者直接:
logits = self.lm_head(dec_out)      # nn.Linear(d_model, V)

这些是原始分数——每个词表 token、每个位置、每条样本各一个。 还不是概率。如果你的损失是 CrossEntropyLoss,不要对它们做 softmax; 为了数值稳定,它内部会自己做。

阶段 23 · Softmax → 概率 (只在你真正需要时)

probs = F.softmax(logits, dim=-1)   # [B, S_tgt, V] —— 在 V 上每行和为 1

在 B·S_tgt 个位置的每一个上,你现在都有一个覆盖整个词表的分布。 这 37000 个数里大多数都极小。只有少数几个集中在正确答案附近。

输出头的参数量

光 W_lm 就是 V × d_model = 37000 × 512 ≈ 19M 个参数—— 通常是模型里最大的一块权重。论文把它和输入嵌入矩阵绑定,把权重内存减半, 并(经验上)有助于泛化。

完整的前向传播,端到端

Transformer.forward —— 忠于论文
def forward(self, src_ids, tgt_in_ids):
    # src_ids:    [B, S_src]   tgt_in_ids: [B, S_tgt]
    src = self.src_embed(src_ids) + self.pos(src_ids)        # [B, S_src, d]
    tgt = self.tgt_embed(tgt_in_ids) + self.pos(tgt_in_ids)    # [B, S_tgt, d]

    memory = src
    for layer in self.encoder_layers:
        memory = layer(memory)                                  # [B, S_src, d]

    causal = self.causal_mask(tgt_in_ids.shape[1])
    x = tgt
    for layer in self.decoder_layers:
        x = layer(x, memory, causal)                            # [B, S_tgt, d]

    logits = self.lm_head(x)                                     # [B, S_tgt, V]
    return logits

def train_step(model, optimizer, batch):
    # batch:长度为 B 的 list[(src_str, tgt_str)]
    src_ids = tokenize_src(batch)         # [B, S_src]    int64
    dec_in, labels = tokenize_tgt(batch)  # 各为 [B, S_tgt]

    src = model.src_embed(src_ids)*math.sqrt(d) + model.pos(src_ids)
    tgt = model.tgt_embed(dec_in)*math.sqrt(d) + model.pos(dec_in)

    memory = src
    for layer in model.encoder_layers: memory = layer(memory)

    causal = model.causal_mask(dec_in.shape[1])
    x = tgt
    for layer in model.decoder_layers: x = layer(x, memory, causal)

    logits = model.lm_head(x)              # [B, S_tgt, V]
    loss = F.cross_entropy(
        logits.view(-1, V), labels.view(-1),
        ignore_index=-100, label_smoothing=0.1)

    optimizer.zero_grad(); loss.backward(); optimizer.step()
    return loss.item()                     # 标量

@torch.no_grad()
def generate(model, src_text, max_len=32):
    src_ids = tokenize_src([(src_text, "")])      # [1, S_src]

    src = model.src_embed(src_ids)*math.sqrt(d) + model.pos(src_ids)
    memory = src
    for layer in model.encoder_layers:
        memory = layer(memory)                          # [1, S_src, d]

    dec_in = torch.tensor([[SOS]])                         # [1, 1]
    for _ in range(max_len):
        tgt = model.tgt_embed(dec_in)*math.sqrt(d) + model.pos(dec_in)
        causal = model.causal_mask(dec_in.shape[1])
        x = tgt
        for layer in model.decoder_layers:
            x = layer(x, memory, causal)                 # [1, t, d]
        logits = model.lm_head(x)                          # [1, t, V]
        next_id = logits[:, -1, :].argmax(-1, keepdim=True)
        if next_id.item() == EOS: break
        dec_in = torch.cat([dec_in, next_id], dim=1)    # [1, t+1]

    return detokenize(dec_in[0, 1:].tolist())          # str

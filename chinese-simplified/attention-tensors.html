一个张量，直到 logits

Attention Is All You Need Transformer 是一套流程：取一个源 token ID 张量， 送入编码器，再将目标 token 送入对编码器进行交叉注意力的解码器， 投影到词表，输出下一 token 的 logits。 本页追踪形状——从输入 ID 到最终 [B, T_tgt, V] 概率的每个阶段，用颜色标注， 让你读论文时不会迷失在张量里。

论文默认值（base 配置）
B (batch) = 2 # 批大小
T_src (src len) = 8 # 源序列 token 数
T_tgt (tgt len) = 7 # 目标序列 token 数
d_model = 512 # 嵌入维度
h (heads) = 8 # 注意力头数
d_k = d_v = 64 # d_model / h
d_ff = 2048# FFN 内部维度（4·d_model）
V (vocab) = 37000# 英→德 BPE 词表
N (layers) = 6 # 编码器层数 = 解码器层数
颜色图例（全局统一）
B 批次 T token 数 d_model h 头数 d_k d_ff

当某个维度在两个阶段之间发生变化时，它会高亮显示。看着同一个张量被 拆分为多个头、相乘、缩放、softmax，再拼接回来。

阶段 1–3 · 从 token ID 到可用于注意力的张量

在进行任何注意力运算之前，输入必须先变成向量张量。三个快速阶段： 整数 ID 输入，实值向量输出，并通过加法将位置信息融入其中。

阶段 1 · token ID
分词器输出。批次中两个样本，各八个 token。
$\text{ids}\in\mathbb{Z}^{B\times T}$
shape [B=2, T=8] · dtype: int32 
# 两个句子，各填充/截断至 T=8 个 token
ids = [[ 101, 2057, 2293, 3019, 2025, 2748, 102, 0 ],
       [ 101, 7592, 2088,  102,    0,    0,   0, 0 ]]
# 形状：[B=2, T=8]
阶段 2 · 嵌入查找

权重矩阵 $E\in\mathbb{R}^{V\times d_{model}}$ 每行对应词表中的一个词条。 按 ID 查找。张量增加了一个维度：每个整数被替换为大小为 d_model 的实值向量。

$X = E[\text{ids}] \;\in\; \mathbb{R}^{B\times T\times d_{model}}$
[2, 8] → [2, 8, 512] 
import torch.nn as nn
embed = nn.Embedding(num_embeddings=30522, embedding_dim=512)
X = embed(ids)            # [B, T] -> [B, T, d_model]
print(X.shape)            # torch.Size([2, 8, 512])

参数量：$V \cdot d_{model} = 30522 \cdot 512 \approx 15.6\text{M}$ — 单是嵌入表就比任何单一层的其他部分都大。

阶段 3 · + 位置编码

自注意力具有置换等变性——若无位置信息，"the cat sat" 和 "sat cat the" 将产生相同的输出。论文在 $X$ 上加入一个固定的正弦模式，形状相同。

$\mathrm{PE}_{(pos, 2i)} = \sin\!\big(pos / 10000^{2i/d_{model}}\big)$
$\mathrm{PE}_{(pos, 2i+1)} = \cos\!\big(pos / 10000^{2i/d_{model}}\big)$
[2, 8, 512] + [8, 512] → [2, 8, 512] 
# PE 形状为 [T, d_model]，沿 B 广播
X = X + PE[:T]            # [B, T, d_model] + [T, d_model] -> [B, T, d_model]

输入与输出形状相同。信息进入数值，而不改变维度。

阶段 3 结束：我们现在得到一个形状为 [B, T, d_model] 的张量，它既包含词语的语义，也包含词语的位置。后续每个阶段都是对这一布局的变换。

阶段 4–11 · 多头注意力，逐维度解析

这正是这篇论文存在的核心理由。每个 token 变成一个 query、key 和 value； 计算相似度、缩放、softmax，再用来混合 value 向量。我们还进行两次形状变换： 一次拆分多头，一次合并回来。

阶段 4 · Q、K、V 投影

三个独立的线性映射，各自将 $\mathbb{R}^{d_{model}}\to\mathbb{R}^{d_{model}}$：

$Q = X W^Q,\quad K = X W^K,\quad V = X W^V$
X [2, 8, 512] Wᵠ [512, 512] → Q [2, 8, 512] 
# 三个线性层，论文中无偏置，实际使用时加偏置也可以
Wq = nn.Linear(d_model, d_model)   # 512*512 = 262 144 个参数
Wk = nn.Linear(d_model, d_model)
Wv = nn.Linear(d_model, d_model)

Q = Wq(X)                              # [B, T, d_model]
K = Wk(X)                              # [B, T, d_model]
V = Wv(X)                              # [B, T, d_model]

三个形状为 $[d_{model}, d_{model}]$ 的矩阵——此处总参数量： $3\cdot 512\cdot 512 = 786\,432$。部分实现将这三个矩阵合并为 一个 $[d_{model}, 3\cdot d_{model}]$ 的单矩阵。

阶段 5 · 为 h 个头重塑并转置

将最后一维切分为 $h$ 块，每块大小为 $d_k = d_{model}/h$，然后转置使头成为外层维度。 无新参数——仅 view + permute。

[2, 8, 512] view→ [2, 8, 8, 64] transpose(1,2)→ [2, 8, 8, 64] 
B, T, d_model = Q.shape
h, d_k = 8, d_model // h         # 8, 64

# 将最后一维拆成 (h, d_k)，然后将头移到紧邻批次的位置
Q = Q.view(B, T, h, d_k).transpose(1, 2)   # [B, h, T, d_k]
K = K.view(B, T, h, d_k).transpose(1, 2)   # [B, h, T, d_k]
V = V.view(B, T, h, d_k).transpose(1, 2)   # [B, h, T, d_k]
为什么需要多头：有 $h=8$ 个独立的 $[T, d_k=64]$ 子空间，每个 头可以专注于不同的关系模式。计算量不变：8 个小的 $64$ 维注意力，代替一个 $512$ 维的。
阶段 6 · Q @ Kᵀ — 两两相似度

对每个头，每个 query 关注每个 key。矩阵乘法收缩 $d_k$ 轴， 为每个批次的每个头产生一个 $T\times T$ 的得分矩阵。

$\text{scores} = Q K^\top$
Q [2, 8, 8, 64] Kᵀ [2, 8, 64, 8] → scores [2, 8, 8, 8] 
scores = Q @ K.transpose(-2, -1)   # [B, h, T, T]
# 元素 (b, head, i, j) = token i 对 token j 的关注程度（在该头中）
阶段 7 · 缩放因子 1/√d_k

若不缩放，$d_k$ 维向量的点积会随 $\sqrt{d_k}$ 增大——过大的值会使 softmax 进入饱和区域，导致梯度消失。将每个得分除以 $\sqrt{d_k}$。

$\text{scaled} = \text{scores} / \sqrt{d_k} = \text{scores} / 8$
[2, 8, 8, 8] ÷ √64 [2, 8, 8, 8] 形状不变 
import math
scores = scores / math.sqrt(d_k)   # 1/√64 = 1/8 = 0.125
阶段 8 · 沿最后一轴的 softmax

将原始得分转化为每个 query 的概率分布。$[T,T]$ 块的每一行之和为 1—— 这些就是注意力权重。

$A = \mathrm{softmax}(\text{scaled},\ \text{dim}=-1)$
[2, 8, 8, 8] → [2, 8, 8, 8] 行和为 1 
import torch.nn.functional as F
A = F.softmax(scores, dim=-1)    # [B, h, T, T]

在解码器自注意力中，softmax 之前会施加一个加性的 $-\infty$ 掩码以强制因果性。

阶段 9 · attention @ V

用注意力权重混合 value 向量。每个 query 的输出是所有 $T$ 个 value 向量的加权和。 收缩轴是内部的 $T$。

A [2, 8, 8, 8] V [2, 8, 8, 64] → out [2, 8, 8, 64] 
head_out = A @ V                  # [B, h, T, d_k]（加权值向量之和）
阶段 10 · 拼接多头（转置 + 重塑）

将各头重新拼合为一个大的最后维度。transpose(1,2) 将头移到紧邻 $d_k$， 再用 reshape 合并。无参数。

[2, 8, 8, 64] transpose→ [2, 8, 8, 64] reshape→ [2, 8, 512] 
out = head_out.transpose(1, 2).contiguous()   # [B, T, h, d_k]（将头轴移回）
out = out.reshape(B, T, d_model)              # [B, T, d_model]（合并头与 d_k）
阶段 11 · 输出投影 W_O

再一个 $\mathbb{R}^{d_{model}\to d_{model}}$ 线性映射。形状不变，但允许模型 在拼接后跨头混合信息。

$\text{MHA}(X) = \mathrm{Concat}(\text{head}_1, \dots, \text{head}_h)\,W^O$
[2, 8, 512] ·W_O [2, 8, 512] 
Wo = nn.Linear(d_model, d_model)   # 又 262 144 个参数
mha_out = Wo(out)                       # [B, T, d_model]

MHA 总参数量： $4 \cdot d_{model}^2 = 4\cdot 512^2 = 1{,}048{,}576$ （Q、K、V、O——各 $d_{model}\times d_{model}$）。

注意力阶段结束：输出形状为 [B, T, d_model] ——与进入时完全相同。注意力是一个保形状的算子。

阶段 12–16 · LayerNorm、FFN 与残差通路

注意力之后，编码器块通过两个残差连接、两个 LayerNorm 和一个位置级 FFN 将一切串联起来。 FFN 是大多数参数所在之处——也被许多人认为是存储事实性知识的地方。

阶段 12 · 残差 + LayerNorm

将原始 $X$ 加回（跳跃连接），然后在 $d_{model}$ 轴上对每个 token 归一化。

$X' = \mathrm{LayerNorm}(X + \mathrm{MHA}(X))$
[2, 8, 512] + [2, 8, 512] → LN → [2, 8, 512] 
ln1 = nn.LayerNorm(d_model)      # 2 * d_model = 1024 个参数（γ, β）
X_post = ln1(X + mha_out)         # [B, T, d_model]

LN 沿最后一轴归一化：每个 $[d_{model}]$ 向量变为均值 0、方差 1， 再由可学习的 $\gamma, \beta\in\mathbb{R}^{d_{model}}$ 重新缩放。批次和 token 位置保持独立。

阶段 13–14 · FFN 线性层 1 + ReLU

位置级前馈网络：对每个 token 独立应用相同的 MLP。 第一个线性层将维度从 $d_{model}$ 扩展到 $d_{ff}$（通常为 $4\cdot d_{model}$）。

$H = \mathrm{ReLU}(X' W_1 + b_1)$
[2, 8, 512] ·W₁ [2, 8, 2048] → ReLU → [2, 8, 2048] 
W1 = nn.Linear(d_model, d_ff)       # 512 * 2048 = 1 048 576 个参数
H = F.relu(W1(X_post))            # [B, T, d_ff]
阶段 15 · FFN 线性层 2

第二个线性层将维度压回：$d_{ff} \to d_{model}$。恢复标准形状，以便下一个块重新开始。

$Y = H W_2 + b_2$
[2, 8, 2048] ·W₂ [2, 8, 512] 
W2 = nn.Linear(d_ff, d_model)       # 2048 * 512 = 又 1 048 576 个参数
ffn_out = W2(H)                       # [B, T, d_model]

FFN 总参数量： $2\cdot d_{model}\cdot d_{ff} = 2\cdot 512\cdot 2048 = 2{,}097{,}152$ —— 大约是所有四个注意力投影加起来的两倍。这正是大型模型"堆参数"的地方。

阶段 16 · 残差 + LayerNorm（再次）

与阶段 12 相同的模式。加上 FFN 前的张量，进行层归一化，编码器块完成。

$Y' = \mathrm{LayerNorm}(X' + Y)$
[2, 8, 512] → [2, 8, 512] 块输出形状 = 块输入形状 
ln2 = nn.LayerNorm(d_model)
out = ln2(X_post + ffn_out)       # [B, T, d_model]
堆叠并重复：论文将 $N=6$ 个相同的编码器块堆叠在一起。每个块的输入形状等于输出形状， 这就是它们可以干净地组合的原因。单块的总编码器参数（base 配置）约为 $4\cdot 512^2 + 2\cdot 512\cdot 2048 + 4\cdot 512 \approx 3.15$M。

14 行实现完整编码器块

EncoderBlock — 最小化 PyTorch 实现
class EncoderBlock(nn.Module):
    def __init__(self, d_model=512, h=8, d_ff=2048):
        super().__init__()
        self.attn = nn.MultiheadAttention(d_model, h, batch_first=True)
        self.ffn  = nn.Sequential(
            nn.Linear(d_model, d_ff), nn.ReLU(), nn.Linear(d_ff, d_model)
        )
        self.ln1 = nn.LayerNorm(d_model)
        self.ln2 = nn.LayerNorm(d_model)

    def forward(self, x):
        # x: [B, T, d_model]（输入形状）
        a, _ = self.attn(x, x, x)
        x = self.ln1(x + a)
        x = self.ln2(x + self.ffn(x))
        return x                       # [B, T, d_model]（输出形状相同）

阶段 17–21 · 解码器块

解码器接收目标 token（右移一位），生成同时了解以下两方面的张量： (a) 它迄今所见的目标 token——通过掩码自注意力； (b) 源序列——通过交叉注意力，Q 来自解码器，K、V 来自编码器输出（"memory"）。 嵌入维度和残差+LN 规范与编码器相同。每层三个子块，论文中共六层。

论文配置（沿用）
B = 2, T_src = 8, T_tgt = 7
d_model = 512, h = 8, d_k = 64
# T_tgt 与 T_src 无关。
# 交叉注意力是它们唯一相遇的地方。
为什么是"右移"？

训练时解码器看到以 <sos> 为前缀的目标序列， 预测以 <eos> 结尾的同一目标序列。

# target  : "die katze sitzt"           ← 真实标签
# dec_in  : "<sos> die katze sitzt"     ← 右移（解码器输入）
# labels  : "die katze sitzt <eos>"     ← 需要预测的内容

阶段 17 · 解码器输入 → 嵌入

[B, T_tgt] → embed lookup → [B, T_tgt, d_model] = [2, 7, 512]

阶段 19 · 掩码自注意力 （因果掩码）

与编码器自注意力完全相同——只是得分矩阵对角线以上的每个位置都被标记为 −∞，使得位置 i 无法查看 i+1, i+2, ...。 经 softmax 后这些位置精确变为 0。

# scores: [B, h, T_tgt, T_tgt] = [2, 8, 7, 7]
mask = torch.triu(torch.ones(T_tgt, T_tgt), diagonal=1).bool()
scores = scores.masked_fill(mask, float('-inf'))
A = F.softmax(scores, dim=-1)        # 下三角，每行之和为 1
[B, h, T_tgt, T_tgt] = [2, 8, 7, 7] softmax 后上三角 = 0

阶段 20 · 交叉注意力 （Q ← 解码器，K、V ← 编码器）

这是编码器最终与解码器"对话"的地方。解码器发出 query； 编码器输出提供 key 和 value。

# dec_x: [B, T_tgt, d_model] = [2, 7, 512]   ← 来自掩码自注意力
# memory: [B, T_src, d_model] = [2, 8, 512]  ← 编码器输出（阶段 16）
Q = W_q(dec_x).view(B, T_tgt, h, d_k).transpose(1, 2)   # [B, h, T_tgt, d_k]
K = W_k(memory).view(B, T_src, h, d_k).transpose(1, 2)   # [B, h, T_src, d_k]
V = W_v(memory).view(B, T_src, h, d_k).transpose(1, 2)   # [B, h, T_src, d_k]
scores = Q @ K.transpose(-2, -1) / math.sqrt(d_k)         # [B, h, T_tgt, T_src] = [2, 8, 7, 8]
A = F.softmax(scores, dim=-1)                              # 最后一轴为 T_src
out = (A @ V).transpose(1, 2).reshape(B, T_tgt, d_model) # [2, 7, 512]

关键点：得分矩阵不是方阵。它的形状是 [B, h, T_tgt, T_src]——每个目标 token 都关注每个源 token。

16 行实现完整解码器块

DecoderBlock — 最小化 PyTorch 实现
class DecoderBlock(nn.Module):
    def __init__(self, d=512, h=8, d_ff=2048):
        super().__init__()
        self.self_attn  = nn.MultiheadAttention(d, h, batch_first=True)
        self.cross_attn = nn.MultiheadAttention(d, h, batch_first=True)
        self.ffn = nn.Sequential(nn.Linear(d, d_ff), nn.ReLU(), nn.Linear(d_ff, d))
        self.ln1, self.ln2, self.ln3 = nn.LayerNorm(d), nn.LayerNorm(d), nn.LayerNorm(d)

    def forward(self, x, memory, causal_mask):
        # x:      [B, T_tgt, d]      memory: [B, T_src, d]（编码器输出）
        a, _ = self.self_attn(x, x, x, attn_mask=causal_mask)
        x = self.ln1(x + a)                 # [B, T_tgt, d]
        c, _ = self.cross_attn(x, memory, memory)
        x = self.ln2(x + c)                 # [B, T_tgt, d]
        x = self.ln3(x + self.ffn(x))
        return x                            # [B, T_tgt, d]

解码器栈的输出（经过 6 个这样的块之后）：[B, T_tgt, d_model] = [2, 7, 512]。下一步：投影到词表。

阶段 22–23 · 从隐状态到 logits

一个 nn.Linear(d_model, V) 将每个 token 的隐状态向量转化为词表中每个词的得分。 沿最后一轴的 softmax 将这些得分转为概率。这就是完整的输出头。 论文中 W_lm 与输入嵌入矩阵共享参数——参数相同，转置使用。

阶段 22 · 线性投影到词表

[B, T_tgt, d_model] @ W_lm.T → [B, T_tgt, V] = [2, 7, 37000] 
# dec_out: [B, T_tgt, d_model]
# W_lm:    [V, d_model]   ← 通常与 embed.weight 共享参数
logits = dec_out @ W_lm.T           # [B, T_tgt, V]
# 或简单地：
logits = self.lm_head(dec_out)      # nn.Linear(d_model, V)

这些是原始得分——每个词表 token、每个位置、每个样本各一个。 尚不是概率。如果损失函数是 CrossEntropyLoss，不要对它们做 softmax； 该函数内部会处理以保持数值稳定性。

阶段 23 · Softmax → 概率 （仅在真正需要时使用）

probs = F.softmax(logits, dim=-1)   # [B, T_tgt, V] — V 轴上每行之和为 1

在 B·T_tgt 个位置的每一处，你现在都有一个覆盖整个词表的分布。 这 37000 个数字中大多数都非常微小。只有少数集中在正确答案附近。

输出头的参数量

仅 W_lm 就有 V × d_model = 37000 × 512 ≈ 1900万参数 —— 通常是模型中单个最大的权重。论文将它与输入嵌入矩阵共享参数， 将权重内存减半，且实验上有助于泛化。

Transformer.forward — 忠于论文的实现
def forward(self, src_ids, tgt_in_ids):
    # src_ids:    [B, T_src]   tgt_in_ids: [B, T_tgt]
    src = self.src_embed(src_ids) + self.pos(src_ids)        # [B, T_src, d]（源嵌入+位置）
    tgt = self.tgt_embed(tgt_in_ids) + self.pos(tgt_in_ids)    # [B, T_tgt, d]（目标嵌入+位置）

    memory = src
    for layer in self.encoder_layers:
        memory = layer(memory)                                  # [B, T_src, d]

    causal = self.causal_mask(tgt_in_ids.shape[1])
    x = tgt
    for layer in self.decoder_layers:
        x = layer(x, memory, causal)                            # [B, T_tgt, d]

    logits = self.lm_head(x)                                     # [B, T_tgt, V]
    return logits

def train_step(model, optimizer, batch):
    # batch: list[(src_str, tgt_str)]，长度为 B
    src_ids = tokenize_src(batch)         # [B, T_src]    int64
    dec_in, labels = tokenize_tgt(batch)  # [B, T_tgt] 各一个

    src = model.src_embed(src_ids)*math.sqrt(d) + model.pos(src_ids)
    tgt = model.tgt_embed(dec_in)*math.sqrt(d) + model.pos(dec_in)

    memory = src
    for layer in model.encoder_layers: memory = layer(memory)

    causal = model.causal_mask(dec_in.shape[1])
    x = tgt
    for layer in model.decoder_layers: x = layer(x, memory, causal)

    logits = model.lm_head(x)              # [B, T_tgt, V]
    loss = F.cross_entropy(
        logits.view(-1, V), labels.view(-1),
        ignore_index=-100, label_smoothing=0.1)

    optimizer.zero_grad(); loss.backward(); optimizer.step()
    return loss.item()                     # 标量

@torch.no_grad()
def generate(model, src_text, max_len=32):
    src_ids = tokenize_src([(src_text, "")])      # [1, T_src]

    src = model.src_embed(src_ids)*math.sqrt(d) + model.pos(src_ids)
    memory = src
    for layer in model.encoder_layers:
        memory = layer(memory)                          # [1, T_src, d]

    dec_in = torch.tensor([[SOS]])                         # [1, 1]（以 <sos> 开始）
    for _ in range(max_len):
        tgt = model.tgt_embed(dec_in)*math.sqrt(d) + model.pos(dec_in)
        causal = model.causal_mask(dec_in.shape[1])
        x = tgt
        for layer in model.decoder_layers:
            x = layer(x, memory, causal)                 # [1, t, d]
        logits = model.lm_head(x)                          # [1, t, V]
        next_id = logits[:, -1, :].argmax(-1, keepdim=True)
        if next_id.item() == EOS: break
        dec_in = torch.cat([dec_in, next_id], dim=1)    # [1, t+1]（追加新 token）

    return detokenize(dec_in[0, 1:].tolist())          # str（去掉 <sos>）

推理过程中的形状摘要： 原始字符串 → [1, T_src] → [1, T_src, d]（memory，仅一次）→ 循环：[1, t] → [1, t, d] → [1, t, V] → [1, V] → [1] → 追加 → 增长 → 最终字符串。

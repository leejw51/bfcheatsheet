AWS S3 CLI — 从 ls 到 put-bucket-policy

两套 CLI,一项存储服务。aws s3 是面向日常工作的友好高层封装 —— 复制、同步、预签名。aws s3api 则是 REST API 的低层镜像,用于版本控制、生命周期、分段上传、桶策略等精细操作。 跟着键盘把两者都过一遍。

aws s3 — 高层命令
类似文件系统的命令:ls、cp、 mv、sync、rm、 mb、rb、presign。 它会自动做很多聪明的事 —— 多线程上传、大文件分段上传、 失败重试 —— 90% 的场景下都用它。
aws s3api — 低层命令
每个 S3 API 动作对应一个子命令:list-objects-v2、 head-object、put-bucket-versioning、 create-multipart-upload…… 当 aws s3 不暴露某项功能时,或当 你想要原始 JSON 输出来配合 --query / jq 时使用。

你将敲一千遍的 S3 URI

# 协议      桶名             键(桶内对象路径)
s3://my-bucket/photos/2026/cat.jpg

# “文件夹”只是键的前缀 —— 它并不真正以对象的形式存在
s3://my-bucket/photos/2026/
注意:S3 的键不是真正的文件系统。“目录” photos/2026/ 只是因为存在以该前缀开头的对象才出现。 空目录是以 / 结尾的零字节对象。

IAM:你最常用到的四个 S3 动作

动作允许的操作资源示例
s3:ListBucket列出指定前缀下的键arn:aws:s3:::my-bucket
s3:GetObject下载对象arn:aws:s3:::my-bucket/*
s3:PutObject上传对象arn:aws:s3:::my-bucket/*
s3:DeleteObject删除对象(或为其打上删除标记)arn:aws:s3:::my-bucket/*
桶级动作 vs 对象级动作:s3:ListBucket 针对 桶的 ARN(不带 /*)。s3:GetObject 针对桶内的对象(带 /*)。混淆这两者 是新策略出现 “AccessDenied” 的头号原因。

列出与查看 — ls、head-object、list-objects-v2、--query

云存储的第一条戒律:动手之前先弄清桶里有什么。 两套 CLI 都能列出对象;s3api 给你 JSON, 可以通过 --query 配合 JMESPath 切片。

$ aws s3 ls

$ aws s3 ls s3://my-bucket

$ aws s3 ls s3://my-bucket/photos/ --

$ aws s3 ls s3://my-bucket --recursive --

aws s3api list-objects-v2 — 程序化列表

# 在前缀 p/ 下列出最多 10 个键,只投影 key + size

    --prefix p/ \
    --max-items 10 \
    --query 'Contents[].{Key:Key,Size:Size}'

$ aws s3api list-objects-v2 --bucket my-bucket --max-keys 1000 \

$ aws s3api list-objects-v2 --bucket my-bucket \
    --query 'Contents[].'

JMESPath 速成 — --query 参数

每条 aws 命令都返回 JSON。--query 'EXPR' 在客户端用 JMESPath 处理 JSON,然后才交给 --output 格式化。 

# 只取桶名
$ aws s3api list-buckets --query 'Buckets[].Name'

# 大于 1 MiB 的对象(数字部分要用反引号)
$ aws s3api list-objects-v2 --bucket B \
    --query 'Contents[?Size > `1048576`].Key'

# 最新的 5 个对象
$ aws s3api list-objects-v2 --bucket B \
    --query 'reverse(sort_by(Contents, &LastModified))[:5].[Key,LastModified]' \
    --output table

# 某个前缀的总大小(求和)
$ aws s3api list-objects-v2 --bucket B --prefix logs/ \
    --query 'sum(Contents[].Size)'
5 从 list-buckets 的结果中只返回桶名。
$ aws s3api list-buckets --query 'Buckets[].'
6 JMESPath:只保留严格大于 1 MiB 的对象。填入关键字。
--query 'Contents[Size > `1048576`].Key'

上传、下载与同步 — cp、mv、sync、exclude/include

这四个动词(cp、mv、 sync、rm)负责进出 S3 的 95% 字节流量。 在动用 --delete 之前,务必掌握 --exclude / --include 的顺序 以及 --dryrun。

cp — 单个文件或递归复制

# 上传
$ aws s3 cp ./report.pdf s3://my-bucket/reports/

# 下载
$ aws s3 cp s3://my-bucket/reports/report.pdf ./

# 递归 —— 整个目录
$ aws s3 cp ./build/ s3://my-bucket/site/ --recursive

# 带存储类型、加密、内容类型与元数据
$ aws s3 cp big.bin s3://my-bucket/data/big.bin \
    --storage-class STANDARD_IA \
    --sse aws:kms --sse-kms-key-id alias/data \
    --content-type application/octet-stream \
    --metadata ingest=2026-05-20,team=ml \
    --acl private
1 把整个本地目录上传到 S3 的某个前缀下。
$ aws s3 cp ./build/ s3://my-bucket/site/ --
2 把上传标记为低频访问。填入存储类型。
$ aws s3 cp file.txt s3://b/k --storage-class

mv — 复制后删除源

# 把构建产物从 staging 提升到 release 前缀
$ aws s3 mv s3://b/staging/v1.tar.gz s3://b/release/v1.tar.gz

# 本地 → S3(成功后会删除本地文件)
$ aws s3 mv ./done.zip s3://b/archive/

sync — 目录级别的超能力

# 镜像本地 → S3,会*新增*与*更新*,但不会删除
$ aws s3 sync ./public/ s3://my-site/

# 带删除的镜像:S3 最终与本地完全一致
$ aws s3 sync ./public/ s3://my-site/ --delete

# 跳过垃圾文件,只发布 JS 与 CSS
$ aws s3 sync ./dist/ s3://my-site/ \
    --exclude "*" \
    --include "*.js" \
    --include "*.css"

# 看看会改什么但不真正执行
$ aws s3 sync ./public/ s3://my-site/ --delete --dryrun
顺序很重要:--exclude / --include 按从左到右求值,后匹配的胜出。 通常先写 --exclude "*",再把想要的部分重新 include 回来。
3 把本地目录镜像到 S3,使本地删除的文件也在 S3 中删除,同时跳过 .tmp 文件。
$ aws s3  local/ s3://bucket/prefix/ -- --exclude "*.tmp"
4 不真正执行,预览破坏性 sync 会做什么。
$ aws s3 sync ./public/ s3://site/ --delete --

      rm — 小心使用,尤其是 --recursive

      # 单个对象
$ aws s3 rm s3://b/old.log

# 整个前缀 —— 不会有任何确认!
$ aws s3 rm s3://b/tmp/ --recursive

# 选择性:清理 logs/ 但保留 .json 文件
$ aws s3 rm s3://b/logs/ --recursive --exclude "*.json"
      版本控制提醒:对启用了版本控制的桶执行 aws s3 rm --recursive 只会增加删除标记 —— 旧版本仍然在。要真正释放空间,必须通过 s3api 显式删除版本。

      s3api 进阶工具 — 每个 S3 API 动作对应一个子命令

      当友好的 aws s3 命令不够用时, 就转到 s3api。它的输出始终是 JSON —— 用 --query 或 jq 处理即可。

      对象级操作

      # 从本地文件上传,显式指定 content-type 与元数据
$ aws s3api put-object \
    --bucket my-bucket \
    --key data/blob.bin \
    --body ./blob.bin \
    --content-type application/octet-stream \
    --metadata sha256=abcd...,team=ml

# 下载到文件
$ aws s3api get-object --bucket my-bucket --key data/blob.bin out.bin

# 服务端复制(字节不会经过你的电脑)
$ aws s3api copy-object \
    --bucket dest-bucket \
    --key archive/blob.bin \
    --copy-source my-bucket/data/blob.bin

# 删除某个特定版本
$ aws s3api delete-object \
    --bucket my-bucket --key data/blob.bin \
    --version-id 3HL4kqtJlcpXroDTDmJ+rmSpXd3dIbrHY
      1 服务端将 srcB/srcK 的键复制到 dstB/dstK。填入子命令与源参数。
      $ aws s3api  \
    --bucket dstB --key dstK \
    -- srcB/srcK

      对象标签

      $ aws s3api put-object-tagging \
    --bucket my-bucket --key data/x.bin \
    --tagging '{"TagSet":[{"Key":"env","Value":"prod"},{"Key":"team","Value":"ml"}]}'

$ aws s3api get-object-tagging --bucket my-bucket --key data/x.bin
      2 读取某个对象当前设置的标签。
      $ aws s3api  --bucket B --key K

      版本控制

      # Turn on versioning (irreversible-ish — you can only Suspend, not Disable)
$ aws s3api put-bucket-versioning \
    --bucket my-bucket \
    --versioning-configuration Status=Enabled

# See what's currently set
$ aws s3api get-bucket-versioning --bucket my-bucket
{ "Status": "Enabled" }

# All versions of an object
$ aws s3api list-object-versions --bucket my-bucket --prefix data/blob.bin
      3 在桶上启用版本控制。填入状态。
      $ aws s3api put-bucket-versioning --bucket B \
    --versioning-configuration Status=

      速查表 — 你最常见到的 s3api 子命令

      子命令必需参数使用场景
      list-buckets无账户级别的桶清单
      list-objects-v2--bucket分页遍历键
      head-object--bucket --key元数据、ETag、版本、加密信息
      head-bucket--bucket桶是否存在 & 我能否访问?

      存储桶与策略 — mb、rb、policy、CORS、lifecycle、公网访问阻断

      桶级别的配置往往决定了安全与成本的成败。 这些命令负责设置版本控制、公网访问阻断、CORS、生命周期规则 以及桶策略。

      创建与删除桶

      # 创建。区域影响 LocationConstraint(us-east-1 是隐式的 / “none”)
$ aws s3 mb s3://my-new-bucket --region us-east-1

# 删除。--force 会一并清空(仅高层命令支持)。
$ aws s3 rb s3://my-new-bucket --force

# s3api 等价命令(选项更多)
$ aws s3api create-bucket --bucket my-new-bucket \
    --create-bucket-configuration LocationConstraint=eu-west-1

$ aws s3api delete-bucket --bucket my-new-bucket
      1 在 us-east-1 中创建一个名为 cat-photos 的新桶。
      $ aws s3  s3://cat-photos --region us-east-1
      2 一次性删除一个非空桶。选择正确的 flag。
      $ aws s3 rb s3://cat-photos --

      阻断所有公网访问(优先做这件事!)

      $ aws s3api put-public-access-block \
    --bucket my-bucket \
    --public-access-block-configuration \
        BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true

$ aws s3api get-public-access-block --bucket my-bucket

      对象锁 — 一次写入多次读取

      # 仅在创建桶时启用 —— 之后无法开启
$ aws s3api create-bucket --bucket vault \
    --object-lock-enabled-for-bucket

# 默认保留期:7 年 COMPLIANCE 模式(对所有人不可变)
$ aws s3api put-object-lock-configuration \
    --bucket vault \
    --object-lock-configuration \
    '{"ObjectLockEnabled":"Enabled","Rule":{"DefaultRetention":{"Mode":"COMPLIANCE","Years":7}}}'
      GOVERNANCE vs COMPLIANCE:GOVERNANCE 允许具备 s3:BypassGovernanceRetention 的特权 IAM 用户绕过保留;COMPLIANCE 则会 阻止所有人(包括 root 用户)直到保留期结束。

      加密与存储类型 — SSE、KMS、IA、Glacier

      根据成本(以及读取延迟)选择存储类型,根据合规要求选择加密模式。 两者相互独立 —— 任何存储类型都支持任何加密模式。

      服务端加密(SSE)

      模式--server-side-encryption密钥由谁管理审计 / KMS 用法
      SSE-S3AES256S3无
      SSE-KMSaws:kmsKMS(你的 CMK)CloudTrail 记录每次 Decrypt
      SSE-KMS DSSEaws:kms:dsseKMS,双重加密合规工作负载
      SSE-C客户在请求头中提供密钥你自己密钥也得自己保管
      无——2023 起默认禁止(S3 自动加 AES256)

$ aws s3 cp f.bin s3://b/k --sse AES256

      分段上传与预签名 URL — 处理大文件与共享链接

      高层命令 aws s3 cp 对超过约 8 MB 的文件会自动启用分段上传。 当你以编程方式驱动上传时,以下是 s3api 的原语。

      分段上传 — 四步走

      ## 1. 开始 —— 拿到一个 UploadId
$ UID=$(aws s3api create-multipart-upload \
    --bucket B --key big.bin \
    --query 'UploadId' --output text)

## 2. 上传每个分段(除最后一段外,最少 5 MB)
$ ETAG1=$(aws s3api upload-part \
    --bucket B --key big.bin \
    --part-number 1 --upload-id $UID \
    --body part1.bin \
    --query 'ETag' --output text)
#  ……重复执行第 2..N 段……

## 3. 完成 —— 把分段列表交给 S3,服务端会拼接
$ aws s3api complete-multipart-upload \
    --bucket B --key big.bin --upload-id $UID \
    --multipart-upload '{"Parts":[{"ETag":"...","PartNumber":1},...]}'

## 4(替代). 取消 —— 释放本会被计费的存储
$ aws s3api abort-multipart-upload \
    --bucket B --key big.bin --upload-id $UID

      练习题 — S3 CLI 速射

      二十多道练习,涵盖最常敲的命令与参数。 aws s3 与 aws s3api 混合出题。

      1 身份检查 —— 哪条命令会返回你的账户 ID 与 ARN?
      $ aws sts

      $ aws s3 ls -- myprofile
      3 为单条命令覆盖区域。
      $ aws s3 ls s3://b -- eu-west-1
      4 把本地目录树递归上传到 S3。

      $ aws s3 sync ./pub/ s3://my-site/ --
      6 同步时排除全部,然后只重新包含 PNG 文件。
      $ aws s3 sync ./img/ s3://b/img/ \
    --exclude "*" \

      Boss 关卡 — 端到端 S3 流水线

      一个场景,十个空白。你正在为一份合规数据集发布新桶: 创建、加锁、启用版本、加密上传、列出、生成预签名分享链接、 安装生命周期、清理旧版本。每个空白都是真实的命令或参数。

      流水线:冷存储研究数据集

      合规要求:KMS 加密、不允许公开访问、启用版本、365 天过期、 90 天后转入 Glacier。你以 eu-west-1 区域、 research 配置文件操作。 

      # 1. 为整个 shell 会话挑选正确的配置文件 + 区域
$ export AWS_=research
$ export AWS_REGION=eu-west-1

# 2. 确认我们确实以 research IAM 用户身份登录
$ aws sts 

# 3. 创建桶
$ aws s3  s3://lab-genomics-2026 --region eu-west-1

# 4. 立刻锁住公网访问
$ aws s3api put- \
    --bucket lab-genomics-2026 \
    --public-access-block-configuration \
        BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true

# 5. 启用版本控制,误删可恢复
$ aws s3api put-bucket-versioning --bucket lab-genomics-2026 \
    --versioning-configuration Status=

# 6. 默认 KMS 加密,这样未来即使忘了 --sse 也仍会加密
$ aws s3api put-bucket-encryption --bucket lab-genomics-2026 \
    --server-side-encryption-configuration \
    '{"Rules":[{"ApplyServerSideEncryptionByDefault":{"SSEAlgorithm":"","KMSMasterKeyID":"alias/lab-data"}}]}'

# 7. 上传第一批,显式加密 + IA 存储类型
$ aws s3 cp ./batch-001/ s3://lab-genomics-2026/raw/2026-05/ \
    --recursive \
    --sse aws:kms --sse-kms-key-id alias/lab-data \
    --storage-class 

# 8. 验证 —— 以 MiB 单位列出
$ aws s3 ls s3://lab-genomics-2026/raw/ --recursive -- --summarize

# 9. 生成 24 小时的分享链接,供 PI 下载某个文件
$ aws s3  s3://lab-genomics-2026/raw/2026-05/cohort-A.tsv --expires-in 86400

# 10. 安装生命周期:90 天转 GLACIER、365 天过期、30 天清理旧版本
$ aws s3api put-bucket- \
    --bucket lab-genomics-2026 \
    --lifecycle-configuration file://lc.json

不要做:把 ~/.aws/credentials 提交到代码库、把密钥粘到 Slack、烤进 Docker 镜像、或塞到 Lambda 环境变量里。优先使用 IAM 角色(EC2 实例角色、Lambda 执行角色、EKS IRSA)— 它们会自动派发短期凭证。

IAM 基础

身份(谁)+ 策略(做什么)+ 资源(在哪里)。每次 API 调用都需依据策略授权。

核心概念

对象含义
User绑定到一个人(或服务)的长期身份。可以拥有访问密钥、控制台密码、MFA。
Role可被 扮演(assume) 的身份。返回短期凭证(STS)。被 EC2、Lambda、EKS、跨账号访问、联合登录使用。
PolicyJSON 文档。身份策略 附加到用户/角色;资源策略 附加到资源(如 S3 桶策略)。
Group共享策略的用户集合。不可嵌套。
STSSecurity Token Service。通过 AssumeRole、AssumeRoleWithWebIdentity 等签发短期凭证。

最小化的 S3 策略

单桶只读
{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Action": ["s3:GetObject", "s3:ListBucket"],
    "Resource": [
      "arn:aws:s3:::my-bucket",
      "arn:aws:s3:::my-bucket/*"
    ]
  }]
}
写入单个前缀
{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Action": ["s3:PutObject", "s3:DeleteObject"],
    "Resource": "arn:aws:s3:::my-bucket/uploads/*"
  }]
}
信任策略(EC2 扮演的角色)
附加在角色上;声明 谁可以扮演我。
{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Principal": {"Service": "ec2.amazonaws.com"},
    "Action": "sts:AssumeRole"
  }]
}

S3 入门

  
桶是某个区域内的扁平命名空间。键(Key)看起来像路径,但并不是文件夹。下面是你每周都会用到的命令。


  
桶(Bucket)

  

    

      
创建 / 列出 / 删除

      
# 创建 — 桶名是全局的
aws s3 mb s3://my-bucket --region us-east-1

aws s3 ls
aws s3 ls s3://my-bucket
aws s3 ls s3://my-bucket --recursive --human-readable --summarize

# 删除(必须为空)
aws s3 rb s3://my-bucket
# 或强制清空 + 删除:
aws s3 rb s3://my-bucket --force

    

    

      
cp — 单个文件

      
aws s3 cp ./report.csv s3://my-bucket/reports/

# 下载
aws s3 cp s3://my-bucket/reports/report.csv ./

# 跨桶
aws s3 cp s3://src/key s3://dst/key

# 带元数据 + 存储类
aws s3 cp ./big.tgz s3://my-bucket/ \
  --storage-class STANDARD_IA \
  --metadata "team=ml,owner=jane" \
  --content-type application/gzip

    

    

      
sync — 目录

      
只复制有差异的文件。默认不会删除本地缺失的远端文件 — 需要时传 --delete。

      
aws s3 sync ./build/ s3://my-bucket/v1/ \
  --exclude "*.tmp" \
  --exclude ".git/*" \
  --include "*.html" \
  --delete \
  --dryrun

    

    

      
rm

      
aws s3 rm s3://my-bucket/old.csv

# 整个前缀(递归)
aws s3 rm s3://my-bucket/tmp/ --recursive

# 带过滤器
aws s3 rm s3://my-bucket/ --recursive \
  --exclude "*" --include "*.log"

    

    

      
预签名 URL

      
任何人都可使用、无需 AWS 凭证、有时效的签名 URL。默认有效期 3600 秒。

      
aws s3 presign s3://my-bucket/report.csv \
  --expires-in 900

# 用于上传(PUT)— 需要 s3api:
aws s3 presign s3://my-bucket/upload.bin \
  --expires-in 600
# 注意:aws s3 presign 签发 GET URL。
# 上传场景请用 SDK 或 s3api 命令做 PUT 预签名。

S3 进阶

  
版本控制、加密、生命周期、网站托管。让一个桶从倾倒场升级为基础设施的特性。


  
版本控制

  

    

      
启用 / 列出 / 删除

      
aws s3api put-bucket-versioning \
  --bucket my-bucket \
  --versioning-configuration Status=Enabled

# 列出版本
aws s3api list-object-versions --bucket my-bucket --prefix reports/

# 删除某个特定版本
aws s3api delete-object \
  --bucket my-bucket --key reports/q1.csv \
  --version-id 3HL4kqtJlcpXroDTDmJ+rmSpXd3dIbrHY+MTRCxf3vjVBH40Nrjfkd

    

    

      
MFA 删除

      
删除某个对象版本或停用版本控制时强制要求 MFA 令牌。仅根账户。

      
aws s3api put-bucket-versioning \
  --bucket my-bucket \
  --versioning-configuration Status=Enabled,MFADelete=Enabled \
  --mfa "arn:aws:iam::123456789012:mfa/root 123456"

    

  


  
加密

  

    

      
默认桶加密

      
# SSE-S3 (AES256, AWS 托管)
aws s3api put-bucket-encryption \
  --bucket my-bucket \
  --server-side-encryption-configuration '{
    "Rules":[{"ApplyServerSideEncryptionByDefault":{
      "SSEAlgorithm":"AES256"}}]}'

# SSE-KMS(客户管理密钥)
aws s3api put-bucket-encryption \
  --bucket my-bucket \
  --server-side-encryption-configuration '{
    "Rules":[{"ApplyServerSideEncryptionByDefault":{
      "SSEAlgorithm":"aws:kms",
      "KMSMasterKeyID":"alias/my-key"}}]}'

    

    

      
SSE-S3 vs SSE-KMS

      
SSE-S3:零配置,密钥归 AWS 所有。免费。除非需要对密钥使用做审计/IAM 控制,否则用它。
SSE-KMS:逐密钥的审计日志、密钥轮换、跨账号授权。每 1 万次请求 0.03 美元 + KMS 密钥费用。

    

    

      
单对象加密

      
aws s3 cp file.txt s3://my-bucket/ \
  --sse aws:kms \
  --sse-kms-key-id alias/my-key

Hello World 工作流

  
三个端到端的菜谱,今天就能跑通。


  
① 在 S3 + CloudFront 上托管静态网站

  

    

      
1. 创建桶

      
BUCKET=my-site-$(uuidgen | tr A-Z a-z)
REGION=us-east-1

aws s3 mb s3://$BUCKET --region $REGION

# 保持禁止公开访问 — 由 CloudFront 来回源内容
aws s3api put-public-access-block \
  --bucket $BUCKET \
  --public-access-block-configuration \
   "BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true"

    

    

      
2. 同步网站

      
aws s3 sync ./site/ s3://$BUCKET/ \
  --delete \
  --cache-control "public,max-age=3600" \
  --exclude "*.html" \
  --exclude "service-worker.js"

# 给 HTML 设短缓存,部署后能快速看到变化
aws s3 sync ./site/ s3://$BUCKET/ \
  --exclude "*" --include "*.html" \
  --cache-control "public,max-age=60"

    

    

      
3. 前置 CloudFront

      
aws cloudfront create-distribution \
  --origin-domain-name $BUCKET.s3.$REGION.amazonaws.com
# 在控制台:开启 OAC,附加一个允许
# CloudFront SourceArn 的桶策略(参见"桶策略"卡片)。

# 部署后失效缓存
aws cloudfront create-invalidation \
  --distribution-id E1ABC... --paths "/*"

zip thumb.zip lambda_function.py

  --function-name thumb \
  --runtime python3.12 \
  --handler lambda_function.handler \
  --role arn:aws:iam::123:role/lambda-thumb \
  --zip-file fileb://thumb.zip

    

    

      
把 S3 接到 Lambda

      
aws lambda add-permission \
  --function-name thumb --statement-id s3 \
  --action lambda:InvokeFunction \
  --principal s3.amazonaws.com \
  --source-arn arn:aws:s3:::my-bucket

aws s3api put-bucket-notification-configuration \
  --bucket my-bucket \
  --notification-configuration '{
    "LambdaFunctionConfigurations":[{
      "LambdaFunctionArn":"arn:aws:lambda:us-east-1:123:function:thumb",
      "Events":["s3:ObjectCreated:*"],
      "Filter":{"Key":{"FilterRules":[
        {"Name":"prefix","Value":"uploads/"},
        {"Name":"suffix","Value":".jpg"}]}}
    }]}'

故障排查

  
当 CLI 调用不顺利时,先读这一页。


  
常见错误

  

    
症状可能原因与修复

    
      
An error occurred (AccessDenied)IAM 策略未授予对应操作。检查身份策略和桶/资源策略两边。先用 aws sts get-caller-identity 确认 哪个 身份。可使用 IAM Policy Simulator。

      
NoSuchBucket桶名拼错,或你处于错误的区域/账号。桶名是全局的;aws s3 ls 只列出 你账号 拥有的桶。

      
The bucket you are attempting to access must be addressed using the specified endpoint区域不匹配。传 --region us-west-2 或设置 AWS_REGION。

      
ExpiredToken / InvalidTokenSSO/STS 凭证已过期。运行 aws sso login --profile foo 或重新扮演角色。

      
Could not connect to the endpoint URL区域错(那个区域没 S3)、VPC 端点策略,或代理。试试 --debug。

      
RequestTimeTooSkewed本机时间漂移。同步时间(NTP)。

      
An error occurred (SlowDown) when calling …S3 在按前缀限流。把 key 分散到更多前缀;退避后重试。每前缀:3500 PUT/COPY/POST/DELETE 与 5500 GET 请求/秒。

      
InvalidAccessKeyId密钥错、被禁用,或密钥属于另一账号。重新核对 ~/.aws/credentials 与 $AWS_PROFILE。

      
EntityAlreadyExists 出现在创建桶时桶名是全局的。换一个名字。

    
  


  
调试工具

  

    

      
--debug

      
打印每个签名请求、请求头、重试次数和响应体。排查权限/区域问题非常好用。日志量很大。

      
aws s3 ls s3://my-bucket --debug 2>&1 | tee aws-debug.log

    

    

      
我是哪个身份?

      
aws sts get-caller-identity --output table

# 环境变量:
env | grep AWS_

# 配置:
aws configure list
aws configure list-profiles

    

    

      
演练(S3 sync/cp/rm)

      
aws s3 sync ./build/ s3://my-bucket/v1/ \
  --delete --dryrun
# 输出 (dryrun) upload: ... → s3://...
# 输出 (dryrun) delete: s3://...

    

    

      
输出格式

      
--output  json | yaml | yaml-stream | text | table

aws ec2 describe-instances \
  --query 'Reservations[].Instances[].InstanceId' \
  --output text

    

    

      
分页

      
# CLI 默认自动分页。可覆盖:
--max-items 50          # 总返回条数
--page-size 200         # 每次调用条数
--no-paginate           # 原始单次响应

    

    

      
端点覆盖(LocalStack、MinIO)

      
aws --endpoint-url http://localhost:4566 \
    s3 ls

# CLI v2:逐服务端点
[profile local]
endpoint_url = http://localhost:4566

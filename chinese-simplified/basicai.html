什么是张量？
张量就是多维数组 —— 标量、向量、矩阵或更高维。 神经网络里的一切都是张量：输入、权重、梯度、输出。 PyTorch 的 torch.Tensor 几乎等同于 NumPy 的 ndarray， 只是它能在 GPU 上跑，并且能记住自己是怎么算出来的（autograd）。
什么是模型？
一个带可学习参数的函数。输入流过一层层（矩阵乘法 + 非线性激活函数）后产生输出。我们把输出与真值比较， 计算梯度，再顺着梯度把权重往低损失的方向轻轻一推。
训练
给模型喂大量"输入 / 目标"对。每一对都做一次：前向 → 算损失 → 反向（求梯度）→ 优化器走一步（更新权重）。重复成千上万次。 损失曲线下降，就这样。这就是深度学习。
推理
训练完后，冻结权重，对新输入只跑前向。更快（无需记账梯度）且确定性强。 用 model.eval() 和 torch.no_grad()。

构建模型 — 你的第一个 nn.Module

每一个 PyTorch 模型都是 nn.Module 的子类。 在 __init__ 里定义层，在 forward 里写前向流程。整个 API 就这点事。

用于 10 类分类的两层 MLP（完整版）
import torch
import torch.nn as nn

class Net(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1  = nn.Linear(784, 128)
        self.relu = nn.ReLU()
        self.fc2  = nn.Linear(128, 10)

    def forward(self, x):
        x = self.fc1(x)
        x = self.relu(x)
        x = self.fc2(x)
        return x

model = Net()

损失与优化器

损失是一个数，告诉你"预测错得有多离谱" —— 越小越好。 优化器用梯度去推动权重，让损失下降。两种损失覆盖 90% 场景； 两种优化器覆盖 95% 场景。

损失
回归 nn.MSELoss —— 均方误差
分类 nn.CrossEntropyLoss —— softmax + log + nll 全部内置
二分类 nn.BCEWithLogitsLoss
优化器
朴素 optim.SGD(params, lr=0.01)
默认稳妥 optim.Adam(params, lr=1e-3)
现代 optim.AdamW —— Adam + 解耦权重衰减
把它们拼起来
import torch.nn as nn
import torch.optim as optim

model     = Net()
loss_fn   = nn.CrossEntropyLoss()
optimizer = optim.Adam(model.parameters(), lr=1e-3)

键入这五行

15 第 1 步 —— 前向。把输入送过模型：

16 第 2 步 —— 计算预测值与目标之间的损失：

17 第 3 步 —— 清空上一批梯度（否则会累加！）：

18 第 4 步 —— 反向传播，对每个参数算出新梯度：

19 第 5 步 —— 用梯度更新权重：

常见 bug：如果你忘了调用 optimizer.zero_grad()， 梯度会跨批次累加，模型会像"过去所有批次都被合在一起" 那样去训练。务必每一步都清零。

推理 — 使用训练好的模型

训练完后，你想对新输入运行模型，且不要算梯度（更快、更省内存）， 也不希望 dropout / batchnorm 出怪事。两个开关搞定。

推理模板
model.eval()                          # dropout 关闭，BN 用滑动统计
with torch.no_grad():                 # 不再追踪梯度
    logits = model(x_new)
    probs  = torch.softmax(logits, dim=-1)

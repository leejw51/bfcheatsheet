BERT 是什么

BERT 是一个仅编码器的 transformer 模型。和 GPT（仅解码器、自回归）不同，每个 token 在每一层都双向地关注其他每个 token。这种双向性正是它的关键。

  • 发布：2018 年 10 月，Devlin et al.（Google AI Language）
  • 架构：transformer 编码器块的堆叠（多头自注意力 + 前馈）
  • 预训练：在 BooksCorpus（8 亿词）+ 英文 Wikipedia（25 亿词）上自监督训练
  • 微调：加一个小型任务头，端到端在标注数据上训练
关键洞见：读过数十亿句子并学会填空的模型，其实对语言已知之甚多。在它顶上接一个微小的分类器，下游任务表现就能远超它的「身板」。
02 /

预训练目标

掩码语言建模（MLM）

随机遮盖 15% 的 WordPiece token。模型预测原始词。损失：仅在被遮盖位置上计算交叉熵。

这 15% 中：

  • 80% 替换为 [MASK]
  • 10% 替换为随机 token
  • 10% 保持不变

这 10/10 的小技巧防止模型偷懒，只在 [MASK] 位置出力 —— 而微调阶段并没有 mask。

为何要双向？单向语言模型若不泄漏未来 token 就无法以其为条件。MLM 让每个位置都能看到完整的双向上下文。

下一句预测（NSP）

给定句子 A 和 B，预测 B 是否真的接在语料中的 A 之后（50%），或来自其他位置的随机句（50%）。在 [CLS] 最终隐状态上的二分类。

输入：  [CLS] A₁ A₂ ... [SEP] B₁ B₂ ... [SEP]
标签：  IsNext / NotNext
暴论：RoBERTa（2019）证明 NSP 几乎没用并去掉了它。SBERT 用句对对比目标取而代之。现代 BERT 风格的模型常常完全跳过 NSP。

组合损失

BERT 的预训练损失是 MLM 与 NSP 损失之和：

$$\mathcal{L} = \mathcal{L}_{\text{MLM}} + \mathcal{L}_{\text{NSP}}$$

$$\mathcal{L}_{\text{MLM}} = -\sum_{i \in M} \log P(x_i \mid \tilde{x})$$

其中 $M$ 是被遮盖位置集合，$\tilde{x}$ 是被破坏后的输入。

输入格式

每个输入位置的向量是三种 embedding 之和：token + 段（segment）+ 位置。三者都是同一隐藏维度 $d$（如 768）的可学习查找表。

$$\mathbf{x}_i = \mathbf{E}_{\text{tok}}(t_i) + \mathbf{E}_{\text{seg}}(s_i) + \mathbf{E}_{\text{pos}}(i)$$

可视化

把鼠标悬停到格子上。每条彩色条代表一种 embedding。最底一行是它们的逐元素之和 —— 这就是进入第一个编码器块的输入。

（数值仅作示意 —— 真正的 embedding 是 768 维且可学习。）

特殊 token

Token作用
[CLS]第一个位置。其最终隐状态作为整段序列的池化表示，用于分类。
[SEP]分隔符。标记一个段的结束；在 NSP 与句对任务中，A、B 之间必须有它。

[PAD]批内对齐用的填充。被 attention mask 屏蔽。

一个编码器块里有什么

每个 L 块都是标准的「post-LN」transformer 编码器：

  1. 多头自注意力，每头 H/A 维
  2. 残差 + LayerNorm
  3. 逐位置前馈：linear → GELU → linear（隐藏维 4H）
  4. 残差 + LayerNorm

$$\text{Attn}(Q,K,V) = \text{softmax}\!\left(\tfrac{QK^\top}{\sqrt{d_k}}\right)V$$

自注意力是完整且双向的 —— 没有因果掩码。

WordPiece 分词

BERT 使用一个 30,522 个 token 的 WordPiece 词表。未知词被切成已知子片段，续片以 ## 前缀标记。

"playing"     → ["play", "##ing"]
"unaffable"   → ["un", "##aff", "##able"]
"transformer" → ["transform", "##er"]
"BERT"        → ["bert"]   (在 bert-base-uncased 中转小写)

青色 = 特殊 token， 黄色 = 整词， 粉色 = ##续片

微调范式

一个预训练 BERT 对应三种（以及更多）下游任务模式。每种情况都加一个小型任务头，在低学习率（通常 2e-5 到 5e-5）下端到端训练 2–4 个周期。

序列分类

情感、NLI、主题。使用最终的 [CLS] 隐状态。

[CLS] → Linear(H→C) → softmax

示例：SST-2、MNLI、QNLI

token 分类

NER、词性标注。把头作用到每个 token 的隐状态。

h_i → Linear(H→C) → softmax（逐 token）

示例：CoNLL-2003 NER

片段抽取（QA）

预测答案片段的起止位置。

h_i → 2 个 logit（start, end）

示例：SQuAD 1.1 / 2.0

双向 vs 自回归

BERT（仅编码器）

在「my dog is __」中预测 cute，使用完整上下文：

my dog is [MASK] today .

双侧都关注。无法自回归生成文本，但能产出强大的上下文表示。

GPT（仅解码器）

在「my dog is」之后预测下一个 token：

my dog is ? today .

因果掩码屏蔽未来 token。天然适合生成；在同等参数量下，理解任务上稍弱（历史上 —— 差距已大幅缩小）。

局限与后继者

BERT 的弱点

  • 注意力对序列长度二次复杂 → 最多 512 token
  • 无法生成 —— 仅编码器，无法采样
  • 预训练/微调不一致 —— 推理时永远没有 [MASK]
  • NSP 信号弱 —— 后续模型纷纷弃用
  • 句向量开箱即用很差；对 [CLS] 做平均池化也不行

后继者与变体

模型年份变化
RoBERTa2019去掉 NSP，更多数据，更长训练，动态掩码
ALBERT2019跨层参数共享；以「句序预测」替代 NSP
DistilBERT2019知识蒸馏，约 40% 更小，约 60% 更快，保留约 97%
ELECTRA2020替换 token 检测 —— 比 MLM 样本效率高得多
SBERT2019孪生网络，借助对比损失输出可用的句向量
DeBERTa2020解耦注意力（内容 vs 位置），增强的 mask 解码器
ModernBERT2024长上下文（8K）、Flash Attention，当前最强编码器

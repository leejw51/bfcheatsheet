bfgpt — 单文件迷你 GPT

bfgpt 是一个下午就能读完、笔记本就能训练的迷你 GPT。 单个 ~800 行的 Python 文件装下了全部内容——模型、数据准备、训练、采样——只需 两个依赖（torch + numpy）。 它是 Karpathy nanoGPT 的忠实精简移植，去掉了 GPT-2 下载路径。分词是字符级的，无需下载分词器——直接从你的文本构建。

self-contained
模型 + 数据准备 + 训练 + 采样全在一个 bfgpt.py 里。检查点甚至内嵌了自己的分词器，采样无需其他文件。
two dependencies
只需 torch 和 numpy。不用 transformers，不用 tiktoken，无需学习任何框架。
runs anywhere
--device auto 自动选择 cuda→mps→cpu。默认参数 CPU 就能跑。
character tokenizer
sorted(set(text)) 就是词表。tiny-shakespeare 有 65 个独特字符。
JSONL in
每行一条 {"text": "..."}。用 --text-field 指向其他字段。
~0.8M params
4 层 / 4 头 / 128 宽，block_size=64。几分钟就能跑出莎士比亚风格的文字。

分词器——你的文本即词表

无需下载分词器，没有 BPE 合并操作，没有 50,257 词元的表。bfgpt 对语料库中的 唯一字符排序去重并编号。该映射（stoi / itos） 就是完整的分词器，存储在 meta.pkl 中。 在下方输入文字，观察模型看到的完整 encode → ids → decode 往返过程。

字符分词器沙盒

vocab_size 0 characters 0 tokens 0 dtype uint16
词表 — id:char（␣ 空格 · ⏎ 换行）
encode → token ids
decode(ids) → 往返

  


  
CharTokenizer — 完整类

  

    
CharTokenizerbfgpt.py:247

    
class CharTokenizer:
    def __init__(self, stoi, itos):
        self.stoi = stoi
        self.itos = itos
        self.vocab_size = len(stoi)

    @classmethod
    def from_text(cls, text):
        chars = sorted(set(text))            # the vocabulary
        stoi  = {ch: i for i, ch in enumerate(chars)}
        itos  = {i: ch for i, ch in enumerate(chars)}
        return cls(stoi, itos)

    def encode(self, s):
        return [self.stoi[c] for c in s]   # str -> list[int]

    def decode(self, ids):
        return ''.join(self.itos[int(i)] for i in ids)

  


  

    为什么这样好：数据的字母表决定词表大小，id 适合放在 2 字节的
    uint16 中，嵌入表很小。代价是：序列更长（每个字符一个 token），
    所以上下文窗口 block_size 计的是字符数，不是词数。

六个让它工作的技巧

  

    
权重共享
wte.weight = lm_head.weight — 输入 embedding 和输出投影共享一个矩阵。

    
pre-norm
x + f(LN(x)) — 在每个子层之前归一化，而非之后。

    
因果掩码
位置 t 只能注意到 ≤ t 的位置；未来被掩为 −∞。

    
flash attention
当 PyTorch ≥ 2.0 提供时使用融合的 scaled_dot_product_attention。

    
缩放初始化
残差投影用 std = 0.02 / √(2·n_layer) 初始化。

    
选择性衰减
权重衰减只作用于 2D 张量；bias 和 LayerNorm 跳过。

CausalSelfAttention — Q、K、V 一次矩阵乘法

  

    
CausalSelfAttention.forwardbfgpt.py:80

    
def forward(self, x):
    B, T, C = x.size()
    q, k, v = self.c_attn(x).split(self.n_embd, dim=2)   # one matmul, split 3 ways
    k = k.view(B, T, self.n_head, C // self.n_head).transpose(1, 2)  # (B, nh, T, hs)
    q = q.view(B, T, self.n_head, C // self.n_head).transpose(1, 2)
    v = v.view(B, T, self.n_head, C // self.n_head).transpose(1, 2)

    if self.flash:
        y = F.scaled_dot_product_attention(q, k, v, is_causal=True)
    else:
        att = (q @ k.transpose(-2, -1)) * (1.0 / math.sqrt(k.size(-1)))
        att = att.masked_fill(self.bias[:, :, :T, :T] == 0, float('-inf'))
        att = F.softmax(att, dim=-1)
        y = att @ v
    y = y.transpose(1, 2).contiguous().view(B, T, C)
    return self.resid_dropout(self.c_proj(y))

  

  
核心思路：因果掩码（或 is_causal=True）是使其成为解码器的关键——每个位置只用过去的信息预测下一个。

GPT.forward — 训练分支 vs 推理分支

  

    
GPT.forwardbfgpt.py:183

    
def forward(self, idx, targets=None):
    b, t = idx.size()
    pos = torch.arange(0, t, device=idx.device)

    tok_emb = self.transformer.wte(idx)   # (b, t, n_embd)
    pos_emb = self.transformer.wpe(pos)   # (t, n_embd)
    x = self.transformer.drop(tok_emb + pos_emb)
    for block in self.transformer.h:
        x = block(x)
    x = self.transformer.ln_f(x)

    if targets is not None:                 # --- 训练 ---
        logits = self.lm_head(x)
        loss = F.cross_entropy(logits.view(-1, logits.size(-1)),
                               targets.view(-1), ignore_index=-1)
    else:                                  # --- 推理 ---
        logits = self.lm_head(x[:, [-1], :])  # 只取最后位置
        loss = None
    return logits, loss

  

  
同一个 forward 的两面：有 targets 时，对每个位置计算损失（并行训练）；没有时，只计算最后位置的 logits——这正是采样所需的。

学习率调度——预热，然后余弦衰减

  

    

      
get_lrbfgpt.py:516

      
def get_lr(it):
    if it < warmup_iters:           # 线性预热
        return lr * (it+1) / (warmup_iters+1)
    if it > lr_decay_iters:         # 下限
        return min_lr
    ratio = (it - warmup_iters) / (lr_decay_iters - warmup_iters)
    coeff = 0.5 * (1.0 + math.cos(math.pi * ratio))  # 余弦
    return min_lr + coeff * (lr - min_lr)

    

    

      
线性预热 100 次迭代至峰值
      learning_rate = 1e-3，然后余弦衰减至
      min_lr = 1e-4，共 2,000 次迭代。

      
$\eta_t = \eta_{\min} + \tfrac{1}{2}\,(1+\cos\pi r)\,(\eta_{\max}-\eta_{\min})$

      
每步还需：前向 → 反向 → 梯度裁剪到 1.0 → AdamW 步进 → zero_grad。下一个 batch 在反向传播期间预取，以重叠 H2D 拷贝。

    

  







  
生成——逐个 token

  

    采样是自回归的：预测下一个字符，追加它，再把整个序列送回去，不断循环。
    当序列超过 block_size 时，从左侧裁剪，让上下文窗口向前滑动。
  


  

    
GPT.generatebfgpt.py:226

    
@torch.no_grad()
def generate(self, idx, max_new_tokens, temperature=1.0, top_k=None):
    for _ in range(max_new_tokens):
        idx_cond = idx if idx.size(1) <= self.config.block_size \
                   else idx[:, -self.config.block_size:]   # 裁剪到上下文
        logits, _ = self(idx_cond)
        logits = logits[:, -1, :] / temperature           # 1. 重塑分布
        if top_k is not None:
            v, _ = torch.topk(logits, min(top_k, logits.size(-1)))
            logits[logits < v[:, [-1]]] = -float('Inf')   # 2. 保留 top-k
        probs = F.softmax(logits, dim=-1)               # 3. 转为概率
        idx_next = torch.multinomial(probs, num_samples=1)  # 4. 采样一个
        idx = torch.cat((idx, idx_next), dim=1)           # 5. 追加并重复
    return idx

链式法则 — 梯度如何反向流动

反向传播就是把一条规则一遍遍地应用。要得到损失对任意东西的梯度,只需沿着从损失返回到它 的路径走一遍,并把沿途的局部导数相乘。那个乘积就是链式法则。本页其余 内容都是从六个角度看待这同一个想法。

一句话
对于这个微型网络 $L \leftarrow z \leftarrow w$,梯度就是若干局部斜率的乘积:
$\dfrac{\partial L}{\partial w} = \dfrac{\partial L}{\partial z}\cdot\dfrac{\partial z}{\partial w}$
中间那个因子 $\partial L/\partial z$ 就是我们称为 dz 的上游梯度。
dz 是枢纽
一个节点的两个输入共享同一个上游梯度 dz。对于 $z = x\cdot w$:
$\textcolor{#4ade80}{dw} = dz\cdot x \qquad \textcolor{#38bdf8}{dx} = dz\cdot w$
dz 是共用的;只有局部因子不同。
为什么会分流
乘法有两个输入,所以梯度会分成两条 支路 —— 而且每条支路拿到的是另一个输入。$x$ 支路带着 $w$, $w$ 支路带着 $x$。一个节点为每个输入送出一个梯度。
矩阵也是一样
对于 $Z = X\!\mathbin{@}\!W$,除了形状之外什么都没变 ——“另一个输入”变成了它的 转置:
$\textcolor{#4ade80}{dW} = X^{\!\top}\!\mathbin{@}\,dZ \qquad \textcolor{#38bdf8}{dX} = dZ\mathbin{@}\,W^{\!\top}$
只存在两种门。反向传播永远只做加法和乘法 —— 一个乘法把梯度按另一个输入缩放,一个加法只是原封不动地把它传过去。 标签 4 就是那个加法门;每一层(矩阵乘法、偏置、残差)都是由这两者搭建出来的。

为什么链式法则是一个乘积

$w$ 从不直接接触损失 —— 它只能通过 $z$ 起作用: $w \to z \to L$。所以如果你稍微推动 $w$ 一下,这个抖动会沿着链条传下去。每一段链路 都有一个“放大系数”,而总效果就是这些系数相乘。 那个乘积正好就是 $\partial L/\partial w$。

把它当成分数来读

这不是证明,但是一个无敌的记忆口诀:把链式法则写成分数,中间那一项就约掉了。

∂Loss / ∂w  =  ∂Loss / ∂z  ×  ∂z / ∂w
中间的 ∂z 同时出现在上下两处,会约掉, 剩下 ∂Loss/∂w。用话来说就是:“每单位 w 引起的损失变化 =(每单位 z 引起的损失变化)×(每单位 w 引起的 z 变化)。” 你是穿过桥梁 $z$ 的。

分流 —— 进来一个梯度,出去两个梯度

一个乘法节点 $z = x\cdot w$ 有两个输入,所以单个上游梯度 dz 会被复制到两条支路。妙处在于: 每条支路都乘以另一个输入。$x$ 支路带着 $w$; $w$ 支路带着 $x$。

为什么每条支路都需要自己的梯度

dw → 更新权重
$w$ 是一个可学习参数。它的梯度驱动更新 w ← w − lr·dw。没有 dw,这一层就无法学习。
dx → 喂给下层
$x$ 是前一层的输出。那一层也需要梯度,所以我们把 dx 往下游传。它会成为下一个节点的 dz —— 链条就这样 一路延续到底。
这个模式:在一个乘法节点上,把 dz 复制给每个 输入,然后把每个副本乘以其余输入的乘积。$x$ 越大 ⇒ dw 越大, 反之亦然。这种对称性就是整个诀窍。

加法门 —— 梯度只是被复制

乘法门会把每条支路按另一个输入缩放。一个加法门 $z = x + w$ 更简单:两个局部导数都是 1,所以上游梯度 dz 原封不动地传过去 —— 同一个值被复制给每个输入。没有缩放,也不需要从前向传播缓存任何东西。

∂z/∂x = 1,   ∂z/∂w = 1   ⇒   dx = dz·1 = dz,   dw = dz·1 = dz

加法 vs 乘法 —— 仅有的两种门

加法 → 复制
$z = x + w$。局部斜率都是 1,所以 dx = dw = dz。梯度被原封不动地广播 给每个输入 —— 没有缩放,没有转置,也不需要从前向传播记住任何东西。
乘法 → 缩放
$z = x \cdot w$。每条支路都被另一个输入缩放: dx = dz·w, dw = dz·x。输出取决于前向传播中 缓存的那些值。
这就是全部的工具箱。反向传播永远只做加法和 乘法。每一层都是由这两种门搭建出来的:一个线性层 Z = X@W + b 是一个乘法(矩阵乘法)再加一个加法(偏置); 一个残差连接 y = x + f(x) 则是纯粹的加法。

为什么残差连接如此好训练

一条跳跃连接 y = x + f(x) 就是一个加法门,所以在反向传播时梯度 会沿两条支路被复制下去:一份完整的 dz 副本进入 f,另一份完全相同的完整副本直接落在 x 上。第二份副本 就是一条梯度高速公路 —— 即便经过 f 的梯度已经 缩小到接近零,它也能毫不衰减地到达深层。

因为 x 在 y = x + f(x) 中出现了两次(一次是原样,一次在 f 内部),它的两份梯度副本会相加: dx = dz + (dz routed back through f)。第一项就是 高速公路 —— 一份干净、未被缩放的上游梯度副本,无论 f 做什么它都能存活下来。 这正是深层 Transformer 始终保持可训练的全部原因。

矩阵 —— 转置只不过是在对齐形状

一个线性层是 $Z = X\!\mathbin{@}\!W$。反向时是和标量门相同的两条支路 —— 唯一的新东西是“乘以另一个输入”变成了“乘以另一个矩阵的转置”, 这样形状才能对得上。

dW = Xᵀ @ dZ
(E,L) = (E, N) @ (N, L)
权重梯度。逐元素 dW[e,l] = Σ_tokens x_e · dz_l —— 输入与 输出梯度的外积,在每个 token 上求和(一个共享的 $W$)。
dX = dZ @ Wᵀ
(N,E) = (N, L) @ (L, E)
输入梯度。逐元素 dx_e = Σ_l dz_l · W[e,l] —— 把输出梯度通过权重推回去。 按每个 token 保留,以便流向下层。

为什么要转置?dW 必须和 $W$ 同形(3×2), dX 必须和 $X$ 同形(2×3)。把 $X$ 或 $W$ 转置后塞进去,形状就会 自动咬合在一起:Xᵀ(3×2)@dZ(2×2)=(3×2), dZ(2×2)@Wᵀ(2×3)=(2×3)。转置不是新的数学 —— 它只是在记账。

前向与反向是一对转置

Z = X @ W   ⇄   dW = Xᵀ @ dZ   ·   dX = dZ @ Wᵀ
小心:先前向再反向(例如 (dZ @ Wᵀ) @ W)得到的是一个 具有 Z 的形状的东西 —— 在形状上是一次往返 —— 但不是 相同的数值,因为 Wᵀ·W 并不是单位矩阵。转置这一对匹配的是形状,不是 数字。

dz 从哪来:softmax + 交叉熵 = P − Y

本页上的每一条链都从一个上游梯度 dz 出发。 对于一个分类器,那个最初的梯度 —— 损失对 logits 的导数 —— 会坍缩成深度学习里 最干净的表达式:P − Y。

一个 token,L = 5 个类别

P − Y 背后的直觉

目标类别(Y = 1)
梯度 = p − 1 → 永远是负的。梯度下降是逆着梯度迈步, 所以这个 logit 会上升(概率 ↑)。方向正确。
其他类别(Y = 0)
梯度 = p − 0 = p → 永远是正的。这个 logit 会下降 (概率 ↓)。同样正确。
这些梯度之和为零:Σ(pᵢ − yᵢ) = Σpᵢ − Σyᵢ = 1 − 1 = 0。从目标那里 拿走的概率会原样分发给其他类别 —— 每次更新都保持平衡。

逐行推导

只需要两个事实:$Y$ 是 one-hot,以及 softmax 的导数。

# 定义
pᵢ = e^zᵢ / Σₖ e^zₖ ← softmax
L = − Σᵢ yᵢ · log(pᵢ) ← 交叉熵
 
# softmax 导数(关键引理)
∂pᵢ/∂zₖ = pᵢ·(δᵢₖ − pₖ) i=k 时 δᵢₖ = 1,否则为 0
 
# 对 ∂L/∂zₖ 用链式法则
∂L/∂zₖ = − Σᵢ yᵢ · (1/pᵢ) · ∂pᵢ/∂zₖ
      = − Σᵢ yᵢ · (1/pᵢ) · pᵢ·(δᵢₖ − pₖ)
      = − Σᵢ yᵢ · (δᵢₖ − pₖ)
      = − yₖ + pₖ · Σᵢ yᵢ Σyᵢ = 1(one-hot)
∂L/∂zₖ = pₖ − yₖ ∎ 写成向量:P − Y
为什么这么漂亮:softmax 的 1/pᵢ 和交叉熵的 pᵢ 正好相消。所有的 exp / log 都消失了,只剩下 “预测 − 目标”。那个干净的 dz 就是标签 2–5 里每一条 链的起点。

CLIP — 让图片和文字进同一间屋

CLIP(Contrastive Language–Image Pre-training,OpenAI 2021)开启了 多模态的淘金热。它训练一个图像编码器和一个文本编码器,把匹配的图像-描述对放到一个共享的 embedding 空间中的同一个点上 —— 不相关的对则相距甚远。训练完成后,只要把类别名写成句子, 就能做零样本分类。

两个编码器,一个空间
图像编码器(ViT-B/32 或 ResNet-50)和文本编码器(Transformer)各自输出一个向量。 每一侧都有一个小小的投影头,把两者映射到同一个 d=512 的 embedding 空间。然后我们就度量 余弦相似度。
对比目标
对一个 N 大小的图像-文本对 batch,构建 N×N 的相似度矩阵。对角线上的格子是真匹配 (正样本);其他都是负样本。把对角线拉高,把非对角线压下去 —— 在行和列上分别做对称 交叉熵。
为什么重要
训练数据:从网上抓取的 4 亿对图像-文本。没有人工标注 —— 描述就是标签。结果是:一个 模型能识别它从未被显式教过的东西,只要你能把它和一句话匹配上。
它驱动了什么
DALL·E 2 的 CLIP 引导、Stable Diffusion 的文本编码器(CLIP-L/14)、 零样本分类器、图像搜索、内容审核、检索、image-to-image、 text-to-image 评分、OpenCLIP、SigLIP、EVA-CLIP…… 全是后裔。

# 伪代码 (Radford et al., 2021)
I_f = image_encoder(I)            # [N, d_i]
T_f = text_encoder(T)             # [N, d_t]

I_e = l2_normalize(W_i @ I_f)     # [N, d]  投影 + 归一化
T_e = l2_normalize(W_t @ T_f)     # [N, d]

logits = (I_e @ T_e.T) * exp(t)   # [N, N] 用温度缩放
labels = arange(N)                # 对角线就是目标
loss   = (CE(logits, labels) + CE(logits.T, labels)) / 2

架构 — 两座塔,一间屋

CLIP 是双编码器。两座独立的塔各自把输入变成定长向量,一个小小的线性投影把 它们对齐到相同维度,L2 归一化后,点积 = 余弦相似度。

对比损失 — 把对角线拉上去

在一个 N 大小的图像-描述对 batch 中,构建 N×N 相似度矩阵。对角线项是仅有的正确匹配; 其他都是 批内负样本。损失是对称交叉熵:行为每张图选对的描述,列为每个描述 选对的图。

L = ½ · ( CE(logits, labels) + CE(logitsᵀ, labels) ), labels = [0, 1, 2, …, N−1]

温度 — softmax 有多锐?

原始相似度 I_e @ T_e.T 落在 [-1, 1] 区间 —— 对一个有用的 softmax 来说太平了。 CLIP 把它乘以一个可学习的 scale exp(t),其中 t 是单个标量参数(初始化为 log(1/0.07) ≈ 2.66)。

P(text=j | image=i) = softmaxj( τ · cos(Ii, Tj) ), 其中 τ = exp(t)

零样本 — 不训练就能分类

CLIP 的超能力:把任何分类任务变成文本匹配问题。把每个类别名套进句子模板 ("a photo of a {class}"),一次性编码全部,然后对任何图像:编码它, 与类 embedding 做点积,softmax → 完成。无需微调,无需标签。

代码练习 — 用真实 CLIP

三套真实世界的栈。open_clip_torch 是事实标准 (模型比原版发布更好)。transformers 提供 HuggingFace 的 CLIPModel。两者包装的是同样的数学。

open_clip — 加载与 embed

完整参考
import torch, open_clip
from PIL import Image

model, _, preprocess = open_clip.create_model_and_transforms(
    "ViT-B-32", pretrained="laion2b_s34b_b79k")
tokenizer = open_clip.get_tokenizer("ViT-B-32")
model.eval()

img    = preprocess(Image.open("cat.jpg")).unsqueeze(0)
labels = ["a photo of a cat", "a photo of a dog"]
text   = tokenizer(labels)

with torch.no_grad():
    img_f = model.encode_image(img)
    txt_f = model.encode_text(text)
    img_f /= img_f.norm(dim=-1, keepdim=True)
    txt_f /= txt_f.norm(dim=-1, keepdim=True)
    probs = (100.0 * img_f @ txt_f.T).softmax(dim=-1)

print("probs", probs)   # [[0.98, 0.02]]

补全 CLIP 训练步

11 个空格。全绿后点 Train。 

import torch
import torch.nn.functional as F

# --- 两个编码器前向 -------------------------------------------
image_features = model.(images)        # [N, d]
text_features  = model.(texts)          # [N, d]

# --- L2 归一化,使点积成为余弦相似度 -------------------------
image_features = image_features / image_features.(dim=-1, keepdim=True)
text_features  = text_features  / text_features.(dim=-1, keepdim=True)

# --- 构建 N × N 相似度 logits --------------------------------
logit_scale = model.logit_scale.()                 # τ = exp(t)
logits_per_image = logit_scale * image_features @ text_features.
logits_per_text  = logits_per_image.

# --- 对角目标的对称交叉熵 ------------------------------------
N      = images.shape[0]
labels = torch.(N, device=images.device)
loss_i = F.cross_entropy(logits_per_image, labels)
loss_t = F.cross_entropy(logits_per_text,  labels)
loss   = (loss_i + ) / 2

# --- 标准 PyTorch 训练步 -------------------------------------
optimizer.zero_grad()
loss.()
optimizer.()

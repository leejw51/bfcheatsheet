ComfyUI — 基于节点的 Stable Diffusion

ComfyUI 是 Stable Diffusion 的节点图编辑器。它没有像 Automatic1111 那样固定的滑块界面, 而是让你把一个个只做一件事的方框连接起来:加载 checkpoint、编码提示词、 去噪潜变量、解码成图像。这张图就是程序。读完这一页,你会亲手连出 一条真正的 txt2img 工作流,调好 KSampler,并通过 HTTP API 调用它。

为什么用节点?
扩散过程的每一步(加载模型、编码提示词、采样、解码)都是一个 节点。连线在节点间传递带类型的数据。整条流水线一目了然 —— 你想怎么重新组合都行。
可复现
ComfyUI 把整张工作流的 JSON 嵌入到每张输出 PNG 的 PNG 元数据 里。 把图拖回画布 —— 工作流就还原了。 同样的种子 → 同样的像素。
可替换
想要 ControlNet、IP-Adapter、AnimateDiff、LoRA 堆叠?加个节点,接根线就行。 基础图保持不变。自定义节点以文件夹的形式放在 custom_nodes/ 下。
vs Automatic1111
A1111 像一个向导:第一次跑很简单,扩展起来很难。ComfyUI 像管道: 前期有学习曲线,之后能无限延伸。高级用户、批量任务,以及任何不平凡的流水线都活在这里。

节点剖析

每个 ComfyUI 节点都长得一样:一个带标题的方框,左边是输入插槽, 右边是输出插槽,身体里是 widget (数字、文本、下拉框)。连线插到插槽里;widget 的值存在节点本身上。

解剖一个 KSampler 节点

输入插槽
在左边。每个插槽都有一个类型 — 只有匹配的连线才能插进来。 试着把 CLIP 线插到 MODEL 插槽?ComfyUI 直接拒绝。
输出插槽
在右边。一个输出可以接到很多输入(一对多)。比如 Load Checkpoint 的同一个 MODEL 输出常常同时接到 KSampler 和 一个 LoRA loader。
widgets
节点的身体部分。数字、滑块、下拉框、文本框。它们把常量值存在节点上 — 不需要拉线。右键任何 widget 都能把它转成输入插槽,这样就能由别的节点驱动。
小提示:双击空白画布会打开节点搜索面板。 在插槽上右键可以转为 widget 或转为输入。

参数参考

widget作用常用默认值
seed初始噪声的 RNG 种子。同种子 + 同图 → 同一张图。任何整数
steps去噪迭代次数。越多越干净但越慢;超过 30 收益递减。20–30
cfg无分类器引导强度。越大越贴近提示词,但容易"烤焦"。5–8
sampler_name数值求解器。euler、dpmpp_2m、uni_pc 是常用主力。dpmpp_2m
schedulersigma 的噪声调度。karras 在小步数下大家最爱。karras
denoise强度:1.0 = 从纯噪声生成(txt2img)。<1.0 = 保留输入(img2img)。1.0 / 0.5

自定义节点 — 扩展工作流

原版 ComfyUI 自带大约 50 个节点。社区有上千个。它们以文件夹的形式安装在 custom_nodes/ 下,服务器启动时 会注册新的节点类。

安装路径
# 1. 安装 ComfyUI-Manager(节点的"包管理器")
cd ComfyUI/custom_nodes
git clone https://github.com/ltdrdata/ComfyUI-Manager

# 2. 重启服务。之后用 Manager 的 UI 安装其他任何东西。
python main.py

必装包

ComfyUI-Manager
包管理器。在 UI 里安装 / 更新 / 禁用其他自定义节点。永远第一个装。
ComfyUI ControlNet Aux
预处理器(Canny、Depth、OpenPose、Lineart)+ ControlNetApply 节点。用参考图控制构图。
ComfyUI_IPAdapter_plus
"图像转 conditioning"。把参考照片当提示词用 — 风格迁移、保留身份。
ComfyUI-AnimateDiff-Evolved
插入一个 motion 模块 → KSampler 就开始产视频。同一张图,batch 维度 = 帧数。
Impact Pack
检测 / 分割 / 细化组合。FaceDetailer 是著名的"一节点修脸"。
rgthree-comfy
体验改进:静音组、快速 bypass、种子管理、给所有节点加进度条。

API 模式 — 用代码驱动 ComfyUI

每条工作流都能保存为 API JSON(齿轮 → "Save (API Format)"), 再通过 HTTP 提交。这样 UI 就成了可选 — 你可以从脚本、后端、Discord 机器人等等 任何地方运行这张图。

提交一个任务
import json, requests, uuid

# 加载通过 齿轮 -> Save (API Format) 导出的工作流 JSON
with open("workflow_api.json") as f:
    workflow = json.load(f)

# 给本次运行打补丁
workflow["6"]["inputs"]["text"] = "a corgi in a spacesuit"
workflow["3"]["inputs"]["seed"] = 12345

payload = {"prompt": workflow, "client_id": str(uuid.uuid4())}
r = requests.post("http://localhost:8188/prompt", json=payload)
prompt_id = r.json()["prompt_id"]
print("queued:", prompt_id)

常用接口

方法路径作用
POST/prompt把一条工作流排队执行;返回 prompt_id。
GET/history/<prompt_id>完成后取输出(图像文件名、节点结果)。
GET/queue列出运行中 + 等待中的任务。
GET/view下载一张输出图:?filename=...&type=output。
WS/ws?clientId=...WebSocket,实时进度 + 节点级预览。
POST/upload/image上传图像(用作 img2img / inpaint 输入)。

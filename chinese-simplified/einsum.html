PyTorch · 一种记号统御所有张量运算

所有缩并都写进一个字符串

torch.einsum 让你用一个下标字符串（比如 'ik,kj->ij'）写出矩阵乘法、批量矩阵乘法、转置、迹、点积、 外积和注意力分数。再也不用记到底该用 @、bmm、 matmul，还是一堆 transpose + sum 的纠缠——你只需 给各个轴命名，再说清楚输出保留哪些。本页讲解规则，提供配方画廊，还有一个 会在小张量上真正运行 einsum 的实时演练场。

01 一句话的核心思想

它是什么

给各个轴命名

给每个输入的每个轴分配一个字母。'bik,bkj->bij' 表示「操作数 A 的下标是 b,i,k；操作数 B 是 b,k,j；结果保留 b,i,j」。

唯一的规则

缺失 = 求和

任何出现在左侧、却没有出现在输出中的字母都会被求和消去。 这里 bij 里没有 k，于是会沿 k 求和——这就是 矩阵乘法。

$$\texttt{'ik,kj->ij'} \quad\Longleftrightarrow\quad C_{ij} = \sum_{k} A_{ik}\, B_{kj}$$
把它当成一句话来读。这些下标就是爱因斯坦求和公式： 左侧列出每个操作数的下标，-> 分隔出输出，而对重复但被丢弃的下标 的求和是隐式的。einsum 只是把这段数学变成了一次函数调用。

01·b 为什么大家爱用它

可读

自带文档

'bhqd,bhkd->bhqk' 一眼就能看出注意力分数在做什么——轴都明明白白写在那里。 而一堆 permute+matmul 把它藏了起来。

通用

一个 API，多种运算

转置、求和、迹、点积、外积、矩阵乘法、批量乘法以及多张量缩并，统统归结为 同一次调用。

更少 bug

没有隐式广播

你把每个轴都写明白了，所以形状不匹配会立刻报错——不会有意料之外的广播 偷偷做出你没想要的事。

三条规则就是整门语言

如何读懂一个下标

-> 之前的字符串描述输入；之后的字符串描述输出。三条 规则就能涵盖一切。

①

相同字母 = 对齐

一个字母在多个操作数间重复使用，意味着那些轴会对齐，并沿该维度逐元素相乘。 在 'ik,kj' 中，A 的 k 与 B 的 k 配对。

②

丢弃字母 = 求和

任何没出现在输出中的输入字母都会被求和。'ik,kj->ij' 丢掉了 k，于是沿 k 求和——这就是一次缩并。

③

保留字母 = 轴

出现在输出中的字母会按你书写的顺序成为结果的一个轴。重排输出字母即可转置：'ij->ji'。

02·a 自由下标 vs. 求和下标

本页通用的两种角色、两种颜色：

自由（free） — 出现在输出中，成为结果的一个轴 求和（summed） — 只出现在左侧，会被缩并消去
$$\underbrace{C_{\textcolor{#fbbf24}{ij}}}_{\text{free}} = \sum_{\textcolor{#818cf8}{k}} A_{\textcolor{#fbbf24}{i}\textcolor{#818cf8}{k}}\, B_{\textcolor{#818cf8}{k}\textcolor{#fbbf24}{j}}$$

02·b 隐式（无箭头）形式

当你省略 -> 时

如果你省略 ->output，einsum 会采用一条隐式规则：每个在左侧 恰好出现一次的下标都会成为自由输出轴，并按字母顺序排序。重复出现的下标会被求和。

  • 'ij'  →  'ij->ij'   恒等（每个字母各出现一次）。
  • 'ii'  →  'ii->'   迹（i 重复，所以被求和）。
  • 'ik,kj'  →  'ik,kj->ij'   矩阵乘法（k 重复 → 被求和；i,j 按字母排序）。
请写显式形式。隐式形式很方便，但按字母排序这条规则常让人意外。 写出 -> 输出能让意图毫不含糊，这也是大多数代码库的做法。

02·c 函数签名与坑

torch.einsum · 调用方式 
import torch

A = torch.randn(2, 3)        # [i=2, k=3]
B = torch.randn(3, 4)        # [k=3, j=4]

C = torch.einsum('ik,kj->ij', A, B)   # [i=2, j=4]  == A @ B

# 操作数按顺序跟在字符串后面传入
# 每个操作数的字母数必须等于它的维数
坑

字母数 ≠ 维数

如果一个操作数有 3 个维度，它的下标就需要正好 3 个字母。给 3 维张量写 'ij' 会报错。

坑

相同字母，相同尺寸

一个重复出现的字母在它出现的每个地方都必须有相同的长度——两个操作数里的 k 必须一致，否则无法对齐。

省略号 ...：对于你不想命名的前导「批量」维度，可以用 '...ik,...kj->...ij'。... 会在任意数量的前导轴上广播——非常适合那些 既要支持批量输入又要支持非批量输入的代码。

缩放点积注意力 · 逐行展开 
# Q,K,V: [B, H, T, d]   (批量, 头数, token, 头维)
scores = torch.einsum('bhqd,bhkd->bhqk', Q, K) / math.sqrt(d)
attn   = scores.softmax(dim=-1)                    # [B,H,q,k]
out    = torch.einsum('bhqk,bhkd->bhqd', attn, V)  # [B,H,q,d]
读一读 scores 这行：q 和 k 是两个 token 轴； d（头维）被共享且被丢弃，所以它被求和——这就是每个 query 与每个 key 的点积。b,h 作为批量轴一路跟随。

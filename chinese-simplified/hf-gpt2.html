HuggingFace GPT-2 — 训练它、分词它、用它生成

GPT-2 是一个 decoder-only transformer,被训练来预测下一个 token。HuggingFace 的 transformers 库三行 Python 就能把它包起来。 本页带你走遍整套技术栈 — pipelines、tokenizer、model、采样旋钮和训练循环 — 并要求你亲手敲关键的几行,把它们焊进肌肉记忆。

四个库
transformers 模型、分词器、pipeline、Trainer
datasets 数据流式 / map / 缓存
tokenizers 高速 Rust BPE / WordPiece
accelerate 多卡、混合精度,免样板代码
什么是 GPT-2?
一个 decoder-only transformer。12 层(124M)、24(345M)、36(774M)、48(1.5B)。 训练数据是 WebText,目标只有一个:预测下一个 token。 因果掩码不让任何位置看到未来。 分词器是字节级 BPE,词表大小 50,257。
两种用法
简单 pipeline("text-generation", model="gpt2")
底层 AutoTokenizer + AutoModelForCausalLM + .generate()
pipeline 用来 demo 很爽。真要做事就降到 model + tokenizer 这一对, 这样能控制采样、批处理和训练。
decoder-only 结构
input ids → token + position embed
[batch, seq] → [batch, seq, 768]
↓
12 × decoder block
带掩码的 self-attn + MLP + LayerNorm + 残差
↓
最终 LayerNorm
稳定隐藏状态
↓
lm_head(权重共享)
[..., 768] → [..., 50257] 词表 logits

快速开始 — 三行代码生成文本

写法 1 — pipelines

最简单的生成
from transformers import pipeline

gen = pipeline("text-generation", model="gpt2")
out = gen("The future of AI is", max_new_tokens=30, do_sample=True)

写法 2 — tokenizer + model

显式两行版
from transformers import AutoTokenizer, AutoModelForCausalLM

tokenizer = AutoTokenizer.from_pretrained("gpt2")
model     = AutoModelForCausalLM.from_pretrained("gpt2")

inputs  = tokenizer("The future of AI is", return_tensors="pt")

分词 — 字符串和张量之间的 BPE 桥梁

GPT-2 看不到字符或单词 — 它看到的是 token IDs。它的字节级 BPE 把常见子词合并成单个单元(比如 "ing"), 把生僻词留作多个 token,并用一个前导空格标记(显示为 Ġ)以保证可逆往返。

标准的生成调用
inputs  = tokenizer(prompt, return_tensors="pt")
outputs = model.generate(
    **inputs,
    max_new_tokens=50,

    top_p=0.9,                 # nucleus:概率累计达 0.9 的最小集合

)

采样策略 — 怎样选下一个 token

每一步,模型会在整个 50,257 词表上输出一个概率分布。 一个策略把这个分布变成一个被选中的 token。不同策略在 连贯性 vs 创造力 vs 计算成本之间做不同的取舍。

策略开关风格适用场景
贪心 Greedy do_sample=False 确定性,容易重复 只有一个正确答案的任务(低温下的翻译)
Beam search num_beams=4 稳妥,有时显得平淡 摘要、结构化输出
采样 Sampling do_sample=True 有创意,可能跑偏 讲故事、头脑风暴
top-k top_k=50 砍掉最差的尾巴 和采样搭配使用更稳
top-p(nucleus) top_p=0.9 按置信度自适应 几乎总是用 — 现代默认
temperature temperature=0.8 控制尖锐度的旋钮 聊天 0.7–1.0,代码 0.2,搞混乱 >1.2
temperature 直觉
在 softmax 之前,logits 会先除以 T。低 T 会让分布更尖锐(接近贪心), 高 T 让分布更平坦(接近均匀)。T→0 退化为贪心。 T=1 时就是从模型本身的分布采样。
top-k 直觉
把概率最高的前 k 个之外的 token 全部丢掉。重新归一化后采样。 砍掉了奇怪、稀有 token 的长尾。但有点死板:对置信度高和低的两步用同一个 k。
top-p(nucleus)
保留累计概率 ≥ p 的最小 token 集合。逐步自适应:置信度高 → 集合很小; 平坦时 → 集合宽。在大多数任务上比纯 top-k 好。 p=0.9 是稳妥默认。
并排对比
# 1. 贪心
model.generate(**inputs, max_new_tokens=30, do_sample=False)

# 2. beam search
model.generate(**inputs, max_new_tokens=30, num_beams=4, early_stopping=True)

# 3. 普通采样
model.generate(**inputs, max_new_tokens=30, do_sample=True, temperature=1.0)

# 4. top-k
model.generate(**inputs, max_new_tokens=30, do_sample=True, top_k=50)

# 5. top-p(nucleus)
model.generate(**inputs, max_new_tokens=30, do_sample=True, top_p=0.9)

# 6. 推荐组合
model.generate(**inputs, max_new_tokens=50,
               do_sample=True, top_k=50, top_p=0.9, temperature=0.8)
实战建议:从 do_sample=True, top_p=0.9, temperature=0.8 开始。 输出太随机,降 T。陷入循环,把 repetition_penalty 调到 1.1–1.3。 觉得太平淡,把 T 提到 1.0 并去掉 top_k。

把这些策略敲一遍

路径 A — Trainer API

端到端微调 gpt2
from transformers import (
    AutoTokenizer, AutoModelForCausalLM,
    Trainer, TrainingArguments, DataCollatorForLanguageModeling,
)
from datasets import load_dataset

tokenizer = AutoTokenizer.from_pretrained("gpt2")
tokenizer.pad_token = tokenizer.eos_token

model = AutoModelForCausalLM.from_pretrained("gpt2")

ds = load_dataset("wikitext", "wikitext-2-raw-v1")

def tok(batch):
    return tokenizer(batch["text"], truncation=True, max_length=512)

ds = ds.map(tok, batched=True, remove_columns=["text"])

collator = DataCollatorForLanguageModeling(
    tokenizer=tokenizer,
    mlm=False,                          # 因果 LM,而非 masked LM
)

args = TrainingArguments(
    output_dir="./gpt2-ft",
    per_device_train_batch_size=4,
    num_train_epochs=3,
    learning_rate=5e-5,
    warmup_steps=100,
    logging_steps=50,
    save_steps=500,
    fp16=True,
)

trainer = Trainer(
    model=model,
    args=args,
    train_dataset=ds["train"],
    eval_dataset=ds["validation"],
    data_collator=collator,
)

trainer.train()
trainer.save_model("./gpt2-ft")

路径 B — 原生 PyTorch 循环

不用 Trainer,纯 torch
import torch
from torch.utils.data import DataLoader
from torch.optim import AdamW

loader    = DataLoader(ds["train"], batch_size=4, collate_fn=collator)
optimizer = AdamW(model.parameters(), lr=5e-5)
device    = "cuda" if torch.cuda.is_available() else "cpu"
model.to(device)
model.train()

for epoch in range(3):
    for batch in loader:
        batch = {k: v.to(device) for k, v in batch.items()}
        outputs = model(**batch)             # labels=input_ids 由 collator 处理
        loss    = outputs.loss               # transformers 直接返回 CE loss
        loss.backward()
        optimizer.step()
        optimizer.zero_grad()
Loss = labels = input_ids:对于因果 LM,输入和 目标是同一个 token 序列 — 模型要从前面的 token 预测每个 token。 DataCollatorForLanguageModeling(mlm=False) 会自动给你设置好 labels。

TrainingArguments — 值得记住的每一个旋钮

TrainingArguments 这个 dataclass 持有 Trainer 涉及的 每一个超参。一共 100+ 个字段。下面这些是你实际会动手设的。

参数典型值作用
output_dir "./out" checkpoint + 日志的去处(必填)
per_device_train_batch_size 4–32 每张 GPU 的批量;有效批量 = 这个 × num_gpus × grad_accum
gradient_accumulation_steps 1–16 显存吃紧时模拟更大批量
num_train_epochs 1–10 遍历训练集的次数
learning_rate 5e-5 transformer 微调的标准值
warmup_steps 100–1000 线性升温 — 防止早期 loss 飙升
weight_decay 0.01 AdamW 的正则化
fp16 / bf16 True 混合精度;Ampere 及以上用 bf16,其它用 fp16
logging_steps 50 每 N 步打印一次 loss
save_steps 500 每 N 步存一次 checkpoint
evaluation_strategy "steps" "no" / "steps" / "epoch"
load_best_model_at_end True 训练完后,自动恢复最好的 checkpoint
report_to "wandb" 或者 "tensorboard" / "none"
接近生产的配置
args = TrainingArguments(
    output_dir="./gpt2-ft",
    per_device_train_batch_size=8,
    gradient_accumulation_steps=4,         # 有效批量 = 32
    num_train_epochs=3,
    learning_rate=5e-5,
    warmup_steps=200,
    weight_decay=0.01,
    fp16=True,
    logging_steps=50,
    save_steps=500,
    evaluation_strategy="steps",
    eval_steps=500,
    load_best_model_at_end=True,
    report_to="none",
)
有效批量大小: per_device_train_batch_size × num_gpus × gradient_accumulation_steps。 一张 24 GB GPU 也许只能塞下批量 8;用 grad-accum 4 可以得到批量 32 的 loss 曲线。

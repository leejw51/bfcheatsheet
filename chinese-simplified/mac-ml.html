Mac 作为机器学习工作站

Apple Silicon(M1/M2/M3/M4)是一台正经的机器学习设备。运行你 IDE 的同一颗芯片 可以训练 transformer、微调 LoRA,并在本地服务 7B 参数模型 — 不必租 GPU、 也不用 nvidia-smi。本页面教你需要的三样工具: arm64 上的 Python 带 MPS 的 PyTorch MLX — 通过亲手输入来学习。

统一内存
在 Apple Silicon 上,CPU 和 GPU 共享同一块内存。没有 cuda 拷贝,也没有 .to(device) 瓶颈。能装进 16 GB 的模型就直接跑起来, 无论你从 PyTorch(MPS)还是 MLX 调用它。
Mac 上的 PyTorch
完整的 PyTorch 配上 MPS 后端(Metal Performance Shaders)。 和其他平台一样的 API — 只需把 cuda 换成 mps。 大多数算子可用;少数会回退到 CPU。
MLX
Apple 的数组框架,从零为 Apple Silicon 打造。类似 NumPy、 默认惰性求值、自动微分、原生 LLM 工具链。更小、更快、 且 Mac 原生。

你将获得什么

1
一个干净的 arm64 Python。已验证为原生架构,在虚拟环境中,准备好安装 ML 包。
2
跑在 GPU 上的 PyTorch。那个标准的 device = torch.device("mps" if ...) 咒语。
3
MLX 的 hello-world。正确使用 mx.value_and_grad 和 mx.eval 的线性回归。
4
本地 LLM。一行 mlx_lm.generate 命令,你就能在终端里和 7B 模型聊天。
5
一条决策准则。并排对比表:何时选 MLX,何时选 PyTorch。

PyTorch + MPS — 你 Mac 上的 GPU

自 PyTorch 1.12(2022 年 5 月)起,MPS 后端使用 Apple 的 Metal Performance Shaders 在集成 GPU 上运行张量算子。同样的 PyTorch API, Mac 风味的设备。

安装
# mac 上常规的 pytorch 轮子已启用 mps
pip install torch torchvision

# nightly 版本通常对 mps 算子覆盖更好
pip install --pre torch torchvision \
    --index-url https://download.pytorch.org/whl/nightly/cpu

设备选择之舞

挑选最佳设备的可移植代码
import torch

# 在 Mac(mps)、Linux(cuda)和仅 CPU 的机器上都能运行

    "mps"  if torch.backends.mps.is_available() else
    "cuda" if torch.cuda.is_available()        else
    "cpu"

CPU 回退安全阀

当算子尚未在 MPS 上实现时
# 在 python 启动前导出此变量(或在导入 torch 之前于 Python 中设置)
export PYTORCH_ENABLE_MPS_FALLBACK=1

# 或者在代码中:
import os
os.environ["PYTORCH_ENABLE_MPS_FALLBACK"] = "1"
import torch
这是干什么的:如果 PyTorch 遇到一个尚未移植到 MPS 的算子,它会 悄悄回退到 CPU 而不是崩溃。慢一些,但你的代码能跑完。截至 PyTorch 2.x,大多数常用算子都可用 — 回退很少见。

微小的端到端示例

一个 1 层 MLP,在噪声上训练,跑在 GPU 上
import torch
import torch.nn as nn

device = torch.device("mps" if torch.backends.mps.is_available() else "cpu")

X = torch.randn(256, 8, device=device)
y = torch.randn(256, 1, device=device)

model = nn.Sequential(
    nn.Linear(8, 32), nn.ReLU(),
    nn.Linear(32, 1),
).to(device)

opt = torch.optim.Adam(model.parameters(), lr=1e-3)
loss_fn = nn.MSELoss()

for step in range(200):
    pred = model(X)
    loss = loss_fn(pred, y)
    opt.zero_grad(); loss.backward(); opt.step()

print("final loss", loss.item())

MLX — Apple 的数组框架

MLX 是 Apple 的 Apple Silicon 机器学习研究框架。 可以把它想成 NumPy 遇上 JAX,再加上 PyTorch 的人体工学。它是惰性的,支持自动微分, 并原生利用统一内存。专为 Mac 设计,出自打造 Mac 的团队。

安装
# 核心数组库
pip install mlx

# LLM 助手(mlx_lm)
pip install mlx-lm
你总会用到的导入
import mlx.core as mx
import mlx.nn   as nn
import mlx.optimizers as optim

三个必须内化的概念

L
默认惰性。mx 的算子构建一张计算图。直到你调用 mx.eval(x) 或读取一个值(例如 print、 numpy())之前,什么都不会真正计算。这让 MLX 能融合 + 调度。
U
统一内存,不是"设备"。没有 .to(device)。 数组住在共享内存里;算子可以自动在 CPU 或 GPU 上运行。
A
自动微分是函数式的。使用 mx.grad(fn) 或 mx.value_and_grad(fn) — 它们返回一个返回梯度的函数。 没有 .backward()。

Hello-world:MLX 中的线性回归

用自动微分,以 25 行代码拟合 y = 2x + 1
import mlx.core as mx
import mlx.nn   as nn
import mlx.optimizers as optim

# 合成数据:y = 2x + 1 + noise
mx.random.seed(0)
X = mx.random.normal((100, 1))
y = 2.0 * X + 1.0 + 0.1 * mx.random.normal((100, 1))

# 模型 = 单个 Linear 层
model = nn.Linear(1, 1)

def loss_fn(model, X, y):
    pred = model(X)
    return mx.mean((pred - y) ** 2)

loss_and_grad = nn.value_and_grad(model, loss_fn)
opt = optim.SGD(learning_rate=0.1)

for step in range(200):
    loss, grads = loss_and_grad(model, X, y)
    opt.update(model, grads)
    mx.eval(model.parameters(), opt.state)   # 强制计算

print(model.weight.item(), model.bias.item())   # ≈ 2.0, 1.0
mx.eval 的诀窍:因为 MLX 是惰性的,梯度并不会 真正被计算,直到你要求它们。每个训练步调用一次 mx.eval(model.parameters(), opt.state) 来物化它们 — 否则计算图会一直增长。

MLX vs PyTorch — 何时用哪个

两者在 Apple Silicon 上都很出色,功能重叠很多。下面教你如何选择。

MLX PyTorch (MPS)
设计目标 apple silicon 仅 Mac,原生 universal Mac · Linux · Windows · CUDA
API 风格 NumPy + 函数式自动微分(类 JAX) NumPy + nn.Module + .backward()
执行方式 惰性 — 用 mx.eval() 物化 即时(默认);torch.compile 可选启用
内存模型 统一 — 不需要设备之舞 显式设备 — .to("mps")
算子覆盖 库较小 — 增长很快 巨大 — 几乎全部可用(MPS 偶有缺口)
LLM 服务 mlx_lm — 量化、LoRA、生成、服务器 使用 transformers + accelerate
生态系统 年轻 — HF 上的 mlx-community,工具链在成长 庞大 — torchvision、HuggingFace、lightning 等等
M 系列性能 通常最快(原生 + 惰性融合) 出色,在重型 CNN 算子上有时更快
调试 惰性让打印变棘手 — 需调用 eval 即时 — 哪里都能 print,直接好用
何时选它 本地 LLM 工作、面向 Mac 的研究、追求 Apple 极致性能 移植 CUDA 代码、使用现成的 HF 模型、跨平台
MLX — 选它当…
  • 你在本地运行 Llama 级别的 LLM
  • 你想在笔记本上做 LoRA 微调
  • 你喜欢 JAX 风格的函数式代码
  • 你的项目仅 Mac — 不需要 CUDA
  • 你想要最少的依赖 + 启动够快
PyTorch — 选它当…
  • 你要把现有的 PyTorch 代码从 CUDA 移植过来
  • 你需要 torchvision / diffusers / transformers
  • 你之后会部署到 Linux GPU
  • 你想要最大的社区 / 文档 / Stack Overflow
  • 你需要小众层(例如复杂的频谱算子)

同一任务,两个框架

MLX 矩阵乘
import mlx.core as mx

A = mx.random.normal((1024, 1024))
B = mx.random.normal((1024, 1024))
C = A @ B
mx.eval(C)
print(C.shape)
PyTorch 矩阵乘

C = A @ B
print(C.shape)
诚实的回答:对 Mac 上的大多数初学者,正确的答案是"两个都用"。 从 PyTorch 开始(一切都有文档),想榨干 LLM 在 Mac 上的最大性能时再用 MLX。

Mac ML 性能技巧

几条习惯就能把快的 Mac ML 和慢的 Mac ML 区分开。多数都是关于尊重 统一内存、避免不必要的工作。

1
全程坚持 arm64。一个 Rosetta 依赖会污染整条链路。 file $(which python) 应当报告 arm64。brew --prefix 同理 (应是 /opt/homebrew,不是 /usr/local)。
2
固定一个较新的 PyTorch。每个版本 MPS 支持都在改进。目标是 PyTorch 2.3+。nightly 版本经常拥有稳定版尚未发布的算子覆盖。
3
批量调用 mx.eval。不要每个算子后都调它 — 而是每个 训练步对需要物化的东西调一次: mx.eval(model.parameters(), opt.state)。
4
推理时量化。4-bit 把内存削减约 4 倍,对聊天的质量损失可忽略。 MLX 和 PyTorch(通过 bitsandbytes — Mac 上有限)都支持。 最简单的方式:从 mlx-community 拉取 -4bit 模型。
5
关注统一内存。Mac 在 CPU 和 GPU 之间共享 RAM。 一台 64 GB 的 Mac 能跑 70B 左右的 4-bit 模型;一台 8 GB 的 Mac 更接近 3B 的 4-bit。 活动监视器 → 内存标签是你的好朋友。
6
不要和惰性图较劲。在 MLX 中,在循环内累积 Python 端状态 (例如 losses.append(loss))而不周期性地 eval 会让计算图爆炸。在 append 前先 mx.eval(loss)。
7
避免 CPU↔MPS 来回切换。PyTorch 中每次 .to("cpu") 或 .numpy() 都会让你付出代价。把整个训练步构建在设备上, 只在边界(日志、保存检查点)处物化。
8
浮点类型很重要。MLX 默认 float32;mx.float16 和 mx.bfloat16 大致可减半内存,且常常加速推理。 PyTorch:torch.set_default_dtype(torch.float16) 或 autocast。
9
有选择地使用 torch.compile。在 MPS 上它可能帮上忙也可能 帮倒忙 — 实测为准。只包裹内部前向通路,把外部训练循环保持为普通 Python。
10
插上电源。Apple Silicon 在电池上会被压制 — 性能核会降频, GPU 时钟会下降。做基准测试时,接电源并使用高性能配置文件。

常见坑

"我的 MPS 代码打印出 inf / nan"
某些算子(尤其是较旧的 PyTorch + 复杂的频谱算子)在 MPS 上会溢出, 而在 CUDA 上不会。试试 PYTORCH_ENABLE_MPS_FALLBACK=1 和较小的学习率。
"MLX 比预期慢"
几乎总是:缺少 mx.eval。检查你是不是每步物化一次, 而不是每个 minibatch 算子都做 — 或者更糟,直到最后才做。
"pip install torch 失败"
你正在 Rosetta 下运行 x86_64 Python。换成 arm64 Python(uv / pyenv / python.org 安装器),并用 file $(which python) 验证。
"我的 LLM 卡死或 OOM"
做 4-bit 量化、降低 max-tokens / 上下文长度,或挑选更小的模型(3B 而不是 7B)。 统一内存是和系统上其他一切共享的。

# mlx_quadratic.py — 在 Apple Silicon 上拟合 y = x^2 + noise
import mlx.core as 
import mlx.nn as nn
import mlx.optimizers as optim

# --- 数据 ---
mx.random.seed(42)
X = mx.random.uniform(low=-1, high=1, shape=(256, 1))
Y = X ** 2 + 0.05 * mx.random.normal((256, 1))

# --- 模型 ---
class MLP(nn.Module):
    def __init__(self):
        super().__init__()
        self.l1 = nn.Linear(, 32)
        self.l2 = nn.Linear(32, 32)
        self.l3 = nn.Linear(32, )

    def __call__(self, x):
        x = nn.relu(self.l1(x))
        x = nn.relu(self.l2(x))
        return self.(x)

model = MLP()

# --- 损失 + 自动微分 ---
def loss_fn(model, X, Y):
    return mx.mean((model(X) - Y) ** 2)

loss_and_grad = nn.(model, loss_fn)
opt = optim.(learning_rate=3e-3)

# --- 训练循环 ---
for step in range(500):
    loss, grads = loss_and_grad(model, X, Y)
    opt.(model, grads)
    mx.(model.parameters(), opt.state)
    if step % 50 == 0:
        print(f"step {step:4d}  loss {loss.item():.4f}")

# --- 保存权重 ---
mx.("model.safetensors", dict(tree_flatten(model.parameters())))

nanoGPT — Karpathy 用约 300 行 PyTorch 写的最简 GPT

karpathy/nanoGPT 是最简单的端到端仓库, 能复现 GPT-2 级别的训练质量。两个文件做掉了几乎所有事: model.py 定义网络, train.py 训练它。在本页一行行敲过来, GPT 就在你脑子里了。

为什么选 nanoGPT
小到一小时就能读完,但又大到能在 OpenWebText 上把 GPT-2(124M)训练到 和原版同样的 loss。单个 PyTorch 文件,零抽象。每个人的入门参考实现。
你将搭出什么
一个 decoder-only Transformer:token + 位置 embedding → N 份 (LN → 因果 attn → 残差 → LN → MLP → 残差) → 最终 LN → 线性头。 这就是一个 GPT。
tiny shakespeare
一个约 1MB 的莎士比亚文本,在笔记本 GPU 上几分钟就能训练完。 char 级别能立刻玩起来,或者用 BPE(tiktoken)做正经分词。两种都通过 data/<name>/prepare.py 提供。
想多大就多大
改六个旋钮就能用同一份代码复现 GPT-2: n_layer, n_head, n_embd, block_size, batch_size, lr。 torchrun 多卡只差一个开关。

四个任务 → 四个文件

model.py — 定义
四个类:CausalSelfAttention、 MLP、 Block、 GPT。一个 @dataclass GPTConfig 持有超参。
train.py — 拟合

model.py — Block
class Block(nn.Module):
    def __init__(self, config):
        super().__init__()
        self.ln_1 = LayerNorm(config.n_embd, bias=config.bias)
        self.attn = CausalSelfAttention(config)
        self.ln_2 = LayerNorm(config.n_embd, bias=config.bias)
        self.mlp  = MLP(config)

    def forward(self, x):
        x = x + self.attn(self.ln_1(x))   # 残差 + pre-norm attn
        x = x + self.mlp(self.ln_2(x))    # 残差 + pre-norm mlp
        return x

MLP

model.py — MLP(4× 隐层,GELU)
class MLP(nn.Module):
    def __init__(self, config):
        super().__init__()
        self.c_fc    = nn.Linear(config.n_embd, 4 * config.n_embd, bias=config.bias)
        self.gelu    = nn.GELU()
        self.c_proj  = nn.Linear(4 * config.n_embd, config.n_embd, bias=config.bias)
        self.dropout = nn.Dropout(config.dropout)

    def forward(self, x):
        return self.dropout(self.c_proj(self.gelu(self.c_fc(x))))

因果自注意力 — GPT 的心脏

每个 token 用学到的相似度权重去混合前面 token(包括自己)的信息。 因果掩码把未来位置抹掉,所以时刻 t 的 token 永远看不到 t+1。 多头把通道维分成 n_head 份独立的注意力计算。

注意力热图(T×T)

越亮 = 注意力权重越大。拖动温度滑块让 softmax 变尖或变平。 切换因果掩码:关掉就成了双向(BERT 风格)的注意力。

因果掩码:开
因果掩码:softmax 之前把上三角换成 -inf,这样那部分注意力权重就变 0。 技巧:用 torch.tril(torch.ones(T,T)) 构造一次,然后反复复用。
model.py — CausalSelfAttention(forward)
def forward(self, x):
    B, T, C = x.size()                            # batch, time, channels
    q, k, v = self.c_attn(x).split(self.n_embd, dim=2)

    # 拆成多头:(B, T, C) → (B, nh, T, hs),其中 C = nh * hs
    q = q.view(B, T, self.n_head, C // self.n_head).transpose(1, 2)
    k = k.view(B, T, self.n_head, C // self.n_head).transpose(1, 2)
    v = v.view(B, T, self.n_head, C // self.n_head).transpose(1, 2)

    att = (q @ k.transpose(-2, -1)) * (1.0 / math.sqrt(k.size(-1)))
    att = att.masked_fill(self.bias[:,:,:T,:T] == 0, float('-inf'))   # 因果
    att = F.softmax(att, dim=-1)
    att = self.attn_dropout(att)
    y = att @ v                                   # (B, nh, T, hs)
    y = y.transpose(1, 2).contiguous().view(B, T, C)   # 重新拼接
    return self.resid_dropout(self.c_proj(y))
model.py — 注册因果掩码 buffer
# 在 __init__ 里:持久化(不可训练)的下三角掩码
self.register_buffer("bias",
    torch.tril(torch.ones(config.block_size, config.block_size))
         .view(1, 1, config.block_size, config.block_size))

while True:
    lr = get_lr(iter_num) if decay_lr else learning_rate
    for g in optimizer.param_groups: g['lr'] = lr

    for micro_step in range(gradient_accumulation_steps):
        with ctx:                                    # bfloat16 autocast

采样 — 把 logits 变成文本

生成是自回归的:喂入 prompt,取最后一个 logit,做 温度缩放,可选地裁到 top-k,过 softmax, 用 multinomial 采样,追加,继续。

8 个模拟 token 上的采样分布

拖滑块。τ → 0 收敛为 argmax;τ → ∞ 变成均匀分布。top_k 把 top-k 之外的全部归零。

1.00
8
套路: logits/τ → 屏蔽 top-k 之外 → softmax → torch.multinomial(probs, 1)。
model.py — generate(标准循环)
@torch.no_grad()
def generate(self, idx, max_new_tokens, temperature=1.0, top_k=None):
    for _ in range(max_new_tokens):
        idx_cond = idx if idx.size(1) <= self.config.block_size else idx[:, -self.config.block_size:]
        logits, _ = self(idx_cond)
        logits = logits[:, -1, :] / temperature        # 只看最后一步
        if top_k is not None:
            v, _ = torch.topk(logits, min(top_k, logits.size(-1)))
            logits[logits < v[:, [-1]]] = -float('inf')
        probs = F.softmax(logits, dim=-1)
        idx_next = torch.multinomial(probs, num_samples=1)
        idx = torch.cat((idx, idx_next), dim=1)
    return idx

nanoGPT vs HuggingFace

nanoGPT
约 300 行,单文件,零抽象。可改可玩。通过 model.py 里的辅助函数 from_pretrained('gpt2') 也能复现 GPT-2 的数值。
transformers(HF)
生产级:tokenizers、几十种架构、各种生成技巧(beam、contrastive、nucleus、 repetition penalty)。表面积更大。上线用 HF;学习用 nanoGPT。

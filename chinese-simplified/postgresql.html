PostgreSQL 命令行 — 反复操练 psql 直到形成肌肉记忆

psql 是 PostgreSQL 的官方终端客户端。它随每个服务器一起发布， 可脚本化、速度极快，并且配备了一整套反斜杠 元命令，胜过任何图形界面。 读完本页，你将亲手 敲过 连接参数、元命令、SQL、导出/恢复、维护一行命令， 以及一个完整的小项目。

什么是 psql？
一个使用 PostgreSQL 通信协议的终端程序。它能执行 SQL，也能运行 反斜杠命令，例如 \dt（列出表）和 \d users（描述表）。支持 Tab 自动补全、历史记录、 通过 -c / -f 编写脚本，以及通过 \copy 批量导入。
为什么不用图形界面？
因为图形界面隐藏了真正的命令、无法脚本化、不能在远程机器上通过 SSH 运行， 而且面对大结果集就会卡死。一旦 \d、 \dt+、\watch 刻进了你的手指，你就会比地球上任何 UI 都更快。
psql 的好搭档
pg_dump / pg_restore 负责备份，pg_isready 用于健康检查， createdb / dropdb 是轻量包装器，还有 vacuumdb、 reindexdb。它们使用同一套连接参数。
约定
默认端口 5432，默认数据库和用户通常与你的操作系统用户同名。 在 psql 里： \? 列出元命令， \h SELECT 显示 SQL 语法， \q 退出。

你将熟记的 psql 速查卡

连接 — 进入 psql

六种连接方式。挑一种适合你当前情况的。

所有连接方式
# 1. 默认值 — 使用 $PGUSER / $PGDATABASE / unix socket
psql

# 2. 显式参数
psql -h db.example.com -p 5432 -U alice -d shop

# 3. URI 形式（适用于环境变量 / CI）
psql postgresql://alice:secret@db.example.com:5432/shop

# 4. 一次性查询，不进入交互式 shell
psql -h host -U alice -d shop -c "SELECT now();"

# 5. 运行一个 .sql 脚本
psql -h host -U alice -d shop -f migrate.sql

# 6. 通过环境变量
PGHOST=db PGUSER=alice PGDATABASE=shop psql

免去每次输入密码的认证方式

~/.pgpass
# ~/.pgpass — 每行一条 (host, port, db, user, password)
# 任意字段都可使用通配符
db.example.com:5432:shop:alice:s3cret
*:*:*:postgres:rootpw

# 必须 chmod 600，否则 psql 会忽略它
chmod 600 ~/.pgpass
在 psql 内安全地修改自己的密码 用 \password。它在客户端进行哈希，明文不会离开终端 —— 比 ALTER USER ... PASSWORD '...' 安全得多， 后者会把密码写入服务器日志。

检查 — 模式、类型、索引、执行计划

连接成功后，第一件事就是四处看看。下面是了解数据库内容的方法。

SQL DDL — 你会真正用到的类型

CREATE TABLE — 丰富的类型
CREATE TABLE users (
    id          UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
    email       TEXT        NOT NULL UNIQUE,
    age         INT         CHECK (age >= 0),
    tags        TEXT[],                                -- 数组
    profile     JSONB       NOT NULL DEFAULT '{}',    -- JSONB
    created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- 外键
CREATE TABLE orders (
    id       BIGSERIAL PRIMARY KEY,
    user_id  UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    total    NUMERIC(10,2) NOT NULL
);

索引 — 三种风味覆盖大多数场景

索引模式
-- 1. 普通 B-tree（默认）
CREATE INDEX idx_orders_user ON orders(user_id);

-- 2. 通过索引实现唯一约束
CREATE UNIQUE INDEX idx_users_email ON users(lower(email));

-- 3. 部分索引 — 只对 "活动" 行建索引
CREATE INDEX idx_open_orders ON orders(user_id)
    WHERE status = 'open';

-- 4. GIN 用于 JSONB / 数组 / 全文
CREATE INDEX idx_users_profile ON users USING GIN (profile);
CREATE INDEX idx_users_tags    ON users USING GIN (tags);

EXPLAIN — 解读你的查询计划

explain analyze
EXPLAIN (ANALYZE, BUFFERS) SELECT * FROM orders WHERE user_id = '...';
--  Index Scan using idx_orders_user on orders  (cost=0.42..8.44 rows=1 width=...)
--    Index Cond: (user_id = '...'::uuid)
--  Planning Time: 0.12 ms
--  Execution Time: 0.04 ms
读 EXPLAIN：执行从顶向下，开销从底向上。在你期望走 Index Scan 的大表上看到 Seq Scan，意味着索引缺失或未被使用。使用 BUFFERS 选项可以查看磁盘读取与缓存命中。

UPSERT、CTE、窗口函数 — 真正的好东西

upsert (INSERT … ON CONFLICT)
INSERT INTO users (id, email)
VALUES ('a-uuid', 'a@x')
ON CONFLICT (id) DO UPDATE SET email = EXCLUDED.email;
CTE + 窗口函数
WITH ranked AS (
    SELECT user_id, total,
           ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY total DESC) AS rn
    FROM orders
)
SELECT * FROM ranked WHERE rn = 1;   -- 每个用户最大订单

实用一行命令

生产环境 DBA 凌晨三点真正会粘贴的查询。请收藏。

最大的 5 张表

体积侦探
SELECT
    schemaname || '.' || relname           AS table,
    pg_size_pretty(pg_total_relation_size(relid))  AS total_size,
    pg_size_pretty(pg_relation_size(relid))        AS data_size,
    pg_size_pretty(pg_total_relation_size(relid) - pg_relation_size(relid)) AS idx_size
FROM pg_catalog.pg_statio_user_tables
ORDER BY pg_total_relation_size(relid) DESC
LIMIT 5;

活动查询 — 此刻谁在做什么

pg_stat_activity
SELECT pid, usename, application_name, state,
       now() - query_start AS runtime,
       left(query, 100) AS q
FROM pg_stat_activity
WHERE state <> 'idle'
ORDER BY runtime DESC;

取消 / 杀死失控的查询

cancel 与 terminate
-- 文明：请求查询停止（连接保持）
SELECT pg_cancel_backend(12345);

-- 核弹：直接断开整条连接
SELECT pg_terminate_backend(12345);

角色与 GRANT

创建只读用户
CREATE ROLE readonly LOGIN PASSWORD 'change-me';
GRANT CONNECT ON DATABASE shop TO readonly;
GRANT USAGE ON SCHEMA public TO readonly;
GRANT SELECT ON ALL TABLES IN SCHEMA public TO readonly;
ALTER DEFAULT PRIVILEGES IN SCHEMA public
    GRANT SELECT ON TABLES TO readonly;

值得了解的扩展

pg_stat_statements
跟踪每条查询的运行时间和计划统计。真正的 DBA 第一件要装的扩展。 
CREATE EXTENSION IF NOT EXISTS pg_stat_statements;
pgcrypto
gen_random_uuid()、crypt()， bcrypt 密码哈希。 
CREATE EXTENSION IF NOT EXISTS pgcrypto;
uuid-ossp
老一辈 UUID 生成器（v1、v3、v4、v5）。在新版 PG 上更推荐 pgcrypto 的 gen_random_uuid()。 
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
pg_trgm
模糊 / 相似度文本搜索。配合 GIN 索引可以让 LIKE '%foo%' 飞起来。 
CREATE EXTENSION IF NOT EXISTS pg_trgm;
pgvector
存储向量嵌入；<-> 是 L2， <=> 是余弦。原生支持 AI/RAG。 
CREATE EXTENSION IF NOT EXISTS vector;

备份与恢复 — pg_dump / pg_restore

两个二进制程序、四个参数。掌握它们，你的数据就安全了。

四种格式

参数格式使用场景
-Fp纯 SQL小型数据库，可读，使用 psql -f 重放
-Fc自定义（二进制）运维默认选择 —— 压缩、并行恢复、表过滤
-Fttar未压缩目录 tarball — 较少用
-Fd目录使用 -j N 进行并行 导出

实用配方

备份
# 单个数据库，自定义格式（推荐）
pg_dump -h host -U alice -Fc shop > shop.dump

# 压缩的纯 SQL
pg_dump -h host -U alice shop | gzip > shop.sql.gz

# 仅模式 / 仅数据
pg_dump -Fc --schema-only shop > shop_schema.dump
pg_dump -Fc --data-only   shop > shop_data.dump

# 并行目录导出（4 个工作线程）
pg_dump -Fd -j 4 -f shop_dir shop

# 集群上每个数据库（含角色）
pg_dumpall -U postgres > cluster.sql
恢复
# 自定义格式 → 用 pg_restore
createdb shop_new
pg_restore -h host -U alice -d shop_new -j 4 shop.dump

# 纯 SQL → 用 psql -f
psql -h host -U alice -d shop_new -f shop.sql

# 列出自定义转储的内容（不实际恢复）
pg_restore -l shop.dump | head

# 只恢复一张表
pg_restore -d shop_new -t users shop.dump

维护 — 让 PG 保持健康

vacuum / analyze / reindex
-- 回收死行 + 刷新规划器统计
VACUUM ANALYZE;
VACUUM ANALYZE orders;            -- 单张表

-- 激进：锁表并重写（很少需要）
VACUUM FULL orders;

-- 重建膨胀的索引
REINDEX TABLE CONCURRENTLY orders;
REINDEX INDEX idx_orders_user;

恢复的黄金法则：始终恢复到一个 全新 的空数据库 （createdb shop_new），验证无误，然后再改名或重新指向你的应用。 永远不要在线上数据库之上直接恢复。

PyTorch + MNIST — read handwriting in 80 lines

MNIST is the hello-world of deep learning: 60,000 hand-drawn digits, each a 28×28 grayscale image, labeled 0–9. With two linear layers and three epochs of training, a tiny PyTorch model gets ~97% test accuracy. Here is the entire program.

the whole thing — copy-paste runnable
import torch
import torch.nn as nn
import torch.nn.functional as F
from torchvision import datasets, transforms
from torch.utils.data import DataLoader

# --- data -------------------------------------------------------
tfm = transforms.Compose([
    transforms.ToTensor(),
    transforms.Normalize((0.1307,), (0.3081,)),
])
train_ds = datasets.MNIST('./data', train=True,  download=True, transform=tfm)
test_ds  = datasets.MNIST('./data', train=False, download=True, transform=tfm)
train_dl = DataLoader(train_ds, batch_size=64, shuffle=True)

# --- model ------------------------------------------------------
class Net(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(28*28, 128)
        self.fc2 = nn.Linear(128, 10)
    def forward(self, x):
        x = x.view(-1, 28*28)            # flatten
        x = F.relu(self.fc1(x))
        return self.fc2(x)               # raw logits

model   = Net()
opt     = torch.optim.SGD(model.parameters(), lr=0.01, momentum=0.9)
loss_fn = nn.CrossEntropyLoss()

# --- training ---------------------------------------------------
for epoch in range(3):
    for x, y in train_dl:
        opt.zero_grad()
        logits = model(x)
        loss   = loss_fn(logits, y)
        loss.backward()
        opt.step()

# --- inference --------------------------------------------------
img = test_ds[0][0].unsqueeze(0)         # [1, 1, 28, 28]
with torch.no_grad():
    pred = model(img).argmax(dim=1).item()
print(pred)   # prints the predicted digit, 0..9
why MNIST

70k images, all 28×28, labeled 0–9. Small enough to train on a laptop CPU in seconds, big enough to actually learn something. Released by Yann LeCun in 1998; still the baseline that everyone starts with.

why this model

Two nn.Linear layers — input 784 → hidden 128 → output 10. ReLU between them. About 101k parameters. A real CNN (LeNet, ResNet) does better, but this MLP already nails ~97%.

The 5 ideas, one card each

1tensors

Images are [B, 1, 28, 28] tensors of floats. Labels are [B] longs (0..9).

2nn.Module

Subclass it. Define layers in __init__, call them in forward. PyTorch tracks the rest.

3autograd

loss.backward() walks the graph of operations and fills .grad on every weight.

4optimizer

opt.step() applies w -= lr * w.grad. opt.zero_grad() resets between batches.

5loss

CrossEntropyLoss takes logits + integer labels — applies softmax + log internally.

Data — tensors, transforms, DataLoader

Three pieces work together: a Dataset (knows how to fetch one example), a transform pipeline (turns PIL images into normalized tensors), and a DataLoader (batches and shuffles).

1. The transform pipeline

PIL image → float tensor → normalized
from torchvision import transforms

tfm = transforms.Compose([
    transforms.ToTensor(),                       # PIL → [1,28,28] float in [0,1]
    transforms.Normalize((0.1307,), (0.3081,)), # (x - μ) / σ
])

0.1307 is the mean pixel value across MNIST; 0.3081 is the standard deviation. Normalizing centers the inputs so gradients behave well.

2. The dataset object

Model — two Linear layers and a ReLU

Every PyTorch model subclasses nn.Module. The pattern is always the same: declare layers in __init__, describe how data flows in forward. PyTorch handles parameters, gradients, and device placement.

the smallest MNIST model that works
import torch.nn as nn
import torch.nn.functional as F

class Net(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(28*28, 128)   # 784 → 128
        self.fc2 = nn.Linear(128,    10)    # 128 → 10

    def forward(self, x):
        x = x.view(-1, 28*28)         # [B,1,28,28] → [B, 784]
        x = F.relu(self.fc1(x))         # [B, 784]    → [B, 128]
        return self.fc2(x)            # [B, 128]    → [B, 10]   (raw logits)

model = Net()
print(sum(p.numel() for p in model.parameters()))   # 101,770 params

Shapes flowing through

What each layer actually does

1x.view(-1, 784)

Reshape without copying. The -1 says "infer the batch dim from the remaining size".

2nn.Linear(784, 128)

Stores a weight matrix W of shape [128, 784] and bias [128]. Computes x @ W.T + b.

3F.relu(z)

max(0, z) elementwise. Cheap, smooth-enough, and the gradient is just an indicator function.

4self.fc2(x)

Final [B,128] → [B,10]. Returns raw logits — no softmax. The loss does that.

5parameters()

Iterator over every learnable tensor. Optimizer needs this.

6nn.Module magic

Anything assigned to self.foo = nn.X(...) is auto-registered for parameters() and state_dict().

Why no softmax in forward()? nn.CrossEntropyLoss is logits + softmax + log + NLL fused into one numerically-stable op. If you softmax yourself, you'll apply it twice and waste accuracy. Always return raw logits.

Loss & Optimizer — how it learns

The loss tells you how wrong a prediction is, as a single number. The optimizer knows how to nudge the weights to make that number smaller. Together they turn random weights into a digit classifier.

nn.CrossEntropyLoss — the standard for classification

logits + integer labels in, scalar loss out
loss_fn = nn.CrossEntropyLoss()

logits = model(x)            # [B, 10]   raw, can be negative
y      = ...                  # [B]       longs in 0..9
loss   = loss_fn(logits, y)   # scalar tensor
$\mathrm{loss}(x, y) = -\log\!\left(\dfrac{\exp(x_y)}{\sum_j \exp(x_j)}\right)$

Read it as: "take the softmax over the 10 logits, look up the entry for the true class, take its negative log." Best loss is 0 (model assigned probability 1 to the right answer); worst is +∞.

The optimizer — SGD with momentum

vanilla SGD, but with a velocity term
opt = torch.optim.SGD(
    model.parameters(),
    lr=0.01,
    momentum=0.9,
)
$v_{t+1} = \mu\, v_t + \nabla L \quad\quad w_{t+1} = w_t - \eta\, v_{t+1}$

Plain gradient descent: $w \leftarrow w - \eta \nabla L$. Momentum smooths the steps with a running velocity — faster, less wobbly. Adam is a fancier choice; for MNIST, plain SGD is enough.

The trio in action

1forward

logits = model(x)
Stores intermediate activations in the autograd graph.

2compute loss

loss = loss_fn(logits, y)
Single scalar, attached to the same graph.

3backward

loss.backward()
Walks the graph filling p.grad for every parameter.

4update

opt.step()
For each parameter: w -= lr * w.grad (with momentum).

5zero gradients

opt.zero_grad()
PyTorch accumulates gradients by default — clear them before the next batch.

6repeat

One full pass over the dataset = one epoch. Three epochs is enough for ~97% on MNIST.

Common gotcha: if you forget opt.zero_grad(), gradients from the previous batch stick around and the model trains the wrong direction. If your loss is NaN, check your learning rate (try lr=0.001) and your normalization.

The Train Loop — five lines, every time

Whether the model has 100k or 100B parameters, the inner loop is identical: zero gradients, forward, loss, backward, step. Memorize this rhythm.

canonical PyTorch train loop
for epoch in range(3):
    model.train()                       # sets dropout/batchnorm modes
    for x, y in train_dl:                # x: [B,1,28,28]   y: [B]
        opt.zero_grad()                 # clear stale grads
        logits = model(x)               # forward → [B, 10]
        loss   = loss_fn(logits, y)      # scalar
        loss.backward()                  # fill .grad on every param
        opt.step()                       # w -= lr * w.grad  (with momentum)

    # --- eval pass after each epoch -----------------------------
    model.eval()
    correct = 0
    with torch.no_grad():
        for x, y in test_dl:
            pred = model(x).argmax(dim=1)
            correct += (pred == y).sum().item()
    print(f'epoch {epoch} acc {correct/len(test_ds):.3%}')

Each step, drawn out

what opt.zero_grad() does

For every parameter p in the optimizer, sets p.grad = 0 (or None, which is faster). Without this, backward() would add the new gradients to the previous batch's gradients.

what loss.backward() does

Starting from the scalar loss, walks the computational graph (recorded during forward) backward through every op, applying the chain rule. Result: every parameter tensor now has a .grad attribute of the same shape.

what opt.step() does

For each parameter group, applies the update rule. SGD with momentum: v = μ·v + p.grad; p -= lr·v. Adam: a fancier per-parameter rule with adaptive scaling.

why model.train() / .eval()

Some layers (Dropout, BatchNorm) behave differently in train vs eval. This MLP doesn't actually care — but flipping the mode is a habit you want from the start. Pair with with torch.no_grad(): in eval to skip building the autograd graph.

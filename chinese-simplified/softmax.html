1什么是 softmax

神经网络最后一层线性层输出的原始分数被称为 logits —— 任意实数，正负皆可。Softmax 把这个向量变成一个 概率分布：每一项都落在 $(0,1)$ 内，整个向量加起来等于 $1$。

从分数到概率

若分类器对 $K$ 个类别输出 logits $\mathbf{z} = (z_1,\dots,z_K)$，softmax 给出

$$\text{softmax}(\mathbf{z})_i \;=\; \frac{e^{z_i}}{\sum_{j=1}^{K} e^{z_j}}.$$

在 transformer 内部它做的是一样的事：把注意力头看到的 $K$ 个 token 的原始相似度分数转成归一化的 注意力权重。

为什么是 soft max？

硬 $\arg\max$ 会扼杀梯度 —— 只有一个槽位胜出，败者分数的微小变化产生零学习信号。Softmax 是它的光滑、可微近似：最大的 logit 占主导，但每个 logit 仍能分到（一小份）概率质量和可用的梯度。

2公式的推导

构造过程

我们想要一个函数：接收任意向量，输出非负数且和为 1。

  1. 让分数变正：用一个严格正映射。自然指数 $e^{z}$ 是最干净的选择 —— 光滑、单调、$e^z>0$ 永远成立。
  2. 归一化：每项除以总和，使总和等于 1。
$$\text{softmax}(\mathbf{z})_i = \frac{\exp(z_i)}{\sum_j \exp(z_j)}.$$

为什么用指数，而不是 $z_i^2$ 或 $|z_i|$？

  • 对所有实数输入 严格为正（不会出现零分母，不必玩符号游戏）。
  • 乘性间隔：$e^{z_i}/e^{z_j} = e^{z_i - z_j}$ 只取决于差值。给每个 logit 加一个常数没有任何影响 —— 见 §3。
  • 最大熵：在固定平均能量 $\mathbb{E}[z]$ 的所有分布中，指数族熵最大。给定 logits，softmax 是最不武断的分布。
  • 对数似然的共轭：与交叉熵搭档时，梯度坍缩成一行 —— 见 §6。

三个要内化的性质

  • 和为 1。 由构造保证：分子之和 = 分母。拖动任意滑块 —— 条形总是填满整行。
  • 对每个输入单调。 增大 $z_i$，$p_i$ 升高（其他下降），永不交叉。
  • 平移不变。 $\text{softmax}(\mathbf{z}+c\mathbf{1}) = \text{softmax}(\mathbf{z})$。分子分母都额外乘 $e^c$ 后相消。这是数值稳定技巧的基础：取 $c = -\max_i z_i$，使最大指数为 $0$ —— 不溢出，不丢信息。

试试 应用 log-sum-exp 技巧 按钮：它把所有 logits 平移 $-\max$。条形不变（平移不变性！），但显示出来的指数现在都 $\le 1$。

4温度

τ 缩放 logits

$$\text{softmax}_\tau(\mathbf{z})_i = \frac{e^{z_i/\tau}}{\sum_j e^{z_j/\tau}}.$$
  • $\tau \to \infty$：每个 $z_i/\tau \to 0$，分布变成 均匀 $1/K$。最大化探索。
  • $\tau \to 0^{+}$：差距被放大，最大 logit 取胜，分布坍缩到 $\arg\max$ 的 one-hot。纯贪心。
  • $\tau = 1$：标准 softmax。

在 LLM 采样里，温度正是这个旋钮：高 $\tau$ 输出多样、出人意料的 token；低 $\tau$ 输出确定、重复的 token。

动手试试

把上方小工具中的 τ 在 $0.05$ 到 $5$ 之间滑动：

  • τ 小 → 胜者的条形被推向 1.0
  • τ = 1 → 普通 softmax
  • τ 大 → 条形趋向 $1/3 \approx 0.333$

退火 —— 训练或采样过程中逐步降低 τ —— 是从统计物理学借来的经典操作；那里 $\tau$ 在玻尔兹曼分布中字面上 就是 温度。

5为什么注意力里要用 softmax

缩放点积注意力

$$\text{Attn}(Q,K,V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right) V.$$

对一个查询 token，$QK^\top$ 给出对每个 key 的相似度分数。Softmax 把这些分数转成 注意力权重：非负、和为 1，每个源 token 一份。

输出于是是 $\sum_j w_j v_j$ —— value 向量的 加权平均。由于 $\sum w_j = 1$，输出的幅度始终与单个 $v_j$ 可比，无论我们注意了多少 token。这正是 softmax（而不是 ReLU 归一化等）成为默认选择的原因。

为什么 softmax 之前要除以 $1/\sqrt{d_k}$？

若 query 和 key 的分量是单位方差的，$QK^\top$ 的方差正比于 $d_k$。$d_k$ 大时 logits 会变得巨大，softmax 饱和，梯度消失。除以 $\sqrt{d_k}$ 让方差保持 $\approx 1$ —— softmax 留在它敏感、可学习的区域。这本质上是温度 $\tau = \sqrt{d_k}$。

6Softmax + 交叉熵：梯度

设定

令 $p_i = \text{softmax}(z)_i$，目标 one-hot 为 $y$，交叉熵

$$L = -\sum_i y_i \log p_i.$$

我们要求 $\partial L/\partial z_i$。Softmax 的雅可比矩阵很乱：

$$\frac{\partial p_i}{\partial z_j} = p_i(\delta_{ij} - p_j).$$

但与 $L$ 复合后，一切优雅地坍缩。

坍缩

链式法则：

$$\frac{\partial L}{\partial z_i} = \sum_k \frac{\partial L}{\partial p_k}\frac{\partial p_k}{\partial z_i} = \sum_k \left(-\frac{y_k}{p_k}\right)\, p_k(\delta_{ki}-p_i).$$

化简：

$$=\;-\sum_k y_k(\delta_{ki}-p_i) \;=\; -y_i + p_i\sum_k y_k \;=\; p_i - y_i.$$

于是 $\nabla_{z}L = \mathbf{p} - \mathbf{y}$。梯度就是 误差向量：预测减目标。简洁、廉价、数值友好。这 就是 softmax + 交叉熵到处搭档的原因。

7数值稳定 —— log-sum-exp 技巧

计算 $e^{1000}$ 会让 64 位浮点数溢出；计算 $e^{-1000}$ 会下溢到 $0$，于是 $\log 0 = -\infty$。朴素 softmax 很脆弱。平移不变性救场。

令 $m = \max_i z_i$。则

$$\text{softmax}(\mathbf{z})_i = \frac{e^{z_i - m}}{\sum_j e^{z_j - m}}.$$

现在最大指数是 $e^0 = 1$；其余都在 $(0,1]$ 内。无溢出，舍入误差最小。

对 $\log\,\text{softmax}$，log-sum-exp 恒等式是规范形式：

$$\log\sum_j e^{z_j} = m + \log\sum_j e^{z_j - m}.$$

每个框架的 log_softmax、nll_loss 和注意力内核里都用了它。

8替代方案，简介

Sparsemax

把 logits 投影到概率单纯形上。输出真正 稀疏 —— 允许出现精确的零。在希望注意力完全忽略大多数 token 时很有用（可解释性、结构化预测）。不那么光滑，需要排序，但几乎处处可微。

Gumbel-softmax

先加 Gumbel 噪声，再用低温度做 softmax：从类别分布 采样 的连续可微松弛。让你能用反向传播训练离散隐变量（例如离散 VAE、硬注意力）。

其他亲戚（entmax-1.5、球面 softmax、softmax 混合……）

Entmax-α 在 softmax（α=1）和 sparsemax（α=2）之间插值。Softmax 混合解决了语言建模中的 “softmax 瓶颈” 问题。所有变体都用某种性质（稀疏性、表达力、计算）换取与原版 softmax 的差别。

第 01 章 · 每个 transformer 的舌头

字符串进，
整数出。

模型不读字符。也不读单词。它们读的是整数。 分词器是那段把原始文本翻译成 transformer 真正能消化的整数语言的小程序—— 并把它生成的整数翻译回你能读懂的文本。没有它，GPT、BERT、LLaMA、Claude—— 全都哑口无言。

一句话讲清
分词器是一个函数 $T : \text{string} \to \mathbb{Z}^n$，它把文本拆成来自固定词表的 token，并发出对应的整数 ID。反分词器 $T^{-1}$ 做相反的事。 
"hello world"
       │
       ▼  encode
[ 31373, 995 ]      # 两个 token，GPT-2 词表
       │
       ▼  decode
"hello world"
为什么不直接用单词？
  • 词表爆炸。英语有 100 万种以上的词形；模型得有一张 100 万行的嵌入表。
  • 超出词表。一个错字或新词都会变成 <UNK>，模型瞬间失明。
  • 词法。"running" / "runner" / "runs" 共享词根，但模型从未见过。
  • 语言。中文没有空格。土耳其语把词素粘在一起。按空格切就废了。
子词分词器一举解决这四点——一个约 3 万到 20 万片段的小词表，通过组合就能覆盖任何字符串。
你将遇到的三大家族

BPE 字节对编码——反复合并最频繁的相邻对。GPT-2/3/4、LLaMA、Mistral。

WordPiece 类似 BPE，但选择能最大化语料似然的合并。BERT、DistilBERT、ELECTRA。

SentencePiece 把空白当作普通字符（▁）；语言无关。内部包装 Unigram 或 BPE。T5、ALBERT、LLaMA。

接下来会发生什么
Token 变成 token ID，再变成嵌入，最终成为形状为 [batch, seq_len, d_model] 的张量。Transformer 的其他每一块—— 注意力、FFN、输出投影——都只是在变换这个张量。 分词做对了，剩下的都是矩阵乘法。

第 02 章 · 每个分词器都做的五步

分词器到底
是怎么工作的。

所有现代分词器最终都按某种顺序做相同的五件事： 归一化、预分词、用学到的合并表切分子词、 把每个片段映射成 ID，以及添加特殊 token。下面是每一步的大白话讲解。

1

归一化

转小写 / NFC Unicode / 去除重音——视模型而定。GPT-2 在这里几乎什么都不做。BERT-uncased 会激进地转小写。

"Hèllo!" → "hello!" (BERT-uncased)
2

预分词

按空白和标点切分，让合并步骤永远不跨越词边界。GPT-2 用正则；SentencePiece 不预切——它直接在原始字节（含空格）上工作。

"hello world!" → ["hello", " world", "!"]
3

应用合并表 / 词表

训练时的慢速步骤会学出一份子词词表。推理时，每个预分词被切成已知的最长片段。

"tokenization" → ["token", "ization"] (BPE)
4

把片段 → 整数 ID

词表是一个字典 str → int。每个片段成为嵌入表中对应的行索引。

["token", "ization"] → [30001, 1634]
5

添加特殊 token 与填充

BERT 在前面加 [CLS]，结尾加 [SEP]。GPT 风格的模型常常什么都不加。批量输入用 <pad> id 填充到统一长度，并由注意力掩码记录哪些位置是真实的。

[101, 30001, 1634, 102, 0, 0] mask: [1, 1, 1, 1, 0, 0]

02 60 秒讲清 BPE——合并步

字节对编码在训练时构建词表：反复找出语料中最频繁的相邻符号对， 把它们的连接作为新符号加入。跑 $V$ 次合并，词表大小就是~初始字节 + $V$。

一个微型示例
从语料 "low low low lower newer" 开始，按字符切分，并加上词边界标记 ▁： 
# 第 0 步——字节
▁ l o w   ▁ l o w   ▁ l o w   ▁ l o w e r   ▁ n e w e r

# 最频繁的对 = (l, o)  →  合并为 "lo"
▁ lo w   ▁ lo w   ▁ lo w   ▁ lo w e r   ▁ n e w e r

# 最频繁的对 = (lo, w)  →  合并为 "low"
▁ low   ▁ low   ▁ low   ▁ low e r   ▁ n e w e r

# 最频繁的对 = (e, r)  →  合并为 "er"
▁ low   ▁ low   ▁ low   ▁ low er   ▁ n e w er

# 最终词表包含：▁, l, o, w, e, r, n, lo, low, er, ...
推理按同一份合并表的顺序走，应用每一个仍然适用的合并。 这就是 BPE 确定且无损的原因——任何字节序列都有唯一的分词结果。

03 一眼看出分词结果

盯着 GPT-2 的输出看一阵后，你会注意到几个迹象：

  • 前导空格很重要。句首的 "hello" 与句中的 " hello" 是不同的 token。GPT-2 甚至在字节级词表里把前导空格打印成 Ġ。
  • 常用词 → 1 个 token。"the"、"and"、"is" 都是单 token。
  • 长词 / 罕见词 → 多个 token。"tokenization" 会拆，"antidisestablishmentarianism" 拆得更狠。
  • 数字和 emoji会被切得很碎——"2024" 可能是 1、2 或 4 个 token，取决于分词器。
  • 非拉丁文字在字节级 BPE 里会变成字节 token：一个 3 字节的 UTF-8 码点要花 3 个 token。SentencePiece 在该语言上训练后能避免这一点。

第 04 章 · 张量从哪儿来

从字符串到 logits，
八种形状一气呵成。

用同一句话——"Hello, world!"——我们追踪从字符串到 logits 的精确形状变化。 B = batch、T = 序列长度、 D = 隐藏维度、V = 词表大小。

1

原始字符串

"Hello, world!"

shape: 标量（还没张量）
2

分词 → id 列表

[15496, 11, 995, 0]（GPT-2 BPE，4 个 token）

shape: [T=4] dtype: int64
3

组批 + 填充

把两句话堆成一个 batch，用 id 0 填充到 T=6：
[[15496, 11, 995, 0, 0, 0],
 [15496, 11, 5440, 16085, 0, 0]]

input_ids: [B=2, T=6] attention_mask: [B=2, T=6]
4

嵌入查表

x = embed[input_ids]——每个 id 对应一张 [V × D] 表中的一行。

x: [B=2, T=6, D=768] embed: [V=50257, D=768]
5

+ 位置嵌入

加上 [T × D] 表（或 RoPE 旋转 Q/K）。现在位置开始有意义。

x: [B=2, T=6, D=768] pos: [T=6, D=768]
6

L 个 transformer 块

每个块都保持形状不变。内部，注意力会瞬间产生 scores: [B, H, T, T]。

x 保持: [B=2, T=6, D=768]
7

输出头 → logits

logits = x @ embed.T——每个位置上每个词表 token 都有一个分数。

logits: [B=2, T=6, V=50257]
8

softmax → 下一个 token id

argmax(logits[:, -1, :], dim=-1) 选出下一个 id；分词器再把它转回文本。

next_id: [B=2] 通过 T⁻¹ 得到"解码后的文本"

04 同样的流程，PyTorch 版

用 HuggingFace 端到端
from transformers import AutoTokenizer, AutoModelForCausalLM
import torch

tok   = AutoTokenizer.from_pretrained("gpt2")
model = AutoModelForCausalLM.from_pretrained("gpt2")

# 1. 编码 → int64 张量字典
batch = tok(["Hello, world!", "Hello, friends, again"],
            padding=True, return_tensors="pt")
print(batch.input_ids.shape)        # torch.Size([2, 6])
print(batch.attention_mask.shape)   # torch.Size([2, 6])

# 2. 前向传播——嵌入 + transformer 块
out = model(**batch)
print(out.logits.shape)            # torch.Size([2, 6, 50257])

# 3. 贪心选下一个 token = 在词表轴上 argmax
next_id = out.logits[:, -1, :].argmax(dim=-1)
print(tok.batch_decode(next_id.unsqueeze(-1)))
显存粗算。8k 上下文、batch=1、fp16 的推理，每层激活大致需要 $T \times D \times 2$ 字节，再加上 KV 缓存 $2 \times L \times T \times D \times 2$ 字节。 LLaMA-7B 在 T=8192 时 KV 缓存大约 ~512 MB——这块预算直接由分词器的压缩比决定。
第 05 章 · 子词的三大家族

BPE · WordPiece · SentencePiece。

三者都产出子词 token，三者都从频率统计开始，但合并规则和处理空白的方式各不相同。 输入一段文字，看每种是怎么切分的。

BPE — 频率合并

挑出最频繁的相邻对，合并，重复。推理时按相同顺序应用合并。贪心且无损。

使用方：GPT-2/3/4、RoBERTa、LLaMA、Mistral、Qwen。

识别特征：前导空格是 token 的一部分（打印为 Ġ 或 ▁）。

WordPiece — 似然合并

类似 BPE，但合并规则最大化语料对数似然，而不是直接看频率。 续接片段会加 ## 前缀。

使用方：BERT、DistilBERT、ELECTRA、MobileBERT。

识别特征："playing" → ["play", "##ing"]

SentencePiece — 字节神圣不可侵犯

把空白作为真实字符（▁ = U+2581），直接在原始文本上学习——不做预分词。 内部包装 Unigram-LM 或 BPE。

使用方：T5、ALBERT、XLNet、mT5、LLaMA。

识别特征：形如 ▁hello、▁world 的 token。

05 选错了会怎样？

  • 分词器不匹配是无声的灾难。用 BPE 训练，用 WordPiece 推理——每次嵌入查表都错位。模型仍能跑，但输出全是乱码。
  • 运行时新增 token是允许的（例如做 chat 模板），但你必须同步扩展嵌入表。忘了这步，遇到新 id 就崩。
  • 不同语言，价钱不同。韩语在 GPT-2 BPE 下，token 数大约是英语的 3 倍。在韩语训练的 SentencePiece 下大约是 1 倍。同样的模型，成本天壤之别。

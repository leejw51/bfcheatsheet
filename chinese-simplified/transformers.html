关卡 01 // 动机

§01为何要 transformer？

2017 年之前，主流的序列模型是 RNN 与 LSTM。它们逐 token 处理，一边把隐状态向前推。这有三个致命问题：

  • 顺序瓶颈。时间维无法并行 —— token t 必须等 t-1。
  • 长距离梯度消失。token 1 的信息要被压过每一步才能到达 token 100。
  • 固定大小的隐状态。整个过去都得塞进一个向量。长程记忆很脆弱。
关键洞见

Attention is all you need

Vaswani 等人发问：如果每个 token 都能直接看到其他每个 token，并在一步内并行完成，会怎样？没有循环，没有卷积，只有一个让每个位置以学到的权重关注每个位置的机制。

结果：训练随硬件线性扩展，梯度按 O(1) 跳数流动，模型原则上可以把段落首词与末词建立联系，就像与近邻一样轻松。

关卡 02 // 输入

§02分词与 embedding

transformer 不读文本，它读整数 ID。分词是一种有损压缩：把字符串变成来自固定词表（通常 30k–200k 项）的 token ID 序列。现代分词器（BPE、WordPiece、Unigram、Tiktoken）按子词切分，让稀有词变成多个常见片段，而不是一个未知 token。

每个整数 ID 再通过 embedding 矩阵 $E \in \mathbb{R}^{V \times d_{model}}$ 映射成稠密向量，$V$ 是词表大小。查表 = 取一行。结果：长度 $L$ 的 token 序列得到形状 [L, d_model] 的张量。

关卡 03 // 顺序

§03位置编码

自注意力是置换不变的：打乱输入 token，（不带位置信息）输出相同。这对语言而言是问题。修复办法：在喂入堆栈前给每个 token embedding 加一个与位置相关的向量。

原始论文使用波长按几何级数增长的正弦/余弦：

$$PE_{(pos,\,2i)} = \sin\!\Big(\tfrac{pos}{10000^{2i/d_{model}}}\Big), \qquad PE_{(pos,\,2i+1)} = \cos\!\Big(\tfrac{pos}{10000^{2i/d_{model}}}\Big)$$

可视化

正弦位置波

每行是一个维度 i。低维振荡快（位置之间区分明显）；高维平滑（粗粒度顺序）。

巧妙之处：$PE_{pos+k}$ 是 $PE_{pos}$ 的固定线性函数，模型很容易学到相对偏移。现代变体（RoPE、ALiBi、可学习绝对位置）权衡不同，但目标一致 —— 注入顺序信息。

关卡 04 // 核心

§04自注意力（Q · K · V）

每个输入 token 通过三组学习矩阵 $W^Q, W^K, W^V \in \mathbb{R}^{d_{model} \times d_k}$ 投影成三个向量：

  • Query (Q) — 「我在找什么？」
  • Key (K) — 「我里头有什么？」
  • Value (V) — 「若被关注，我提供什么？」

缩放点积注意力为：

$$\mathrm{Attention}(Q,K,V) = \mathrm{softmax}\!\Big(\tfrac{QK^\top}{\sqrt{d_k}}\Big)\,V$$

序列长 $L$、头维 $d_k$ 时的形状：Q: [L, d_k] K: [L, d_k] V: [L, d_k]。$QK^\top$ 形状 [L, L] —— 每对 (query, key) 一个相似度分数。沿行做 softmax 变成注意力权重，再用它给 $V$ 加权。

关卡 05 // 集成

§05多头注意力

单一注意力模式往往不够。多头注意力并行运行 $h$ 次注意力，每次有自己的 $W_i^Q, W_i^K, W_i^V$ 投影，再把结果拼接：

$$\mathrm{MHA}(X) = \mathrm{Concat}(\mathrm{head}_1, \ldots, \mathrm{head}_h)\,W^O$$

不同的头学到不同的关系模式 —— 一个头跟踪句法依存，另一个跟踪指代，另一个跟踪简单的位置偏移。拆分让总计算量大致与单头持平：$h=8$ 头时取 $d_k = d_{model}/h$。

level 06 // glue

§06前馈 · 残差 · 归一化

注意力之后，每个位置过一个逐位置的前馈网络 —— 两个线性层中间夹一个非线性（最初是 ReLU，今天通常是 GELU 或 SwiGLU）：

$$\mathrm{FFN}(x) = \max(0,\, xW_1+b_1)\,W_2 + b_2$$

隐藏宽度通常为 4·d_model。模型大部分参数都驻留在这里 —— 一个流行假设是该层充当存储事实知识的 key-value 记忆。

残差连接

注意力子层与 FFN 子层都被裹在残差连接 + 层归一化里：

$$x \leftarrow \mathrm{LayerNorm}(x + \mathrm{Sublayer}(x))$$

为何？残差为梯度从损失到输入开了一条高速公路，让数百层稳定训练。LayerNorm 在特征维上让激活居中且缩放，把每层的优化彼此解耦。

现代变体偏好pre-norm：$x \leftarrow x + \mathrm{Sublayer}(\mathrm{LayerNorm}(x))$ —— 在深层下训练更稳。

关卡 07 // 拓扑

§07编码器 · 解码器 · 掩码

当今主流的三种架构：

  • 仅编码器（BERT）— 双向自注意力。每个 token 都看到其他每个 token。理解类任务（分类、检索）很强。
  • 仅解码器（GPT）— 因果/掩码自注意力。token t 只能看到位置 $\le t$ 的 token。生成很强。这是当今多数 LLM 的形态。
  • 编码器-解码器（T5、原始 NMT）— 编码器读源序列，解码器生成目标序列同时跨注意力到编码器输出。

因果掩码 — 怎么做？

在 softmax 之前，把分数矩阵的上三角设为 $-\infty$。$\mathrm{softmax}(-\infty) = 0$ —— 模型在数学上无法看到未来：

关卡 08 // 目标

§08训练：下一 token 预测

对仅解码器 LLM（GPT 家族），整个训练目标简单到惊人：

给定 token 1..t，预测 token t+1。

模型在每个位置输出词表上的概率分布。损失是对真实下一 token 的平均交叉熵：

$$\mathcal{L} = -\frac{1}{N}\sum_{t=1}^{N} \log P_\theta(x_{t+1} \mid x_1, \ldots, x_t)$$

因果掩码让这个目标很高效：长 $N$ 的序列一次前向，并行得到 $N$ 个预测，每个位置一个。除了把标签整体右移一位，不需要其他 teacher forcing 技巧。

其他一切 —— 指令微调、RLHF、工具使用 —— 都建立在这个目标之上。预训练吞下互联网规模的数据并让模型学到「一切」；微调塑造其行为。

回顾

完整管道

  • 1. token → 整数 ID → embedding（+ 位置编码）
  • 2. N 个 transformer 块的堆叠：MHA → Add&Norm → FFN → Add&Norm
  • 3. 最终线性投影到词表大小，softmax → 下一 token 分布
  • 4. 与真实下一 token 的交叉熵损失，按位置求和
  • 5. 反传到每个权重。重复约 10²² 次。完事。

一個張量，一路到 logits

Attention Is All You Need Transformer 是一道食譜：取一個來源詞元 ID 的張量， 送入編碼器，再將目標詞元送入交叉注意編碼器輸出的解碼器，投影到詞彙表， 最終輸出下一個詞元的 logits。 本頁追蹤形狀——從輸入 ID 到最終 [B, T_tgt, V] 機率的每個階段，以顏色編碼，讓你閱讀論文時不再迷失張量。

論文預設值（base 版本）
B (batch) = 2 # 批次大小
T_src (src len) = 8 # 來源詞元數
T_tgt (tgt len) = 7 # 目標詞元數
d_model = 512 # 嵌入維度
h (heads) = 8 # 注意力頭數
d_k = d_v = 64 # d_model / h
d_ff = 2048# FFN 內部維度（4·d_model）
V (vocab) = 37000# 英→德 BPE 詞彙量
N (layers) = 6 # 編碼器層數 = 解碼器層數
顏色圖例（全頁通用）
B 批次 T 詞元 d_model h 頭數 d_k d_ff

當維度在各階段之間改變時，它會發光。觀察同一個張量如何被分割成各頭、相乘、縮放、softmax，再拼接回一起。

第 1–3 階段 · 從詞元 ID 到可供注意力使用的張量

在任何注意力操作之前，輸入必須先轉換為向量張量。三個快速階段： 整數 ID 輸入，實數值向量輸出，位置資訊透過加法嵌入其中。

第 1 階段 · 詞元 ID
分詞器輸出。批次中兩個範例，各八個詞元。
$\text{ids}\in\mathbb{Z}^{B\times T}$
shape [B=2, T=8] · dtype: int32 
# 兩個句子，各填充/截斷至 T=8 個詞元
ids = [[ 101, 2057, 2293, 3019, 2025, 2748, 102, 0 ],
       [ 101, 7592, 2088,  102,    0,    0,   0, 0 ]]
# 形狀: [B=2, T=8]
第 2 階段 · 嵌入查找

權重矩陣 $E\in\mathbb{R}^{V\times d_{model}}$ 中每個詞彙項目佔一行。 按 ID 查找。張量增加一個維度：每個整數被替換為大小 d_model 的實數值向量。

$X = E[\text{ids}] \;\in\; \mathbb{R}^{B\times T\times d_{model}}$
[2, 8] → [2, 8, 512] 
import torch.nn as nn
embed = nn.Embedding(num_embeddings=30522, embedding_dim=512)
X = embed(ids)            # [B, T] -> [B, T, d_model]
print(X.shape)            # torch.Size([2, 8, 512])

參數量：$V \cdot d_{model} = 30522 \cdot 512 \approx 15.6\text{M}$ — 僅嵌入表就比任何單一層的其他部分都大。

第 3 階段 · + 位置編碼

自注意力具有排列等變性——沒有位置資訊時，"the cat sat" 和 "sat cat the" 會產生相同的輸出。論文添加了一個與 $X$ 形狀相同的固定正弦模式。

$\mathrm{PE}_{(pos, 2i)} = \sin\!\big(pos / 10000^{2i/d_{model}}\big)$
$\mathrm{PE}_{(pos, 2i+1)} = \cos\!\big(pos / 10000^{2i/d_{model}}\big)$
[2, 8, 512] + [8, 512] → [2, 8, 512] 
# PE 的形狀為 [T, d_model]，在 B 上廣播
X = X + PE[:T]            # [B, T, d_model] + [T, d_model] -> [B, T, d_model]

輸入與輸出形狀相同。資訊進入了值，而不是維度。

第 3 階段結束：我們現在有一個形狀為 [B, T, d_model] 的張量，它同時包含了詞語的語義和位置資訊。後續每個階段都是對這種佈局的變換。

第 4–11 階段 · 多頭注意力，逐維剖析

這正是這篇論文存在的原因。每個詞元成為查詢（query）、鍵（key）和值（value）； 計算相似度，縮放，softmax，再用來混合值。我們還需重塑形狀兩次：一次拆分多頭，一次重新合併。

第 4 階段 · Q、K、V 投影

三個獨立的線性映射，每個 $\mathbb{R}^{d_{model}}\to\mathbb{R}^{d_{model}}$：

$Q = X W^Q,\quad K = X W^K,\quad V = X W^V$
X [2, 8, 512] Wᵠ [512, 512] → Q [2, 8, 512] 
# 三個線性層，論文中無偏置，但實踐中有偏置也可以
Wq = nn.Linear(d_model, d_model)   # 512*512 = 262 144 參數
Wk = nn.Linear(d_model, d_model)
Wv = nn.Linear(d_model, d_model)

Q = Wq(X)                              # [B, T, d_model]
K = Wk(X)                              # [B, T, d_model]
V = Wv(X)                              # [B, T, d_model]

三個形狀為 $[d_{model}, d_{model}]$ 的矩陣 — 此處總參數量： $3\cdot 512\cdot 512 = 786\,432$。在某些實作中，這三個矩陣融合成一個 $[d_{model}, 3\cdot d_{model}]$ 矩陣。

第 5 階段 · 為 h 個頭重塑並轉置

將最後一個維度切分為 $h$ 個大小為 $d_k = d_{model}/h$ 的塊，再轉置使各頭成為外層維度。 無新參數——僅是 view + permute 操作。

[2, 8, 512] view→ [2, 8, 8, 64] transpose(1,2)→ [2, 8, 8, 64] 
B, T, d_model = Q.shape
h, d_k = 8, d_model // h         # 8, 64

# 將最後一個維度拆分為 (h, d_k)，然後將各頭移到緊靠批次的位置
Q = Q.view(B, T, h, d_k).transpose(1, 2)   # [B, h, T, d_k]
K = K.view(B, T, h, d_k).transpose(1, 2)   # [B, h, T, d_k]
V = V.view(B, T, h, d_k).transpose(1, 2)   # [B, h, T, d_k]
為何多頭很重要：有了 $h=8$ 個獨立的 $[T, d_k=64]$ 子空間，每個頭可專精於不同的關聯模式。 計算量不變：8 個小型 $64$ 維注意力，而非一個 $512$ 維的。
第 6 階段 · Q @ Kᵀ — 成對相似度

對每個頭，每個查詢都關注每個鍵。矩陣乘法收縮 $d_k$ 軸， 每個批次每個頭產生一個 $T\times T$ 的分數矩陣。

$\text{scores} = Q K^\top$
Q [2, 8, 8, 64] Kᵀ [2, 8, 64, 8] → scores [2, 8, 8, 8] 
scores = Q @ K.transpose(-2, -1)   # [B, h, T, T]
# 元素 (b, head, i, j) = 詞元 i 在該頭中對詞元 j 的注意力程度
第 7 階段 · 除以 1/√d_k 縮放

不縮放的話，$d_k$ 維向量的點積會以 $\sqrt{d_k}$ 的速度增長——過大的值會使 softmax 進入飽和區，導致梯度消失。將每個分數除以 $\sqrt{d_k}$。

$\text{scaled} = \text{scores} / \sqrt{d_k} = \text{scores} / 8$
[2, 8, 8, 8] ÷ √64 [2, 8, 8, 8] 形狀不變 
import math
scores = scores / math.sqrt(d_k)   # 1/√64 = 1/8 = 0.125
第 8 階段 · 沿最後一個軸做 softmax

將原始分數轉換為每個查詢的機率分布。$[T,T]$ 塊的每行和為 1——這些就是注意力權重。

$A = \mathrm{softmax}(\text{scaled},\ \text{dim}=-1)$
[2, 8, 8, 8] → [2, 8, 8, 8] 各行和為 1 
import torch.nn.functional as F
A = F.softmax(scores, dim=-1)    # [B, h, T, T]

對於解碼器自注意力，在 softmax 之前施加一個加法 $-\infty$ 遮罩來強制因果性。

第 9 階段 · 注意力 @ V

用注意力權重混合值向量。每個查詢的輸出是所有 $T$ 個值向量的加權和。收縮軸是內部的 $T$。

A [2, 8, 8, 8] V [2, 8, 8, 64] → out [2, 8, 8, 64] 
head_out = A @ V                  # [B, h, T, d_k]
第 10 階段 · 拼接各頭（轉置 + 重塑）

將各頭重新合併為一個大的最後一個維度。transpose(1,2) 將頭移至 $d_k$ 旁， 再用 reshape 合併。無參數操作。

[2, 8, 8, 64] transpose→ [2, 8, 8, 64] reshape→ [2, 8, 512] 
out = head_out.transpose(1, 2).contiguous()   # [B, T, h, d_k]
out = out.reshape(B, T, d_model)              # [B, T, d_model]
第 11 階段 · 輸出投影 W_O

再一個 $\mathbb{R}^{d_{model}\to d_{model}}$ 線性映射。形狀相同，但讓模型在拼接後跨頭混合資訊。

$\text{MHA}(X) = \mathrm{Concat}(\text{head}_1, \dots, \text{head}_h)\,W^O$
[2, 8, 512] ·W_O [2, 8, 512] 
Wo = nn.Linear(d_model, d_model)   # 另外 262 144 個參數
mha_out = Wo(out)                       # [B, T, d_model]

MHA 總參數量： $4 \cdot d_{model}^2 = 4\cdot 512^2 = 1{,}048{,}576$ （Q、K、V、O — 每個都是 $d_{model}\times d_{model}$）。

注意力結束：我們離開時的形狀為 [B, T, d_model] ——與進入時完全相同的形狀。注意力是一個形狀不變的算子。

第 12–16 階段 · LayerNorm、FFN 與殘差公路

注意力之後，編碼器區塊用兩個殘差連接、兩個 LayerNorm 和一個逐位置 FFN 把一切縫合在一起。 FFN 是大多數參數的所在——也是許多人認為事實知識儲存的地方。

第 12 階段 · 殘差 + LayerNorm

加回原始 $X$（跳躍連接），然後在 $d_{model}$ 上對每個詞元進行正規化。

$X' = \mathrm{LayerNorm}(X + \mathrm{MHA}(X))$
[2, 8, 512] + [2, 8, 512] → LN → [2, 8, 512] 
ln1 = nn.LayerNorm(d_model)      # 2 * d_model = 1024 個參數 (γ, β)
X_post = ln1(X + mha_out)         # [B, T, d_model]

LN 沿最後一個軸正規化：每個 $[d_{model}]$ 向量變為均值 0、方差 1，再由學習的 $\gamma, \beta\in\mathbb{R}^{d_{model}}$ 重新縮放。批次和詞元位置保持獨立。

第 13–14 階段 · FFN 線性層 1 + ReLU

逐位置前饋網路：對每個詞元獨立應用相同的 MLP。 第一個線性層將 $d_{model} \to d_{ff}$（通常為 $4\cdot d_{model}$）。

$H = \mathrm{ReLU}(X' W_1 + b_1)$
[2, 8, 512] ·W₁ [2, 8, 2048] → ReLU → [2, 8, 2048] 
W1 = nn.Linear(d_model, d_ff)       # 512 * 2048 = 1 048 576 個參數
H = F.relu(W1(X_post))            # [B, T, d_ff]
第 15 階段 · FFN 線性層 2

第二個線性層投影回去：$d_{ff} \to d_{model}$。恢復標準形狀，讓下一個區塊可以重新開始。

$Y = H W_2 + b_2$
[2, 8, 2048] ·W₂ [2, 8, 512] 
W2 = nn.Linear(d_ff, d_model)       # 2048 * 512 = 另外 1 048 576 個參數
ffn_out = W2(H)                       # [B, T, d_model]

FFN 總參數量： $2\cdot d_{model}\cdot d_{ff} = 2\cdot 512\cdot 2048 = 2{,}097{,}152$ — 大約是四個注意力投影總和的兩倍。這是大型模型集中參數的地方。

第 16 階段 · 殘差 + LayerNorm（再次）

與第 12 階段相同的模式。加上 FFN 前的張量，層歸一化，編碼器區塊完成。

$Y' = \mathrm{LayerNorm}(X' + Y)$
[2, 8, 512] → [2, 8, 512] 區塊輸出形狀 = 區塊輸入形狀 
ln2 = nn.LayerNorm(d_model)
out = ln2(X_post + ffn_out)       # [B, T, d_model]
堆疊與重複：論文堆疊了 $N=6$ 個相同的編碼器區塊。每個區塊的輸入形狀等於輸出形狀， 這就是它們可以整潔組合的原因。 一個區塊的編碼器總參數量（base 版本）：大約 $4\cdot 512^2 + 2\cdot 512\cdot 2048 + 4\cdot 512 \approx 3.15$M。

整個編碼器區塊，14 行搞定

EncoderBlock — 最簡 PyTorch 實作
class EncoderBlock(nn.Module):
    def __init__(self, d_model=512, h=8, d_ff=2048):
        super().__init__()
        self.attn = nn.MultiheadAttention(d_model, h, batch_first=True)
        self.ffn  = nn.Sequential(
            nn.Linear(d_model, d_ff), nn.ReLU(), nn.Linear(d_ff, d_model)
        )
        self.ln1 = nn.LayerNorm(d_model)
        self.ln2 = nn.LayerNorm(d_model)

    def forward(self, x):
        # x: [B, T, d_model]
        a, _ = self.attn(x, x, x)
        x = self.ln1(x + a)
        x = self.ln2(x + self.ffn(x))
        return x                       # [B, T, d_model]

第 17–21 階段 · 解碼器區塊

解碼器接收目標詞元（右移一位），產生一個張量，包含關於 (a) 迄今所見目標詞元的資訊——透過遮罩自注意力， 以及 (b) 來源的資訊——透過交叉注意力，其中 Q 來自解碼器，K、V 來自編碼器輸出（「memory」）。 與編碼器相同的嵌入維度、相同的殘差+LN 規範。每層三個子區塊，論文中共六層。

論文設定（沿用）
B = 2, T_src = 8, T_tgt = 7
d_model = 512, h = 8, d_k = 64
# T_tgt 與 T_src 相互獨立。
# 交叉注意力是它們唯一相遇的地方。
為什麼要「右移」？

訓練期間解碼器看到前置 <sos> 的黃金目標， 並預測以 <eos> 結尾的相同目標。

# target  : "die katze sitzt"           ← 真實答案
# dec_in  : "<sos> die katze sitzt"     ← 右移（輸入）
# labels  : "die katze sitzt <eos>"     ← 我們要預測的

第 17 階段 · 解碼器輸入 → 嵌入

[B, T_tgt] → 嵌入查找 → [B, T_tgt, d_model] = [2, 7, 512]

第 19 階段 · 遮罩自注意力 （因果遮罩）

與編碼器自注意力相同——除了分數矩陣對角線上方的每個位置都被蓋上 −∞， 使位置 i 永遠無法看到 i+1, i+2, ...。softmax 後這些位置恰好變為 0。

# scores: [B, h, T_tgt, T_tgt] = [2, 8, 7, 7]
mask = torch.triu(torch.ones(T_tgt, T_tgt), diagonal=1).bool()
scores = scores.masked_fill(mask, float('-inf'))
A = F.softmax(scores, dim=-1)        # 下三角，各行和為 1
[B, h, T_tgt, T_tgt] = [2, 8, 7, 7] softmax 後上三角 = 0

第 20 階段 · 交叉注意力 （Q ← 解碼器，K、V ← 編碼器）

這是編碼器最終與解碼器交流的地方。解碼器發出查詢；編碼器輸出提供鍵和值。

# dec_x: [B, T_tgt, d_model] = [2, 7, 512]   ← 來自遮罩自注意力
# memory: [B, T_src, d_model] = [2, 8, 512]  ← 編碼器輸出（第 16 階段）
Q = W_q(dec_x).view(B, T_tgt, h, d_k).transpose(1, 2)   # [B, h, T_tgt, d_k]
K = W_k(memory).view(B, T_src, h, d_k).transpose(1, 2)   # [B, h, T_src, d_k]
V = W_v(memory).view(B, T_src, h, d_k).transpose(1, 2)   # [B, h, T_src, d_k]
scores = Q @ K.transpose(-2, -1) / math.sqrt(d_k)         # [B, h, T_tgt, T_src] = [2, 8, 7, 8]
A = F.softmax(scores, dim=-1)                              # 最後一個軸是 T_src
out = (A @ V).transpose(1, 2).reshape(B, T_tgt, d_model) # [2, 7, 512]

關鍵點：分數矩陣不是方形的。它是 [B, h, T_tgt, T_src]——每個目標詞元對每個來源詞元都有注意力。

解碼器區塊，16 行搞定

DecoderBlock — 最簡 PyTorch 實作
class DecoderBlock(nn.Module):
    def __init__(self, d=512, h=8, d_ff=2048):
        super().__init__()
        self.self_attn  = nn.MultiheadAttention(d, h, batch_first=True)
        self.cross_attn = nn.MultiheadAttention(d, h, batch_first=True)
        self.ffn = nn.Sequential(nn.Linear(d, d_ff), nn.ReLU(), nn.Linear(d_ff, d))
        self.ln1, self.ln2, self.ln3 = nn.LayerNorm(d), nn.LayerNorm(d), nn.LayerNorm(d)

    def forward(self, x, memory, causal_mask):
        # x:      [B, T_tgt, d]      memory: [B, T_src, d]
        a, _ = self.self_attn(x, x, x, attn_mask=causal_mask)
        x = self.ln1(x + a)                 # [B, T_tgt, d]
        c, _ = self.cross_attn(x, memory, memory)
        x = self.ln2(x + c)                 # [B, T_tgt, d]
        x = self.ln3(x + self.ffn(x))
        return x                            # [B, T_tgt, d]

解碼器堆疊的輸出（6 層之後）：[B, T_tgt, d_model] = [2, 7, 512]。下一站：投影到詞彙表。

第 22–23 階段 · 從隱藏狀態到 logits

一個 nn.Linear(d_model, V) 將每個詞元的隱藏向量轉換為詞彙表中每個詞的分數。 沿最後一個軸的 softmax 將這些分數轉換為機率。這就是整個輸出頭。 論文中，W_lm 與輸入嵌入矩陣共享參數——相同的參數，轉置使用。

第 22 階段 · 線性投影到詞彙表

[B, T_tgt, d_model] @ W_lm.T →（矩陣乘積） [B, T_tgt, V] = [2, 7, 37000] 
# dec_out: [B, T_tgt, d_model]
# W_lm:    [V, d_model]   ← 通常與 embed.weight 共享
logits = dec_out @ W_lm.T           # [B, T_tgt, V]
# 或更簡單地：
logits = self.lm_head(dec_out)      # nn.Linear(d_model, V)

這些是原始分數——每個詞彙詞元、每個位置、每個範例各一個。 還不是機率。如果你的損失是 CrossEntropyLoss，不要對它們做 softmax； 它在內部出於數值穩定性會自行處理。

第 23 階段 · Softmax → 機率 （僅在真正需要時）

probs = F.softmax(logits, dim=-1)   # [B, T_tgt, V] — V 上各行和為 1

在 B·T_tgt 個位置的每一個，你現在擁有整個詞彙表的分布。 那 37,000 個數字中的大多數都很微小。少數幾個集中在正確答案附近。

輸出頭的參數量

僅 W_lm 就有 V × d_model = 37000 × 512 ≈ 19M 個參數—— 通常是模型中最大的單一權重。論文將其與輸入嵌入矩陣共享，使權重記憶體減半， 且（實證上）有助於泛化。

完整的前向傳播，端到端

Transformer.forward — 忠於論文
def forward(self, src_ids, tgt_in_ids):
    # src_ids:    [B, T_src]   tgt_in_ids: [B, T_tgt]
    src = self.src_embed(src_ids) + self.pos(src_ids)        # [B, T_src, d]
    tgt = self.tgt_embed(tgt_in_ids) + self.pos(tgt_in_ids)    # [B, T_tgt, d]

    memory = src
    for layer in self.encoder_layers:
        memory = layer(memory)                                  # [B, T_src, d]

    causal = self.causal_mask(tgt_in_ids.shape[1])
    x = tgt
    for layer in self.decoder_layers:
        x = layer(x, memory, causal)                            # [B, T_tgt, d]

    logits = self.lm_head(x)                                     # [B, T_tgt, V]
    return logits

def train_step(model, optimizer, batch):
    # batch: list[(src_str, tgt_str)]，長度為 B
    src_ids = tokenize_src(batch)         # [B, T_src]    int64
    dec_in, labels = tokenize_tgt(batch)  # [B, T_tgt] 各一

    src = model.src_embed(src_ids)*math.sqrt(d) + model.pos(src_ids)
    tgt = model.tgt_embed(dec_in)*math.sqrt(d) + model.pos(dec_in)

    memory = src
    for layer in model.encoder_layers: memory = layer(memory)

    causal = model.causal_mask(dec_in.shape[1])
    x = tgt
    for layer in model.decoder_layers: x = layer(x, memory, causal)

    logits = model.lm_head(x)              # [B, T_tgt, V]
    loss = F.cross_entropy(
        logits.view(-1, V), labels.view(-1),
        ignore_index=-100, label_smoothing=0.1)

    optimizer.zero_grad(); loss.backward(); optimizer.step()
    return loss.item()                     # 純量

@torch.no_grad()
def generate(model, src_text, max_len=32):
    src_ids = tokenize_src([(src_text, "")])      # [1, T_src]

    src = model.src_embed(src_ids)*math.sqrt(d) + model.pos(src_ids)
    memory = src
    for layer in model.encoder_layers:
        memory = layer(memory)                          # [1, T_src, d]

    dec_in = torch.tensor([[SOS]])                         # [1, 1]
    for _ in range(max_len):
        tgt = model.tgt_embed(dec_in)*math.sqrt(d) + model.pos(dec_in)
        causal = model.causal_mask(dec_in.shape[1])
        x = tgt
        for layer in model.decoder_layers:
            x = layer(x, memory, causal)                 # [1, t, d]
        logits = model.lm_head(x)                          # [1, t, V]
        next_id = logits[:, -1, :].argmax(-1, keepdim=True)
        if next_id.item() == EOS: break
        dec_in = torch.cat([dec_in, next_id], dim=1)    # [1, t+1]

    return detokenize(dec_in[0, 1:].tolist())          # str

推論期間的形狀摘要： 原始字符串 → [1, T_src] → [1, T_src, d]（memory，一次）→ 循環：[1, t] → [1, t, d] → [1, t, V] → [1, V] → [1] → 附加 → 增長 → 最終字符串。

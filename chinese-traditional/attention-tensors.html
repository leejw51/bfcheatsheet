01 Attention Is All You Need · 張量形狀

一個張量,一路走到 logits

Attention Is All You Need 裡的 Transformer 就是一套配方:拿一個源 token ID 張量, 送過編碼器(encoder),把目標 token 送過一個對編碼器做交叉注意力(cross-attention)的 解碼器(decoder),投影到詞表,最後吐出下一個 token 的 logits。 本頁追蹤形狀——從輸入 ID 到最終 [B, S_tgt, V] 概率的每一個階段,按顏色編碼,讓你讀論文時永遠不會跟丟這個張量。

論文默認配置(base)
B (batch) = 2 # 批大小
S_src (src len) = 10 # 源 token 數
S_tgt (tgt len) = 7 # 目標 token 數
d_model = 512 # 嵌入維度
h (heads) = 8 # 注意力頭數
d_k = d_v = 64 # d_model / h
d_ff = 2048# FFN 內部維度(4·d_model)
V (vocab) = 37000# 英→德 BPE 詞表
N (layers) = 6 # 編碼器層數 = 解碼器層數
顏色圖例(全頁通用)

02 輸入與嵌入

從 token ID 到一個準備好上下文的張量

在做任何注意力之前,輸入必須先變成一個向量張量。三個快速階段: 輸入整數 ID,輸出實數向量,並通過相加把位置信息烘焙進去。

階段 1 · token ID
分詞器的輸出。一個批裡有兩條樣本,每條 8 個 token。
$\text{ids}\in\mathbb{Z}^{B\times S}$
形狀 [B=2, S=8] · dtype: int32 
# 兩個句子,各填充/截斷至 S=8 個 token
ids = [[ 101, 2057, 2293, 3019, 2025, 2748, 102, 0 ],
       [ 101, 7592, 2088,  102,    0,    0,   0, 0 ]]
# 形狀:[B=2, S=8]
階段 2 · 嵌入查表

權重矩陣 $E\in\mathbb{R}^{V\times d_{model}}$ 為詞表裡每個條目保存一行。 按 ID 查表。張量多出一維:每個整數被一個大小為 d_model 的實數向量替換。

$X = E[\text{ids}] \;\in\; \mathbb{R}^{B\times S\times d_{model}}$
[2, 10] → [2, 10, 512] 
import torch.nn as nn
embed = nn.Embedding(num_embeddings=30522, embedding_dim=512)
X = embed(ids)            # [B, S] -> [B, S, d_model]
print(X.shape)            # torch.Size([2, 10, 512])

參數量:$V \cdot d_{model} = 30522 \cdot 512 \approx 15.6\text{M}$——光這張嵌入表 就比任何單層的其餘部分都大。

階段 3 · + 位置編碼

自注意力對位置是等變的(permutation-equivariant)——沒有位置信息時,"the cat sat" 和 "sat cat the" 會產生相同的輸出。論文加上了一個與 $X$ 同形狀的固定正弦模式。

$\mathrm{PE}_{(pos, 2i)} = \sin\!\big(pos / 10000^{2i/d_{model}}\big)$
$\mathrm{PE}_{(pos, 2i+1)} = \cos\!\big(pos / 10000^{2i/d_{model}}\big)$
[2, 10, 512] + [10, 512] → [2, 10, 512] 
# PE 形狀為 [S, d_model],沿 B 廣播
X = X + PE[:S]            # [B, S, d_model] + [S, d_model] -> [B, S, d_model]

進去什麼形狀,出來還是什麼形狀。信息進的是數值,不是維度。

階段 3 結束: 現在我們有一個形狀為 [B, S, d_model] 的張量,它既關於這些詞,也關於它們在哪裡。之後的每個階段 都是在這個佈局上做變換。

03 多頭注意力

階段 4–11 · 注意力,逐維拆解

這篇論文存在的全部理由。每個 token 都變成一個查詢(query)、鍵(key)和值(value); 計算相似度、縮放、做 softmax,再用來混合這些值。我們還會 reshape 兩次: 一次拆分多頭,一次再把它們拼回去。

為什麼現在每個形狀都一目瞭然: 這裡 h = 8 個頭、 S = 10 個序列位置、d_k = 64——每一維都有各自獨特的數字, 所以你總能分清哪個軸是哪個。(早期版本用序列長度 8,正好和 8 個頭撞上: [2, 8, 8, 64] 根本看不清。換成 S=10 後變成 [2, 8, 10, 64]——清清楚楚。)

把鼠標懸停或點擊某個頭,聚焦那個頭的 S×d_k 切片。 同一份數據,重新分組——沒有新參數。

每個單元格 (i, j) = 查詢位置 i 對鍵位置 j 投入多少注意力。 每一行都是一個和為 1 的 softmax。選一個頭——你看到的是 h = 8 張獨立的 10×10 圖之一。(權重僅作示意。)

階段 4 · Q、K、V 投影

三個獨立的線性映射,每個都是 $\mathbb{R}^{d_{model}}\to\mathbb{R}^{d_{model}}$:

$Q = X W^Q,\quad K = X W^K,\quad V = X W^V$
X [2, 10, 512] Wᵠ [512, 512] → Q [2, 10, 512] 
# 三個線性層,論文中無偏置,實際使用時加偏置也可以
Wq = nn.Linear(d_model, d_model)   # 512*512 = 262 144 個參數
Wk = nn.Linear(d_model, d_model)
Wv = nn.Linear(d_model, d_model)

Q = Wq(X)                              # [B, S, d_model]
K = Wk(X)                              # [B, S, d_model]
V = Wv(X)                              # [B, S, d_model]

三個形狀為 $[d_{model}, d_{model}]$ 的矩陣——這裡的總參數量: $3\cdot 512\cdot 512 = 786\,432$。某些實現會把這三個融合成一個 $[d_{model}, 3\cdot d_{model}]$ 的矩陣。

階段 5 · 為 h 個頭做 reshape 與 transpose

把最後一維切成 $h$ 塊,每塊 $d_k = d_{model}/h$,然後 transpose,讓頭變成更外層的一維。 沒有新參數——只是 view + permute。

[2, 10, 512] view→ [2, 10, 8, 64] transpose(1,2)→ [2, 8, 10, 64] 
B, S, d_model = Q.shape
h, d_k = 8, d_model // h         # 8, 64

# 將最後一維拆成 (h, d_k),然後把頭移到緊鄰批次的位置
Q = Q.view(B, S, h, d_k).transpose(1, 2)   # [B, h, S, d_k]
K = K.view(B, S, h, d_k).transpose(1, 2)   # [B, h, S, d_k]
V = V.view(B, S, h, d_k).transpose(1, 2)   # [B, h, S, d_k]
為什麼頭很重要: 有了 $h=8$ 個獨立的 $[S, d_k=64]$ 子空間,每個頭 都能專門負責一種不同的關係模式。計算量不變:8 個小的 $64$ 維注意力,而不是一個 $512$ 維的。
階段 6 · Q @ Kᵀ —— 兩兩相似度

對每個頭,每個查詢都關注每個鍵。這次矩陣乘會收縮 $d_k$ 軸, 為每個頭、每個 batch 產生一個 $S\times S$ 的分數矩陣。

$\text{scores} = Q K^\top$
Q [2, 8, 10, 64] Kᵀ [2, 8, 64, 10] → scores [2, 8, 10, 10] 
scores = Q @ K.transpose(-2, -1)   # [B, h, S, S]
# 元素 (b, head, i, j) = token i 對 token j 的關注程度(在該頭中)
階段 7 · 按 1/√d_k 縮放

不縮放的話,$d_k$ 維向量的點積會以 $\sqrt{d_k}$ 的量級增長——過大的值會把 softmax 推入飽和區,梯度隨之消失。把每個分數都除以 $\sqrt{d_k}$。

$\text{scaled} = \text{scores} / \sqrt{d_k} = \text{scores} / 8$
[2, 8, 10, 10] ÷ √64 [2, 8, 10, 10] 形狀不變 
import math
scores = scores / math.sqrt(d_k)   # 1/√64 = 1/8 = 0.125
階段 8 · 沿最後一維做 softmax

把原始分數變成每個查詢的一個概率分佈。$[S,S]$ 塊的每一行 都和為 1——這些就是注意力權重。

$A = \mathrm{softmax}(\text{scaled},\ \text{dim}=-1)$
[2, 8, 10, 10] → [2, 8, 10, 10] 每行和為 1 
import torch.nn.functional as F
A = F.softmax(scores, dim=-1)    # [B, h, S, S]

對解碼器自注意力,會在 softmax 之前加上一個 $-\infty$ 加性掩碼來強制因果性。

階段 9 · attention @ V

用注意力權重去混合值向量。每個查詢的輸出都是全部 $S$ 個值向量的加權和。 被收縮的是內側的 $S$ 軸。

A [2, 8, 10, 10] V [2, 8, 10, 64] → out [2, 8, 10, 64] 
head_out = A @ V                  # [B, h, S, d_k]
階段 10 · 拼接多頭(transpose + reshape)

把多個頭粘回成一個大的最後一維。transpose(1,2) 把頭移到緊挨 $d_k$ 的位置, 然後 reshape 把它們合併。無參數。

[2, 8, 10, 64] transpose→ [2, 10, 8, 64] reshape→ [2, 10, 512] 
out = head_out.transpose(1, 2).contiguous()   # [B, S, h, d_k] —— 此時各頭相鄰
out = out.reshape(B, S, d_model)              # reshape 把 8 個頭首尾相接地合併 = Concat(head_0..head_7)
這個 reshape 就是拼接(concat)。 論文寫的是 $\mathrm{MHA}(Q,K,V) = \mathrm{Concat}(\text{head}_1,\dots,\text{head}_h)\,W^O$,但這裡 並沒有真正的 torch.cat。做完 transpose(1,2) 後,對每一個 S = 10 的 token 位置, h = 8 個頭在最後兩維裡彼此相鄰: [2, 10, 8, 64]。 reshape 隨後把這最後兩維拍平 ($h\cdot d_k = 8\cdot 64 = $ 512),把各頭沿特徵維首尾相接地排開。 這種在連續內存上的並排佈局正是拼接——所以 concat 並沒有被省略,它恰恰就是這個 reshape 在做的事 (.contiguous() 只是讓 transpose 之後的 reshape 合法)。每個 token 的 512 維向量是:
[512] = [head₀ 64]‖[head₁ 64]‖ … ‖[head₇ 64] 等價於 torch.cat([head_0, …, head_7], dim=-1)
階段 11 · 輸出投影 W_O

再來一個 $\mathbb{R}^{d_{model}\to d_{model}}$ 的線性映射。形狀不變,但讓模型 在拼接之後能跨頭混合信息。

$\text{MHA}(X) = \mathrm{Concat}(\text{head}_1, \dots, \text{head}_h)\,W^O$
[2, 10, 512] ·W_O [2, 10, 512] 
Wo = nn.Linear(d_model, d_model)   # 又 262 144 個參數
mha_out = Wo(out)                       # [B, S, d_model]

MHA 總參數量: $4 \cdot d_{model}^2 = 4\cdot 512^2 = 1{,}048{,}576$ (Q、K、V、O——各 $d_{model}\times d_{model}$)。

注意力結束: 我們帶著形狀 [B, S, d_model] 離開——和進來時的形狀一模一樣。注意力是一個保持形狀的算子。

04 FFN 與編碼器塊

LayerNorm、FFN,以及殘差高速公路

注意力之後,編碼器塊用兩條殘差連接、兩個 LayerNorm,以及一個逐位置(position-wise)的 FFN 把一切縫合起來。FFN 是大部分參數所在之處——也是很多人認為事實性知識被存儲的地方。

階段 12 · 殘差 + LayerNorm

把原始的 $X$ 加回來(跳躍連接),然後對每個 token 沿 $d_{model}$ 做歸一化。

$X' = \mathrm{LayerNorm}(X + \mathrm{MHA}(X))$
[2, 10, 512] + [2, 10, 512] → LN → [2, 10, 512] 
ln1 = nn.LayerNorm(d_model)      # 2 * d_model = 1024 個參數(γ, β)
X_post = ln1(X + mha_out)         # [B, S, d_model]

LN 沿最後一維歸一化:每個 $[d_{model}]$ 向量被變成均值 0、方差 1,再由可學習的 $\gamma, \beta\in\mathbb{R}^{d_{model}}$ 重新縮放。批次和 token 位置都保持互相獨立。

階段 13–14 · FFN 線性層 1 + ReLU

一個逐位置的前饋網絡:同一個 MLP 獨立地作用到每個 token 上。 第一個線性層把 $d_{model} \to d_{ff}$ 擴展(通常是 $4\cdot d_{model}$)。

$H = \mathrm{ReLU}(X' W_1 + b_1)$
[2, 10, 512] ·W₁ [2, 10, 2048] → ReLU → [2, 10, 2048] 
W1 = nn.Linear(d_model, d_ff)       # 512 * 2048 = 1 048 576 個參數
H = F.relu(W1(X_post))            # [B, S, d_ff]
階段 15 · FFN 線性層 2

第二個線性層投影回去:$d_{ff} \to d_{model}$。恢復成標準形狀,好讓下一個塊從頭開始。

$Y = H W_2 + b_2$
[2, 10, 2048] ·W₂ [2, 10, 512] 
W2 = nn.Linear(d_ff, d_model)       # 2048 * 512 = 又 1 048 576 個參數
ffn_out = W2(H)                       # [B, S, d_model]

FFN 總參數量: $2\cdot d_{model}\cdot d_{ff} = 2\cdot 512\cdot 2048 = 2{,}097{,}152$—— 大約是全部四個注意力投影加起來的兩倍。大模型的重量就壓在這裡。

階段 16 · 殘差 + LayerNorm(再來一次)

和階段 12 一樣的套路。加上 FFN 之前的張量,做 layer-norm,編碼器塊就完成了。

$Y' = \mathrm{LayerNorm}(X' + Y)$
[2, 10, 512] → [2, 10, 512] 塊輸出形狀 = 塊輸入形狀 
ln2 = nn.LayerNorm(d_model)
out = ln2(X_post + ffn_out)       # [B, S, d_model]
堆疊並重復: 論文堆了 $N=6$ 個相同的編碼器塊。每一個的輸入形狀 都等於它的輸出形狀,這正是它們能幹淨地串接起來的原因。 編碼器總參數量(單塊,base):大約 $4\cdot 512^2 + 2\cdot 512\cdot 2048 + 4\cdot 512 \approx 3.15$M。

14 行寫完整個編碼器塊

EncoderBlock —— 最小化 PyTorch
class EncoderBlock(nn.Module):
    def __init__(self, d_model=512, h=8, d_ff=2048):
        super().__init__()
        self.attn = nn.MultiheadAttention(d_model, h, batch_first=True)
        self.ffn  = nn.Sequential(
            nn.Linear(d_model, d_ff), nn.ReLU(), nn.Linear(d_ff, d_model)
        )
        self.ln1 = nn.LayerNorm(d_model)
        self.ln2 = nn.LayerNorm(d_model)

    def forward(self, x):
        # x: [B, S, d_model]
        a, _ = self.attn(x, x, x)
        x = self.ln1(x + a)
        x = self.ln2(x + self.ffn(x))
        return x                       # [B, S, d_model] —— 塊輸出

05 解碼器塊

解碼器塊 —— 先帶掩碼自注意力,再交叉注意力

解碼器接收目標 token(右移一位),產生一個同時瞭解兩件事的張量:(a) 它目前為止看過的目標 token ——通過帶掩碼的自注意力——以及 (b) 源端——通過交叉注意力, 其中 Q 來自解碼器,K、V 來自編碼器輸出(即 "memory")。與編碼器相同的嵌入維度、相同的殘差 + LN 規範。每層三個子塊,論文裡六層。

論文配置(仍然)
B = 2, S_src = 10, S_tgt = 7
d_model = 512, h = 8, d_k = 64
# S_tgt 與 S_src 互相獨立。
# 交叉注意力是它們唯一相遇的地方。
為什麼要"右移"?

訓練時,解碼器看到的是前綴加了 <sos> 的標準目標,並預測以 <eos> 結尾的同一個目標。

# target  : "die katze sitzt"           ← 真值
# dec_in  : "<sos> die katze sitzt"     ← 右移(輸入)
# labels  : "die katze sitzt <eos>"     ← 我們要預測的

階段 17 · 解碼器輸入 → 嵌入

[B, S_tgt] → 嵌入查表 → [B, S_tgt, d_model] = [2, 7, 512]

階段 19 · 帶掩碼的自注意力 (因果掩碼)

和編碼器自注意力完全相同——只是分數矩陣會在對角線以上的每個位置蓋上 −∞,這樣位置 i 永遠看不到 i+1, i+2, ...。經過 softmax 後這些位置恰好變成 0。

# scores: [B, h, S_tgt, S_tgt] = [2, 8, 7, 7]
mask = torch.triu(torch.ones(S_tgt, S_tgt), diagonal=1).bool()
scores = scores.masked_fill(mask, float('-inf'))
A = F.softmax(scores, dim=-1)        # 下三角,每行和為 1
[B, h, S_tgt, S_tgt] = [2, 8, 7, 7] 上三角在 softmax 後 = 0

A 掩碼 — 交互

為什麼解碼器不能偷看未來

訓練時解碼器會一次性看到整個目標(teacher forcing)。為了仍然學會"預測下一個 token", 我們禁止位置 i 關注任何更靠後的位置 j > i——這些單元格在 softmax 之前被設為 −∞,所以 softmax 之後它們恰好為 0。切換掩碼、滑動查詢位置來感受一下。

因果掩碼:開啟
查詢 i →
鍵 j → (S_tgt = 7)
允許 (j ≤ i) 屏蔽 (−∞)

掩碼關閉時,每個查詢都能看到每個鍵——這就是編碼器自注意力。掩碼開啟時, 網格變成下三角:位置 0 只看到自己,位置 6 看到全部 7 個。正是這一點讓模型既能在所有位置上 並行訓練,又能在推理時從左到右逐步生成。

Padding 掩碼

另外,S_src = 10 的源序列末尾有 padding 零(第 0 行:9 個真實 token + 1 個 pad;第 1 行:3 個真實 + 7 個 pad)。 這些位置也會被掩成 −∞,這樣注意力永遠不會流進 padding。點擊某個鍵位置把它切換為 padding, 看它整列變黑。

階段 20 · 交叉注意力 (Q ← 解碼器,K、V ← 編碼器)

這裡編碼器終於和解碼器對上話了。解碼器發出查詢;編碼器輸出提供鍵和值。

# dec_x: [B, S_tgt, d_model] = [2, 7, 512]   ← 來自帶掩碼的自注意力
# memory: [B, S_src, d_model] = [2, 10, 512]  ← 編碼器輸出(階段 16)
Q = W_q(dec_x).view(B, S_tgt, h, d_k).transpose(1, 2)   # [B, h, S_tgt, d_k]
K = W_k(memory).view(B, S_src, h, d_k).transpose(1, 2)   # [B, h, S_src, d_k]
V = W_v(memory).view(B, S_src, h, d_k).transpose(1, 2)   # [B, h, S_src, d_k]
scores = Q @ K.transpose(-2, -1) / math.sqrt(d_k)         # [B, h, S_tgt, S_src] = [2, 8, 7, 10]
A = F.softmax(scores, dim=-1)                              # 最後一維是 S_src
out = (A @ V).transpose(1, 2).reshape(B, S_tgt, d_model) # [2, 7, 512]

關鍵點:這個分數矩陣不是方陣。它是 [B, h, S_tgt, S_src]——每個目標 token 都關注每個源 token。

16 行寫完解碼器塊

DecoderBlock —— 最小化 PyTorch
class DecoderBlock(nn.Module):
    def __init__(self, d=512, h=8, d_ff=2048):
        super().__init__()
        self.self_attn  = nn.MultiheadAttention(d, h, batch_first=True)
        self.cross_attn = nn.MultiheadAttention(d, h, batch_first=True)
        self.ffn = nn.Sequential(nn.Linear(d, d_ff), nn.ReLU(), nn.Linear(d_ff, d))
        self.ln1, self.ln2, self.ln3 = nn.LayerNorm(d), nn.LayerNorm(d), nn.LayerNorm(d)

    def forward(self, x, memory, causal_mask):
        # x:      [B, S_tgt, d]      memory: [B, S_src, d]
        a, _ = self.self_attn(x, x, x, attn_mask=causal_mask)
        x = self.ln1(x + a)                 # [B, S_tgt, d]
        c, _ = self.cross_attn(x, memory, memory)
        x = self.ln2(x + c)                 # [B, S_tgt, d]
        x = self.ln3(x + self.ffn(x))
        return x                            # [B, S_tgt, d]

解碼器堆棧的輸出(經過 6 個這樣的塊之後): [B, S_tgt, d_model] = [2, 7, 512]。下一站:投影到詞表。

06 輸出 → logits

從隱藏狀態到 logits

一個 nn.Linear(d_model, V) 就把每個 token 的隱藏向量變成詞表裡每個詞的分數。 沿最後一維做 softmax,把這些分數變成概率。這就是整個輸出頭。在論文裡,W_lm 與輸入嵌入矩陣綁定——同一份參數,轉置而已。

階段 22 · 線性投影到詞表

[B, S_tgt, d_model] @ W_lm.T → [B, S_tgt, V] = [2, 7, 37000] 
# dec_out: [B, S_tgt, d_model]
# W_lm:    [V, d_model]   ← 常與 embed.weight 綁定
logits = dec_out @ W_lm.T           # [B, S_tgt, V]
# 或者直接:
logits = self.lm_head(dec_out)      # nn.Linear(d_model, V)

這些是原始分數——每個詞表 token、每個位置、每條樣本各一個。 還不是概率。如果你的損失是 CrossEntropyLoss,不要對它們做 softmax; 為了數值穩定,它內部會自己做。

階段 23 · Softmax → 概率 (只在你真正需要時)

probs = F.softmax(logits, dim=-1)   # [B, S_tgt, V] —— 在 V 上每行和為 1

在 B·S_tgt 個位置的每一個上,你現在都有一個覆蓋整個詞表的分佈。 這 37000 個數裡大多數都極小。只有少數幾個集中在正確答案附近。

輸出頭的參數量

光 W_lm 就是 V × d_model = 37000 × 512 ≈ 19M 個參數—— 通常是模型裡最大的一塊權重。論文把它和輸入嵌入矩陣綁定,把權重內存減半, 並(經驗上)有助於泛化。

完整的前向傳播,端到端

Transformer.forward —— 忠於論文
def forward(self, src_ids, tgt_in_ids):
    # src_ids:    [B, S_src]   tgt_in_ids: [B, S_tgt]
    src = self.src_embed(src_ids) + self.pos(src_ids)        # [B, S_src, d]
    tgt = self.tgt_embed(tgt_in_ids) + self.pos(tgt_in_ids)    # [B, S_tgt, d]

    memory = src
    for layer in self.encoder_layers:
        memory = layer(memory)                                  # [B, S_src, d]

    causal = self.causal_mask(tgt_in_ids.shape[1])
    x = tgt
    for layer in self.decoder_layers:
        x = layer(x, memory, causal)                            # [B, S_tgt, d]

    logits = self.lm_head(x)                                     # [B, S_tgt, V]
    return logits

def train_step(model, optimizer, batch):
    # batch:長度為 B 的 list[(src_str, tgt_str)]
    src_ids = tokenize_src(batch)         # [B, S_src]    int64
    dec_in, labels = tokenize_tgt(batch)  # 各為 [B, S_tgt]

    src = model.src_embed(src_ids)*math.sqrt(d) + model.pos(src_ids)
    tgt = model.tgt_embed(dec_in)*math.sqrt(d) + model.pos(dec_in)

    memory = src
    for layer in model.encoder_layers: memory = layer(memory)

    causal = model.causal_mask(dec_in.shape[1])
    x = tgt
    for layer in model.decoder_layers: x = layer(x, memory, causal)

    logits = model.lm_head(x)              # [B, S_tgt, V]
    loss = F.cross_entropy(
        logits.view(-1, V), labels.view(-1),
        ignore_index=-100, label_smoothing=0.1)

    optimizer.zero_grad(); loss.backward(); optimizer.step()
    return loss.item()                     # 標量

@torch.no_grad()
def generate(model, src_text, max_len=32):
    src_ids = tokenize_src([(src_text, "")])      # [1, S_src]

    src = model.src_embed(src_ids)*math.sqrt(d) + model.pos(src_ids)
    memory = src
    for layer in model.encoder_layers:
        memory = layer(memory)                          # [1, S_src, d]

    dec_in = torch.tensor([[SOS]])                         # [1, 1]
    for _ in range(max_len):
        tgt = model.tgt_embed(dec_in)*math.sqrt(d) + model.pos(dec_in)
        causal = model.causal_mask(dec_in.shape[1])
        x = tgt
        for layer in model.decoder_layers:
            x = layer(x, memory, causal)                 # [1, t, d]
        logits = model.lm_head(x)                          # [1, t, V]
        next_id = logits[:, -1, :].argmax(-1, keepdim=True)
        if next_id.item() == EOS: break
        dec_in = torch.cat([dec_in, next_id], dim=1)    # [1, t+1]

    return detokenize(dec_in[0, 1:].tolist())          # str

AWS S3 CLI — 從 ls 到 put-bucket-policy

兩個 CLI 家族,一個儲存服務。aws s3 是日常工作的友善高階封裝 — 複製、同步、預簽。aws s3api 則是 REST API 的低階對映,適合精細操作如版本控管、生命週期、分段上傳與 bucket 政策。動手把兩個都打過一遍。

aws s3 — 高階
類似檔案系統的指令:ls、cp、 mv、sync、rm、 mb、rb、presign。 會自動做聰明的事 — 多執行緒上傳、大檔自動分段、重試 — 九成情況下你會用它。
aws s3api — 低階
每個 S3 API 動詞對應一個子指令:list-objects-v2、 head-object、put-bucket-versioning、 create-multipart-upload …… 當 aws s3 沒有暴露你需要的功能時用它,或者你想要原始 JSON 輸出 以丟進 --query / jq。

你會打到一千次的 S3 URI

# scheme   bucket 名稱       key(bucket 內的物件路徑)
s3://my-bucket/photos/2026/cat.jpg

# 「資料夾」其實只是 key 前綴 — 它並不是真的存在的物件
s3://my-bucket/photos/2026/
陷阱:S3 的 key 並不是真正的檔案系統。「目錄」 photos/2026/ 之所以存在,只是因為有 key 以那個前綴開頭的物件。 空資料夾是以 / 結尾的零位元組物件。

IAM:你最常用的四個 S3 動作

Action允許做什麼Resource 範例
s3:ListBucket列出指定前綴的 keyarn:aws:s3:::my-bucket
s3:GetObject下載物件arn:aws:s3:::my-bucket/*
s3:PutObject上傳物件arn:aws:s3:::my-bucket/*
s3:DeleteObject刪除物件(或放上刪除標記)arn:aws:s3:::my-bucket/*
Bucket 對 object 動作:s3:ListBucket 對的是 bucket ARN(沒有 /*);s3:GetObject 對的是裡面的物件(有 /*)。把這兩個搞混 是新政策出現「AccessDenied」的頭號原因。

JMESPath 速成 — --query flag

每個 aws 呼叫都回傳 JSON。--query 'EXPR' 在用戶端執行 JMESPath,再交給 --output 格式化。 

# 只取 bucket 名稱
$ aws s3api list-buckets --query 'Buckets[].Name'

# 大於 1 MiB 的物件(數字要用反引號)
$ aws s3api list-objects-v2 --bucket B \
    --query 'Contents[?Size > `1048576`].Key'

# 最新的 5 個物件
$ aws s3api list-objects-v2 --bucket B \
    --query 'reverse(sort_by(Contents, &LastModified))[:5].[Key,LastModified]' \
    --output table

# 某前綴的總大小(加總)
$ aws s3api list-objects-v2 --bucket B --prefix logs/ \
    --query 'sum(Contents[].Size)'
5 從 list-buckets 只回傳 bucket 名稱。
$ aws s3api list-buckets --query 'Buckets[].'
6 JMESPath:只保留嚴格大於 1 MiB 的物件。填入關鍵字。
--query 'Contents[Size > `1048576`].Key'

上傳、下載、同步 — cp、mv、sync、exclude/include

這四個動詞(cp、mv、 sync、rm)處理進出 S3 95% 的位元組。在按下 --delete 之前,先把 --exclude / --include 的順序與 --dryrun 練熟。

cp — 單檔或遞迴

# 上傳
$ aws s3 cp ./report.pdf s3://my-bucket/reports/

# 下載
$ aws s3 cp s3://my-bucket/reports/report.pdf ./

# 遞迴 — 整個目錄
$ aws s3 cp ./build/ s3://my-bucket/site/ --recursive

# 加上儲存類別、加密、content-type 與 metadata
$ aws s3 cp big.bin s3://my-bucket/data/big.bin \
    --storage-class STANDARD_IA \
    --sse aws:kms --sse-kms-key-id alias/data \
    --content-type application/octet-stream \
    --metadata ingest=2026-05-20,team=ml \
    --acl private
1 把整個本地目錄上傳到 S3 的某個前綴。
$ aws s3 cp ./build/ s3://my-bucket/site/ --
2 把上傳標記為 Infrequent-Access。填入儲存類別。
$ aws s3 cp file.txt s3://b/k --storage-class

mv — 複製 + 刪除來源

# 把建置產物從 staging 升到 release 前綴
$ aws s3 mv s3://b/staging/v1.tar.gz s3://b/release/v1.tar.gz

# 本地 → S3(成功後刪除本地檔)
$ aws s3 mv ./done.zip s3://b/archive/

sync — 目錄級別的超能力

# 鏡像本地 → S3,*新增*與*更新*但永不刪除
$ aws s3 sync ./public/ s3://my-site/

# 鏡像並刪除:S3 結果與本地完全一致
$ aws s3 sync ./public/ s3://my-site/ --delete

# 跳過垃圾檔,只傳 JS 與 CSS
$ aws s3 sync ./dist/ s3://my-site/ \
    --exclude "*" \
    --include "*.js" \
    --include "*.css"

# 只看會做什麼,不真的做
$ aws s3 sync ./public/ s3://my-site/ --delete --dryrun
順序很重要:--exclude / --include 由左至右評估,最後相符者勝。 先用 --exclude "*",再把要的部分重新 include 進來。
3 把本地目錄鏡像到 S3,讓本地刪除的檔案也在 S3 被刪除,同時跳過 .tmp 檔案。
$ aws s3  local/ s3://bucket/prefix/ -- --exclude "*.tmp"
4 預覽具破壞性的 sync 會做什麼,但不真的執行。
$ aws s3 sync ./public/ s3://site/ --delete --

      rm — 小心,尤其是搭配 --recursive

      # 單一物件
$ aws s3 rm s3://b/old.log

# 整個前綴 — 不會問你!
$ aws s3 rm s3://b/tmp/ --recursive

$ aws s3 rm s3://b/logs/ --recursive --exclude "*.json"

      s3api 進階工具 — 一個動詞對應一個 S3 API 動作

      當友善的 aws s3 指令不夠用時,降到 s3api。輸出永遠是 JSON — 用 --query 或 jq 串接處理。

      物件層級操作

      # 從本地檔案上傳,明確指定 content-type 與 metadata
$ aws s3api put-object \
    --bucket my-bucket \
    --key data/blob.bin \
    --body ./blob.bin \
    --content-type application/octet-stream \
    --metadata sha256=abcd...,team=ml

# 下載到檔案
$ aws s3api get-object --bucket my-bucket --key data/blob.bin out.bin

# 伺服器端複製(位元組不會經過你的筆電)
$ aws s3api copy-object \
    --bucket dest-bucket \
    --key archive/blob.bin \
    --copy-source my-bucket/data/blob.bin

# 刪除特定一個版本
$ aws s3api delete-object \
    --bucket my-bucket --key data/blob.bin \
    --version-id 3HL4kqtJlcpXroDTDmJ+rmSpXd3dIbrHY
      1 在伺服器端把 srcB/srcK 的 key 複製到 dstB/dstK。填入動詞與來源 flag。
      $ aws s3api  \
    --bucket dstB --key dstK \
    -- srcB/srcK

      大表速查 — 你最常見到的 s3api 動詞

      動詞必要 flag用途
      list-buckets無整個帳號的 bucket 清單
      list-objects-v2--bucket分頁列出 key
      head-object--bucket --keymetadata、ETag、版本、加密

      Bucket 與政策 — mb、rb、policy、CORS、lifecycle、public-access block

      Bucket 層級的設定是安全與成本最常出問題的地方。 這些指令會設定版本控管、公開存取封鎖、CORS、生命週期規則與 bucket 政策。

      建立與移除 bucket

      # 建立。region 影響 LocationConstraint(us-east-1 是隱含 / 「沒有」)
$ aws s3 mb s3://my-new-bucket --region us-east-1

# 刪除。--force 會把它連同內容清空(僅高階指令)。
$ aws s3 rb s3://my-new-bucket --force

# s3api 對應(更多旋鈕)
$ aws s3api create-bucket --bucket my-new-bucket \
    --create-bucket-configuration LocationConstraint=eu-west-1

$ aws s3api delete-bucket --bucket my-new-bucket
      1 在 us-east-1 建立名為 cat-photos 的新 bucket。
      $ aws s3  s3://cat-photos --region us-east-1
      2 一招刪掉一個非空的 bucket。挑出 flag。
      $ aws s3 rb s3://cat-photos --

      封鎖所有公開存取(請先做這個!)

      $ aws s3api put-public-access-block \
    --bucket my-bucket \
    --public-access-block-configuration \
        BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true

$ aws s3api get-public-access-block --bucket my-bucket
      帳號層級護盾:同樣的旗標也存在於 aws s3control put-public-access-block --account-id … — 也在那邊設定一次, 這樣未來的 bucket 政策不會不小心開洞。

      Object Lock — 寫一次讀多次

      # 只能在建立 bucket 時啟用 — 之後無法開啟
$ aws s3api create-bucket --bucket vault \
    --object-lock-enabled-for-bucket

# 預設保存:7 年 COMPLIANCE 模式(對所有人都不可變更)
$ aws s3api put-object-lock-configuration \
    --bucket vault \
    --object-lock-configuration \
    '{"ObjectLockEnabled":"Enabled","Rule":{"DefaultRetention":{"Mode":"COMPLIANCE","Years":7}}}'
      GOVERNANCE 對 COMPLIANCE:GOVERNANCE 允許具特權的 IAM 使用者透過 s3:BypassGovernanceRetention 覆寫保存期間;COMPLIANCE 則 封鎖所有人(包括 root user),直到保存期結束。

      加密與儲存類別 — SSE、KMS、IA、Glacier

      為了成本(和取回延遲)選擇儲存類別,為了合規選擇加密模式。兩者互相獨立 — 每個類別都支援每一種加密模式。

      伺服器端加密(SSE)

      模式--server-side-encryption金鑰管理者稽核 / KMS 用途
      SSE-S3AES256S3無
      SSE-KMSaws:kmsKMS(你的 CMK)CloudTrail 記錄每次 Decrypt
      SSE-KMS DSSEaws:kms:dsseKMS,雙層加密受監管的工作負載
      SSE-C客戶在 header 提供金鑰你金鑰也得你自己保管
      無——2023 年起預設禁止(S3 自動套用 AES256)
       
      # SSE-S3(AES-256,S3 管理金鑰)
$ aws s3 cp f.bin s3://b/k --sse AES256

# SSE-KMS 並指定金鑰
$ aws s3 cp f.bin s3://b/k \
    --sse aws:kms \
    --sse-kms-key-id alias/data-prod

# Bucket 預設加密(任何新上傳即使沒指定也會用 KMS)
$ aws s3api put-bucket-encryption --bucket B \
    --server-side-encryption-configuration \
    '{"Rules":[{"ApplyServerSideEncryptionByDefault":{"SSEAlgorithm":"aws:kms","KMSMasterKeyID":"alias/data-prod"}}]}'

      2 用 SSE-S3 上傳。--sse 的值是什麼?

      分段上傳與預簽 URL — 給大檔與分享連結用

      高階 aws s3 cp 對於超過約 8 MB 的檔案會自動使用分段上傳。 當你以程式驅動上傳時,這裡是 s3api 的原語。

      分段上傳 — 四步舞

      ## 1. 啟動 — 取得 UploadId
$ UID=$(aws s3api create-multipart-upload \
    --bucket B --key big.bin \
    --query 'UploadId' --output text)

## 2. 上傳每個分段(最少 5 MB,最後一段例外)
$ ETAG1=$(aws s3api upload-part \
    --bucket B --key big.bin \
    --part-number 1 --upload-id $UID \
    --body part1.bin \
    --query 'ETag' --output text)

      練習 — S3 CLI 連續射擊

      超過二十題練習,涵蓋最常打的指令與 flag。aws s3 與 aws s3api 混合出題。

      1 身分驗證 — 哪個呼叫會回傳你的帳號 ID 與 ARN?
      $ aws sts
      2 為單一指令使用非預設 profile。
      $ aws s3 ls -- myprofile
      3 為單一指令覆寫 region。
      $ aws s3 ls s3://b -- eu-west-1
      4 把本地目錄樹遞迴複製到 S3。
      $ aws s3 cp ./site/ s3://my-site/ --
      5 帶刪除的 sync。
      $ aws s3 sync ./pub/ s3://my-site/ --
      6 Sync 時排除一切,然後只重新納入 PNG 檔。
      $ aws s3 sync ./img/ s3://b/img/ \
    --exclude "*" \
    -- "*.png"
      7 產生有效一天的預簽 URL(秒)。
      $ aws s3 presign s3://b/k --expires-in
      8 建立新 bucket。填入高階動詞。
      $ aws s3  s3://photos --region us-east-1
      9 移除非空的 bucket。
      $ aws s3  s3://photos --force
      10 SSE-S3 的加密值。
      $ aws s3 cp f.txt s3://b/k --sse
      11 SSE-KMS 的加密值。
      $ aws s3 cp f.txt s3://b/k --server-side-encryption
      12 Intelligent-Tiering 的儲存類別。
      $ aws s3 cp f.txt s3://b/k --storage-class
      13 只取 metadata(無 body)的 s3api 動詞。
      $ aws s3api  --bucket B --key K
      14 在某前綴下分頁取最多 1000 個物件。
      $ aws s3api  --bucket B --prefix p/ --max-keys 1000
      15 開啟版本控管。填入狀態字串。
      $ aws s3api put-bucket-versioning --bucket B --versioning-configuration Status=
      16 從本地 JSON 檔套用 bucket 政策。填入 file:// URI。
      $ aws s3api put-bucket-policy --bucket B --policy
      17 封鎖所有公開存取 — 填入 put- 後的動詞。
      $ aws s3api put- --bucket B \
    --public-access-block-configuration BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true
      18 JMESPath:從每個列出的物件只投影 Key 欄位。
      --query 'Contents[].'
      19 取消卡住的分段上傳。
      $ aws s3api -multipart-upload --bucket B --key K --upload-id $UID
      20 在伺服器端把 key 從一個 bucket 複製到另一個。
      $ aws s3api copy-object --bucket dest --key newkey -- srcBucket/srcKey
      21 刪除物件的某個特定版本。
      $ aws s3api delete-object --bucket B --key K -- 3HL4kqtJ
      22 顯示物件的每個版本(包含刪除標記)。
      $ aws s3api  --bucket B --prefix data/blob.bin
      23 JMESPath 過濾:保留嚴格大於 1 MiB 的物件。
      --query 'Contents[?Size  `1048576`].Key'
      24 取得 bucket 的 JSON 格式 CORS 規則。
      $ aws s3api get-bucket- --bucket B

      大魔王 — 端到端 S3 流水線

      一個情境,十個空格。你正在為受監管的資料集建立一個新 bucket: 建立、鎖好、開啟版本控管、加密上傳、列出、產生分享連結、安裝生命週期、清理舊版本。 每個空格都是真實的指令或 flag。

      流水線:冷儲存研究資料集

      合規要求:KMS 加密、無公開存取、版本控管、365 天到期、 90 天後轉到 Glacier。你從 research profile 在 eu-west-1 上操作。 

      # 1. 為整個 shell 工作階段選對 profile + region
$ export AWS_=research
$ export AWS_REGION=eu-west-1

# 2. 確認真的是以 research IAM 使用者登入
$ aws sts 

# 3. 建立 bucket
$ aws s3  s3://lab-genomics-2026 --region eu-west-1

# 4. 立刻鎖住公開存取
$ aws s3api put- \
    --bucket lab-genomics-2026 \
    --public-access-block-configuration \
        BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true

# 5. 開啟版本控管,讓不小心刪除可以還原
$ aws s3api put-bucket-versioning --bucket lab-genomics-2026 \
    --versioning-configuration Status=

# 6. 預設用 KMS 加密,讓未來「忘了帶 --sse」的上傳仍會被加密
$ aws s3api put-bucket-encryption --bucket lab-genomics-2026 \
    --server-side-encryption-configuration \
    '{"Rules":[{"ApplyServerSideEncryptionByDefault":{"SSEAlgorithm":"","KMSMasterKeyID":"alias/lab-data"}}]}'

# 7. 上傳第一批,明確指定加密 + IA 儲存類別
$ aws s3 cp ./batch-001/ s3://lab-genomics-2026/raw/2026-05/ \
    --recursive \
    --sse aws:kms --sse-kms-key-id alias/lab-data \
    --storage-class 

# 8. 驗證 — 以 MiB 列出
$ aws s3 ls s3://lab-genomics-2026/raw/ --recursive -- --summarize

# 9. 為 PI 產生 24 小時的分享連結以下載一個檔案
$ aws s3  s3://lab-genomics-2026/raw/2026-05/cohort-A.tsv --expires-in 86400

# 10. 安裝生命週期:90 天轉 GLACIER、365 天到期、舊版本 30 天清掉
$ aws s3api put-bucket- \
    --bucket lab-genomics-2026 \
    --lifecycle-configuration file://lc.json

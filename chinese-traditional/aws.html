安裝

安裝 AWS CLI v2、設定憑證,並驗證能正常打到你的帳號。五分鐘搞定。

安裝

macOS
# pkg 安裝程式
curl "https://awscli.amazonaws.com/AWSCLIV2.pkg" -o "AWSCLIV2.pkg"
sudo installer -pkg AWSCLIV2.pkg -target /

# 或透過 Homebrew
brew install awscli

aws --version
# aws-cli/2.15.x Python/3.x Darwin/24.x
Linux
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install

# Apple Silicon / ARM
curl "https://awscli.amazonaws.com/awscli-exe-linux-aarch64.zip" -o "awscliv2.zip"
Windows
# MSI 安裝程式
msiexec.exe /i https://awscli.amazonaws.com/AWSCLIV2.msi

# 或於 Powershell:
winget install -e --id Amazon.AWSCLI

具名 profile
大多數團隊都有多個帳號(dev、staging、prod),每個帳號用一個 profile。
aws configure --profile prod
# 每個指令都加:
aws s3 ls --profile prod
# 或用環境變數:
export AWS_PROFILE=prod

[prod]

aws_secret_access_key = ...

# ~/.aws/config
[default]
region = us-east-1
output = json

[profile prod]
region = us-west-2
output = json

IAM 基礎

身分(誰)+ 政策(可做什麼)+ 資源(在哪)。每個 API 呼叫都會根據政策授權。

概念

名稱說明
User長期身分,綁定到一個人(或服務)。可有 access key、主控台密碼、MFA。
Role可被承擔(assume)的身分,會回傳短效憑證(STS)。EC2、Lambda、EKS、跨帳號存取、聯邦驗證都會用。
PolicyJSON 文件。身分政策掛在使用者/角色上;資源政策掛在資源上(例如 S3 桶政策)。
Group共享政策的使用者集合,不可巢狀。
STSSecurity Token Service。透過 AssumeRole、AssumeRoleWithWebIdentity 等核發短效憑證。

最小可用 S3 政策

對單一桶唯讀
{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Action": ["s3:GetObject", "s3:ListBucket"],
    "Resource": [
      "arn:aws:s3:::my-bucket",
      "arn:aws:s3:::my-bucket/*"
    ]
  }]
}
寫入單一前綴
{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Action": ["s3:PutObject", "s3:DeleteObject"],
    "Resource": "arn:aws:s3:::my-bucket/uploads/*"
  }]
}
信任政策(角色由 EC2 承擔)
附加在角色上,用來宣告誰可以承擔我。

  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Principal": {"Service": "ec2.amazonaws.com"},
    "Action": "sts:AssumeRole"
  }]
}

S3 入門精華

  
桶(bucket)是 region 內的扁平命名空間。鍵(key)看起來像路徑,但其實不是資料夾。下面是你每週都會用到的指令。


  
桶

  

    

      
建立 / 列出 / 刪除

      
# 建立 —— 桶名是全球性的
aws s3 mb s3://my-bucket --region us-east-1

aws s3 ls
aws s3 ls s3://my-bucket
aws s3 ls s3://my-bucket --recursive --human-readable --summarize

# 刪除(必須是空的)
aws s3 rb s3://my-bucket
# 或強制清空後再刪:
aws s3 rb s3://my-bucket --force

    

    

      
cp —— 單一檔案

      
aws s3 cp ./report.csv s3://my-bucket/reports/

# 下載
aws s3 cp s3://my-bucket/reports/report.csv ./

# 跨桶
aws s3 cp s3://src/key s3://dst/key

# 加上 metadata 與 storage class
aws s3 cp ./big.tgz s3://my-bucket/ \
  --storage-class STANDARD_IA \
  --metadata "team=ml,owner=jane" \
  --content-type application/gzip

    

    

      
sync —— 整個目錄

      
只複製內容不同的檔案。預設不會刪除遠端有但本地缺的檔案 —— 要刪請加 --delete。

      
aws s3 sync ./build/ s3://my-bucket/v1/ \
  --exclude "*.tmp" \
  --exclude ".git/*" \
  --include "*.html" \
  --delete \
  --dryrun

    

    

      
rm

      
aws s3 rm s3://my-bucket/old.csv

# 整個前綴(遞迴)
aws s3 rm s3://my-bucket/tmp/ --recursive

# 加上篩選條件
aws s3 rm s3://my-bucket/ --recursive \
  --exclude "*" --include "*.log"

    

    

      
預先簽署 URL(presigned URL)

      
不需 AWS 憑證、有時效的簽署 URL。預設有效期是 3600 秒。

      
aws s3 presign s3://my-bucket/report.csv \
  --expires-in 900

# 用於上傳(PUT) —— 需要 s3api:
aws s3 presign s3://my-bucket/upload.bin \
  --expires-in 600
# 注意:aws s3 presign 只發 GET 形式的 URL。
# 要做 PUT 預簽請改用 SDK 或 s3api 指令。

S3 進階

  
版本、加密、生命週期、靜態網站。把桶從堆棧空間升級成基礎設施。


  
版本控制

  

    

      
啟用 / 列出 / 刪除

      
aws s3api put-bucket-versioning \
  --bucket my-bucket \
  --versioning-configuration Status=Enabled

# 列出版本
aws s3api list-object-versions --bucket my-bucket --prefix reports/

# 刪除指定版本
aws s3api delete-object \
  --bucket my-bucket --key reports/q1.csv \
  --version-id 3HL4kqtJlcpXroDTDmJ+rmSpXd3dIbrHY+MTRCxf3vjVBH40Nrjfkd

    

    

      
MFA Delete

      
強制要求 MFA token 才能刪除物件版本或關閉版本控制。僅 root 帳號可用。

      
aws s3api put-bucket-versioning \
  --bucket my-bucket \
  --versioning-configuration Status=Enabled,MFADelete=Enabled \
  --mfa "arn:aws:iam::123456789012:mfa/root 123456"

    

  


  
加密

  

    

      
桶預設加密

      
# SSE-S3(AES256,由 AWS 託管)
aws s3api put-bucket-encryption \
  --bucket my-bucket \
  --server-side-encryption-configuration '{
    "Rules":[{"ApplyServerSideEncryptionByDefault":{
      "SSEAlgorithm":"AES256"}}]}'

# SSE-KMS(客戶自管金鑰)
aws s3api put-bucket-encryption \
  --bucket my-bucket \
  --server-side-encryption-configuration '{
    "Rules":[{"ApplyServerSideEncryptionByDefault":{
      "SSEAlgorithm":"aws:kms",
      "KMSMasterKeyID":"alias/my-key"}}]}'

    

    

      
SSE-S3 vs SSE-KMS

      
SSE-S3:零設定,AWS 自管金鑰。免費。除非需要對金鑰使用做稽核或 IAM 控制,否則用它就好。
SSE-KMS:每把金鑰有稽核紀錄、可輪替、可跨帳號授權。每 1 萬請求 $0.03 + KMS 金鑰費用。

    

    

      
逐物件加密

      
aws s3 cp file.txt s3://my-bucket/ \
  --sse aws:kms \
  --sse-kms-key-id alias/my-key

① 用 S3 + CloudFront 托管靜態網站

  

    

      
1. 建立桶

      
BUCKET=my-site-$(uuidgen | tr A-Z a-z)
REGION=us-east-1

aws s3 mb s3://$BUCKET --region $REGION

# 維持公開存取封鎖 —— 由 CloudFront 來提供內容
aws s3api put-public-access-block \
  --bucket $BUCKET \
  --public-access-block-configuration \
   "BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true"

    

    

      
2. 同步網站內容

      
aws s3 sync ./site/ s3://$BUCKET/ \
  --delete \
  --cache-control "public,max-age=3600" \
  --exclude "*.html" \
  --exclude "service-worker.js"

# HTML 用短時快取,讓部署能很快被看見
aws s3 sync ./site/ s3://$BUCKET/ \
  --exclude "*" --include "*.html" \
  --cache-control "public,max-age=60"

    

    

      
3. 在前面放 CloudFront

      
aws cloudfront create-distribution \
  --origin-domain-name $BUCKET.s3.$REGION.amazonaws.com
# 在主控台:啟用 OAC,並附上一條桶政策,
# 允許 CloudFront 的 SourceArn(參考「桶政策」卡片)。

# 部署後使快取失效
aws cloudfront create-invalidation \
  --distribution-id E1ABC... --paths "/*"

疑難排解

  
CLI 不順手時,先看這頁。


  
常見錯誤

  

    
症狀常見原因與修法

    
      
An error occurred (AccessDenied)IAM 政策沒有授權該動作。請同時檢查身分政策與桶/資源政策。先用 aws sts get-caller-identity 確認是哪個身分發出請求,並善用 IAM Policy Simulator。

      
NoSuchBucket桶名打錯,或你在錯誤的 region/帳號。桶名是全域的;aws s3 ls 只列出你帳號擁有的桶。

      
The bucket you are attempting to access must be addressed using the specified endpointregion 對不上。請加 --region us-west-2 或設定 AWS_REGION。

      
ExpiredToken / InvalidTokenSSO/STS 憑證過期。請執行 aws sso login --profile foo 或重新承擔角色。

      
Could not connect to the endpoint URLregion 錯誤(該 region 沒有 S3)、VPC endpoint 政策、或代理設定。試試 --debug。

      
RequestTimeTooSkewed系統時鐘偏移。請同步 NTP。

      
An error occurred (SlowDown) when calling …S3 對單一前綴限速。請把鍵分散到更多前綴、退避後重試。每前綴上限:3500 PUT/COPY/POST/DELETE 與 5500 GET req/s。

      
InvalidAccessKeyId金鑰錯誤、已停用,或屬於另一個帳號。請重新檢查 ~/.aws/credentials 與 $AWS_PROFILE。

除錯工具

  

    

      
--debug

      
會印出每個簽署過的請求、標頭、重試次數與回應內容。權限/region 問題的良伴。輸出非常多。

      
aws s3 ls s3://my-bucket --debug 2>&1 | tee aws-debug.log

    

    

      
我是哪個身分?

      
aws sts get-caller-identity --output table

# 環境變數:
env | grep AWS_

# 設定:
aws configure list
aws configure list-profiles

    

    

      
Dry-run(S3 sync/cp/rm)

      
aws s3 sync ./build/ s3://my-bucket/v1/ \
  --delete --dryrun
# 印出 (dryrun) upload: ... → s3://...
# 印出 (dryrun) delete: s3://...

    

    

      
輸出格式

      
--output  json | yaml | yaml-stream | text | table

aws ec2 describe-instances \
  --query 'Reservations[].Instances[].InstanceId' \
  --output text

--no-paginate           # 直接拿一次原始回應

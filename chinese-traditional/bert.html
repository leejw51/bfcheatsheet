01 /

BERT 是什麼

BERT 是個純 transformer 編碼器模型。與 GPT（純解碼器、自迴歸）不同，每一層中每個 token 都會雙向關注其他所有 token。雙向性正是它的關鍵。

  • 發布時間：2018 年 10 月，由 Devlin 等人（Google AI Language）提出
  • 架構：transformer 編碼器區塊堆疊（多頭自注意力 + 前饋網路）
  • 預訓練：在 BooksCorpus（8 億詞）+ 英文維基百科（25 億詞）上自監督訓練
  • 微調：加上小型任務頭，在標註資料上端到端訓練
關鍵洞見：讀過數十億個句子並學會填空的模型，對語言知道得很多。在頂端裝個小分類器，下游任務的表現就會超越其體量。
02 /

預訓練目標

遮罩語言建模（MLM）

隨機遮罩 15% 的 WordPiece token。模型預測原本的 token。損失：僅在被遮罩位置上的交叉熵。

那 15% 中：

  • 80% 替換為 [MASK]
  • 10% 替換為隨機 token
  • 10% 維持不變

10/10 的小技巧避免模型偷懶——只在 [MASK] 位置產出強輸出，因為微調時不會有遮罩。

為何雙向？單向語言模型若不洩漏資訊就無法以未來 token 為條件。MLM 讓每個位置都能看到完整的雙向脈絡。

下一句預測（NSP）

給定兩個句子 A 與 B，預測 B 是否真的接在 A 之後（50%），還是來自他處的隨機句子（50%）。對 [CLS] token 的最後隱狀態做二元分類。

輸入：[CLS] A₁ A₂ ... [SEP] B₁ B₂ ... [SEP]
標籤：IsNext / NotNext
火辣觀點：RoBERTa（2019）證明 NSP 大致沒用，於是丟掉。SBERT 用句對對比目標取代。現代 BERT 風格的模型常常完全跳過 NSP。

合併損失

BERT 的預訓練損失是 MLM 與 NSP 損失之和：

$$\mathcal{L} = \mathcal{L}_{\text{MLM}} + \mathcal{L}_{\text{NSP}}$$

$$\mathcal{L}_{\text{MLM}} = -\sum_{i \in M} \log P(x_i \mid \tilde{x})$$

其中 $M$ 是被遮罩位置的集合，$\tilde{x}$ 是被破壞的輸入。

03 /

輸入格式

每個輸入位置的向量是三個 embedding 的和：token + segment + position。三者皆為大小相同（如 768）的可學習表。

$$\mathbf{x}_i = \mathbf{E}_{\text{tok}}(t_i) + \mathbf{E}_{\text{seg}}(s_i) + \mathbf{E}_{\text{pos}}(i)$$

一個編碼器區塊內部

L 個區塊每個都是標準的「post-LN」transformer 編碼器：

  1. 多頭自注意力，每個頭 H/A 維度
  2. 殘差 + LayerNorm
  3. 逐位置前饋：linear → GELU → linear（隱藏大小 4H）
  4. 殘差 + LayerNorm

$$\text{Attn}(Q,K,V) = \text{softmax}\!\left(\tfrac{QK^\top}{\sqrt{d_k}}\right)V$$

自注意力是完整且雙向的——沒有因果遮罩。

05 /

WordPiece 分詞

BERT 使用 30,522 個 token 的 WordPiece 詞彙表。未知字會被切成已知子片，延續片以 ## 為前綴。

"playing"     → ["play", "##ing"]
"unaffable"   → ["un", "##aff", "##able"]
"transformer" → ["transform", "##er"]
"BERT"        → ["bert"]   （在 bert-base-uncased 中為小寫）

動手試試

藍綠色 = 特殊 token， 黃色 = 完整字詞， 粉紅色 = ##延續片

06 /

微調範式

一個預訓練好的 BERT，三種（以及更多）下游任務模式。每種情況都加上小型任務專屬頭，端到端訓練，學習率較低（通常 2e-5 到 5e-5），訓練 2-4 個週期。

序列分類

情感、NLI、主題。使用最後的 [CLS] 隱狀態。

[CLS] → Linear(H→C) → softmax

範例：SST-2、MNLI、QNLI

Token 分類

NER、POS 標註。對每個 token 的隱狀態套用頭。

h_i → Linear(H→C) → softmax（每個 token）

範例：CoNLL-2003 NER

區段抽取（QA）

預測答案區段的起始與結束位置。

h_i → 2 個 logits（start、end）

範例：SQuAD 1.1 / 2.0

07 /

雙向 vs 自迴歸

BERT（純編碼器）

在「my dog is __」中以完整脈絡預測 cute：

my dog is [MASK] today .

關注兩側。無法自迴歸地產生文字，但能產出強的脈絡表示。

GPT（純解碼器）

在「my dog is」之後預測下一個 token：

my dog is ? today .

因果遮罩擋住未來 token。天生適合生成，相同參數量下對理解任務歷史上較弱（不過差距已縮小）。

限制與後繼者

BERT 的弱點

  • 注意力對序列長度為平方複雜度 → 最多 512 token
  • 無法生成——純編碼器，無法取樣
  • 預訓練/微調不一致——[MASK] 在推論時不會出現
  • NSP 訊號薄弱——後續模型把它丟了
  • 句子向量開箱即用效果差；平均池化的 [CLS] 表現不佳

後繼者與變體

模型年份變化
RoBERTa2019丟棄 NSP，更多資料、更長訓練、動態遮罩
ALBERT2019跨層共享參數；以句序預測取代 NSP
DistilBERT2019知識蒸餾，~40% 較小、~60% 較快，保留約 97% 效能
ELECTRA2020替代 token 偵測——比 MLM 更具樣本效率

bfgpt — 單一檔案的迷你 GPT

bfgpt 是一個可以在一個下午讀完、 並在筆電上訓練的迷你 GPT。一個約 800 行的 Python 檔案涵蓋一切—— 模型、資料準備、訓練和取樣——僅有 兩個依賴 (torch + numpy)。它是 Karpathy nanoGPT 的精煉移植版，去掉了 GPT-2 下載路徑。分詞採字元層級， 無需下載任何分詞器——詞彙表直接從你的文字中建立。

自給自足
模型 + 資料準備 + 訓練 + 取樣，全在一個 bfgpt.py 中。檢查點甚至內嵌了自己的分詞器，取樣不需要任何其他東西。
兩個依賴
只需 torch 和 numpy。沒有 transformers，沒有 tiktoken，無需學習任何框架。
隨處可跑
--device auto 自動解析 cuda→mps→cpu。預設值在 CPU 上即可執行。
字元分詞器
sorted(set(text)) 就是詞彙表。tiny-shakespeare 有 65 個唯一字元。
JSONL 輸入
每行一筆 {"text": "..."}。用 --text-field 指向不同欄位。
~0.8M 參數
4 層 / 4 頭 / 128 寬的模型，block_size=64。幾分鐘內就能產生莎士比亞風格的文字。

分詞器 — 你的文字就是詞彙表

沒有下載的分詞器，沒有 BPE 合併，沒有 50,257 個 token 的表格。bfgpt 取你語料庫中 唯一字元的排序集合並編號。這個映射 (stoi / itos) 就是完整的分詞器， 儲存在 meta.pkl 中。在下方輸入文字，觀察模型所看到的 encode → ids → decode 完整來回轉換。

CharTokenizer — 完整類別

  

    
CharTokenizerbfgpt.py:247

    
class CharTokenizer:
    def __init__(self, stoi, itos):
        self.stoi = stoi
        self.itos = itos
        self.vocab_size = len(stoi)

    @classmethod
    def from_text(cls, text):
        chars = sorted(set(text))            # the vocabulary
        stoi  = {ch: i for i, ch in enumerate(chars)}
        itos  = {i: ch for i, ch in enumerate(chars)}
        return cls(stoi, itos)

    def encode(self, s):
        return [self.stoi[c] for c in s]   # str -> list[int]

    def decode(self, ids):
        return ''.join(self.itos[int(i)] for i in ids)

  


  

    這樣做的好處：你的資料字母表決定了詞彙量大小，id 放在
    2 位元組的 uint16 中，嵌入表也很小。代價是序列較長
    （每個字元一個 token），因此上下文視窗 block_size 計的是
    字元，而非單詞。

讓它運作的六個技巧

  

    
權重共享
wte.weight = lm_head.weight — 輸入嵌入和輸出投影共用同一個矩陣。

    
前正規化
x + f(LN(x)) — 在每個子層之前正規化，而非之後。

    
因果遮罩
位置 t 只能關注 ≤ t 的位置；未來被遮罩為 −∞。

    
Flash Attention
當 PyTorch ≥ 2.0 支援時，使用融合的 scaled_dot_product_attention。

    
縮放初始化
殘差投影以 std = 0.02 / √(2·n_layer) 初始化。

    
選擇性衰減
權重衰減只作用於二維張量；偏置和 LayerNorm 跳過。

CausalSelfAttention — Q、K、V 一次矩陣乘法

  

    
CausalSelfAttention.forwardbfgpt.py:80

    
def forward(self, x):
    B, T, C = x.size()
    q, k, v = self.c_attn(x).split(self.n_embd, dim=2)   # one matmul, split 3 ways
    k = k.view(B, T, self.n_head, C // self.n_head).transpose(1, 2)  # (B, nh, T, hs)
    q = q.view(B, T, self.n_head, C // self.n_head).transpose(1, 2)
    v = v.view(B, T, self.n_head, C // self.n_head).transpose(1, 2)

    if self.flash:
        y = F.scaled_dot_product_attention(q, k, v, is_causal=True)
    else:
        att = (q @ k.transpose(-2, -1)) * (1.0 / math.sqrt(k.size(-1)))
        att = att.masked_fill(self.bias[:, :, :T, :T] == 0, float('-inf'))
        att = F.softmax(att, dim=-1)
        y = att @ v
    y = y.transpose(1, 2).contiguous().view(B, T, C)
    return self.resid_dropout(self.c_proj(y))

  

  
核心概念：因果遮罩（或 is_causal=True）使這成為一個解碼器——每個位置只用過去來預測下一個位置。

GPT.forwardbfgpt.py:183

    
def forward(self, idx, targets=None):
    b, t = idx.size()
    pos = torch.arange(0, t, device=idx.device)

    tok_emb = self.transformer.wte(idx)   # (b, t, n_embd)
    pos_emb = self.transformer.wpe(pos)   # (t, n_embd)
    x = self.transformer.drop(tok_emb + pos_emb)
    for block in self.transformer.h:
        x = block(x)
    x = self.transformer.ln_f(x)

    if targets is not None:                 # --- training ---
        logits = self.lm_head(x)
        loss = F.cross_entropy(logits.view(-1, logits.size(-1)),
                               targets.view(-1), ignore_index=-1)
    else:                                  # --- inference ---
        logits = self.lm_head(x[:, [-1], :])  # last position only
        loss = None
    return logits, loss

  

  
同一個 forward 的兩面：有 targets 時，它計算每個位置的損失（並行訓練）；沒有時，它只處理最後一個位置的 logits——取樣只需這些。

學習率排程 — 預熱後餘弦衰減

  

    

      
get_lrbfgpt.py:516

      
def get_lr(it):
    if it < warmup_iters:           # linear warmup
        return lr * (it+1) / (warmup_iters+1)
    if it > lr_decay_iters:         # floor
        return min_lr
    ratio = (it - warmup_iters) / (lr_decay_iters - warmup_iters)
    coeff = 0.5 * (1.0 + math.cos(math.pi * ratio))  # cosine
    return min_lr + coeff * (lr - min_lr)

    

    

      
線性升温 100 次迭代至峰值
      learning_rate = 1e-3，然後餘弦滑降至
      min_lr = 1e-4，共 2,000 次迭代。

      
$\eta_t = \eta_{\min} + \tfrac{1}{2}\,(1+\cos\pi r)\,(\eta_{\max}-\eta_{\min})$

      
每步還要：前向 → 反向 → 梯度裁剪至 1.0 → AdamW 步驟 → zero_grad。下一批次在反向傳播期間預取，以重疊 H2D 複製。

    

  







  
生成 — 逐 token 產生

  

    取樣是自回歸式的：預測下一個字元，附加它，把整個序列送回去，重複。
    當序列超過 block_size 後，從左側裁剪，使上下文視窗向前滑動。
  


  

    
GPT.generatebfgpt.py:226

    
@torch.no_grad()
def generate(self, idx, max_new_tokens, temperature=1.0, top_k=None):
    for _ in range(max_new_tokens):
        idx_cond = idx if idx.size(1) <= self.config.block_size \
                   else idx[:, -self.config.block_size:]   # crop to context
        logits, _ = self(idx_cond)
        logits = logits[:, -1, :] / temperature           # 1. reshape distribution
        if top_k is not None:
            v, _ = torch.topk(logits, min(top_k, logits.size(-1)))
            logits[logits < v[:, [-1]]] = -float('Inf')   # 2. keep top-k
        probs = F.softmax(logits, dim=-1)               # 3. to probabilities
        idx_next = torch.multinomial(probs, num_samples=1)  # 4. draw one
        idx = torch.cat((idx, idx_next), dim=1)           # 5. append & repeat
    return idx

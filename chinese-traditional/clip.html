CLIP — 圖片與文字共處一室

CLIP(Contrastive Language–Image Pre-training,OpenAI 2021)就是引發 多模態淘金熱的模型。它教會一個圖像編碼器和一個文字編碼器,把配對的圖文對放在共享嵌入 空間中相同的位置 — 而把無關的對拉得很遠。一旦訓練完成,只要把類別名寫成句子,就能做 零樣本 分類。

兩個編碼器、一個空間
圖像編碼器(ViT-B/32 或 ResNet-50)和文字編碼器(Transformer)各自輸出一個向量。 每邊的小型 投影頭 (projection head) 把兩者映射到相同的 d=512 嵌入空間,然後計算 cosine 相似度。
對比目標
對於一批 N 個圖文對,建立 N×N 相似度矩陣。對角線是真正的配對(正樣本); 其他都是負樣本。把對角拉高、把非對角壓低 — 對列與行同時做對稱交叉熵。
為什麼重要
訓練資料:從網路抓取的 4 億張圖文對。沒有人工標註 — 圖片說明就是標籤。結果: 一個模型能辨識它從未被明確教過的東西,只要把它與一個句子配對。
它驅動了什麼
DALL·E 2 的 CLIP 引導、Stable Diffusion 的文字編碼器(CLIP-L/14)、 零樣本分類器、影像搜尋、內容審核、檢索、image-to-image、 text-to-image 評分、OpenCLIP、SigLIP、EVA-CLIP… 全是後代。

一口氣講完的食譜

# 偽程式碼 (Radford et al., 2021)
I_f = image_encoder(I)            # [N, d_i]
T_f = text_encoder(T)             # [N, d_t]

I_e = l2_normalize(W_i @ I_f)     # [N, d]  投影 + 正規化
T_e = l2_normalize(W_t @ T_f)     # [N, d]

logits = (I_e @ T_e.T) * exp(t)   # [N, N] 以溫度縮放
labels = arange(N)                # 對角是目標
loss   = (CE(logits, labels) + CE(logits.T, labels)) / 2

架構 — 兩座塔、一個空間

CLIP 是一個 雙塔編碼器 (dual encoder)。兩座獨立的塔各自把輸入轉換為固定長度的 向量,再用小型線性投影把兩者對齊到同一維度,L2 正規化,再做內積 = cosine 相似度。

雙塔示意圖

圖像編碼器選項
ViT-B/32 12 層、patch 32、86M 參數(原始論文預設)
ViT-L/14 24 層、patch 14、304M 參數(Stable Diffusion 用這個)
RN50, RN101 ResNet 變體,以注意力池化取代 GAP
文字編碼器細節
arch 12 層 Transformer、8 頭、512 維、masked self-attention
tokens 49,152 字彙 BPE;每個序列最多 77 token
pool [EOS] token 位置的 hidden state,再做線性投影

logits_per_image = image_features @ text_features.   # [N, N]

對比損失 — 把對角線拉上去

在一批 N 個圖文對中,建立 N×N 相似度矩陣。對角項是唯一的正確配對; 其餘是 批次內負樣本 (in-batch negatives)。損失是對稱交叉熵:行為每張圖挑出 正確的說明,列為每個說明挑出正確的圖。

L = ½ · ( CE(logits, labels) + CE(logitsᵀ, labels) ), labels = [0, 1, 2, …, N−1]

溫度 — softmax 有多銳利?

原始相似度 I_e @ T_e.T 落在 [-1, 1] — 對有用的 softmax 來說太平了。CLIP 乘上一個學習過的縮放因子 exp(t),其中 t 是單一純量參數(初始化為 log(1/0.07) ≈ 2.66)。

P(text=j | image=i) = softmaxj( τ · cos(Ii, Tj) ), with τ = exp(t)

self.logit_scale.data.clamp_(0, math.log())

零樣本 — 不訓練就能分類

CLIP 的超能力:把任何分類任務變成文字配對問題。把每個類別名套進句子模板 ("a photo of a {class}"),一次編碼全部;對任意圖像:編碼、與類別嵌入做內積、 softmax → 完成。不用微調、不用標籤。

程式練習 — 使用真實 CLIP

三套真實世界的 stack。open_clip_torch 是事實上的標準 (模型比原始版本更好)。transformers 提供 HuggingFace 的 CLIPModel。兩者都包裝相同的數學。

open_clip — 載入 & 嵌入

完整參考
import torch, open_clip
from PIL import Image

model, _, preprocess = open_clip.create_model_and_transforms(
    "ViT-B-32", pretrained="laion2b_s34b_b79k")
tokenizer = open_clip.get_tokenizer("ViT-B-32")
model.eval()

img    = preprocess(Image.open("cat.jpg")).unsqueeze(0)
labels = ["a photo of a cat", "a photo of a dog"]
text   = tokenizer(labels)

with torch.no_grad():
    img_f = model.encode_image(img)
    txt_f = model.encode_text(text)
    img_f /= img_f.norm(dim=-1, keepdim=True)
    txt_f /= txt_f.norm(dim=-1, keepdim=True)
    probs = (100.0 * img_f @ txt_f.T).softmax(dim=-1)

print("probs", probs)   # [[0.98, 0.02]]

完成 CLIP 訓練步驟

11 個空格。全綠後按 訓練。 

import torch
import torch.nn.functional as F

# --- 兩個編碼器的前向傳播 --------------------------------------
image_features = model.(images)        # [N, d]
text_features  = model.(texts)          # [N, d]

# --- L2 正規化讓內積等於 cosine sim ------------------------------
image_features = image_features / image_features.(dim=-1, keepdim=True)
text_features  = text_features  / text_features.(dim=-1, keepdim=True)

# --- 建立 N × N 相似度 logits ------------------------------
logit_scale = model.logit_scale.()                 # τ = exp(t)
logits_per_image = logit_scale * image_features @ text_features.
logits_per_text  = logits_per_image.

# --- 對角目標的對稱交叉熵 ----------------
N      = images.shape[0]
labels = torch.(N, device=images.device)
loss_i = F.cross_entropy(logits_per_image, labels)
loss_t = F.cross_entropy(logits_per_text,  labels)
loss   = (loss_i + ) / 2

# --- 標準的 PyTorch 步驟 --------------------------------------
optimizer.zero_grad()
loss.()
optimizer.()

PyTorch · 一套記法統治所有張量運算

所有縮並，一條字串搞定

torch.einsum 讓你用一條下標字串（像 'ik,kj->ij'）就能寫出矩陣相乘、批次 matmul、轉置、跡、內積、 外積以及注意力分數。不用再記到底該用 @、bmm、 matmul，還是一堆糾纏的 transpose + sum —— 你只要 替每個軸命名，再說輸出要保留哪些。本頁帶你看規則、一份食譜集，還有一個 真的會在小張量上跑 einsum 的即時遊樂場。

01 一句話的核心概念

這是什麼

替每個軸命名

給每個輸入的每個軸一個字母。'bik,bkj->bij' 的意思是「運算元 A 的下標是 b,i,k；運算元 B 是 b,k,j；結果保留 b,i,j。」

唯一的規則

沒出現 = 被加總

任何出現在左側、卻沒出現在輸出的字母，都會被加總掉。 這裡 bij 裡少了 k，所以會對 k 加總 —— 這就是 矩陣相乘。

$$\texttt{'ik,kj->ij'} \quad\Longleftrightarrow\quad C_{ij} = \sum_{k} A_{ik}\, B_{kj}$$
把它當成一句話來讀。這些下標就是愛因斯坦求和公式： 左側列出每個運算元的索引，-> 隔開輸出，而對那個 重複但被丟掉的索引做加總則是隱含的。einsum 只是把這個數學寫成一次函式呼叫。

易讀

自帶文件

'bhqd,bhkd->bhqk' 一眼就告訴你注意力分數在做什麼 —— 軸全都 寫在那裡。一堆 permute+matmul 反而把它藏起來。

通用

更少 bug

整套語言，三條規則

怎麼讀一條下標

-> 之前的字串描述輸入；之後的字串描述輸出。三條 規則就涵蓋一切。

①

同字母 = 對齊

一個字母跨運算元重複使用，代表那些軸要對齊，並沿著該維度逐元素相乘。 在 'ik,kj' 裡，A 的 k 與 B 的 k 配對。

②

被丟掉的字母 = 加總

任何沒出現在輸出的輸入字母都會被加總掉。'ik,kj->ij' 丟掉了 k，所以會對 k 加總 —— 這就是縮並。

③

保留的字母 = 軸

出現在輸出的字母會成為結果的一個軸，順序就照你寫的來。把輸出字母重新排序就能 轉置：'ij->ji'。

02·a 自由索引 vs. 被加總索引

兩種角色、兩種顏色，貫穿整頁：

自由（free） —— 出現在輸出，成為結果的一個軸 被加總（summed） —— 只在左側出現，會被縮並掉
$$\underbrace{C_{\textcolor{#fbbf24}{ij}}}_{\text{free}} = \sum_{\textcolor{#818cf8}{k}} A_{\textcolor{#fbbf24}{i}\textcolor{#818cf8}{k}}\, B_{\textcolor{#818cf8}{k}\textcolor{#fbbf24}{j}}$$

02·b 隱含（無箭頭）形式

當你省略 -> 時

如果你略掉 ->輸出，einsum 會套用隱含規則：左側 恰好出現一次的每個索引都會成為自由輸出軸，並依字母順序排序。重複出現的 索引則會被加總。

  • 'ij'  →  'ij->ij'   恆等（每個字母各一次）。
  • 'ii'  →  'ii->'   跡（i 重複，所以被加總）。
  • 'ik,kj'  →  'ik,kj->ij'   矩陣相乘（k 重複 → 被加總；i,j 依序排）。
明確寫出來。隱含形式雖然方便，但字母排序規則常讓人意外。 把 -> 輸出寫出來能讓意圖毫不含糊，也是多數程式碼庫的做法。

02·c 函式簽名與陷阱

torch.einsum · 這個呼叫 
import torch

A = torch.randn(2, 3)        # [i=2, k=3]
B = torch.randn(3, 4)        # [k=3, j=4]

C = torch.einsum('ik,kj->ij', A, B)   # [i=2, j=4]  == A @ B

# 運算元依序接在字串之後傳入
# 每個運算元的字母數必須等於它的維度數
陷阱

字母數 ≠ 維度數

如果一個運算元有 3 個維度，它的下標就要剛好有 3 個字母。對一個 3 維張量寫 'ij' 會報錯。

陷阱

同字母，同長度

重複的字母在每個出現的地方長度都必須一致 —— 兩個運算元裡的 k 必須相符，否則無法對齊。

省略號 ...：對於你不想命名的前置「批次」維度，可以用 '...ik,...kj->...ij'。... 會在任意數量的前置軸上廣播 —— 對於 既要支援批次又要支援非批次輸入的程式碼非常好用。

03·c 用 einsum 寫出的注意力區塊

縮放點積注意力 · 完整拆解 
# Q,K,V: [B, H, T, d]   (批次、頭數、token、頭維度)
scores = torch.einsum('bhqd,bhkd->bhqk', Q, K) / math.sqrt(d)
attn   = scores.softmax(dim=-1)                    # [B,H,q,k]
out    = torch.einsum('bhqk,bhkd->bhqd', attn, V)  # [B,H,q,d]
讀一下 scores 那一行：q 和 k 是兩個 token 軸； d（頭維度）被共用且被丟掉，所以會被加總 —— 這就是 每個 query 對每個 key 的點積。b,h 則作為批次軸隨行。

05·a 讀懂下標

01 轉置矩陣 A[i,j] —— 交換輸出的軸。
torch.einsum('ij->', A)
02 對各欄 j 加總，每一列 i 保留一個值（列加總）。
torch.einsum('ij->', A)
03 矩陣相乘：A[i,k] 乘上 B[?,?] → C[i,j]。填入 B 的下標。
torch.einsum('ik,->ij', A, B)

04 完成這條規則：出現在左側、卻沒出現在輸出的索引，會被 ______ 掉。

05 方陣的跡（對角線元素之和）。填入輸入下標。
torch.einsum('->', A)
06 把對角線取出成一個向量（保留 i，不加總）。

torch.einsum('i,j->', u, v)

05·c 批次運算與注意力

08 批次 matmul：對每個批次 b，A[b,i,k] · B[b,k,j]。填入輸出。
torch.einsum('bik,bkj->', A, B)
09 注意力分數：Q[b,i,d] 與 K[b,j,d] 在頭維度 d 上做點積。
torch.einsum('bid,bjd->', Q, K)
10 脈絡向量：用注意力對 values 加權。attn[b,i,j] · V[b,j,d] → [b,i,d]。
torch.einsum('bij,->bid', attn, V)
11 PyTorch 函式本身 —— 寫出這個呼叫的名稱。
C = torch.('ik,kj->ij', A, B)

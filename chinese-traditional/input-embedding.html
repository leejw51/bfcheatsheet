輸入嵌入如何學習

分詞器把一串整數如 [464, 3797, 3332] 交給模型。輸入嵌入將每個 ID 轉換成大小為 $d_{model}$ 的可學習稠密向量。但它並非手工設計——它是一個普通的權重矩陣， 從隨機雜訊出發，由反向傳播訓練，與所有其他參數完全一致。本指南依照 Annotated Transformer， 精確說明這張表是如何建立、輸入與學習的。

01 一句話核心

它是什麼

可學習的查詢

一個矩陣 $E \in \mathbb{R}^{V \times D}$ ——詞表中每個條目對應一個可訓練的列。詞元的嵌入就是它的那一列，按索引取出。

如何學習

梯度下降

各列從隨機（Xavier 初始化）出發。每個訓練步，損失梯度只會推動批次中出現過的那些列。重複數百萬次後，幾何結構自然浮現。

「我們使用可學習的嵌入。」 論文明確指出：輸入與輸出詞元嵌入並非固定——它們是權重。沒有獨立的「嵌入訓練」階段；這張表與注意力層及前饋層在同一個語言/翻譯損失下共同最佳化。

01·b 它在哪裡

形狀 · 嵌入新增特徵軸 D
詞元 ID [B, T] → 嵌入 × √D [B, T, D] + PE[:T] [T, D] → 編碼器 [B, T, D]

$B$ = 批次，$T$ = 序列長度，$D = d_{model}$（基礎模型為 512）。分詞器產生 [B, T] 的 ID；嵌入層查詢每個 ID 對應的 D 維向量並縮放；再由 位置編碼 加入順序資訊。

如何使用本頁：按順序走過標籤頁 01 → 07。第 03 頁畫出查詢表與查表過程；第 04 頁是核心——前向傳播 → 損失 → 反向傳播的循環，附有即時「訓練一步」沙盒；第 05 頁展示浮現的語意；第 07 頁為你計分。按 ? 查看快速鍵。
Embeddings 模組

每個詞元一個可訓練的列

在 Annotated Transformer 中，嵌入是一個封裝 nn.Embedding 的小模組。前向傳播做列查詢，然後乘以 $\sqrt{d_{model}}$ 縮放。唯一需要學習的就是這個權重矩陣。

Embeddings 類別 · 原始碼 
class Embeddings(nn.Module):
    def __init__(self, d_model, vocab):
        super(Embeddings, self).__init__()
        self.lut = nn.Embedding(vocab, d_model)   # 權重: [V, D]
        self.d_model = d_model
    def forward(self, x):
        return self.lut(x) * math.sqrt(self.d_model)
每個符號的含義
  • vocab · V  詞表大小——列數。（nn.Embedding 的第一個參數。）
  • d_model · D  嵌入寬度——基礎 Transformer 為 512。
  • lut  可訓練的權重矩陣 [V, D]——字面意義上的「查詢表（look-up table）」。
  • x  形狀為 [B, T] 的整數詞元 ID；輸出為 [B, T, D]。
為什麼要乘以 √d_model？ 論文說：「在嵌入層，我們將權重乘以 √d_model。」 這樣可以提升嵌入的量值，使其與下一步加入的單位尺度位置編碼相當——否則位置資訊會淹沒內容資訊。當 D = 512 時，係數為 √512 ≈ 22.6。
通常是最大的一層。 參數量恰好為 V × D。32k 詞表搭配 D = 512 約為 16.4M 個權重——全在這張表裡，全靠梯度下降學習。

核心 · 列如何學習

訓練這張查詢表

嵌入層沒有特殊的訓練程序——它隨著模型的正常循環更新。一個批次前向傳播到損失；loss.backward() 將梯度送回每個權重，包括被查到的嵌入列；最佳化器執行一步。關鍵在於，只有批次中出現的列才會獲得梯度——更新是稀疏的。

04·b 為什麼更新是稀疏的

因為查表等同於一個 one-hot 矩陣乘法，$e_{id} = \mathbf{1}_{id}^{\top} E$，對查詢表的梯度在除了第 id 列以外全為零。每個被查到的詞元都把接收到的梯度存入自己的列，並在批次中的出現次數上求和。

$$\frac{\partial L}{\partial E_{j}} \;=\; \sum_{t\,:\,ids_t = j} \frac{\partial L}{\partial e_{t}} \qquad\Longrightarrow\qquad \frac{\partial L}{\partial E_{j}} = 0 \;\text{ 對未使用的列 } j$$
稀疏更新沙盒 · 只有批次中的列會移動

這個批次是一個短句。它的詞元（含列 ID）是唯一會獲得梯度的列。反覆點擊訓練一步：高亮列可見地漂移，而凍結的列始終不動——這就是稀疏嵌入梯度的實際效果。

04·c 驅動訓練的損失與最佳化器

標籤平滑損失 · 反向傳播 · 更新步 
# 目標分佈已平滑化：真實詞元獲得置信度，ε 分散到其他詞元
criterion = LabelSmoothing(size=V, padding_idx=0, smoothing=0.1)   # 內部使用 KLDivLoss

def step(out, targets, norm):
    log_probs = generator(out)                 # Linear → log_softmax  [.., V]
    loss = criterion(log_probs, targets) / norm
    loss.backward()                          # ∇ 流入 E 的被使用列
    opt.step()                                # Noam 排程 Adam 更新
    opt.optimizer.zero_grad()                 # 清除梯度以備下一批次
    return loss
$$lrate = d_{model}^{-0.5}\cdot \min\!\big(step^{-0.5},\; step\cdot warmup^{-1.5}\big)$$

學習率在前 4000 步上升，然後按 1/√step 衰減——早期更新溫和。

初始化很重要。 第 1 步之前，所有維度大於 1 的權重——包括嵌入表——都用 nn.init.xavier_uniform(p) 初始化。因此各列從小的隨機向量出發；你在第 05 頁看到的所有結構，都是從這裡學習得來的。
訓練產生的結果

語意成為幾何

訓練足夠多步後，扮演相似角色的詞元會被推向相似的列——因為這樣預測得更準，模型得到獎勵。兩列之間的餘弦相似度現在衡量模型認為它們有多相似，而方向也有了意義（著名的 king − man + woman ≈ queen）。選擇一個查詢詞元，看看它的鄰居如何亮起。

一個矩陣，三項工作

共享權重

論文更進一步：輸入嵌入、輸出（目標）嵌入，以及產生 logit 的 softmax 前線性層，三者共享同一個權重矩陣。訓練其中一個，就等於同時訓練所有三個——參數更少，且輸入/輸出幾何保持一致。

原文：「我們在兩個嵌入層和 softmax 前線性變換之間共享相同的權重矩陣。」 logit 因此為 h @ E.T——字面意思是「這個隱藏狀態在每個詞元的列方向上投影了多少？」
輸出端 · Generator 
class Generator(nn.Module):
    # "定義標準的線性 + softmax 生成步驟。"
    def __init__(self, d_model, vocab):
        super(Generator, self).__init__()
        self.proj = nn.Linear(d_model, vocab)   # 權重: [V, D] — 與 E 共享
    def forward(self, x):
        return F.log_softmax(self.proj(x), dim=-1)

給沒耐心操作員的實戰手冊
— 派生 / 協調 / 合併 —

multi­agent.

讓 Claude Code 並行運作的四種方式。
檔案：multi-agent-guide.html
執行階段：claude-code/cli
~/repo — claude --multi-agent

一個 Claude 視窗本身就是個小團隊。四個視窗則是一間工作室。 本指南介紹該團隊可以採用的四種型態——以及何時該選擇何種型態。

依協調方式選擇，而非人頭數。附帶任務 → 子代理。 獨立任務 → 代理檢視。需要協調 → 團隊。完全隔離 → 每個終端機一個 worktree。

即時 · 互動
i.
四種方法 · 擇一或疊加使用
方法 01 · 工作階段內委派

子代理。

擁有自己的脈絡視窗的專家工作者，完成聚焦任務後將摘要回傳給呼叫的工作階段。 用它們來保持主線整潔——研究、驗證、探索。

「重點不是再多一個代理。重點是讓你的主要對話保持簡潔。」

以 Markdown 檔案形式定義在 .claude/agents/ 之下。 每個都有自己的系統提示、工具白名單，並可選用較小模型，使工作成本低廉。

脈絡 · 隔離 並行 · 是 檔案衝突 · 共用 cwd 適合 · 研究、稽核
.claude/agents/researcher.md 
---
name: Researcher
description: Finds prior art & cites sources
model: haiku
tools:
  - bash
  - read
  - webfetch
---

# You are an expert researcher.
# Return citations as a bullet list.
提示 · 主工作階段 
$ /agents
  → 管理定義

$ Have @Researcher dig up the rate-limit RFC, then summarize.
→ 在隔離脈絡中派生 Researcher…
← "Found 3 sources, see notes/rfc-rate-limit.md"
方法 02 · 背景儀表板

代理檢視。

獨立背景工作階段的儀表板。發送提示後可以離開，稍後再查看輸出。 每一列都是自己的程序，並且能在終端機關閉後繼續存活。

「發出後再回來查看。當你有五件小事但只有一顆腦袋時非常有用。」

以 claude agents 開啟。在儀表板中： Space 預覽、Enter 進入、 ← 離開。

脈絡 · 每個工作階段獨立 協調 · 由你 持久性 · 關閉後仍存活 適合 · 並行雜事
shell 
# 派發背景工作階段
$ claude --bg "investigate the flaky e2e test"

# 用自訂代理派發
$ claude --agent code-reviewer --bg "review PR #1234"

# 開啟儀表板
$ claude agents

┌─ active ───────────────────────────┐
│ ▸ flaky-e2e        running   2m 14s │
│ ▸ pr-1234-review   running     38s │
│ ▸ docs-cleanup     done      11m 02s │
└────────────────────────────────────┘
方法 03 · 實驗性 · 協調式

代理團隊。

多個工作階段針對同一目標協作，共用任務清單、代理間訊息， 並由 lead 管理工作進度。比工作池更接近一間工作室。

「一位審查安全、一位審查效能、一位審查測試——他們會互相對話。」

屬於實驗性功能且預設停用。在 settings.json 中啟用， 然後請 Claude 為你的目標啟動一個團隊。

脈絡 · 每位成員獨立 協調 · 內建 狀態 · 實驗性 適合 · 多面向審查
settings.json 
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
提示 
$ Spin up a team to review PR #142.
  Three reviewers, working in parallel:
    1. security  — auth, input, secrets
    2. perf     — hot paths, allocs, queries
    3. tests    — coverage, flakiness, mocks
  Lead consolidates & posts one review.

→ team:r142 派生完成，含 3 位成員
→ 建立共用任務清單（12 項）
← 已將整合後審查發表至 #142
方法 04 · 最簡單的一種 · 強烈推薦

多個終端機。

打開另一個終端機分頁。執行 claude。完成——你就有兩個代理了。 搭配 git worktrees 即可獲得真正的隔離： 每個代理在自己的分支與工作樹上，不會發生編輯衝突。

「最乾淨的做法也是最笨的做法：多開幾個分頁。」

當任務長時間執行、彼此無關，且你希望完整看到每個代理在做什麼時最適合。 完成後透過 PR 合併。

設定 · 零 隔離 · 完整（worktree） 可見性 · 完全 適合 · 並行功能開發
shell · 每個代理一個 worktree 
# 從主 repo 開始
$ git worktree add ../repo-feat-a -b feat-a
$ git worktree add ../repo-feat-b -b feat-b

# 終端機 1 — 功能 A
$ cd ../repo-feat-a && claude

# 終端機 2 — 功能 B
$ cd ../repo-feat-b && claude

# 之後透過 PR 合併 ─ 沒有衝突
tmux · 分割窗格變體 
$ tmux new -s squad
  ctrl-b "       水平分割
  ctrl-b %       垂直分割
  ctrl-b o       切換窗格

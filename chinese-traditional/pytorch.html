Autograd

自動微分。PyTorch 存在的根本原因。

🔄 梯度流

前向傳遞 → 計算輸出。反向傳遞 ← 透過鏈鎖律傳播 ∂L/∂x。

前向激發 反向流動的梯度
陷阱  梯度在每次 backward() 呼叫間會 累積。每一步前都要呼叫 optimizer.zero_grad()（或傳 set_to_none=True），否則更新會混入不同批次的梯度。
requires_grad & backward
標記要追蹤梯度的輸入，再對純量 loss 呼叫 .backward()。
x = torch.tensor([1., 2.], requires_grad=True)
y = (x ** 2).sum()
y.backward()
print(x.grad)   # tensor([2., 4.])
no_grad / detach
推論時跳過 autograd 以節省記憶體。
with torch.no_grad():
    preds = model(x)

z = y.detach()       # 從計算圖剝離

# 也可作為 decorator
@torch.no_grad()
def predict(x): ...
zero_grad
梯度預設會累積 — 每一步前先重置。
optimizer.zero_grad(set_to_none=True)
loss.backward()
optimizer.step()

自訂 Function
為冷門運算定義自己的 forward/backward。
class MyReLU(torch.autograd.Function):
    @staticmethod
    def forward(ctx, x):
        ctx.save_for_backward(x)
        return x.clamp(min=0)

    @staticmethod
    def backward(ctx, grad_out):
        (x,) = ctx.saved_tensors
        return grad_out * (x > 0)

GradScaler (fp16)
放大 loss 讓微小梯度在 fp16 下不消失 → 在 step() 前還原。
scaler = torch.amp.GradScaler("cuda")

for x, y in loader:
    optimizer.zero_grad(set_to_none=True)
    with torch.autocast("cuda", dtype=torch.float16):
        loss = loss_fn(model(x), y)
    scaler.scale(loss).backward()
    scaler.unscale_(optimizer)             # 為了梯度裁剪
    torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
    scaler.step(optimizer)
    scaler.update()

FSDP — 分片所有東西

何時選哪個
速判規則。
# DDP — 模型放得進 1 張 GPU。
#   + 最簡單、單步最快
#   + 每個 rank 都有完整模型（評估/儲存方便）
#   - 記憶體：每個 rank O(模型大小)

# FSDP — 模型放不進 1 張 GPU。
#   + 記憶體：O(模型 / world_size)
#   + 可結合 bf16 / activation checkpointing
#   - 額外 all-gather → 通訊量略增

# 數十億級模型可採張量平行 + FSDP。

常見地雷

忘了 zero_grad：每次 backward() 都會累積梯度。記得重置。
view vs reshape：transpose / permute 之後張量不再連續 — view 會出錯。要先用 reshape 或 .contiguous()。
CrossEntropyLoss 接受 logits，不是 softmax 結果。先做 softmax 會讓梯度算錯。
BatchNorm 在 eval()：會使用累積統計而非批次統計。推論時忘了 model.eval()，輸出會依批次組成而變化。
tensor.item() 只能用於單元素張量，會強制 CPU 同步 — 熱迴圈中要避免。

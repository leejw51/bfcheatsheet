1什麼是 softmax

神經網路最後一層線性層輸出稱為 logits 的原始分數——任意實數，正負皆可。Softmax 將該向量轉換為機率分佈：每個項落在 $(0,1)$ 之間，整個向量總和為 $1$。

從分數到機率

若分類器對 $K$ 個類別輸出 logits $\mathbf{z} = (z_1,\dots,z_K)$，softmax 為

$$\text{softmax}(\mathbf{z})_i \;=\; \frac{e^{z_i}}{\sum_{j=1}^{K} e^{z_j}}.$$

在 transformer 內部，它對注意力頭關注的 $K$ 個 token 做相同的工作：將原始相似度分數轉換為標準化的注意力權重。

為何要「軟」max？

硬的 $\arg\max$ 會抹除梯度——只有一個位置勝出，輸家分數的微小變化產生零學習訊號。Softmax 是平滑、可微分的近似：最大的 logit 主導，但每個 logit 仍能分到（少量）機率質量與可用的梯度。

2公式的推導

建構過程

我們要的是一個函數，能將任意向量轉換為非負且總和為 1 的數值。

  1. 讓分數變正：套用嚴格正映射。自然指數 $e^{z}$ 是最乾淨的選擇——平滑、單調、$e^z>0$ 永遠成立。
  2. 標準化：每個都除以總和，使總和為 1。
$$\text{softmax}(\mathbf{z})_i = \frac{\exp(z_i)}{\sum_j \exp(z_j)}.$$

為何用指數，而不是 $z_i^2$ 或 $|z_i|$？

  • 對所有實數輸入嚴格為正（不會有零分母，也不必玩正負號遊戲）。
  • 乘性差距：$e^{z_i}/e^{z_j} = e^{z_i - z_j}$ 只取決於差。對每個 logit 加上常數毫無影響——見 §3。
  • 最大熵：在固定平均能量 $\mathbb{E}[z]$ 的分佈中，指數族擁有最高熵。Softmax 是與 logits 一致、最不武斷的分佈。
  • 與對數似然共軛：與交叉熵搭配時，梯度收斂到一行——見 §6。

三個必須內化的性質

  • 總和為 1。由建構方式保證：分子總和 = 分母。拖動任何滑桿——長條總是填滿那一列。
  • 對每個輸入單調。$z_i$ 增加，$p_i$ 增加（其餘減少），永遠不交叉。
  • 位移不變性。$\text{softmax}(\mathbf{z}+c\mathbf{1}) = \text{softmax}(\mathbf{z})$。分子分母都會多一個 $e^c$ 因子並抵消。這是數值穩定技巧的基礎：取 $c = -\max_i z_i$，使最大指數為 $0$——不會溢位、不損失資訊。

試試 套用 log-sum-exp 技巧 按鈕：它會將 logits 平移 $-\max$。長條不會動（位移不變性！），但可見的指數值現在都 $\le 1$。

4溫度

τ 縮放 logits

$$\text{softmax}_\tau(\mathbf{z})_i = \frac{e^{z_i/\tau}}{\sum_j e^{z_j/\tau}}.$$
  • $\tau \to \infty$：所有 $z_i/\tau \to 0$，分佈變成均勻分佈 $1/K$。最大探索。
  • $\tau \to 0^{+}$：差距爆炸，最大 logit 勝出，分佈塌陷成 $\arg\max$ 處的 one-hot。純貪婪。
  • $\tau = 1$：標準 softmax。

在 LLM 取樣中，溫度就是這個旋鈕：高 $\tau$ 產生多元、令人意外的 token；低 $\tau$ 產生確定、重複的 token。

動手試試

在上方小工具中，將 τ 在 $0.05$ 與 $5$ 之間滑動：

  • τ 小 → 勝出者的長條向 1.0 靠近
  • τ = 1 → 普通 softmax
  • τ 大 → 長條向 $1/3 \approx 0.333$ 趨平

退火——在訓練或取樣中逐步降低 τ——是借自統計物理的經典手法，那裡的 $\tau$ 確實就是 Boltzmann 分佈中的溫度。

5為什麼注意力中要用 softmax

縮放點積注意力

$$\text{Attn}(Q,K,V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right) V.$$

對每個 query token，$QK^\top$ 產生與每個 key 的相似度分數。Softmax 將這些分數轉換為注意力權重：非負、總和為 1，每個來源 token 一個。

輸出為 $\sum_j w_j v_j$——value 向量的加權平均。因為 $\sum w_j = 1$，輸出的大小始終可與單個 $v_j$ 比較，不論我們關注多少 token。這就是為何預設用 softmax（而不是諸如 ReLU 標準化）。

為何要在 softmax 前除以 $\sqrt{d_k}$？

若 query 和 key 的成分具單位變異數，$QK^\top$ 的變異數 $\propto d_k$。$d_k$ 大時 logits 變得巨大，softmax 飽和、梯度消失。除以 $\sqrt{d_k}$ 使變異數保持 $\approx 1$——softmax 留在敏感、可學習的區域。本質上這就是溫度 $\tau = \sqrt{d_k}$。

6Softmax + 交叉熵：梯度

設定

令 $p_i = \text{softmax}(z)_i$，目標 one-hot $y$，交叉熵

$$L = -\sum_i y_i \log p_i.$$

我們要 $\partial L/\partial z_i$。Softmax 的 Jacobian 很雜亂：

$$\frac{\partial p_i}{\partial z_j} = p_i(\delta_{ij} - p_j).$$

但與 $L$ 複合後，事情變得漂亮。

美麗的塌陷

連鎖律：

$$\frac{\partial L}{\partial z_i} = \sum_k \frac{\partial L}{\partial p_k}\frac{\partial p_k}{\partial z_i} = \sum_k \left(-\frac{y_k}{p_k}\right)\, p_k(\delta_{ki}-p_i).$$

化簡：

$$=\;-\sum_k y_k(\delta_{ki}-p_i) \;=\; -y_i + p_i\sum_k y_k \;=\; p_i - y_i.$$

所以 $\nabla_{z}L = \mathbf{p} - \mathbf{y}$。梯度就是誤差向量：預測減去目標。乾淨、便宜、數值友善。這就是為什麼 softmax + 交叉熵到處成對出現。

7數值穩定性——log-sum-exp 技巧

計算 $e^{1000}$ 會讓 64 位元浮點溢位；計算 $e^{-1000}$ 會下溢為 $0$，且 $\log 0 = -\infty$。樸素的 softmax 很脆弱。位移不變性救了我們。

令 $m = \max_i z_i$。則

$$\text{softmax}(\mathbf{z})_i = \frac{e^{z_i - m}}{\sum_j e^{z_j - m}}.$$

現在最大指數為 $e^0 = 1$；其餘都在 $(0,1]$。不會溢位，捨入誤差最小。

對 $\log\,\text{softmax}$ 而言，log-sum-exp 恆等式是標準形式：

$$\log\sum_j e^{z_j} = m + \log\sum_j e^{z_j - m}.$$

每個框架的 log_softmax、nll_loss 與注意力核心都會用到。

8替代方案，簡介

Sparsemax

將 logits 投影到機率單純形上。輸出真正稀疏——容許精確的零。當你希望注意力完全忽略大多數 token（可解釋性、結構化預測）時很有用。沒那麼平滑、需要排序步驟，但幾乎處處可微。

Gumbel-softmax

加上 Gumbel 雜訊後以低溫度做 softmax：對類別取樣的連續、可微鬆弛。讓你能用反向傳播訓練離散潛變數（如離散 VAEs、硬注意力）。

其他相關（entmax-1.5、球形 softmax、softmax 混合…）

Entmax-α 在 softmax（α=1）與 sparsemax（α=2）之間插值。Softmax 混合解決語言模型中的「softmax 瓶頸」。它們都用某種性質——稀疏性、表達力、計算量——換取與普通 softmax 不同的特性。

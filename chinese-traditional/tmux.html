1. tmux 是什麼?

tmux 是個 終端機多工器:一個終端機視窗能容納許多獨立的 shell — 並列、分頁,而且(最重要的)與你的 SSH 連線分離。

三大超能力:

  • 持續性 — 闔上筆電,明天再連線回來,你的 shell 還停在離開時的位置
  • 布局 — 將視窗切成多個窗格;同時跑 vim、tail -f 與 htop
  • 可腳本化 — 用一個指令啟動專案布局、與隊友共享工作階段、自動化建置

2. 心智模型

tmux 是個 客戶端/伺服器 程式。你的終端機與一個長時間執行的 tmux 伺服器溝通。伺服器擁有:

server
└── session "work"
    ├── window 0: editor
    │   ├── pane 0: vim
    │   └── pane 1: shell
    └── window 1: logs
        └── pane 0: tail -f app.log

attach = 你的終端機連上工作階段。detach = 你的終端機斷開,但工作階段仍在伺服器上執行。關閉 SSH 連線 ≠ 殺掉工作階段。

3. 前綴鍵

幾乎每個 tmux 工作階段內部的指令都從 前綴鍵 開始。預設是 Ctrl+b。你按下前綴、鬆開,再按下一個鍵。

記法:C-b " 表示 Ctrl+b 然後 "(也就是 Shift+')。許多人會把前綴重新對應到 Ctrl+a,因為更好按。

試試下方的模擬器 — 按下 Ctrl+b,看指示燈亮起來。

6. 窗格

窗格將視窗切成多個矩形。tmux 的記法以混亂出名:

  • C-b " 進行 水平 切割(上/下 — 一條水平分隔線)。記憶口訣:" 是兩個堆疊的點。
  • C-b % 進行 垂直 切割(左/右 — 一條垂直分隔線)。記憶口訣:% 看起來像兩個方塊之間的斜線。

7. 複製模式與卷軸回看

你的滑鼠可以滾動終端機,但在 tmux 中 真正的 方式是 複製模式。用 C-b [ 進入。現在你可以用 vi 鍵移動、搜尋、複製文字。

動作按鍵(在複製模式中)
進入複製模式C-b [
移動h j k l 或方向鍵
向前 / 向後搜尋/ / ?
開始選取Space(vi)或設 mode-keys vi 後用 v
複製選取內容Enter(預設)或 vi 模式下 y
貼上C-b ]
離開複製模式q

在 ~/.tmux.conf 中以 setw -g mode-keys vi 設為 vi 風格。

8. 設定 — ~/.tmux.conf

從 tmux 內部重新載入設定:按 C-b : 然後輸入 source-file ~/.tmux.conf,或設成綁定。

# 將前綴重新對應到 Ctrl-a
unbind C-b
set -g prefix C-a
bind C-a send-prefix

# 視窗/窗格編號從 1 開始(更容易按到 1..9)
set -g base-index 1
setw -g pane-base-index 1
set -g renumber-windows on

# vim 風格的窗格導航
bind h select-pane -L
bind j select-pane -D
bind k select-pane -U
bind l select-pane -R

# 合理的切割,保留目前路徑
bind | split-window -h -c "#{pane_current_path}"
bind - split-window -v -c "#{pane_current_path}"

# 滑鼠 + vi 複製模式
set -g mouse on
setw -g mode-keys vi
bind r source-file ~/.tmux.conf \; display "config reloaded"

# 加大卷軸緩衝
set -g history-limit 50000

9. 外掛(TPM)

TPM = Tmux Plugin Manager。將以下加到 ~/.tmux.conf:

set -g @plugin 'tmux-plugins/tpm'
set -g @plugin 'tmux-plugins/tmux-sensible'
set -g @plugin 'tmux-plugins/tmux-resurrect'   # 儲存/還原工作階段
set -g @plugin 'tmux-plugins/tmux-continuum'   # 開機自動還原
run '~/.tmux/plugins/tpm/tpm'

tmux 內部:C-b I 安裝、C-b U 更新。tmux-resurrect 可儲存完整工作階段樹的快照;tmux-continuum 每 15 分鐘自動儲存並在開機時重新載入 — 筆電重開 = 工作不流失。

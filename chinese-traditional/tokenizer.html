第 01 章 · 每個 transformer 的舌頭

字串 進,
整數 出。

模型不讀字元,也不讀單字。它們讀的是 整數。 分詞器就是那支小程式,把原始文字翻譯成 transformer 真正吃進去的整數語言 —— 再把它生成的整數翻譯回你看得懂的文字。少了它,GPT、BERT、LLaMA、Claude —— 全都啞口無言。

用一句話說
分詞器是一個函數 $T : \text{string} \to \mathbb{Z}^n$,把文字依固定 詞彙表 切成一個個 詞元(token),並輸出它們的整數 ID。反分詞器 $T^{-1}$ 則做相反的事。 
"hello world"
       │
       ▼  encode
[ 31373, 995 ]      # 兩個 token,GPT-2 詞彙表
       │
       ▼  decode
"hello world"
為什麼不直接用單字?
  • 詞彙爆炸。英文有超過 100 萬種詞形,模型得撐起一張百萬列的嵌入表。
  • 超出詞彙表。一個錯字或新詞就變成 <UNK>,模型瞬間失明。
  • 構詞學。"running"、"runner"、"runs" 共享一個字根,模型卻看不見。
  • 語言差異。中文沒有空格,土耳其文把詞素黏起來。靠空白切分必敗。
子詞分詞器一次解決四個問題 —— 一份約 3 萬到 20 萬塊的小詞彙表,藉組合就能涵蓋任何字串。
你會遇到的三大家族

BPE Byte-Pair Encoding —— 反覆合併最高頻的相鄰配對。GPT-2/3/4、LLaMA、Mistral。

WordPiece 像 BPE,但選的是讓語料概似度最大化的合併。BERT、DistilBERT、ELECTRA。

SentencePiece 把空白當成普通字元(▁);語言無關。內裡可以包 Unigram 或 BPE。T5、ALBERT、LLaMA。

接下來會發生什麼
詞元變成 token ID、再變成 嵌入,最後成為形狀為 [batch, seq_len, d_model] 的張量。Transformer 後面所有元件 —— 注意力、FFN、輸出投影 —— 都只是這張量的變換。分詞做對了,剩下的就是矩陣乘法。

第 02 章 · 每個分詞器都會做的五件事

分詞器 實際上
怎麼運作。

所有現代分詞器最終都做同樣的五件事,只是順序略有不同: 正規化、預分詞、用學到的合併表切成子詞、 每個片段對映到 ID、加上特殊 token。以下用白話一個個說明。

1

正規化

轉小寫 / NFC Unicode / 去重音 —— 看模型而定。GPT-2 在這幾乎不做事;BERT-uncased 則積極轉小寫。

"Hèllo!" → "hello!" (BERT-uncased)
2

預分詞

依空白與標點切開,確保合併步驟不會跨越單字邊界。GPT-2 用一段正規表達式;SentencePiece 則 不 預切 —— 它直接在原始位元組(包含空白)上動作。

"hello world!" → ["hello", " world", "!"]
3

套用合併 / 詞彙表

訓練時的緩慢步驟學到一份子詞詞彙表。推論時,每個預分詞片段被切成最長已知的片段。

"tokenization" → ["token", "ization"] (BPE)
4

把片段 → 整數 ID

詞彙表就是一份字典 str → int。每個片段成為它在嵌入表中的列索引。

["token", "ization"] → [30001, 1634]
5

加上特殊 token 與 padding

BERT 在前面加 [CLS]、結尾接 [SEP]。GPT 系模型通常什麼也不加。批次輸入會用 <pad> id 補到統一長度,並由 注意力遮罩 紀錄哪些位置是真的。

[101, 30001, 1634, 102, 0, 0] mask: [1, 1, 1, 1, 0, 0]

02 60 秒看懂 BPE —— 合併步驟

Byte-Pair Encoding 在 訓練時 建詞彙表:反覆找出語料中最高頻的相鄰符號對, 把它們的串接加進詞彙表當新符號。跑 $V$ 次合併,最終得到大小為 ~初始位元組數 + $V$ 的詞彙表。

一個迷你的實作範例
語料從 "low low low lower newer" 開始,先切成字元,並用詞邊界標記 ▁: 
# 第 0 步 — 位元組
▁ l o w   ▁ l o w   ▁ l o w   ▁ l o w e r   ▁ n e w e r

# 最高頻配對 = (l, o)  →  合併為 "lo"
▁ lo w   ▁ lo w   ▁ lo w   ▁ lo w e r   ▁ n e w e r

# 最高頻配對 = (lo, w)  →  合併為 "low"
▁ low   ▁ low   ▁ low   ▁ low e r   ▁ n e w e r

# 最高頻配對 = (e, r)  →  合併為 "er"
▁ low   ▁ low   ▁ low   ▁ low er   ▁ n e w er

# 最終詞彙表包含: ▁, l, o, w, e, r, n, lo, low, er, ...
推論時沿用 同一份 合併清單,依序套用每一筆仍然成立的合併。所以 BPE 是確定且無損的 —— 任何位元組序列都有唯一的分詞結果。

03 在腦袋裡讀懂分詞

盯著 GPT-2 輸出久了,你會看出幾個徵兆:

  • 前導空格很重要。字串開頭的 "hello" 與句中位置的 " hello" 是不同的 token。GPT-2 在位元組層級的詞彙表中甚至把前導空格印成 Ġ。
  • 常見字 → 1 個 token。"the"、"and"、"is" 都是單一 token。
  • 長 / 罕見字 → 多個 token。"tokenization" 會被切開,"antidisestablishmentarianism" 切得更碎。
  • 數字與表情符號會被切得很細 —— "2024" 在不同分詞器底下可能是 1、2 或 4 個 token。
  • 非拉丁字符在位元組層級 BPE 裡會變成 位元組 token:一個 3 位元組的 UTF-8 字碼點要花 3 個 token。SentencePiece 在以該語言訓練時可避開。

第 04 章 · 張量從哪來

從字串到 logits
八個形狀。

用同一句話 —— "Hello, world!" —— 我們追蹤從字串到 logits 確切的形狀變化。 B = batch、T = 序列長度、 D = 隱藏維度、V = 詞彙表大小。

1

原始字串

"Hello, world!"

shape: 純量(尚無張量)
2

分詞 → ID 串列

[15496, 11, 995, 0] (GPT-2 BPE,4 個 token)

shape: [T=4] dtype: int64
3

批次 + 補齊

把兩個句子堆成一個 batch,並用 id 0 補齊到 T=6:
[[15496, 11, 995, 0, 0, 0],
 [15496, 11, 5440, 16085, 0, 0]]

input_ids: [B=2, T=6] attention_mask: [B=2, T=6]
4

嵌入查表

x = embed[input_ids] —— 每個 id 對應 [V × D] 表中的一列。

x: [B=2, T=6, D=768] embed: [V=50257, D=768]
5

+ 位置嵌入

加上一張 [T × D] 表(或者 RoPE 改成旋轉 Q/K)。從這一步起,位置開始有意義。

x: [B=2, T=6, D=768] pos: [T=6, D=768]
6

L 個 transformer 區塊

每個區塊保持形狀不變。內部,注意力會短暫產生 scores: [B, H, T, T]。

x 維持: [B=2, T=6, D=768]
7

輸出頭 → logits

logits = x @ embed.T —— 每個位置對詞彙表中每個 token 各得一個分數。

logits: [B=2, T=6, V=50257]
8

softmax → 下一個 token id

argmax(logits[:, -1, :], dim=-1) 挑出下一個 id;分詞器再把它變回文字。

next_id: [B=2] 透過 T⁻¹ 變回 "解碼後的文字"

04 同樣的流程,在 PyTorch 裡

用 HuggingFace 從頭到尾跑一次
from transformers import AutoTokenizer, AutoModelForCausalLM
import torch

tok   = AutoTokenizer.from_pretrained("gpt2")
model = AutoModelForCausalLM.from_pretrained("gpt2")

# 1. encode → int64 張量字典
batch = tok(["Hello, world!", "Hello, friends, again"],
            padding=True, return_tensors="pt")
print(batch.input_ids.shape)        # torch.Size([2, 6])
print(batch.attention_mask.shape)   # torch.Size([2, 6])

# 2. forward —— 嵌入 + transformer 區塊
out = model(**batch)
print(out.logits.shape)            # torch.Size([2, 6, 50257])

# 3. greedy 取下一個 token = 在 vocab 軸上 argmax
next_id = out.logits[:, -1, :].argmax(dim=-1)
print(tok.batch_decode(next_id.unsqueeze(-1)))
記憶體粗估。fp16、batch=1、context=8k 的推論,每層活化值 大約需要 $T \times D \times 2$ 位元組,再加上 KV 快取 $2 \times L \times T \times D \times 2$ 位元組。以 LLaMA-7B 為例,T=8192 時 KV 快取 約 ~512 MB —— 這個預算直接被你分詞器的壓縮率掌控。
第 05 章 · 三大子詞家族

BPE · WordPiece · SentencePiece。

三者都產生子詞 token、都從頻率統計起家,但合併規則與處理空白的方式各有不同。 輸入一段字串,看看每一種怎麼切。

BPE — 依頻率合併

挑出最高頻的相鄰配對、合併、重複。推論時依序套用同樣的合併。貪婪且無損。

使用者:GPT-2/3/4、RoBERTa、LLaMA、Mistral、Qwen。

辨識特徵:前導空格屬於 token 的一部分(印成 Ġ 或 ▁)。

WordPiece — 依概似度合併

像 BPE,但合併規則改為最大化語料對數概似,而非單純頻率。 延續片段以 ## 為前綴。

使用者:BERT、DistilBERT、ELECTRA、MobileBERT。

辨識特徵:"playing" → ["play", "##ing"]

SentencePiece — 位元組神聖不可侵

把空白當成真正的字元(▁ = U+2581),直接從原始文字學起 —— 不做預分詞。 內裡可包 Unigram-LM 或 BPE。

使用者:T5、ALBERT、XLNet、mT5、LLaMA。

辨識特徵:長得像 ▁hello、▁world 的 token。

05 選錯了會怎樣?

  • 分詞器不一致是無聲的災難。用 BPE 訓練、用 WordPiece 推論 —— 每一次嵌入查表都是錯的。模型仍照跑、輸出全是亂碼。
  • 執行時新增 token是允許的(例如聊天範本),但你必須 同步擴大嵌入表 來對齊。漏掉這步,新 id 一上場就崩潰。
  • 不同語言、不同帳單。韓文在 GPT-2 BPE 下大約是英文的 3 倍 token。但用韓文訓練的 SentencePiece 大約只要 1 倍。同一個模型,成本截然不同。

level 01 // 動機

§01為何要 transformers?

2017 年以前,主流的序列模型是 RNN 與 LSTM。它們一次處理一個詞元,把隱藏狀態往前傳遞。這帶來三個致命缺陷:

  • 序列瓶頸。無法在時間維度上平行處理 — 詞元 t 需要詞元 t-1。
  • 遠距離梯度消失。來自詞元 1 的資訊得在抵達詞元 100 之前被擠壓過每一步。
  • 固定大小的隱藏狀態。整個過去都得塞進一個向量裡。長距離記憶很脆弱。
關鍵洞察

注意力就是你需要的一切

Vaswani 等人提出疑問:如果每個詞元都能直接看見其他每個詞元,而且是一個平行步驟內完成呢?沒有遞迴。沒有卷積。只有一個學到的機制,讓每個位置以學到的權重關注其他每個位置。

結果是:訓練可隨硬體線性擴展、梯度以 O(1) 跳數流動,模型在原則上能將段落的第一個字與最後一個字之間的關聯,建立得跟與其相鄰字一樣容易。

level 02 // 輸入

§02分詞與嵌入

Transformer 無法閱讀文字。它讀的是整數 ID。分詞是一種失真壓縮機制,把字串轉成來自固定詞彙表(通常 30k–200k 條目)的詞元 ID 序列。現代分詞器(BPE、WordPiece、Unigram、Tiktoken)依子詞片段切分,讓罕見字成為一串常見片段而不是未知詞元。

每個整數 ID 接著透過嵌入矩陣 $E \in \mathbb{R}^{V \times d_{model}}$ 對映到稠密向量,其中 $V$ 是詞彙表大小。查表 = 取一列。結果是:對長度 $L$ 的詞元序列,你會得到形狀為 [L, d_model] 的張量。

level 03 // 順序

§03位置編碼

自注意力是排列不變的:打亂輸入詞元(若沒有位置資訊)會得到一樣的輸出。對語言來說這是個問題。修正方式:在送進整個堆疊之前,於每個詞元嵌入加上一個與位置相關的向量。

原始論文採用幾何遞增波長的正弦:

$$PE_{(pos,\,2i)} = \sin\!\Big(\tfrac{pos}{10000^{2i/d_{model}}}\Big), \qquad PE_{(pos,\,2i+1)} = \cos\!\Big(\tfrac{pos}{10000^{2i/d_{model}}}\Big)$$

視覺化

正弦位置波形

每一列是一個維度 i。較低維度震盪快(各位置看起來差很多);較高維度則平滑(展現大範圍順序)。

聰明之處在於:$PE_{pos+k}$ 是 $PE_{pos}$ 的固定線性函數,所以模型能輕鬆學到相對位移。現代變體(RoPE、ALiBi、學習式絕對位置)各有取捨,但目標一致 — 注入順序資訊。

level 04 // 核心

§04自注意力(Q · K · V)

每個輸入詞元透過三個學到的矩陣 $W^Q, W^K, W^V \in \mathbb{R}^{d_{model} \times d_k}$ 投影成三個向量:

  • Query(Q,查詢) — 「我在找什麼?」
  • Key(K,鍵) — 「我擁有什麼?」
  • Value(V,值) — 「若被關注,我貢獻什麼?」

縮放點積注意力為:

$$\mathrm{Attention}(Q,K,V) = \mathrm{softmax}\!\Big(\tfrac{QK^\top}{\sqrt{d_k}}\Big)\,V$$

對於序列長度 $L$ 與單頭維度 $d_k$ 的形狀:Q: [L, d_k] K: [L, d_k] V: [L, d_k]。乘積 $QK^\top$ 為 [L, L] — 每個 (query, key) 對有一個相似度分數。沿列做 softmax 即可轉成注意力權重,再用以重新加權 $V$。

level 05 // 集成

§05多頭注意力

單一注意力模式通常還不夠。多頭注意力會平行執行 $h$ 個注意力運算,各有自己的 $W_i^Q, W_i^K, W_i^V$ 投影,並把結果串接:

$$\mathrm{MHA}(X) = \mathrm{Concat}(\mathrm{head}_1, \ldots, \mathrm{head}_h)\,W^O$$

每個 head 學會關注一種不同的關聯模式 — 一個 head 可能追蹤句法依存,另一個追蹤共指,再另一個處理簡單的位置位移。這種拆分讓總計算量大致與單頭相當:當 $h=8$ 時取 $d_k = d_{model}/h$。

level 06 // 黏合

§06前饋網路 · 殘差 · normalization

注意力之後,每個位置會通過位置式的前饋網路(feed-forward network) — 兩個線性層中間夾一個非線性(原始為 ReLU,如今通常是 GELU 或 SwiGLU):

$$\mathrm{FFN}(x) = \max(0,\, xW_1+b_1)\,W_2 + b_2$$

隱藏層寬度通常是 4·d_model。模型大部分參數住在這裡 — 一個流行的假說認為這層充當儲存事實知識的鍵值記憶體。

殘差連接

注意力子層與 FFN 子層都被包在殘差連接加上 layer norm 之中:

$$x \leftarrow \mathrm{LayerNorm}(x + \mathrm{Sublayer}(x))$$

為什麼?殘差為從損失回到輸入的梯度開出一條高速公路,讓上百層也能穩定訓練。LayerNorm 沿特徵維度將激活值置中與縮放,讓每一層的最佳化彼此解耦。

現代變體偏好 pre-norm:$x \leftarrow x + \mathrm{Sublayer}(\mathrm{LayerNorm}(x))$ — 在深層下訓練更穩定。

level 07 // 拓撲

§07編碼器 · 解碼器 · 遮罩

當今主流有三種架構口味:

  • Encoder-only(僅編碼器)(BERT) — 雙向自注意力。每個詞元都看得到其他每個詞元。適合理解任務(分類、檢索)。
  • Decoder-only(僅解碼器)(GPT) — 因果/遮罩式自注意力。詞元 t 只能關注 $\le t$ 的詞元。適合生成。多數現代 LLM 採用此架構。
  • Encoder-decoder(編解碼器)(T5、原始 NMT) — 編碼器讀來源,解碼器在生成目標時對編碼器輸出做 cross-attention。

因果遮罩 — 怎麼做?

在 softmax 之前,把分數矩陣的上三角項設成 $-\infty$。$-\infty$ 的 softmax 是 0 — 因此模型在數學上根本看不到未來:

level 08 // 目標

§08訓練:下一個詞元的預測

對僅解碼器的 LLM(GPT 家族),整個訓練目標簡單到令人驚訝:

給定詞元 1..t,預測詞元 t+1。

模型在每個位置輸出一個關於詞彙表的機率分佈。損失是相對於實際下一詞元的平均交叉熵(cross-entropy):

$$\mathcal{L} = -\frac{1}{N}\sum_{t=1}^{N} \log P_\theta(x_{t+1} \mid x_1, \ldots, x_t)$$

因果遮罩讓這件事很有效率:對長度 $N$ 的序列做一次前向傳遞,可同時得到 $N$ 個預測,每個位置一個。除了把標籤右移一格,不需要任何 teacher forcing 把戲。

其餘一切 — 指令微調、RLHF、工具使用 — 都建立在這個目標之上。預訓練是吸收網際網路規模資料、模型學到「萬事萬物」的階段。微調則是塑造其行為的階段。

回顧

完整管線

  • 1. 詞元 → 整數 ID → 嵌入(+ 位置編碼)
  • 2. 由 N 個 transformer 區塊組成的堆疊:MHA → Add&Norm → FFN → Add&Norm
  • 3. 投影到詞彙表大小的最終線性層、softmax → 下一詞元分佈
  • 4. 對照真實下一詞元的交叉熵損失,沿位置加總
  • 5. 對每個權重做反向傳播。重複約 10²² 次。完成。

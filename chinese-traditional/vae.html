變分自編碼器 — 帶有滲漏瓶頸的生成式建模

VAE 是潛在編碼為 機率分布 而非單一點的自編碼器。編碼器輸出 $(\mu, \log\sigma^2)$,我們透過 重參數化技巧 (reparameterization trick) 抽樣 $z \sim \mathcal{N}(\mu, \sigma^2)$,解碼器重建 $\hat{x}$。以重建損失 + KL(到單位高斯)訓練。訓練完成後,丟棄編碼器並從 $z \sim \mathcal{N}(0, I)$ 抽樣 來 生成新資料。

vs. 自編碼器
一般的 AE 把 $x \to z \to \hat{x}$ 做為確定性函數壓縮。潛在空間有空洞 — 隨機抽 $z$ 會得到垃圾。VAE 強迫潛在分布看起來像 $\mathcal{N}(0, I)$,這樣每個 $z$ 解碼後 都是合理的東西。
為什麼要機率化?
我們想要 生成:可以抽出新樣本的方法。VAE 學習 $p_\theta(x) = \int p_\theta(x|z)\,p(z)\,dz$,其中 $p(z) = \mathcal{N}(0, I)$。 最大化 $\log p(x)$ 的下界 — ELBO。
那個技巧
抽樣不可微分。重參數化: $z = \mu + \sigma \odot \varepsilon$, $\varepsilon \sim \mathcal{N}(0, I)$。 現在梯度可以流過 $\mu$ 與 $\sigma$,彷彿 $z$ 只是它們的函數。

ELBO 一行寫完

$\mathcal{L}_{\text{VAE}} = \underbrace{\mathbb{E}_{q(z|x)}[-\log p(x|z)]}_{\text{reconstruction}} \; + \; \beta \cdot \underbrace{\mathrm{KL}(q(z|x)\,\|\,p(z))}_{\text{regularizer}}$

第一項:解碼器從抽樣的 $z$ 重建 $x$ 的好壞。第二項:編碼器的 $q(z|x) = \mathcal{N}(\mu, \sigma^2)$ 偏離先驗 $p(z) = \mathcal{N}(0, I)$ 多遠。 編碼器必須在編碼的 同時 也保持貼近先驗。

心智模型

# VAE 前向傳播
mu, logvar = encoder(x)                 # 兩個輸出頭
std        = (0.5 * logvar).exp()       # σ = exp(½·logσ²)
eps        = torch.randn_like(std)      # ε ∼ N(0, I)
z          = mu + std * eps             # 重參數化
x_hat      = decoder(z)
loss       = recon(x_hat, x) + beta * kl(mu, logvar)

編碼器產生兩個向量

與只產生一個潛在向量的普通 AE 不同,VAE 編碼器有 兩個輸出頭: 一個輸出平均 $\mu$,一個輸出 $\log \sigma^2$(對數變異數 — 預測對數變異數而不是 $\sigma$ 讓網路輸出無限制,而 $\sigma$ 經過 $\exp$ 後仍保持為正)。

PyTorch 編碼器
import torch
import torch.nn as nn

class Encoder(nn.Module):
    def __init__(self, in_dim=784, hidden=400, latent=20):
        super().__init__()
        self.fc1     = nn.Linear(in_dim, hidden)
        self.fc_mu   = nn.Linear(hidden, latent)   # mean head
        self.fc_lv   = nn.Linear(hidden, latent)   # log-var head

    def forward(self, x):
        h = torch.relu(self.fc1(x))
        return self.fc_mu(h), self.fc_lv(h)        # (μ, log σ²)
解碼器 — 反向之旅
class Decoder(nn.Module):
    def __init__(self, latent=20, hidden=400, out_dim=784):
        super().__init__()
        self.fc1 = nn.Linear(latent, hidden)
        self.fc2 = nn.Linear(hidden, out_dim)

    def forward(self, z):
        h = torch.relu(self.fc1(z))
        return torch.sigmoid(self.fc2(h))    # pixels in [0, 1]

重參數化 — 讓一切變得可微分的技巧

我們需要對一個 樣本 $z \sim \mathcal{N}(\mu, \sigma^2)$ 做反向傳播。純隨機抽樣 沒有梯度 — 「從一個分布抽樣」沒有鏈鎖法則的路徑。 解法:把隨機性移到網路外面。

天真做法 (不行)
z = torch.normal(mu, std)
# ⚠ 沒有梯度路徑
# 回到 mu / std
重參數化 (可以)
eps = torch.randn_like(std)
z   = mu + std * eps
# ε 對 μ、σ 來說是常數
# ∂z/∂μ = 1, ∂z/∂σ = ε
$\sigma = \exp(\tfrac{1}{2}\log\sigma^2), \quad \varepsilon \sim \mathcal{N}(0, I), \quad z = \mu + \sigma \odot \varepsilon$

損失 — 重建 + KL = ELBO

最大化 ELBO 等同於最小化兩項:解碼器重建 $x$ 的差距,以及編碼器分布偏離 $\mathcal{N}(0, I)$ 的程度。

$\mathcal{L} = \underbrace{\mathrm{BCE}(\hat{x}, x)}_{\text{reconstruction}} \; + \; \beta \cdot \underbrace{-\tfrac{1}{2}\sum_{j}\bigl(1 + \log\sigma_j^2 - \mu_j^2 - \sigma_j^2\bigr)}_{\mathrm{KL}(\,\mathcal{N}(\mu,\sigma^2)\,\|\,\mathcal{N}(0,I)\,)}$
重建項
逐像素的 F.binary_cross_entropy 用於二元 / Bernoulli 像素 (MNIST),或 F.mse_loss 用於 高斯像素。使用 reduction='sum' 以對齊 ELBO 的解析推導,然後再除以批次大小。
KL 項 — 封閉形式
兩個高斯之間的 KL 有乾淨的解析形式,不需要抽樣: 
kl = -0.5 * torch.sum(
    1 + logvar - mu.pow(2) - logvar.exp()
)

β-VAE — 滑動 β 感受權衡

β = 1 標準 VAE。 β > 1 先驗壓力更強 → 潛在空間更平滑但重建更模糊。 β → 0 退化成普通自編碼器。

符號技巧: 我們 最大化 的 ELBO 是 $\mathbb{E}[\log p(x|z)] - \mathrm{KL}$。 我們 最小化 的損失是它的負值 — 一個正的重建項加上一個正的 KL 項。 同樣的東西、相反的符號。

潛在空間漫步 — z[0]、z[1] 是什麼意思?

訓練之後,在潛在空間中滑動會讓解碼輸出連續變形。當 $\beta > 0$ 時,潛在空間會被組織起來: 鄰近的 $z$ 解碼為相似的東西,而個別軸常常會學到可解釋的變動因子。

程式練習 — 組裝完整 VAE

現在打出把所有東西黏起來的部分:reparameterize、 完整前向傳播、損失函數,以及單一訓練步驟。

標準的 VAE forward + loss(完整版)
import torch
import torch.nn as nn
import torch.nn.functional as F

class VAE(nn.Module):
    def __init__(self, in_dim=784, hidden=400, latent=20):
        super().__init__()
        self.fc1     = nn.Linear(in_dim, hidden)
        self.fc_mu   = nn.Linear(hidden, latent)
        self.fc_lv   = nn.Linear(hidden, latent)
        self.fc3     = nn.Linear(latent, hidden)
        self.fc4     = nn.Linear(hidden, in_dim)

    def encode(self, x):
        h = torch.relu(self.fc1(x))
        return self.fc_mu(h), self.fc_lv(h)

    def reparameterize(self, mu, logvar):
        std = (0.5 * logvar).exp()
        eps = torch.randn_like(std)
        return mu + std * eps

    def decode(self, z):
        h = torch.relu(self.fc3(z))
        return torch.sigmoid(self.fc4(h))

    def forward(self, x):
        mu, logvar = self.encode(x.view(-1, 784))
        z          = self.reparameterize(mu, logvar)
        return self.decode(z), mu, logvar

def vae_loss(x_hat, x, mu, logvar, beta=1.0):
    recon = F.binary_cross_entropy(x_hat, x.view(-1, 784), reduction='sum')
    kl    = -0.5 * torch.sum(1 + logvar - mu.pow(2) - logvar.exp())
    return recon + beta * kl

CLIP — pictures and words in the same room

CLIP (Contrastive Language–Image Pre-training, OpenAI 2021) is the model that started the multimodal gold rush. It teaches an image encoder and a text encoder to put matching image–caption pairs at the same point in a shared embedding space — and unrelated pairs far apart. Once trained, you can do zero-shot classification by just writing the class names as sentences.

two encoders, one space
Image encoder (ViT-B/32 or ResNet-50) and text encoder (Transformer) each emit a vector. A small projection head on each side maps both into the same d=512 embedding space. Then we measure cosine similarity.
contrastive objective
For a batch of N image–text pairs, build the N×N similarity matrix. The diagonal cells are the true matches (positives); everything off-diagonal is a negative. Pull diagonals up, push off-diagonals down — symmetric cross-entropy on rows and columns.
why it matters
Training data: 400M image–text pairs scraped from the web. No human labels — captions are the labels. Result: a model that recognizes things it was never explicitly taught, just by matching them to a sentence.
what it powers
DALL·E 2's CLIP guidance, Stable Diffusion's text encoder (CLIP-L/14), zero-shot classifiers, image search, content moderation, retrieval, image-to-image, text-to-image scoring, OpenCLIP, SigLIP, EVA-CLIP… all descendants.

The recipe in one breath

# pseudocode (Radford et al., 2021)
I_f = image_encoder(I)            # [N, d_i]
T_f = text_encoder(T)             # [N, d_t]

I_e = l2_normalize(W_i @ I_f)     # [N, d]  project + normalize
T_e = l2_normalize(W_t @ T_f)     # [N, d]

logits = (I_e @ T_e.T) * exp(t)   # [N, N] scaled by temperature
labels = arange(N)                # diagonal is the target
loss   = (CE(logits, labels) + CE(logits.T, labels)) / 2

Architecture — two towers, one room

CLIP is a dual encoder. Two independent towers each turn their input into a fixed-length vector, a small linear projection lines them up to the same dimension, L2-normalize, then dot-product = cosine similarity.

the two-tower diagram

image encoder choices
ViT-B/32 12 layers, patch 32, 86M params (default in original paper)
ViT-L/14 24 layers, patch 14, 304M params (Stable Diffusion uses this)
RN50, RN101 ResNet variants, attention pool replacing GAP
text encoder details
arch 12-layer Transformer, 8 heads, 512-dim, masked self-attention
tokens 49,152-vocab BPE; max 77 tokens per sequence
pool hidden state at [EOS] token, then linear projection

Contrastive Loss — pull the diagonal up

Inside a batch of N image–caption pairs, build the N×N similarity matrix. Diagonal entries are the only correct matches; the rest are in-batch negatives. The loss is symmetric cross-entropy: rows pick the right caption for each image, columns pick the right image for each caption.

L = ½ · ( CE(logits, labels) + CE(logitsᵀ, labels) ), labels = [0, 1, 2, …, N−1]

Temperature — how sharp is the softmax?

The raw similarities I_e @ T_e.T live in [-1, 1] — way too flat for a useful softmax. CLIP multiplies by a learned scale exp(t), where t is a single scalar parameter (initialized to log(1/0.07) ≈ 2.66).

P(text=j | image=i) = softmaxj( τ · cos(Ii, Tj) ), with τ = exp(t)

Zero-Shot — classify without training

CLIP's superpower: turn any classification task into a text-matching problem. Wrap each class name in a sentence template ("a photo of a {class}"), encode all of them once, then for any image: encode it, dot with the class embeddings, softmax → done. No fine-tuning, no labels.

Code Drills — using real CLIP

Three real-world stacks. open_clip_torch is the de-facto standard (better models than the original release). transformers ships HuggingFace's CLIPModel. Both wrap the same math.

open_clip — load & embed

full reference
import torch, open_clip
from PIL import Image

model, _, preprocess = open_clip.create_model_and_transforms(
    "ViT-B-32", pretrained="laion2b_s34b_b79k")
tokenizer = open_clip.get_tokenizer("ViT-B-32")
model.eval()

img    = preprocess(Image.open("cat.jpg")).unsqueeze(0)
labels = ["a photo of a cat", "a photo of a dog"]
text   = tokenizer(labels)

with torch.no_grad():
    img_f = model.encode_image(img)
    txt_f = model.encode_text(text)
    img_f /= img_f.norm(dim=-1, keepdim=True)
    txt_f /= txt_f.norm(dim=-1, keepdim=True)
    probs = (100.0 * img_f @ txt_f.T).softmax(dim=-1)

print("probs", probs)   # [[0.98, 0.02]]

import torch
import torch.nn.functional as F

# --- Forward both encoders --------------------------------------
image_features = model.(images)        # [N, d]
text_features  = model.(texts)          # [N, d]

# --- L2-normalize so the dot product is cosine sim --------------
image_features = image_features / image_features.(dim=-1, keepdim=True)
text_features  = text_features  / text_features.(dim=-1, keepdim=True)

# --- Build N × N similarity logits ------------------------------
logit_scale = model.logit_scale.()                 # τ = exp(t)
logits_per_image = logit_scale * image_features @ text_features.
logits_per_text  = logits_per_image.

# --- Symmetric cross-entropy on diagonal targets ----------------
N      = images.shape[0]
labels = torch.(N, device=images.device)
loss_i = F.cross_entropy(logits_per_image, labels)
loss_t = F.cross_entropy(logits_per_text,  labels)
loss   = (loss_i + ) / 2

# --- Standard PyTorch step --------------------------------------
optimizer.zero_grad()
loss.()
optimizer.()

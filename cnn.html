What is a CNN?

A Convolutional Neural Network is the family of architectures that powered the deep-learning vision revolution (LeNet → AlexNet → ResNet). Instead of flattening an image into one giant vector and learning a separate weight per pixel, a CNN slides a tiny shared kernel across the image and reuses the same weights everywhere — learning local patterns like edges, corners, and textures.

why convolution beats fully-connected
  • Spatial locality — pixels near each other are correlated, far ones aren't. The kernel only looks at a small neighborhood.
  • Weight sharing — the same kernel is reused at every position, so a 3×3 filter has just 9 params (per channel) regardless of image size.
  • Translation equivariance — a cat in the top-left or bottom-right activates the same filter. Move the input, the output moves with it.
why not just use a dense layer?
A 224×224 RGB image flattened is 150,528 inputs. A single fully-connected layer with 4,096 neurons needs ~600M parameters. A 3×3 conv with 64 filters? ~1,800 parameters. That's a 300,000× reduction — and it generalizes better, because the inductive bias matches the structure of images.

The Convolution Operation

Convolution is element-wise multiply, then sum. Place a small kernel over a patch of the input, multiply matching cells, add them all up — that's one output pixel. Slide the kernel one step (the stride) and repeat.

$\text{out}[i, j] = \displaystyle\sum_{di=0}^{K-1}\sum_{dj=0}^{K-1}\, \text{in}[i+di,\ j+dj]\cdot \text{kernel}[di, dj]$

Output spatial size formula

$H_{\text{out}} = \left\lfloor \dfrac{H_{\text{in}} + 2P - K}{S} \right\rfloor + 1$

H input height, K kernel size, P padding, S stride. Same formula for width.

Stride & Padding

stride = 2 (downsamples)

Skip every other position. Output is roughly half the size.

[5, 5] → [2, 2]
padding = 1 (preserves size)

Wrap the input with a border of zeros. With $K=3, S=1, P=1$: output = input. This is what padding='same' means.

[5, 5] → [5, 5]

In code

2-D convolution — naïve Python
def conv2d(x, k, stride=1, pad=0):
    # x: [H, W],   k: [Kh, Kw]
    H, W   = x.shape
    Kh, Kw = k.shape
    # pad with zeros
    xp = np.pad(x, pad)
    Hp, Wp = xp.shape
    Ho = (Hp - Kh) // stride + 1
    Wo = (Wp - Kw) // stride + 1
    out = np.zeros((Ho, Wo))
    for i in range(Ho):
        for j in range(Wo):
            patch = xp[i*stride:i*stride+Kh, j*stride:j*stride+Kw]
            out[i, j] = (patch * k).sum()
    return out

Pooling & ReLU

Convolution gives you a stack of feature maps; ReLU introduces non-linearity so the network can model curves, and pooling halves the spatial resolution so the next conv sees a wider receptive field with the same kernel size.

ReLU — the rectifier

$\mathrm{ReLU}(z) = \max(0,\ z)$

Negative numbers vanish. Positives are unchanged. Zero gradient on the negative side (the famous "dying ReLU" risk).

Max-pool 2×2 — keep only the strongest

4×4 → 2×2 max pooling

Slide a 2×2 window with stride 2 over the input. Each window's max becomes one output cell. Halves both dimensions, no learnable parameters.

why pool?
  • Reduces spatial size → cheaper compute downstream.
  • Adds translation invariance — small shifts barely change the output.
  • Doubles the receptive field of the next conv layer for free.
  • Zero parameters — pooling is pure aggregation.
why ReLU?
  • Non-linearity → the network can model curves, not just lines.
  • Cheap: one comparison per element.
  • Sparse activations → many zeros → fast.
  • No vanishing gradient on the positive side, unlike sigmoid/tanh.

In code

# PyTorch — the canonical "Conv → ReLU → Pool" block
import torch.nn as nn

block = nn.Sequential(
    nn.Conv2d(3, 32, kernel_size=3, padding=1),  # [B,3,H,W] → [B,32,H,W]
    nn.ReLU(),                                  # elementwise, same shape
    nn.MaxPool2d(2),                            # [B,32,H,W] → [B,32,H/2,W/2]
)

Stacking the Pieces

A classic image classifier (think LeNet, VGG, AlexNet) is just the same block repeated: ConvReLUPool × N. Eventually the feature maps are tiny but deep — at that point we flatten to a vector and finish with one or two fully-connected layers plus a softmax.

Receptive field grows as you stack

A single 3×3 conv only sees a 3×3 patch. Stack two 3×3 convs (no padding) and the second layer sees a 5×5 patch of the original input. Add a 2×2 pool and the next conv's receptive field doubles again. After 4-5 stages you're effectively looking at the whole image with tiny shared kernels.

1conv1 (3×3)

RF = 3×3 — sees an edge or tiny texture.

2conv2 (3×3)

RF = 5×5 — combines edges into corners.

3after pool

RF doubles → next 3×3 sees a 14×14 area at the input.

Parameter count for a Conv layer

$\text{params} = K \cdot K \cdot C_{\text{in}} \cdot C_{\text{out}} \;+\; C_{\text{out}}\;\;\text{(bias)}$

Example: Conv2d(3, 16, 3) → $3 \cdot 3 \cdot 3 \cdot 16 + 16 = $ 448 params, and the same kernel runs over every spatial location.

The full classifier in PyTorch

a tiny LeNet-style CNN
import torch.nn as nn

class TinyCNN(nn.Module):
    def __init__(self):
        super().__init__()
        self.features = nn.Sequential(
            nn.Conv2d(1, 32, 3),  nn.ReLU(),  nn.MaxPool2d(2),
            nn.Conv2d(32, 64, 3), nn.ReLU(),  nn.MaxPool2d(2),
        )
        self.classifier = nn.Sequential(
            nn.Flatten(),
            nn.Linear(64 * 5 * 5, 10),  # 10 MNIST classes
        )
    def forward(self, x):           # x: [B, 1, 28, 28]
        x = self.features(x)        #    [B, 64,  5,  5]
        return self.classifier(x)  #    [B, 10]

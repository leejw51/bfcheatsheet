AWS S3 CLI — od ls po put-bucket-policy

Dvě rodiny CLI, jedna úložná služba. aws s3 je přívětivý high-level wrapper pro každodenní práci — copy, sync, presign. aws s3api je low-level zrcadlo REST API, pro jemné operace jako versioning, lifecycly, multipart uploady a bucket policy. Proklepejte si obojí psaním.

aws s3 — high-level
Příkazy s pocitem souborového systému: ls, cp, mv, sync, rm, mb, rb, presign. Dělá za vás chytré věci automaticky — vícevláknové uploady, multipart pro velké soubory, opakování — a 90 % času saháte právě po něm.
aws s3api — low-level
Jeden subcommand na každé sloveso S3 API: list-objects-v2, head-object, put-bucket-versioning, create-multipart-upload, … Použijte ho, když potřebujete funkci, kterou aws s3 nevystavuje, nebo když chcete syrový JSON výstup pro --query / jq.

S3 URI, které napíšete tisíckrát

# schéma   název bucketu   klíč (cesta k objektu uvnitř bucketu)
s3://my-bucket/photos/2026/cat.jpg

# "složka" je jen prefix klíče — jako objekt vlastně neexistuje
s3://my-bucket/photos/2026/
Pozor: S3 klíče nejsou reálný souborový systém. „Adresář" photos/2026/ existuje jen proto, že existují objekty, jejichž klíče začínají tímto prefixem. Prázdné složky jsou nulobytové objekty končící na /.

Výpis & inspekce — ls, head-object, list-objects-v2, --query

První přikázání cloud storage: vědět, co je v bucketu, než na to sáhnete. Obě CLI rodiny vypisují objekty; s3api dává JSON, který si můžete krájet pomocí JMESPath přes --query.

aws s3 ls — přívětivý výpis

# Všechny buckety v účtu
$ aws s3 ls

2026-04-12 09:01:55 logs-bucket

# Objekty v rootu jednoho bucketu
$ aws s3 ls s3://my-bucket

$ aws s3 ls s3://my-bucket/photos/ --recursive --human-readable --summarize

2026-05-01 14:11:03   1.7 MiB photos/2026/dog.jpg

Total Objects: 2
   Total Size: 3.8 MiB
1 Rekurzivně vypište vše pod prefixem photos/ v my-bucket.
$ aws s3 ls s3://my-bucket/photos/ --
2 Přidejte flag, který tiskne velikosti v MiB / GiB místo bytů.
$ aws s3 ls s3://my-bucket --recursive --

JMESPath rychlokurz — flag --query

Každé volání aws vrací JSON. --query 'EXPR' spustí JMESPath na klientu, ještě před tím, než --output formátuje. 

# Jen názvy bucketů
$ aws s3api list-buckets --query 'Buckets[].Name'

# Objekty větší než 1 MiB (všimněte si zpětných uvozovek u čísla)
$ aws s3api list-objects-v2 --bucket B \
    --query 'Contents[?Size > `1048576`].Key'

# 5 nejnovějších objektů
$ aws s3api list-objects-v2 --bucket B \
    --query 'reverse(sort_by(Contents, &LastModified))[:5].[Key,LastModified]' \
    --output table

# Celková velikost prefixu (suma)
$ aws s3api list-objects-v2 --bucket B --prefix logs/ \
    --query 'sum(Contents[].Size)'
5 Vraťte jen názvy bucketů z list-buckets.
$ aws s3api list-buckets --query 'Buckets[].'
6 JMESPath: ponechte jen objekty striktně větší než 1 MiB. Doplňte klíčové slovo.
--query 'Contents[Size > `1048576`].Key'

Upload, download, sync — cp, mv, sync, exclude/include

Tato čtyři slovesa (cp, mv, sync, rm) zvládnou 95 % bytů, které putují do S3 nebo z něj. Zvládněte pořadí --exclude / --include a --dryrun dřív, než vůbec napíšete --delete.

cp — jeden soubor nebo rekurzivně

# Upload
$ aws s3 cp ./report.pdf s3://my-bucket/reports/

# Download
$ aws s3 cp s3://my-bucket/reports/report.pdf ./

# Rekurzivně — celý adresář
$ aws s3 cp ./build/ s3://my-bucket/site/ --recursive

# S třídou úložiště, šifrováním, content-type a metadaty
$ aws s3 cp big.bin s3://my-bucket/data/big.bin \
    --storage-class STANDARD_IA \
    --sse aws:kms --sse-kms-key-id alias/data \
    --content-type application/octet-stream \
    --metadata ingest=2026-05-20,team=ml \
    --acl private
1 Nahrajte celý lokální adresář do prefixu v S3.
$ aws s3 cp ./build/ s3://my-bucket/site/ --
2 Označte upload jako Infrequent-Access. Doplňte třídu úložiště.
$ aws s3 cp file.txt s3://b/k --storage-class

mv — kopie + smazání zdroje

# Posunutí buildu ze staging do release prefixu
$ aws s3 mv s3://b/staging/v1.tar.gz s3://b/release/v1.tar.gz

# Přesun lokál → S3 (po úspěchu lokální soubor smaže)
$ aws s3 mv ./done.zip s3://b/archive/

sync — supersíla na úrovni adresářů

# Zrcadlit lokál → S3, *přidat* a *aktualizovat*, ale nikdy nemazat
$ aws s3 sync ./public/ s3://my-site/

# Zrcadlit s mazáním: S3 přesně odpovídá lokálu
$ aws s3 sync ./public/ s3://my-site/ --delete

# Vyhodit smetí, posílat jen JS a CSS
$ aws s3 sync ./dist/ s3://my-site/ \
    --exclude "*" \
    --include "*.js" \
    --include "*.css"

# Co by se změnilo, bez skutečného provedení
$ aws s3 sync ./public/ s3://my-site/ --delete --dryrun
Záleží na pořadí: --exclude / --include se vyhodnocují zleva doprava a vyhrává poslední shoda. Vzor --exclude "*" nejdřív, pak znovu zahrňte to, co chcete.
3 Zrcadlete lokální adresář do S3 tak, aby smazané lokální soubory byly smazány i v S3, a přitom přeskočte .tmp soubory.
$ aws s3  local/ s3://bucket/prefix/ -- --exclude "*.tmp"
4 Zobrazte náhled toho, co by destruktivní sync udělal, aniž by to provedl.
$ aws s3 sync ./public/ s3://site/ --delete --

      rm — opatrně, hlavně s --recursive

      # Jeden objekt
$ aws s3 rm s3://b/old.log

# Celý prefix — bez potvrzení!
$ aws s3 rm s3://b/tmp/ --recursive

# Selektivně: smazat logs/ ale ponechat .json
$ aws s3 rm s3://b/logs/ --recursive --exclude "*.json"

      s3api Power Tools — jedno sloveso na akci S3 API

      Když přívětivé příkazy aws s3 dojdou, sestupte na s3api. Výstup je vždy JSON — proženete přes --query nebo jq.

      Operace na úrovni objektů

      # Upload z lokálního souboru s explicitním content-type a metadaty
$ aws s3api put-object \
    --bucket my-bucket \
    --key data/blob.bin \
    --body ./blob.bin \
    --content-type application/octet-stream \
    --metadata sha256=abcd...,team=ml

# Stažení do souboru
$ aws s3api get-object --bucket my-bucket --key data/blob.bin out.bin

# Server-side kopie (žádné byty přes váš laptop)
$ aws s3api copy-object \
    --bucket dest-bucket \
    --key archive/blob.bin \
    --copy-source my-bucket/data/blob.bin

# Smazat jednu konkrétní verzi
$ aws s3api delete-object \
    --bucket my-bucket --key data/blob.bin \
    --version-id 3HL4kqtJlcpXroDTDmJ+rmSpXd3dIbrHY
      1 Server-side zkopírujte klíč z srcB/srcK do dstB/dstK. Doplňte sloveso a flag pro zdroj.
      $ aws s3api  \
    --bucket dstB --key dstK \
    -- srcB/srcK

      Buckety & politiky — mb, rb, policy, CORS, lifecycle, public-access block

      Konfigurace na úrovni bucketu je místo, kde nejčastěji žije nebo umírá bezpečnost a náklady. Tyto příkazy nastavují versioning, blokování veřejného přístupu, CORS, lifecycle pravidla a bucket policy.

      Vytvořit & smazat bucket

      # Vytvořit. Region rozhoduje o LocationConstraint (us-east-1 je implicitní / „none")
$ aws s3 mb s3://my-new-bucket --region us-east-1

# Smazat. --force ho zároveň vyprázdní (jen high-level).
$ aws s3 rb s3://my-new-bucket --force

# Ekvivalenty v s3api (víc knoflíků)
$ aws s3api create-bucket --bucket my-new-bucket \
    --create-bucket-configuration LocationConstraint=eu-west-1

$ aws s3api delete-bucket --bucket my-new-bucket
      1 Vytvořte nový bucket s názvem cat-photos v us-east-1.
      $ aws s3  s3://cat-photos --region us-east-1
      2 Smažte neprázdný bucket najednou. Zvolte flag.
      $ aws s3 rb s3://cat-photos --

      Zablokovat veškerý veřejný přístup (udělejte hned!)

      $ aws s3api put-public-access-block \
    --bucket my-bucket \
    --public-access-block-configuration \
        BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true

$ aws s3api get-public-access-block --bucket my-bucket
      Štít na úrovni účtu: stejné flagy existují jako aws s3control put-public-access-block --account-id … — nastavte je i tam, aby budoucí bucket policy nemohla omylem otevřít díru.

      Object Lock — write-once-read-many

      # Zapíná se jen při vytváření bucketu — později to už nejde
$ aws s3api create-bucket --bucket vault \
    --object-lock-enabled-for-bucket

# Defaultní retence: 7 let v módu COMPLIANCE (neměnné pro všechny)
$ aws s3api put-object-lock-configuration \
    --bucket vault \
    --object-lock-configuration \
    '{"ObjectLockEnabled":"Enabled","Rule":{"DefaultRetention":{"Mode":"COMPLIANCE","Years":7}}}'
      GOVERNANCE vs COMPLIANCE: GOVERNANCE umožňuje privilegovanému IAM uživateli obejít retenci přes s3:BypassGovernanceRetention; COMPLIANCE blokuje všechny, včetně root uživatele, dokud retenční doba neskončí.

      Multipart & pre-signed URL — pro velké soubory a sdílené odkazy

      High-level aws s3 cp používá multipart automaticky pro soubory větší než ~8 MB. Když řídíte uploady programaticky, tady jsou primitiva s3api.

      Multipart upload — čtyřkrokový tanec

      ## 1. Start — získat UploadId
$ UID=$(aws s3api create-multipart-upload \
    --bucket B --key big.bin \
    --query 'UploadId' --output text)

## 2. Nahrát každou část (5 MB minimum, kromě poslední)
$ ETAG1=$(aws s3api upload-part \
    --bucket B --key big.bin \
    --part-number 1 --upload-id $UID \
    --body part1.bin \
    --query 'ETag' --output text)
#  …opakovat pro části 2..N…

## 3. Dokončit — předat S3 seznam částí, server je sešije
$ aws s3api complete-multipart-upload \
    --bucket B --key big.bin --upload-id $UID \
    --multipart-upload '{"Parts":[{"ETag":"...","PartNumber":1},...]}'

## 4 (alt). Zrušit — uvolní úložiště, za které byste jinak platili
$ aws s3api abort-multipart-upload \
    --bucket B --key big.bin --upload-id $UID

      Procvičovací drilly — rapid-fire S3 CLI

      Více než dvacet drillů pokrývajících nejvíce psané příkazy a flagy. Mix aws s3 a aws s3api.

      1 Kontrola identity — které volání vrátí ID účtu a ARN?
      $ aws sts
      2 Použít nedefaultní profil pro jeden příkaz.
      $ aws s3 ls -- myprofile
      3 Přepsat region pro jeden příkaz.
      $ aws s3 ls s3://b -- eu-west-1
      4 Rekurzivně zkopírovat lokální strom do S3.
      $ aws s3 cp ./site/ s3://my-site/ --
      5 Sync s mazáním.
      $ aws s3 sync ./pub/ s3://my-site/ --
      6 Sync, ale vyloučit vše, pak znovu zahrnout jen PNG soubory.
      $ aws s3 sync ./img/ s3://b/img/ \
    --exclude "*" \
    -- "*.png"
      7 Vygenerovat denní pre-signed URL (v sekundách).
      $ aws s3 presign s3://b/k --expires-in
      8 Vytvořit nový bucket. Doplňte high-level sloveso.
      $ aws s3  s3://photos --region us-east-1
      9 Odstranit neprázdný bucket.
      $ aws s3  s3://photos --force
      10 Hodnota pro SSE-S3 šifrování.
      $ aws s3 cp f.txt s3://b/k --sse
      11 Hodnota pro SSE-KMS šifrování.
      $ aws s3 cp f.txt s3://b/k --server-side-encryption
      12 Třída úložiště pro Intelligent-Tiering.
      $ aws s3 cp f.txt s3://b/k --storage-class
      13 Sloveso s3api, které vrátí jen metadata (žádné tělo).
      $ aws s3api  --bucket B --key K
      14 Stránkovat až 1000 objektů pod prefixem.
      $ aws s3api  --bucket B --prefix p/ --max-keys 1000
      15 Zapnout versioning. Doplňte status.
      $ aws s3api put-bucket-versioning --bucket B --versioning-configuration Status=
      16 Aplikovat bucket policy z lokálního JSON souboru. Doplňte file:// URI.
      $ aws s3api put-bucket-policy --bucket B --policy
      17 Zablokovat veškerý veřejný přístup — doplňte sloveso po put-.
      $ aws s3api put- --bucket B \
    --public-access-block-configuration BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true
      18 JMESPath: promítnout jen pole Key každého vypsaného objektu.
      --query 'Contents[].'
      19 Zrušit zaseknutý multipart upload.
      $ aws s3api -multipart-upload --bucket B --key K --upload-id $UID
      20 Server-side zkopírovat klíč z jednoho bucketu do druhého.
      $ aws s3api copy-object --bucket dest --key newkey -- srcBucket/srcKey
      21 Smazat jednu konkrétní verzi objektu.
      $ aws s3api delete-object --bucket B --key K -- 3HL4kqtJ
      22 Zobrazit každou verzi (a delete-marker) objektu.
      $ aws s3api  --bucket B --prefix data/blob.bin
      23 JMESPath filtr: ponechat objekty striktně větší než 1 MiB.
      --query 'Contents[?Size  `1048576`].Key'
      24 Získat CORS pravidla bucketu ve formátu JSON.
      $ aws s3api get-bucket- --bucket B

      Boss drill — End-to-end S3 pipeline

      Jeden scénář, deset políček. Spouštíte nový bucket pro regulovaný dataset: vytvořte ho, zamkněte, zapněte versioning, šifrovaně nahrajte, vypište, vystavte sdílecí odkaz, nainstalujte lifecycle a vyčistěte staré verze. Každé políčko je skutečný příkaz nebo flag.

      Pipeline: studený archiv výzkumného datasetu

      Compliance říká: KMS-šifrováno, žádný veřejný přístup, versionováno, expirace 365 dní, přesun na Glacier po 90 dnech. Pracujete z profilu research v eu-west-1. 

      # 1. Vyberte správný profil + region pro celou shell session
$ export AWS_=research
$ export AWS_REGION=eu-west-1

# 2. Ověřte, že jsme opravdu přihlášeni jako research IAM uživatel
$ aws sts 

# 3. Vytvořte bucket
$ aws s3  s3://lab-genomics-2026 --region eu-west-1

# 4. Co nejdřív zamkněte veřejný přístup
$ aws s3api put- \
    --bucket lab-genomics-2026 \
    --public-access-block-configuration \
        BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true

# 5. Zapněte versioning, ať jsou náhodná smazání obnovitelná
$ aws s3api put-bucket-versioning --bucket lab-genomics-2026 \
    --versioning-configuration Status=

# 6. Default-šifrování přes KMS — i upload, který „zapomene" --sse, bude šifrován
$ aws s3api put-bucket-encryption --bucket lab-genomics-2026 \
    --server-side-encryption-configuration \
    '{"Rules":[{"ApplyServerSideEncryptionByDefault":{"SSEAlgorithm":"","KMSMasterKeyID":"alias/lab-data"}}]}'

# 7. Nahrajte první dávku s explicitním šifrováním + IA třídou úložiště
$ aws s3 cp ./batch-001/ s3://lab-genomics-2026/raw/2026-05/ \
    --recursive \
    --sse aws:kms --sse-kms-key-id alias/lab-data \
    --storage-class 

# 8. Ověřte — vypsat s velikostmi v MiB
$ aws s3 ls s3://lab-genomics-2026/raw/ --recursive -- --summarize

# 9. Vygenerujte 24h sdílecí odkaz, aby si PI mohl jeden soubor stáhnout
$ aws s3  s3://lab-genomics-2026/raw/2026-05/cohort-A.tsv --expires-in 86400

# 10. Nainstalujte lifecycle: přesun na GLACIER po 90d, expirace po 365d, smazání starých verzí po 30d
$ aws s3api put-bucket- \
    --bucket lab-genomics-2026 \
    --lifecycle-configuration file://lc.json

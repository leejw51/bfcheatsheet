Vítejte — vaše první neuronová síť v PyTorch

Než dočtete tuto stránku, naťukáte kód kompletní neuronové sítě: nadefinujete ji, natrénujete ji a spustíte inferenci. Žádné copy-paste — záleží na každé klávese. Vyplňte každou mezeru správným tokenem a stránka vás obodí — s počítadlem série, animovanými diagramy a finálním mini projektem, který se „natrénuje“ ve vašem prohlížeči.

Co je tenzor?
Tenzor je vícerozměrné pole — číslo, vektor, matice nebo víc. Vše uvnitř neuronové sítě jsou tenzory: vstupy, váhy, gradienty, výstupy. PyTorchí torch.Tensor je v podstatě jako ndarray z NumPy, navíc umí běžet na GPU a pamatovat si, jak vznikl (autograd).
Co je model?
Funkce s naučitelnými parametry. Vstupy proudí přes vrstvy (maticová násobení + nelinearity) a tvoří výstupy. Parametry upravujeme srovnáváním výstupů s pravdou, výpočtem gradientů a posunem vah z kopce dolů.
Trénování
Modelu ukážete mnoho dvojic vstup/cíl. Pro každou: dopředný průchod → výpočet ztráty → zpětný průchod (gradienty) → krok optimalizátoru (aktualizace vah). Opakujte tisíckrát. Křivka ztráty klesá. A to je vše. Tohle je hluboké učení.
Inference
Po natrénování zmrazte váhy a spouštějte jen dopředné průchody na nových vstupech. Rychlejší (žádné účetnictví gradientů) a deterministické. Použijte model.eval() a torch.no_grad().

Sestavte model — váš první nn.Module

Každý PyTorch model je podtřída nn.Module. Vrstvy definujete v __init__ a dopředný průchod v forward. To je celé API.

2-vrstvé MLP pro 10třídní klasifikaci (kompletně)
import torch
import torch.nn as nn

class Net(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1  = nn.Linear(784, 128)
        self.relu = nn.ReLU()
        self.fc2  = nn.Linear(128, 10)

    def forward(self, x):
        x = self.fc1(x)
        x = self.relu(x)
        x = self.fc2(x)
        return x

model = Net()

Ztráta a optimalizátor

Ztráta je číslo říkající „jak moc byla predikce špatná“ — menší je lepší. Optimalizátor používá gradienty, aby pošťouchl váhy a ztráta klesla. Dvě ztráty pokrývají 90 % případů; dva optimalizátory pokrývají 95 %.

ztráty
regrese nn.MSELoss — střední kvadratická chyba
klasifikace nn.CrossEntropyLoss — softmax + log + nll v jednom
binární nn.BCEWithLogitsLoss
optimalizátory
jednoduchý optim.SGD(params, lr=0.01)
dobrá výchozí volba optim.Adam(params, lr=1e-3)
moderní optim.AdamW — Adam + oddělený weight decay
dáme to dohromady
import torch.nn as nn
import torch.optim as optim

model     = Net()
loss_fn   = nn.CrossEntropyLoss()
optimizer = optim.Adam(model.parameters(), lr=1e-3)

Inference — použití natrénovaného modelu

Po trénování chcete model spouštět na nových vstupech bez výpočtu gradientů (rychlejší, méně paměti) a bez zlobení dropoutu / batchnormu. Toto zařídí dva přepínače.

šablona inference
model.eval()                          # dropout vypnutý, BN používá běžící statistiky
with torch.no_grad():                 # bez sledování gradientů
    logits = model(x_new)
    probs  = torch.softmax(logits, dim=-1)
    pred   = probs.argmax(dim=-1)        # identifikátor třídy

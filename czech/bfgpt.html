bfgpt — malý GPT v jednom souboru

bfgpt je maličký GPT, který přečtete za odpoledne a natrénujete na laptopu. Jeden ~800-řádkový Python soubor obsahuje vše — model, přípravu dat, trénink i vzorkování — s přesně dvěma závislostmi (torch + numpy). Je to věrný, zhuštěný port Karpathyho nanoGPT bez stahování GPT-2. Tokenizace je na úrovni znaků, takže žádný tokenizer se nestahuje — vytváří se přímo z vašeho textu.

soběstačný
Model + příprava dat + trénink + vzorkování v jediném bfgpt.py. Checkpoint dokonce obsahuje vlastní tokenizer, takže vzorkování nepotřebuje nic jiného.
dvě závislosti
Jen torch a numpy. Žádné transformers, žádný tiktoken, žádný framework k učení.
běží kdekoliv
--device auto detekuje cuda→mps→cpu. Výchozí nastavení funguje i na CPU.
znakový tokenizer
sorted(set(text)) je slovník. tiny-shakespeare dává 65 unikátních znaků.
JSONL vstup
Jeden {"text": "..."} na řádek. Na jiné pole ukažte pomocí --text-field.
~0.8M parametrů
Model 4 vrstvy / 4 hlavy / šířka 128 s block_size=64. Dost malý, aby blábolil Shakespeara za minuty.

Tokenizer — váš text je slovník

Žádný stažený tokenizer, žádné BPE sloučení, žádná tabulka 50 257 tokenů. bfgpt vezme setříděnou množinu unikátních znaků ve vašem korpusu a očísluje je. Toto mapování (stoi / itos) je celý tokenizer, uložený v meta.pkl. Pište níže a sledujte přesný cyklus encode → ids → decode, který model vidí.

písmenný tokenizer — sandbox

vocab_size 0 characters 0 tokens 0 dtype uint16
slovník — id:znak (␣ mezera · ⏎ nový řádek)
encode → token ids
decode(ids) → zpáteční cesta

  


  
CharTokenizer — celá třída

  

    
CharTokenizerbfgpt.py:247

    
class CharTokenizer:
    def __init__(self, stoi, itos):
        self.stoi = stoi
        self.itos = itos
        self.vocab_size = len(stoi)

    @classmethod
    def from_text(cls, text):
        chars = sorted(set(text))            # the vocabulary
        stoi  = {ch: i for i, ch in enumerate(chars)}
        itos  = {i: ch for i, ch in enumerate(chars)}
        return cls(stoi, itos)

    def encode(self, s):
        return [self.stoi[c] for c in s]   # str -> list[int]

    def decode(self, ids):
        return ''.join(self.itos[int(i)] for i in ids)

  


  

    Proč je to elegantní: abeceda vašich dat určuje velikost slovníku, id se vejdou do

    dlouhé (jeden token na znak), takže kontextové okno block_size počítá
    znaky, ne slova.

Šest triků, které to zprovozní

  

    
sdílení vah
wte.weight = lm_head.weight — vstupní embedding a výstupní projekce sdílejí jednu matici.

    
pre-norm
x + f(LN(x)) — normalizace před každou podvrstvou, ne po ní.

    
kauzální maska
pozice t může sledovat jen ≤ t; budoucnost je maskována na −∞.

    
flash attention
používá fúzovanou scaled_dot_product_attention je-li dostupná v PyTorch ≥ 2.0.

    
škálovaná inicializace
reziduální projekce se inicializují s std = 0.02 / √(2·n_layer).

    
selektivní váhový útlum
weight decay jen pro 2D tensory; biasy & LayerNormy jsou přeskočeny.

CausalSelfAttention — Q, K, V v jednom maticovém násobení

  

    
CausalSelfAttention.forwardbfgpt.py:80

    
def forward(self, x):
    B, T, C = x.size()
    q, k, v = self.c_attn(x).split(self.n_embd, dim=2)   # one matmul, split 3 ways
    k = k.view(B, T, self.n_head, C // self.n_head).transpose(1, 2)  # (B, nh, T, hs)
    q = q.view(B, T, self.n_head, C // self.n_head).transpose(1, 2)
    v = v.view(B, T, self.n_head, C // self.n_head).transpose(1, 2)

    if self.flash:
        y = F.scaled_dot_product_attention(q, k, v, is_causal=True)
    else:
        att = (q @ k.transpose(-2, -1)) * (1.0 / math.sqrt(k.size(-1)))
        att = att.masked_fill(self.bias[:, :, :T, :T] == 0, float('-inf'))
        att = F.softmax(att, dim=-1)
        y = att @ v
    y = y.transpose(1, 2).contiguous().view(B, T, C)
    return self.resid_dropout(self.c_proj(y))

  

  
Klíčová myšlenka: kauzální maska (nebo is_causal=True) je to, co z toho dělá dekodér — každá pozice předpovídá tu další pouze na základě minulosti.

GPT.forward — tréninková větev vs. inferenční větev

  

    
GPT.forwardbfgpt.py:183

    
def forward(self, idx, targets=None):
    b, t = idx.size()
    pos = torch.arange(0, t, device=idx.device)

    tok_emb = self.transformer.wte(idx)   # (b, t, n_embd)
    pos_emb = self.transformer.wpe(pos)   # (t, n_embd)
    x = self.transformer.drop(tok_emb + pos_emb)
    for block in self.transformer.h:
        x = block(x)
    x = self.transformer.ln_f(x)

    if targets is not None:                 # --- training ---
        logits = self.lm_head(x)
        loss = F.cross_entropy(logits.view(-1, logits.size(-1)),
                               targets.view(-1), ignore_index=-1)
    else:                                  # --- inference ---
        logits = self.lm_head(x[:, [-1], :])  # last position only
        loss = None
    return logits, loss

  

  
Dvě tváře téhož forward: s targets počítá ztrátu přes každou pozici (paralelní trénink); bez nich se stará jen o logity poslední pozice — to je vše, co vzorkování potřebuje.

Plán tempa učení — rozehřátí, pak kosinus

  

    

      
get_lrbfgpt.py:516

      
def get_lr(it):
    if it < warmup_iters:           # linear warmup
        return lr * (it+1) / (warmup_iters+1)
    if it > lr_decay_iters:         # floor
        return min_lr
    ratio = (it - warmup_iters) / (lr_decay_iters - warmup_iters)
    coeff = 0.5 * (1.0 + math.cos(math.pi * ratio))  # cosine
    return min_lr + coeff * (lr - min_lr)

    

    

      
Lineární nárůst po 100 iteracích až na vrchol
      learning_rate = 1e-3, pak kosinusový sestup na
      min_lr = 1e-4 do 2 000 iterací.

      
$\eta_t = \eta_{\min} + \tfrac{1}{2}\,(1+\cos\pi r)\,(\eta_{\max}-\eta_{\min})$

      
Každý krok také: forward → backward → ořezat normu gradientu na 1.0 → krok AdamW → zero_grad. Příští batch se přednačítá během backwardu, aby se překryl H2D přenos.

    

  







  
Generování — jeden token najednou

  

    Vzorkování je autoregresivní: předpověz další znak, připoj ho, vlož vše znovu, opakuj.
    Jak sekvence přeroste block_size, ořeže se zleva,
    takže kontextové okno posune dopředu.
  


  

    
GPT.generatebfgpt.py:226

    
@torch.no_grad()
def generate(self, idx, max_new_tokens, temperature=1.0, top_k=None):
    for _ in range(max_new_tokens):
        idx_cond = idx if idx.size(1) <= self.config.block_size \
                   else idx[:, -self.config.block_size:]   # crop to context
        logits, _ = self(idx_cond)
        logits = logits[:, -1, :] / temperature           # 1. reshape distribution
        if top_k is not None:
            v, _ = torch.topk(logits, min(top_k, logits.size(-1)))
            logits[logits < v[:, [-1]]] = -float('Inf')   # 2. keep top-k
        probs = F.softmax(logits, dim=-1)               # 3. to probabilities
        idx_next = torch.multinomial(probs, num_samples=1)  # 4. draw one
        idx = torch.cat((idx, idx_next), dim=1)           # 5. append & repeat
    return idx

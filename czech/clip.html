CLIP — obrázky a slova ve stejné místnosti

CLIP (Contrastive Language–Image Pre-training, OpenAI 2021) je model, který spustil multimodální zlatou horečku. Učí image encoder a text encoder umístit odpovídající páry obrázek–popisek na stejné místo ve sdíleném embedding prostoru — a nesouvisející páry daleko od sebe. Po natrénování můžete dělat zero-shot klasifikaci pouhým psaním názvů tříd jako vět.

dva enkodéry, jeden prostor
Image encoder (ViT-B/32 nebo ResNet-50) a text encoder (Transformer) každý vydají vektor. Malá projekční hlava na každé straně je mapuje do stejného d=512 embedding prostoru. Pak měříme kosinovou podobnost.
kontrastivní cíl
Pro dávku N párů obrázek–text sestavte N×N matici podobnosti. Diagonální buňky jsou skutečné shody (pozitivy); vše mimo diagonálu jsou negativy. Vytáhněte diagonálu nahoru, zatlačte mimo-diagonálu dolů — symetrická cross-entropy na řádky a sloupce.
proč na tom záleží
Tréninková data: 400M párů obrázek–text staženo z webu. Žádné lidské labely — popisky jsou labely. Výsledek: model, který rozpozná věci, kterým nikdy explicitně nebyl učen, jen jejich srovnáním s větou.
co pohání
CLIP guidance v DALL·E 2, text encoder Stable Diffusion (CLIP-L/14), zero-shot klasifikátory, hledání obrázků, moderaci obsahu, retrieval, image-to-image, text-to-image scoring, OpenCLIP, SigLIP, EVA-CLIP… všechny potomci.

Recept jedním dechem

# pseudokód (Radford et al., 2021)
I_f = image_encoder(I)            # [N, d_i]
T_f = text_encoder(T)             # [N, d_t]

I_e = l2_normalize(W_i @ I_f)     # [N, d]  projekce + normalizace
T_e = l2_normalize(W_t @ T_f)     # [N, d]

logits = (I_e @ T_e.T) * exp(t)   # [N, N] škálováno teplotou
labels = arange(N)                # diagonála je cíl
loss   = (CE(logits, labels) + CE(logits.T, labels)) / 2

Architektura — dvě věže, jedna místnost

CLIP je dual encoder. Dvě nezávislé věže každá promění svůj vstup na vektor pevné délky, malá lineární projekce je sjednotí na stejnou dimenzi, L2-normalizace, pak skalární součin = kosinová podobnost.

Napište architekturu

1 Dvě věže nesdílí nic, dokud nedojdou k projekci. Kosinová podobnost potřebuje, aby vektory žily ve stejném d-dim prostoru — to je to, co dělají W_i a W_t. Pojmenujte operaci, která činí magnitudy nepodstatné před skalárním součinem.
image_features = image_features / image_features.(dim=-1, keepdim=True)
text_features  = text_features  / text_features.norm(dim=-1, keepdim=True)
2 Původní CLIP image encoder používá vision transformery s patches velikosti 32 (ViT-B/32). Napište jméno třídy (case-sensitive).
image_encoder = (patch_size=32, width=768, layers=12)
3 Text encoder pooluje hidden state na pozici kterého speciálního tokenu?
# vezmeme features na eot embeddingu (eot_token je nejvyšší číslo v každé sekvenci)
x = x[torch.arange(x.shape[0]), text.argmax(dim=-1)]
# to je [] token
4 Po obou projekcích spočítáte matici párových podobností pro dávku. Image features [N, d], text features [N, d] — správná tensorová operace je:
logits_per_image = image_features @ text_features.   # [N, N]

Kontrastivní Loss — vytáhnout diagonálu nahoru

Uvnitř dávky N párů obrázek–popisek sestavte N×N matici podobnosti. Diagonální položky jsou jediné správné shody; zbytek jsou in-batch negativy. Loss je symetrická cross-entropy: řádky vyberou správný popisek pro každý obrázek, sloupce vyberou správný obrázek pro každý popisek.

L = ½ · ( CE(logits, labels) + CE(logitsᵀ, labels) ), labels = [0, 1, 2, …, N−1]

hrajte si s 4×4 dávkou — táhněte buňky, sledujte loss

Každá buňka je logit (podobnost × teplota) mezi obrázkem i (řádek) a textem j (sloupec). Klikněte na buňku a použijte ↑ / ↓ pro posun. Diagonální buňky jsou orámovány modře — tam chcete hmotu.

Cíl: dotlačte diagonály vysoko relativně k jejich řádku/sloupci. Loss = 0 znamená, že každý argmax řádku a každý argmax sloupce sedí na diagonále. Nenatrénovaný CLIP začíná blízko log(N) ≈ 1.39.

Teplota — jak ostrý je softmax?

Hrubé podobnosti I_e @ T_e.T žijí v [-1, 1] — příliš ploché pro užitečný softmax. CLIP násobí naučenou škálou exp(t), kde t je jeden skalární parametr (inicializován na log(1/0.07) ≈ 2.66).

P(text=j | image=i) = softmaxj( τ · cos(Ii, Tj) ), s τ = exp(t)

Zero-Shot — klasifikujte bez tréninku

Superschopnost CLIPu: proměňte jakoukoli klasifikační úlohu v problém srovnávání textů. Zabalte každý název třídy do větné šablony ("a photo of a {class}"), enkódujte je všechny jednou, pak pro libovolný obrázek: enkódujte jej, skalárně násobte s embeddingy tříd, softmax → hotovo. Drily Kódu — používání skutečného CLIPu

Tři reálné stacky. open_clip_torch je de-facto standard (lepší modely než původní release). transformers dodává HuggingFace CLIPModel. Oba obalují stejnou matematiku.

open_clip — načtení & embed

úplná reference
import torch, open_clip
from PIL import Image

model, _, preprocess = open_clip.create_model_and_transforms(
    "ViT-B-32", pretrained="laion2b_s34b_b79k")
tokenizer = open_clip.get_tokenizer("ViT-B-32")
model.eval()

img    = preprocess(Image.open("cat.jpg")).unsqueeze(0)
labels = ["a photo of a cat", "a photo of a dog"]
text   = tokenizer(labels)

with torch.no_grad():
    img_f = model.encode_image(img)
    txt_f = model.encode_text(text)
    img_f /= img_f.norm(dim=-1, keepdim=True)
    txt_f /= txt_f.norm(dim=-1, keepdim=True)
    probs = (100.0 * img_f @ txt_f.T).softmax(dim=-1)

print("probs", probs)   # [[0.98, 0.02]]

Boss — napište CLIP forward na jeden zátah

Závěrečná zkouška: každé políčko v tomto kódovém bloku je část CLIP forward + loss. Když jsou všechna zelená, stránka simuluje trénink 4×4 CLIP dávky a uvidíte, jak se matice podobnosti zostřuje na své diagonále.

dokončete CLIP tréninkový krok

11 políček. Klikněte na Trénovat, když jsou všechna zelená. 

import torch
import torch.nn.functional as F

# --- Forward přes oba enkodéry ----------------------------------
image_features = model.(images)        # [N, d]
text_features  = model.(texts)          # [N, d]

# --- L2-normalizace, aby skalární součin byl kosinová podobnost --
image_features = image_features / image_features.(dim=-1, keepdim=True)
text_features  = text_features  / text_features.(dim=-1, keepdim=True)

# --- Sestavte N × N similarity logits ---------------------------
logit_scale = model.logit_scale.()                 # τ = exp(t)
logits_per_image = logit_scale * image_features @ text_features.
logits_per_text  = logits_per_image.

# --- Symetrická cross-entropy na diagonálních cílech ------------
N      = images.shape[0]
labels = torch.(N, device=images.device)
loss_i = F.cross_entropy(logits_per_image, labels)
loss_t = F.cross_entropy(logits_per_text,  labels)
loss   = (loss_i + ) / 2

# --- Standardní PyTorch krok ------------------------------------
optimizer.zero_grad()
loss.()
optimizer.()

4×4 matice podobnosti zostřující se na diagonále

cyan = pozitiv (diagonála) · magenta = negativ · jasnější = vyšší logit
Natrénováno! Diagonála zaklapla. Skutečný CLIP běží přesně tuto smyčku po 32 epoch nad 400M páry na ~256 V100. Stejných pět magických řádků jako u běžného PyTorch modelu — jen zajímavější loss.

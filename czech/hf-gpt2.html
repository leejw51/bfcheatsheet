HuggingFace GPT-2 — natrénujte ho, tokenizujte ho, generujte s ním

GPT-2 is a decoder-only transformer trained to predict the next token. HuggingFace's transformers library wraps it in three lines of Python. This page walks you through the whole stack — pipelines, tokenizer, model, sampling knobs, and training loops — and makes you type the key bits to lock them in.

the four libraries
transformers models, tokenizers, pipelines, Trainer
datasets stream / map / cache training data
tokenizers fast Rust BPE / WordPiece
accelerate multi-GPU, mixed precision, no boilerplate
what is GPT-2?
A decoder-only transformer. 12 layers (124M), 24 (345M), 36 (774M), 48 (1.5B). Trained on WebText with one objective: predict the next token. A causal mask stops every position from peeking at the future. Tokenizer is byte-level BPE with a 50,257-token vocabulary.
two ways to use it
easy pipeline("text-generation", model="gpt2")
low-level AutoTokenizer + AutoModelForCausalLM + .generate()
The pipeline is great for demos. For real work, drop down to the model + tokenizer pair so you can control sampling, batching, and training.
decoder-only architecture
input ids → token + position embed
[batch, seq] → [batch, seq, 768]
↓
12 × decoder block
masked self-attn + MLP + LayerNorm + residual
↓
final LayerNorm
stabilises hidden states
↓
lm_head (weight-tied)
[..., 768] → [..., 50257] vocab logits

Rychlý start — generujte text na 3 řádcích

Two recipes. The pipeline hides everything; the explicit two-liner gives you full control. Both run the exact same model under the hood.

Recipe 1 — pipelines

simplest possible generation
from transformers import pipeline

gen = pipeline("text-generation", model="gpt2")
out = gen("The future of AI is", max_new_tokens=30, do_sample=True)
print(out[0]["generated_text"])

Recipe 2 — tokenizer + model

explicit two-liner
from transformers import AutoTokenizer, AutoModelForCausalLM

tokenizer = AutoTokenizer.from_pretrained("gpt2")
model     = AutoModelForCausalLM.from_pretrained("gpt2")

inputs  = tokenizer("The future of AI is", return_tensors="pt")
outputs = model.generate(**inputs, max_new_tokens=30, do_sample=True)
text    = tokenizer.decode(outputs[0], skip_special_tokens=True)
print(text)
Class lookup: AutoModelForCausalLM is the right head for autoregressive text generation. Other heads are AutoModelForSequenceClassification, AutoModelForMaskedLM (BERT-family), AutoModelForQuestionAnswering, etc.

Type the two-liner

Tokenizace — BPE most mezi řetězci a tenzory

GPT-2 doesn't see characters or words — it sees token IDs. Its byte-level BPE merges common subwords into single units (e.g. "ing"), keeps rare words as multiple tokens, and uses a leading space marker (renders as Ġ) so reversible round-trips are guaranteed.

Watch out: GPT-2 has no pad token by default. Before batching, do tokenizer.pad_token = tokenizer.eos_token — this reuses end-of-sequence as pad, which is the canonical fix for left-padding causal LMs.

canonical generation call
inputs  = tokenizer(prompt, return_tensors="pt")
outputs = model.generate(
    **inputs,
    max_new_tokens=50,
    do_sample=True,            # False = greedy/beam, True = stochastic
    top_k=50,                  # keep top-50 logits, mask rest
    top_p=0.9,                 # nucleus: smallest set whose probs sum to 0.9
    temperature=0.8,           # <1 sharper, >1 wilder
    repetition_penalty=1.1,    # discourage repeats
    pad_token_id=tokenizer.eos_token_id,
)
text = tokenizer.decode(outputs[0], skip_special_tokens=True)

Strategie samplingu — výběr dalšího tokenu

At every step the model emits a probability over the entire 50,257-token vocabulary. A strategy turns that distribution into one chosen token. Different strategies trade off coherence vs. creativity vs. compute cost.

strategyflagvibeuse when
Greedy do_sample=False deterministic, repetitive tasks with one right answer (translation in low-temp mode)
Beam search num_beams=4 safe, sometimes blandly fluent summarization, structured output
Sampling do_sample=True creative, can drift storytelling, brainstorm
top-k top_k=50 caps the worst tail pair with sampling for stability
top-p (nucleus) top_p=0.9 adapts to confidence almost always — modern default
temperature temperature=0.8 knob over sharpness 0.7–1.0 for chat, 0.2 for code, >1.2 for chaos
temperature intuition
Logits are divided by T before softmax. Low T sharpens (greedy-like), high T flattens (uniform-like). At T→0 you recover greedy. At T=1 you sample from the model's actual distribution.
top-k intuition
Throw away every token except the top k most likely. Renormalize and sample. Cuts the long tail of weird, rare tokens. Brittle: same k for a confident step and a flat one.
top-p (nucleus)
Keep the smallest set of tokens whose cumulative probability ≥ p. Adapts per-step: confident step → tiny set; flat step → broad set. Beats raw top-k on most tasks. p=0.9 is the safe default.
side by side
# 1. greedy
model.generate(**inputs, max_new_tokens=30, do_sample=False)

# 2. beam search
model.generate(**inputs, max_new_tokens=30, num_beams=4, early_stopping=True)

# 3. plain sampling
model.generate(**inputs, max_new_tokens=30, do_sample=True, temperature=1.0)

# 4. top-k
model.generate(**inputs, max_new_tokens=30, do_sample=True, top_k=50)

# 5. top-p (nucleus)
model.generate(**inputs, max_new_tokens=30, do_sample=True, top_p=0.9)

# 6. recommended combo
model.generate(**inputs, max_new_tokens=50,
               do_sample=True, top_k=50, top_p=0.9, temperature=0.8)
Practical advice: start with do_sample=True, top_p=0.9, temperature=0.8. If output is too random, lower T. If it loops, raise repetition_penalty to 1.1–1.3. If it sounds bland, bump T to 1.0 and remove top_k.

Path A — the Trainer API

end-to-end fine-tune of gpt2
from transformers import (
    AutoTokenizer, AutoModelForCausalLM,
    Trainer, TrainingArguments, DataCollatorForLanguageModeling,
)
from datasets import load_dataset

tokenizer = AutoTokenizer.from_pretrained("gpt2")
tokenizer.pad_token = tokenizer.eos_token

model = AutoModelForCausalLM.from_pretrained("gpt2")

ds = load_dataset("wikitext", "wikitext-2-raw-v1")

def tok(batch):
    return tokenizer(batch["text"], truncation=True, max_length=512)

ds = ds.map(tok, batched=True, remove_columns=["text"])

collator = DataCollatorForLanguageModeling(
    tokenizer=tokenizer,
    mlm=False,                          # causal LM, not masked LM
)

args = TrainingArguments(
    output_dir="./gpt2-ft",
    per_device_train_batch_size=4,
    num_train_epochs=3,
    learning_rate=5e-5,
    warmup_steps=100,
    logging_steps=50,
    save_steps=500,
    fp16=True,
)

trainer = Trainer(
    model=model,
    args=args,
    train_dataset=ds["train"],
    eval_dataset=ds["validation"],
    data_collator=collator,
)

trainer.train()
trainer.save_model("./gpt2-ft")

Path B — raw PyTorch loop

no Trainer, just torch
import torch
from torch.utils.data import DataLoader
from torch.optim import AdamW

loader    = DataLoader(ds["train"], batch_size=4, collate_fn=collator)
optimizer = AdamW(model.parameters(), lr=5e-5)
device    = "cuda" if torch.cuda.is_available() else "cpu"
model.to(device)
model.train()

for epoch in range(3):
    for batch in loader:
        batch = {k: v.to(device) for k, v in batch.items()}
        outputs = model(**batch)             # labels=input_ids handled by collator
        loss    = outputs.loss               # transformers returns CE loss
        loss.backward()
        optimizer.step()
        optimizer.zero_grad()
Loss = labels = input_ids: for causal LMs, the input and the target are the same token sequence — the model learns to predict each token from the ones before it. DataCollatorForLanguageModeling(mlm=False) sets up the labels for you.

TrainingArguments — každý ovladač, který stojí za to znát

The TrainingArguments dataclass holds every hyperparameter the Trainer touches. There are 100+ fields. These dozen are the ones you'll actually set.

argumenttypicalwhat it does
output_dir "./out" where checkpoints + logs land (required)
per_device_train_batch_size 4–32 batch per GPU; effective batch = this × num_gpus × grad_accum
gradient_accumulation_steps 1–16 fake bigger batches when GPU memory is tight
num_train_epochs 1–10 passes over the training set
learning_rate 5e-5 standard for fine-tuning transformers
warmup_steps 100–1000 linear ramp-up — prevents early loss spikes
weight_decay 0.01 AdamW regularization
fp16 / bf16 True mixed precision; bf16 on Ampere+, fp16 elsewhere
logging_steps 50 print loss every N steps
save_steps 500 checkpoint every N steps
evaluation_strategy "steps" "no" / "steps" / "epoch"
load_best_model_at_end True after training, restore the best checkpoint
report_to "wandb" or "tensorboard" / "none"
production-ish config
args = TrainingArguments(
    output_dir="./gpt2-ft",
    per_device_train_batch_size=8,
    gradient_accumulation_steps=4,         # effective batch = 32
    num_train_epochs=3,
    learning_rate=5e-5,
    warmup_steps=200,
    weight_decay=0.01,
    fp16=True,
    logging_steps=50,
    save_steps=500,
    evaluation_strategy="steps",
    eval_steps=500,
    load_best_model_at_end=True,
    report_to="none",
)
Effective batch size: per_device_train_batch_size × num_gpus × gradient_accumulation_steps. A 24 GB GPU might fit batch 8; using grad-accum 4 gives the loss curve of batch 32.

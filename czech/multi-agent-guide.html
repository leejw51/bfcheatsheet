čtyři způsoby, jak rozběhnout Claude Code paralelně.
SOUBOR: multi-agent-guide.html

~/repo — claude --multi-agent

Jedno okno Claude je už malý tým. Čtyři okna jsou studio. Tato příručka ukazuje čtyři podoby, které ten tým může mít — a kdy je která ta správná.

Volte podle koordinace, ne podle počtu hlav. Vedlejší úkol → sub-agent. Nezávislé → agent view. Koordinované → tým. Úplná izolace → worktree na terminál.

přístup 01 · delegace v rámci relace

Sub-agenti.

Specializovaný pracovník, který běží v vlastním kontextovém okně, dokončí zaměřený úkol a vrátí shrnutí volající relaci. Použijte je k udržení hlavního vlákna čistého — výzkum, ověření, průzkum.

„Smyslem není další agent. Smyslem je udržet vaši hlavní konverzaci nezahlcenou.“

Definujte je jako Markdown soubory v .claude/agents/. Každý dostane vlastní system prompt, allowlist nástrojů a (volitelně) menší model, takže je práce levná.

kontext · izolovaný paralelně · ano konflikty souborů · sdílený cwd vhodné pro · výzkum, audity
.claude/agents/researcher.md 
---
name: Researcher
description: Finds prior art & cites sources
model: haiku
tools:
  - bash
  - read
  - webfetch
---

# You are an expert researcher.
# Return citations as a bullet list.
prompt · hlavní relace 
$ /agents
  → manage definitions

$ Have @Researcher dig up the rate-limit RFC, then summarize.
→ spawning Researcher in isolated context…
← "Found 3 sources, see notes/rfc-rate-limit.md"
přístup 02 · dashboard na pozadí

Agent view.

Přehled nezávislých relací běžících na pozadí. Pošlete prompt, odejděte, později se mrkněte na výstup. Každý řádek je vlastní proces a přežije zavření terminálu.

„Vystřel-a-zkontroluj-později. Užitečné, když máte pět malých úkolů a jeden mozek.“

Otevřete pomocí claude agents. Uvnitř dashboardu: Space pro náhled, Enter pro připojení, ← pro odpojení.

kontext · na relaci koordinace · vy persistence · přežije zavření vhodné pro · paralelní rutiny
shell 
# dispatch a background session
$ claude --bg "investigate the flaky e2e test"

# dispatch with a custom agent
$ claude --agent code-reviewer --bg "review PR #1234"

# open the dashboard
$ claude agents

│ ▸ pr-1234-review   running     38s │
│ ▸ docs-cleanup     done      11m 02s │

přístup 03 · experimentální · koordinovaný

Týmy agentů.

Více relací pracujících na stejném cíli se sdíleným seznamem úkolů, zprávami mezi agenty a leadem řídícím práci. Blíže studiu než pracovnímu poolu.

„Jeden reviewer pro bezpečnost, jeden pro výkon, jeden pro testy — a ti se mezi sebou baví.“

Experimentální, ve výchozím stavu vypnuté. Povolte v settings.json, pak požádejte Claude, aby vytvořil tým pro váš cíl.

kontext · na člena týmu koordinace · vestavěná status · experimentální vhodné pro · vícestrannou recenzi
settings.json 
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
prompt 
$ Spin up a team to review PR #142.
  Three reviewers, working in parallel:
    1. security  — auth, input, secrets
    2. perf     — hot paths, allocs, queries
    3. tests    — coverage, flakiness, mocks
  Lead consolidates & posts one review.

→ team:r142 spawned with 3 teammates
→ shared task list created (12 items)
← consolidated review posted to #142
přístup 04 · ten jednoduchý · vřele doporučeno

Mnoho terminálů.

Otevřete další tab terminálu. Spusťte claude. Hotovo — máte dva agenty. Spojte to s git worktrees a získáte skutečnou izolaci: každý agent na vlastní větvi a working tree, žádné konflikty editace.

„Nejčistší vzorec je zároveň nejhloupější: víc tabů.“

Nejlepší, když jsou úkoly dlouhé, nesouvisející a chcete plnou viditelnost do toho, co každý agent dělá. Sloučení skrze PR až nakonec.

nastavení · žádné izolace · plná (worktree) viditelnost · úplná vhodné pro · paralelní featury
shell · worktree-na-agenta 
# from main repo
$ git worktree add ../repo-feat-a -b feat-a
$ git worktree add ../repo-feat-b -b feat-b

# terminal 1 — feature A
$ cd ../repo-feat-a && claude

# terminal 2 — feature B
$ cd ../repo-feat-b && claude

# merge later via PR ─ no conflicts
tmux · varianta split-pane 
$ tmux new -s squad
  ctrl-b "       split horizontal
  ctrl-b %       split vertical
  ctrl-b o       cycle panes

# 4 panes · 4 worktrees · 4 claudes

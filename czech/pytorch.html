Automatická diferenciace. Důvod, proč PyTorch existuje.

🔄 Tok gradientů

Dopředný průchod → počítá výstupy. Zpětný průchod ← propaguje ∂L/∂x řetízkovým pravidlem.

dopředná aktivace gradient tekoucí zpět
Past  Gradienty se akumulují mezi voláními backward(). Vždy volej optimizer.zero_grad() (nebo předej set_to_none=True) před každým krokem, jinak se ti aktualizace promíchá s gradienty z různých dávek.
requires_grad & backward
Označ vstupy, pro které chceš gradienty, a zavolej .backward() na skalární loss.

y = (x ** 2).sum()
y.backward()
print(x.grad)   # tensor([2., 4.])

custom Function
Definuj si vlastní forward/backward pro speciální operace.
class MyReLU(torch.autograd.Function):
    @staticmethod
    def forward(ctx, x):
        ctx.save_for_backward(x)
        return x.clamp(min=0)

    @staticmethod
    def backward(ctx, grad_out):
        (x,) = ctx.saved_tensors
        return grad_out * (x > 0)

GradScaler (fp16)
Škáluje ztrátu, aby drobné gradienty přežily fp16 → před step() škálu odstraní.
scaler = torch.amp.GradScaler("cuda")

for x, y in loader:
    optimizer.zero_grad(set_to_none=True)
    with torch.autocast("cuda", dtype=torch.float16):
        loss = loss_fn(model(x), y)
    scaler.scale(loss).backward()
    scaler.unscale_(optimizer)             # pro ořez gradientu
    torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
    scaler.step(optimizer)
    scaler.update()

Rychlá rozhodovací pravidla.
# DDP — model se vejde na 1 GPU.
#   + nejjednodušší, nejrychlejší na krok
#   + každý rank má plný model (snadný eval/save)
#   - paměť: O(model) na rank

Časté chyby

Zapomenutý zero_grad: gradienty se akumulují s každým voláním backward(). Vždy resetuj.
view vs reshape: po transpose / permute není tenzor souvislý — view vyhodí chybu. Použij reshape nebo nejdřív .contiguous().
CrossEntropyLoss bere logity, ne výstupy softmaxu. Aplikování softmaxu předem dá špatné gradienty.
BatchNorm v eval(): používají se running statistiky, ne dávkové. Zapomenutí model.eval() při inferenci způsobí, že výstupy závisí na složení dávky.
tensor.item() funguje jen pro tenzory s jedním prvkem a vynutí CPU synchronizaci — nepoužívej v horké smyčce.

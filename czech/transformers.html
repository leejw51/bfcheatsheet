úroveň 01 // motivace

§01Proč transformery?

Před rokem 2017 byly dominantními modely sekvencí RNN a LSTM. Zpracovávaly tokeny jeden po druhém a nesly skrytý stav vpřed. To mělo tři ochromující problémy:

  • Sekvenční úzké hrdlo. Nelze paralelizovat napříč časovou dimenzí — token t potřebuje token t-1.
  • Mizející gradienty na dálku. Informace z tokenu 1 musí projít každým krokem, než dorazí k tokenu 100.
  • Skrytý stav pevné velikosti. Celá minulost se musí vměstnat do jednoho vektoru. Paměť na dlouhé vzdálenosti je křehká.
klíčový vhled

Pozornost je vše, co potřebujete

Vaswani a kol. se zeptali: co kdyby každý token mohl pohlédnout na každý jiný token přímo, v jednom paralelním kroku? Žádná rekurence. Žádná konvoluce. Jen naučený mechanismus, který každé pozici umožňuje věnovat pozornost každé jiné pozici s naučenou vahou.

Výsledek: trénování, které škáluje lineárně s hardwarem, gradienty tekoucí v O(1) skocích a model, který v principu může vztáhnout první slovo odstavce k poslednímu stejně snadno jako k jeho sousedovi.

úroveň 02 // vstup

§02Tokenizace a vkládání

Transformer neumí číst text. Čte celočíselné identifikátory. Tokenizace je ztrátový kompresní postup, který mění řetězec znaků na posloupnost identifikátorů tokenů z pevného slovníku (typicky 30–200 tisíc položek). Moderní tokenizéry (BPE, WordPiece, Unigram, Tiktoken) se dělí na podslovní kusy, takže vzácná slova se stanou posloupností běžných úseků namísto neznámého tokenu.

Každý celočíselný identifikátor je pak namapován na hustý vektor přes matici vkládání $E \in \mathbb{R}^{V \times d_{model}}$, kde $V$ je velikost slovníku. Vyhledání = jeden řádek matice. Výsledek: pro posloupnost $L$ tokenů dostanete tenzor tvaru [L, d_model].

Pozn.: tohle je hračkářský tokenizér na bílé znaky a interpunkci. Skutečné BPE by rozdělilo tokenization na něco jako token + ization.

úroveň 03 // pořadí

§03Polohové kódování

Sebepozornost je permutačně invariantní: zamíchejte vstupní tokeny a (bez polohové informace) dostanete stejný výstup. To je problém pro jazyk. Řešení: před vstupem do stacku přidejte ke každému vkládání tokenu vektor závisející na pozici.

Původní článek používá sinusoidy s geometricky rostoucí vlnovou délkou:

$$PE_{(pos,\,2i)} = \sin\!\Big(\tfrac{pos}{10000^{2i/d_{model}}}\Big), \qquad PE_{(pos,\,2i+1)} = \cos\!\Big(\tfrac{pos}{10000^{2i/d_{model}}}\Big)$$

vizualizér

Sinusoidní polohové vlny

Každý řádek je jedna dimenze i. Nižší dimenze oscilují rychle (pozice vypadají velmi odlišně); vyšší dimenze jsou hladké (pořadí ve velkém měřítku).

Chytrá vlastnost: $PE_{pos+k}$ je pevná lineární funkce $PE_{pos}$, takže se model může snadno učit relativní posuny. Moderní varianty (RoPE, ALiBi, naučené absolutní) nabízejí různé kompromisy, ale cíl je stejný — vnést pořadí.

úroveň 04 // jádro

§04Sebepozornost (Q · K · V)

Každý vstupní token je promítnut do tří vektorů přes tři naučené matice $W^Q, W^K, W^V \in \mathbb{R}^{d_{model} \times d_k}$:

  • Dotaz / Query (Q) — „co hledám?“
  • Klíč / Key (K) — „co obsahuji?“
  • Hodnota / Value (V) — „čím přispěji, pokud mi bude věnována pozornost?“

Škálovaná skalárně-součinová pozornost je pak:

$$\mathrm{Attention}(Q,K,V) = \mathrm{softmax}\!\Big(\tfrac{QK^\top}{\sqrt{d_k}}\Big)\,V$$

Tvary pro délku sekvence $L$ a dimenzi hlavy $d_k$: Q: [L, d_k] K: [L, d_k] V: [L, d_k]. Součin $QK^\top$ má tvar [L, L] — jedno skóre podobnosti na pár (dotaz, klíč). Softmax po řádcích z toho udělá váhy pozornosti, a pak znovu zvážíme $V$.

úroveň 05 // ansámbl

§05Vícehlavá pozornost

Jeden vzor pozornosti je málokdy dostačující. Vícehlavá pozornost spouští $h$ paralelních operací pozornosti, každá s vlastními projekcemi $W_i^Q, W_i^K, W_i^V$, a výsledky zřetězí:

$$\mathrm{MHA}(X) = \mathrm{Concat}(\mathrm{head}_1, \ldots, \mathrm{head}_h)\,W^O$$

Každá hlava se učí věnovat pozornost jinému relačnímu vzoru — jedna hlava může sledovat syntaktické závislosti, jiná koreference, další jednoduché posuny pozic. Rozdělení udržuje celkový výpočet zhruba shodný s jednou hlavou: u $h=8$ hlav volíme $d_k = d_{model}/h$.

úroveň 06 // tmel

§06Feed-forward · reziduum · norma

Po pozornosti prochází každá pozice samostatně skrz dopřednou síť — dvě lineární vrstvy s nelinearitou mezi nimi (původně ReLU, dnes obvykle GELU nebo SwiGLU):

$$\mathrm{FFN}(x) = \max(0,\, xW_1+b_1)\,W_2 + b_2$$

Šířka skryté vrstvy je typicky 4·d_model. Tady sídlí většina parametrů modelu — a populární hypotéza je, že tato vrstva funguje jako paměť klíč–hodnota uchovávající faktické znalosti.

Reziduální spojení

Jak vrstva pozornosti, tak vrstva FFN jsou obaleny reziduálními spojeními + vrstvovou normalizací:

$$x \leftarrow \mathrm{LayerNorm}(x + \mathrm{Sublayer}(x))$$

Proč? Reziduály vytvářejí gradientovou dálnici od ztráty zpět k vstupu, což stovkám vrstev umožňuje stabilní trénink. LayerNorm udržuje aktivace centrované a škálované napříč rysovou dimenzí, čímž odděluje optimalizaci jednotlivých vrstev.

Moderní varianty preferují pre-norm: $x \leftarrow x + \mathrm{Sublayer}(\mathrm{LayerNorm}(x))$ — dává stabilnější trénink do velké hloubky.

úroveň 07 // topologie

§07Enkodér · dekodér · maskování

Dnes dominují tři architektonické varianty:

  • Pouze enkodér (BERT) — obousměrná sebepozornost. Každý token vidí každý jiný token. Skvělé pro úlohy porozumění (klasifikace, vyhledávání).
  • Pouze dekodér (GPT) — kauzální/maskovaná sebepozornost. Token t může věnovat pozornost pouze tokenům $\le t$. Skvělé pro generování. Tohle používá většina moderních LLM.
  • Enkodér-dekodér (T5, původní NMT) — enkodér čte zdroj, dekodér generuje cíl a přitom využívá křížovou pozornost na výstupu enkodéru.

Kauzální maskování — jak?

Před softmaxem nastavte horní trojúhelníkové prvky matice skóre na $-\infty$. Softmax z $-\infty$ je 0 — model je tak matematicky neschopen pohlédnout do budoucnosti:

úroveň 08 // cíl

§08Trénink: predikce dalšího tokenu

U LLM jen s dekodérem (rodina GPT) je celý trénovací cíl šokujícím způsobem jednoduchý:

Při daných tokenech 1..t predikujte token t+1.

Model produkuje pravděpodobnostní rozdělení nad slovníkem na každé pozici. Ztráta je průměrná křížová entropie proti skutečnému dalšímu tokenu:

$$\mathcal{L} = -\frac{1}{N}\sum_{t=1}^{N} \log P_\theta(x_{t+1} \mid x_1, \ldots, x_t)$$

Kauzální maskování to dělá efektivním: jediný průchod vpřed nad sekvencí délky $N$ vrátí $N$ predikcí paralelně, jednu na pozici. Krom posunutí cílů o jedna nejsou žádné triky s teacher-forcingem potřeba.

Vše ostatní — instrukční doladění, RLHF, použití nástrojů — sedí nad tímto cílem. Předtrénování je část, která vstřebává data v internetovém měřítku a kde se model učí „všechno“. Doladění je část, která tvaruje jeho chování.

shrnutí

Celá pipeline

  • 1. Tokeny → celočíselné identifikátory → vkládání (+ polohové kódování)
  • 2. Stack N transformer bloků: MHA → Add&Norm → FFN → Add&Norm
  • 3. Závěrečná lineární projekce na velikost slovníku, softmax → rozdělení dalšího tokenu
  • 4. Ztráta křížové entropie proti pravdivému dalšímu tokenu, sečtená přes pozice
  • 5. Zpětná propagace přes každou váhu. Opakujte ~10²² krát. Hotovo.

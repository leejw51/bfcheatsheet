Un tenseur, jusqu'aux logits

Le transformer Attention Is All You Need est une recette : prendre un tenseur d'identifiants de tokens sources, le pousser dans un encodeur, pousser les tokens cibles dans un décodeur qui effectue une cross-attention vers l'encodeur, projeter sur le vocabulaire, et obtenir les logits du prochain token. Cette page suit la forme — chaque étape des IDs d'entrée jusqu'aux probabilités finales [B, T_tgt, V], colorées pour lire le papier sans jamais perdre le tenseur.

valeurs par défaut du papier (base)
B (batch) = 2 # taille du batch
T_src (src len) = 8 # tokens source
T_tgt (tgt len) = 7 # tokens cible
d_model = 512 # dim. embedding
h (heads) = 8 # têtes d'attention
d_k = d_v = 64 # d_model / h
d_ff = 2048# dim. interne FFN (4·d_model)
V (vocab) = 37000# vocab BPE EN→DE
N (layers) = 6 # couches enc = couches dec
légende des couleurs (utilisée partout)
B batch T tokens d_model h têtes d_k d_ff

Quand une dimension change entre deux étapes, elle s'illumine. Observez le même tenseur se diviser en têtes, se multiplier, se mettre à l'échelle, passer par le softmax, puis se recoudre.

Étapes 1–3 · Des IDs de tokens à un tenseur prêt pour le contexte

Avant toute attention, l'entrée doit devenir un tenseur de vecteurs. Trois étapes rapides : IDs entiers en entrée, vecteurs réels en sortie, avec l'info positionnelle intégrée par addition.

étape 1 · IDs de tokens
Sortie du tokenizer. Deux exemples dans un batch, huit tokens chacun.
$\text{ids}\in\mathbb{Z}^{B\times T}$
shape [B=2, T=8] · dtype: int32 
# deux phrases, chacune paddée/tronquée à T=8 tokens
ids = [[ 101, 2057, 2293, 3019, 2025, 2748, 102, 0 ],
       [ 101, 7592, 2088,  102,    0,    0,   0, 0 ]]
# forme : [B=2, T=8]
étape 2 · lookup embedding

Une matrice de poids $E\in\mathbb{R}^{V\times d_{model}}$ contient une ligne par entrée du vocabulaire. Lookup par ID. Le tenseur gagne une dimension : un vecteur à valeurs réelles de taille d_model remplace chaque entier.

$X = E[\text{ids}] \;\in\; \mathbb{R}^{B\times T\times d_{model}}$
[2, 8] → [2, 8, 512] 
import torch.nn as nn
embed = nn.Embedding(num_embeddings=30522, embedding_dim=512)
X = embed(ids)            # [B, T] -> [B, T, d_model]
print(X.shape)            # torch.Size([2, 8, 512])

Nombre de params : $V \cdot d_{model} = 30522 \cdot 512 \approx 15.6\text{M}$ — la table d'embedding seule est plus grande que le reste de n'importe quelle couche individuelle.

étape 3 · + encodage positionnel

La self-attention est équivariante par permutation — sans info de position, « the cat sat » et « sat cat the » produisent les mêmes sorties. Le papier ajoute un motif sinusoïdal fixe de même forme que $X$.

$\mathrm{PE}_{(pos, 2i)} = \sin\!\big(pos / 10000^{2i/d_{model}}\big)$
$\mathrm{PE}_{(pos, 2i+1)} = \cos\!\big(pos / 10000^{2i/d_{model}}\big)$
[2, 8, 512] + [8, 512] → [2, 8, 512] 
# PE a la forme [T, d_model], se diffuse sur B
X = X + PE[:T]            # [B, T, d_model] + [T, d_model] -> [B, T, d_model]

Même forme en entrée, même forme en sortie. L'information va dans les valeurs, pas dans les dims.

Fin de l'étape 3 : nous avons maintenant un tenseur de forme [B, T, d_model] qui est à la fois sur les mots et sur leur position. Chaque étape ultérieure est une transformation sur cette structure.

Étapes 4–11 · Attention multi-têtes, dimension par dimension

La raison d'être de ce papier. Chaque token devient une requête, une clé et une valeur ; les similarités sont calculées, mises à l'échelle, passées par softmax et utilisées pour mélanger les valeurs. Nous redécoupons aussi deux fois : une fois pour séparer les têtes, une fois pour les recoller.

étape 4 · projections Q, K, V

Trois projections linéaires indépendantes, chacune $\mathbb{R}^{d_{model}}\to\mathbb{R}^{d_{model}}$ :

$Q = X W^Q,\quad K = X W^K,\quad V = X W^V$
X [2, 8, 512] Wᵠ [512, 512] → Q [2, 8, 512] 
# trois couches linéaires, pas de biais dans le papier mais c'est acceptable en pratique
Wq = nn.Linear(d_model, d_model)   # 512*512 = 262 144 params
Wk = nn.Linear(d_model, d_model)
Wv = nn.Linear(d_model, d_model)

Q = Wq(X)                              # [B, T, d_model]
K = Wk(X)                              # [B, T, d_model]
V = Wv(X)                              # [B, T, d_model]

Trois matrices de forme $[d_{model}, d_{model}]$ — total des params ici : $3\cdot 512\cdot 512 = 786\,432$. Dans certaines implémentations ces trois sont fusionnées en une seule matrice $[d_{model}, 3\cdot d_{model}]$.

étape 5 · redécoupage & transposition pour h têtes

Diviser le dernier axe en $h$ morceaux de $d_k = d_{model}/h$ chacun, puis transposer pour que les têtes deviennent une dimension extérieure. Aucun nouveau paramètre — juste view + permute.

[2, 8, 512] view→ [2, 8, 8, 64] transpose(1,2)→ [2, 8, 8, 64] 
B, T, d_model = Q.shape
h, d_k = 8, d_model // h         # 8, 64

# diviser le dernier axe en (h, d_k), puis déplacer les têtes à côté du batch
Q = Q.view(B, T, h, d_k).transpose(1, 2)   # [B, h, T, d_k]
K = K.view(B, T, h, d_k).transpose(1, 2)   # [B, h, T, d_k]
V = V.view(B, T, h, d_k).transpose(1, 2)   # [B, h, T, d_k]
Pourquoi les têtes sont importantes : avec $h=8$ sous-espaces $[T, d_k=64]$ séparés, chaque tête peut se spécialiser sur un motif relationnel différent. Le coût de calcul est identique : 8 petites attentions de dimension $64$ au lieu d'une seule de dimension $512$.
étape 6 · Q @ Kᵀ — similarité deux à deux

Pour chaque tête, chaque requête assiste chaque clé. Le matmul contracte l'axe $d_k$, produisant une matrice de scores $T\times T$ par tête et par batch.

$\text{scores} = Q K^\top$
Q [2, 8, 8, 64] Kᵀ [2, 8, 64, 8] → scores [2, 8, 8, 8] 
scores = Q @ K.transpose(-2, -1)   # [B, h, T, T]
# entrée (b, tête, i, j) = dans quelle mesure le token i assiste le token j (dans cette tête)
étape 7 · mise à l'échelle par 1/√d_k

Sans mise à l'échelle, les produits scalaires de vecteurs de dimension $d_k$ croissent comme $\sqrt{d_k}$ — de grandes valeurs poussent le softmax dans le régime de saturation où les gradients s'évanouissent. Diviser chaque score par $\sqrt{d_k}$.

$\text{scaled} = \text{scores} / \sqrt{d_k} = \text{scores} / 8$
[2, 8, 8, 8] ÷ √64 [2, 8, 8, 8] forme inchangée 
import math
scores = scores / math.sqrt(d_k)   # 1/√64 = 1/8 = 0,125
étape 8 · softmax sur le dernier axe

Transformer les scores bruts en distribution de probabilité par requête. Chaque ligne du bloc $[T,T]$ somme à 1 — ce sont les poids d'attention.

$A = \mathrm{softmax}(\text{scaled},\ \text{dim}=-1)$
[2, 8, 8, 8] → [2, 8, 8, 8] les lignes somment à 1 
import torch.nn.functional as F
A = F.softmax(scores, dim=-1)    # [B, h, T, T]

Pour la self-attention du décodeur, un masque additif $-\infty$ est appliqué avant le softmax pour imposer la causalité.

étape 9 · attention @ V

Mélanger les vecteurs de valeurs en utilisant les poids d'attention. La sortie de chaque requête est une somme pondérée de tous les $T$ vecteurs de valeurs. L'axe de contraction est le $T$ intérieur.

A [2, 8, 8, 8] V [2, 8, 8, 64] → out [2, 8, 8, 64] 
head_out = A @ V                  # [B, h, T, d_k]
étape 10 · concaténation des têtes (transposition + redécoupage)

Recoller les têtes en un grand dernier axe. transpose(1,2) déplace les têtes à côté de $d_k$, puis reshape les fusionne. Sans paramètre.

[2, 8, 8, 64] transpose→ [2, 8, 8, 64] reshape→ [2, 8, 512] 
out = head_out.transpose(1, 2).contiguous()   # [B, T, h, d_k]
out = out.reshape(B, T, d_model)              # [B, T, d_model]
étape 11 · projection de sortie W_O

Une projection linéaire $\mathbb{R}^{d_{model}\to d_{model}}$ de plus. Même forme, mais permet au modèle de mélanger l'information entre les têtes après la concaténation.

$\text{MHA}(X) = \mathrm{Concat}(\text{head}_1, \dots, \text{head}_h)\,W^O$
[2, 8, 512] ·W_O [2, 8, 512] 
Wo = nn.Linear(d_model, d_model)   # 262 144 params supplémentaires
mha_out = Wo(out)                       # [B, T, d_model]

Total params MHA : $4 \cdot d_{model}^2 = 4\cdot 512^2 = 1{,}048{,}576$ (Q, K, V, O — chacun $d_{model}\times d_{model}$).

Fin de l'attention : on repart avec la forme [B, T, d_model] — exactement la même forme qu'en entrée. L'attention est un opérateur à forme constante.

Étapes 12–16 · LayerNorm, FFN et l'autoroute résiduelle

Après l'attention, le bloc encodeur assemble tout avec deux connexions résiduelles, deux LayerNorms, et un FFN position-wise. Le FFN concentre la plupart des paramètres — et c'est là que, selon beaucoup, les connaissances factuelles sont stockées.

étape 12 · résiduel + LayerNorm

Additionner le $X$ original (connexion de saut), puis normaliser par token sur $d_{model}$.

$X' = \mathrm{LayerNorm}(X + \mathrm{MHA}(X))$
[2, 8, 512] + [2, 8, 512] → LN → [2, 8, 512] 
ln1 = nn.LayerNorm(d_model)      # 2 * d_model = 1024 params (γ, β)
X_post = ln1(X + mha_out)         # [B, T, d_model]

LN normalise sur le dernier axe : chaque vecteur $[d_{model}]$ devient moyenne-0, var-1, puis remis à l'échelle par $\gamma, \beta\in\mathbb{R}^{d_{model}}$ appris. Les positions de batch et de tokens restent indépendantes.

étapes 13–14 · FFN linéaire 1 + ReLU

Un réseau feed-forward position-wise : le même MLP appliqué à chaque token indépendamment. Le premier linéaire élargit $d_{model} \to d_{ff}$ (typiquement $4\cdot d_{model}$).

$H = \mathrm{ReLU}(X' W_1 + b_1)$
[2, 8, 512] ·W₁ [2, 8, 2048] → ReLU → [2, 8, 2048] 
W1 = nn.Linear(d_model, d_ff)       # 512 * 2048 = 1 048 576 params
H = F.relu(W1(X_post))            # [B, T, d_ff]
étape 15 · FFN linéaire 2

Le second linéaire projette en retour : $d_{ff} \to d_{model}$. Restaure la forme canonique pour que le bloc suivant puisse repartir de zéro.

$Y = H W_2 + b_2$
[2, 8, 2048] ·W₂ [2, 8, 512] 
W2 = nn.Linear(d_ff, d_model)       # 2048 * 512 = 1 048 576 params supplémentaires
ffn_out = W2(H)                       # [B, T, d_model]

Total params FFN : $2\cdot d_{model}\cdot d_{ff} = 2\cdot 512\cdot 2048 = 2{,}097{,}152$ — environ deux fois le coût des quatre projections d'attention combinées. C'est là que les grands modèles placent leur poids.

étape 16 · résiduel + LayerNorm (à nouveau)

Même schéma qu'à l'étape 12. Additionner le tenseur pré-FFN, normaliser par couche, et le bloc encodeur est terminé.

$Y' = \mathrm{LayerNorm}(X' + Y)$
[2, 8, 512] → [2, 8, 512] block-output shape = block-input shape 
ln2 = nn.LayerNorm(d_model)
out = ln2(X_post + ffn_out)       # [B, T, d_model]
Empilement & répétition : le papier empile $N=6$ blocs encodeurs identiques. La forme d'entrée de chacun est égale à sa forme de sortie, ce qui explique pourquoi ils se composent proprement. Params totaux par bloc encodeur (base) : environ $4\cdot 512^2 + 2\cdot 512\cdot 2048 + 4\cdot 512 \approx 3{,}15$M.

Le bloc encodeur entier en 14 lignes

EncoderBlock — PyTorch minimal
class EncoderBlock(nn.Module):
    def __init__(self, d_model=512, h=8, d_ff=2048):
        super().__init__()
        self.attn = nn.MultiheadAttention(d_model, h, batch_first=True)
        self.ffn  = nn.Sequential(
            nn.Linear(d_model, d_ff), nn.ReLU(), nn.Linear(d_ff, d_model)
        )
        self.ln1 = nn.LayerNorm(d_model)
        self.ln2 = nn.LayerNorm(d_model)

    def forward(self, x):
        # x : [B, T, d_model]
        a, _ = self.attn(x, x, x)
        x = self.ln1(x + a)
        x = self.ln2(x + self.ffn(x))
        return x                       # [B, T, d_model]

Étapes 17–21 · Le bloc décodeur

Le décodeur prend les tokens cibles (décalés d'un rang à droite) et produit un tenseur qui connaît à la fois (a) les tokens cibles vus jusqu'ici — via la self-attn masquée — et (b) la source — via la cross-attn, où Q vient du décodeur et K, V viennent de la sortie de l'encodeur (la “mémoire”). Même dim d'embedding, même discipline résiduelle+LN que l'encodeur. Trois sous-blocs par couche, six couches dans le papier.

config du papier (inchangée)
B = 2, T_src = 8, T_tgt = 7
d_model = 512, h = 8, d_k = 64
# T_tgt est indépendant de T_src.
# la cross-attn est le seul endroit où ils se rencontrent.
pourquoi « décalé à droite » ?

Pendant l'entraînement le décodeur voit la cible gold préfixée par <sos> et prédit la même cible se terminant par <eos>.

# target  : "die katze sitzt"           ← vérité
# dec_in  : "<sos> die katze sitzt"     ← décalé à droite (entrée)
# labels  : "die katze sitzt <eos>"     ← ce que l'on prédit

Étape 17 · Entrée décodeur → embedding

[B, T_tgt] → lookup embed → [B, T_tgt, d_model] = [2, 7, 512]

Étape 19 · Self-attention masquée (masque causal)

Identique à la self-attn de l'encodeur — sauf que la matrice de scores reçoit −∞ sur toutes les positions au-dessus de la diagonale, de sorte que la position i ne peut jamais regarder i+1, i+2, .... Après softmax ces positions deviennent exactement 0.

# scores : [B, h, T_tgt, T_tgt] = [2, 8, 7, 7]
mask = torch.triu(torch.ones(T_tgt, T_tgt), diagonal=1).bool()
scores = scores.masked_fill(mask, float('-inf'))
A = F.softmax(scores, dim=-1)        # triangle inférieur, les lignes somment à 1
[B, h, T_tgt, T_tgt] = [2, 8, 7, 7] triangle supérieur = 0 après softmax

Étape 20 · Cross-attention (Q ← décodeur, K, V ← encodeur)

C'est ici que l'encodeur parle enfin au décodeur. Le décodeur émet des requêtes ; la sortie de l'encodeur fournit les clés et les valeurs.

# dec_x : [B, T_tgt, d_model] = [2, 7, 512]   ← de la self-attn masquée
# memory : [B, T_src, d_model] = [2, 8, 512]  ← sortie encodeur (étape 16)
Q = W_q(dec_x).view(B, T_tgt, h, d_k).transpose(1, 2)   # [B, h, T_tgt, d_k]
K = W_k(memory).view(B, T_src, h, d_k).transpose(1, 2)   # [B, h, T_src, d_k]
V = W_v(memory).view(B, T_src, h, d_k).transpose(1, 2)   # [B, h, T_src, d_k]
scores = Q @ K.transpose(-2, -1) / math.sqrt(d_k)         # [B, h, T_tgt, T_src] = [2, 8, 7, 8]
A = F.softmax(scores, dim=-1)                              # dernier axe = T_src
out = (A @ V).transpose(1, 2).reshape(B, T_tgt, d_model) # [2, 7, 512]

Point clé : la matrice de scores n'est pas carrée. Elle est [B, h, T_tgt, T_src] — chaque token cible assiste chaque token source.

Le bloc décodeur en 16 lignes

DecoderBlock — PyTorch minimal
class DecoderBlock(nn.Module):
    def __init__(self, d=512, h=8, d_ff=2048):
        super().__init__()
        self.self_attn  = nn.MultiheadAttention(d, h, batch_first=True)
        self.cross_attn = nn.MultiheadAttention(d, h, batch_first=True)
        self.ffn = nn.Sequential(nn.Linear(d, d_ff), nn.ReLU(), nn.Linear(d_ff, d))
        self.ln1, self.ln2, self.ln3 = nn.LayerNorm(d), nn.LayerNorm(d), nn.LayerNorm(d)

    def forward(self, x, memory, causal_mask):
        # x :      [B, T_tgt, d]      memory : [B, T_src, d]
        a, _ = self.self_attn(x, x, x, attn_mask=causal_mask)
        x = self.ln1(x + a)                 # [B, T_tgt, d]
        c, _ = self.cross_attn(x, memory, memory)
        x = self.ln2(x + c)                 # [B, T_tgt, d]
        x = self.ln3(x + self.ffn(x))
        return x                            # [B, T_tgt, d]

Sortie de la pile décodeur (après 6 de ces blocs) : [B, T_tgt, d_model] = [2, 7, 512]. Prochaine étape : projection sur le vocabulaire.

Étapes 22–23 · De l'état caché aux logits

Un seul nn.Linear(d_model, V) transforme chaque vecteur caché par token en un score pour chaque mot du vocabulaire. Softmax sur le dernier axe transforme ces scores en probabilités. C'est toute la tête de sortie. Dans le papier, W_lm est liée à la matrice d'embedding d'entrée — mêmes paramètres, transposés.

Étape 22 · Projection linéaire sur le vocabulaire

[B, T_tgt, d_model] @ W_lm.T → [B, T_tgt, V] = [2, 7, 37000] 
# dec_out : [B, T_tgt, d_model]
# W_lm :    [V, d_model]   ← souvent liée à embed.weight
logits = dec_out @ W_lm.T           # [B, T_tgt, V]
# ou simplement :
logits = self.lm_head(dec_out)      # nn.Linear(d_model, V)

Ce sont des scores bruts — un par token du vocabulaire, par position, par exemple. Pas encore des probabilités. Ne leur appliquez pas softmax si votre perte est CrossEntropyLoss ; il le fait en interne pour la stabilité numérique.

Étape 23 · Softmax → probabilités (seulement quand vous en avez vraiment besoin)

probs = F.softmax(logits, dim=-1)   # [B, T_tgt, V] — les lignes sur V somment à 1

À chacune des B·T_tgt positions vous avez maintenant une distribution sur l'ensemble du vocabulaire. La plupart de ces 37 000 chiffres sont minuscules. Une poignée se concentre près de la bonne réponse.

Nombre de paramètres de la tête de sortie

W_lm seule représente V × d_model = 37000 × 512 ≈ 19M paramètres — généralement le plus grand poids du modèle. Le papier la lie à la matrice d'embedding d'entrée, réduisant de moitié la mémoire des poids et (empiriquement) améliorant la généralisation.

La passe avant complète, de bout en bout

Transformer.forward — fidèle au papier
def forward(self, src_ids, tgt_in_ids):
    # src_ids:    [B, T_src]   tgt_in_ids: [B, T_tgt]
    src = self.src_embed(src_ids) + self.pos(src_ids)        # [B, T_src, d]
    tgt = self.tgt_embed(tgt_in_ids) + self.pos(tgt_in_ids)    # [B, T_tgt, d]

    memory = src
    for layer in self.encoder_layers:
        memory = layer(memory)                                  # [B, T_src, d]

    causal = self.causal_mask(tgt_in_ids.shape[1])
    x = tgt
    for layer in self.decoder_layers:
        x = layer(x, memory, causal)                            # [B, T_tgt, d]

    logits = self.lm_head(x)                                     # [B, T_tgt, V]
    return logits  # logits retournés

Entraînement — tout le papier, une seule étape

Une seule étape d'entraînement sur un batch de paires de chaînes (source, cible). Nous tokenisons les deux, construisons les entrées encodeur & décodeur, exécutons le transformer complet (embeds → positionnel → 6 couches encodeur → 6 couches décodeur → tête linéaire), calculons la cross-entropie contre les labels décalés à droite, rétropropageons et effectuons une étape d'optimiseur. Entrée initiale : une liste de paires de chaînes. Sortie finale : une perte scalaire + paramètres mis à jour.

config du papier (inchangée — fixe sur toutes les étapes)
B = 2, T_src = 8, T_tgt = 7, d_model = 512, h = 8, d_k = 64, d_ff = 2048, V = 37000, N = 6
tokens affichés : <sos>=1, <eos>=2, <pad>=0 ; les IDs anglais/allemands sont illustratifs.

un batch de B=2 paires (src, tgt)
list[(str, str)], len = B

données

batch = [
  ("the cat sat", "die katze sitzt"),
  ("i love you",  "ich liebe dich"),
]

tenseur

# rien encore — chaînes Python pures
type(batch)        # list

Étape 2 · Tokeniser la source — chaînes → IDs entiers

tokenizer BPE · pad à T_src
(B, T_src) [2, 8] int64

données

def train_step(model, optimizer, batch):
    # batch : list[(src_str, tgt_str)] de longueur B
    src_ids = tokenize_src(batch)         # [B, T_src]    int64
    dec_in, labels = tokenize_tgt(batch)  # [B, T_tgt] chacun

    src = model.src_embed(src_ids)*math.sqrt(d) + model.pos(src_ids)
    tgt = model.tgt_embed(dec_in)*math.sqrt(d) + model.pos(dec_in)

    memory = src
    for layer in model.encoder_layers: memory = layer(memory)

    causal = model.causal_mask(dec_in.shape[1])
    x = tgt
    for layer in model.decoder_layers: x = layer(x, memory, causal)

    logits = model.lm_head(x)              # [B, T_tgt, V]
    loss = F.cross_entropy(
        logits.view(-1, V), labels.view(-1),
        ignore_index=-100, label_smoothing=0.1)

    optimizer.zero_grad(); loss.backward(); optimizer.step()
    return loss.item()                     # scalaire

@torch.no_grad()
def generate(model, src_text, max_len=32):
    src_ids = tokenize_src([(src_text, "")])      # [1, T_src]

    src = model.src_embed(src_ids)*math.sqrt(d) + model.pos(src_ids)
    memory = src
    for layer in model.encoder_layers:
        memory = layer(memory)                          # [1, T_src, d]

    dec_in = torch.tensor([[SOS]])                         # [1, 1]
    for _ in range(max_len):
        tgt = model.tgt_embed(dec_in)*math.sqrt(d) + model.pos(dec_in)
        causal = model.causal_mask(dec_in.shape[1])
        x = tgt
        for layer in model.decoder_layers:
            x = layer(x, memory, causal)                 # [1, t, d]
        logits = model.lm_head(x)                          # [1, t, V]
        next_id = logits[:, -1, :].argmax(-1, keepdim=True)
        if next_id.item() == EOS: break
        dec_in = torch.cat([dec_in, next_id], dim=1)    # [1, t+1]

    return detokenize(dec_in[0, 1:].tolist())          # str

Récapitulatif des formes lors de l'inférence : chaîne brute → [1, T_src] → [1, T_src, d] (mémoire, une fois) → boucle : [1, t] → [1, t, d] → [1, t, V] → [1, V] → [1] → ajouter → grandir → chaîne finale.

Bac à sable de formes en direct

Modifiez B, T, d_model, h. Chaque étape se recalcule — les pilules de forme, les comptages de paramètres, et le $d_k$ dérivé se mettent tous à jour. Essayez $d_{model}=768, h=12$ (BERT-base). Ou $d_{model}=4096, h=32$ (LLaMA-7B). Même recette, chiffres différents.

contrôles

B T d_model h d_k d_ff surbrillance orange = dim changée depuis l'étape précédente
Ce qui reste constant : les formes d'entrée et de sortie de chaque bloc encodeur sont identiques, $[B, T, d_{model}]$. C'est ce qui rend trivial l'empilement de $N$ d'entre eux.
Ce qui change à l'intérieur : le tenseur se dilate à 4D pour l'attention et à $4\cdot d_{model}$ dans le FFN, puis se réduit à nouveau.

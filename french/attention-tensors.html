01 Attention Is All You Need · formes des tenseurs

Un seul tenseur, jusqu'aux logits

Le transformeur Attention Is All You Need est une recette : prenez un tenseur d'IDs de tokens source, faites-le passer dans un encodeur, faites passer les tokens cible dans un décodeur qui réalise une attention croisée sur l'encodeur, projetez vers le vocabulaire, et il en ressort les logits du token suivant. Cette page suit la forme — chaque étape, des IDs d'entrée jusqu'aux probabilités finales [B, S_tgt, V], codée par couleur pour que vous puissiez lire l'article sans jamais perdre le tenseur.

valeurs par défaut de l'article (base)
B (batch) = 2 # taille du lot
S_src (src len) = 10 # tokens source
S_tgt (tgt len) = 7 # tokens cible
d_model = 512 # dim. du plongement
h (heads) = 8 # têtes d'attention
d_k = d_v = 64 # d_model / h
d_ff = 2048# dim. interne du FFN (4·d_model)
V (vocab) = 37000# vocab. BPE EN→DE
N (layers) = 6 # couches enc. = couches déc.
légende des couleurs (utilisée partout)
B lot S tokens d_model h têtes d_k d_ff

Quand une dimension change d'une étape à l'autre, elle s'illumine. Observez le même tenseur être découpé en têtes, multiplié, mis à l'échelle, passé au softmax, puis recousu.

02 Entrées et plongements

Des IDs de tokens à un tenseur prêt pour le contexte

Avant toute attention, l'entrée doit devenir un tenseur de vecteurs. Trois étapes rapides : des IDs entiers en entrée, des vecteurs à valeurs réelles en sortie, avec l'information positionnelle intégrée par addition.

étape 1 · IDs des tokens
Tokenizer output. Two examples in a batch, eight tokens each.
$\text{ids}\in\mathbb{Z}^{B\times S}$
forme [B=2, S=8] · dtype: int32 
# two sentences, each padded/truncated to S=8 tokens
ids = [[ 101, 2057, 2293, 3019, 2025, 2748, 102, 0 ],
       [ 101, 7592, 2088,  102,    0,    0,   0, 0 ]]
# shape: [B=2, S=8]
étape 2 · recherche du plongement

Une matrice de poids $E\in\mathbb{R}^{V\times d_{model}}$ contient une ligne par entrée du vocabulaire. Recherche par ID. Le tenseur gagne une dimension : un vecteur à valeurs réelles de taille d_model remplace chaque entier.

$X = E[\text{ids}] \;\in\; \mathbb{R}^{B\times S\times d_{model}}$
[2, 10] → [2, 10, 512] 
import torch.nn as nn
embed = nn.Embedding(num_embeddings=30522, embedding_dim=512)
X = embed(ids)            # [B, S] -> [B, S, d_model]
print(X.shape)            # torch.Size([2, 10, 512])

Nombre de paramètres : $V \cdot d_{model} = 30522 \cdot 512 \approx 15.6\text{M}$ — la table de plongement à elle seule est plus grosse que le reste de n'importe quelle couche.

étape 3 · + encodage positionnel

L'auto-attention est équivariante par permutation — sans information de position, « the cat sat » et « sat cat the » produisent les mêmes sorties. L'article ajoute un motif sinusoïdal fixe de la même forme que $X$.

$\mathrm{PE}_{(pos, 2i)} = \sin\!\big(pos / 10000^{2i/d_{model}}\big)$
$\mathrm{PE}_{(pos, 2i+1)} = \cos\!\big(pos / 10000^{2i/d_{model}}\big)$
[2, 10, 512] + [10, 512] → [2, 10, 512] 
# PE has shape [S, d_model], broadcasts over B
X = X + PE[:S]            # [B, S, d_model] + [S, d_model] -> [B, S, d_model]

Même forme en entrée, même forme en sortie. L'information va dans les valeurs, pas dans les dimensions.

Fin de l'étape 3 : nous avons maintenant un tenseur de forme [B, S, d_model] qui parle à la fois des mots et de leur position. Chaque étape suivante est une transformation sur cette disposition.

03 Attention multi-têtes

Étapes 4–11 · attention, dimension par dimension

La raison d'être de cet article. Chaque token devient une requête, une clé et une valeur ; les similarités sont calculées, mises à l'échelle, passées au softmax, et utilisées pour mélanger les valeurs. On reshape aussi deux fois : une fois pour découper les têtes, une fois pour les recoller.

Pourquoi chaque forme se lit clairement désormais : ici h = 8 têtes, S = 10 positions de séquence, d_k = 64 — chaque dimension a un nombre distinct, donc on peut toujours dire quel axe est lequel. (Les versions antérieures utilisaient une longueur de séquence de 8, qui entrait en collision avec les 8 têtes : [2, 8, 8, 64] était illisible. Avec S=10 cela devient [2, 8, 10, 64] — sans ambiguïté.)

étape 4 · projections Q, K, V

Trois applications linéaires indépendantes, chacune $\mathbb{R}^{d_{model}}\to\mathbb{R}^{d_{model}}$ :

$Q = X W^Q,\quad K = X W^K,\quad V = X W^V$
X [2, 10, 512] Wᵠ [512, 512] → Q [2, 10, 512] 
# trois couches linéaires, pas de biais dans l'article mais un biais convient en pratique
Wq = nn.Linear(d_model, d_model)   # 512*512 = 262 144 params
Wk = nn.Linear(d_model, d_model)
Wv = nn.Linear(d_model, d_model)

Q = Wq(X)                              # [B, S, d_model]
K = Wk(X)                              # [B, S, d_model]
V = Wv(X)                              # [B, S, d_model]

Trois matrices de forme $[d_{model}, d_{model}]$ — total des paramètres ici : $3\cdot 512\cdot 512 = 786\,432$. Dans certaines implémentations, ces trois matrices sont fusionnées en une seule matrice $[d_{model}, 3\cdot d_{model}]$.

étape 5 · reshape & transpose pour h têtes

Découpez la dernière dimension en $h$ blocs de $d_k = d_{model}/h$ chacun, puis transposez pour que les têtes deviennent une dimension externe. Aucun nouveau paramètre — juste view + permute.

[2, 10, 512] view→ [2, 10, 8, 64] transpose(1,2)→ [2, 8, 10, 64] 
B, S, d_model = Q.shape
h, d_k = 8, d_model // h         # 8, 64

# découpe la dernière dim en (h, d_k), puis place les têtes près du lot
Q = Q.view(B, S, h, d_k).transpose(1, 2)   # [B, h, S, d_k]
K = K.view(B, S, h, d_k).transpose(1, 2)   # [B, h, S, d_k]
V = V.view(B, S, h, d_k).transpose(1, 2)   # [B, h, S, d_k]
Pourquoi les têtes comptent : avec $h=8$ sous-espaces $[S, d_k=64]$ distincts, chaque tête peut se spécialiser sur un motif relationnel différent. Le calcul est inchangé : 8 petites attentions de dim. $64$ au lieu d'une seule de dim. $512$.
étape 6 · Q @ Kᵀ — similarité par paires

Pour chaque tête, chaque requête prête attention à chaque clé. Le produit matriciel contracte l'axe $d_k$, produisant une matrice de scores $S\times S$ par tête et par élément du lot.

$\text{scores} = Q K^\top$
Q [2, 8, 10, 64] Kᵀ [2, 8, 64, 10] → scores [2, 8, 10, 10] 
scores = Q @ K.transpose(-2, -1)   # [B, h, S, S]
# entrée (b, head, i, j) = à quel point le token i prête attention au token j (dans cette tête)
étape 7 · mise à l'échelle par 1/√d_k

Sans mise à l'échelle, les produits scalaires de vecteurs de dim. $d_k$ croissent en $\sqrt{d_k}$ — de grandes valeurs poussent le softmax dans le régime de saturation où les gradients s'annulent. Divisez chaque score par $\sqrt{d_k}$.

$\text{scaled} = \text{scores} / \sqrt{d_k} = \text{scores} / 8$
[2, 8, 10, 10] ÷ √64 [2, 8, 10, 10] forme inchangée 
import math
scores = scores / math.sqrt(d_k)   # 1/√64 = 1/8 = 0.125
étape 8 · softmax sur le dernier axe

Transforme les scores bruts en une distribution de probabilité par requête. Chaque ligne du bloc $[S,S]$ a une somme de 1 — ce sont les poids d'attention.

$A = \mathrm{softmax}(\text{scaled},\ \text{dim}=-1)$
[2, 8, 10, 10] → [2, 8, 10, 10] somme des lignes = 1 
import torch.nn.functional as F
A = F.softmax(scores, dim=-1)    # [B, h, S, S]

Pour l'auto-attention du décodeur, un masque additif de $-\infty$ est appliqué avant le softmax pour imposer la causalité.

étape 9 · attention @ V

Mélange les vecteurs de valeur à l'aide des poids d'attention. La sortie de chaque requête est une somme pondérée de tous les $S$ vecteurs de valeur. L'axe contracté est le $S$ interne.

A [2, 8, 10, 10] V [2, 8, 10, 64] → out [2, 8, 10, 64] 
head_out = A @ V                  # [B, h, S, d_k]
étape 10 · concaténation des têtes (transpose + reshape)

Recolle les têtes en une grande dernière dimension. transpose(1,2) déplace les têtes à côté de $d_k$, puis reshape les fusionne. Sans paramètre.

[2, 8, 10, 64] transpose→ [2, 10, 8, 64] reshape→ [2, 10, 512] 
out = head_out.transpose(1, 2).contiguous()   # [B, S, h, d_k] — têtes désormais adjacentes
out = out.reshape(B, S, d_model)              # reshape fusionne les 8 têtes bout à bout = Concat(head_0..head_7)
Le reshape est la concaténation. L'article écrit $\mathrm{MHA}(Q,K,V) = \mathrm{Concat}(\text{head}_1,\dots,\text{head}_h)\,W^O$, mais il n'y a aucun torch.cat littéral ici. Après transpose(1,2), les h = 8 têtes sont adjacentes dans les deux derniers axes pour chacune des S = 10 positions de token : [2, 10, 8, 64]. reshape aplatit ensuite ces deux derniers axes ($h\cdot d_k = 8\cdot 64 = $ 512), disposant les têtes bout à bout le long de la dimension des caractéristiques. Cette disposition côte à côte sur une mémoire contiguë est exactement la concaténation — donc la concaténation n'est pas omise, c'est précisément ce que fait le reshape (.contiguous() rend simplement le reshape valide après la transposition). Le vecteur de 512 de chaque token est :
[512] = [head₀ 64]‖[head₁ 64]‖ … ‖[head₇ 64] équivaut à torch.cat([head_0, …, head_7], dim=-1)
étape 11 · projection de sortie W_O

Une application linéaire $\mathbb{R}^{d_{model}\to d_{model}}$ de plus. Même forme, mais permet au modèle de mélanger l'information entre les têtes après la concaténation.

$\text{MHA}(X) = \mathrm{Concat}(\text{head}_1, \dots, \text{head}_h)\,W^O$
[2, 10, 512] ·W_O [2, 10, 512] 
Wo = nn.Linear(d_model, d_model)   # encore 262 144 paramètres
mha_out = Wo(out)                       # [B, S, d_model]

Total des paramètres MHA : $4 \cdot d_{model}^2 = 4\cdot 512^2 = 1{,}048{,}576$ (Q, K, V, O — chacune $d_{model}\times d_{model}$).

Fin de l'attention : nous repartons avec la forme [B, S, d_model] — exactement la même forme qu'à l'entrée. L'attention est un opérateur à forme constante.

04 FFN et bloc encodeur

LayerNorm, FFN et l'autoroute résiduelle

Après l'attention, le bloc encodeur assemble tout avec deux connexions résiduelles, deux LayerNorms et un FFN appliqué position par position. C'est dans le FFN que vit la majorité des paramètres — et où, selon beaucoup, le savoir factuel est stocké.

étape 12 · résiduel + LayerNorm

Réajoute le $X$ original (connexion saute-couche), puis normalise par token sur $d_{model}$.

$X' = \mathrm{LayerNorm}(X + \mathrm{MHA}(X))$
[2, 10, 512] + [2, 10, 512] → LN → [2, 10, 512] 
ln1 = nn.LayerNorm(d_model)      # 2 * d_model = 1024 paramètres (γ, β)
X_post = ln1(X + mha_out)         # [B, S, d_model]

Le LN normalise sur le dernier axe : chaque vecteur $[d_{model}]$ devient de moyenne 0 et variance 1, puis ré-échelonné par des $\gamma, \beta\in\mathbb{R}^{d_{model}}$ appris. Le lot et les positions de token restent indépendants.

étapes 13–14 · FFN linéaire 1 + ReLU

Un réseau feed-forward appliqué position par position : le même MLP appliqué à chaque token indépendamment. Le premier linéaire étend $d_{model} \to d_{ff}$ (typiquement $4\cdot d_{model}$).

$H = \mathrm{ReLU}(X' W_1 + b_1)$
[2, 10, 512] ·W₁ [2, 10, 2048] → ReLU → [2, 10, 2048] 
W1 = nn.Linear(d_model, d_ff)       # 512 * 2048 = 1 048 576 paramètres
H = F.relu(W1(X_post))            # [B, S, d_ff]
stage 15 · FFN linear 2

Second linear projects back: $d_{ff} \to d_{model}$. Restores the canonical shape so the next block can start fresh.

$Y = H W_2 + b_2$
[2, 10, 2048] ·W₂ [2, 10, 512] 
W2 = nn.Linear(d_ff, d_model)       # 2048 * 512 = another 1 048 576 params
ffn_out = W2(H)                       # [B, S, d_model]

Total FFN params: $2\cdot d_{model}\cdot d_{ff} = 2\cdot 512\cdot 2048 = 2{,}097{,}152$ — roughly twice the cost of all four attention projections combined. This is where big models put their weight.

étape 16 · résiduel + LayerNorm (à nouveau)

Même motif qu'à l'étape 12. Ajoute le tenseur d'avant-FFN, applique la normalisation par couche, et le bloc encodeur est terminé.

$Y' = \mathrm{LayerNorm}(X' + Y)$
[2, 10, 512] → [2, 10, 512] forme de sortie du bloc = forme d'entrée du bloc 
ln2 = nn.LayerNorm(d_model)
out = ln2(X_post + ffn_out)       # [B, S, d_model]
Empiler & répéter : l'article empile $N=6$ blocs encodeur identiques. La forme d'entrée de chacun égale sa forme de sortie, ce qui explique qu'ils se composent proprement. Total des paramètres de l'encodeur (un bloc, base) : environ $4\cdot 512^2 + 2\cdot 512\cdot 2048 + 4\cdot 512 \approx 3.15$M.

Tout le bloc encodeur en 14 lignes

EncoderBlock — PyTorch minimal
class EncoderBlock(nn.Module):
    def __init__(self, d_model=512, h=8, d_ff=2048):
        super().__init__()
        self.attn = nn.MultiheadAttention(d_model, h, batch_first=True)
        self.ffn  = nn.Sequential(
            nn.Linear(d_model, d_ff), nn.ReLU(), nn.Linear(d_ff, d_model)
        )
        self.ln1 = nn.LayerNorm(d_model)
        self.ln2 = nn.LayerNorm(d_model)

    def forward(self, x):
        # x: [B, S, d_model]
        a, _ = self.attn(x, x, x)
        x = self.ln1(x + a)
        x = self.ln2(x + self.ffn(x))
        return x                       # [B, S, d_model]

05 Bloc décodeur

Le bloc décodeur — auto-attention masquée, puis attention croisée

Le décodeur prend les tokens cible (décalés d'un cran à droite) et produit un tenseur qui connaît à la fois (a) les tokens cible vus jusqu'ici — via l'auto-attention masquée — et (b) la source — via l'attention croisée, où Q vient du décodeur et K, V viennent de la sortie de l'encodeur (la « mémoire »). Même dimension de plongement, même discipline résiduel+LN que l'encodeur. Trois sous-blocs par couche, six couches dans l'article.

config. de l'article (toujours)
B = 2, S_src = 10, S_tgt = 7
d_model = 512, h = 8, d_k = 64
# S_tgt est indépendant de S_src.
# l'attention croisée est le seul endroit où ils se rencontrent.
pourquoi « décalé à droite » ?

Pendant l'entraînement, le décodeur voit la cible de référence préfixée par <sos> et prédit la même cible se terminant par <eos>.

# cible   : "die katze sitzt"           ← vérité
# dec_in  : "<sos> die katze sitzt"     ← décalé à droite (entrée)
# étiq.   : "die katze sitzt <eos>"     ← ce que l'on prédit

Étape 17 · Entrée du décodeur → plongement

[B, S_tgt] → embed lookup → [B, S_tgt, d_model] = [2, 7, 512]

Étape 19 · Auto-attention masquée (masque causal)

Identique à l'auto-attention de l'encodeur — sauf que la matrice de scores reçoit −∞ sur chaque position au-dessus de la diagonale, de sorte que la position i ne peut jamais regarder i+1, i+2, .... Après le softmax, ces positions deviennent exactement 0.

# scores: [B, h, S_tgt, S_tgt] = [2, 8, 7, 7]
mask = torch.triu(torch.ones(S_tgt, S_tgt), diagonal=1).bool()
scores = scores.masked_fill(mask, float('-inf'))
A = F.softmax(scores, dim=-1)        # triangle inférieur, somme des lignes = 1
[B, h, S_tgt, S_tgt] = [2, 8, 7, 7] triangle supérieur = 0 après softmax

A Masquage — interactif

Pourquoi le décodeur ne peut pas tricher en regardant le futur

Pendant l'entraînement, le décodeur voit toute la cible d'un coup (teacher forcing). Pour quand même apprendre à « prédire le token suivant », on interdit à la position i de prêter attention à toute position ultérieure j > i — ces cellules sont mises à −∞ avant le softmax, donc après le softmax elles valent exactement 0. Basculez le masque et glissez la position de requête pour le ressentir.

masque causal : ON
requête i →
key j → (S_tgt = 7)
allowed (j ≤ i) blocked (−∞)

With the mask OFF, every query can see every key — that's encoder self-attention. With it ON, the grid is lower-triangular: position 0 sees only itself, position 6 sees all 7. This is exactly what lets the model be trained on all positions in parallel yet still generate left-to-right at inference.

Masque de padding

Separately, the S_src = 10 source had trailing padding zeros (row 0: 9 real tokens + 1 pad; row 1: 3 real + 7 pad). Those positions are also masked to −∞ so attention never flows into padding. Click a key position to toggle it as padding and watch its whole column black out.

Étape 20 · Attention croisée (Q ← décodeur, K, V ← encodeur)

This is where the encoder finally talks to the decoder. The decoder emits queries; the encoder output supplies keys and values.

# dec_x: [B, S_tgt, d_model] = [2, 7, 512]   ← from masked self-attn
# memory: [B, S_src, d_model] = [2, 10, 512]  ← encoder output (stage 16)
Q = W_q(dec_x).view(B, S_tgt, h, d_k).transpose(1, 2)   # [B, h, S_tgt, d_k]
K = W_k(memory).view(B, S_src, h, d_k).transpose(1, 2)   # [B, h, S_src, d_k]
V = W_v(memory).view(B, S_src, h, d_k).transpose(1, 2)   # [B, h, S_src, d_k]
scores = Q @ K.transpose(-2, -1) / math.sqrt(d_k)         # [B, h, S_tgt, S_src] = [2, 8, 7, 10]
A = F.softmax(scores, dim=-1)                              # last axis is S_src
out = (A @ V).transpose(1, 2).reshape(B, S_tgt, d_model) # [2, 7, 512]

Crucially: the score matrix is not square. It is [B, h, S_tgt, S_src] — every target token attends over every source token.

Le bloc décodeur en 16 lignes

DecoderBlock — PyTorch minimal
class DecoderBlock(nn.Module):
    def __init__(self, d=512, h=8, d_ff=2048):
        super().__init__()
        self.self_attn  = nn.MultiheadAttention(d, h, batch_first=True)
        self.cross_attn = nn.MultiheadAttention(d, h, batch_first=True)
        self.ffn = nn.Sequential(nn.Linear(d, d_ff), nn.ReLU(), nn.Linear(d_ff, d))
        self.ln1, self.ln2, self.ln3 = nn.LayerNorm(d), nn.LayerNorm(d), nn.LayerNorm(d)

    def forward(self, x, memory, causal_mask):
        # x:      [B, S_tgt, d]      memory: [B, S_src, d]
        a, _ = self.self_attn(x, x, x, attn_mask=causal_mask)
        x = self.ln1(x + a)                 # [B, S_tgt, d]
        c, _ = self.cross_attn(x, memory, memory)
        x = self.ln2(x + c)                 # [B, S_tgt, d]
        x = self.ln3(x + self.ffn(x))
        return x                            # [B, S_tgt, d]

Output of the decoder stack (after 6 of these): [B, S_tgt, d_model] = [2, 7, 512]. Next stop: project to vocabulary.

06 Sortie → logits

De l'état caché aux logits

A single nn.Linear(d_model, V) turns each per-token hidden vector into a score for every word in the vocabulary. Softmax along the last axis turns those scores into probabilities. That's the entire output head. In the paper, W_lm is tied to the input embedding matrix — same parameters, transposed.

Étape 22 · Projection linéaire vers le vocabulaire

[B, S_tgt, d_model] @ W_lm.T → [B, S_tgt, V] = [2, 7, 37000] 
# dec_out: [B, S_tgt, d_model]
# W_lm :   [V, d_model]   ← souvent lié à embed.weight
logits = dec_out @ W_lm.T           # [B, S_tgt, V]
# or simply:
logits = self.lm_head(dec_out)      # nn.Linear(d_model, V)

These are raw scores — one per vocabulary token, per position, per example. Not probabilities yet. Don't softmax them if your loss is CrossEntropyLoss; it does that internally for numerical stability.

Étape 23 · Softmax → probabilités (seulement quand vous en avez vraiment besoin)

probs = F.softmax(logits, dim=-1)   # [B, S_tgt, V] — rows over V sum to 1

At each of the B·S_tgt positions you now have a distribution over the entire vocabulary. Most of those 37,000 numbers are tiny. A handful concentrate near the right answer.

Nombre de paramètres de la tête de sortie

W_lm alone is V × d_model = 37000 × 512 ≈ 19M parameters — usually the single largest weight in the model. The paper ties it to the input embedding matrix, halving the weight memory and (empirically) helping generalization.

La passe avant complète, de bout en bout

Transformer.forward — paper-faithful
def forward(self, src_ids, tgt_in_ids):
    # src_ids:    [B, S_src]   tgt_in_ids: [B, S_tgt]
    src = self.src_embed(src_ids) + self.pos(src_ids)        # [B, S_src, d]
    tgt = self.tgt_embed(tgt_in_ids) + self.pos(tgt_in_ids)    # [B, S_tgt, d]

    memory = src
    for layer in self.encoder_layers:
        memory = layer(memory)                                  # [B, S_src, d]

    causal = self.causal_mask(tgt_in_ids.shape[1])
    x = tgt
    for layer in self.decoder_layers:
        x = layer(x, memory, causal)                            # [B, S_tgt, d]

    logits = self.lm_head(x)                                     # [B, S_tgt, V]
    return logits

Entraînement — tout l'article, en une étape

The whole training step in one shot

train_step — paper-faithful, ~20 lines
def train_step(model, optimizer, batch):
    # batch: list[(src_str, tgt_str)] of length B
    src_ids = tokenize_src(batch)         # [B, S_src]    int64
    dec_in, labels = tokenize_tgt(batch)  # [B, S_tgt] each

    src = model.src_embed(src_ids)*math.sqrt(d) + model.pos(src_ids)
    tgt = model.tgt_embed(dec_in)*math.sqrt(d) + model.pos(dec_in)

    memory = src
    for layer in model.encoder_layers: memory = layer(memory)

    causal = model.causal_mask(dec_in.shape[1])
    x = tgt
    for layer in model.decoder_layers: x = layer(x, memory, causal)

    logits = model.lm_head(x)              # [B, S_tgt, V]
    loss = F.cross_entropy(
        logits.view(-1, V), labels.view(-1),
        ignore_index=-100, label_smoothing=0.1)

    optimizer.zero_grad(); loss.backward(); optimizer.step()
    return loss.item()                     # scalar

Summary of shapes during one training step: raw strings (B) → [B, S_src] → [B, S_src, d] → [B, S_tgt, d] → [B, S_tgt, V] → scalar loss → ∂L/∂θ on every param.

The whole inference loop in one shot

generate — greedy, ~20 lines
@torch.no_grad()
def generate(model, src_text, max_len=32):
    src_ids = tokenize_src([(src_text, "")])      # [1, S_src]

    src = model.src_embed(src_ids)*math.sqrt(d) + model.pos(src_ids)
    memory = src
    for layer in model.encoder_layers:
        memory = layer(memory)                          # [1, S_src, d]

    dec_in = torch.tensor([[SOS]])                         # [1, 1]
    for _ in range(max_len):
        tgt = model.tgt_embed(dec_in)*math.sqrt(d) + model.pos(dec_in)
        causal = model.causal_mask(dec_in.shape[1])
        x = tgt
        for layer in model.decoder_layers:
            x = layer(x, memory, causal)                 # [1, t, d]
        logits = model.lm_head(x)                          # [1, t, V]
        next_id = logits[:, -1, :].argmax(-1, keepdim=True)
        if next_id.item() == EOS: break
        dec_in = torch.cat([dec_in, next_id], dim=1)    # [1, t+1]

    return detokenize(dec_in[0, 1:].tolist())          # str

Summary of shapes during inference: raw string → [1, S_src] → [1, S_src, d] (memory, once) → loop: [1, t] → [1, t, d] → [1, t, V] → [1, V] → [1] → append → grow → final string.

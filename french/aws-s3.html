AWS S3 CLI — de ls à put-bucket-policy

Deux familles CLI, un seul service de stockage. aws s3 est l'enveloppe haut niveau et conviviale pour le quotidien — copier, synchroniser, presigner. aws s3api est le miroir bas niveau de l'API REST, pour les opérations fines comme le versioning, les cycles de vie, les uploads multipart et les politiques de bucket. Tapez pour les apprendre toutes les deux.

aws s3 — haut niveau
Des commandes au feeling système de fichiers : ls, cp, mv, sync, rm, mb, rb, presign. Elle fait des choses intelligentes automatiquement — uploads multi-threads, multipart pour les gros fichiers, retries — et c'est ce que vous utiliserez 90 % du temps.
aws s3api — bas niveau
Une sous-commande par verbe de l'API S3 : list-objects-v2, head-object, put-bucket-versioning, create-multipart-upload, … À utiliser quand vous avez besoin d'une fonctionnalité que aws s3 n'expose pas, ou quand vous voulez du JSON brut à passer à --query / jq.

L'URI S3 que vous taperez mille fois

# schéma   nom du bucket    clé (chemin de l'objet dans le bucket)
s3://my-bucket/photos/2026/cat.jpg

# un « dossier » n'est qu'un préfixe de clé — il n'existe pas vraiment comme objet
s3://my-bucket/photos/2026/
Piège : les clés S3 ne sont pas un vrai système de fichiers. Le « répertoire » photos/2026/ n'existe que parce qu'il y a des objets dont les clés commencent par ce préfixe. Les dossiers vides sont des objets de zéro octet finissant par /.

IAM : les quatre actions S3 les plus utilisées

ActionCe qu'elle autoriseExemple de ressource
s3:ListBucketLister les clés avec un préfixe donnéarn:aws:s3:::my-bucket
s3:GetObjectTélécharger un objetarn:aws:s3:::my-bucket/*

Actions sur bucket vs sur objet : s3:ListBucket cible l'ARN du bucket (sans /*). s3:GetObject cible les objets à l'intérieur (avec /*). Confondre les deux est la cause n°1 d'« AccessDenied » sur une politique fraîche.

Lister & inspecter — ls, head-object, list-objects-v2, --query

Premier commandement du stockage cloud : savoir ce qu'il y a dans le bucket avant d'y toucher. Les deux familles CLI listent les objets ; s3api vous donne du JSON que vous pouvez découper avec JMESPath via --query.

aws s3 ls — le listing convivial

# Tous les buckets du compte
$ aws s3 ls
2026-04-01 10:22:13 my-bucket
2026-04-12 09:01:55 logs-bucket

# Objets à la racine d'un bucket
$ aws s3 ls s3://my-bucket
                           PRE photos/
2026-05-01 14:11:02     1024 readme.txt

# Récursif, lisible par un humain, avec ligne de résumé
$ aws s3 ls s3://my-bucket/photos/ --recursive --human-readable --summarize
2026-05-01 14:11:02   2.1 MiB photos/2026/cat.jpg
2026-05-01 14:11:03   1.7 MiB photos/2026/dog.jpg

Total Objects: 2
   Total Size: 3.8 MiB
1 Lister récursivement tout ce qui est sous le préfixe photos/ dans my-bucket.
$ aws s3 ls s3://my-bucket/photos/ --
2 Ajoutez le drapeau qui affiche les tailles en MiB / GiB plutôt qu'en octets.
$ aws s3 ls s3://my-bucket --recursive --

aws s3api list-objects-v2 — listage programmatique

# Lister jusqu'à 10 clés sous le préfixe p/, projeter juste key + size
$ aws s3api list-objects-v2 \
    --bucket my-bucket \
    --prefix p/ \
    --max-items 10 \
    --query 'Contents[].{Key:Key,Size:Size}'

# Pagination via jeton de continuation
$ aws s3api list-objects-v2 --bucket my-bucket --max-keys 1000 \
    --starting-token "$NEXT_TOKEN"
3 Projeter uniquement le champ Key de chaque objet listé.
$ aws s3api list-objects-v2 --bucket my-bucket \
    --query 'Contents[].'

JMESPath en express — le drapeau --query

Chaque appel aws retourne du JSON. --query 'EXPR' exécute JMESPath dessus côté client, avant que --output ne le formate. 

# Juste les noms de buckets
$ aws s3api list-buckets --query 'Buckets[].Name'

# Objets plus gros que 1 MiB (notez les backticks pour les nombres)
$ aws s3api list-objects-v2 --bucket B \
    --query 'Contents[?Size > `1048576`].Key'

# Les 5 objets les plus récents
$ aws s3api list-objects-v2 --bucket B \
    --query 'reverse(sort_by(Contents, &LastModified))[:5].[Key,LastModified]' \
    --output table

# Taille totale d'un préfixe (somme)
$ aws s3api list-objects-v2 --bucket B --prefix logs/ \
    --query 'sum(Contents[].Size)'
5 Retourner uniquement les noms de buckets depuis list-buckets.
$ aws s3api list-buckets --query 'Buckets[].'
6 JMESPath : ne garder que les objets strictement plus gros qu'1 MiB. Complétez le mot-clé.
--query 'Contents[Size > `1048576`].Key'

Upload, Download, Sync — cp, mv, sync, exclude/include

Ces quatre verbes (cp, mv, sync, rm) gèrent 95 % des octets qui entrent ou sortent de S3. Maîtrisez l'ordre de --exclude / --include et --dryrun avant de taper un jour --delete.

cp — fichier unique ou récursif

# Upload
$ aws s3 cp ./report.pdf s3://my-bucket/reports/

# Download
$ aws s3 cp s3://my-bucket/reports/report.pdf ./

# Récursif — répertoire entier
$ aws s3 cp ./build/ s3://my-bucket/site/ --recursive

# Avec classe de stockage, chiffrement, content-type et metadata
$ aws s3 cp big.bin s3://my-bucket/data/big.bin \
    --storage-class STANDARD_IA \
    --sse aws:kms --sse-kms-key-id alias/data \
    --content-type application/octet-stream \
    --metadata ingest=2026-05-20,team=ml \
    --acl private
1 Uploader un répertoire local entier vers un préfixe dans S3.
$ aws s3 cp ./build/ s3://my-bucket/site/ --
2 Marquer l'upload comme Infrequent-Access. Complétez la classe de stockage.
$ aws s3 cp file.txt s3://b/k --storage-class

mv — copie + suppression de la source

# Promouvoir un artefact de build du préfixe staging au préfixe release
$ aws s3 mv s3://b/staging/v1.tar.gz s3://b/release/v1.tar.gz

# Déplacer local → S3 (supprime le fichier local en cas de succès)
$ aws s3 mv ./done.zip s3://b/archive/

sync — le superpouvoir au niveau répertoire

# Mirroir local → S3, *ajoute* et *met à jour* mais ne supprime jamais
$ aws s3 sync ./public/ s3://my-site/

# Mirroir avec suppression : S3 finit par correspondre exactement au local
$ aws s3 sync ./public/ s3://my-site/ --delete

# Ignorer les fichiers parasites, n'envoyer que JS et CSS
$ aws s3 sync ./dist/ s3://my-site/ \
    --exclude "*" \
    --include "*.js" \
    --include "*.css"

# Voir ce qui changerait sans le faire
$ aws s3 sync ./public/ s3://my-site/ --delete --dryrun
L'ordre compte : --exclude / --include sont évalués de gauche à droite et la dernière correspondance gagne. Mettez d'abord le motif --exclude "*", puis ré-incluez ce que vous voulez.
3 Mirroir d'un répertoire local vers S3 pour que les fichiers locaux supprimés soient aussi supprimés dans S3, en ignorant les fichiers .tmp.
$ aws s3  local/ s3://bucket/prefix/ -- --exclude "*.tmp"
4 Prévisualiser ce que ferait un sync destructif sans l'exécuter.
$ aws s3 sync ./public/ s3://site/ --delete --

      rm — prudence, surtout avec --recursive

      # Un seul objet
$ aws s3 rm s3://b/old.log

# Préfixe entier — ne demande rien !
$ aws s3 rm s3://b/tmp/ --recursive

# Sélectif : éliminer logs/ mais garder les fichiers .json
$ aws s3 rm s3://b/logs/ --recursive --exclude "*.json"
      Avertissement versioning : aws s3 rm --recursive sur un bucket versionné n'ajoute que des delete markers — les anciennes versions sont toujours là. Pour vraiment libérer de l'espace, il faut supprimer explicitement les versions via s3api.

      Outils puissants s3api — un verbe par action de l'API S3

      Quand les commandes conviviales aws s3 ne suffisent plus, descendez vers s3api. La sortie est toujours du JSON — passez-la dans --query ou jq.

      Opérations au niveau objet

      # Upload d'un fichier local avec content-type et metadata explicites
$ aws s3api put-object \
    --bucket my-bucket \
    --key data/blob.bin \
    --body ./blob.bin \
    --content-type application/octet-stream \
    --metadata sha256=abcd...,team=ml

# Téléchargement vers un fichier
$ aws s3api get-object --bucket my-bucket --key data/blob.bin out.bin

# Copie côté serveur (aucun octet ne passe par votre laptop)
$ aws s3api copy-object \
    --bucket dest-bucket \
    --key archive/blob.bin \
    --copy-source my-bucket/data/blob.bin

# Supprimer une version spécifique
$ aws s3api delete-object \
    --bucket my-bucket --key data/blob.bin \
    --version-id 3HL4kqtJlcpXroDTDmJ+rmSpXd3dIbrHY
      1 Copie côté serveur d'une clé de srcB/srcK vers dstB/dstK. Complétez le verbe et le drapeau de source.
      $ aws s3api  \
    --bucket dstB --key dstK \
    -- srcB/srcK

      Tagging d'objets

      $ aws s3api put-object-tagging \
    --bucket my-bucket --key data/x.bin \
    --tagging '{"TagSet":[{"Key":"env","Value":"prod"},{"Key":"team","Value":"ml"}]}'

$ aws s3api get-object-tagging --bucket my-bucket --key data/x.bin
      2 Lire les tags actuellement posés sur un objet.

      # Activer le versioning (quasi-irréversible — vous pouvez seulement Suspendre, pas Désactiver)
$ aws s3api put-bucket-versioning \

# Voir l'état actuel
$ aws s3api get-bucket-versioning --bucket my-bucket

$ aws s3api list-object-versions --bucket my-bucket --prefix data/blob.bin

      Buckets & politiques — mb, rb, policy, CORS, lifecycle, public-access block

      La configuration au niveau bucket est l'endroit où la sécurité et le coût se jouent le plus souvent. Ces commandes mettent en place le versioning, les blocages d'accès public, le CORS, les règles de cycle de vie et les politiques de bucket.

      Créer & supprimer un bucket

      # Créer. La région compte pour LocationConstraint (us-east-1 est implicite / « none »)
$ aws s3 mb s3://my-new-bucket --region us-east-1

# Supprimer. --force le vide aussi (haut niveau seulement).
$ aws s3 rb s3://my-new-bucket --force

# Équivalents s3api (plus de boutons)
$ aws s3api create-bucket --bucket my-new-bucket \
    --create-bucket-configuration LocationConstraint=eu-west-1

$ aws s3api delete-bucket --bucket my-new-bucket
      1 Créer un nouveau bucket appelé cat-photos dans us-east-1.
      $ aws s3  s3://cat-photos --region us-east-1
      2 Supprimer un bucket non vide d'un seul coup. Choisissez le drapeau.
      $ aws s3 rb s3://cat-photos --

      Bloquer tout accès public (faites-le en premier !)

      $ aws s3api put-public-access-block \
    --bucket my-bucket \
    --public-access-block-configuration \
        BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true

$ aws s3api get-public-access-block --bucket my-bucket
      Bouclier au niveau compte : les mêmes drapeaux existent en aws s3control put-public-access-block --account-id … — réglez-les là aussi pour qu'une future politique de bucket ne puisse pas accidentellement ouvrir une faille.

      Politique de bucket

      # policy.json — autoriser GetObject pour tout le monde (bucket public en lecture seule)
{
  "Version": "2012-10-17",
  "Statement": [{
    "Sid":    "PublicRead",
    "Effect": "Allow",
    "Principal": "*",
    "Action": "s3:GetObject",
    "Resource": "arn:aws:s3:::my-public-bucket/*"
  }]
}

$ aws s3api put-bucket-policy --bucket my-public-bucket \
    --policy file://policy.json

$ aws s3api get-bucket-policy --bucket my-public-bucket
$ aws s3api delete-bucket-policy --bucket my-public-bucket

      $ aws s3api put-bucket-policy --bucket B --policy

      Règles de cycle de vie

      # lc.json — passer les logs en IA à 30j, GLACIER à 90j, expirer à 365j
{
  "Rules": [{
    "ID": "logs-tier-down",
    "Status": "Enabled",
    "Filter": { "Prefix": "logs/" },
    "Transitions": [
      { "Days": 30,  "StorageClass": "STANDARD_IA" },
      { "Days": 90,  "StorageClass": "GLACIER" }
    ],
    "Expiration": { "Days": 365 },
    "NoncurrentVersionExpiration": { "NoncurrentDays": 90 }
  }]
}

$ aws s3api put-bucket-lifecycle-configuration \
    --bucket my-bucket \
    --lifecycle-configuration file://lc.json

$ aws s3api get-bucket-lifecycle-configuration --bucket my-bucket
      4 Installer une configuration de cycle de vie depuis lc.json.
      $ aws s3api put-bucket- \
    --bucket B --lifecycle-configuration file://lc.json

      Object Lock — write-once-read-many

      # Activable uniquement à la création du bucket — impossible à activer plus tard
$ aws s3api create-bucket --bucket vault \
    --object-lock-enabled-for-bucket

# Rétention par défaut : 7 ans en mode COMPLIANCE (immuable pour tout le monde)
$ aws s3api put-object-lock-configuration \
    --bucket vault \
    --object-lock-configuration \
    '{"ObjectLockEnabled":"Enabled","Rule":{"DefaultRetention":{"Mode":"COMPLIANCE","Years":7}}}'
      GOVERNANCE vs COMPLIANCE : GOVERNANCE permet à un utilisateur IAM privilégié d'outrepasser la rétention avec s3:BypassGovernanceRetention ; COMPLIANCE bloque tout le monde, y compris l'utilisateur root, jusqu'à la fin de la période de rétention.

      Chiffrement & classes de stockage — SSE, KMS, IA, Glacier

      Choisissez une classe de stockage pour le coût (et la latence de récupération) et un mode de chiffrement pour la conformité. Les deux sont indépendants — chaque classe supporte chaque mode de chiffrement.

      Multipart & URLs pré-signées — pour gros fichiers et liens partagés

      Le aws s3 cp haut niveau utilise multipart automatiquement pour les fichiers plus gros que ~8 Mo. Quand vous pilotez les uploads par programme, voici les primitives s3api.

      Upload multipart — la danse en quatre temps

      ## 1. Démarrer — récupérer un UploadId
$ UID=$(aws s3api create-multipart-upload \
    --bucket B --key big.bin \
    --query 'UploadId' --output text)

## 2. Uploader chaque partie (5 Mo minimum, sauf la dernière)
$ ETAG1=$(aws s3api upload-part \
    --bucket B --key big.bin \
    --part-number 1 --upload-id $UID \
    --body part1.bin \
    --query 'ETag' --output text)
#  …répéter pour les parties 2..N…

## 3. Terminer — donner la liste des parties à S3 qui les recolle côté serveur
$ aws s3api complete-multipart-upload \
    --bucket B --key big.bin --upload-id $UID \
    --multipart-upload '{"Parts":[{"ETag":"...","PartNumber":1},...]}'

## 4 (alt). Annuler — libère le stockage que vous payeriez sinon
$ aws s3api abort-multipart-upload \
    --bucket B --key big.bin --upload-id $UID

## Voir les uploads orphelins (ils coûtent de l'argent !)
$ aws s3api list-multipart-uploads --bucket B
      Ajoutez une règle de cycle de vie avec AbortIncompleteMultipartUpload: { DaysAfterInitiation: 7 } pour que les uploads échoués ne fassent pas silencieusement gonfler votre facture.
      1 Arrêter un upload multipart bloqué pour libérer son stockage.
      $ aws s3api  \
    --bucket B --key K --upload-id $UID

      Exercices pratiques — S3 CLI en rafale

      Plus de vingt exercices couvrant les commandes et drapeaux les plus tapés. Mélange de aws s3 et aws s3api.

      1 Vérification d'identité — quel appel retourne votre ID de compte et votre ARN ?
      $ aws sts
      2 Utiliser un profil non par défaut pour une seule commande.
      $ aws s3 ls -- myprofile
      3 Surcharger la région pour une seule commande.
      $ aws s3 ls s3://b -- eu-west-1
      4 Copie récursive d'une arborescence locale vers S3.
      $ aws s3 cp ./site/ s3://my-site/ --
      5 Sync avec suppression.
      $ aws s3 sync ./pub/ s3://my-site/ --
      6 Sync mais exclure tout, puis ré-inclure uniquement les PNG.
      $ aws s3 sync ./img/ s3://b/img/ \
    --exclude "*" \
    -- "*.png"
      7 Générer une URL pré-signée d'un jour (en secondes).
      $ aws s3 presign s3://b/k --expires-in
      8 Créer un nouveau bucket. Complétez le verbe haut niveau.
      $ aws s3  s3://photos --region us-east-1
      9 Supprimer un bucket non vide.
      $ aws s3  s3://photos --force
      10 Valeur de chiffrement SSE-S3.
      $ aws s3 cp f.txt s3://b/k --sse
      11 Valeur de chiffrement SSE-KMS.
      $ aws s3 cp f.txt s3://b/k --server-side-encryption
      12 Classe de stockage pour Intelligent-Tiering.
      $ aws s3 cp f.txt s3://b/k --storage-class
      13 Verbe s3api qui ne récupère que les métadonnées (sans corps).
      $ aws s3api  --bucket B --key K
      14 Parcourir jusqu'à 1000 objets sous un préfixe.
      $ aws s3api  --bucket B --prefix p/ --max-keys 1000
      15 Activer le versioning. Complétez la chaîne de statut.
      $ aws s3api put-bucket-versioning --bucket B --versioning-configuration Status=
      16 Appliquer une politique de bucket depuis un fichier JSON local. Complétez l'URI file://.
      $ aws s3api put-bucket-policy --bucket B --policy
      17 Bloquer tout accès public — complétez le verbe après put-.
      $ aws s3api put- --bucket B \
    --public-access-block-configuration BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true
      18 JMESPath : projeter uniquement le champ Key de chaque objet listé.
      --query 'Contents[].'
      19 Annuler un upload multipart bloqué.
      $ aws s3api -multipart-upload --bucket B --key K --upload-id $UID
      20 Copie côté serveur d'une clé d'un bucket à un autre.
      $ aws s3api copy-object --bucket dest --key newkey -- srcBucket/srcKey
      21 Supprimer une version spécifique d'un objet.
      $ aws s3api delete-object --bucket B --key K -- 3HL4kqtJ
      22 Afficher toutes les versions (et delete-markers) d'un objet.
      $ aws s3api  --bucket B --prefix data/blob.bin
      23 Filtre JMESPath : ne garder que les objets strictement plus gros qu'1 MiB.
      --query 'Contents[?Size  `1048576`].Key'
      24 Récupérer les règles CORS au format JSON d'un bucket.
      $ aws s3api get-bucket- --bucket B

      Boss Drill — Pipeline S3 de bout en bout

      Un scénario, dix blancs. Vous mettez en production un nouveau bucket pour un dataset réglementé : le créer, le verrouiller, activer le versioning, uploader chiffré, lister, presigner un lien de partage, installer un cycle de vie, et nettoyer les anciennes versions. Chaque blanc est une vraie commande ou un drapeau.

      Pipeline : dataset de recherche en stockage froid

      La conformité dit : chiffré KMS, pas d'accès public, versionné, expiration à 365 jours, passage en Glacier après 90 jours. Vous opérez depuis le profil research en eu-west-1. 

      # 1. Choisir le bon profil + région pour toute la session shell
$ export AWS_=research
$ export AWS_REGION=eu-west-1

# 2. Confirmer qu'on est bien connecté en tant qu'utilisateur IAM research
$ aws sts 

# 3. Créer le bucket
$ aws s3  s3://lab-genomics-2026 --region eu-west-1

# 4. Verrouiller l'accès public au plus vite
$ aws s3api put- \
    --bucket lab-genomics-2026 \
    --public-access-block-configuration \
        BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true

# 5. Activer le versioning pour récupérer les suppressions accidentelles
$ aws s3api put-bucket-versioning --bucket lab-genomics-2026 \
    --versioning-configuration Status=

# 6. Chiffrement par défaut KMS pour que tout futur upload qui « oublie » --sse soit quand même chiffré
$ aws s3api put-bucket-encryption --bucket lab-genomics-2026 \
    --server-side-encryption-configuration \
    '{"Rules":[{"ApplyServerSideEncryptionByDefault":{"SSEAlgorithm":"","KMSMasterKeyID":"alias/lab-data"}}]}'

# 7. Uploader le premier batch avec chiffrement explicite + classe IA
$ aws s3 cp ./batch-001/ s3://lab-genomics-2026/raw/2026-05/ \
    --recursive \
    --sse aws:kms --sse-kms-key-id alias/lab-data \
    --storage-class 

# 8. Vérifier — lister avec les tailles en MiB
$ aws s3 ls s3://lab-genomics-2026/raw/ --recursive -- --summarize

# 9. Générer un lien de partage de 24h pour que le PI télécharge un fichier
$ aws s3  s3://lab-genomics-2026/raw/2026-05/cohort-A.tsv --expires-in 86400

# 10. Installer le cycle de vie : passer en GLACIER à 90j, expirer à 365j, tuer les anciennes versions à 30j
$ aws s3api put-bucket- \
    --bucket lab-genomics-2026 \
    --lifecycle-configuration file://lc.json

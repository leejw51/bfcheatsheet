ComfyUI — Stable Diffusion à base de nœuds

ComfyUI est un éditeur de graphe de nœuds pour Stable Diffusion. Au lieu d'une interface figée à curseurs (comme Automatic1111), vous câblez des boîtes qui font chacune une chose : charger un checkpoint, encoder un prompt, débruiter des latents, décoder en image. Le graphe est le programme. À la fin de cette page, vous aurez câblé un véritable workflow txt2img, réglé un KSampler et appelé l'API HTTP.

Pourquoi des nœuds ?
Chaque étape de la diffusion (charger le modèle, encoder le prompt, échantillonner, décoder) devient un nœud. Les fils transportent des données typées entre eux. Vous voyez tout le pipeline d'un coup d'œil — et vous pouvez le réorganiser à votre guise.
Reproductible
ComfyUI intègre tout le JSON du workflow dans les métadonnées PNG de chaque sortie. Glissez l'image sur la zone de travail — le graphe est restauré. Même seed → mêmes pixels.
Modulable
Besoin de ControlNet, IP-Adapter, AnimateDiff, d'une pile de LoRA ? Ajoutez un nœud, branchez un fil. Le graphe de base reste identique. Les nœuds personnalisés se livrent en dossiers dans custom_nodes/.
vs Automatic1111
A1111 est un assistant : démarrage facile, extension difficile. ComfyUI est de la plomberie : courbe d'apprentissage au début, flexibilité infinie ensuite. Les utilisateurs avancés, les runs en lot et tout pipeline non-trivial vivent ici.

Anatomie d'un nœud

Chaque nœud ComfyUI a la même forme : une boîte titrée avec des slots d'entrée à gauche, des slots de sortie à droite, et des widgets dans le corps (nombres, texte, listes déroulantes). Les fils se branchent dans les slots ; les valeurs des widgets sont stockées sur le nœud lui-même.

disséquer un nœud KSampler

slots d'entrée
Bord gauche. Chaque slot a un type — seuls les fils correspondants peuvent s'y brancher. Essayez de déposer un fil CLIP dans un slot MODEL et ComfyUI refuse.
slots de sortie
Bord droit. Une sortie peut alimenter plusieurs entrées (un-vers-plusieurs). La même sortie MODEL de Load Checkpoint va couramment vers KSampler et un loader de LoRA.
widgets
Corps du nœud. Nombres, curseurs, listes, zones de texte. Ils stockent des valeurs constantes sur le nœud — pas besoin de fil. Clic-droit sur un widget pour le convertir en slot d'entrée si vous voulez qu'il soit piloté par un autre nœud.
Astuce : double-cliquez sur le canevas vide pour ouvrir la palette de recherche de nœuds. Clic-droit sur un slot pour convertir en widget ou convertir en entrée.

Fils & types

Les fils sont typés. Chaque type a sa propre couleur pour qu'on voie d'un coup d'œil ce qui circule où. ComfyUI refuse de connecter des types incompatibles.

VAEencodeur/décodeur entre pixels & latentsLoad Checkpoint, VAELoader

Règle : un fil ne connecte que des types identiques. CONDITIONING en sortie ne va que vers CONDITIONING en entrée. Les couleurs sont un contrat visuel.

Nœuds personnalisés — étendre le graphe

ComfyUI de base livre ~50 nœuds. La communauté en propose des milliers. Ils s'installent en dossiers sous custom_nodes/ et enregistrent de nouvelles classes de nœuds au démarrage du serveur.

# 2. redémarrer le serveur. utilisez ensuite l'UI Manager pour installer le reste.
python main.py

Packs incontournables

ComfyUI-Manager
Le gestionnaire de paquets. Installe / met à jour / désactive les autres nœuds personnalisés depuis l'UI. Toujours le premier à ajouter.
ComfyUI ControlNet Aux
Préprocesseurs (Canny, Depth, OpenPose, Lineart) + le nœud ControlNetApply. Pilotez la composition avec des images de référence.
ComfyUI_IPAdapter_plus
Image-vers-conditioning. Traitez une photo de référence comme un prompt — transfert de style, préservation d'identité.
ComfyUI-AnimateDiff-Evolved
Branchez un module de mouvement → KSampler produit maintenant de la vidéo. Même graphe, dimension batch = images.
Impact Pack
Combos détection / segmentation / detailer. FaceDetailer est célèbre pour réparer les visages en un seul nœud.
rgthree-comfy
Nœuds de confort : muting de groupes, bypass rapide, gestion des seeds, barres de progression sur tout.

Mode API — piloter ComfyUI depuis le code

Tout workflow peut être enregistré en JSON API (engrenage → « Save (API Format) ») et soumis en HTTP. L'UI devient alors optionnelle — vous pouvez exécuter le graphe depuis un script, un backend, un bot Discord, n'importe quoi.

soumettre un job
import json, requests, uuid

# charger le JSON du workflow exporté via engrenage -> Save (API Format)
with open("workflow_api.json") as f:
    workflow = json.load(f)

# patcher le prompt pour cette exécution
workflow["6"]["inputs"]["text"] = "a corgi in a spacesuit"
workflow["3"]["inputs"]["seed"] = 12345

payload = {"prompt": workflow, "client_id": str(uuid.uuid4())}
r = requests.post("http://localhost:8188/prompt", json=payload)
prompt_id = r.json()["prompt_id"]
print("queued:", prompt_id)

Endpoints utiles

méthodechemince qu'il fait
POST/promptMet un workflow en file pour exécution ; retourne prompt_id.
GET/history/<prompt_id>Récupère les sorties (noms d'images, résultats de nœuds) une fois terminé.
GET/queueListe les jobs en cours + en attente.
GET/viewTélécharge une image de sortie : ?filename=...&type=output.
WS/ws?clientId=...WebSocket pour la progression en direct + previews nœud par nœud.
POST/upload/imageTéléverse une image (pour les entrées img2img / inpaint).

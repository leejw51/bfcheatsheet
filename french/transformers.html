niveau 01 // motivation

§01Pourquoi les transformers ?

Avant 2017, les modèles de séquences dominants étaient les RNN et les LSTM. Ils traitaient les tokens un à un, en propageant un état caché. Cela posait trois problèmes paralysants :

  • Goulot d'étranglement séquentiel. Impossible de paralléliser sur la dimension temporelle — le token t a besoin du token t-1.
  • Gradients qui disparaissent avec la distance. L'information du token 1 est compressée à travers chaque étape avant d'atteindre le token 100.
  • État caché de taille fixe. Tout le passé doit tenir dans un seul vecteur. La mémoire à longue portée est fragile.
idée clé

L'attention est tout ce qu'il faut

Vaswani et al. ont demandé : et si chaque token pouvait regarder chaque autre token directement, en une seule étape parallèle ? Pas de récurrence. Pas de convolution. Juste un mécanisme appris qui permet à chaque position d'accorder de l'attention à chaque autre position avec un poids appris.

Résultat : un entraînement qui passe à l'échelle linéairement avec le matériel, des gradients qui circulent en O(1) sauts, et un modèle qui peut en principe relier le premier mot d'un paragraphe au dernier aussi facilement qu'à son voisin.

niveau 02 // entrée

§02Tokenisation et plongements

Un transformer ne peut pas lire du texte. Il lit des identifiants entiers. La tokenisation est le schéma de compression avec perte qui transforme une chaîne de caractères en une séquence d'identifiants de tokens issus d'un vocabulaire fixe (typiquement 30k–200k entrées). Les tokeniseurs modernes (BPE, WordPiece, Unigram, Tiktoken) découpent en sous-mots, de sorte que les mots rares deviennent une séquence de morceaux courants plutôt qu'un token inconnu.

Chaque identifiant entier est ensuite mappé vers un vecteur dense via une matrice de plongements $E \in \mathbb{R}^{V \times d_{model}}$ où $V$ est la taille du vocabulaire. Lookup = une ligne de matrice. Résultat : pour une séquence de $L$ tokens vous obtenez un tenseur de forme [L, d_model].

niveau 03 // ordre

§03Encodage positionnel

L'auto-attention est invariante par permutation : mélangez les tokens d'entrée et (sans information positionnelle) vous obtenez la même sortie. C'est un problème pour le langage. La solution : ajouter un vecteur dépendant de la position à chaque plongement de token avant d'alimenter la pile.

L'article original utilise des sinusoïdes de longueur d'onde croissante géométriquement :

$$PE_{(pos,\,2i)} = \sin\!\Big(\tfrac{pos}{10000^{2i/d_{model}}}\Big), \qquad PE_{(pos,\,2i+1)} = \cos\!\Big(\tfrac{pos}{10000^{2i/d_{model}}}\Big)$$

visualiseur

Vagues de positions sinusoïdales

Chaque ligne est une dimension i. Les dimensions basses oscillent rapidement (les positions semblent très différentes) ; les dimensions hautes sont lisses (ordre à grande échelle).

La propriété astucieuse : $PE_{pos+k}$ est une fonction linéaire fixe de $PE_{pos}$, le modèle peut donc apprendre facilement les décalages relatifs. Les variantes modernes (RoPE, ALiBi, absolu appris) offrent des compromis différents mais l'objectif est le même — injecter l'ordre.

niveau 04 // le cœur

§04Auto-attention (Q · K · V)

Chaque token d'entrée est projeté en trois vecteurs via trois matrices apprises $W^Q, W^K, W^V \in \mathbb{R}^{d_{model} \times d_k}$ :

  • Requête (Q) — « que cherche-je ? »
  • Clé (K) — « que contiens-je ? »
  • Valeur (V) — « que vais-je apporter, si on m'écoute ? »

L'attention par produit scalaire normalisé est alors :

$$\mathrm{Attention}(Q,K,V) = \mathrm{softmax}\!\Big(\tfrac{QK^\top}{\sqrt{d_k}}\Big)\,V$$

Formes pour une longueur de séquence $L$ et une dimension de tête $d_k$ : Q: [L, d_k] K: [L, d_k] V: [L, d_k]. Le produit $QK^\top$ a la forme [L, L] — un score de similarité par couple (requête, clé). Le softmax sur les lignes le transforme en poids d'attention, puis on repondère $V$.

niveau 05 // ensemble

§05Attention multi-tête

Un seul motif d'attention est rarement suffisant. L'attention multi-tête exécute $h$ opérations d'attention en parallèle, chacune avec ses propres projections $W_i^Q, W_i^K, W_i^V$, et concatène les résultats :

$$\mathrm{MHA}(X) = \mathrm{Concat}(\mathrm{head}_1, \ldots, \mathrm{head}_h)\,W^O$$

Chaque tête apprend à attendre selon un motif relationnel différent — une tête peut suivre les dépendances syntaxiques, une autre la coréférence, une autre de simples décalages de position. La répartition maintient le coût total de calcul à peu près égal à celui d'une seule tête : avec $h=8$ têtes on choisit $d_k = d_{model}/h$.

niveau 06 // ciment

§06Feed-forward · résiduel · norme

Après l'attention, chaque position passe à travers un réseau feed-forward appliqué position par position — deux couches linéaires avec une non-linéarité au milieu (à l'origine ReLU, aujourd'hui généralement GELU ou SwiGLU) :

$$\mathrm{FFN}(x) = \max(0,\, xW_1+b_1)\,W_2 + b_2$$

La largeur cachée est typiquement 4·d_model. C'est ici que vit la majorité des paramètres du modèle — et une hypothèse populaire est que cette couche agit comme une mémoire clé-valeur stockant les connaissances factuelles.

Connexions résiduelles

La sous-couche d'attention et la sous-couche FFN sont toutes deux enveloppées dans des connexions résiduelles + layer norm :

$$x \leftarrow \mathrm{LayerNorm}(x + \mathrm{Sublayer}(x))$$

Pourquoi ? Les résidus créent une autoroute de gradient depuis la perte jusqu'à l'entrée, permettant à des centaines de couches de s'entraîner de manière stable. La LayerNorm garde les activations centrées et mises à l'échelle sur la dimension des caractéristiques, découplant l'optimisation de chaque couche des autres.

Les variantes modernes préfèrent la pré-norm : $x \leftarrow x + \mathrm{Sublayer}(\mathrm{LayerNorm}(x))$ — donne un entraînement plus stable en profondeur.

niveau 07 // topologie

§07Encodeur · décodeur · masquage

Trois variantes d'architecture dominent aujourd'hui :

  • Encodeur seul (BERT) — auto-attention bidirectionnelle. Chaque token voit chaque autre token. Idéal pour les tâches de compréhension (classification, recherche).
  • Décodeur seul (GPT) — auto-attention causale/masquée. Le token t ne peut attendre que les tokens $\le t$. Idéal pour la génération. C'est ce que la plupart des LLM modernes utilisent.
  • Encodeur-décodeur (T5, NMT original) — l'encodeur lit la source, le décodeur génère la cible tout en faisant de la cross-attention sur la sortie de l'encodeur.

Masquage causal — comment ?

Avant le softmax, mettez les entrées triangulaires supérieures de la matrice de scores à $-\infty$. Softmax de $-\infty$ vaut 0 — le modèle est donc mathématiquement incapable de regarder dans le futur :

niveau 08 // objectif

§08Entraînement : prédiction du prochain token

Pour les LLM décodeur seul (la famille GPT), tout l'objectif d'entraînement est d'une simplicité saisissante :

Étant donnés les tokens 1..t, prédire le token t+1.

Le modèle produit une distribution de probabilité sur le vocabulaire à chaque position. La perte est l'entropie croisée moyenne par rapport au véritable prochain token :

$$\mathcal{L} = -\frac{1}{N}\sum_{t=1}^{N} \log P_\theta(x_{t+1} \mid x_1, \ldots, x_t)$$

Le masquage causal rend cela efficace : une seule passe avant sur une séquence de longueur $N$ donne $N$ prédictions en parallèle, une par position. Pas besoin d'astuces de teacher forcing au-delà du décalage des étiquettes d'un cran.

Tout le reste — instruction tuning, RLHF, utilisation d'outils — vient se greffer sur cet objectif. Le pré-entraînement est la partie qui absorbe les données à l'échelle d'internet et où le modèle apprend « tout ». Le fine-tuning est la partie qui façonne son comportement.

récap

Le pipeline complet

  • 1. Tokens → identifiants entiers → plongements (+ encodage positionnel)
  • 2. Pile de N blocs transformer : MHA → Add&Norm → FFN → Add&Norm
  • 3. Projection linéaire finale vers la taille du vocabulaire, softmax → distribution du prochain token
  • 4. Perte d'entropie croisée par rapport à la vérité terrain du prochain token, sommée sur les positions
  • 5. Rétropropagation à travers chaque poids. Répétez ~10²² fois. Terminé.

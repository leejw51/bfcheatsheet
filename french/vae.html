Auto-encodeur variationnel — modélisation générative avec un goulot d'étranglement perméable

Un VAE est un auto-encodeur où le code latent est une distribution de probabilité, pas un point unique. L'encodeur produit $(\mu, \log\sigma^2)$, on échantillonne $z \sim \mathcal{N}(\mu, \sigma^2)$ via le truc de reparamétrisation, et le décodeur reconstruit $\hat{x}$. On entraîne avec une perte de reconstruction + KL vers une gaussienne unitaire. Une fois entraîné, on jette l'encodeur et on échantillonne $z \sim \mathcal{N}(0, I)$ pour générer de nouvelles données.

vs. auto-encodeur
Un AE classique compresse $x \to z \to \hat{x}$ comme une fonction déterministe. L'espace latent a des trous — échantillonner un $z$ aléatoire donne du bruit. Un VAE force le latent à ressembler à $\mathcal{N}(0, I)$, ainsi tout $z$ se décode en quelque chose de plausible.
pourquoi probabiliste ?
On veut générer : un moyen de tirer de nouveaux échantillons. Un VAE apprend $p_\theta(x) = \int p_\theta(x|z)\,p(z)\,dz$ où $p(z) = \mathcal{N}(0, I)$. On maximise une borne inférieure de $\log p(x)$ — l'ELBO.
le truc
L'échantillonnage n'est pas différentiable. Reparamétrer : $z = \mu + \sigma \odot \varepsilon$, $\varepsilon \sim \mathcal{N}(0, I)$. Désormais les gradients passent par $\mu$ et $\sigma$ comme si $z$ n'était qu'une fonction d'eux.

L'ELBO en une ligne

$\mathcal{L}_{\text{VAE}} = \underbrace{\mathbb{E}_{q(z|x)}[-\log p(x|z)]}_{\text{reconstruction}} \; + \; \beta \cdot \underbrace{\mathrm{KL}(q(z|x)\,\|\,p(z))}_{\text{régularisation}}$

Premier terme : à quel point le décodeur reconstruit $x$ depuis un $z$ échantillonné. Second terme : à quel point le $q(z|x) = \mathcal{N}(\mu, \sigma^2)$ de l'encodeur s'est éloigné du prior $p(z) = \mathcal{N}(0, I)$. L'encodeur doit encoder et rester proche du prior.

loss       = recon(x_hat, x) + beta * kl(mu, logvar)

Architecture — encodeur, échantillonneur, décodeur

Le VAE comporte trois composants. L'encodeur est un réseau de neurones déterministe avec deux têtes de sortie. L'échantillonneur injecte du bruit. Le décodeur est un réseau de neurones classique.

L'encodeur produit deux vecteurs

Contrairement à un AE normal qui produit un seul vecteur latent, un encodeur de VAE possède deux têtes de sortie : une pour la moyenne $\mu$, une pour $\log \sigma^2$ (log-variance — prédire la log-variance plutôt que $\sigma$ laisse la sortie du réseau libre, tandis que $\sigma$ reste positif après $\exp$).

encodeur en PyTorch
import torch
import torch.nn as nn

class Encoder(nn.Module):
    def __init__(self, in_dim=784, hidden=400, latent=20):
        super().__init__()
        self.fc1     = nn.Linear(in_dim, hidden)
        self.fc_mu   = nn.Linear(hidden, latent)   # tête moyenne
        self.fc_lv   = nn.Linear(hidden, latent)   # tête log-variance

    def forward(self, x):
        h = torch.relu(self.fc1(x))
        return self.fc_mu(h), self.fc_lv(h)        # (μ, log σ²)
décodeur — le voyage retour
class Decoder(nn.Module):
    def __init__(self, latent=20, hidden=400, out_dim=784):
        super().__init__()
        self.fc1 = nn.Linear(latent, hidden)
        self.fc2 = nn.Linear(hidden, out_dim)

    def forward(self, z):
        h = torch.relu(self.fc1(z))
        return torch.sigmoid(self.fc2(h))    # pixels dans [0, 1]

Reparamétrisation — le truc qui rend tout différentiable

Il faut rétro-propager à travers un échantillon $z \sim \mathcal{N}(\mu, \sigma^2)$. Un échantillonnage aléatoire pur n'a pas de gradient — pas de chemin de règle de chaîne à travers « tirer d'une distribution ». La solution : déplacer l'aléa à l'extérieur du réseau.

Perte — reconstruction + KL = ELBO

Maximiser l'ELBO équivaut à minimiser deux termes : à quel point le décodeur reconstruit mal $x$, et à quel point la distribution de l'encodeur s'écarte de $\mathcal{N}(0, I)$.

$\mathcal{L} = \underbrace{\mathrm{BCE}(\hat{x}, x)}_{\text{reconstruction}} \; + \; \beta \cdot \underbrace{-\tfrac{1}{2}\sum_{j}\bigl(1 + \log\sigma_j^2 - \mu_j^2 - \sigma_j^2\bigr)}_{\mathrm{KL}(\,\mathcal{N}(\mu,\sigma^2)\,\|\,\mathcal{N}(0,I)\,)}$
terme de reconstruction
F.binary_cross_entropy par pixel pour des pixels binaires / Bernoulli (MNIST), ou F.mse_loss pour des pixels gaussiens. Utilisez reduction='sum' pour coller à la dérivation analytique de l'ELBO, puis divisez par la taille du batch.
terme KL — forme close
Pour deux gaussiennes, la KL a une forme analytique propre. Pas besoin d'échantillonner : 
kl = -0.5 * torch.sum(
    1 + logvar - mu.pow(2) - logvar.exp()
)

le VAE canonique forward + perte (complet)
import torch
import torch.nn as nn
import torch.nn.functional as F

class VAE(nn.Module):
    def __init__(self, in_dim=784, hidden=400, latent=20):
        super().__init__()
        self.fc1     = nn.Linear(in_dim, hidden)
        self.fc_mu   = nn.Linear(hidden, latent)
        self.fc_lv   = nn.Linear(hidden, latent)
        self.fc3     = nn.Linear(latent, hidden)
        self.fc4     = nn.Linear(hidden, in_dim)

    def encode(self, x):
        h = torch.relu(self.fc1(x))
        return self.fc_mu(h), self.fc_lv(h)

    def reparameterize(self, mu, logvar):
        std = (0.5 * logvar).exp()
        eps = torch.randn_like(std)
        return mu + std * eps

    def decode(self, z):
        h = torch.relu(self.fc3(z))
        return torch.sigmoid(self.fc4(h))

    def forward(self, x):
        mu, logvar = self.encode(x.view(-1, 784))
        z          = self.reparameterize(mu, logvar)
        return self.decode(z), mu, logvar

def vae_loss(x_hat, x, mu, logvar, beta=1.0):
    recon = F.binary_cross_entropy(x_hat, x.view(-1, 784), reduction='sum')
    kl    = -0.5 * torch.sum(1 + logvar - mu.pow(2) - logvar.exp())
    return recon + beta * kl

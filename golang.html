Variables & Types

Go is statically typed but feels lighter than Java thanks to type inference and short-declaration syntax.

Declarations
var name string           // zero value: ""
var age int = 42
var ok = true             // type inferred

// short form (inside functions only)
name := "jane"
age, ok := 42, true

// block
var (
    host = "localhost"
    port = 8080
)
Constants
const Pi = 3.14159
const Name string = "go"

// iota: auto-incrementing 0,1,2,...
const (
    Sunday = iota   // 0
    Monday              // 1
    Tuesday             // 2
)

// typed bitmask via iota
type Flag uint8
const (
    Read   Flag = 1 << iota  // 1
    Write                       // 2
    Exec                        // 4
)
Basic types
// integers
int  int8  int16  int32  int64
uint uint8 uint16 uint32 uint64
byte   // alias for uint8
rune   // alias for int32 (one Unicode codepoint)

// floats / complex
float32 float64
complex64 complex128

Zero values
There's no "uninitialized". Every variable gets a typed zero.
var i int      // 0
var f float64  // 0
var b bool     // false
var s string   // ""
var p *int     // nil
var m map[string]int  // nil — read OK, write panics
var sl []int    // nil — append works
Type conversion
No implicit conversion — even between numeric types. Be explicit.
var i int = 42
var f float64 = float64(i)
var u uint    = uint(f)

// strings ⇄ bytes / runes
b := []byte("hello")
s := string(b)
r := []rune("héllo")  // for unicode iteration
Type definitions vs aliases
// new named type — not interchangeable with int
type UserID int

// alias — interchangeable
type ID = int

var u UserID = 7
// var n int = u           // compile error
var n int = int(u)    // explicit OK

Control Flow

One loop, one if, one switch — but each carries its weight.

if (with init)
if err := doWork(); err != nil {
    return err
}

// scoped variable disappears after the if
if v, ok := m["key"]; ok {
    fmt.Println(v)
} else {
    fmt.Println("missing")
}
for — Go's only loop
// classic three-clause
for i := 0; i < 10; i++ { … }

// while-style
for n < 100 { n *= 2 }

// infinite
for { … }

// range
for i, v := range slice { … }
for k, v := range m { … }
for _, line := range lines { … }
for i := range 10 { … }   // Go 1.22+
switch (no fallthrough by default)
switch os := runtime.GOOS; os {
case "darwin":
    fmt.Println("mac")
case "linux", "freebsd":
    fmt.Println("unix")
default:
    fmt.Println(os)
}

// expression switch on conditions
switch {
case n < 0:   return "neg"
case n == 0:  return "zero"
default:      return "pos"
}
type switch
switch v := x.(type) {
case int:
    fmt.Println("int", v*2)
case string:
    fmt.Println("str", len(v))
case nil:
    fmt.Println("nil")
default:
    fmt.Printf("%T\n", v)
}
Labels, break, continue, goto
outer:
for _, row := range grid {
    for _, cell := range row {
        if cell == target {
            break outer
        }
    }
}
Pitfall (pre-1.22): in older Go, the loop variable was reused across iterations — capturing it in a closure or goroutine bound the same address every time. Go 1.22 fixed this so each iteration has fresh v. If you target older Go, do v := v inside the loop.

Functions

Multiple returns, named returns, variadic args, defer, closures. Functions are values.

Basics & multiple returns
func add(a, b int) int { return a + b }

// Go's idiom: (value, error)
func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, errors.New("divide by zero")
    }
    return a / b, nil
}

q, err := divide(10, 3)
Named returns
func split(sum int) (x, y int) {
    x = sum * 4 / 9
    y = sum - x
    return      // "naked return"
}
Variadic
func sum(nums ...int) int {
    total := 0
    for _, n := range nums { total += n }
    return total
}

sum(1, 2, 3)
xs := []int{1, 2, 3}
sum(xs...)         // spread
defer
Runs at function return. LIFO order. Args are evaluated when defer is registered.
func readFile(path string) (string, error) {
    f, err := os.Open(path)
    if err != nil { return "", err }
    defer f.Close()    // always runs on the way out

    b, err := io.ReadAll(f)
    return string(b), err
}
Function values & closures
func counter() func() int {
    n := 0
    return func() int {
        n++
        return n
    }
}

next := counter()
fmt.Println(next(), next(), next())  // 1 2 3

Pointers

Go has pointers but no pointer arithmetic. They exist for two reasons: avoiding copies and letting a callee mutate the caller's value.

Take address, deref
x := 42
p := &x         // pointer to x
fmt.Println(*p)  // 42
*p = 100
fmt.Println(x)   // 100
Pass-by-value vs pointer
func grow(n int)    { n *= 2 }      // no effect
func growP(n *int) { *n *= 2 }    // mutates caller

x := 3
grow(x)        // x still 3
growP(&x)      // x now 6
new vs &T{}
// new(T) → *T pointing at zero-valued T
p := new(int)        // *int pointing at 0
*p = 7

// composite literal — more idiomatic
u := &User{Name: "jane"}
When to use pointers
Use pointer: mutation, large struct (avoid copy), interface satisfaction with pointer receivers, optional/nil sentinel.
Avoid: small immutable structs, slices/maps/channels (already reference types).
Pitfall: taking the address of a loop variable in old Go gave you the same address each iteration. In Go 1.22+ each iteration has a fresh variable, so &v in a range loop is safe. On older versions: do v := v first.

Structs & Methods

Structs hold data. Methods are functions with a receiver. Embedding gives you composition.

Define & instantiate
type User struct {
    ID    int      `json:"id"`
    Name  string   `json:"name"`
    Email string   `json:"email,omitempty"`
}

u := User{ID: 1, Name: "jane"}
v := User{1, "jane", ""}   // positional, fragile
p := &User{Name: "alex"}     // pointer to fresh struct
Methods (value vs pointer receiver)
type Counter struct{ n int }

// value receiver — works on a copy
func (c Counter) Get() int   { return c.n }

// pointer receiver — mutates caller
func (c *Counter) Inc()      { c.n++ }

c := Counter{}
c.Inc()                    // auto &c
fmt.Println(c.Get())
Embedding (composition)
type Logger struct{ Prefix string }
func (l Logger) Log(msg string) {
    fmt.Println(l.Prefix, msg)
}

type Server struct {
    Logger              // embedded
    Addr string
}

s := Server{Logger: Logger{"[srv]"}}
s.Log("started")        // promoted method
Anonymous struct
resp := struct {
    OK   bool
    Data []int
}{OK: true, Data: []int{1, 2, 3}}
Rule: if any method needs a pointer receiver, give them all pointer receivers for consistency. The method set of a value type does not include pointer-receiver methods, which can break interface satisfaction.

Interfaces

An interface is a set of method signatures. Anything that implements all of them satisfies it — implicitly. No implements keyword.

Define & satisfy
type Stringer interface {
    String() string
}

type Point struct{ X, Y int }

func (p Point) String() string {
    return fmt.Sprintf("(%d,%d)", p.X, p.Y)
}

// Point implicitly satisfies Stringer
var s Stringer = Point{3, 4}
empty interface / any
// any === interface{} (Go 1.18+)
var v any = 42
v = "hello"
v = []int{1, 2}

// type assertion — extract concrete type
n, ok := v.(int)
if ok { fmt.Println("got int", n) }
Common stdlib interfaces
type error interface      { Error() string }
type Stringer interface   { String() string }
type Reader interface     { Read(p []byte) (n int, err error) }
type Writer interface     { Write(p []byte) (n int, err error) }
type Closer interface     { Close() error }
Embedding interfaces
type ReadCloser interface {
    Reader
    Closer
}
nil interface ≠ nil pointer: an interface variable holds (type, value). If you store a nil *MyError in an error variable, the interface itself is non-nil. if err != nil will be true even though the underlying pointer is nil. Avoid returning typed nils as errors.

Slices, Maps, Arrays

Arrays are fixed-size and rare in Go code. Slices are the workhorse. Maps are unordered.

Slice basics
var s []int                  // nil slice (len=0, cap=0)
s = []int{1, 2, 3}            // literal
s = make([]int, 5)             // len 5, cap 5
s = make([]int, 0, 10)           // len 0, cap 10

// re-slice (shares the backing array!)
sub := s[1:3]
// to copy: dst := make([]int, len(sub)); copy(dst, sub)
append (and growth)
s := []int{}
s = append(s, 1)           // cap usually doubles when full
s = append(s, 2, 3, 4)
s = append(s, other...)    // merge another slice

// pre-size for known size — avoids reallocations
results := make([]int, 0, len(input))
Maps
m := make(map[string]int)
m["a"] = 1
v, ok := m["a"]            // ok==true if present
delete(m, "a")
n := len(m)

// literal
ages := map[string]int{
    "jane": 30,
    "alex": 28,
}

// iteration order is RANDOMIZED
for k, v := range ages { … }
Arrays (fixed size)
var a [3]int            // [0 0 0]
b := [3]int{1, 2, 3}
c := [...]int{1, 2, 3}    // size inferred

// arrays are values — copied on assignment
d := b
d[0] = 99
// b unchanged
slices.* helpers (Go 1.21+)
import "slices"

slices.Sort(s)
slices.SortFunc(users, func(a, b User) int {
    return cmp.Compare(a.Name, b.Name)
})
slices.Contains(s, x)
slices.Index(s, x)
slices.Reverse(s)
slices.Equal(a, b)
slices.Concat(a, b)
maps.* helpers (Go 1.21+)
import "maps"

keys := slices.Sorted(maps.Keys(m))
maps.Copy(dst, src)
maps.DeleteFunc(m, func(k string, v int) bool {
    return v < 0
})

Concurrency

Don't communicate by sharing memory; share memory by communicating. Goroutines + channels are Go's whole pitch.

goroutine
go work(item)             // run in new goroutine
go func() {
    fmt.Println("hi")
}()

// goroutines are cheap (~2 KB stack)
// the runtime multiplexes them onto OS threads
channels
ch := make(chan int)         // unbuffered
ch := make(chan int, 10)     // buffered, cap 10

ch <- 42                     // send (blocks if full / no receiver)
v := <-ch                    // receive
v, ok := <-ch                // ok=false on closed empty
close(ch)                    // senders only — never the receiver

// range until closed
for v := range ch { fmt.Println(v) }
select
select {
case v := <-a:
    fmt.Println("a", v)
case v := <-b:
    fmt.Println("b", v)
case out <- 7:
    fmt.Println("sent")
case <-time.After(time.Second):
    fmt.Println("timeout")
default:
    fmt.Println("nothing ready")
}
sync.WaitGroup
var wg sync.WaitGroup
for _, item := range items {
    wg.Add(1)
    go func(it Item) {
        defer wg.Done()
        work(it)
    }(item)
}
wg.Wait()
sync.Mutex
type Counter struct {
    mu sync.Mutex
    n  int
}
func (c *Counter) Inc() {
    c.mu.Lock()
    defer c.mu.Unlock()
    c.n++
}
context for cancellation
ctx, cancel := context.WithTimeout(parent, 5*time.Second)
defer cancel()

select {
case <-ctx.Done():
    return ctx.Err()
case v := <-resultCh:
    return v, nil
}

Errors & Generics

Errors are values. Wrap them, inspect them, but don't panic for normal flow.

Create & wrap
errors.New("bad input")
fmt.Errorf("open %s: %w", path, err)   // %w wraps

// sentinel
var ErrNotFound = errors.New("not found")

// custom type
type ParseError struct{ Line int; Msg string }
func (e *ParseError) Error() string {
    return fmt.Sprintf("line %d: %s", e.Line, e.Msg)
}
Inspect: errors.Is / errors.As
if errors.Is(err, ErrNotFound) {
    // matches sentinel through wraps
}

var pe *ParseError
if errors.As(err, &pe) {
    fmt.Println("line", pe.Line)
}

// errors.Join (Go 1.20+)
err := errors.Join(err1, err2)
panic / recover
Use only for truly unrecoverable bugs (or top-level HTTP middleware).
defer func() {
    if r := recover(); r != nil {
        log.Println("recovered:", r)
    }
}()
Generics (Go 1.18+)
func Map[T, U any](xs []T, f func(T) U) []U {
    out := make([]U, len(xs))
    for i, x := range xs {
        out[i] = f(x)
    }
    return out
}

// type sets via constraints
type Number interface {
    ~int | ~float64
}
func Sum[T Number](xs []T) T { … }

Stdlib quick-reference

The standard library is the killer feature. Here are the bits you'll reach for first.

net/http server

Hello server
package main

import (
    "fmt"
    "net/http"
)

func main() {
    mux := http.NewServeMux()
    mux.HandleFunc("GET /hello/{name}",
        func(w http.ResponseWriter, r *http.Request) {
            name := r.PathValue("name")
            fmt.Fprintf(w, "hi %s", name)
        })
    http.ListenAndServe(":8080", mux)
}
HTTP client
client := &http.Client{Timeout: 5*time.Second}
req, _ := http.NewRequestWithContext(ctx,
    http.MethodGet, "https://api/x", nil)
req.Header.Set("Authorization", "Bearer "+tok)

resp, err := client.Do(req)
if err != nil { return err }
defer resp.Body.Close()

body, err := io.ReadAll(resp.Body)
Middleware pattern
func withLog(h http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        start := time.Now()
        h.ServeHTTP(w, r)
        slog.Info("req", "path", r.URL.Path,
                 "dur", time.Since(start))
    })
}

testing

Table-driven test
func TestAdd(t *testing.T) {
    for _, tc := range []struct {
        name    string
        a, b, w int
    }{
        {"zero", 0, 0, 0},
        {"pos",  2, 3, 5},
        {"neg", -1, 1, 0},
    } {
        t.Run(tc.name, func(t *testing.T) {
            if g := add(tc.a, tc.b); g != tc.w {
                t.Errorf("got %d, want %d", g, tc.w)
            }
        })
    }
}

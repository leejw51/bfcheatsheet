One tensor, all the way to logits

The Attention Is All You Need transformer is a recipe: take a tensor of source token IDs, push it through an encoder, push the target tokens through a decoder that cross-attends to the encoder, project to the vocabulary, and out come next-token logits. This page tracks shape — every stage from input IDs to final [B, T_tgt, V] probabilities, color-coded so you can read the paper without ever losing the tensor.

paper defaults (base)
B (batch) = 2 # batch size
T_src (src len) = 8 # source tokens
T_tgt (tgt len) = 7 # target tokens
d_model = 512 # embedding dim
h (heads) = 8 # attention heads
d_k = d_v = 64 # d_model / h
d_ff = 2048# FFN inner dim (4·d_model)
V (vocab) = 37000# EN→DE BPE vocab
N (layers) = 6 # enc layers = dec layers
color legend (used everywhere)
B batch T tokens d_model h heads d_k d_ff

When a dimension changes between stages, it lights up. Watch the same tensor get split into heads, multiplied, scaled, soft-maxed, and stitched back together.

Stages 1–3 · From token IDs to a contextual-ready tensor

Before any attention, the input has to become a tensor of vectors. Three quick stages: integer IDs in, real-valued vectors out, with positional info baked in by addition.

stage 1 · token IDs
Tokenizer output. Two examples in a batch, eight tokens each.
$\text{ids}\in\mathbb{Z}^{B\times T}$
shape [B=2, T=8] · dtype: int32 
# two sentences, each padded/truncated to T=8 tokens
ids = [[ 101, 2057, 2293, 3019, 2025, 2748, 102, 0 ],
       [ 101, 7592, 2088,  102,    0,    0,   0, 0 ]]
# shape: [B=2, T=8]
stage 2 · embedding lookup

A weight matrix $E\in\mathbb{R}^{V\times d_{model}}$ holds one row per vocabulary entry. Lookup by ID. The tensor gains a dimension: a real-valued vector of size d_model replaces each integer.

$X = E[\text{ids}] \;\in\; \mathbb{R}^{B\times T\times d_{model}}$
[2, 8] → [2, 8, 512] 
import torch.nn as nn
embed = nn.Embedding(num_embeddings=30522, embedding_dim=512)
X = embed(ids)            # [B, T] -> [B, T, d_model]
print(X.shape)            # torch.Size([2, 8, 512])

Param count: $V \cdot d_{model} = 30522 \cdot 512 \approx 15.6\text{M}$ — the embedding table alone is bigger than the rest of any single layer.

stage 3 · + positional encoding

Self-attention is permutation-equivariant — without position info, "the cat sat" and "sat cat the" produce the same outputs. The paper adds a fixed sinusoidal pattern of the same shape as $X$.

$\mathrm{PE}_{(pos, 2i)} = \sin\!\big(pos / 10000^{2i/d_{model}}\big)$
$\mathrm{PE}_{(pos, 2i+1)} = \cos\!\big(pos / 10000^{2i/d_{model}}\big)$
[2, 8, 512] + [8, 512] → [2, 8, 512] 
# PE has shape [T, d_model], broadcasts over B
X = X + PE[:T]            # [B, T, d_model] + [T, d_model] -> [B, T, d_model]

Same shape in, same shape out. The information goes into the values, not the dims.

End of stage 3: we now have a tensor of shape [B, T, d_model] that's both about the words and about where they are. Every later stage is a transformation on this layout.

Stages 4–11 · Multi-head attention, dimension by dimension

The whole reason this paper exists. Each token becomes a query, key, and value; similarities are computed, scaled, soft-maxed, and used to mix the values. We also reshape twice: once to split heads, once to glue them back.

stages 4 · Q, K, V projections

Three independent linear maps, each $\mathbb{R}^{d_{model}}\to\mathbb{R}^{d_{model}}$:

$Q = X W^Q,\quad K = X W^K,\quad V = X W^V$
X [2, 8, 512] Wᵠ [512, 512] → Q [2, 8, 512] 
# three linear layers, no bias in the paper but a bias is fine in practice
Wq = nn.Linear(d_model, d_model)   # 512*512 = 262 144 params
Wk = nn.Linear(d_model, d_model)
Wv = nn.Linear(d_model, d_model)

Q = Wq(X)                              # [B, T, d_model]
K = Wk(X)                              # [B, T, d_model]
V = Wv(X)                              # [B, T, d_model]

Three matrices of shape $[d_{model}, d_{model}]$ — total params here: $3\cdot 512\cdot 512 = 786\,432$. In some implementations these three are fused into a single $[d_{model}, 3\cdot d_{model}]$ matrix.

stage 5 · reshape & transpose for h heads

Slice the last dim into $h$ chunks of $d_k = d_{model}/h$ each, then transpose so heads become an outer dim. No new parameters — just view + permute.

[2, 8, 512] view→ [2, 8, 8, 64] transpose(1,2)→ [2, 8, 8, 64] 
B, T, d_model = Q.shape
h, d_k = 8, d_model // h         # 8, 64

# split last dim into (h, d_k), then move heads next to batch
Q = Q.view(B, T, h, d_k).transpose(1, 2)   # [B, h, T, d_k]
K = K.view(B, T, h, d_k).transpose(1, 2)   # [B, h, T, d_k]
V = V.view(B, T, h, d_k).transpose(1, 2)   # [B, h, T, d_k]
Why heads matter: with $h=8$ separate $[T, d_k=64]$ subspaces, each head can specialize on a different relational pattern. Compute is unchanged: 8 small $64$-dim attentions instead of one $512$-dim one.
stage 6 · Q @ Kᵀ — pairwise similarity

For each head, every query attends to every key. The matmul contracts the $d_k$ axis, producing a $T\times T$ score matrix per head per batch.

$\text{scores} = Q K^\top$
Q [2, 8, 8, 64] Kᵀ [2, 8, 64, 8] → scores [2, 8, 8, 8] 
scores = Q @ K.transpose(-2, -1)   # [B, h, T, T]
# entry (b, head, i, j) = how much token i attends to token j (in that head)
stage 7 · scale by 1/√d_k

Without scaling, dot products of $d_k$-dim vectors grow as $\sqrt{d_k}$ — large values push softmax into the saturating regime where gradients vanish. Divide every score by $\sqrt{d_k}$.

$\text{scaled} = \text{scores} / \sqrt{d_k} = \text{scores} / 8$
[2, 8, 8, 8] ÷ √64 [2, 8, 8, 8] shape unchanged 
import math
scores = scores / math.sqrt(d_k)   # 1/√64 = 1/8 = 0.125
stage 8 · softmax along the last axis

Turn raw scores into a probability distribution per query. Each row of the $[T,T]$ block sums to 1 — these are the attention weights.

$A = \mathrm{softmax}(\text{scaled},\ \text{dim}=-1)$
[2, 8, 8, 8] → [2, 8, 8, 8] rows sum to 1 
import torch.nn.functional as F
A = F.softmax(scores, dim=-1)    # [B, h, T, T]

For decoder self-attention, an additive $-\infty$ mask is applied before softmax to enforce causality.

stage 9 · attention @ V

Mix value vectors using the attention weights. Each query's output is a weighted sum of all $T$ value vectors. The contracting axis is the inner $T$.

A [2, 8, 8, 8] V [2, 8, 8, 64] → out [2, 8, 8, 64] 
head_out = A @ V                  # [B, h, T, d_k]
stage 10 · concat heads (transpose + reshape)

Glue the heads back into one big last-dim. transpose(1,2) moves heads next to $d_k$, then reshape merges them. Param-free.

[2, 8, 8, 64] transpose→ [2, 8, 8, 64] reshape→ [2, 8, 512] 
out = head_out.transpose(1, 2).contiguous()   # [B, T, h, d_k]
out = out.reshape(B, T, d_model)              # [B, T, d_model]
stage 11 · output projection W_O

One more $\mathbb{R}^{d_{model}\to d_{model}}$ linear map. Same shape, but lets the model mix information across heads after concat.

$\text{MHA}(X) = \mathrm{Concat}(\text{head}_1, \dots, \text{head}_h)\,W^O$
[2, 8, 512] ·W_O [2, 8, 512] 
Wo = nn.Linear(d_model, d_model)   # another 262 144 params
mha_out = Wo(out)                       # [B, T, d_model]

Total MHA params: $4 \cdot d_{model}^2 = 4\cdot 512^2 = 1{,}048{,}576$ (Q, K, V, O — each $d_{model}\times d_{model}$).

End of attention: we left with shape [B, T, d_model] — exactly the same shape we entered with. Attention is a same-shape operator.

Stages 12–16 · LayerNorm, FFN, and the residual highway

After attention, the encoder block stitches everything with two residual connections, two LayerNorms, and one position-wise FFN. The FFN is where most of the parameters live — and where, many believe, factual knowledge is stored.

stage 12 · residual + LayerNorm

Add the original $X$ back (skip connection), then normalize per-token across $d_{model}$.

$X' = \mathrm{LayerNorm}(X + \mathrm{MHA}(X))$
[2, 8, 512] + [2, 8, 512] → LN → [2, 8, 512] 
ln1 = nn.LayerNorm(d_model)      # 2 * d_model = 1024 params (γ, β)
X_post = ln1(X + mha_out)         # [B, T, d_model]

LN normalizes across the last axis: each $[d_{model}]$ vector becomes mean-0, var-1, then re-scaled by learned $\gamma, \beta\in\mathbb{R}^{d_{model}}$. Both batch and token positions stay independent.

stages 13–14 · FFN linear 1 + ReLU

A position-wise feed-forward network: same MLP applied to every token independently. First linear expands $d_{model} \to d_{ff}$ (typically $4\cdot d_{model}$).

$H = \mathrm{ReLU}(X' W_1 + b_1)$
[2, 8, 512] ·W₁ [2, 8, 2048] → ReLU → [2, 8, 2048] 
W1 = nn.Linear(d_model, d_ff)       # 512 * 2048 = 1 048 576 params
H = F.relu(W1(X_post))            # [B, T, d_ff]
stage 15 · FFN linear 2

Second linear projects back: $d_{ff} \to d_{model}$. Restores the canonical shape so the next block can start fresh.

$Y = H W_2 + b_2$
[2, 8, 2048] ·W₂ [2, 8, 512] 
W2 = nn.Linear(d_ff, d_model)       # 2048 * 512 = another 1 048 576 params
ffn_out = W2(H)                       # [B, T, d_model]

Total FFN params: $2\cdot d_{model}\cdot d_{ff} = 2\cdot 512\cdot 2048 = 2{,}097{,}152$ — roughly twice the cost of all four attention projections combined. This is where big models put their weight.

stage 16 · residual + LayerNorm (again)

Same pattern as stage 12. Add the pre-FFN tensor, layer-normalize, and the encoder block is done.

$Y' = \mathrm{LayerNorm}(X' + Y)$
[2, 8, 512] → [2, 8, 512] block-output shape = block-input shape 
ln2 = nn.LayerNorm(d_model)
out = ln2(X_post + ffn_out)       # [B, T, d_model]
Stack & repeat: the paper stacks $N=6$ identical encoder blocks. Each one's input shape equals its output shape, which is why they compose cleanly. Total encoder params (one block, base): roughly $4\cdot 512^2 + 2\cdot 512\cdot 2048 + 4\cdot 512 \approx 3.15$M.

The whole encoder block in 14 lines

EncoderBlock — minimal PyTorch
class EncoderBlock(nn.Module):
    def __init__(self, d_model=512, h=8, d_ff=2048):
        super().__init__()
        self.attn = nn.MultiheadAttention(d_model, h, batch_first=True)
        self.ffn  = nn.Sequential(
            nn.Linear(d_model, d_ff), nn.ReLU(), nn.Linear(d_ff, d_model)
        )
        self.ln1 = nn.LayerNorm(d_model)
        self.ln2 = nn.LayerNorm(d_model)

    def forward(self, x):
        # x: [B, T, d_model]
        a, _ = self.attn(x, x, x)
        x = self.ln1(x + a)
        x = self.ln2(x + self.ffn(x))
        return x                       # [B, T, d_model]

Stages 17–21 · The decoder block

The decoder takes target tokens (shifted right by one) and produces a tensor that knows about both (a) the target tokens it has seen so far — via masked self-attn — and (b) the source — via cross-attn, where Q comes from the decoder and K, V come from the encoder output (the “memory”). Same embedding dim, same residual+LN discipline as the encoder. Three sub-blocks per layer, six layers in the paper.

paper config (still)
B = 2, T_src = 8, T_tgt = 7
d_model = 512, h = 8, d_k = 64
# T_tgt is independent of T_src.
# cross-attn is the only place they meet.
why “shifted right”?

During training the decoder sees the gold target prefixed with <sos> and predicts the same target ending in <eos>.

# target  : "die katze sitzt"           ← truth
# dec_in  : "<sos> die katze sitzt"     ← shifted right (input)
# labels  : "die katze sitzt <eos>"     ← what we predict

Stage 17 · Decoder input → embedding

[B, T_tgt] → embed lookup → [B, T_tgt, d_model] = [2, 7, 512]

Stage 19 · Masked self-attention (causal mask)

Identical to encoder self-attn — except the score matrix gets −∞ stamped on every position above the diagonal so position i can never look at i+1, i+2, .... After softmax those positions become exactly 0.

# scores: [B, h, T_tgt, T_tgt] = [2, 8, 7, 7]
mask = torch.triu(torch.ones(T_tgt, T_tgt), diagonal=1).bool()
scores = scores.masked_fill(mask, float('-inf'))
A = F.softmax(scores, dim=-1)        # lower triangle, rows sum to 1
[B, h, T_tgt, T_tgt] = [2, 8, 7, 7] upper-triangle = 0 after softmax

Stage 20 · Cross-attention (Q ← decoder, K, V ← encoder)

This is where the encoder finally talks to the decoder. The decoder emits queries; the encoder output supplies keys and values.

# dec_x: [B, T_tgt, d_model] = [2, 7, 512]   ← from masked self-attn
# memory: [B, T_src, d_model] = [2, 8, 512]  ← encoder output (stage 16)
Q = W_q(dec_x).view(B, T_tgt, h, d_k).transpose(1, 2)   # [B, h, T_tgt, d_k]
K = W_k(memory).view(B, T_src, h, d_k).transpose(1, 2)   # [B, h, T_src, d_k]
V = W_v(memory).view(B, T_src, h, d_k).transpose(1, 2)   # [B, h, T_src, d_k]
scores = Q @ K.transpose(-2, -1) / math.sqrt(d_k)         # [B, h, T_tgt, T_src] = [2, 8, 7, 8]
A = F.softmax(scores, dim=-1)                              # last axis is T_src
out = (A @ V).transpose(1, 2).reshape(B, T_tgt, d_model) # [2, 7, 512]

Crucially: the score matrix is not square. It is [B, h, T_tgt, T_src] — every target token attends over every source token.

The decoder block in 16 lines

DecoderBlock — minimal PyTorch
class DecoderBlock(nn.Module):
    def __init__(self, d=512, h=8, d_ff=2048):
        super().__init__()
        self.self_attn  = nn.MultiheadAttention(d, h, batch_first=True)
        self.cross_attn = nn.MultiheadAttention(d, h, batch_first=True)
        self.ffn = nn.Sequential(nn.Linear(d, d_ff), nn.ReLU(), nn.Linear(d_ff, d))
        self.ln1, self.ln2, self.ln3 = nn.LayerNorm(d), nn.LayerNorm(d), nn.LayerNorm(d)

    def forward(self, x, memory, causal_mask):
        # x:      [B, T_tgt, d]      memory: [B, T_src, d]
        a, _ = self.self_attn(x, x, x, attn_mask=causal_mask)
        x = self.ln1(x + a)                 # [B, T_tgt, d]
        c, _ = self.cross_attn(x, memory, memory)
        x = self.ln2(x + c)                 # [B, T_tgt, d]
        x = self.ln3(x + self.ffn(x))
        return x                            # [B, T_tgt, d]

Output of the decoder stack (after 6 of these): [B, T_tgt, d_model] = [2, 7, 512]. Next stop: project to vocabulary.

Stages 22–23 · From hidden state to logits

A single nn.Linear(d_model, V) turns each per-token hidden vector into a score for every word in the vocabulary. Softmax along the last axis turns those scores into probabilities. That's the entire output head. In the paper, W_lm is tied to the input embedding matrix — same parameters, transposed.

Stage 22 · Linear projection to vocabulary

[B, T_tgt, d_model] @ W_lm.T → [B, T_tgt, V] = [2, 7, 37000] 
# dec_out: [B, T_tgt, d_model]
# W_lm:    [V, d_model]   ← often tied to embed.weight
logits = dec_out @ W_lm.T           # [B, T_tgt, V]
# or simply:
logits = self.lm_head(dec_out)      # nn.Linear(d_model, V)

These are raw scores — one per vocabulary token, per position, per example. Not probabilities yet. Don't softmax them if your loss is CrossEntropyLoss; it does that internally for numerical stability.

Stage 23 · Softmax → probabilities (only when you actually need them)

probs = F.softmax(logits, dim=-1)   # [B, T_tgt, V] — rows over V sum to 1

At each of the B·T_tgt positions you now have a distribution over the entire vocabulary. Most of those 37,000 numbers are tiny. A handful concentrate near the right answer.

Parameter count of the output head

W_lm alone is V × d_model = 37000 × 512 ≈ 19M parameters — usually the single largest weight in the model. The paper ties it to the input embedding matrix, halving the weight memory and (empirically) helping generalization.

The full forward, end-to-end

Transformer.forward — paper-faithful
def forward(self, src_ids, tgt_in_ids):
    # src_ids:    [B, T_src]   tgt_in_ids: [B, T_tgt]
    src = self.src_embed(src_ids) + self.pos(src_ids)        # [B, T_src, d]
    tgt = self.tgt_embed(tgt_in_ids) + self.pos(tgt_in_ids)    # [B, T_tgt, d]

    memory = src
    for layer in self.encoder_layers:
        memory = layer(memory)                                  # [B, T_src, d]

    causal = self.causal_mask(tgt_in_ids.shape[1])
    x = tgt
    for layer in self.decoder_layers:
        x = layer(x, memory, causal)                            # [B, T_tgt, d]

    logits = self.lm_head(x)                                     # [B, T_tgt, V]
    return logits

Training — the whole paper, one step

A single training step on a batch of (source, target) string pairs. We tokenize both, build the encoder & decoder inputs, run the full transformer (embeds → positional → 6 encoder layers → 6 decoder layers → linear head), compute cross-entropy against the shifted-right labels, backprop, and take an optimizer step. Initial input: a list of string pairs. Final output: a scalar loss + updated parameters.

The whole training step in one shot

train_step — paper-faithful, ~20 lines
def train_step(model, optimizer, batch):
    # batch: list[(src_str, tgt_str)] of length B
    src_ids = tokenize_src(batch)         # [B, T_src]    int64
    dec_in, labels = tokenize_tgt(batch)  # [B, T_tgt] each

    src = model.src_embed(src_ids)*math.sqrt(d) + model.pos(src_ids)
    tgt = model.tgt_embed(dec_in)*math.sqrt(d) + model.pos(dec_in)

    memory = src
    for layer in model.encoder_layers: memory = layer(memory)

    causal = model.causal_mask(dec_in.shape[1])
    x = tgt
    for layer in model.decoder_layers: x = layer(x, memory, causal)

    logits = model.lm_head(x)              # [B, T_tgt, V]
    loss = F.cross_entropy(
        logits.view(-1, V), labels.view(-1),
        ignore_index=-100, label_smoothing=0.1)

    optimizer.zero_grad(); loss.backward(); optimizer.step()
    return loss.item()                     # scalar

Summary of shapes during one training step: raw strings (B) → [B, T_src] → [B, T_src, d] → [B, T_tgt, d] → [B, T_tgt, V] → scalar loss → ∂L/∂θ on every param.

The whole inference loop in one shot

generate — greedy, ~20 lines
@torch.no_grad()
def generate(model, src_text, max_len=32):
    src_ids = tokenize_src([(src_text, "")])      # [1, T_src]

    src = model.src_embed(src_ids)*math.sqrt(d) + model.pos(src_ids)
    memory = src
    for layer in model.encoder_layers:
        memory = layer(memory)                          # [1, T_src, d]

    dec_in = torch.tensor([[SOS]])                         # [1, 1]
    for _ in range(max_len):
        tgt = model.tgt_embed(dec_in)*math.sqrt(d) + model.pos(dec_in)
        causal = model.causal_mask(dec_in.shape[1])
        x = tgt
        for layer in model.decoder_layers:
            x = layer(x, memory, causal)                 # [1, t, d]
        logits = model.lm_head(x)                          # [1, t, V]
        next_id = logits[:, -1, :].argmax(-1, keepdim=True)
        if next_id.item() == EOS: break
        dec_in = torch.cat([dec_in, next_id], dim=1)    # [1, t+1]

    return detokenize(dec_in[0, 1:].tolist())          # str

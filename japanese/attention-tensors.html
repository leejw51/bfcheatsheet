1つのテンソル、logitsまで一気に

Attention Is All You Need の Transformer はひとつのレシピです：ソーストークンIDのテンソルを受け取り、 エンコーダーを通し、ターゲットトークンをエンコーダーにクロスアテンションするデコーダーに通し、 語彙に射影して、次トークンのlogitsを出力する。 このページでは形状を追います — 入力IDから最終的な [B, T_tgt, V]確率まで、すべてのステージを色分けして示し、論文を読む際にテンソルを見失わないようにします。

論文デフォルト (ベース)
B (batch) = 2 # バッチサイズ
T_src (src len) = 8 # ソーストークン数
T_tgt (tgt len) = 7 # ターゲットトークン数
d_model = 512 # 埋め込み次元
h (heads) = 8 # アテンションヘッド数
d_k = d_v = 64 # d_model / h
d_ff = 2048# FFN内部次元 (4·d_model)
V (vocab) = 37000# EN→DE BPEボキャブラリー
N (layers) = 6 # encレイヤー数 = decレイヤー数
カラーレジェンド（全体で使用）
B バッチ T トークン d_model h ヘッド d_k d_ff

ステージ間で次元が変わると、それが点灯します。同じテンソルがヘッドに分割され、 乗算・スケーリング・softmax・結合される過程を観察してください。

エンコーダーブロックの図解

このページの使い方：タブを順番に移動してください。ライブ形状は 最強のタブです — B、T、d_model、h を変更すると全ステージがリアルタイム更新され、それぞれ相対テンソルサイズを示すバーが表示されます。 色付きの次元をクリック（例： T=8）するとページ全体でその次元をトレースできます。 ショートカットは?キーで確認できます。

ステージ1〜3 · トークンIDからコンテキスト対応テンソルへ

アテンションの前に、入力はベクトルのテンソルに変換される必要があります。3つのステップで完了します： 整数IDを入力し、実数値ベクトルを出力として得て、加算により位置情報を埋め込みます。

ステージ1 · トークンID
トークナイザーの出力。バッチ内の2つの例、それぞれ8トークン。
$\text{ids}\in\mathbb{Z}^{B\times T}$
shape [B=2, T=8] · dtype: int32 
# 2文、それぞれT=8トークンにパディング/切り詰め
ids = [[ 101, 2057, 2293, 3019, 2025, 2748, 102, 0 ],
       [ 101, 7592, 2088,  102,    0,    0,   0, 0 ]]
# 形状: [B=2, T=8]
ステージ2 · 埋め込みルックアップ

重み行列 $E\in\mathbb{R}^{V\times d_{model}}$ はボキャブラリーエントリーごとに1行保持しています。 IDでルックアップします。テンソルが1次元増えます：各整数がd_modelサイズの実数値ベクトルに置き換わります。

$X = E[\text{ids}] \;\in\; \mathbb{R}^{B\times T\times d_{model}}$
[2, 8] → [2, 8, 512] 
import torch.nn as nn
embed = nn.Embedding(num_embeddings=30522, embedding_dim=512)
X = embed(ids)            # [B, T] -> [B, T, d_model]
print(X.shape)            # torch.Size([2, 8, 512])

パラメーター数: $V \cdot d_{model} = 30522 \cdot 512 \approx 15.6\text{M}$ — 埋め込みテーブルだけで 他のどの単一レイヤーよりも大きいです。

ステージ3 · + 位置エンコーディング

自己アテンションは置換等変です — 位置情報なしでは「the cat sat」と 「sat cat the」は同じ出力を生成します。論文では$X$と同じ形状の固定サイン波パターンを加算します。

$\mathrm{PE}_{(pos, 2i)} = \sin\!\big(pos / 10000^{2i/d_{model}}\big)$
$\mathrm{PE}_{(pos, 2i+1)} = \cos\!\big(pos / 10000^{2i/d_{model}}\big)$
[2, 8, 512] + [8, 512] → [2, 8, 512] 
# PEの形状は [T, d_model]、B次元にブロードキャストされる
X = X + PE[:T]            # [B, T, d_model] + [T, d_model] -> [B, T, d_model]

入力と出力の形状は同じです。情報は次元ではなく値に入ります。

ステージ3終了： 形状 [B, T, d_model] のテンソルが得られました。これは単語の意味とその位置の両方を含んでいます。 以降のすべてのステージはこのレイアウトに対する変換です。

ステージ4〜11 · マルチヘッドアテンション、次元ごとの解説

この論文が存在する理由そのものです。各トークンがクエリ、キー、バリューになり、 類似度が計算され、スケーリング・softmax処理・バリューの混合が行われます。 また、ヘッドを分割するため1回、結合するため1回の計2回リシェイプします。

ステージ4 · Q、K、V 射影

3つの独立した線形マップ、それぞれ $\mathbb{R}^{d_{model}}\to\mathbb{R}^{d_{model}}$：

$Q = X W^Q,\quad K = X W^K,\quad V = X W^V$
X [2, 8, 512] Wᵠ [512, 512] → Q [2, 8, 512] 
# 3つの線形レイヤー、論文ではバイアスなしだが実装上はバイアスありでも可
Wq = nn.Linear(d_model, d_model)   # 512*512 = 262 144 params
Wk = nn.Linear(d_model, d_model)
Wv = nn.Linear(d_model, d_model)

Q = Wq(X)                              # [B, T, d_model]
K = Wk(X)                              # [B, T, d_model]
V = Wv(X)                              # [B, T, d_model]

形状 $[d_{model}, d_{model}]$ の3つの行列 — ここでのパラメーター合計： $3\cdot 512\cdot 512 = 786\,432$。いくつかの実装ではこの3つを単一の $[d_{model}, 3\cdot d_{model}]$ 行列に融合します。

ステージ5 · h ヘッド用にリシェイプ & 転置

最後の次元を $d_k = d_{model}/h$ の $h$ チャンクに分割し、ヘッドが外側の次元になるよう転置します。 新しいパラメーターなし — view + permute のみです。

[2, 8, 512] view→ [2, 8, 8, 64] transpose(1,2)→ [2, 8, 8, 64] 
B, T, d_model = Q.shape
h, d_k = 8, d_model // h         # 8, 64

# 最後の次元を (h, d_k) に分割し、ヘッドをバッチの隣に移動
Q = Q.view(B, T, h, d_k).transpose(1, 2)   # [B, h, T, d_k]
K = K.view(B, T, h, d_k).transpose(1, 2)   # [B, h, T, d_k]
V = V.view(B, T, h, d_k).transpose(1, 2)   # [B, h, T, d_k]
ヘッドが重要な理由： $h=8$ の独立した $[T, d_k=64]$ 部分空間で、各 ヘッドが異なる関係パターンに特化できます。計算量は変わりません：1つの$512$次元アテンションの代わりに8つの小さな$64$次元アテンション。
ステージ6 · Q @ Kᵀ — ペアワイズ類似度

各ヘッドで、すべてのクエリがすべてのキーにアテンションします。行列積が $d_k$ 軸を縮約し、 バッチ・ヘッドごとに $T\times T$ のスコア行列を生成します。

$\text{scores} = Q K^\top$
Q [2, 8, 8, 64] Kᵀ [2, 8, 64, 8] → scores [2, 8, 8, 8] 
scores = Q @ K.transpose(-2, -1)   # [B, h, T, T]
# 要素 (b, head, i, j) = トークン i がトークン j にアテンションする度合い（そのヘッドで）
ステージ7 · 1/√d_k でスケール

スケーリングなしでは、$d_k$次元ベクトルの内積が$\sqrt{d_k}$の割合で増大し — 大きな値が softmaxを勾配が消える飽和領域に押し込みます。すべてのスコアを $\sqrt{d_k}$で割ります。

$\text{scaled} = \text{scores} / \sqrt{d_k} = \text{scores} / 8$
[2, 8, 8, 8] ÷ √64 [2, 8, 8, 8] 形状は変わらず 
import math
scores = scores / math.sqrt(d_k)   # 1/√64 = 1/8 = 0.125
ステージ8 · 最終軸に沿って softmax

クエリごとに生スコアを確率分布に変換します。$[T,T]$ブロックの各行の合計は1になります — これらがアテンション重みです。

$A = \mathrm{softmax}(\text{scaled},\ \text{dim}=-1)$
[2, 8, 8, 8] → [2, 8, 8, 8] 各行の合計=1 
import torch.nn.functional as F
A = F.softmax(scores, dim=-1)    # [B, h, T, T]

デコーダー自己アテンションでは、因果性を強制するために加法的な $-\infty$ マスクが softmaxの前に適用されます。

ステージ9 · attention @ V

アテンション重みを使ってバリューベクトルを混合します。各クエリの出力はすべての $T$ 個のバリューベクトルの 加重和です。縮約軸は内側の $T$ です。

A [2, 8, 8, 8] V [2, 8, 8, 64] → out [2, 8, 8, 64] 
head_out = A @ V                  # [B, h, T, d_k]
ステージ10 · ヘッドを結合（転置 + リシェイプ）

ヘッドを1つの大きな最終次元に結合します。transpose(1,2)でヘッドを $d_k$の隣に移動し、reshapeで結合します。パラメーターは不要。

[2, 8, 8, 64] transpose→ [2, 8, 8, 64] reshape→ [2, 8, 512] 
out = head_out.transpose(1, 2).contiguous()   # [B, T, h, d_k]
out = out.reshape(B, T, d_model)              # [B, T, d_model]
ステージ11 · 出力射影 W_O

もう1つの $\mathbb{R}^{d_{model}\to d_{model}}$ 線形マップ。形状は同じですが、結合後に ヘッド間の情報をモデルが混合できるようにします。

$\text{MHA}(X) = \mathrm{Concat}(\text{head}_1, \dots, \text{head}_h)\,W^O$
[2, 8, 512] ·W_O [2, 8, 512] 
Wo = nn.Linear(d_model, d_model)   # さらに262 144パラメーター
mha_out = Wo(out)                       # [B, T, d_model]

MHA パラメーター合計： $4 \cdot d_{model}^2 = 4\cdot 512^2 = 1{,}048{,}576$ (Q, K, V, O — それぞれ $d_{model}\times d_{model}$)。

アテンション終了： 形状 [B, T, d_model] で終わりました — 入力と全く同じ形状です。アテンションは形状保持演算子です。

ステージ12〜16 · LayerNorm、FFN、残差ハイウェイ

アテンションの後、エンコーダーブロックは2つの残差接続、2つのLayerNorm、 そして1つの位置ごとFFNで全てをつなぎます。FFNはパラメーターの大部分が存在する場所であり — 多くの研究者が事実的知識が保存される場所と考えています。

ステージ12 · 残差 + LayerNorm

元の $X$ を加算し（スキップ接続）、トークンごとに $d_{model}$ 全体で正規化します。

$X' = \mathrm{LayerNorm}(X + \mathrm{MHA}(X))$
[2, 8, 512] + [2, 8, 512] → LN → [2, 8, 512] 
ln1 = nn.LayerNorm(d_model)      # 2 * d_model = 1024 params (γ, β)
X_post = ln1(X + mha_out)         # [B, T, d_model]

LNは最終軸で正規化します：各 $[d_{model}]$ ベクトルが平均0・分散1になり、 学習済みの $\gamma, \beta\in\mathbb{R}^{d_{model}}$ で再スケールされます。バッチとトークン位置は両方独立を保ちます。

ステージ13〜14 · FFN 線形1 + ReLU

位置ごとの Feed-Forward Network：同じMLPがすべてのトークンに独立して適用されます。 最初の線形が $d_{model} \to d_{ff}$（通常 $4\cdot d_{model}$）に拡張します。

$H = \mathrm{ReLU}(X' W_1 + b_1)$
[2, 8, 512] ·W₁ [2, 8, 2048] → ReLU → [2, 8, 2048] 
W1 = nn.Linear(d_model, d_ff)       # 512 * 2048 = 1 048 576 params
H = F.relu(W1(X_post))            # [B, T, d_ff]
ステージ15 · FFN 線形2

2番目の線形が戻す：$d_{ff} \to d_{model}$。次のブロックが新鮮にスタートできるよう 正規の形状に復元します。

$Y = H W_2 + b_2$
[2, 8, 2048] ·W₂ [2, 8, 512] 
W2 = nn.Linear(d_ff, d_model)       # 2048 * 512 = さらに1 048 576 params
ffn_out = W2(H)                       # [B, T, d_model]

FFN パラメーター合計： $2\cdot d_{model}\cdot d_{ff} = 2\cdot 512\cdot 2048 = 2{,}097{,}152$ — 4つのアテンション射影の合計のほぼ2倍。大規模モデルが重量を置く場所です。

ステージ16 · 残差 + LayerNorm（再び）

ステージ12と同じパターン。FFN前のテンソルを加算し、レイヤー正規化を行い、 エンコーダーブロックが完了します。

$Y' = \mathrm{LayerNorm}(X' + Y)$
[2, 8, 512] → [2, 8, 512] ブロック出力形状 = ブロック入力形状 
ln2 = nn.LayerNorm(d_model)
out = ln2(X_post + ffn_out)       # [B, T, d_model]
スタック & 繰り返し： 論文は $N=6$ の同一エンコーダーブロックを積み重ねます。各ブロックの 入力形状は出力形状と等しいため、きれいに合成できます。 エンコーダーパラメーター合計（1ブロック、ベース）：おおよそ $4\cdot 512^2 + 2\cdot 512\cdot 2048 + 4\cdot 512 \approx 3.15$M。

エンコーダーブロック全体を14行で

EncoderBlock — 最小限の PyTorch
class EncoderBlock(nn.Module):
    def __init__(self, d_model=512, h=8, d_ff=2048):
        super().__init__()
        self.attn = nn.MultiheadAttention(d_model, h, batch_first=True)
        self.ffn  = nn.Sequential(
            nn.Linear(d_model, d_ff), nn.ReLU(), nn.Linear(d_ff, d_model)
        )
        self.ln1 = nn.LayerNorm(d_model)
        self.ln2 = nn.LayerNorm(d_model)

    def forward(self, x):
        # x: [B, T, d_model]
        a, _ = self.attn(x, x, x)
        x = self.ln1(x + a)
        x = self.ln2(x + self.ffn(x))
        return x                       # [B, T, d_model]

ステージ17〜21 · デコーダーブロック

デコーダーはターゲットトークン（右に1つシフト）を受け取り、(a) これまでに見たターゲットトークン — マスク済み自己アテンション経由 — と、(b) ソース — クロスアテンション経由 （QはデコーダーからKとVはエンコーダー出力（“memory”）から）の両方を知るテンソルを生成します。 埋め込み次元とエンコーダーと同じ残差+LN構造。レイヤーごとに3つのサブブロック、論文では6レイヤー。

論文設定（変わらず）
B = 2, T_src = 8, T_tgt = 7
d_model = 512, h = 8, d_k = 64
# T_tgt は T_src から独立しています。
# クロスアテンションだけが両者の出会いの場。

# target  : "die katze sitzt"           ← 正解

ステージ17 · デコーダー入力 → 埋め込み

[B, T_tgt] → embed lookup → [B, T_tgt, d_model] = [2, 7, 512]

ステージ19 · マスク済み自己アテンション （因果マスク）

エンコーダー自己アテンションと同一 — ただしスコア行列の対角線より上のすべての位置に −∞が押印されるため、位置 i は i+1, i+2, ...を絶対に見ることができません。softmax後にそれらの位置は正確に0になります。

# scores: [B, h, T_tgt, T_tgt] = [2, 8, 7, 7]
mask = torch.triu(torch.ones(T_tgt, T_tgt), diagonal=1).bool()
scores = scores.masked_fill(mask, float('-inf'))
A = F.softmax(scores, dim=-1)        # 下三角、各行の合計=1
[B, h, T_tgt, T_tgt] = [2, 8, 7, 7] softmax後、上三角 = 0

ステージ20 · クロスアテンション （Q ← デコーダー、K、V ← エンコーダー）

ここがエンコーダーが最終的にデコーダーと対話する場所です。デコーダーがクエリを 出力し、エンコーダー出力がキーとバリューを供給します。

# dec_x: [B, T_tgt, d_model] = [2, 7, 512]   ← マスク済み自己アテンションから
# memory: [B, T_src, d_model] = [2, 8, 512]  ← エンコーダー出力（ステージ16）
Q = W_q(dec_x).view(B, T_tgt, h, d_k).transpose(1, 2)   # [B, h, T_tgt, d_k]
K = W_k(memory).view(B, T_src, h, d_k).transpose(1, 2)   # [B, h, T_src, d_k]
V = W_v(memory).view(B, T_src, h, d_k).transpose(1, 2)   # [B, h, T_src, d_k]
scores = Q @ K.transpose(-2, -1) / math.sqrt(d_k)         # [B, h, T_tgt, T_src] = [2, 8, 7, 8]
A = F.softmax(scores, dim=-1)                              # 最終軸はT_src
out = (A @ V).transpose(1, 2).reshape(B, T_tgt, d_model) # [2, 7, 512]

重要な点：スコア行列は正方行列ではありません。 [B, h, T_tgt, T_src] — すべてのターゲットトークンがすべてのソーストークンにアテンションします。

デコーダーブロック全体を16行で

DecoderBlock — 最小限の PyTorch
class DecoderBlock(nn.Module):
    def __init__(self, d=512, h=8, d_ff=2048):
        super().__init__()
        self.self_attn  = nn.MultiheadAttention(d, h, batch_first=True)
        self.cross_attn = nn.MultiheadAttention(d, h, batch_first=True)
        self.ffn = nn.Sequential(nn.Linear(d, d_ff), nn.ReLU(), nn.Linear(d_ff, d))
        self.ln1, self.ln2, self.ln3 = nn.LayerNorm(d), nn.LayerNorm(d), nn.LayerNorm(d)

    def forward(self, x, memory, causal_mask):
        # x:      [B, T_tgt, d]      memory: [B, T_src, d]
        a, _ = self.self_attn(x, x, x, attn_mask=causal_mask)
        x = self.ln1(x + a)                 # [B, T_tgt, d]
        c, _ = self.cross_attn(x, memory, memory)
        x = self.ln2(x + c)                 # [B, T_tgt, d]
        x = self.ln3(x + self.ffn(x))
        return x                            # [B, T_tgt, d]

デコーダースタックの出力（6つ後）: [B, T_tgt, d_model] = [2, 7, 512]。次のステップ：語彙に射影。

ステージ22〜23 · 隠れ状態から logits へ

単一の nn.Linear(d_model, V) が各トークンの隠れベクトルを語彙内のすべての単語のスコアに変換します。 最終軸に沿ったsoftmaxがそれらのスコアを確率に変換します。 これが出力ヘッド全体です。論文ではW_lmは入力埋め込み行列と共有（tied）されています — 同じパラメーター、転置で使用。

ステージ22 · 語彙への線形射影

[B, T_tgt, d_model] @ W_lm.T → [B, T_tgt, V] = [2, 7, 37000] 
# dec_out: [B, T_tgt, d_model]
# W_lm:    [V, d_model]   ← しばしば embed.weight と共有
logits = dec_out @ W_lm.T           # [B, T_tgt, V]
# または単純に:
logits = self.lm_head(dec_out)      # nn.Linear(d_model, V)

これらは生スコアです — 語彙トークンごと、位置ごと、サンプルごとに1つ。 まだ確率ではありません。損失がCrossEntropyLossなら softmax しないでください； 数値安定性のために内部で行います。

ステージ23 · Softmax → 確率 （実際に必要な場合のみ）

probs = F.softmax(logits, dim=-1)   # [B, T_tgt, V] — V方向の各行の合計=1

B·T_tgtの各位置で語彙全体の分布が得られます。その37,000個の数値のほとんどは極小です。 一握りが正解の近くに集中します。

出力ヘッドのパラメーター数

W_lmだけでV × d_model = 37000 × 512 ≈ 19Mパラメーター — 通常、モデル内で最も大きな単一の重みです。論文ではこれを入力埋め込み行列と共有し、 重みメモリを半分にして（経験的に）汎化に役立てています。

Transformer.forward — 論文に忠実
def forward(self, src_ids, tgt_in_ids):
    # src_ids:    [B, T_src]   tgt_in_ids: [B, T_tgt]
    src = self.src_embed(src_ids) + self.pos(src_ids)        # [B, T_src, d]
    tgt = self.tgt_embed(tgt_in_ids) + self.pos(tgt_in_ids)    # [B, T_tgt, d]

    memory = src
    for layer in self.encoder_layers:
        memory = layer(memory)                                  # [B, T_src, d]

    causal = self.causal_mask(tgt_in_ids.shape[1])
    x = tgt
    for layer in self.decoder_layers:
        x = layer(x, memory, causal)                            # [B, T_tgt, d]

    logits = self.lm_head(x)                                     # [B, T_tgt, V]
    return logits

def train_step(model, optimizer, batch):
    # batch: list[(src_str, tgt_str)]、長さB
    src_ids = tokenize_src(batch)         # [B, T_src]    int64
    dec_in, labels = tokenize_tgt(batch)  # それぞれ [B, T_tgt]

    src = model.src_embed(src_ids)*math.sqrt(d) + model.pos(src_ids)
    tgt = model.tgt_embed(dec_in)*math.sqrt(d) + model.pos(dec_in)

    memory = src
    for layer in model.encoder_layers: memory = layer(memory)

    causal = model.causal_mask(dec_in.shape[1])
    x = tgt
    for layer in model.decoder_layers: x = layer(x, memory, causal)

    logits = model.lm_head(x)              # [B, T_tgt, V]
    loss = F.cross_entropy(
        logits.view(-1, V), labels.view(-1),
        ignore_index=-100, label_smoothing=0.1)

    optimizer.zero_grad(); loss.backward(); optimizer.step()
    return loss.item()                     # スカラー

@torch.no_grad()
def generate(model, src_text, max_len=32):
    src_ids = tokenize_src([(src_text, "")])      # [1, T_src]

    src = model.src_embed(src_ids)*math.sqrt(d) + model.pos(src_ids)
    memory = src
    for layer in model.encoder_layers:
        memory = layer(memory)                          # [1, T_src, d]

    dec_in = torch.tensor([[SOS]])                         # [1, 1]
    for _ in range(max_len):
        tgt = model.tgt_embed(dec_in)*math.sqrt(d) + model.pos(dec_in)
        causal = model.causal_mask(dec_in.shape[1])
        x = tgt
        for layer in model.decoder_layers:
            x = layer(x, memory, causal)                 # [1, t, d]
        logits = model.lm_head(x)                          # [1, t, V]
        next_id = logits[:, -1, :].argmax(-1, keepdim=True)
        if next_id.item() == EOS: break
        dec_in = torch.cat([dec_in, next_id], dim=1)    # [1, t+1]

    return detokenize(dec_in[0, 1:].tolist())          # str

調整ノブ

B T d_model h d_k d_ff オレンジ強調 = 前ステージから次元が変化
一定のもの： 各エンコーダーブロックの入力と出力の形状は同一で、 $[B, T, d_{model}]$。これが$N$個のスタッキングを簡単にする理由です。
内部で変化するもの： テンソルはアテンションのために4Dに拡張され、 FFN内部では $4\cdot d_{model}$ になり、その後元に戻ります。

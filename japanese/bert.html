BERTとは何か

BERTはtransformer-encoder-onlyなモデルです。GPT(decoder-only、自己回帰)と異なり、すべてのトークンが各層で双方向的に他のすべてのトークンに注意を向けます。この双方向性が要点です。

  • 公開: 2018年10月、Devlin et al. (Google AI Language)
  • アーキテクチャ: transformerエンコーダブロックの積み重ね(マルチヘッド自己注意 + フィードフォワード)
  • 事前学習: BooksCorpus (8億語) + 英語Wikipedia (25億語) で自己教師あり学習
  • ファインチューニング: 小さなタスクヘッドを追加し、ラベル付きデータでエンドツーエンドに学習
核心的洞察: 数十億の文を読み、空所を埋める方法を学んだモデルは言語について多くを知っています。小さな分類器を上に乗せれば、下流タスクで本来のサイズ以上の力を発揮します。
02 /

事前学習タスク

マスク言語モデリング (MLM)

WordPieceトークンの15%をランダムにマスクします。モデルは元のトークンを予測。損失はマスク位置のみのクロスエントロピー。

その15%の内訳:

  • 80%を[MASK]に置換
  • 10%をランダムなトークンに置換
  • 10%はそのまま

この10/10トリックにより、モデルがマスクのないファインチューニング時に[MASK]位置でしか強い出力を出さない怠慢を防ぎます。

なぜ双方向? 一方向の言語モデルはリークなしに未来のトークンを条件付けできません。MLMは各位置が双方向の完全な文脈を見られるようにします。

次文予測 (NSP)

2つの文AとBが与えられ、Bが実際にAの後に続くか(50%)、それとも別の場所からのランダム文か(50%)を予測。[CLS]トークンの最終隠れ状態に対する2値分類。

入力:   [CLS] A₁ A₂ ... [SEP] B₁ B₂ ... [SEP]
ラベル: IsNext / NotNext
辛口意見: RoBERTa (2019)はNSPがほぼ無用だと示し、これを排除しました。SBERTは文ペア対照学習目的に置き換えました。現代のBERT系モデルはしばしばNSPを完全に省きます。

合成損失

BERTの事前学習損失はMLM損失とNSP損失の和です。

$$\mathcal{L} = \mathcal{L}_{\text{MLM}} + \mathcal{L}_{\text{NSP}}$$

$$\mathcal{L}_{\text{MLM}} = -\sum_{i \in M} \log P(x_i \mid \tilde{x})$$

$M$はマスクされた位置の集合、$\tilde{x}$は破壊された入力です。

03 /

入力フォーマット

各入力位置のベクトルは3つの埋め込みの和です: トークン + セグメント + 位置。3つすべて同じ隠れサイズ$d$ (例: 768)の学習されたテーブルです。

$$\mathbf{x}_i = \mathbf{E}_{\text{tok}}(t_i) + \mathbf{E}_{\text{seg}}(s_i) + \mathbf{E}_{\text{pos}}(i)$$

ビジュアライザー

セルにホバーしてください。各色のバーは1つの埋め込みタイプです。下段は要素ごとの和 — これが最初のエンコーダブロックに入ります。

(値は説明用 — 実際の埋め込みは768次元で学習されます。)

1つのエンコーダブロックの中身

L個の各ブロックは標準的な「post-LN」transformerエンコーダです。

  1. ヘッドあたりH/A次元のマルチヘッド自己注意
  2. 残差接続 + LayerNorm
  3. 位置ごとのフィードフォワード: linear → GELU → linear (隠れサイズ4H)
  4. 残差接続 + LayerNorm

$$\text{Attn}(Q,K,V) = \text{softmax}\!\left(\tfrac{QK^\top}{\sqrt{d_k}}\right)V$$

自己注意は完全な双方向 — 因果マスクなし。

05 /

WordPieceトークン化

BERTは30,522トークンのWordPiece語彙を使います。未知の単語は既知のサブピースに分割され、継続ピースには##がプレフィックスされます。

"playing"     → ["play", "##ing"]
"unaffable"   → ["un", "##aff", "##able"]
"transformer" → ["transform", "##er"]
"BERT"        → ["bert"]   (bert-base-uncasedでは小文字化)

試してみよう

teal = 特殊トークン、 yellow = 完全な単語、 pink = ##継続

06 /

ファインチューニングのパラダイム

1つの事前学習済みBERTから3つ(以上)の下流タスクパターン。各ケースで小さなタスク固有のヘッドを追加し、低い学習率(通常2e-5から5e-5)で2〜4エポックエンドツーエンドに学習します。

シーケンス分類

感情分析、NLI、トピック。最終[CLS]の隠れ状態を使用。

[CLS] → Linear(H→C) → softmax

例: SST-2、MNLI、QNLI

トークン分類

NER、POSタグ付け。各トークンの隠れ状態にヘッドを適用。

h_i → Linear(H→C) → softmax (トークンごと)

例: CoNLL-2003 NER

スパン抽出 (QA)

回答スパンの開始・終了位置を予測。

h_i → 2 logits (start, end)

例: SQuAD 1.1 / 2.0

07 /

双方向 vs 自己回帰

BERT (encoder-only)

"my dog is __" のcuteを完全な文脈で予測:

my dog is [MASK] today .

両側に注意を向ける。自己回帰的にテキスト生成はできないが、強い文脈表現を生む。

GPT (decoder-only)

"my dog is" の次トークンを予測:

my dog is ? today .

因果マスクが未来のトークンをブロック。生成には自然だが、同じパラメータ数では理解タスクで劣る(歴史的に — その差は縮まっています)。

限界と後継モデル

BERTの弱点

  • シーケンス長に対する2乗の注意計算 → 最大512トークン
  • 生成不可 — エンコーダのみ、サンプリングできない
  • 事前学習とファインチューニングのミスマッチ — [MASK]は推論時には現れない
  • NSPは弱いシグナル — 後のモデルは削除
  • 文埋め込みが標準では悪い; 平均プールされた[CLS]はパフォーマンス不足

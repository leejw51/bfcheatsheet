連鎖律 — 勾配はどう逆向きに流れるのか

バックプロパゲーションは、1 つのルールを何度も適用するだけです。損失の 何か に対する勾配を 求めるには、損失からそこへ戻る経路をたどり、途中で 局所微分を掛け合わせる だけ。その積こそが 連鎖律 です。 このページの残りは、その 1 つの考え方を 6 つの角度から見たものにすぎません。

一文で言うと
小さなネットワーク $L \leftarrow z \leftarrow w$ では、勾配は局所的な傾きの積です:
$\dfrac{\partial L}{\partial w} = \dfrac{\partial L}{\partial z}\cdot\dfrac{\partial z}{\partial w}$
真ん中の因子 $\partial L/\partial z$ が、私たちが dz と呼ぶ 上流からの勾配 です。
dz が要
ノードの両方の入力は 同じ 上流勾配 dz を共有します。$z = x\cdot w$ の場合:
$\textcolor{#4ade80}{dw} = dz\cdot x \qquad \textcolor{#38bdf8}{dx} = dz\cdot w$
dz は共通で、異なるのは 局所的な 因子だけです。
なぜ分岐するのか
掛け算には入力が 2 つ あるので、勾配は 2 つ の枝に分岐します — そして各枝は もう一方 の入力を受け取ります。$x$ の枝は $w$ を運び、 $w$ の枝は $x$ を運びます。ノードは入力ごとに 1 つの勾配を送り出します。
行列でも同じ
$Z = X\!\mathbin{@}\!W$ でも、変わるのは形だけです — 「もう一方の入力」がその 転置 になります:
$\textcolor{#4ade80}{dW} = X^{\!\top}\!\mathbin{@}\,dZ \qquad \textcolor{#38bdf8}{dX} = dZ\mathbin{@}\,W^{\!\top}$
ゲートは 2 種類しかありません。 バックプロパゲーションがやることは 足す ことと 掛ける ことだけ — 掛け算 は勾配をもう一方の入力で スケールし、足し算 はそれをそのままコピーして通すだけです。タブ 4 がその足し算ゲートで、 どの層 (matmul、バイアス、残差) もこの 2 つから組み立てられています。

連鎖律が 積 である理由

$w$ は損失に直接触れることはなく、$z$ を 通して 作用するだけです: $w \to z \to L$。だから $w$ を少し動かすと、その揺れが連鎖を伝わっていきます。各リンクには 「増幅率」があり、全体の効果はそれらの率を 掛け合わせた ものになります。 その積こそが、まさに $\partial L/\partial w$ です。

分数として読む

証明ではありませんが、最強の語呂合わせです: 連鎖律を分数として書くと、真ん中の項が約分されます。

∂Loss / ∂w  =  ∂Loss / ∂z  ×  ∂z / ∂w
真ん中の ∂z が上と下で出会って 約分され、 ∂Loss/∂w が残ります。言葉で言えば「w あたりの損失変化 = (z あたりの損失変化) × (w あたりの z 変化)」。 橋となる $z$ を 通り抜ける のです。

分岐 — 入る勾配は 1 つ、出る勾配は 2 つ

掛け算ノード $z = x\cdot w$ には入力が 2 つあるので、1 つの上流勾配 dz は 2 つの枝にコピーされます。ひねりはこうです: 各枝には もう一方 の入力が掛けられます。$x$ の枝は $w$ を運び、 $w$ の枝は $x$ を運びます。

各枝になぜ独自の勾配が必要なのか

dw → 重みを更新する
$w$ は 学習可能なパラメータ です。その勾配が更新 w ← w − lr·dw を駆動します。dw がなければ、その層は学習できません。
dx → 下の層へ渡す
$x$ は 前の層の出力 です。その層にも勾配が必要なので、 dx を下流へ渡します。それが次のノードの dz になり — 連鎖は ずっと下まで続きます。
パターン: 掛け算では、dz をすべての入力にコピーし、 各コピーに もう一方 の入力の積を掛けます。$x$ が大きいほど dw も大きくなり、その逆も同様です。この対称性こそがすべてのトリックです。

足し算ゲート — 勾配はただ コピー されるだけ

掛け算ゲートは各枝を もう一方 の入力でスケールしていました。足し算 ゲート $z = x + w$ はもっと単純です: 局所微分は両方とも 1 なので、上流勾配 dz は そのまま 通り抜けます — 同じ値が すべての入力にコピーされるのです。スケールもなく、順伝播から何もキャッシュしません。

∂z/∂x = 1,   ∂z/∂w = 1   ⇒   dx = dz·1 = dz,   dw = dz·1 = dz

足し算 vs 掛け算 — ゲートはこの 2 つだけ

足し算 → コピー
$z = x + w$。局所的な傾きは両方とも 1 なので、 dx = dw = dz。勾配は 変わらずブロードキャスト されて すべての入力に届きます — スケールなし、転置なし、順伝播から覚えておくものは何もありません。
掛け算 → スケール
$z = x \cdot w$。各枝は もう一方 の入力でスケールされます: dx = dz·w、 dw = dz·x。出てくるものは、順伝播でキャッシュした値に依存します。
これで道具箱はすべてそろいました。 バックプロパゲーションがやることは 足す ことと 掛ける ことだけ。どの層もこの 2 つのゲートから組み立てられています: 線形層 Z = X@W + b は 掛け算 (matmul) のあとに 足し算 (バイアス) であり、 残差 接続 y = x + f(x) は純粋な足し算です。

なぜ残差接続はこれほどよく学習するのか

スキップ接続 y = x + f(x) は足し算ゲートなので、逆伝播では勾配が両方の枝へコピーされます: dz の完全なコピーが f に入り、同一 の完全な コピーがそのまま x に届きます。その 2 つ目のコピーが 勾配のハイウェイ です — f を通る勾配がゼロ近くまで縮んでしまっても、深い層まで減衰せずに届きます。

x は y = x + f(x) の中に 2 回 現れる (生のまま 1 回、f の 中で 1 回) ので、その 2 つの勾配コピーは 足し合わされます: dx = dz + (dz routed back through f)。最初の項がハイウェイ — f が何をしようとも生き残る、上流勾配のきれいでスケールされていないコピーです。 これこそが、深いトランスフォーマーが学習可能であり続ける理由のすべてです。

行列 — 転置は形を合わせるだけ

線形層は $Z = X\!\mathbin{@}\!W$ です。逆伝播はスカラーゲートと まったく同じ 2 つの枝で、 新しいのは「もう一方の入力を掛ける」が「もう一方の行列を 転置して 掛ける」に なることだけ。こうして形がそろいます。

dW = Xᵀ @ dZ
(E,L) = (E, N) @ (N, L)
重みの勾配。 要素ごとに dW[e,l] = Σ_tokens x_e · dz_l — 入力と出力勾配の 外積を すべてのトークンにわたって合計 したもの (共有された $W$ は 1 つ)。
dX = dZ @ Wᵀ
(N,E) = (N, L) @ (L, E)
入力の勾配。 要素ごとに dx_e = Σ_l dz_l · W[e,l] — 出力勾配を重みを通して押し戻します。 トークンごとに保持されるので、下の層へと流れます。

順伝播と逆伝播は転置のペア

Z = X @ W   ⇄   dW = Xᵀ @ dZ   ·   dX = dZ @ Wᵀ
注意: 順伝播してから逆伝播する (例 (dZ @ Wᵀ) @ W) と、 Z の形 を持つものが返ってきます — 形 の往復です — が、 Wᵀ·W は単位行列ではないので、同じ 値 にはなりません。 転置のペアが合わせるのは形であって、数値ではありません。

dz はどこから来るのか: softmax + 交差エントロピー = P − Y

このページのすべての連鎖は、上流勾配 dz から始まりました。 分類器の場合、その一番最初の勾配 — ロジットに対する損失 — は、ディープラーニングで 最もきれいな式に集約されます: P − Y。

P − Y の背後にある直感

ターゲットクラス (Y = 1)
勾配 = p − 1 → 常に負。勾配降下は勾配の 逆 方向に進むので、 このロジットは 上がり ます (確率 ↑)。正しい方向です。
他のクラス (Y = 0)
勾配 = p − 0 = p → 常に正。このロジットは 下がり ます (確率 ↓)。これも正しい方向です。
勾配の合計はゼロになります: Σ(pᵢ − yᵢ) = Σpᵢ − Σyᵢ = 1 − 1 = 0。 ターゲットから取られた確率が、他のクラスに正確に配分されます — どの更新も常に釣り合いが取れています。

導出を 1 行ずつ

必要な事実は 2 つだけ: $Y$ がワンホットであること、そして softmax の微分です。

# 定義
pᵢ = e^zᵢ / Σₖ e^zₖ ← softmax
L = − Σᵢ yᵢ · log(pᵢ) ← 交差エントロピー
 
# softmax の微分 (重要な補題)
∂pᵢ/∂zₖ = pᵢ·(δᵢₖ − pₖ) δᵢₖ = 1 (i=k のとき) それ以外は 0
 
# ∂L/∂zₖ の連鎖律
∂L/∂zₖ = − Σᵢ yᵢ · (1/pᵢ) · ∂pᵢ/∂zₖ
      = − Σᵢ yᵢ · (1/pᵢ) · pᵢ·(δᵢₖ − pₖ)
      = − Σᵢ yᵢ · (δᵢₖ − pₖ)
      = − yₖ + pₖ · Σᵢ yᵢ Σyᵢ = 1 (ワンホット)
∂L/∂zₖ = pₖ − yₖ ∎ ベクトルとして: P − Y
なぜこんなに美しいのか: softmax の 1/pᵢ と交差エントロピーの pᵢ がちょうど約分されます。すべての exp / log が消え、 「予測 − ターゲット」だけが残ります。そのきれいな dz が、 タブ 2〜5 のすべての連鎖の出発点です。

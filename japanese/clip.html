CLIP — 画像とテキストを同じ部屋に

CLIP (Contrastive Language–Image Pre-training, OpenAI 2021) はマルチモーダル ゴールドラッシュの口火を切ったモデルです。画像エンコーダーとテキストエンコーダーを訓練し、 マッチする画像・キャプションのペアを共有埋め込み空間内の同じ点に — 関係ないペアは 遠くに配置します。一度学習すれば、クラス名を文として書くだけで ゼロショット 分類ができるようになります。

2 つのエンコーダー、1 つの空間
画像エンコーダー (ViT-B/32 または ResNet-50) とテキストエンコーダー (Transformer) がそれぞれベクトルを出力。両側の小さな 射影ヘッド (projection head) が どちらも同じ d=512 の埋め込み空間にマップし、 コサイン類似度を測ります。
対比損失の目的
N 個の画像・テキストペアのバッチに対し、 N×N の類似度行列を作ります。対角セルが本物の マッチ (正例)、それ以外は負例。対角を上げ、対角外を下げる — 行 と 列の 両方で対称なクロスエントロピーをとります。
なぜ重要か
学習データ: ウェブから集めた 4 億組の画像・テキストペア。人手のラベルなし — キャプションが ラベル代わり。結果、明示的に教えていないものを、文と照合するだけで認識するモデルが完成。
CLIP が動かしているもの
DALL·E 2 の CLIP ガイダンス、Stable Diffusion のテキストエンコーダー (CLIP-L/14)、 ゼロショット分類器、画像検索、コンテンツモデレーション、検索、image-to-image、 text-to-image スコアリング、OpenCLIP、SigLIP、EVA-CLIP… すべてその子孫。

I_f = image_encoder(I)            # [N, d_i]
T_f = text_encoder(T)             # [N, d_t]

I_e = l2_normalize(W_i @ I_f)     # [N, d]  射影 + 正規化
T_e = l2_normalize(W_t @ T_f)     # [N, d]

labels = arange(N)                # 対角がターゲット
loss   = (CE(logits, labels) + CE(logits.T, labels)) / 2

アーキテクチャ — 2 つの塔、1 つの部屋

CLIP は デュアルエンコーダー。2 つの独立した塔がそれぞれ入力を固定長ベクトルに変換、 小さな線形射影で同じ次元に揃え、L2 正規化、その後内積 = コサイン類似度。

4 両方の射影後、バッチ内のペアごとの類似度 行列 を計算します。画像特徴 [N, d]、テキスト特徴 [N, d] — 正しいテンソル演算は:

対比損失 — 対角を引き上げる

N 個の画像・キャプションペアのバッチ内で N×N 類似度行列を作ります。対角だけが正しいマッチで、 それ以外は バッチ内負例。損失は対称クロスエントロピー: 行は各画像に対して正しい キャプションを選び、列は各キャプションに対して正しい画像を選びます。

L = ½ · ( CE(logits, labels) + CE(logitsᵀ, labels) ), labels = [0, 1, 2, …, N−1]

4×4 バッチで遊ぶ — セルをドラッグして損失を見る

各セルは画像 i (行) とテキスト j (列) の間の logit (類似度 × 温度)。 セルをクリックして ↑ / ↓ で動かしてみよう。対角セルは青で囲まれています — そこに重みを集中させたい。

温度 — softmax の鋭さ

生の類似度 I_e @ T_e.T は [-1, 1] の範囲 — softmax には平らすぎ。CLIP は 学習可能なスケール exp(t) を掛けます (t は 1 つのスカラーパラメータ、初期値 log(1/0.07) ≈ 2.66)。

P(text=j | image=i) = softmaxj( τ · cos(Ii, Tj) ), with τ = exp(t)

温度を動かして softmax がシャープになる様子を見る

τ の上限: CLIP は学習の不安定化を防ぐため、 t を log(100) ≈ 4.6 でクランプ。スライダーを右端まで動かすと、バーがほぼ one-hot に収束します。

self.logit_scale = nn.Parameter(torch.ones([]) * np.(1 / 0.07))

self.logit_scale.data.clamp_(0, math.log())

ゼロショット — 学習なしで分類する

コードドリル — 本物の CLIP を使う

実世界の 3 つのスタック。open_clip_torch が事実上の 標準 (オリジナルリリースよりも優れたモデル)。transformers は HuggingFace の CLIPModel を提供。両者とも 同じ数学をラップしています。

open_clip — 読み込み & 埋め込み

完全リファレンス
import torch, open_clip
from PIL import Image

model, _, preprocess = open_clip.create_model_and_transforms(
    "ViT-B-32", pretrained="laion2b_s34b_b79k")
tokenizer = open_clip.get_tokenizer("ViT-B-32")
model.eval()

img    = preprocess(Image.open("cat.jpg")).unsqueeze(0)
labels = ["a photo of a cat", "a photo of a dog"]
text   = tokenizer(labels)

with torch.no_grad():
    img_f = model.encode_image(img)
    txt_f = model.encode_text(text)
    img_f /= img_f.norm(dim=-1, keepdim=True)
    txt_f /= txt_f.norm(dim=-1, keepdim=True)
    probs = (100.0 * img_f @ txt_f.T).softmax(dim=-1)

print("probs", probs)   # [[0.98, 0.02]]

ボス — CLIP の forward を一発で書く

最終試験: このコードブロックの空欄はすべて CLIP の forward + 損失の一部。すべて緑になったら、 ページが 4×4 CLIP バッチの学習をシミュレートし、類似度行列が対角に向かってシャープに なる様子が見られます。

CLIP 学習ステップを完成させる

11 個の空欄。すべて緑になったら Train を押す。 

import torch
import torch.nn.functional as F

# --- 両エンコーダーの forward ----------------------------------
image_features = model.(images)        # [N, d]
text_features  = model.(texts)          # [N, d]

# --- L2 正規化で内積をコサイン類似度に -------------------------
image_features = image_features / image_features.(dim=-1, keepdim=True)
text_features  = text_features  / text_features.(dim=-1, keepdim=True)

# --- N × N 類似度 logit を構築 --------------------------------
logit_scale = model.logit_scale.()                 # τ = exp(t)
logits_per_image = logit_scale * image_features @ text_features.
logits_per_text  = logits_per_image.

# --- 対角ターゲットでの対称クロスエントロピー ------------------
N      = images.shape[0]
labels = torch.(N, device=images.device)
loss_i = F.cross_entropy(logits_per_image, labels)
loss_t = F.cross_entropy(logits_per_text,  labels)
loss   = (loss_i + ) / 2

# --- 標準的な PyTorch ステップ --------------------------------
optimizer.zero_grad()
loss.()
optimizer.()

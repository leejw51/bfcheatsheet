ComfyUI — ノードベースの Stable Diffusion

ComfyUI は Stable Diffusion 用のノードグラフエディタ。スライダーが固定された UI (Automatic1111 のような) ではなく、それぞれ 1 つの仕事をするボックスを配線します: チェックポイントを読み込む、プロンプトをエンコード、潜在ノイズを除去、画像にデコード。 グラフが そのままプログラム。このページの最後には実際の txt2img ワークフローを配線し、 KSampler を調整し、HTTP API を呼び出すまで終わっています。

Why nodes?
Every step of diffusion (load model, encode prompt, sample, decode) becomes a node. Wires carry typed data between them. You see the full pipeline at once — and you can rearrange it however you like.
Reproducible
ComfyUI embeds the entire workflow JSON in the PNG metadata of every output. Drop the image back onto the canvas — graph restored. Same seed → same pixels.
Swappable
Need ControlNet, IP-Adapter, AnimateDiff, a LoRA stack? Add a node, route a wire. The base graph stays the same. Custom nodes ship as folders in custom_nodes/.
vs Automatic1111
A1111 is a wizard: easy first run, hard to extend. ComfyUI is plumbing: learning curve up front, infinite flexibility after. Power users, batch runs, and any non-trivial pipeline live here.

The big picture (memorize this)

# the txt2img graph in plain English
Load Checkpoint  ——MODEL——>  KSampler
                 ——CLIP——>   CLIP Text Encode (positive)  ——COND——> KSampler
                 ——CLIP——>   CLIP Text Encode (negative)  ——COND——> KSampler

Empty Latent Image  —LATENT—> KSampler  —LATENT—> VAE Decode  —IMAGE—> Save Image

Anatomy of a Node

Every ComfyUI node is the same shape: a titled box with input slots on the left, output slots on the right, and widgets in the body (numbers, text, dropdowns). Wires plug into slots; widget values are stored on the node itself.

dissect a KSampler node

input slots
Left edge. Each slot has a type — only matching wires can plug in. Try to drop a CLIP wire into a MODEL slot and ComfyUI refuses.
output slots
Right edge. One output can feed many inputs (one-to-many). The same MODEL output of Load Checkpoint commonly goes to KSampler and a LoRA loader.
widgets
Body of the node. Numbers, sliders, dropdowns, text boxes. They store constant values on the node — no wire needed. Right-click any widget to convert it to an input slot if you want it driven by another node.
Tip: double-click empty canvas to open the node search palette. Right-click a slot to convert to widget or convert to input.

Custom Nodes — extend the graph

Stock ComfyUI ships ~50 nodes. The community has thousands. They install as folders under custom_nodes/ and register new node classes at server startup.

install path
# 1. install ComfyUI-Manager (the package manager for nodes)
cd ComfyUI/custom_nodes
git clone https://github.com/ltdrdata/ComfyUI-Manager

# 2. restart server. now use the Manager UI to install anything else.
python main.py

Must-have packs

ComfyUI-Manager
The package manager. Install / update / disable other custom nodes from the UI. Always the first thing to add.
ComfyUI ControlNet Aux
Preprocessors (Canny, Depth, OpenPose, Lineart) + the ControlNetApply node. Steer composition with reference images.
ComfyUI_IPAdapter_plus
Image-to-conditioning. Treat a reference photo like a prompt — style transfer, identity preservation.
ComfyUI-AnimateDiff-Evolved
Drop in a motion module → KSampler now produces video. Same graph, batch dimension = frames.
Impact Pack
Detection / segmentation / detailer combos. FaceDetailer is famously a face-fixer in one node.
rgthree-comfy
Quality-of-life nodes: muting groups, fast bypass, seed wrangling, progress bars on everything.

API Mode — drive ComfyUI from code

Every workflow can be saved as API JSON (gear → "Save (API Format)") and submitted by HTTP. The UI is then optional — you can run the graph from a script, a backend, a Discord bot, anything.

submit a job
import json, requests, uuid

# load the workflow JSON exported via gear -> Save (API Format)
with open("workflow_api.json") as f:
    workflow = json.load(f)

# patch the prompt for this run
workflow["6"]["inputs"]["text"] = "a corgi in a spacesuit"
workflow["3"]["inputs"]["seed"] = 12345

payload = {"prompt": workflow, "client_id": str(uuid.uuid4())}
r = requests.post("http://localhost:8188/prompt", json=payload)
prompt_id = r.json()["prompt_id"]
print("queued:", prompt_id)

Useful endpoints

methodpathwhat it does
POST/promptQueue a workflow for execution; returns prompt_id.
GET/history/<prompt_id>Fetch outputs (image filenames, node results) once done.
GET/queueList running + pending jobs.
GET/viewDownload an output image: ?filename=...&type=output.
WS/ws?clientId=...WebSocket for live progress + node-by-node previews.
POST/upload/imageUpload an image (for img2img / inpaint inputs).

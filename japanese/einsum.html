PyTorch · テンソル演算を統べる1つの記法

あらゆる縮約を1つの文字列で

torch.einsum を使えば、行列積、バッチ行列積、転置、トレース、内積、 外積、アテンションスコアまで、'ik,kj->ij' のような 1本の添字文字列で書けます。@ なのか bmm なのか matmul なのか、あるいは transpose + sum の絡まりなのかを覚える必要はありません。ただ 軸に名前を付け、出力に残す軸を宣言するだけです。このページでは、ルール、レシピ集、そして 小さなテンソルで実際に einsum を走らせるライブのプレイグラウンドを紹介します。

01 一行でわかる発想

これは何か

軸に名前を付ける

すべての入力のすべての軸に文字を割り当てます。'bik,bkj->bij' は「オペランド A の添字は b,i,k、オペランド B は b,k,j、結果は b,i,j を残す」という意味です。

唯一のルール

出力に無い = 和をとる

左辺に現れて出力には現れない文字は、すべて和をとって消えるのです。 ここでは k が bij に無いので k について和をとります。これが 行列積です。

$$\texttt{'ik,kj->ij'} \quad\Longleftrightarrow\quad C_{ij} = \sum_{k} A_{ik}\, B_{kj}$$
文章のように読む。 この添字は、まさにアインシュタインの総和記法そのものです。 左辺は各オペランドの添字を並べ、-> が出力を区切り、繰り返されつつ脱落した添字に対する総和は 暗黙に行われます。einsum はその数式を関数呼び出しに変換するだけです。

01·b なぜ使われるのか

読みやすい

自己文書化

'bhqd,bhkd->bhqk' はアテンションスコアが何をするかを正確に伝えます。軸がそのまま見えている からです。permute+matmul の積み重ねでは、それが隠れてしまいます。

汎用的

1つのAPIで多くの演算

転置、総和、トレース、内積、外積、行列積、バッチ処理、多テンソル縮約まで、すべてが同じ呼び出しに まとまります。

バグが減る

暗黙のブロードキャストなし

すべての軸を明示するので、形状の不一致は大きな声でエラーになります。意図しない偶発的なブロードキャストが 勝手なことをすることもありません。

言語の全体が3つのルールに

添字の読み方

-> の前の文字列が入力を、後ろの文字列が出力を表します。3つのルールですべてをカバーできます。

①

同じ文字 = 揃える

文字がオペランドをまたいで再利用されると、それらの軸が揃えられ、その次元に沿って要素ごとに 掛け合わされます。'ik,kj' では A の k と B の k が対になります。

②

脱落した文字 = 和をとる

出力に現れない入力側の文字は、和をとられます。'ik,kj->ij' は k を落とすので、k について和をとります。これが縮約です。

③

残した文字 = 軸

出力に現れる文字は、書いた順に結果の軸になります。出力の文字を並べ替えれば転置になります: 'ij->ji'。

02·a 自由添字 vs. 和をとる添字

このページ全体で使う、2つの役割と2つの色:

自由(free) — 出力に現れ、結果の軸になる 和(summed) — 左辺だけに現れ、縮約されて消える
$$\underbrace{C_{\textcolor{#fbbf24}{ij}}}_{\text{free}} = \sum_{\textcolor{#818cf8}{k}} A_{\textcolor{#fbbf24}{i}\textcolor{#818cf8}{k}}\, B_{\textcolor{#818cf8}{k}\textcolor{#fbbf24}{j}}$$

02·b 暗黙形式（矢印なし）

-> を省略したとき

->output を省くと、einsum は暗黙のルールを使います。左辺に ちょうど1回だけ現れる添字はすべて自由な出力軸となり、アルファベット順に並べられます。繰り返し現れる 添字は和をとられます。

  • 'ij'  →  'ij->ij'   恒等（各文字が1回ずつ）。
  • 'ii'  →  'ii->'   トレース（i が繰り返すので和をとる）。
  • 'ik,kj'  →  'ik,kj->ij'   行列積（k が繰り返す → 和をとる、i,j はソート済み）。
明示しよう。 暗黙形式は便利ですが、アルファベット順ソートのルールは人を驚かせがちです。 -> で出力を書けば意図が曖昧でなくなり、ほとんどのコードベースもそうしています。

02·c シグネチャと落とし穴

torch.einsum · 呼び出し 
import torch

A = torch.randn(2, 3)        # [i=2, k=3]
B = torch.randn(3, 4)        # [k=3, j=4]

C = torch.einsum('ik,kj->ij', A, B)   # [i=2, j=4]  == A @ B

# オペランドは文字列の後ろに、順番どおりに渡す
# 各オペランドの文字数は、その次元数と一致しなければならない
落とし穴

文字数 ≠ 次元数

オペランドが3次元なら、その添字はちょうど3文字必要です。3次元テンソルに 'ij' を使うと エラーになります。

落とし穴

同じ文字は同じサイズ

繰り返される文字は、現れるすべての場所で同じ長さでなければなりません。両方のオペランドの k が一致しないと、軸を揃えられません。

省略記号 ...: 名前を付けたくない先頭の「バッチ」次元には '...ik,...kj->...ij' を使います。... は任意の数の先頭軸にわたってブロードキャストするので、 バッチありとバッチなしの両方の入力で動くべきコードに最適です。

03·c アテンションブロックを einsum で

スケールド・ドットプロダクト・アテンション · 展開版 
# Q,K,V: [B, H, T, d]   (バッチ, ヘッド, トークン, ヘッド次元)
scores = torch.einsum('bhqd,bhkd->bhqk', Q, K) / math.sqrt(d)
attn   = scores.softmax(dim=-1)                    # [B,H,q,k]
out    = torch.einsum('bhqk,bhkd->bhqd', attn, V)  # [B,H,q,d]
scores の行を読む: q と k は2つのトークン軸、 d（ヘッド次元）は共有されつつ脱落するので和をとられます。これが各クエリと各キーの内積です。 b,h はバッチ軸として一緒に運ばれます。

05·a 添字を読む

01 行列 A[i,j] を転置する。出力の軸を入れ替えよう。
torch.einsum('ij->', A)
02 列 j について和をとり、行 i ごとに1値を残す（行ごとの和）。
torch.einsum('ij->', A)
03 行列積: A[i,k] と B[?,?] → C[i,j]。B の添字を埋めよう。
torch.einsum('ik,->ij', A, B)

05·b 総和のルール

04 ルールを完成させよう: 左辺にあって出力に無い添字は、______ をとられる（英語で）。
it is  over
05 正方行列のトレース（対角成分の和）。入力の添字を埋めよう。
torch.einsum('->', A)
06 対角成分をベクトルとして取り出す（i を残し、和はとらない）。
torch.einsum('ii->', A)
07 ベクトル u[i] と v[j] の外積 → 行列 [i,j]。
torch.einsum('i,j->', u, v)

05·c バッチ処理とアテンション

08 バッチ行列積: バッチ b ごとに A[b,i,k] · B[b,k,j]。出力を埋めよう。
torch.einsum('bik,bkj->', A, B)
09 アテンションスコア: ヘッド次元 d について Q[b,i,d] と K[b,j,d] の内積をとる。
torch.einsum('bid,bjd->', Q, K)
10 コンテキストベクトル: 値をアテンションで重み付けする。attn[b,i,j] · V[b,j,d] → [b,i,d]。
torch.einsum('bij,->bid', attn, V)
11 PyTorch の関数そのもの。呼び出す関数名を答えよう。
C = torch.('ik,kj->ij', A, B)
11問すべて解けましたか？ これで転置、リダクション、トレース、外積/内積、行列積、 バッチ行列積、そして完全なアテンションブロックまで、einsum を読み書きできます。この言語の全体はただこれだけです: 軸に名前を付け、出力に残す軸を並べ、それ以外はすべて和をとる。

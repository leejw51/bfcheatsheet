入力埋め込みが学ぶしくみ

トークナイザーはモデルに [464, 3797, 3332] のような整数リストを渡す。 入力埋め込みは各IDを、サイズ $d_{model}$ の学習済み密ベクトルへ変換する。 だがそれは手作業で設計されるものではない — ランダムノイズから始まり、他のすべてのパラメータと同様にバックプロパゲーションで学習される 、普通の重み行列だ。このガイドは The Annotated Transformer に沿って、 そのテーブルがどのように構築され、供給され、学習されるかを正確に示す。

01 一言で言えば

何であるか

学習可能なルックアップ

行列 $E \in \mathbb{R}^{V \times D}$ — 語彙エントリごとに1つの学習可能な行。トークンの埋め込みは、インデックスで取り出されたその行にすぎない。

どのように学ぶか

勾配降下法

行はランダム（Xavier初期化）から始まる。各学習ステップで、損失勾配はバッチに現れた行だけを少しずつ動かす。数百万回繰り返すと、ジオメトリが自然と整理される。

「学習済み埋め込みを使用する」 論文は明示している: 入力と出力の トークン埋め込みは固定ではなく — 重みだ。別途「埋め込み学習フェーズ」は存在しない; テーブルはアテンションとフィードフォワード層とともに、単一の 言語/翻訳損失のもとで共同最適化される。

01·b どこに位置するか

形状 · 埋め込みが特徴軸 D を追加する
トークンID [B, T] → 埋め込み × √D [B, T, D] + PE[:T] [T, D] → エンコーダ [B, T, D]

$B$ = バッチ、$T$ = シーケンス長、$D = d_{model}$（ベースモデルでは512）。トークナイザーが IDの [B, T] を作り; 埋め込みがIDごとに D 次元ベクトルを引き出してスケールし; その後 位置エンコーディングが順序を加える。

このページの使い方: タブを 01 → 07 と進む。タブ03でテーブルとルックアップを描画; タブ04が核心 — 順伝播 → 損失 → 逆伝播のサイクルと、ライブ「ステップを学習」サンドボックス; タブ05で現れる意味を示す; タブ07でスコアを付ける。 ショートカットは ? を押して。
Embeddings モジュール

トークンごとに1つの学習可能な行

Annotated Transformer では、埋め込みは nn.Embedding をラップする小さなモジュール。 順伝播は行のルックアップ、その後 $\sqrt{d_{model}}$ によるスケーリングだ。ここで学習するのは重み行列だけ。

Embeddings クラス · 原文のまま 
class Embeddings(nn.Module):
    def __init__(self, d_model, vocab):
        super(Embeddings, self).__init__()
        self.lut = nn.Embedding(vocab, d_model)   # 重み: [V, D]
        self.d_model = d_model
    def forward(self, x):
        return self.lut(x) * math.sqrt(self.d_model)
各シンボルの意味
  • vocab · V  語彙サイズ — 行の数。(nn.Embedding の第1引数。)
  • d_model · D  埋め込みの幅 — ベース Transformer では 512。
  • lut  学習可能な重み行列 [V, D] — 文字通り「ルックアップテーブル」。
  • x  形状 [B, T] の整数トークンID; 出力は [B, T, D]。
なぜ × √d_model？ 論文では「埋め込み層では、それらの重みを √d_model で乗じる」と述べている。 これにより埋め込みの大きさが、次に加算される単位スケールの位置エンコーディングと比較可能になる — そうしなければ位置情報が内容を圧倒してしまう。D = 512 の場合、その係数は √512 ≈ 22.6。
最も大きな層になることが多い。 パラメータ数はちょうど V × D。 D = 512 の32k語彙では ≈ 16.4M の重み — すべてこの一つのテーブルに、すべて勾配降下法で学習される。
ルックアップを見る

ルックアップラボ

学習が触れる前に、テーブルが何であるかを見てみよう。各行は1トークンのベクトル; 各列は1次元; 色が値を表す。トークンID をスライドすると — その行が光り、 下流に流れる D 次元ベクトルとして取り出される。ルックアップは純粋なインデックス操作: E[id] — 計算は不要。

核心 · 行がどのように学ぶか

テーブルの学習

埋め込みに特別な学習手順はない — モデルの通常のループに乗るだけだ。バッチが順伝播で損失まで流れ; loss.backward() がすべての重みに勾配を送り、ルックアップされた埋め込みの行も含めて; オプティマイザーがステップを踏む。重要なのは、バッチに現れた行だけが勾配を受ける — 更新はスパースだ。

04·b なぜ更新がスパースなのか

ルックアップはワンホット行列積 $e_{id} = \mathbf{1}_{id}^{\top} E$ と等しいため、 テーブルに関する勾配は行 id 以外はすべてゼロ。 ルックアップされた各トークンは、バッチ内での出現回数の合計として自分の行に勾配が蓄積される。

$$\frac{\partial L}{\partial E_{j}} \;=\; \sum_{t\,:\,ids_t = j} \frac{\partial L}{\partial e_{t}} \qquad\Longrightarrow\qquad \frac{\partial L}{\partial E_{j}} = 0 \;\text{ for unused rows } j$$
スパース更新サンドボックス · バッチの行だけが動く

このバッチは短い1文。そのトークン（行IDつき）だけが勾配を受ける行だ。 学習ステップを繰り返し押してみよう: ハイライトされた行は目に見えてドリフトし、 凍結された行は決して変化しない — スパース埋め込み勾配の実演だ。

−1+1 枠付き = バッチ内（∇を受ける） · 暗い = 凍結

04·c 学習を駆動する損失 & オプティマイザー

ラベル平滑化損失 · 逆伝播 · ステップ 
# 目標分布は平滑化: 正解トークンに確信度、残りに ε を分散
criterion = LabelSmoothing(size=V, padding_idx=0, smoothing=0.1)   # 内部で KLDivLoss

def step(out, targets, norm):
    log_probs = generator(out)                 # Linear → log_softmax  [.., V]
    loss = criterion(log_probs, targets) / norm
    loss.backward()                          # ∇ が E の使用行へ流れる
    opt.step()                                # Noam スケジュール Adam 更新
    opt.optimizer.zero_grad()                 # 次のバッチのため勾配をクリア
    return loss

学習が生み出すもの

意味がジオメトリになる

十分なステップの後、似た役割を持つトークンは似た行へ押し寄せられる — そうすることが予測の精度向上につながるからだ。コサイン類似度で2行を比べると、 モデルがそれらをどれほど似ていると考えているかが分かり、方向に意味が生まれる （有名な king − man + woman ≈ queen）。クエリトークンを選んで、その近傍が光るのを見てみよう。

学習済みトイ埋め込み空間 · クエリトークンへのコサイン類似度

各点はトークンのベクトル（2次元に投影）。選択されたクエリが輝き; 線が他のトークンへ伸び、 類似度が高いほど明るい。ランクリストはコサイン類似度の高い順だ。

一つの行列、三つの役割

共有重み

論文はさらに進む: 入力埋め込み、出力（ターゲット）埋め込み、そしてロジットを生成するソフトマックス前の線形変換が すべて同じ重み行列を共有する。一つを学習すれば三つすべてが学習される — パラメータが減り、入出力のジオメトリが一貫する。

原文のまま: 「2つの埋め込み層とソフトマックス前の線形変換の間で同じ重み行列を共有する」 ロジットはその後 h @ E.T — 文字通り「この隠れ状態は各トークンの行にどれだけ向いているか？」
出力側 · Generator 
class Generator(nn.Module):
    # "標準の線形 + ソフトマックス生成ステップを定義する。"
    def __init__(self, d_model, vocab):
        super(Generator, self).__init__()
        self.proj = nn.Linear(d_model, vocab)   # 重み: [V, D] — E と共有
    def forward(self, x):
        return F.log_softmax(self.proj(x), dim=-1)

マルチ­エージェント。

Claude Codeを並列で動かす4つの方法。
FILE: multi-agent-guide.html
RUNTIME: claude-code/cli
~/repo — claude --multi-agent

Claudeのウィンドウ1つで既に小さなチームです。4つのウィンドウはスタジオです。 このガイドはそのチームが取り得る4つの形 — そしてどの形がいつ 正解なのかを示します。

人数ではなく協調方法で選びます。サイドタスク → サブエージェント。 独立 → エージェントビュー。協調 → チーム。完全な分離 → ターミナルごとのワークツリー。

アプローチ 01 · セッション内委譲

サブエージェント。

独自のコンテキストウィンドウで動作する専門ワーカー。集中したタスクを 完了させ、要約を呼び出し元のセッションに返します。メインスレッドを きれいに保つために使います — 調査、検証、探索。

「ポイントは別のエージェントを増やすことではない。メインの 会話を散らかさないことが目的だ。」

.claude/agents/配下のMarkdownファイルとして定義します。 各エージェントは独自のシステムプロンプト、ツール許可リスト、(必要なら) より小さなモデルを持つので作業が安く済みます。

コンテキスト · 分離 並列 · はい ファイル衝突 · 共有CWD 適している · 調査、監査
.claude/agents/researcher.md 
---
name: Researcher
description: 先行研究を調査し、出典を引用する
model: haiku
tools:
  - bash
  - read
  - webfetch
---

# あなたは熟練したリサーチャーです。
# 引用を箇条書きで返してください。
プロンプト · メインセッション 
$ /agents
  → 定義を管理

$ @Researcherにレートリミットの RFC を調査して要約させて。
→ 分離コンテキストでResearcherを起動中…
← "3つの出典を発見、notes/rfc-rate-limit.md参照"
アプローチ 02 · バックグラウンドダッシュボード

エージェントビュー。

独立したバックグラウンドセッションのダッシュボード。プロンプトを送信し、 離れて、後で出力を覗きます。各行が独立したプロセスで、 ターミナルを閉じても残ります。

「投げて後で確認。5つの小さな仕事と 脳1つしかない時に便利。」

claude agentsで開きます。ダッシュボード内では、 Spaceで覗き見、Enterでアタッチ、 ←でデタッチします。

コンテキスト · セッションごと 協調 · あなた 永続性 · 閉じても残る 適している · 並列雑務
shell 
# バックグラウンドセッションを送信
$ claude --bg "不安定なe2eテストを調査"

# カスタムエージェントで送信
$ claude --agent code-reviewer --bg "PR #1234をレビュー"

# ダッシュボードを開く
$ claude agents

┌─ active ───────────────────────────┐
│ ▸ flaky-e2e        running   2m 14s │
│ ▸ pr-1234-review   running     38s │
│ ▸ docs-cleanup     done      11m 02s │
└────────────────────────────────────┘
アプローチ 03 · 実験的 · 協調型

エージェントチーム。

複数のセッションが同じ目標に向かって、共有タスクリスト、 エージェント間メッセージ、作業を統括するリードと共に動きます。 ワーカープールというよりスタジオに近いです。

「セキュリティ担当、パフォーマンス担当、テスト担当を1人ずつ — 彼らはお互いに会話する。」

実験的でデフォルトでは無効です。 settings.jsonで有効化し、Claudeに目標のための チームを編成するように依頼します。

コンテキスト · メンバーごと 協調 · 組み込み ステータス · 実験的 適している · 多軸レビュー
settings.json 
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
プロンプト 
$ PR #142レビューのためのチームを編成。
  並列で動く3人のレビュアー:
    1. security  — 認証、入力、シークレット
    2. perf     — ホットパス、アロケーション、クエリ
    3. tests    — カバレッジ、不安定性、モック
  リードが集約して1つのレビューを投稿。

→ team:r142 が3人のメンバーで起動
→ 共有タスクリスト作成 (12アイテム)
← 統合レビューを #142 に投稿しました
アプローチ 04 · シンプルなやつ · 強く推奨

たくさんのターミナル。

別のターミナルタブを開きます。claudeを実行。完了 — エージェントが2つになりました。これをgit ワークツリーと組み合わせると 真の分離が得られます: 各エージェントが独自のブランチとワーキングツリーを持ち、 編集の衝突がありません。

「最もきれいなパターンは、最も間抜けなパターンでもある: もっとタブを。」

タスクが長時間で無関係で、各エージェントが何をしているか 完全に把握したいときに最適です。完了後はPRでマージします。

セットアップ · ゼロ 分離 · 完全 (ワークツリー) 可視性 · 完全 適している · 並列の機能開発
shell · エージェントごとのワークツリー 
# メインリポジトリから
$ git worktree add ../repo-feat-a -b feat-a
$ git worktree add ../repo-feat-b -b feat-b

# ターミナル 1 — 機能 A
$ cd ../repo-feat-a && claude

# ターミナル 2 — 機能 B
$ cd ../repo-feat-b && claude

# 後でPRでマージ ─ 衝突なし
tmux · ペイン分割版 
$ tmux new -s squad
  ctrl-b "       水平分割
  ctrl-b %       垂直分割
  ctrl-b o       ペイン切り替え

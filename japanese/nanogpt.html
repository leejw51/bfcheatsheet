nanoGPT — Karpathy 製の約 300 行の PyTorch ミニマル GPT

karpathy/nanoGPT は GPT-2 品質を再現する エンドツーエンドで最もシンプルなリポジトリ。2 つのファイルがほぼすべてを担います: model.py がネットワークを定義、 train.py が学習。このページで打ち込んでいけば GPT を完全に理解できます。

why nanoGPT
Tiny enough to read in an hour, big enough to train GPT-2 (124M) on OpenWebText to the same loss as the original. Single PyTorch file, no abstractions. The reference implementation everyone learns from.
what you'll build
A decoder-only Transformer: token + position embeddings → N copies of (LN → causal attn → residual → LN → MLP → residual) → final LN → linear head. That's a GPT.
tiny shakespeare
~1MB of Shakespeare you can train in minutes on a laptop GPU. Char-level for immediate fun, or BPE (tiktoken) for proper tokenization. Both shipped via data/<name>/prepare.py.
scale up at will
The same code reproduces GPT-2 by changing six knobs: n_layer, n_head, n_embd, block_size, batch_size, lr. torchrun multi-GPU is one flag away.

Repo layout — four files run the show

Clone karpathy/nanoGPT. The whole training stack lives in four scripts plus a per-dataset prepare.py.

nanoGPT/ ├── model.py # GPT, Block, CausalSelfAttention, MLP — ~300 LOC ├── train.py # config + AdamW + DDP + the loop ├── sample.py # load a checkpoint, generate text ├── configurator.py # tiny override system: train.py config/foo.py ├── config/ # named configs (train_shakespeare_char.py, ...) ├── data/ │ ├── shakespeare/ │ │ └── prepare.py # tokenizes input.txt → train.bin / val.bin (uint16) │ └── openwebtext/ │ └── prepare.py # BPE via tiktoken on OWT └── README.md

The four tasks → the four files

model.py — define
Four classes: CausalSelfAttention, MLP, Block, GPT. A @dataclass GPTConfig holds the hyperparameters.
train.py — fit
Configures dataset, model, AdamW. Cosine LR schedule, gradient clipping, mixed precision, DDP. Saves ckpt.pt by val loss.
sample.py — infer
Loads checkpoint, runs model.generate(idx, max_new_tokens, temperature, top_k) autoregressively. Pure forward passes inside torch.no_grad.
prepare.py — tokenize
Reads raw text, encodes (char-level or BPE), writes train.bin / val.bin as flat np.uint16 arrays. Train loads them via np.memmap.

The Block — pre-norm Transformer in 8 lines

A Block wraps two sub-layers in residual connections, each preceded by a LayerNorm. nanoGPT uses pre-norm: LN sits inside the residual branch, not on the highway. This is the trick that makes deep stacks trainable.

model.py — Block
class Block(nn.Module):
    def __init__(self, config):
        super().__init__()
        self.ln_1 = LayerNorm(config.n_embd, bias=config.bias)
        self.attn = CausalSelfAttention(config)
        self.ln_2 = LayerNorm(config.n_embd, bias=config.bias)
        self.mlp  = MLP(config)

    def forward(self, x):
        x = x + self.attn(self.ln_1(x))   # residual + pre-norm attn
        x = x + self.mlp(self.ln_2(x))    # residual + pre-norm mlp
        return x

The MLP

model.py — MLP (4× hidden, GELU)
class MLP(nn.Module):
    def __init__(self, config):
        super().__init__()
        self.c_fc    = nn.Linear(config.n_embd, 4 * config.n_embd, bias=config.bias)
        self.gelu    = nn.GELU()
        self.c_proj  = nn.Linear(4 * config.n_embd, config.n_embd, bias=config.bias)
        self.dropout = nn.Dropout(config.dropout)

    def forward(self, x):
        return self.dropout(self.c_proj(self.gelu(self.c_fc(x))))

Causal self-attention — the heart of GPT

Each token mixes information from previous tokens (and itself), weighted by learned similarity scores. The causal mask blanks out future positions so a token at time t never sees t+1. Multi-head splits the channel dim into n_head independent attention computations.

attention heatmap (T×T)

Brighter = higher attention weight. Drag the temperature slider to sharpen / soften the softmax. Toggle the causal mask: with mask off you get bidirectional (BERT-style) attention.

causal mask: ON
Causal mask: upper triangle replaced with -inf before softmax, so attention weights there go to 0. The trick: build it once with torch.tril(torch.ones(T,T)) and reuse it.
model.py — CausalSelfAttention (forward)
def forward(self, x):
    B, T, C = x.size()                            # batch, time, channels
    q, k, v = self.c_attn(x).split(self.n_embd, dim=2)

    # split into heads: (B, T, C) → (B, nh, T, hs)  where C = nh * hs
    q = q.view(B, T, self.n_head, C // self.n_head).transpose(1, 2)
    k = k.view(B, T, self.n_head, C // self.n_head).transpose(1, 2)
    v = v.view(B, T, self.n_head, C // self.n_head).transpose(1, 2)

    att = (q @ k.transpose(-2, -1)) * (1.0 / math.sqrt(k.size(-1)))
    att = att.masked_fill(self.bias[:,:,:T,:T] == 0, float('-inf'))   # causal
    att = F.softmax(att, dim=-1)
    att = self.attn_dropout(att)
    y = att @ v                                   # (B, nh, T, hs)
    y = y.transpose(1, 2).contiguous().view(B, T, C)   # reassemble
    return self.resid_dropout(self.c_proj(y))

# in __init__: persistent (non-learnable) lower-triangular mask

train.py — the loop (simplified)
while True:
    lr = get_lr(iter_num) if decay_lr else learning_rate
    for g in optimizer.param_groups: g['lr'] = lr

    for micro_step in range(gradient_accumulation_steps):
        with ctx:                                    # bfloat16 autocast
            logits, loss = model(X, Y)
            loss = loss / gradient_accumulation_steps
        X, Y = get_batch('train')                   # prefetch next
        scaler.scale(loss).backward()

    if grad_clip != 0.0:
        scaler.unscale_(optimizer)
        torch.nn.utils.clip_grad_norm_(model.parameters(), grad_clip)
    scaler.step(optimizer)
    scaler.update()
    optimizer.zero_grad(set_to_none=True)
    iter_num += 1
train.py — weight-decay only on 2D params
# bias and LayerNorm/embedding 1D params get NO weight decay
decay = [p for _, p in param_dict.items() if p.dim() >= 2]
nodecay = [p for _, p in param_dict.items() if p.dim() < 2]
optim_groups = [
    {'params': decay,   'weight_decay': weight_decay},
    {'params': nodecay, 'weight_decay': 0.0},
]

Why bf16? bfloat16 has the same exponent range as float32 — no GradScaler needed (unlike float16). Faster than fp32, more stable than fp16. Default on Ampere+ GPUs.

Sampling — turning logits into text

Generation is autoregressive: feed the prompt, take the last logit, apply temperature, optionally clip to top-k, softmax, multinomial sample, append, repeat.

sampling distribution over 8 mock tokens

Drag the sliders. τ → 0 sharpens to argmax; τ → ∞ flattens to uniform. top_k zeros out everything outside the top-k.

1.00
8
Recipe: logits/τ → mask out non-top-k → softmax → torch.multinomial(probs, 1).
model.py — generate (the canonical loop)
@torch.no_grad()
def generate(self, idx, max_new_tokens, temperature=1.0, top_k=None):
    for _ in range(max_new_tokens):
        idx_cond = idx if idx.size(1) <= self.config.block_size else idx[:, -self.config.block_size:]
        logits, _ = self(idx_cond)
        logits = logits[:, -1, :] / temperature        # last step only
        if top_k is not None:
            v, _ = torch.topk(logits, min(top_k, logits.size(-1)))
            logits[logits < v[:, [-1]]] = -float('inf')
        probs = F.softmax(logits, dim=-1)
        idx_next = torch.multinomial(probs, num_samples=1)
        idx = torch.cat((idx, idx_next), dim=1)
    return idx

nanoGPT vs. HuggingFace

nanoGPT
~300 LOC, single file, zero abstractions. Hackable. Same numbers as GPT-2 with from_pretrained('gpt2') imported via the helper in model.py.
transformers (HF)
Production-grade: tokenizers, dozens of architectures, generation tricks (beam, contrastive, nucleus, repetition penalty). Bigger surface area. Use HF for shipping; use nanoGPT for learning.

パーセプトロン — 1個のニューロン、2つのアイデア

パーセプトロンは最もシンプルなニューラルネットワーク: 入力の重み付き和をとり、バイアスを加え、 符号に応じて 0 か 1 を発火させる単一ニューロンです。Frank Rosenblatt が 1958 年に提案し、現代のあらゆる深層ネットはこの 1 つのアイデアの変種を積み重ねたものです。

ニューロン
入力 $x_1, x_2, \dots, x_n$ に重み $w_1, w_2, \dots, w_n$ を掛けて合計し、 バイアス $b$ でオフセットします。活性化 (activation) 関数 (ここではステップ関数) が結果を予測値に変換します。
$\hat{y} = \mathrm{step}(w_1 x_1 + w_2 x_2 + \cdots + w_n x_n + b)$
ステップ活性化
$\mathrm{step}(z) = \begin{cases} 1 & z \ge 0 \\ 0 & z < 0 \end{cases}$

Heaviside 関数または sign 関数とも呼ばれます。ニューロンを 「発火」させる ON/OFF スイッチです。現代のネットでは ReLU や sigmoid に置き換えられます。 ステップ関数の微分はほぼどこでも 0 で、勾配降下と相性が悪いからです。

学習
ラベル付きの例を 1 つずつ見せ、予測を計算します。間違っていたら、次回少しだけ間違いが 小さくなるように重みを調整します:
$w_i \leftarrow w_i + \eta\,(y - \hat{y})\,x_i$
$b \leftarrow b + \eta\,(y - \hat{y})$
$\eta$ (eta) は 学習率 (learning rate)。誤分類がなくなるまで繰り返します。
推論
重みが固定されたら、予測は内積 1 回 + ステップ関数だけ。勾配もループも不要 — $\hat{y} = \mathrm{step}(\mathbf{w}\cdot\mathbf{x} + b)$ で終わりです。 推論呼び出しの正体はこれだけ。

数式で書き出すと

2 入力のパーセプトロンの場合:

$z = w_1 x_1 + w_2 x_2 + b \quad\Longrightarrow\quad \hat{y} = \mathrm{step}(z)$

境界線 $w_1 x_1 + w_2 x_2 + b = 0$ は平面を 2 つの半空間に分けます。 $z$ が正の点はクラス 1、負の点はクラス 0 です。

パーセプトロン学習則

Rosenblatt のアルゴリズムは驚くほどシンプル: 点を誤分類したら、重みベクトルをその点に向けて (または逆向きに) 少しだけ動かすだけ。すべての例について繰り返せば — 線形分離可能なデータなら — 必ず収束します。

更新式 — 1 例 $(x, y)$ について
$\hat{y} = \mathrm{step}(\mathbf{w}\cdot\mathbf{x} + b)$
$\mathbf{w} \leftarrow \mathbf{w} + \eta\,(y - \hat{y})\,\mathbf{x}$
$b \leftarrow b + \eta\,(y - \hat{y})$

3 通りのケース:

  • 正解$y - \hat{y} = 0$ — 更新なし。
  • 1 を見逃し$y - \hat{y} = +1$ — $\mathbf{w}$ を $\mathbf{x}$ の方向へ。
  • 誤発火$y - \hat{y} = -1$ — $\mathbf{w}$ を $\mathbf{x}$ から離す。

err = y − ŷ
(−1, 0, +1 のいずれか)

w += η · err · x

リファレンス実装

パーセプトロン学習 — 純粋 NumPy
import numpy as np

def step(z):
    return np.where(z >= 0, 1, 0)

def train_perceptron(X, y, lr=0.1, epochs=20):
    n, d = X.shape
    w = np.zeros(d)
    b = 0.0
    for epoch in range(epochs):
        for i in range(n):
            z    = np.dot(w, X[i]) + b
            yhat = step(z)
            err  = y[i] - yhat
            w   += lr * err * X[i]   # 重み更新
            b   += lr * err          # バイアス更新
    return w, b

# AND データセット
X = np.array([[0,0],[0,1],[1,0],[1,1]])
y = np.array([ 0,         0,         0,         1      ])

w, b = train_perceptron(X, y)
print(w, b)   # [0.2, 0.1] -0.2 のようなもの
収束: パーセプトロン学習則は、分離超平面が存在するなら有限ステップで 必ず それを見つけられます — ただし存在する場合に限ります。クラスが重なっていたり XOR 状だと、永遠に振動します。理由はタブ 6 で。

推論 — 学習済みパーセプトロンを使う

重みが固定されると、予測は たった 1 行の計算 です: 内積 + バイアス + ステップ関数。 ループも勾配も状態もなし。これが世界で一番シンプルな「ニューラル推論」。

スカラー形式
$\hat{y} = \mathrm{step}\!\left(\sum_{i=1}^{n} w_i x_i + b\right)$
ベクトル形式
$\hat{y} = \mathrm{step}(\mathbf{w}^{\!\top}\mathbf{x} + b)$
推論 — バッチ向けにベクトル化
import numpy as np

# 学習済み重み (例: AND 用)
w = np.array([1.0, 1.0])
b = -1.5

# 入力のバッチ、形 (N, 2)
X_new = np.array([[0,0], [0,1], [1,0], [1,1]])

logits = X_new @ w + b           # 形 (N,)
preds  = np.where(logits >= 0, 1, 0)

print(preds)                     # [0 0 0 1]
なぜこんなに速いか: 推論は層あたり 1 回の行列・ベクトル積。 本物の LLM はパーセプトロンが何十億個も積み重なっただけ — 個々のニューロンの推論は これと同じです。

XOR 問題 — 1 個のニューロンでは足りない理由

単一パーセプトロンは 直線 しか引けません。AND と OR は線形分離可能 — 1 本の線で 1 と 0 を分けられます。XOR はそうではない。 $(0,1)$ と $(1,0)$ を、$(0,0)$ と $(1,1)$ から 1 本の線で分けてみてください。引けないはず。

1 は対角の角にあります — 単位正方形の中をどう線で切っても 0 と分離できません。

XOR — 直線では切れない

マイナス 2 つ、プラス 2 つ — どんな直線でも分離できません。

解決策: パーセプトロンを積み重ねる → MLP

隠れ層 (hidden layer) を 1 つ加えるだけで、任意の論理関数を組み立てられます。XOR = (A OR B) AND NOT (A AND B) — 2 個のニューロンが 3 個目に繋がる構造です。

歴史メモ: Minsky と Papert の 1969 年の本 Perceptrons が、単層では XOR を解けないことを示し、「AI の冬」が訪れました。隠れ層を介した誤差逆伝播 (backpropagation) という解決策が広まったのは 1980 年代。その多層パーセプトロン (MLP) こそ、現代のあらゆる深層 ネットワークの直接の祖先です。

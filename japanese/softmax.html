1Softmaxとは

ニューラルネットワークの最終線形層が出力する生のスコアはロジットと呼ばれ、正負どちらの実数も取りえます。Softmaxはそのベクトルを確率分布に変換します。各成分は$(0,1)$の範囲に収まり、ベクトル全体の合計は$1$になります。

スコアから確率へ

分類器が$K$クラスに対してロジット$\mathbf{z} = (z_1,\dots,z_K)$を出力する場合、softmaxは次を返します。

$$\text{softmax}(\mathbf{z})_i \;=\; \frac{e^{z_i}}{\sum_{j=1}^{K} e^{z_j}}.$$

Transformer内部でも同じ仕事をします。アテンションヘッドが見る$K$個のトークンに対して、生の類似度スコアを正規化されたアテンション重みに変換します。

なぜ「ソフトな」max?

ハードな$\arg\max$は勾配を消してしまいます。1つだけが勝者となり、敗者スコアの僅かな変化からは学習信号がゼロです。Softmaxはなめらかで微分可能な近似です。最大のロジットが支配しますが、すべてのロジットが（小さな）確率の取り分と利用可能な勾配を持ちます。

2数式の導出

構成

任意のベクトルを取り、合計が1の非負の数を生成する関数が欲しいわけです。

  1. スコアを正にする: 厳密に正の写像を適用。自然指数$e^{z}$が最もきれいな選択です。なめらかで単調、$e^z>0$が常に成立。
  2. 正規化: それぞれを総和で割って合計を1にする。
$$\text{softmax}(\mathbf{z})_i = \frac{\exp(z_i)}{\sum_j \exp(z_j)}.$$

なぜ指数関数なのか? $z_i^2$や$|z_i|$ではダメ?

  • 厳密に正: 任意の実数入力に対して(分母がゼロにならず、符号の問題もない)。
  • 乗法的なギャップ: $e^{z_i}/e^{z_j} = e^{z_i - z_j}$は差のみに依存。すべてのロジットに定数を加えても何も変わらない — §3を参照。
  • 最大エントロピー: 平均エネルギー$\mathbb{E}[z]$を固定した分布の中で、指数族はエントロピー最大。Softmaxはロジットと整合する最大限に偏りのない分布です。
  • 対数尤度との共役性: クロスエントロピーと組み合わせると勾配が1行に圧縮されます — §6を参照。

体得すべき3つの性質

  • 合計が1。 構成上、分子の総和=分母です。どのスライダーをドラッグしても、バーは行を埋めます。
  • 各入力に関して単調。 $z_i$を増やすと$p_i$は上がり(他は下がる)、決して交差しません。
  • シフト不変性。 $\text{softmax}(\mathbf{z}+c\mathbf{1}) = \text{softmax}(\mathbf{z})$。分子と分母の両方に$e^c$の係数が掛かりキャンセルします。これが数値安定化トリックの基礎です。$c = -\max_i z_i$を選べば最大の指数が$0$に — オーバーフローも情報損失もなし。

log-sum-expトリックを適用ボタンを試してください。ロジットを$-\max$だけシフトします。バーは動きません(シフト不変性!)が、見える指数はすべて$\le 1$になります。

4温度

τはロジットをスケールする

$$\text{softmax}_\tau(\mathbf{z})_i = \frac{e^{z_i/\tau}}{\sum_j e^{z_j/\tau}}.$$
  • $\tau \to \infty$: すべての$z_i/\tau \to 0$、分布は一様$1/K$になる。最大の探索。
  • $\tau \to 0^{+}$: ギャップが膨れ上がり、最大ロジットが勝ち、分布は$\arg\max$でのone-hotに収束。純粋な貪欲。
  • $\tau = 1$: 標準のsoftmax。

LLMサンプリングでは、温度はまさにこのつまみです。高い$\tau$は多様で意外なトークンを生み、低い$\tau$は決定的で反復的なトークンを生みます。

試してみよう

上のウィジェットでτを$0.05$から$5$までスライドしてください。

  • τが小さい → 勝者のバーが1.0に近づく
  • τ = 1 → 通常のsoftmax
  • τが大きい → バーが$1/3 \approx 0.333$に近づき平らになる

アニーリング — 学習やサンプリング中にτを徐々に下げる — は統計物理から借りた定番テクニックです。そこでは$\tau$は文字通りボルツマン分布における温度です。

5アテンションでsoftmaxが使われる理由

スケールドドット積アテンション

$$\text{Attn}(Q,K,V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right) V.$$

あるクエリトークンに対して、$QK^\top$は各キーとの類似度スコアを生成します。Softmaxはそれらのスコアをアテンション重みに変換します。非負で合計1、各ソーストークンに1つずつ。

出力は$\sum_j w_j v_j$ — つまりバリューベクトルの加重平均です。$\sum w_j = 1$なので、何個のトークンに注意を向けようと出力の大きさは単一の$v_j$と比較可能なまま保たれます。これがsoftmax(ReLU正規化などではなく)がデフォルトである理由です。

なぜsoftmaxの前に$1/\sqrt{d_k}$?

クエリとキーが単位分散の成分を持つ場合、$QK^\top$の分散は$\propto d_k$。$d_k$が大きいとロジットが膨れ、softmaxが飽和し、勾配が消失します。$\sqrt{d_k}$で割ると分散$\approx 1$に保たれ、softmaxは敏感で学習可能なレジームに留まります。これは本質的に温度$\tau = \sqrt{d_k}$です。

6Softmax + クロスエントロピー: 勾配

セットアップ

$p_i = \text{softmax}(z)_i$、ターゲットone-hot$y$、クロスエントロピーを

$$L = -\sum_i y_i \log p_i.$$

とします。$\partial L/\partial z_i$が欲しい。Softmaxのヤコビアンは煩雑です。

$$\frac{\partial p_i}{\partial z_j} = p_i(\delta_{ij} - p_j).$$

しかし$L$と合成すると、見事に簡略化されます。

簡略化

連鎖律。

$$\frac{\partial L}{\partial z_i} = \sum_k \frac{\partial L}{\partial p_k}\frac{\partial p_k}{\partial z_i} = \sum_k \left(-\frac{y_k}{p_k}\right)\, p_k(\delta_{ki}-p_i).$$

整理すると。

$$=\;-\sum_k y_k(\delta_{ki}-p_i) \;=\; -y_i + p_i\sum_k y_k \;=\; p_i - y_i.$$

つまり$\nabla_{z}L = \mathbf{p} - \mathbf{y}$。勾配は誤差ベクトル: 予測 − ターゲット。クリーンで安価で数値的にやさしい。これがsoftmax + クロスエントロピーが至る所でペアになっている理由です。

7数値安定性 — log-sum-expトリック

$e^{1000}$を計算すると64ビット浮動小数点をオーバーフローし、$e^{-1000}$を計算するとアンダーフローして$0$になり、$\log 0 = -\infty$。素朴なsoftmaxは脆弱です。シフト不変性が我々を救います。

$m = \max_i z_i$とすると

$$\text{softmax}(\mathbf{z})_i = \frac{e^{z_i - m}}{\sum_j e^{z_j - m}}.$$

これで最大の指数は$e^0 = 1$、それ以外はすべて$(0,1]$。オーバーフローなし、丸め誤差最小。

$\log\,\text{softmax}$にはlog-sum-exp恒等式が標準形です。

$$\log\sum_j e^{z_j} = m + \log\sum_j e^{z_j - m}.$$

あらゆるフレームワークのlog_softmax、nll_loss、アテンションカーネルで使われています。

8代替案、簡潔に

Sparsemax

ロジットを確率単体上に射影。出力は本当にスパース — 厳密な0が許されます。アテンションが大半のトークンを完全に無視させたい場合に有用(解釈可能性、構造化予測)。なめらかさは劣り、ソートステップを要しますが、ほぼ至る所で微分可能です。

Gumbel-softmax

Gumbelノイズを加えてから低温度のsoftmaxを取る: カテゴリカル分布からのサンプリングを連続的・微分可能に緩和したもの。離散潜在変数を逆伝播で学習可能にします(例: 離散VAE、ハードアテンション)。

他の関連手法 (entmax-1.5、球面softmax、softmaxの混合…)

Entmax-αはsoftmax (α=1)とsparsemax (α=2)の間を補間します。Mixture-of-softmaxesは言語モデルにおける「softmaxボトルネック」に対処します。すべて何らかの性質 — スパース性、表現力、計算量 — を素のsoftmaxに対してトレードオフしています。

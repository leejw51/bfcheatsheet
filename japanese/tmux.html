1. tmux とは?

tmux はターミナルマルチプレクサです。ひとつのターミナルウィンドウの中に、独立したシェルを複数並べたり、タブで切り替えたり、そして何より SSH 接続から切り離した状態で動かし続ける ことができます。

3 つの強力な機能:

  • 永続性 — ノート PC を閉じても、翌日再接続すればシェルは作業をやめた地点でそのまま待っている
  • レイアウト — ウィンドウをペインに分割し、vim、tail -f、htop を同時に走らせる
  • スクリプト可能 — ひとつのコマンドでプロジェクト用レイアウトを起動し、セッションを同僚と共有し、セットアップを自動化

2. メンタルモデル

tmux は クライアント/サーバー 型のプログラムです。あなたのターミナルは、長時間動き続ける tmux サーバーと通信します。サーバーが所有しているのは:

server
└── session "work"
    ├── window 0: editor
    │   ├── pane 0: vim
    │   └── pane 1: shell
    └── window 1: logs
        └── pane 0: tail -f app.log

attach = ターミナルがセッションに接続する。detach = ターミナルが切断されるが、セッションはサーバー上で動き続ける。SSH 接続を閉じることはセッションを終了させることとは違います。

3. プレフィックスキー

セッション内のほぼすべての tmux コマンドは プレフィックスキー から始まります。デフォルトは Ctrl+b。プレフィックスを押し、離してから、次のキーを押します。

表記: C-b " は Ctrl+b のあと "(つまり Shift+')。多くの人は届きやすさのためプレフィックスを Ctrl+a に再マップします。

下のシミュレーターで試してみよう — Ctrl+b を押すとインジケータが点灯します。

6. ペイン

ペインはウィンドウを矩形に分割します。tmux の表記は混乱しがちで有名です:

  • C-b " は 水平方向に分割(上下 — 水平な仕切り)。覚え方: " は縦に並んだ 2 つの点。
  • C-b % は 垂直方向に分割(左右 — 垂直な仕切り)。覚え方: % は 2 つの箱の間にあるスラッシュに見える。

ペイン間を移動C-b ↑/↓/←/→

7. コピーモードとスクロールバック

マウスでターミナルをスクロールすることもできますが、tmux で 本来の 方法は コピーモード です。C-b [ で入ります。すると vi キーで移動、検索、テキストのヤンクができます。

操作キー(コピーモード内)
コピーモードに入るC-b [
移動h j k l または矢印
前方/後方検索/ / ?
選択開始Space(vi)または mode-keys vi 設定下では v
選択範囲をコピーEnter(デフォルト)または vi モードでは y
貼り付けC-b ]
コピーモードを抜けるq

vi スタイルにするには ~/.tmux.conf で setw -g mode-keys vi を設定。

8. 設定 — ~/.tmux.conf

tmux 内から設定を再読込するには C-b : のあと source-file ~/.tmux.conf。あるいはバインドしてしまいましょう。

# プレフィックスを Ctrl-a に再マップ
unbind C-b
set -g prefix C-a
bind C-a send-prefix

# ウィンドウ/ペインのインデックスを 1 から開始 (1..9 のキーが届きやすい)
set -g base-index 1
setw -g pane-base-index 1
set -g renumber-windows on

# vim スタイルのペイン移動
bind h select-pane -L
bind j select-pane -D
bind k select-pane -U
bind l select-pane -R

# 現在のパスを保つまともな分割
bind | split-window -h -c "#{pane_current_path}"
bind - split-window -v -c "#{pane_current_path}"

# マウス + vi コピーモード
set -g mouse on
setw -g mode-keys vi
bind r source-file ~/.tmux.conf \; display "config reloaded"

# スクロールバックを大きく
set -g history-limit 50000

9. プラグイン (TPM)

TPM = Tmux Plugin Manager。~/.tmux.conf に追記:

set -g @plugin 'tmux-plugins/tpm'
set -g @plugin 'tmux-plugins/tmux-sensible'
set -g @plugin 'tmux-plugins/tmux-resurrect'   # セッションを保存/復元
set -g @plugin 'tmux-plugins/tmux-continuum'   # 起動時に自動復元
run '~/.tmux/plugins/tpm/tpm'

tmux 内で: C-b I でインストール、C-b U で更新。tmux-resurrect はセッションツリー全体をチェックポイントできます。tmux-continuum は 15 分ごとに自動保存し起動時にリロード — ノート PC を再起動しても作業は失われない。

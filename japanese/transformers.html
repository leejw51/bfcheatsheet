レベル01 // 動機

§01なぜTransformerなのか?

2017年以前、系列モデルの主役はRNNとLSTMでした。これらはトークンを1つずつ処理し、隠れ状態を前方へ運びます。しかし致命的な問題が3つありました:

  • 逐次処理ボトルネック。時間方向に並列化できません — トークンtを計算するにはトークンt-1が必要です。
  • 距離による勾配消失。1番目のトークンの情報は、100番目に届くまでにすべてのステップで圧縮されます。
  • 固定サイズの隠れ状態。過去すべてを1つのベクトルに収めなければなりません。長距離記憶は脆弱です。
重要な洞察

アテンションさえあればよい

Vaswaniらは問いました: もしすべてのトークンが他のすべてのトークンを直接、1つの並列ステップで見られたら? 再帰なし。畳み込みなし。各位置が学習された重みで他の各位置にアテンションを向ける、学習可能な機構だけ。

結果: 学習はハードウェアに線形にスケールし、勾配はO(1)ホップで流れ、段落の最初の単語と最後の単語を、隣接する単語と同じくらい簡単に関係づけられるモデルになりました。

レベル02 // 入力

§02トークン化と埋め込み

Transformerはテキストを直接読めません。整数IDを読みます。トークン化とは、文字列を固定語彙(典型的には30k〜200kエントリ)から得たトークンIDの列に変換する非可逆圧縮スキームです。現代のトークナイザ(BPE, WordPiece, Unigram, Tiktoken)はサブワード単位で分割するので、レアな単語は未知トークンではなくよく見るチャンクの列になります。

各整数IDは埋め込み行列 $E \in \mathbb{R}^{V \times d_{model}}$ を介して密ベクトルにマップされます($V$は語彙サイズ)。ルックアップ = 1行を取り出すだけ。結果として、$L$トークンの系列は [L, d_model] 形状のテンソルになります。

レベル03 // 順序

§03位置エンコーディング

自己注意は順列不変です: 入力トークンをシャッフルしても(位置情報がなければ)同じ出力になります。言語にとっては問題です。解決策: スタックに入力する前に、各トークン埋め込みに位置依存ベクトルを加えます。

原論文では幾何級数的に増える波長のサイン波を使います:

$$PE_{(pos,\,2i)} = \sin\!\Big(\tfrac{pos}{10000^{2i/d_{model}}}\Big), \qquad PE_{(pos,\,2i+1)} = \cos\!\Big(\tfrac{pos}{10000^{2i/d_{model}}}\Big)$$

ビジュアライザ

サイン波による位置波形

各行は次元iです。低い次元は速く振動し(位置ごとの差が大きい)、高い次元は滑らかです(大規模な順序情報)。

巧妙な性質: $PE_{pos+k}$ は $PE_{pos}$ の固定線形関数なので、モデルは相対オフセットを簡単に学習できます。現代の派生(RoPE, ALiBi, 学習済み絶対位置)はトレードオフが異なりますが、目的は同じ — 順序を注入することです。

レベル04 // 中核

§04自己注意 (Q · K · V)

すべての入力トークンは、3つの学習行列 $W^Q, W^K, W^V \in \mathbb{R}^{d_{model} \times d_k}$ によって3つのベクトルに射影されます:

  • Query (Q) — 「私は何を探しているのか?」
  • Key (K) — 「私は何を持っているのか?」
  • Value (V) — 「アテンションされた場合、何を寄与するのか?」

スケール付きドット積アテンションは次の式です:

$$\mathrm{Attention}(Q,K,V) = \mathrm{softmax}\!\Big(\tfrac{QK^\top}{\sqrt{d_k}}\Big)\,V$$

系列長 $L$、ヘッド次元 $d_k$ における形状: Q: [L, d_k] K: [L, d_k] V: [L, d_k]。積 $QK^\top$ は [L, L] — (query, key)ペアごとに1つの類似度スコア。行に沿ってソフトマックスをかけるとアテンション重みになり、その重みで$V$を再加重します。

レベル05 // アンサンブル

§05マルチヘッドアテンション

1つのアテンションパターンだけでは不十分です。マルチヘッドアテンションは$h$個のアテンション演算を並列に走らせ、それぞれが独自の $W_i^Q, W_i^K, W_i^V$ 射影を持ち、結果を連結します:

$$\mathrm{MHA}(X) = \mathrm{Concat}(\mathrm{head}_1, \ldots, \mathrm{head}_h)\,W^O$$

各ヘッドは異なる関係パターンへのアテンションを学習します — あるヘッドは構文依存を、別のヘッドは共参照を、また別のヘッドは単純な位置オフセットを追跡するかもしれません。分割によって、合計計算量はほぼシングルヘッドと同じに保たれます: $h=8$ ヘッドなら $d_k = d_{model}/h$ を取ります。

レベル06 // 接着剤

§06順伝播 · 残差 · norm

アテンションの後、各位置は位置ごとの順伝播ネットワークを通過します — 間に非線形性を挟んだ2つの線形層(元はReLU、現在は通常GELUまたはSwiGLU):

$$\mathrm{FFN}(x) = \max(0,\, xW_1+b_1)\,W_2 + b_2$$

隠れ層の幅は通常4·d_modelです。ここにモデルのパラメータの大半が住んでおり、この層がkey-valueメモリのように事実知識を保存しているという仮説は人気です。

残差接続

アテンション副層もFFN副層も、残差接続+レイヤー正規化で包まれます:

$$x \leftarrow \mathrm{LayerNorm}(x + \mathrm{Sublayer}(x))$$

なぜか? 残差は損失から入力までの勾配ハイウェイを作り、何百層もの学習を安定化します。LayerNormは特徴次元にわたって活性化を中央化・スケールし、各層の最適化を他から切り離します。

現代の派生はpre-normを好みます: $x \leftarrow x + \mathrm{Sublayer}(\mathrm{LayerNorm}(x))$ — 深層での学習がより安定します。

レベル07 // 構成

§07エンコーダ · デコーダ · マスク

今日支配的なアーキテクチャは3種類です:

  • エンコーダのみ (BERT) — 双方向自己注意。すべてのトークンが他のすべてのトークンを見る。理解タスク(分類、検索)に最適。
  • デコーダのみ (GPT) — causal/masked 自己注意。トークンtは$\le t$のトークンしか見られない。生成に最適。現代のLLMの多くがこれ。
  • エンコーダ・デコーダ (T5, 元祖NMT) — エンコーダがソースを読み、デコーダはエンコーダ出力にクロスアテンションしながらターゲットを生成。

因果マスク — 仕組み

ソフトマックスの前に、スコア行列の上三角を$-\infty$にセット。$-\infty$のソフトマックスは0なので、モデルは数学的に未来を覗くことが不可能です:

レベル08 // 目的関数

§08学習: 次トークン予測

デコーダのみのLLM(GPTファミリー)では、学習目的全体が驚くほどシンプルです:

トークン1..tが与えられたら、トークンt+1を予測。

モデルは各位置で語彙上の確率分布を出力します。損失は実際の次トークンに対する平均交差エントロピーです:

$$\mathcal{L} = -\frac{1}{N}\sum_{t=1}^{N} \log P_\theta(x_{t+1} \mid x_1, \ldots, x_t)$$

Causal maskingがこれを効率的にします: 長さ$N$の系列に対する1回の順伝播で、各位置1つずつ$N$個の予測が並列に得られます。ラベルを1つシフトするだけで、teacher forcingのトリックは不要です。

残りすべて — 命令チューニング、RLHF、ツール使用 — はこの目的の上に乗っています。事前学習はインターネット規模のデータを吸収し、モデルが「すべて」を学ぶ部分。ファインチューニングは振る舞いを形作る部分です。

まとめ

完全なパイプライン

  • 1. トークン → 整数ID → 埋め込み(+ 位置エンコーディング)
  • 2. N個のtransformerブロックのスタック: MHA → Add&Norm → FFN → Add&Norm
  • 3. 語彙サイズへの最終線形射影、ソフトマックス → 次トークン分布
  • 4. 真の次トークンに対する交差エントロピー損失、位置で総和
  • 5. 全重みを通じて逆伝播。約10²²回繰り返す。完了。

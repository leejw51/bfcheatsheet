変分オートエンコーダー — 漏れのあるボトルネックによる生成モデル

VAE は潜在コードが単一の点ではなく 確率分布 となる オートエンコーダーです。エンコーダーは $(\mu, \log\sigma^2)$ を出力し、再パラメータ化トリック (reparameterization trick) により $z \sim \mathcal{N}(\mu, \sigma^2)$ をサンプリング、デコーダーが $\hat{x}$ を復元します。 再構成損失 + 単位ガウス分布への KL で学習。学習後はエンコーダーを捨て、 $z \sim \mathcal{N}(0, I)$ をサンプリングして 新しいデータを生成 できます。

通常の AE との違い
通常の AE は $x \to z \to \hat{x}$ を決定的な関数で圧縮。潜在空間に穴があり、 ランダムな $z$ をサンプルしてもゴミが出ます。VAE は潜在を $\mathcal{N}(0, I)$ に近づけるので、 どんな $z$ も妥当なものに復号されます。
なぜ確率的に?
欲しいのは 生成: 新しいサンプルを引く方法。VAE は $p_\theta(x) = \int p_\theta(x|z)\,p(z)\,dz$ ($p(z) = \mathcal{N}(0, I)$) を学習。 $\log p(x)$ の下界 — ELBO — を最大化します。
トリック
サンプリングは微分不可能。再パラメータ化: $z = \mu + \sigma \odot \varepsilon$, $\varepsilon \sim \mathcal{N}(0, I)$。 これで勾配が $\mu$ と $\sigma$ を通り、$z$ をそれらの関数として扱えます。

ELBO を一行で

$\mathcal{L}_{\text{VAE}} = \underbrace{\mathbb{E}_{q(z|x)}[-\log p(x|z)]}_{\text{reconstruction}} \; + \; \beta \cdot \underbrace{\mathrm{KL}(q(z|x)\,\|\,p(z))}_{\text{regularizer}}$

第 1 項: デコーダーがサンプル $z$ から $x$ を再構成できる度合い。第 2 項: エンコーダーの $q(z|x) = \mathcal{N}(\mu, \sigma^2)$ が事前分布 $p(z) = \mathcal{N}(0, I)$ からどれだけ 離れたか。エンコーダーはエンコードしつつ、事前分布の近くにとどまる必要があります。

メンタルモデル

# VAE の順伝播
mu, logvar = encoder(x)                 # 2 つのヘッド
std        = (0.5 * logvar).exp()       # σ = exp(½·logσ²)
eps        = torch.randn_like(std)      # ε ∼ N(0, I)
z          = mu + std * eps             # 再パラメータ化
x_hat      = decoder(z)
loss       = recon(x_hat, x) + beta * kl(mu, logvar)

エンコーダーは 2 つのベクトルを生成

通常の AE が 1 つの潜在ベクトルを生成するのと違い、VAE のエンコーダーには 2 つの出力ヘッド: 1 つは平均 $\mu$、もう 1 つは $\log \sigma^2$ (対数分散 — $\sigma$ ではなく対数分散を予測すれば、 ネット出力は制約なし、$\exp$ 後の $\sigma$ は正のまま)。

PyTorch のエンコーダー
import torch
import torch.nn as nn

class Encoder(nn.Module):
    def __init__(self, in_dim=784, hidden=400, latent=20):
        super().__init__()
        self.fc1     = nn.Linear(in_dim, hidden)
        self.fc_mu   = nn.Linear(hidden, latent)   # 平均ヘッド
        self.fc_lv   = nn.Linear(hidden, latent)   # log 分散ヘッド

    def forward(self, x):
        h = torch.relu(self.fc1(x))
        return self.fc_mu(h), self.fc_lv(h)        # (μ, log σ²)
デコーダー — 逆方向の旅
class Decoder(nn.Module):
    def __init__(self, latent=20, hidden=400, out_dim=784):
        super().__init__()
        self.fc1 = nn.Linear(latent, hidden)
        self.fc2 = nn.Linear(hidden, out_dim)

    def forward(self, z):
        h = torch.relu(self.fc1(z))
        return torch.sigmoid(self.fc2(h))    # [0, 1] 範囲のピクセル

損失 — 再構成 + KL = ELBO

ELBO の最大化は 2 項の最小化と等価: デコーダーがどれだけ $x$ をうまく再構成できないか、 そしてエンコーダーの分布が $\mathcal{N}(0, I)$ からどれだけ離れているか。

$\mathcal{L} = \underbrace{\mathrm{BCE}(\hat{x}, x)}_{\text{reconstruction}} \; + \; \beta \cdot \underbrace{-\tfrac{1}{2}\sum_{j}\bigl(1 + \log\sigma_j^2 - \mu_j^2 - \sigma_j^2\bigr)}_{\mathrm{KL}(\,\mathcal{N}(\mu,\sigma^2)\,\|\,\mathcal{N}(0,I)\,)}$
再構成項
バイナリ / Bernoulli ピクセル (MNIST) なら ピクセル単位の F.binary_cross_entropy、 Gaussian ピクセルなら F.mse_loss。 解析的な ELBO 導出に合わせるため reduction='sum' を使い、その後バッチサイズで割ります。
KL 項 — 閉形式
2 つのガウス分布の KL はきれいな解析形があり、サンプリング不要: 
kl = -0.5 * torch.sum(
    1 + logvar - mu.pow(2) - logvar.exp()
)

β-VAE — β を動かしてトレードオフを体感

β = 1 標準 VAE。 β > 1 事前圧力が強まる → 滑らかな潜在だが再構成はぼやける。 β → 0 単なるオートエンコーダーに退化。

コードドリル — VAE 全体を組み立てる

すべてをつなぐ部品を打ち込みましょう: reparameterize、 完全な順伝播、損失関数、1 ステップの学習。

VAE の典型的な順伝播 + 損失 (完全版)
import torch
import torch.nn as nn
import torch.nn.functional as F

class VAE(nn.Module):
    def __init__(self, in_dim=784, hidden=400, latent=20):
        super().__init__()
        self.fc1     = nn.Linear(in_dim, hidden)
        self.fc_mu   = nn.Linear(hidden, latent)
        self.fc_lv   = nn.Linear(hidden, latent)
        self.fc3     = nn.Linear(latent, hidden)
        self.fc4     = nn.Linear(hidden, in_dim)

    def encode(self, x):
        h = torch.relu(self.fc1(x))
        return self.fc_mu(h), self.fc_lv(h)

    def reparameterize(self, mu, logvar):
        std = (0.5 * logvar).exp()
        eps = torch.randn_like(std)
        return mu + std * eps

    def decode(self, z):
        h = torch.relu(self.fc3(z))
        return torch.sigmoid(self.fc4(h))

    def forward(self, x):
        mu, logvar = self.encode(x.view(-1, 784))
        z          = self.reparameterize(mu, logvar)
        return self.decode(z), mu, logvar

def vae_loss(x_hat, x, mu, logvar, beta=1.0):
    recon = F.binary_cross_entropy(x_hat, x.view(-1, 784), reduction='sum')
    kl    = -0.5 * torch.sum(1 + logvar - mu.pow(2) - logvar.exp())
    return recon + beta * kl

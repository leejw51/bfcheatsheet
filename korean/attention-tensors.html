텐서 하나로, 로짓까지

Attention Is All You Need 트랜스포머는 하나의 레시피입니다: 소스 토큰 ID 텐서를 인코더에 통과시키고, 타깃 토큰을 인코더에 크로스-어텐션하는 디코더에 통과시킨 뒤, 어휘로 투영하면 다음 토큰 로짓이 나옵니다. 이 페이지는 형태(shape)를 추적합니다 — 입력 ID부터 최종 [B, T_tgt, V] 확률까지 모든 단계를 색상으로 구분해, 논문을 읽을 때 텐서를 놓치지 않도록 합니다.

논문 기본값 (base)
B (batch) = 2 # 배치 크기
T_src (src len) = 8 # 소스 토큰 수
T_tgt (tgt len) = 7 # 타깃 토큰 수
d_model = 512 # 임베딩 차원
h (heads) = 8 # 어텐션 헤드 수
d_k = d_v = 64 # d_model / h
d_ff = 2048# FFN 내부 차원 (4·d_model)
V (vocab) = 37000# EN→DE BPE 어휘
N (layers) = 6 # 인코더 레이어 수 = 디코더 레이어 수
색상 범례 (모든 탭 공통)
B 배치 T 토큰 d_model h 헤드 d_k d_ff

단계 사이에서 차원이 바뀌면 강조됩니다. 동일한 텐서가 헤드로 분할되고, 곱해지고, 스케일링되고, softmax를 거쳐 다시 합쳐지는 과정을 확인하세요.

인코더 블록 도식

이 페이지 사용법: 탭을 순서대로 이동하세요. 실시간 형태 탭이 핵심입니다 — B, T, d_model, h를 바꾸면 모든 단계가 실시간으로 업데이트되며, 각 단계의 텐서 크기를 나타내는 막대도 함께 변합니다. 색상이 있는 차원 클릭 (예: T=8)하면 전체 페이지에서 해당 차원을 추적할 수 있습니다. ?를 눌러 단축키를 확인하세요.

단계 1–3 · 토큰 ID에서 문맥 준비 텐서로

어텐션 전에 입력이 벡터 텐서로 변환되어야 합니다. 세 가지 빠른 단계: 정수 ID 입력, 실수 벡터 출력, 위치 정보는 덧셈으로 주입됩니다.

단계 1 · 토큰 ID
토크나이저 출력. 배치에 두 예제, 각각 여덟 토큰.
$\text{ids}\in\mathbb{Z}^{B\times T}$
shape [B=2, T=8] · dtype: int32 
# 두 문장, 각각 T=8 토큰으로 패딩/잘라내기
ids = [[ 101, 2057, 2293, 3019, 2025, 2748, 102, 0 ],
       [ 101, 7592, 2088,  102,    0,    0,   0, 0 ]]
# 형태: [B=2, T=8]
단계 2 · 임베딩 조회

가중치 행렬 $E\in\mathbb{R}^{V\times d_{model}}$은 어휘 항목마다 하나의 행을 갖습니다. ID로 조회하면 텐서에 차원이 추가됩니다: 각 정수가 크기 d_model의 실수 벡터로 대체됩니다.

$X = E[\text{ids}] \;\in\; \mathbb{R}^{B\times T\times d_{model}}$
[2, 8] → [2, 8, 512] 
import torch.nn as nn
embed = nn.Embedding(num_embeddings=30522, embedding_dim=512)
X = embed(ids)            # [B, T] -> [B, T, d_model]
print(X.shape)            # torch.Size([2, 8, 512])  형태 확인

파라미터 수: $V \cdot d_{model} = 30522 \cdot 512 \approx 15.6\text{M}$ — 임베딩 테이블 하나만으로도 단일 레이어 전체보다 큽니다.

단계 3 · + 위치 인코딩

셀프-어텐션은 순열 등변(permutation-equivariant)입니다 — 위치 정보가 없으면 "the cat sat"과 "sat cat the"가 동일한 출력을 냅니다. 논문은 $X$와 동일한 형태의 고정 사인파 패턴을 더합니다.

$\mathrm{PE}_{(pos, 2i)} = \sin\!\big(pos / 10000^{2i/d_{model}}\big)$
$\mathrm{PE}_{(pos, 2i+1)} = \cos\!\big(pos / 10000^{2i/d_{model}}\big)$
[2, 8, 512] + [8, 512] → [2, 8, 512] 
# PE 형태 [T, d_model], B 축에 브로드캐스트
X = X + PE[:T]            # [B, T, d_model] + [T, d_model] -> [B, T, d_model]

입력 형태 = 출력 형태. 정보는 값 안에 들어가며, 차원에는 변화가 없습니다.

단계 3 종료: 이제 [B, T, d_model] 형태의 텐서가 생겼습니다. 이 텐서는 단어의 의미와 위치를 모두 담고 있습니다. 이후의 모든 단계는 이 구조에 대한 변환입니다.

단계 4–11 · 멀티헤드 어텐션, 차원 단위로

이 논문이 존재하는 이유입니다. 각 토큰이 쿼리, 키, 값이 되고; 유사도를 계산하고, 스케일링하고, softmax를 거쳐, 값을 혼합하는 데 사용됩니다. 형태 변환도 두 번 있습니다: 헤드 분할과 헤드 재결합.

단계 4 · Q, K, V 투영

각각 $\mathbb{R}^{d_{model}}\to\mathbb{R}^{d_{model}}$인 독립적인 세 선형 사상:

$Q = X W^Q,\quad K = X W^K,\quad V = X W^V$
X [2, 8, 512] Wᵠ [512, 512] → Q [2, 8, 512] 
# 세 선형 레이어, 논문엔 편향 없지만 실제로는 있어도 됨
Wq = nn.Linear(d_model, d_model)   # 512*512 = 262 144 params
Wk = nn.Linear(d_model, d_model)
Wv = nn.Linear(d_model, d_model)

Q = Wq(X)                              # [B, T, d_model]
K = Wk(X)                              # [B, T, d_model]
V = Wv(X)                              # [B, T, d_model]

$[d_{model}, d_{model}]$ 형태의 행렬 세 개 — 여기서 총 파라미터: $3\cdot 512\cdot 512 = 786\,432$. 일부 구현에서는 세 행렬을 하나의 $[d_{model}, 3\cdot d_{model}]$ 행렬로 융합합니다.

단계 5 · h개 헤드로 재형성 & 전치

마지막 차원을 각각 $d_k = d_{model}/h$인 $h$개의 청크로 잘라낸 후, 헤드가 외부 차원이 되도록 전치합니다. 새 파라미터 없음 — view + permute만 사용합니다.

[2, 8, 512] view→ [2, 8, 8, 64] transpose(1,2)→ [2, 8, 8, 64] 
B, T, d_model = Q.shape
h, d_k = 8, d_model // h         # 8, 64

# 마지막 차원을 (h, d_k)로 분할하고 헤드를 배치 옆으로 이동
Q = Q.view(B, T, h, d_k).transpose(1, 2)   # [B, h, T, d_k]
K = K.view(B, T, h, d_k).transpose(1, 2)   # [B, h, T, d_k]
V = V.view(B, T, h, d_k).transpose(1, 2)   # [B, h, T, d_k]
헤드가 중요한 이유: $h=8$개의 독립적인 $[T, d_k=64]$ 부분공간에서, 각 헤드는 서로 다른 관계 패턴에 특화될 수 있습니다. 계산량은 동일합니다: 하나의 $512$-차원 어텐션 대신 8개의 작은 $64$-차원 어텐션.
단계 6 · Q @ Kᵀ — 쌍별 유사도

각 헤드에서 모든 쿼리가 모든 키에 어텐션합니다. 행렬 곱이 $d_k$ 축을 축소하여 배치·헤드당 $T\times T$ 점수 행렬을 생성합니다.

$\text{scores} = Q K^\top$
Q [2, 8, 8, 64] Kᵀ [2, 8, 64, 8] → scores [2, 8, 8, 8] 
scores = Q @ K.transpose(-2, -1)   # [B, h, T, T]
# (b, head, i, j) = 해당 헤드에서 토큰 i가 토큰 j에 얼마나 어텐션하는지
단계 7 · 1/√d_k 스케일링

스케일링 없이는 $d_k$-차원 벡터의 내적이 $\sqrt{d_k}$만큼 커지며, 큰 값은 softmax를 포화 구간으로 밀어 그래디언트가 소실됩니다. 모든 점수를 $\sqrt{d_k}$로 나눕니다.

$\text{scaled} = \text{scores} / \sqrt{d_k} = \text{scores} / 8$
[2, 8, 8, 8] ÷ √64 [2, 8, 8, 8] 형태 유지 
import math
scores = scores / math.sqrt(d_k)   # 1/√64 = 1/8 = 0.125  스케일링
단계 8 · 마지막 축에 softmax

원시 점수를 쿼리별 확률 분포로 변환합니다. $[T,T]$ 블록의 각 행의 합이 1이 됩니다 — 이것이 어텐션 가중치입니다.

$A = \mathrm{softmax}(\text{scaled},\ \text{dim}=-1)$
[2, 8, 8, 8] → [2, 8, 8, 8] 각 행의 합 = 1 
import torch.nn.functional as F
A = F.softmax(scores, dim=-1)    # [B, h, T, T]

디코더 셀프-어텐션에서는 인과성을 위해 softmax 이전에 $-\infty$ 마스크를 더합니다.

단계 9 · 어텐션 @ V

어텐션 가중치를 사용해 값 벡터를 혼합합니다. 각 쿼리의 출력은 모든 $T$ 값 벡터의 가중합입니다. 수축하는 축은 내부 $T$입니다.

A [2, 8, 8, 8] V [2, 8, 8, 64] → out [2, 8, 8, 64] 
head_out = A @ V                  # [B, h, T, d_k]
단계 10 · 헤드 연결 (전치 + 재형성)

헤드를 하나의 큰 마지막 차원으로 다시 붙입니다. transpose(1,2)로 헤드를 $d_k$ 옆으로 이동시킨 뒤 reshape으로 병합합니다. 파라미터 없음.

[2, 8, 8, 64] transpose→ [2, 8, 8, 64] reshape→ [2, 8, 512] 
out = head_out.transpose(1, 2).contiguous()   # [B, T, h, d_k]  헤드 이동
out = out.reshape(B, T, d_model)              # [B, T, d_model]  헤드 병합
단계 11 · 출력 투영 W_O

하나의 $\mathbb{R}^{d_{model}\to d_{model}}$ 선형 사상. 형태는 동일하지만, concat 이후 헤드 간 정보를 혼합하게 해줍니다.

$\text{MHA}(X) = \mathrm{Concat}(\text{head}_1, \dots, \text{head}_h)\,W^O$
[2, 8, 512] ·W_O [2, 8, 512] 
Wo = nn.Linear(d_model, d_model)   # 추가 262 144 params
mha_out = Wo(out)                       # [B, T, d_model]

MHA 총 파라미터: $4 \cdot d_{model}^2 = 4\cdot 512^2 = 1{,}048{,}576$ (Q, K, V, O — 각각 $d_{model}\times d_{model}$).

어텐션 종료: 형태는 [B, T, d_model] — 입력과 완전히 동일한 형태입니다. 어텐션은 형태 보존 연산자입니다.

단계 12–16 · LayerNorm, FFN, 잔차 고속도로

어텐션 이후 인코더 블록은 두 개의 잔차 연결, 두 개의 LayerNorm, 하나의 위치별 FFN으로 마무리됩니다. FFN에 파라미터가 대부분 있으며 — 많은 이들이 사실적 지식이 저장되는 곳이라고 믿습니다.

단계 12 · 잔차 연결 + LayerNorm

원래 $X$를 다시 더하고 (스킵 연결), 토큰별로 $d_{model}$ 전체를 정규화합니다.

$X' = \mathrm{LayerNorm}(X + \mathrm{MHA}(X))$
[2, 8, 512] + [2, 8, 512] → LN → [2, 8, 512] 
ln1 = nn.LayerNorm(d_model)      # 2 * d_model = 1024 파라미터 (γ, β)
X_post = ln1(X + mha_out)         # [B, T, d_model]

LN은 마지막 축으로 정규화합니다: 각 $[d_{model}]$ 벡터가 평균=0, 분산=1이 된 후 학습된 $\gamma, \beta\in\mathbb{R}^{d_{model}}$로 재조정됩니다. 배치와 토큰 위치는 독립적입니다.

단계 13–14 · FFN 선형 1 + ReLU

위치별 피드포워드 네트워크: 동일한 MLP가 각 토큰에 독립적으로 적용됩니다. 첫 번째 선형 레이어가 $d_{model} \to d_{ff}$ (일반적으로 $4\cdot d_{model}$)로 확장합니다.

$H = \mathrm{ReLU}(X' W_1 + b_1)$
[2, 8, 512] ·W₁ [2, 8, 2048] → ReLU → [2, 8, 2048] 
W1 = nn.Linear(d_model, d_ff)       # 512 * 2048 = 1 048 576 파라미터
H = F.relu(W1(X_post))            # [B, T, d_ff]
단계 15 · FFN 선형 2

두 번째 선형 레이어가 $d_{ff} \to d_{model}$로 투영합니다. 표준 형태를 복원하여 다음 블록이 새로 시작할 수 있습니다.

$Y = H W_2 + b_2$
[2, 8, 2048] ·W₂ [2, 8, 512] 
W2 = nn.Linear(d_ff, d_model)       # 2048 * 512 = 추가 1 048 576 파라미터
ffn_out = W2(H)                       # [B, T, d_model]

FFN 총 파라미터: $2\cdot d_{model}\cdot d_{ff} = 2\cdot 512\cdot 2048 = 2{,}097{,}152$ — 네 개의 어텐션 투영을 합친 것의 약 두 배입니다. 대형 모델이 파라미터를 가장 많이 두는 곳입니다.

단계 16 · 잔차 연결 + LayerNorm (재등장)

단계 12와 동일한 패턴. FFN 이전 텐서를 더하고, 레이어 정규화하면 인코더 블록이 완성됩니다.

$Y' = \mathrm{LayerNorm}(X' + Y)$
[2, 8, 512] → [2, 8, 512] 블록 출력 형태 = 블록 입력 형태 
ln2 = nn.LayerNorm(d_model)
out = ln2(X_post + ffn_out)       # [B, T, d_model]
쌓고 반복: 논문은 동일한 인코더 블록을 $N=6$개 쌓습니다. 각 블록의 입력 형태와 출력 형태가 동일하므로 깔끔하게 조합됩니다. 인코더 총 파라미터 (블록 하나, base 기준): 대략 $4\cdot 512^2 + 2\cdot 512\cdot 2048 + 4\cdot 512 \approx 3.15$M.

인코더 블록 전체 — 14줄로

EncoderBlock — 최소 PyTorch
class EncoderBlock(nn.Module):
    def __init__(self, d_model=512, h=8, d_ff=2048):
        super().__init__()
        self.attn = nn.MultiheadAttention(d_model, h, batch_first=True)
        self.ffn  = nn.Sequential(
            nn.Linear(d_model, d_ff), nn.ReLU(), nn.Linear(d_ff, d_model)
        )
        self.ln1 = nn.LayerNorm(d_model)
        self.ln2 = nn.LayerNorm(d_model)

    def forward(self, x):
        # x: [B, T, d_model]  입력 텐서
        a, _ = self.attn(x, x, x)
        x = self.ln1(x + a)
        x = self.ln2(x + self.ffn(x))
        return x                       # [B, T, d_model]

단계 17–21 · 디코더 블록

디코더는 타깃 토큰(오른쪽으로 한 칸 이동)을 받아, (a) 지금까지 본 타깃 토큰 — 마스크 셀프-어텐션을 통해 — 과 (b) 소스 — 크로스-어텐션(Q는 디코더, K, V는 인코더 출력인 “memory”) 을 통해 — 모두 알고 있는 텐서를 생성합니다. 임베딩 차원과 잔차+LN 규율은 인코더와 동일합니다. 레이어당 세 서브블록, 논문에서 여섯 레이어.

논문 설정 (동일)
B = 2, T_src = 8, T_tgt = 7
d_model = 512, h = 8, d_k = 64
# T_tgt는 T_src와 독립적입니다.
# 크로스-어텐션이 두 시퀀스가 만나는 유일한 곳.
왜 “오른쪽으로 이동”인가?

학습 중 디코더는 <sos>로 시작하는 골드 타깃을 보고, <eos>로 끝나는 동일한 타깃을 예측합니다.

# target  : "die katze sitzt"           ← 정답
# dec_in  : "<sos> die katze sitzt"     ← 오른쪽으로 이동 (입력)
# labels  : "die katze sitzt <eos>"     ← 예측 대상

단계 17 · 디코더 입력 → 임베딩

[B, T_tgt] → 임베딩 조회 → [B, T_tgt, d_model] = [2, 7, 512]

단계 19 · 마스크 셀프-어텐션 (인과적 마스크)

인코더 셀프-어텐션과 동일하지만, 점수 행렬의 대각선 위 모든 위치에 −∞를 도장 찍어 위치 i가 i+1, i+2, ...를 볼 수 없게 합니다. softmax 이후 해당 위치는 정확히 0이 됩니다.

# scores: [B, h, T_tgt, T_tgt] = [2, 8, 7, 7]
mask = torch.triu(torch.ones(T_tgt, T_tgt), diagonal=1).bool()
scores = scores.masked_fill(mask, float('-inf'))
A = F.softmax(scores, dim=-1)        # 하삼각, 각 행의 합 = 1
[B, h, T_tgt, T_tgt] = [2, 8, 7, 7] 상삼각 = softmax 후 0

단계 20 · 크로스-어텐션 (Q ← 디코더, K, V ← 인코더)

인코더가 드디어 디코더와 대화하는 곳입니다. 디코더가 쿼리를 내보내고; 인코더 출력이 키와 값을 제공합니다.

# dec_x: [B, T_tgt, d_model] = [2, 7, 512]   ← 마스크 셀프-어텐션 출력
# memory: [B, T_src, d_model] = [2, 8, 512]  ← 인코더 출력 (단계 16)
Q = W_q(dec_x).view(B, T_tgt, h, d_k).transpose(1, 2)   # [B, h, T_tgt, d_k]
K = W_k(memory).view(B, T_src, h, d_k).transpose(1, 2)   # [B, h, T_src, d_k]
V = W_v(memory).view(B, T_src, h, d_k).transpose(1, 2)   # [B, h, T_src, d_k]
scores = Q @ K.transpose(-2, -1) / math.sqrt(d_k)         # [B, h, T_tgt, T_src] = [2, 8, 7, 8]
A = F.softmax(scores, dim=-1)                              # 마지막 축은 T_src
out = (A @ V).transpose(1, 2).reshape(B, T_tgt, d_model) # [2, 7, 512]

중요: 점수 행렬은 정방형이 아닙니다. 형태는 [B, h, T_tgt, T_src] — 모든 타깃 토큰이 모든 소스 토큰에 어텐션합니다.

디코더 블록 전체 — 16줄로

DecoderBlock — 최소 PyTorch
class DecoderBlock(nn.Module):
    def __init__(self, d=512, h=8, d_ff=2048):
        super().__init__()
        self.self_attn  = nn.MultiheadAttention(d, h, batch_first=True)
        self.cross_attn = nn.MultiheadAttention(d, h, batch_first=True)
        self.ffn = nn.Sequential(nn.Linear(d, d_ff), nn.ReLU(), nn.Linear(d_ff, d))
        self.ln1, self.ln2, self.ln3 = nn.LayerNorm(d), nn.LayerNorm(d), nn.LayerNorm(d)

    def forward(self, x, memory, causal_mask):
        # x:      [B, T_tgt, d]      memory: [B, T_src, d]  입력 텐서들
        a, _ = self.self_attn(x, x, x, attn_mask=causal_mask)
        x = self.ln1(x + a)                 # [B, T_tgt, d]
        c, _ = self.cross_attn(x, memory, memory)
        x = self.ln2(x + c)                 # [B, T_tgt, d]
        x = self.ln3(x + self.ffn(x))
        return x                            # [B, T_tgt, d]

디코더 스택 출력 (6개 후): [B, T_tgt, d_model] = [2, 7, 512]. 다음 단계: 어휘로 투영.

단계 22–23 · 은닉 상태에서 로짓으로

하나의 nn.Linear(d_model, V)가 각 토큰별 은닉 벡터를 어휘의 모든 단어에 대한 점수로 변환합니다. 마지막 축을 따라 softmax를 적용하면 확률이 됩니다. 이것이 출력 헤드 전부입니다. 논문에서 W_lm은 입력 임베딩 행렬과 공유(tied) 됩니다 — 동일한 파라미터, 전치만 됩니다.

단계 22 · 어휘로 선형 투영

[B, T_tgt, d_model] @ W_lm.T → [B, T_tgt, V] = [2, 7, 37000] 
# dec_out: [B, T_tgt, d_model]
# W_lm:    [V, d_model]   ← 종종 embed.weight와 공유
logits = dec_out @ W_lm.T           # [B, T_tgt, V]
# 또는 간단히:
logits = self.lm_head(dec_out)      # nn.Linear(d_model, V)

이것은 원시 점수 — 위치별, 예제별 어휘 토큰 하나당 점수입니다. 아직 확률이 아닙니다. 손실이 CrossEntropyLoss라면 softmax를 적용하지 마세요; 수치 안정성을 위해 내부에서 처리합니다.

단계 23 · Softmax → 확률 (실제로 필요한 경우에만)

probs = F.softmax(logits, dim=-1)   # [B, T_tgt, V] — V에 대한 각 행의 합 = 1

B·T_tgt개의 각 위치마다 전체 어휘에 대한 분포가 생깁니다. 37,000개의 숫자 대부분은 아주 작습니다. 소수만 정답 근처에 집중됩니다.

출력 헤드 파라미터 수

W_lm 하나만으로 V × d_model = 37000 × 512 ≈ 19M 파라미터 — 보통 모델에서 가장 큰 가중치입니다. 논문은 입력 임베딩 행렬과 공유하여 가중치 메모리를 절반으로 줄이고 (경험적으로) 일반화를 돕습니다.

전체 순전파, 처음부터 끝까지

Transformer.forward — 논문 충실 구현
def forward(self, src_ids, tgt_in_ids):
    # src_ids:    [B, T_src]   tgt_in_ids: [B, T_tgt]
    src = self.src_embed(src_ids) + self.pos(src_ids)        # [B, T_src, d]  소스 임베딩
    tgt = self.tgt_embed(tgt_in_ids) + self.pos(tgt_in_ids)    # [B, T_tgt, d]  타깃 임베딩

    memory = src
    for layer in self.encoder_layers:
        memory = layer(memory)                                  # [B, T_src, d]  인코더

    causal = self.causal_mask(tgt_in_ids.shape[1])
    x = tgt
    for layer in self.decoder_layers:
        x = layer(x, memory, causal)                            # [B, T_tgt, d]  디코더

    logits = self.lm_head(x)                                     # [B, T_tgt, V]  출력 헤드
    return logits

train_step — 논문 충실 구현, ~20줄
def train_step(model, optimizer, batch):
    # batch: list[(src_str, tgt_str)], 길이 B
    src_ids = tokenize_src(batch)         # [B, T_src]    int64
    dec_in, labels = tokenize_tgt(batch)  # [B, T_tgt] 각각

    src = model.src_embed(src_ids)*math.sqrt(d) + model.pos(src_ids)
    tgt = model.tgt_embed(dec_in)*math.sqrt(d) + model.pos(dec_in)

    memory = src
    for layer in model.encoder_layers: memory = layer(memory)

    causal = model.causal_mask(dec_in.shape[1])
    x = tgt
    for layer in model.decoder_layers: x = layer(x, memory, causal)

    logits = model.lm_head(x)              # [B, T_tgt, V]
    loss = F.cross_entropy(
        logits.view(-1, V), labels.view(-1),
        ignore_index=-100, label_smoothing=0.1)

    optimizer.zero_grad(); loss.backward(); optimizer.step()
    return loss.item()                     # 스칼라

한 학습 스텝의 형태 요약: 원시 문자열 (B) → [B, T_src] → [B, T_src, d] → [B, T_tgt, d] → [B, T_tgt, V] → 스칼라 손실 → 모든 파라미터의 ∂L/∂θ.

@torch.no_grad()
def generate(model, src_text, max_len=32):
    src_ids = tokenize_src([(src_text, "")])      # [1, T_src]

    src = model.src_embed(src_ids)*math.sqrt(d) + model.pos(src_ids)
    memory = src
    for layer in model.encoder_layers:
        memory = layer(memory)                          # [1, T_src, d]

    dec_in = torch.tensor([[SOS]])                         # [1, 1]  시작
    for _ in range(max_len):
        tgt = model.tgt_embed(dec_in)*math.sqrt(d) + model.pos(dec_in)
        causal = model.causal_mask(dec_in.shape[1])
        x = tgt
        for layer in model.decoder_layers:
            x = layer(x, memory, causal)                 # [1, t, d]
        logits = model.lm_head(x)                          # [1, t, V]
        next_id = logits[:, -1, :].argmax(-1, keepdim=True)
        if next_id.item() == EOS: break
        dec_in = torch.cat([dec_in, next_id], dim=1)    # [1, t+1]  자라남

    return detokenize(dec_in[0, 1:].tolist())          # str  최종 번역

추론 중 형태 요약: 원시 문자열 → [1, T_src] → [1, T_src, d] (memory, 한 번) → 루프: [1, t] → [1, t, d] → [1, t, V] → [1, V] → [1] → 추가 → 증가 → 최종 문자열.

B, T, d_model, h를 편집하세요. 모든 단계가 재계산됩니다 — 형태 알약, 파라미터 수, 유도된 $d_k$가 모두 업데이트됩니다. $d_{model}=768, h=12$ (BERT-base)를 시도해 보세요. 또는 $d_{model}=4096, h=32$ (LLaMA-7B). 같은 레시피, 다른 숫자들.

조절 노브

B T d_model h d_k d_ff 주황 강조 = 이전 단계 대비 차원 변경
변하지 않는 것: 모든 인코더 블록의 입력·출력 형태는 $[B, T, d_{model}]$로 동일합니다. 그것이 $N$개를 쌓는 것을 쉽게 만드는 이유입니다.
내부에서 변하는 것: 텐서는 어텐션을 위해 4D로 팽창하고, FFN 내부에서 $4\cdot d_{model}$까지 확장된 후, 다시 수축합니다.

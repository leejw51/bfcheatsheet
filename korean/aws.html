IAM 기초

Identity (누가) + Policy (무엇을) + Resource (어디서). 모든 API 호출은 정책에 대해 권한 부여됩니다.

개념

대상설명
User사람 (또는 서비스)에 묶인 장기 ID. 액세스 키, 콘솔 비밀번호, MFA를 가질 수 있습니다.
Role맡을 수 있는 ID. 단기 자격 증명 (STS)을 반환합니다. EC2, Lambda, EKS, 교차 계정 접근, 페더레이션이 사용합니다.
PolicyJSON 문서. Identity 정책은 user/role에 연결합니다. Resource 정책은 리소스 (예: S3 버킷 정책)에 연결합니다.
Group정책을 공유하는 사용자 묶음. 중첩 불가.
STSSecurity Token Service. AssumeRole, AssumeRoleWithWebIdentity 등을 통해 단기 자격 증명을 발급합니다.

단일 버킷에 대한 읽기 전용

  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Action": ["s3:GetObject", "s3:ListBucket"],
    "Resource": [
      "arn:aws:s3:::my-bucket",
      "arn:aws:s3:::my-bucket/*"
    ]
  }]
}
단일 prefix에 쓰기

  "Version": "2012-10-17",

  }]
}
신뢰 정책 (EC2가 맡는 역할)
역할에 연결됩니다; 누가 나를 맡을 수 있는가를 명시합니다.
{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Principal": {"Service": "ec2.amazonaws.com"},
    "Action": "sts:AssumeRole"
  }]
}

S3 기본

  
버킷은 리전 안의 평면 네임스페이스입니다. 키는 경로처럼 보이지만 폴더가 아닙니다. 아래는 매주 실행할 명령들입니다.


  
버킷

  

    

      
생성 / 목록 / 삭제

      
# 생성 — 버킷 이름은 전역
aws s3 mb s3://my-bucket --region us-east-1

aws s3 ls
aws s3 ls s3://my-bucket
aws s3 ls s3://my-bucket --recursive --human-readable --summarize

# 삭제 (비어 있어야 함)
aws s3 rb s3://my-bucket
# 또는 강제로 비우고 삭제:
aws s3 rb s3://my-bucket --force

    

    

      
cp — 단일 파일

      
aws s3 cp ./report.csv s3://my-bucket/reports/

# 다운로드
aws s3 cp s3://my-bucket/reports/report.csv ./

# 버킷 간
aws s3 cp s3://src/key s3://dst/key

# 메타데이터 + 스토리지 클래스 포함
aws s3 cp ./big.tgz s3://my-bucket/ \
  --storage-class STANDARD_IA \
  --metadata "team=ml,owner=jane" \
  --content-type application/gzip

    

    

      
sync — 디렉토리

      
달라진 파일만 복사합니다. 기본적으로 로컬에 없는 원격 파일을 삭제하지 않습니다 — 그러려면 --delete를 전달하세요.

      
aws s3 sync ./build/ s3://my-bucket/v1/ \
  --exclude "*.tmp" \
  --exclude ".git/*" \
  --include "*.html" \
  --delete \
  --dryrun

    

    

      
rm

      
aws s3 rm s3://my-bucket/old.csv

# 전체 prefix (재귀)
aws s3 rm s3://my-bucket/tmp/ --recursive

# 필터 사용
aws s3 rm s3://my-bucket/ --recursive \
  --exclude "*" --include "*.log"

    

    

      
사전 서명된 URL

      
AWS 자격 증명 없이 누구나 사용할 수 있는 시간 제한 서명 URL. 기본 만료는 3600초.

      
aws s3 presign s3://my-bucket/report.csv \

S3 고급

  
버전 관리, 암호화, 라이프사이클, 호스팅. 버킷을 단순 저장소에서 인프라로 바꿔주는 기능들.


  
버전 관리

  

    

      
활성화 / 목록 / 삭제

      
aws s3api put-bucket-versioning \
  --bucket my-bucket \
  --versioning-configuration Status=Enabled

# 버전 목록
aws s3api list-object-versions --bucket my-bucket --prefix reports/

# 특정 버전 삭제
aws s3api delete-object \
  --bucket my-bucket --key reports/q1.csv \
  --version-id 3HL4kqtJlcpXroDTDmJ+rmSpXd3dIbrHY+MTRCxf3vjVBH40Nrjfkd

    

    

      
MFA Delete

      
객체 버전을 삭제하거나 버전 관리를 비활성화하려면 MFA 토큰을 강제합니다. 루트 계정만 가능.

      
aws s3api put-bucket-versioning \
  --bucket my-bucket \
  --versioning-configuration Status=Enabled,MFADelete=Enabled \
  --mfa "arn:aws:iam::123456789012:mfa/root 123456"

    

  


  
암호화

  

    

      
기본 버킷 암호화

      
# SSE-S3 (AES256, AWS 관리)
aws s3api put-bucket-encryption \
  --bucket my-bucket \
  --server-side-encryption-configuration '{
    "Rules":[{"ApplyServerSideEncryptionByDefault":{
      "SSEAlgorithm":"AES256"}}]}'

# SSE-KMS (고객 관리 키)
aws s3api put-bucket-encryption \
  --bucket my-bucket \
  --server-side-encryption-configuration '{
    "Rules":[{"ApplyServerSideEncryptionByDefault":{
      "SSEAlgorithm":"aws:kms",
      "KMSMasterKeyID":"alias/my-key"}}]}'

    

    

      
SSE-S3 vs SSE-KMS

      
SSE-S3: 무설정, AWS가 키를 소유. 무료. 키 사용에 대한 감사/IAM 제어가 필요하지 않으면 사용하세요.
SSE-KMS: 키별 감사 로그, 키 회전, 교차 계정 부여. $0.03/10k 요청 + KMS 키 요금.

    

    

      
객체별 암호화

      
aws s3 cp file.txt s3://my-bucket/ \
  --sse aws:kms \
  --sse-kms-key-id alias/my-key

① S3 + CloudFront로 정적 사이트 호스팅

  

    

      
1. 버킷 만들기

      
BUCKET=my-site-$(uuidgen | tr A-Z a-z)
REGION=us-east-1

aws s3 mb s3://$BUCKET --region $REGION

# 공개 접근은 차단된 채로 둡니다 — 콘텐츠는 CloudFront가 제공합니다
aws s3api put-public-access-block \
  --bucket $BUCKET \
  --public-access-block-configuration \
   "BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true"

    

    

      
2. 사이트 동기화

      
aws s3 sync ./site/ s3://$BUCKET/ \
  --delete \
  --cache-control "public,max-age=3600" \
  --exclude "*.html" \
  --exclude "service-worker.js"

# HTML은 짧게 캐시해서 배포가 빠르게 보이도록 합니다
aws s3 sync ./site/ s3://$BUCKET/ \
  --exclude "*" --include "*.html" \
  --cache-control "public,max-age=60"

    

    

      
3. CloudFront를 앞에 두기

      
aws cloudfront create-distribution \
  --origin-domain-name $BUCKET.s3.$REGION.amazonaws.com
# 콘솔에서: OAC를 켜고, CloudFront SourceArn을
# 허용하는 버킷 정책을 연결하세요 ("버킷 정책" 카드 참조).

# 배포 후 캐시 무효화
aws cloudfront create-invalidation \
  --distribution-id E1ABC... --paths "/*"

③ S3 → Lambda 트리거

  

    

      
신뢰 + 실행 역할

      
aws iam create-role \
  --role-name lambda-thumb \
  --assume-role-policy-document '{
    "Version":"2012-10-17",
    "Statement":[{"Effect":"Allow",
      "Principal":{"Service":"lambda.amazonaws.com"},
      "Action":"sts:AssumeRole"}]}'

aws iam attach-role-policy \
  --role-name lambda-thumb \
  --policy-arn arn:aws:iam::aws:policy/service-role/AWSLambdaBasicExecutionRole

aws iam put-role-policy --role-name lambda-thumb \
  --policy-name s3rw --policy-document '{
    "Version":"2012-10-17",
    "Statement":[{"Effect":"Allow",
      "Action":["s3:GetObject","s3:PutObject"],
      "Resource":"arn:aws:s3:::my-bucket/*"}]}'

    

    

      
함수 생성

      
zip thumb.zip lambda_function.py

aws lambda create-function \
  --function-name thumb \
  --runtime python3.12 \
  --handler lambda_function.handler \
  --role arn:aws:iam::123:role/lambda-thumb \
  --zip-file fileb://thumb.zip

    

    

      
S3 → Lambda 연결

      
aws lambda add-permission \
  --function-name thumb --statement-id s3 \
  --action lambda:InvokeFunction \
  --principal s3.amazonaws.com \
  --source-arn arn:aws:s3:::my-bucket

aws s3api put-bucket-notification-configuration \
  --bucket my-bucket \
  --notification-configuration '{
    "LambdaFunctionConfigurations":[{
      "LambdaFunctionArn":"arn:aws:lambda:us-east-1:123:function:thumb",
      "Events":["s3:ObjectCreated:*"],
      "Filter":{"Key":{"FilterRules":[
        {"Name":"prefix","Value":"uploads/"},
        {"Name":"suffix","Value":".jpg"}]}}
    }]}'

SSM Parameter Store

      
무료 티어 (Standard, 4 KB). 비밀에는 SecureString을 사용하세요 — KMS로 암호화됩니다.

      
aws ssm put-parameter \
  --name /prod/api/URL \
  --value https://api.example.com \
  --type String --overwrite

aws ssm get-parameter \
  --name /prod/api/URL \
  --with-decryption \
  --query Parameter.Value --output text

# 트리 전체 나열
aws ssm get-parameters-by-path \
  --path /prod/ --recursive --with-decryption

일반 오류

  

    
증상원인 & 해결

    
      
An error occurred (AccessDenied)IAM 정책이 해당 액션을 허용하지 않습니다. identity 정책과 버킷/리소스 정책을 모두 확인하세요. 어느 ID인지 먼저 aws sts get-caller-identity로 확인하세요. IAM Policy Simulator를 사용하세요.

      
NoSuchBucket버킷 이름 오타이거나, 잘못된 리전/계정에 있습니다. 버킷 이름은 전역입니다; aws s3 ls는 당신의 계정이 소유한 버킷만 나열합니다.

      
The bucket you are attempting to access must be addressed using the specified endpoint리전 불일치. --region us-west-2를 전달하거나 AWS_REGION을 설정하세요.

      
ExpiredToken / InvalidTokenSSO/STS 자격 증명이 만료됐습니다. aws sso login --profile foo를 실행하거나 역할을 다시 맡으세요.

      
Could not connect to the endpoint URL잘못된 리전 (해당 리전에 S3 없음), VPC 엔드포인트 정책, 또는 프록시. --debug를 시도하세요.

      
RequestTimeTooSkewed시계가 어긋났습니다. 시간을 동기화하세요 (NTP).

      
An error occurred (SlowDown) when calling …S3가 prefix별로 속도 제한 중입니다. 키를 더 많은 prefix로 분산시키세요; 백오프 후 재시도. Prefix별 한계: 3500 PUT/COPY/POST/DELETE 및 5500 GET req/s.

      
InvalidAccessKeyId잘못된 키, 비활성화된 키, 또는 키가 다른 계정 소유입니다. ~/.aws/credentials와 $AWS_PROFILE을 다시 확인하세요.

      
버킷 생성 시 EntityAlreadyExists버킷 이름은 전역입니다. 다른 이름을 고르세요.

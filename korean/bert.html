BERT, 진짜로 이해하기.

Bidirectional Encoder Representations from Transformers — NLP를 조용히 재정의한 2018년 Google의 모델. 이 가이드는 설명, 인터랙티브 데모, 타이핑 드릴 연습 모드를 결합하여 아키텍처가 머릿속에 박히도록 합니다.

01 /

BERT란 무엇인가

BERT는 트랜스포머 인코더 전용 모델입니다. GPT(디코더 전용, 자기회귀적)와 달리, 모든 토큰이 모든 레이어에서 양방향으로 다른 모든 토큰에 어텐드합니다. 이 양방향성이 핵심 포인트입니다.

  • 출시: 2018년 10월, Devlin 외 (Google AI Language)
  • 아키텍처: 트랜스포머 인코더 블록 스택 (멀티헤드 셀프 어텐션 + 피드포워드)
  • 사전 학습: BooksCorpus (800M 단어) + 영어 Wikipedia (2.5B 단어)에서 자기지도 학습
  • 파인튜닝: 작은 태스크 헤드를 추가하고 레이블된 데이터에서 종단 간 학습
핵심 통찰: 수십억 개의 문장을 읽고 빈칸 채우기를 학습한 모델은 언어에 대해 많은 것을 압니다. 위에 작은 분류기를 끼워넣으면 다운스트림 태스크에서 그 무게보다 훨씬 큰 펀치를 날립니다.
02 /

사전 학습 목표

마스크된 언어 모델링 (MLM)

WordPiece 토큰의 15%를 무작위로 마스킹합니다. 모델은 원본을 예측합니다. 손실: 마스킹된 위치에 대해서만 교차 엔트로피.

그 15% 중:

  • 80%는 [MASK]로 대체
  • 10%는 무작위 토큰으로 대체
  • 10%는 그대로 유지

10/10 트릭은 모델이 게을러져서 마스크가 없는 파인튜닝 동안 [MASK] 위치에서만 강한 출력을 내는 것을 방지합니다.

왜 양방향인가? 단방향 언어 모델은 누출 없이는 미래 토큰을 조건으로 사용할 수 없습니다. MLM은 모든 위치가 전체 양방향 컨텍스트를 볼 수 있게 합니다.

다음 문장 예측 (NSP)

두 문장 A와 B가 주어졌을 때, B가 실제로 코퍼스에서 A를 따라오는지(50%) 다른 곳에서 무작위로 선택된 문장인지(50%) 예측합니다. [CLS] 토큰의 최종 히든 상태에 대한 이진 분류.

Input:  [CLS] A₁ A₂ ... [SEP] B₁ B₂ ... [SEP]
Label:  IsNext / NotNext
핫 테이크: RoBERTa (2019)는 NSP가 대부분 쓸모없다는 것을 보였고 이를 삭제했습니다. SBERT는 이를 문장 쌍 대조 목표로 대체했습니다. 현대적인 BERT 스타일 모델은 종종 NSP를 완전히 건너뜁니다.

결합 손실

BERT의 사전 학습 손실은 MLM과 NSP 손실의 합입니다:

$$\mathcal{L} = \mathcal{L}_{\text{MLM}} + \mathcal{L}_{\text{NSP}}$$

$$\mathcal{L}_{\text{MLM}} = -\sum_{i \in M} \log P(x_i \mid \tilde{x})$$

여기서 $M$은 마스킹된 위치의 집합이고 $\tilde{x}$는 손상된 입력입니다.

03 /

입력 형식

모든 입력 위치의 벡터는 세 임베딩의 합입니다: 토큰 + 세그먼트 + 위치. 세 가지 모두 같은 히든 크기 $d$ (예: 768)의 학습된 테이블입니다.

$$\mathbf{x}_i = \mathbf{E}_{\text{tok}}(t_i) + \mathbf{E}_{\text{seg}}(s_i) + \mathbf{E}_{\text{pos}}(i)$$

시각화

셀 위에 마우스를 올려보세요. 각 색깔 막대는 하나의 임베딩 유형입니다. 맨 아래 행은 그것들의 원소별 합 — 첫 번째 인코더 블록에 들어가는 것입니다.

(값은 예시용 — 실제 임베딩은 768차원이며 학습됨.)

특수 토큰

토큰역할
[CLS]첫 번째 위치. 그 최종 히든 상태는 분류에 사용되는 풀링된 시퀀스 표현입니다.
[SEP]구분자. 세그먼트 끝을 표시; NSP 및 쌍 태스크에서 문장 A와 B 사이에 필요합니다.
[MASK]MLM 사전 학습 중에 사용됨. 추론 시점에는 나타나지 않아야 합니다.
[PAD]배치에서 시퀀스를 정렬하기 위한 패딩. 어텐션 마스크에 의해 마스킹됩니다.
[UNK]어휘 외 폴백. WordPiece를 고려하면 드물지만 존재합니다.

인코더 블록 하나의 내부

L개의 각 블록은 표준 "post-LN" 트랜스포머 인코더입니다:

  1. 헤드당 H/A 차원의 멀티헤드 셀프 어텐션
  2. 잔차(Residual) + LayerNorm
  3. 위치별 피드포워드: linear → GELU → linear (히든 크기 4H)
  4. 잔차(Residual) + LayerNorm

$$\text{Attn}(Q,K,V) = \text{softmax}\!\left(\tfrac{QK^\top}{\sqrt{d_k}}\right)V$$

셀프 어텐션은 완전 양방향입니다 — 인과 마스크 없음.

05 /

WordPiece 토큰화

BERT는 30,522 토큰 WordPiece 어휘를 사용합니다. 알 수 없는 단어는 알려진 서브 조각으로 나뉘며, 연속 조각에는 ## 접두사가 붙습니다.

"playing"     → ["play", "##ing"]
"unaffable"   → ["un", "##aff", "##able"]
"transformer" → ["transform", "##er"]
"BERT"        → ["bert"]   (lowercased in bert-base-uncased)

시도해보기

teal = 특수 토큰, yellow = 완전한 단어, pink = ##연속

06 /

파인튜닝 패러다임

하나의 사전 학습된 BERT, 세 가지(그리고 더 많은) 다운스트림 태스크 패턴. 각 경우에 작은 태스크 전용 헤드를 추가하고 낮은 학습률 (보통 2e-5에서 5e-5)로 2-4 에포크 동안 종단 간 학습합니다.

시퀀스 분류

감정, NLI, 주제. 최종 [CLS] 히든 상태를 사용.

[CLS] → Linear(H→C) → softmax

예: SST-2, MNLI, QNLI

토큰 분류

NER, POS 태깅. 헤드를 모든 토큰의 히든 상태에 적용.

h_i → Linear(H→C) → softmax (토큰별)

예: CoNLL-2003 NER

스팬 추출 (QA)

답변 스팬의 시작 & 끝 위치 예측.

h_i → 2 로짓 (start, end)

예: SQuAD 1.1 / 2.0

07 /

양방향 vs 자기회귀적

BERT (인코더 전용)

전체 컨텍스트로 "my dog is __"에서 cute 예측:

my dog is [MASK] today .

양쪽에 어텐드합니다. 자기회귀적으로 텍스트를 생성할 수는 없지만, 강력한 문맥 표현을 만듭니다.

GPT (디코더 전용)

"my dog is" 다음의 다음 토큰 예측:

my dog is ? today .

인과 마스크가 미래 토큰을 차단합니다. 생성에 자연스럽게 적합하고, 동일한 파라미터 수에서 이해 태스크에는 더 약합니다 (역사적으로 — 격차가 좁혀졌음).

한계 및 후속 모델

BERT의 약점

  • 시퀀스 길이에 대한 2차 어텐션 → 최대 512 토큰
  • 생성 없음 — 인코더 전용, 샘플링 불가
  • 사전학습/파인튜닝 불일치 — [MASK]는 추론 시 절대 나타나지 않음
  • NSP는 약한 신호 — 후속 모델들이 폐기
  • 기본 상태로는 문장 임베딩이 좋지 않음; 평균 풀링된 [CLS]는 성능이 떨어짐

후속 모델 및 변종

모델연도특징
RoBERTa2019NSP 제거, 더 많은 데이터, 긴 학습, 동적 마스킹
ALBERT2019레이어 간 파라미터 공유; NSP 대신 문장 순서 예측
DistilBERT2019지식 증류, ~40% 더 작고, ~60% 더 빠르고, ~97% 성능 유지
ELECTRA2020대체 토큰 탐지 — MLM보다 훨씬 샘플 효율적
SBERT2019대조 손실로 사용 가능한 문장 임베딩을 만드는 샴 네트워크
DeBERTa2020분리된 어텐션 (내용 vs 위치), 향상된 마스크 디코더
ModernBERT2024긴 컨텍스트 (8K), Flash Attention, 현재 최고의 인코더
BERT 현장 가이드 — 자기 주도 학습용으로 제작

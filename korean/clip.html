CLIP — 사진과 단어를 같은 공간에

CLIP(Contrastive Language–Image Pre-training, OpenAI 2021)은 멀티모달 골드러시를 시작한 모델입니다. 이미지 인코더와 텍스트 인코더를 학습시켜, 매칭되는 이미지–캡션 쌍은 공유 임베딩 공간의 같은 지점에 두고 — 관련 없는 쌍은 멀리 떨어뜨리도록 합니다. 학습이 끝나면 클래스 이름을 문장으로 써주는 것만으로 제로샷 분류가 가능합니다.

두 인코더, 하나의 공간
이미지 인코더(ViT-B/32 또는 ResNet-50)와 텍스트 인코더(Transformer)가 각각 벡터를 출력합니다. 양쪽의 작은 프로젝션 헤드가 둘 다 같은 d=512 임베딩 공간으로 매핑합니다. 그런 다음 코사인 유사도를 측정합니다.
대조 학습 목표
N개의 이미지–텍스트 쌍 배치에 대해, N×N 유사도 행렬을 만듭니다. 대각선 셀이 실제 매칭(양성)이고, 그 외 모든 칸은 음성입니다. 대각선은 끌어올리고, 대각선 외는 눌러내립니다 — 행 과 열에 대해 대칭 교차 엔트로피를 적용합니다.
왜 중요한가
학습 데이터: 웹에서 스크래핑한 이미지–텍스트 쌍 4억 개. 사람이 단 라벨은 없습니다 — 캡션이 곧 라벨입니다. 결과: 한 번도 명시적으로 배운 적 없는 것을, 문장 매칭만으로 인식하는 모델입니다.
무엇을 가능하게 하는가
DALL·E 2의 CLIP 가이던스, Stable Diffusion의 텍스트 인코더(CLIP-L/14), 제로샷 분류기, 이미지 검색, 콘텐츠 모더레이션, 검색, 이미지-투-이미지, 텍스트-투-이미지 점수 매기기, OpenCLIP, SigLIP, EVA-CLIP… 모두 그 후손입니다.

한 호흡으로 보는 레시피

# 의사코드 (Radford 외, 2021)
I_f = image_encoder(I)            # [N, d_i]
T_f = text_encoder(T)             # [N, d_t]

I_e = l2_normalize(W_i @ I_f)     # [N, d]  프로젝션 + 정규화
T_e = l2_normalize(W_t @ T_f)     # [N, d]

logits = (I_e @ T_e.T) * exp(t)   # [N, N] 온도로 스케일링
labels = arange(N)                # 대각선이 정답
loss   = (CE(logits, labels) + CE(logits.T, labels)) / 2

아키텍처 — 두 개의 타워, 하나의 방

CLIP은 듀얼 인코더입니다. 두 개의 독립적인 타워가 각자의 입력을 고정 길이 벡터로 변환하고, 작은 선형 프로젝션이 둘을 같은 차원으로 정렬한 후, L2 정규화하고, 내적 = 코사인 유사도를 계산합니다.

대조 손실 — 대각선을 끌어올리기

N개의 이미지–캡션 쌍 배치 안에서, N×N 유사도 행렬을 만듭니다. 대각선 항목만이 유일한 정답 매칭이고, 나머지는 모두 배치 내 음성입니다. 손실은 대칭 교차 엔트로피입니다: 행은 각 이미지에 맞는 캡션을 고르고, 열은 각 캡션에 맞는 이미지를 고릅니다.

L = ½ · ( CE(logits, labels) + CE(logitsᵀ, labels) ), labels = [0, 1, 2, …, N−1]

4×4 배치로 놀아보기 — 셀을 드래그하면 손실이 변합니다

각 셀은 이미지 i(행)와 텍스트 j(열) 사이의 로짓(유사도 × 온도)입니다. 셀을 클릭한 다음 ↑ / ↓로 값을 미세 조정하세요. 대각선 셀은 파란색 테두리로 표시됩니다 — 질량이 여기로 모여야 합니다.

온도 — 소프트맥스는 얼마나 날카로운가?

원시 유사도 I_e @ T_e.T는 [-1, 1] 범위에 있습니다 — 유용한 소프트맥스에는 너무 평평합니다. CLIP은 학습 가능한 스케일 exp(t)를 곱하는데, 여기서 t는 단일 스칼라 파라미터입니다 (초기값은 log(1/0.07) ≈ 2.66).

P(text=j | image=i) = softmaxj( τ · cos(Ii, Tj) ), with τ = exp(t)

제로샷 — 학습 없이 분류하기

CLIP의 초능력: 모든 분류 작업을 텍스트 매칭 문제로 바꿉니다. 각 클래스 이름을 문장 템플릿("a photo of a {class}")으로 감싸 한 번에 인코딩한 다음, 어떤 이미지든 인코딩하고 클래스 임베딩과 내적, 소프트맥스 → 끝. 파인튜닝도, 라벨도 필요 없습니다.

코드 드릴 — 실제 CLIP 사용

세 개의 실전 스택. open_clip_torch가 사실상 표준입니다 (원본 릴리스보다 더 좋은 모델). transformers는 HuggingFace의 CLIPModel을 제공합니다. 둘 다 같은 수학을 감싼 것입니다.

open_clip — 로드 & 임베딩

전체 레퍼런스
import torch, open_clip
from PIL import Image

model, _, preprocess = open_clip.create_model_and_transforms(
    "ViT-B-32", pretrained="laion2b_s34b_b79k")
tokenizer = open_clip.get_tokenizer("ViT-B-32")
model.eval()

img    = preprocess(Image.open("cat.jpg")).unsqueeze(0)
labels = ["a photo of a cat", "a photo of a dog"]
text   = tokenizer(labels)

with torch.no_grad():
    img_f = model.encode_image(img)
    txt_f = model.encode_text(text)
    img_f /= img_f.norm(dim=-1, keepdim=True)
    txt_f /= txt_f.norm(dim=-1, keepdim=True)
    probs = (100.0 * img_f @ txt_f.T).softmax(dim=-1)

print("probs", probs)   # [[0.98, 0.02]]

보스 — CLIP 순전파를 한 번에 작성

최종 시험: 이 코드 블록의 모든 빈칸은 CLIP 순전파 + 손실의 한 부분입니다. 모두 초록색이 되면, 페이지가 4×4 CLIP 배치 학습을 시뮬레이션해 유사도 행렬이 대각선 위로 날카로워지는 것을 보여줍니다.

CLIP 학습 스텝 완성하기

11개 빈칸. 모두 초록색이 되면 학습 클릭. 

import torch
import torch.nn.functional as F

# --- 양 인코더 순전파 -------------------------------------------
image_features = model.(images)        # [N, d]
text_features  = model.(texts)          # [N, d]

# --- 내적이 코사인 유사도가 되도록 L2 정규화 -------------------
image_features = image_features / image_features.(dim=-1, keepdim=True)
text_features  = text_features  / text_features.(dim=-1, keepdim=True)

# --- N × N 유사도 로짓 구성 -------------------------------------
logit_scale = model.logit_scale.()                 # τ = exp(t)
logits_per_image = logit_scale * image_features @ text_features.
logits_per_text  = logits_per_image.

# --- 대각선 타깃에 대한 대칭 교차 엔트로피 ---------------------
N      = images.shape[0]
labels = torch.(N, device=images.device)
loss_i = F.cross_entropy(logits_per_image, labels)
loss_t = F.cross_entropy(logits_per_text,  labels)
loss   = (loss_i + ) / 2

# --- 표준 PyTorch 스텝 ------------------------------------------
optimizer.zero_grad()
loss.()
optimizer.()

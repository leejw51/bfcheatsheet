ComfyUI — 노드 기반 Stable Diffusion

ComfyUI는 Stable Diffusion을 위한 노드 그래프 에디터입니다. 슬라이더로 채워진 고정 UI(예: Automatic1111)와 달리, 각자 한 가지 일만 하는 박스를 직접 와이어링합니다. 체크포인트 로드, 프롬프트 인코딩, 라텐트 디노이징, 이미지 디코딩. 그래프 자체가 곧 프로그램입니다. 이 페이지를 끝낼 즈음에는 실제 txt2img 워크플로우를 와이어링하고, KSampler를 튜닝하고, HTTP API까지 호출하게 됩니다.

왜 노드인가?
디퓨전의 모든 단계(모델 로드, 프롬프트 인코딩, 샘플링, 디코딩)가 하나의 노드가 됩니다. 와이어는 그들 사이로 타입이 정해진 데이터를 나릅니다. 전체 파이프라인을 한눈에 볼 수 있고 — 원하는 대로 자유롭게 재배치할 수 있죠.
재현성
ComfyUI는 모든 출력 이미지의 PNG 메타데이터에 워크플로우 JSON 전체를 박아 넣습니다. 그 이미지를 캔버스에 다시 떨어뜨리면 — 그래프가 복원됩니다. 같은 시드 → 같은 픽셀.
교체 가능성
ControlNet, IP-Adapter, AnimateDiff, LoRA 스택이 필요한가요? 노드를 추가하고 와이어를 연결하면 끝. 베이스 그래프는 그대로입니다. 커스텀 노드는 custom_nodes/ 안에 폴더로 들어옵니다.
vs Automatic1111
A1111은 마법사 스타일: 첫 실행은 쉽지만 확장은 어렵습니다. ComfyUI는 배관 스타일: 초기 학습 비용은 있지만 이후 무한한 유연성. 파워 유저, 배치 실행, 그리고 단순치 않은 모든 파이프라인은 여기 살죠.

노드의 해부학

모든 ComfyUI 노드는 같은 모양입니다: 제목이 달린 박스에 왼쪽에는 입력 슬롯, 오른쪽에는 출력 슬롯, 그리고 몸통에는 위젯(숫자, 텍스트, 드롭다운)이 있죠. 슬롯에 와이어를 꽂고, 위젯 값은 노드 자체에 저장됩니다.

KSampler 노드 해부

입력 슬롯
왼쪽 가장자리. 각 슬롯은 타입을 갖고 있어 — 같은 타입끼리만 연결됩니다. CLIP 와이어를 MODEL 슬롯에 꽂으려 하면 ComfyUI가 거부하죠.
출력 슬롯
오른쪽 가장자리. 출력 하나가 여러 입력에 동시에 들어갈 수 있습니다(1대다). Load Checkpoint의 같은 MODEL 출력이 KSampler 와 LoRA 로더에 동시에 가는 게 흔한 패턴입니다.
위젯
노드 몸통. 숫자, 슬라이더, 드롭다운, 텍스트박스. 와이어 없이 노드 자체에 상수 값을 저장합니다. 다른 노드가 값을 주도록 하고 싶다면 위젯을 우클릭해서 입력 슬롯으로 변환하면 됩니다.
팁: 빈 캔버스를 더블클릭하면 노드 검색 팔레트가 열립니다. 슬롯을 우클릭하면 위젯으로 변환하거나 입력으로 변환할 수 있어요.

API 모드 — 코드로 ComfyUI 구동

모든 워크플로우는 API JSON으로 저장(기어 → "Save (API Format)")해서 HTTP로 제출할 수 있습니다. 그러면 UI는 선택 사항이 되죠 — 스크립트, 백엔드, Discord 봇, 어디서나 그래프를 실행할 수 있습니다.

잡 제출
import json, requests, uuid

# 기어 -> Save (API Format)으로 내보낸 워크플로우 JSON 로드
with open("workflow_api.json") as f:
    workflow = json.load(f)

# 이번 실행을 위해 프롬프트 패치
workflow["6"]["inputs"]["text"] = "a corgi in a spacesuit"
workflow["3"]["inputs"]["seed"] = 12345

payload = {"prompt": workflow, "client_id": str(uuid.uuid4())}
r = requests.post("http://localhost:8188/prompt", json=payload)
prompt_id = r.json()["prompt_id"]
print("queued:", prompt_id)

유용한 엔드포인트

methodpath역할
POST/prompt워크플로우를 실행 큐에 추가; prompt_id 반환.
GET/history/<prompt_id>완료 후 출력(이미지 파일명, 노드 결과)을 가져옴.
GET/queue실행 중 + 대기 중인 잡 목록.
GET/view출력 이미지 다운로드: ?filename=...&type=output.
WS/ws?clientId=...실시간 진행도 + 노드별 미리보기용 WebSocket.
POST/upload/image이미지 업로드 (img2img / 인페인트 입력용).

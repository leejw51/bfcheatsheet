HuggingFace GPT-2 — 학습시키고, 토큰화하고, 생성하기

GPT-2는 다음 토큰을 예측하도록 학습된 디코더 전용 (decoder-only) 트랜스포머입니다. HuggingFace의 transformers 라이브러리는 이걸 Python 세 줄로 감싸줍니다. 이 페이지는 전체 스택을 따라가며 — 파이프라인, 토크나이저, 모델, 샘플링 노브, 학습 루프 — 핵심 부분을 직접 타이핑해서 머릿속에 새기게 합니다.

네 가지 라이브러리
transformers 모델, 토크나이저, 파이프라인, Trainer
datasets 학습 데이터 스트림 / map / 캐시
tokenizers 빠른 Rust BPE / WordPiece
accelerate 멀티 GPU, 혼합 정밀도, 보일러플레이트 없음
GPT-2란?
디코더 전용 트랜스포머. 12 레이어 (124M), 24 (345M), 36 (774M), 48 (1.5B). WebText로 단 하나의 목표 — 다음 토큰 예측 — 만으로 학습됨. 모든 위치가 미래를 엿보지 못하도록 인과 마스크 (causal mask)가 막아줍니다. 토크나이저는 50,257 토큰 어휘의 바이트 레벨 BPE입니다.
사용하는 두 가지 방법
쉬움 pipeline("text-generation", model="gpt2")
저수준 AutoTokenizer + AutoModelForCausalLM + .generate()
파이프라인은 데모용으로 좋습니다. 실전에서는 모델 + 토크나이저 짝으로 내려가면 샘플링, 배치, 학습을 직접 제어할 수 있습니다.
디코더 전용 아키텍처
input ids → 토큰 + 위치 임베딩
[batch, seq] → [batch, seq, 768]
↓
12 × 디코더 블록
마스크된 self-attn + MLP + LayerNorm + 잔차 연결
↓
최종 LayerNorm
은닉 상태를 안정화
↓
lm_head (가중치 공유)
[..., 768] → [..., 50257] 어휘 로짓

빠른 시작 — 3줄로 텍스트 생성

두 가지 레시피. 파이프라인은 모든 걸 숨기고, 명시적인 두 줄짜리는 완전한 제어를 줍니다. 내부적으로는 둘 다 똑같은 모델을 굴립니다.

레시피 1 — 파이프라인

가장 간단한 생성
from transformers import pipeline

gen = pipeline("text-generation", model="gpt2")
out = gen("The future of AI is", max_new_tokens=30, do_sample=True)
print(out[0]["generated_text"])

레시피 2 — 토크나이저 + 모델

명시적인 두 줄
from transformers import AutoTokenizer, AutoModelForCausalLM

tokenizer = AutoTokenizer.from_pretrained("gpt2")
model     = AutoModelForCausalLM.from_pretrained("gpt2")

inputs  = tokenizer("The future of AI is", return_tensors="pt")
outputs = model.generate(**inputs, max_new_tokens=30, do_sample=True)
text    = tokenizer.decode(outputs[0], skip_special_tokens=True)
print(text)
클래스 찾기: AutoModelForCausalLM은 자기회귀적 텍스트 생성을 위한 적절한 헤드입니다. 다른 헤드로는 AutoModelForSequenceClassification, AutoModelForMaskedLM (BERT 계열), AutoModelForQuestionAnswering 등이 있습니다.

토큰화 — 문자열과 텐서를 잇는 BPE 다리

GPT-2는 글자나 단어를 보지 않고 — 토큰 ID를 봅니다. 바이트 레벨 BPE가 자주 등장하는 서브워드를 하나의 단위로 합치고 (예: "ing"), 드문 단어는 여러 토큰으로 나누며, 앞쪽 공백 마커 (Ġ로 표시)를 써서 가역적인 왕복을 보장합니다.

🔤 가짜 토크나이저 놀이터

텍스트를 입력하거나 붙여넣고 Tokenize를 누르세요. 이 페이지에서는 진짜 BPE를 브라우저에서 실행할 수 없으므로, 출력은 공백 + 구두점으로 빠르게 나누고 결정적인 ID 해시를 사용하는 근사값입니다. 출력의 모양은 GPT-2와 같습니다: 서브워드 조각과 조각마다 정수 id 하나.

생성 — .generate()의 모든 인자가 하는 일

"창의성"의 모든 마법은 model.generate()에 있습니다. 여섯 개의 인자로 95%의 경우를 처리합니다. 슬라이더를 움직여서 짜놓은 데모가 어떻게 반응하는지 보세요.

표준 생성 호출
inputs  = tokenizer(prompt, return_tensors="pt")
outputs = model.generate(
    **inputs,
    max_new_tokens=50,
    do_sample=True,            # False = 그리디/빔, True = 확률적
    top_k=50,                  # 상위 50 로짓만 남기고 나머지는 마스크
    top_p=0.9,                 # nucleus: 확률 합이 0.9가 되는 가장 작은 집합
    temperature=0.8,           # <1 더 날카롭게, >1 더 거칠게
    repetition_penalty=1.1,    # 반복을 억제
    pad_token_id=tokenizer.eos_token_id,
)
text = tokenizer.decode(outputs[0], skip_special_tokens=True)

샘플링 전략 — 다음 토큰 고르기

매 단계마다 모델은 50,257 토큰 어휘 전체에 대한 확률을 출력합니다. 전략은 그 분포를 한 토큰의 선택으로 바꿉니다. 전략마다 일관성 vs. 창의성 vs. 계산 비용의 트레이드오프가 다릅니다.

전략플래그느낌쓸 때
그리디 do_sample=False 결정적, 반복적 정답이 하나인 작업 (저온도 모드의 번역)
빔 서치 num_beams=4 안전하지만 가끔 무미건조하게 유창함 요약, 정형화된 출력
샘플링 do_sample=True 창의적, 헤맬 수 있음 스토리텔링, 브레인스토밍
top-k top_k=50 최악의 꼬리를 자름 안정성을 위해 샘플링과 짝지어 사용
top-p (nucleus) top_p=0.9 확신도에 적응 거의 항상 — 현대의 기본값
temperature temperature=0.8 날카로움 노브 채팅은 0.7–1.0, 코드는 0.2, 카오스는 >1.2
temperature 직관
softmax 전에 로짓을 T로 나눕니다. 낮은 T는 날카롭게 (그리디 같음), 높은 T는 평평하게 (균등 같음). T→0이면 그리디로 회귀. T=1이면 모델의 실제 분포에서 샘플링합니다.
top-k 직관
가장 가능성 높은 상위 k개를 제외하고 모든 토큰을 버립니다. 정규화 후 샘플링. 이상하고 드문 토큰의 긴 꼬리를 자릅니다. 약점: 확신 있는 단계와 평평한 단계에 같은 k를 씁니다.
top-p (nucleus)
누적 확률이 ≥ p가 되는 가장 작은 토큰 집합을 유지합니다. 단계별로 적응합니다: 확신 있는 단계 → 작은 집합; 평평한 단계 → 넓은 집합. 대부분의 작업에서 raw top-k를 이깁니다. p=0.9가 안전한 기본값.
나란히 비교
# 1. 그리디
model.generate(**inputs, max_new_tokens=30, do_sample=False)

# 2. 빔 서치
model.generate(**inputs, max_new_tokens=30, num_beams=4, early_stopping=True)

# 3. 일반 샘플링
model.generate(**inputs, max_new_tokens=30, do_sample=True, temperature=1.0)

# 4. top-k
model.generate(**inputs, max_new_tokens=30, do_sample=True, top_k=50)

# 5. top-p (nucleus)
model.generate(**inputs, max_new_tokens=30, do_sample=True, top_p=0.9)

# 6. 추천 조합
model.generate(**inputs, max_new_tokens=50,
               do_sample=True, top_k=50, top_p=0.9, temperature=0.8)
실전 조언: do_sample=True, top_p=0.9, temperature=0.8로 시작하세요. 출력이 너무 무작위하면 T를 낮추세요. 반복하면 repetition_penalty를 1.1–1.3으로 올리세요. 무미건조하면 T를 1.0으로 올리고 top_k를 빼세요.

경로 A — Trainer API

gpt2의 처음부터 끝까지 파인튜닝
from transformers import (
    AutoTokenizer, AutoModelForCausalLM,
    Trainer, TrainingArguments, DataCollatorForLanguageModeling,
)
from datasets import load_dataset

tokenizer = AutoTokenizer.from_pretrained("gpt2")
tokenizer.pad_token = tokenizer.eos_token

model = AutoModelForCausalLM.from_pretrained("gpt2")

ds = load_dataset("wikitext", "wikitext-2-raw-v1")

def tok(batch):
    return tokenizer(batch["text"], truncation=True, max_length=512)

ds = ds.map(tok, batched=True, remove_columns=["text"])

collator = DataCollatorForLanguageModeling(
    tokenizer=tokenizer,
    mlm=False,                          # 인과 LM, 마스크 LM 아님
)

args = TrainingArguments(
    output_dir="./gpt2-ft",
    per_device_train_batch_size=4,
    num_train_epochs=3,
    learning_rate=5e-5,
    warmup_steps=100,
    logging_steps=50,
    save_steps=500,
    fp16=True,
)

trainer = Trainer(
    model=model,
    args=args,
    train_dataset=ds["train"],
    eval_dataset=ds["validation"],
    data_collator=collator,
)

trainer.train()
trainer.save_model("./gpt2-ft")

경로 B — raw PyTorch 루프

Trainer 없이, torch만
import torch
from torch.utils.data import DataLoader
from torch.optim import AdamW

loader    = DataLoader(ds["train"], batch_size=4, collate_fn=collator)
optimizer = AdamW(model.parameters(), lr=5e-5)
device    = "cuda" if torch.cuda.is_available() else "cpu"
model.to(device)
model.train()

for epoch in range(3):
    for batch in loader:
        batch = {k: v.to(device) for k, v in batch.items()}
        outputs = model(**batch)             # labels=input_ids는 collator가 처리
        loss    = outputs.loss               # transformers는 CE 손실 반환
        loss.backward()
        optimizer.step()
        optimizer.zero_grad()
Loss = labels = input_ids: 인과 LM에서는 입력과 타겟이 같은 토큰 시퀀스입니다 — 모델은 앞의 토큰들로부터 각 토큰을 예측하도록 학습합니다. DataCollatorForLanguageModeling(mlm=False)가 레이블을 알아서 설정해줍니다.

TrainingArguments — 알 만한 모든 노브

TrainingArguments 데이터클래스는 Trainer가 건드리는 모든 하이퍼파라미터를 담고 있습니다. 100개 이상의 필드가 있습니다. 다음 12개가 실제로 자주 설정하는 것들입니다.

인자전형값역할
output_dir "./out" 체크포인트 + 로그가 저장되는 곳 (필수)
per_device_train_batch_size 4–32 GPU당 배치; 유효 배치 = 이 값 × num_gpus × grad_accum
gradient_accumulation_steps 1–16 GPU 메모리가 빡빡할 때 더 큰 배치를 흉내냄
num_train_epochs 1–10 학습 세트를 몇 번 반복
learning_rate 5e-5 트랜스포머 파인튜닝의 표준
warmup_steps 100–1000 선형 워밍업 — 초기 손실 폭주를 방지
weight_decay 0.01 AdamW 정규화
fp16 / bf16 True 혼합 정밀도; Ampere+ 에선 bf16, 그 외에선 fp16
logging_steps 50 N 스텝마다 손실 출력
save_steps 500 N 스텝마다 체크포인트
evaluation_strategy "steps" "no" / "steps" / "epoch"
load_best_model_at_end True 학습 후 최고 체크포인트로 복원
report_to "wandb" 또는 "tensorboard" / "none"
실전 비슷한 설정

    report_to="none",
)
유효 배치 크기: per_device_train_batch_size × num_gpus × gradient_accumulation_steps. 24 GB GPU에 배치 8이 들어간다면, grad-accum 4로 배치 32의 손실 곡선을 얻을 수 있습니다.

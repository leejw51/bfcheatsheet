입력 임베딩이 학습되는 방법

토크나이저는 모델에 [464, 3797, 3332] 같은 정수 목록을 넘깁니다. 입력 임베딩은 각 ID를 $d_{model}$ 크기의 학습된 밀집 벡터로 변환합니다. 하지만 이건 손으로 설계한 게 아니라 — 랜덤 노이즈에서 시작해서 역전파로 학습되는 평범한 가중치 행렬입니다, 다른 모든 파라미터와 똑같이요. 이 가이드는 The Annotated Transformer를 따라 그 테이블이 어떻게 만들어지고, 공급되고, 학습되는지 정확히 보여줍니다.

01 한 줄 요약

그게 뭔데

학습 가능한 룩업

행렬 $E \in \mathbb{R}^{V \times D}$ — 어휘(vocab) 항목마다 학습 가능한 행 하나. 토큰의 임베딩은 단순히 인덱스로 가져온 해당 행입니다.

어떻게 학습하나

그래디언트 하강

행들은 랜덤(Xavier 초기화)으로 시작합니다. 매 학습 스텝마다, 손실 그래디언트가 배치에 등장한 행들만 살짝 밀어냅니다. 수백만 번 반복하면 기하 구조가 저절로 정리됩니다.

"우리는 학습된 임베딩을 사용한다." 논문이 명확히 밝히고 있습니다: 입력과 출력 토큰 임베딩은 고정된 게 아니라 가중치입니다. 별도의 "임베딩 학습" 단계는 없습니다; 테이블은 어텐션 및 피드포워드 레이어와 함께 단일 언어/번역 손실 아래 공동으로 최적화됩니다.

01·b 어디에 위치하나

텐서 형태 · 임베딩이 특징 축 D를 추가합니다
토큰 ID [B, T] → 임베드 × √D [B, T, D] + PE[:T] [T, D] → 인코더 [B, T, D]

$B$ = 배치, $T$ = 시퀀스 길이, $D = d_{model}$ (기본 모델에서 512). 토크나이저가 ID의 [B, T]를 만들고; 임베딩이 ID마다 D-벡터를 룩업하여 스케일링하고; 그 다음 위치 인코딩이 순서를 더합니다.

이 페이지 사용법: 탭 01 → 07을 순서대로 보세요. 탭 03은 테이블과 룩업을 그려줍니다; 탭 04가 핵심 — 순전파 → 손실 → 역전파 사이클, "학습 스텝" 라이브 샌드박스 포함; 탭 05는 의미가 나타나는 모습을 보여줍니다; 탭 07은 점수를 매깁니다. ?를 눌러 단축키를 확인하세요.
Embeddings 모듈

토큰마다 학습 가능한 행 하나

Annotated Transformer에서 임베딩은 nn.Embedding을 감싸는 작은 모듈입니다. 순전파는 행 룩업 후 $\sqrt{d_{model}}$로 스케일링합니다. 여기서 학습하는 건 가중치 행렬뿐입니다.

Embeddings 클래스 · 원문 그대로 
class Embeddings(nn.Module):
    def __init__(self, d_model, vocab):
        super(Embeddings, self).__init__()
        self.lut = nn.Embedding(vocab, d_model)   # 가중치: [V, D]
        self.d_model = d_model
    def forward(self, x):
        return self.lut(x) * math.sqrt(self.d_model)
각 기호가 의미하는 것
  • vocab · V  어휘 크기 — 행의 수. (nn.Embedding의 첫 번째 인자.)
  • d_model · D  임베딩 너비 — 기본 Transformer에서 512.
  • lut  학습 가능한 가중치 행렬 [V, D] — 말 그대로 "룩업 테이블".
  • x  형태 [B, T]의 정수 토큰 ID; 출력은 [B, T, D].
왜 × √d_model? 논문에는 "임베딩 레이어에서, 그 가중치에 √d_model을 곱한다."고 적혀 있습니다. 이는 임베딩의 크기를 키워서, 바로 다음에 더해지는 단위 스케일의 위치 인코딩과 비슷한 수준이 되게 합니다 — 그렇지 않으면 위치 신호가 내용을 압도합니다. D = 512에서 그 계수는 √512 ≈ 22.6입니다.
종종 가장 큰 레이어입니다. 파라미터 수는 정확히 V × D입니다. D = 512에 32k vocab이면 ≈ 16.4M 가중치 — 이 테이블 하나에 전부, 모두 그래디언트 하강으로 학습됩니다.

핵심 · 행들이 학습되는 방법

테이블 학습

임베딩에는 특별한 학습 절차가 없습니다 — 모델의 일반적인 루프를 그냥 탑니다. 배치가 순전파로 손실에 도달하고; loss.backward()가 그래디언트를 모든 가중치에 역전파하는데, 룩업된 임베딩 행들을 포함해서; 옵티마이저가 스텝합니다. 중요한 건, 배치에 등장한 행들만 그래디언트를 받는다는 점 — 업데이트는 희소합니다.

04·b 왜 업데이트가 희소한가

룩업은 원-핫 행렬곱과 같아서, $e_{id} = \mathbf{1}_{id}^{\top} E$이므로, 테이블에 대한 그래디언트는 행 id를 제외한 모든 곳에서 0입니다. 룩업된 각 토큰은 자기 행에 들어오는 그래디언트를 받는데, 배치 내 등장 횟수만큼 합산됩니다.

$$\frac{\partial L}{\partial E_{j}} \;=\; \sum_{t\,:\,ids_t = j} \frac{\partial L}{\partial e_{t}} \qquad\Longrightarrow\qquad \frac{\partial L}{\partial E_{j}} = 0 \;\text{ for unused rows } j$$
희소 업데이트 샌드박스 · 배치 행들만 이동

이 배치는 짧은 문장 하나입니다. 그 토큰들(행 ID 포함)만 그래디언트를 받습니다. 학습 스텝을 반복해서 누르세요: 하이라이트된 행들이 눈에 띄게 이동하는 동안 고정된 행들은 절대 바뀌지 않습니다 — 희소 임베딩 그래디언트를 직접 보여줍니다.

−1+1 윤곽선 = 배치에 포함 (∇ 받음) · 흐림 = 고정

실제 학습은 거대한 코퍼스에서 수백만 스텝 동안 이걸 반복합니다. 빈번한 토큰(예: the)은 끊임없이 업데이트되고; 희귀 토큰은 배치에 거의 등장하지 않아 천천히 이동합니다.

04·c 이를 구동하는 손실 & 옵티마이저

레이블 스무딩 손실 · backward · step 
# 타깃 분포를 스무딩: 실제 토큰에 confidence, 나머지에 ε 분산
criterion = LabelSmoothing(size=V, padding_idx=0, smoothing=0.1)   # 내부에 KLDivLoss

def step(out, targets, norm):
    log_probs = generator(out)                 # Linear → log_softmax  [.., V]
    loss = criterion(log_probs, targets) / norm
    loss.backward()                          # ∇가 E의 사용된 행들로 흐름
    opt.step()                                # Noam 스케줄 Adam 업데이트
    opt.optimizer.zero_grad()                 # 다음 배치를 위해 그래드 초기화
    return loss
$$lrate = d_{model}^{-0.5}\cdot \min\!\big(step^{-0.5},\; step\cdot warmup^{-1.5}\big)$$

학습이 만들어내는 것

의미가 기하학이 된다

충분한 스텝 후, 비슷한 역할을 하는 토큰들은 비슷한 행 쪽으로 밀려납니다 — 그게 예측을 잘 하는 데 유리하기 때문입니다. 이제 두 행 사이의 코사인 유사도는 모델이 얼마나 비슷하다고 생각하는지를 측정하며, 방향도 의미를 갖게 됩니다(유명한 king − man + woman ≈ queen). 쿼리 토큰을 선택하면 이웃들이 빛납니다.

학습된 장난감 임베딩 공간 · 쿼리 토큰까지의 코사인 유사도

각 점은 토큰 벡터(2-D로 투영됨). 선택된 쿼리가 빛나고; 선이 다른 것들에 뻗어나가는데, 더 유사할수록 더 밝습니다. 순위 목록은 코사인 유사도, 높은 것부터.

하나의 행렬, 세 가지 역할

공유된 가중치

논문은 한 발 더 나아갑니다: 입력 임베딩, 출력(타깃) 임베딩, 그리고 로짓을 생성하는 소프트맥스 전 선형 레이어가 모두 같은 가중치 행렬을 공유합니다. 하나를 학습하면 셋 다 학습됩니다 — 파라미터가 줄고, 입력/출력 기하학이 일관되게 유지됩니다.

원문 그대로: "두 임베딩 레이어와 소프트맥스 전 선형 변환 사이에 같은 가중치 행렬을 공유한다." 로짓은 h @ E.T — 말 그대로 "이 히든 스테이트가 각 토큰의 행 방향으로 얼마나 향하는가?"
출력 측 · Generator 
class Generator(nn.Module):
    # "표준 선형 + 소프트맥스 생성 단계를 정의한다."
    def __init__(self, d_model, vocab):
        super(Generator, self).__init__()
        self.proj = nn.Linear(d_model, vocab)   # 가중치: [V, D] — E에 타이
    def forward(self, x):
        return F.log_softmax(self.proj(x), dim=-1)

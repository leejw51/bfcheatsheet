llama.cpp — LLM을 로컬에서, C/C++로 실행

llama.cpp는 LLaMA 계열 모델(Llama 1/2/3, Mistral, Qwen, Gemma, Phi, DeepSeek 등)을 위한 고성능 추론 엔진으로, 순수 C/C++로 작성되었고 런타임에 Python 의존성이 전혀 없습니다. CPU, Apple Silicon (Metal), NVIDIA (CUDA), AMD (ROCm), Vulkan을 지원하는 모든 GPU에서 실행됩니다. 모델은 단일 .gguf 파일로 존재합니다. 이 페이지를 끝낼 때쯤이면 빌드, 양자화, 실행, 서빙 명령어를 직접 타이핑해 본 상태가 됩니다.

왜 llama.cpp인가?
의존성 제로의 C++. 1초 안에 부팅. 양자화된 모델(Q4_K_M)은 7B 모델을 약 14 GB(fp16)에서 약 4 GB로 줄여줍니다. 노트북, 라즈베리 파이, M 시리즈 맥, 24 GB GPU에서 모두 같은 코드 경로로 동작합니다. 모두가 따라 만드는 표준 구현체입니다.
GGUF가 뭔가?
GGUF (GPT-Generated Unified Format)는 가중치 + 토크나이저 + 채팅 템플릿 + 메타데이터를 묶은 단일 파일 바이너리입니다. 메모리 매핑 가능, 버전 관리되며, 로컬 LLM의 사실상 표준 교환 포맷입니다. 이전의 GGML 포맷을 대체합니다.

    -p "Why is the sky blue?" -n 256 -ngl 99

./build/bin/llama-server -m Meta-Llama-3.1-8B-Instruct-Q4_K_M.gguf \

GGUF 및 양자화

양자화는 가중치를 더 적은 비트로 저장해 모델을 줄입니다. 가중치당 정밀도는 떨어지지만, 더 똑똑한 블록 단위 방식(_K_M 계열)이 품질을 놀라울 만큼 잘 유지합니다. 아래 슬라이더로 크기/품질 트레이드오프를 직접 느껴 보세요.

llama-cli — 일회성 및 채팅

직접 생성을 위한 프론트엔드. 프롬프트를 넣으면 토큰을 받습니다. 대화 모드는 -cnv를 추가하거나, stdin으로 파이프하세요. 아래 모든 플래그는 llama-server에서도 동일하게 동작합니다.

가장 많이 쓰는 플래그

플래그 치트시트
-m .gguf 모델 경로
-p 프롬프트 문자열
-f 파일에서 프롬프트 읽기
-n n_predict — 최대 새 토큰 수 (-1 = ∞)
-c 컨텍스트 크기 (예: 4096, 8192)
-b 프롬프트 배치 크기 (기본 2048)
-cnv 대화 / 채팅 모드
-sys 시스템 프롬프트
-ngl GPU에 오프로드할 레이어 수 (99 = 전부)
-t CPU 스레드
--temp 샘플링 온도
--top-k top-k 샘플링
--top-p nucleus 샘플링
--repeat-penalty 반복 방지
--mlock 가중치를 RAM에 락
--seed RNG 시드 (-1 = 랜덤)
정석 생성
llama-cli \
    -m models/Meta-Llama-3.1-8B-Instruct-Q4_K_M.gguf \
    -p "Explain backprop in 2 sentences." \
    -n 256 \
    -c 4096 \
    -ngl 99 \
    --temp 0.7 --top-p 0.9
인터랙티브 채팅 (Llama-3 채팅 템플릿 자동 인식)
llama-cli -m model.gguf -cnv \
    -sys "You are a terse assistant." \
    -ngl 99 -c 8192

llama-server — OpenAI 호환 HTTP

llama-server는 같은 모델을 HTTP로 노출하며, OpenAI API의 일부를 따릅니다. 많은 SDK에서 드롭인 교체 가능. /에 내장 WebUI 포함.

서버 시작
llama-server \
    -m models/Meta-Llama-3.1-8B-Instruct-Q4_K_M.gguf \
    --host 0.0.0.0 --port 8080 \
    -c 8192 -ngl 99 \
    --chat-template llama3 \
    --api-key sk-local # 선택적 인증

엔드포인트 (OpenAI 호환)

chat — /v1/chat/completions
{ messages: [...] }를 받습니다. 서버는 GGUF에 박힌 채팅 템플릿을 자동 적용합니다. "stream": true로 스트리밍 지원.
completion — /v1/completions
원시 텍스트 보완. { prompt: "..." }. 채팅 템플릿을 완전히 건너뜁니다.
embeddings — /v1/embeddings
임베딩 모델용 — --embedding으로 시작하세요. OpenAI 클라이언트와 호환되는 벡터를 반환합니다.
네이티브 — /completion, /tokenize, /health
llama.cpp 네이티브 엔드포인트, 추가 기능 포함: 그래마, JSON 스키마, 슬롯 관리. WebUI는 http://localhost:8080에서 확인.

curl http://localhost:8080/v1/chat/completions \

        "messages": [
            {"role": "system", "content": "You are concise."},
            {"role": "user",   "content": "What is GGUF?"}
        ],
        "temperature": 0.7,
        "max_tokens": 200
    }'
python (openai SDK)
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:8080/v1",
    api_key="sk-local",        # --api-key가 없으면 아무거나 동작

resp = client.chat.completions.create(
    model="local",             # 무시됨, 모델은 하나만 로드됨

        {"role": "user",   "content": "Why is the sky blue?"},
    ],
    stream=True,

for chunk in resp:
    print(chunk.choices[0].delta.content or "", end="", flush=True)

모델이 들어갈까? 두 가지 비용이 있습니다 — 가중치(로드 시 고정)와 KV 캐시(컨텍스트 길이에 따라 증가). 파라미터 + 양자화로 가중치 RAM을, 컨텍스트 + KV 캐시 타입으로 런타임 RAM을 확인하세요.

샘플링 파라미터

모델이 로짓을 만든 뒤, 샘플러가 그것을 단일 토큰으로 좁힙니다. 기본값은 합리적이며, 아래 노브들이 튜닝의 99%를 커버합니다.

temperature — --temp
로짓을 T로 나눕니다. 0 = greedy / 결정적; 0.7 ≈ 창의적 채팅; 1.0+ = 거칠어짐. 코드 생성: 0.0–0.3. 스토리: 0.7–1.0.
top-k — --top-k
확률 상위 K개만 유지. 40이 클래식 기본값, 0은 비활성.
top-p (nucleus) — --top-p
누적 확률이 p 이상이 되는 가장 작은 토큰 집합을 유지. 최신 기본값: 0.9–0.95. 1.0으로 비활성.
repeat-penalty — --repeat-penalty
이미 본 토큰에 패널티. 1.0 = 끔. 1.1–1.2가 대부분의 루프를 해결합니다. 너무 높으면 부자연스러움.
min-p — --min-p
min_p × max_prob 미만의 토큰을 제거. top-p보다 새로운 방식, 선호하는 사람들이 있습니다.
seed — --seed
RNG 시드. -1 = 랜덤. 시드 고정으로 재현 가능한 실행을 만드세요.

전체 라이프사이클: 양자화된 Llama 3.1 8B를 받고, llama-server를 빌드해 시작한 뒤, HTTP로 채팅. 빠진 부분을 채우고 실행을 클릭해 시뮬레이션된 엔드투엔드 데모를 보세요.

프로그램 완성하기

9개 빈칸. 모두 초록이 되면 엔드투엔드 실행을 누르세요. 

# 1. Apple Silicon GPU로 llama.cpp 빌드
git clone https://github.com/ggml-org/llama.cpp
cd llama.cpp
cmake -B build -DGGML_=ON
cmake --build build --config Release -j

# 2. Llama 3.1 8B Instruct의 Q4_K_M 양자화 다운로드
huggingface-cli download bartowski/Meta-Llama-3.1-8B-Instruct-GGUF \
    Meta-Llama-3.1-8B-Instruct-.gguf \
    --local-dir models/

# 3. OpenAI 호환 서버 시작
./build/bin/llama- \
    -m models/Meta-Llama-3.1-8B-Instruct-Q4_K_M.gguf \
    --host 0.0.0.0 --port  \
    -c 8192 - 99

# 4. 다른 터미널에서 채팅 (curl)
curl http://localhost:8080/v1/chat/ \
    -H "Content-Type: application/json" \
    -d '{"messages":[{"role":"user","content":"Hi!"}]}'

# 5. 또는 Python에서 (인프로세스, 서버 없이)
from llama_cpp import 
llm = Llama(model_path="models/Meta-Llama-3.1-8B-Instruct-Q4_K_M.gguf",
            n_ctx=8192, n_gpu_layers=)
out = llm("Q: largest planet?\nA:", max_tokens=)

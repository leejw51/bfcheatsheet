multi­agent.

Claude Code를 병렬로 작동시키는 네 가지 방법.
파일: multi-agent-guide.html
런타임: claude-code/cli
~/repo — claude --multi-agent

Claude 창 하나도 이미 작은 팀입니다. 네 개의 창은 스튜디오입니다. 이 가이드는 그 팀이 취할 수 있는 네 가지 형태와 — 각 형태가 언제 올바른 형태인지 보여줍니다.

인원수가 아닌 조율 방식으로 선택하세요. 사이드 작업 → 서브 에이전트. 독립적 → 에이전트 뷰. 조율됨 → 팀. 완전 격리 → 터미널당 워크트리.

접근법 01 · 세션 내 위임

서브 에이전트.

자체 컨텍스트 윈도우에서 실행되어 집중된 작업을 마치고 호출 세션에 요약을 반환하는 전문 작업자. 메인 스레드를 깨끗하게 유지하기 위해 사용하세요 — 조사, 검증, 탐색.

“요점은 또 다른 에이전트가 아닙니다. 요점은 메인 대화를 어수선하지 않게 유지하는 것입니다.”

.claude/agents/ 아래에 Markdown 파일로 정의합니다. 각각 자체 시스템 프롬프트, 도구 허용 목록, 그리고 (선택적으로) 더 작은 모델을 가지므로 작업이 저렴합니다.

컨텍스트 · 격리됨 병렬 · 예 파일 충돌 · 공유 cwd 적합 · 조사, 감사
.claude/agents/researcher.md 
---
name: Researcher
description: Finds prior art & cites sources
model: haiku
tools:
  - bash
  - read
  - webfetch
---

# You are an expert researcher.
# Return citations as a bullet list.
prompt · main session 
$ /agents
  → manage definitions

$ Have @Researcher dig up the rate-limit RFC, then summarize.
→ spawning Researcher in isolated context…
← "Found 3 sources, see notes/rfc-rate-limit.md"
접근법 02 · 백그라운드 대시보드

에이전트 뷰.

독립적인 백그라운드 세션의 대시보드. 프롬프트를 디스패치하고 자리를 떠난 다음 나중에 출력을 들여다보세요. 각 행은 자체 프로세스이며 터미널을 닫아도 살아남습니다.

“쏘고 나중에 확인. 5개의 작은 작업과 하나의 두뇌가 있을 때 유용합니다.”

claude agents로 엽니다. 대시보드 안에서: Space로 엿보기, Enter로 부착, ←로 분리.

컨텍스트 · 세션당 조율 · 당신 지속성 · 닫아도 유지 적합 · 병렬 잡일
shell 
# dispatch a background session
$ claude --bg "investigate the flaky e2e test"

# dispatch with a custom agent
$ claude --agent code-reviewer --bg "review PR #1234"

# open the dashboard
$ claude agents

┌─ active ───────────────────────────┐
│ ▸ flaky-e2e        running   2m 14s │
│ ▸ pr-1234-review   running     38s │
│ ▸ docs-cleanup     done      11m 02s │
└────────────────────────────────────┘
접근법 03 · 실험적 · 조율됨

에이전트 팀.

공유된 작업 목록, 에이전트 간 메시지, 작업을 관리하는 리더와 함께 같은 목표를 향해 작업하는 여러 세션. 작업자 풀보다는 스튜디오에 가깝습니다.

“보안용 리뷰어 한 명, 성능용 한 명, 테스트용 한 명 — 그들은 서로 대화합니다.”

실험적이며 기본적으로 비활성화되어 있습니다. settings.json에서 활성화한 다음, Claude에게 목표를 위한 팀을 꾸리도록 요청하세요.

컨텍스트 · 팀원당 조율 · 내장 상태 · 실험적 적합 · 다축 리뷰
settings.json 
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
prompt 
$ Spin up a team to review PR #142.
  Three reviewers, working in parallel:
    1. security  — auth, input, secrets
    2. perf     — hot paths, allocs, queries
    3. tests    — coverage, flakiness, mocks
  Lead consolidates & posts one review.

→ team:r142 spawned with 3 teammates
→ shared task list created (12 items)
← consolidated review posted to #142
접근법 04 · 단순한 방식 · 강력 추천

여러 터미널.

다른 터미널 탭을 엽니다. claude를 실행합니다. 끝 — 두 개의 에이전트가 생겼습니다. 이를 git 워크트리와 결합하면 진정한 격리를 얻습니다: 각 에이전트가 자체 브랜치와 작업 트리에 있어 편집 충돌이 없습니다.

“가장 깔끔한 패턴이 가장 단순한 패턴이기도 합니다: 더 많은 탭.”

작업이 장시간 실행되고, 관련 없으며, 각 에이전트가 무엇을 하는지 완전히 파악하고 싶을 때 가장 좋습니다. 완료되면 PR로 병합하세요.

설정 · 없음 격리 · 완전 (워크트리) 가시성 · 전체 적합 · 병렬 기능
shell · worktree-per-agent 
# from main repo
$ git worktree add ../repo-feat-a -b feat-a
$ git worktree add ../repo-feat-b -b feat-b

# terminal 1 — feature A
$ cd ../repo-feat-a && claude

# terminal 2 — feature B
$ cd ../repo-feat-b && claude

# merge later via PR ─ no conflicts
tmux · split-pane variant 
$ tmux new -s squad
  ctrl-b "       split horizontal
  ctrl-b %       split vertical
  ctrl-b o       cycle panes

POW! Make a Comic with Nano Banana

"Nano Banana" is the affectionate nickname for Google's Gemini 2.5 Flash Image model — a multimodal image generator that takes text and reference images, edits in place, and (most importantly for us) keeps a character looking like the same character across panels. That last property is what makes it usable for comics in the first place.

what nano banana actually is
A fast, multimodal image model from Google DeepMind. You can use it through Google AI Studio Gemini API Gemini app. The official model id is gemini-2.5-flash-image. It accepts both text and one or more reference images in the same prompt.
why it's good for comics
Three reasons:
  • Character consistency — feed a reference, the same face survives shot changes.
  • In-place edits — "keep everything, change her expression to surprised" works.
  • Speed — fast enough to iterate panel-by-panel.

The whole pipeline at a glance

1script

Write the dialogue and beats. Decide panel count (3-strip vs full page).

2character sheet

Generate a model-sheet image: front / side / back / expressions. This is your consistency anchor.

3panels

For each panel: [ref image] + prompt. Always re-supply the reference.

4iterate

Edit in place: "same panel, change weather to rain", "change pose", "fix hands".

5letter

Add speech bubbles in Figma / Canva / Photoshop. Nano Banana renders text but post-lettering reads cleaner.

6stitch

Drop panels into a page layout, gutters, page numbers. Done.

PLAN! Script Before You Generate

The biggest mistake first-timers make: open Gemini, type "draw me a comic". You'll get one panel of generic art. Comics are a sequence, and the model can't sequence for you — you have to plan the beats first.

1. Pick your format

3-panel strip

Setup → turn → punchline. Newspaper-style. Easiest to iterate on.

1-page (4–6 panels)

One scene. Mix shot sizes — wide → close-up → reaction.

Multi-page

Real story. Plan a beat sheet first, then page layout, then panels.

2. Write a beat sheet

For each panel, jot down: shot type · who's there · action · dialogue · mood.

example: 3-panel strip — "Aria finds the cat"
# Panel 1 — establishing
shot:     wide
who:      Aria
where:    rainy alley at night, neon signs
action:   walking, looking at phone
dialogue: "Where did it go..."
mood:     noir, lonely

# Panel 2 — discovery
shot:     medium
who:      Aria + cat
where:    behind dumpster, same alley
action:   crouching, surprised
dialogue: "There you are!"
mood:     tender, soft glow

# Panel 3 — reaction / punchline
shot:     close-up
who:      cat
where:    cradled in hands
action:   cat staring deadpan
dialogue: "meow."
mood:     comedic, warm

3. Decide your style anchor

Pick one short style phrase you'll repeat in every panel prompt. This is one of the cheapest consistency tricks.

manga

"black-and-white manga, screentone shading, ink linework"

american comic

"american comic book art, bold inks, flat color, halftone"

watercolor

"watercolor illustration, loose lines, soft pastels"

pixel

"16-bit pixel art, limited palette, scanlines"

Tip: if you change the style phrase mid-comic, the look will drift. Lock it in before you start, and copy-paste it verbatim into every panel prompt.

SHEET! The Consistency Anchor

The single most important file in your project is the character sheet — also called a model sheet. It's one image showing the character from multiple angles and expressions. You'll attach this image to every panel prompt.

Generate the sheet first

Send Nano Banana a prompt like the one below, with no reference image yet:

prompt — generate character sheet
Character sheet for "Aria", a young detective in a tan trenchcoat,
short black bob, green eyes, age 24. Show four full-body poses on a
plain neutral grey background:
  - front view, neutral expression
  - 3/4 view, smiling
  - side profile
  - back view
Plus three head shots in the corner: surprised, angry, laughing.
Style: american comic book art, bold inks, flat color.
Label each pose. No background scene.

What it looks like

A model sheet has four standardized views plus a row of expressions:

FRONT
3/4
SIDE
BACK
happy surprised angry tired

The sheet doesn't have to be pretty. It has to be recognizable from any angle.

Why a model sheet beats a single portrait

single portrait (bad)
The model sees one angle and one expression. When you ask for "side view, angry", it has to invent both — and small inventions accumulate into a different person.
model sheet (good)
The model has explicit reference for every angle and expression you'll need. New panels become interpolation instead of invention. Identity holds.
Pro move: after the first sheet, pick one panel where the character looks perfect and crop a clean head shot. Add that head shot as a second reference image when generating close-ups. Two refs is better than one.

Naming

Give the character a name and use it in every prompt — "Aria", not "the woman". Named entities seem to lock in identity better than descriptions alone.

ZAP! The Panel Prompt Pattern

Every panel prompt has the same five-part shape. Once you internalize it, you can crank out panels at conversational speed.

the formula
[character ref image]
[shot type] of [character name] [action] in [setting].
[mood / lighting].
[style anchor].
[panel-specific instructions]

Worked example — Panel 1 of "Aria finds the cat"

prompt + attachments
# attached: aria_charsheet.png

Wide establishing shot of Aria walking down a rainy alley at night,
holding her phone, looking confused. Neon signs reflecting in
puddles. Noir lighting, deep shadows, single rim light.

Style: american comic book art, bold inks, flat color, halftone
shading. Square aspect ratio.

Keep Aria's face, hair, and trenchcoat exactly as in the reference.

Shot-type vocabulary

wide / establishing

Whole scene, character small. Sets the location.

medium

Knees-up. Daily comic default.

close-up

Head and shoulders. For emotion beats.

extreme close

Just the eyes. Dramatic punctuation.

over-the-shoulder

Looking past one character at another. Conversation.

low angle

Camera from below. Makes subject feel powerful.

high angle

From above. Subject feels small / vulnerable.

POV

Through character's eyes. Rare but punchy.

Mood vocabulary

noir

Hard shadows, single light source, blue/teal tint, rain.

sunny / warm

Golden hour, soft shadows, yellow cast.

dramatic

Rim light, strong contrast, particles in air.

cozy

Warm interior lighting, soft palette, no hard edges.

Five consistency tricks (do all of them)

1attach the ref every time

Don't trust prior conversation memory. Re-attach the character sheet on every panel.

2name the character

Use the same proper name across prompts. "Aria does X", not "she does X".

3lock the style phrase

Identical wording every panel. "american comic book art, bold inks, flat color, halftone".

4describe what stays

"Keep face, hair, outfit exactly as reference." Spell out what shouldn't drift.

5fix the aspect ratio

Same aspect across panels — square or 4:3. Mixed ratios make a strip feel sloppy.

6seed when possible

If your interface exposes a seed, pin it for a sequence. Cheaper than re-prompting.

FIX! Iterate, Edit, Letter

First-pass generations are rarely perfect. The Nano Banana superpower is in-place editing — you can keep most of the image and surgically change one thing.

The edit instruction pattern

template
Keep everything else exactly, change X to Y.

Examples of edits that work

expression

Keep the panel exactly, change Aria's expression to surprised, mouth slightly open.

pose

Same scene, change Aria's pose so she's crouching down to the cat instead of standing.

weather / time

Same composition, change rain to light snow, time to dusk instead of night.

add a character

Same panel, add a small black cat sitting next to the dumpster.

fix hands

Keep the rest, redraw both hands holding the phone correctly. Five fingers each.

simplify

Same panel, remove the background characters. Only Aria visible.

When to redo vs. when to edit

edit in place if...
  • composition is right, one element is wrong
  • the character looks correct (don't risk losing identity)
  • you only need to swap mood / time / weather
redo from scratch if...
  • shot type is wrong (close-up instead of wide)
  • character identity has drifted
  • art style mismatches other panels

Lettering — how to handle dialogue

Nano Banana can render speech bubbles and short text in-image, and it's gotten good enough that one-off panels are fine. For a polished comic, generate the panels without text and add bubbles in post.

Agenerate clean

Add "no text, no speech bubbles, no captions" to your panel prompt. Get clean art.

Bletter in figma / canva

Drop bubbles, captions, sound effects on top. Use a comic font (Bangers, Anime Ace, BadaBoom).

Cstitch the page

Arrange panels in a 2×2 / 2×3 grid. White gutters between. Export PNG / PDF.

The lettered output (mock)

Onomatopoeia tip: sound effects look great as a hand-lettered overlay rather than baked into the image. Try POW! CRASH! over a finished panel.

Sequencing — chain three panels

Once you've nailed Panel 1, the next two are the same prompt with different shot / action / mood. The character, setting, style, and ref images stay locked.

three-panel prompt template
# attached: aria_charsheet.png, aria_headshot.png

Generate three comic panels in sequence, same character (Aria),
same alley setting, same style.

Style anchor (use exactly this in all three):
  american comic book art, bold inks, flat color, halftone shading.

Panel 1 — wide establishing shot of Aria walking down a rainy
  alley at night, looking at her phone. Noir lighting.

Panel 2 — medium shot of Aria crouching down behind a dumpster,
  surprised expression, reaching out to a small black cat.

Panel 3 — close-up of the cat staring deadpan at the camera,
  cradled in Aria's hands. Warm soft light.

Aspect ratio for all: square. No text or speech bubbles in the
output — leave space for lettering.

PostgreSQL CLI — 손에 박힐 때까지 psql 드릴

psql은 PostgreSQL의 공식 터미널 클라이언트입니다. 모든 서버에 함께 제공되며, 스크립트로 돌릴 수 있고, 매우 빠르며, 어떤 GUI도 따라올 수 없는 백슬래시 메타 명령이 가득합니다. 이 페이지를 끝낼 때쯤이면 접속 플래그, 메타 명령, SQL, dump/restore, 유지보수 한 줄 명령, 그리고 작은 엔드투엔드 프로젝트까지 직접 타이핑해 본 상태가 됩니다.

psql이 뭔가?
PostgreSQL 와이어 프로토콜로 통신하는 터미널 프로그램입니다. SQL은 물론 백슬래시 명령도 실행합니다 — 예: \dt(테이블 목록), \d users(테이블 설명). 탭 자동 완성, 히스토리, -c / -f로 스크립팅, \copy로 대량 로드.
왜 GUI가 아닌가?
GUI는 실제 명령을 숨기고, 스크립트가 안 되며, SSH 너머의 원격 서버에서 못 돌고, 큰 결과 세트에서 깨지기 때문입니다. \d, \dt+, \watch가 손가락에 박히면 지구상 어떤 UI보다 빨라집니다.
psql의 친구들
백업용 pg_dump / pg_restore, 헬스체크용 pg_isready, 얇은 래퍼인 createdb / dropdb, vacuumdb, reindexdb. 모두 같은 접속 플래그를 씁니다.
관습
기본 포트 5432, 기본 DB와 사용자는 보통 OS 사용자와 같습니다. psql 안에서: \?는 메타 명령 목록, \h SELECT는 SQL 문법, \q는 종료.

외워둘 psql 치트시트

필수
# 접속
psql -h db.example.com -p 5432 -U app -d shop

# psql 안에서
\l            # 데이터베이스 목록
\c shop       # "shop" DB로 접속
\dt           # 테이블 목록
\d users      # "users" 테이블 설명
\df           # 함수 목록
\du           # 롤 목록
\timing on    # 모든 쿼리에 실행 시간 표시
\watch 2      # 마지막 쿼리를 2초마다 재실행
\copy users TO 'users.csv' CSV HEADER
\q            # 종료
이 페이지 사용법: 섹션을 탭으로 넘기고, 빈칸에 직접 타이핑하세요(Enter로 확인, Tab으로 이동). 연속 정답을 노리세요. 종이 폭죽이 터지면 섹션을 끝낸 겁니다.

준비되셨나요? 위쪽의 2. 접속을 클릭하세요. 모든 세션이 시작되는 곳에서 시작합니다 — 들어가기.

접속 — psql에 들어가기

접속하는 여섯 가지 방법. 상황에 맞는 걸 고르세요.

모든 접속 스타일
# 1. 기본값 — $PGUSER / $PGDATABASE / unix 소켓 사용
psql

# 2. 명시적 플래그
psql -h db.example.com -p 5432 -U alice -d shop

# 3. URI 형태 (환경 변수 / CI에 좋음)
psql postgresql://alice:secret@db.example.com:5432/shop

# 4. 일회성 쿼리, 인터랙티브 셸 없이
psql -h host -U alice -d shop -c "SELECT now();"

# 5. .sql 스크립트 실행
psql -h host -U alice -d shop -f migrate.sql

# 6. 환경 변수와 함께
PGHOST=db PGUSER=alice PGDATABASE=shop psql

매번 비밀번호 입력 없이 인증

~/.pgpass
# ~/.pgpass — 한 줄에 (host, port, db, user, password)
# 모든 필드에 와일드카드 허용
db.example.com:5432:shop:alice:s3cret
*:*:*:postgres:rootpw

# 반드시 chmod 600 — 아니면 psql이 무시함
chmod 600 ~/.pgpass
psql 안에서 본인 비밀번호 설정은 \password로. 클라이언트 측에서 해시되므로 평문이 터미널을 벗어나지 않아 훨씬 안전합니다 — 서버 로그에 남는 ALTER USER ... PASSWORD '...'보다.

접속 플래그 타이핑

1 호스트 db.example.com, 포트 5432, 사용자 alice, DB shop으로 접속. 호스트 플래그를 채우세요.
psql  db.example.com -p 5432 -U alice -d shop
2 같은 접속, 사용자 플래그를 채우세요.
psql -h db.example.com -p 5432  alice -d shop
3 URI 형태 — Postgres 기본 포트를 입력하세요.
psql postgresql://user:pass@host:/dbname
4 psql 종료.
postgres=# \
5 psql 안에서 본인 비밀번호를 안전하게 설정.
postgres=# \
6 psql이 ~/.pgpass를 읽기 위해 필요한 권한은? (8진수)
chmod  ~/.pgpass
7 SQL 문법의 내장 도움말 표시.
postgres=# \ SELECT
계속 타이핑하세요: \?는 모든 메타 명령 목록. \h는 모든 SQL 문 목록. 둘이 가장 좋은 친구입니다.

메타 명령 치트 카드

백슬래시 메타 명령은 특별합니다 — 서버로 보내지 않고 psql이 로컬에서 해석합니다. 상세 출력은 +를 붙이고, 설명은 \d를 앞에 붙입니다.

살펴보기 — 스키마, 타입, 인덱스, 플랜

접속한 뒤 가장 먼저 하는 일은 둘러보기입니다. 데이터베이스에 뭐가 있는지 알아내는 방법.

SQL DDL — 실제로 쓰는 타입들

CREATE TABLE — 풍부한 타입
CREATE TABLE users (
    id          UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
    email       TEXT        NOT NULL UNIQUE,
    age         INT         CHECK (age >= 0),
    tags        TEXT[],                                -- ARRAY
    profile     JSONB       NOT NULL DEFAULT '{}',    -- JSONB
    created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- 외래 키
CREATE TABLE orders (
    id       BIGSERIAL PRIMARY KEY,
    user_id  UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    total    NUMERIC(10,2) NOT NULL
);

인덱스 — 대부분의 일을 커버하는 세 가지 종류

인덱스 패턴
-- 1. 평범한 B-tree (기본)
CREATE INDEX idx_orders_user ON orders(user_id);

-- 2. 인덱스를 통한 유니크 제약
CREATE UNIQUE INDEX idx_users_email ON users(lower(email));

-- 3. 부분 — "live" 행만 인덱싱
CREATE INDEX idx_open_orders ON orders(user_id)
    WHERE status = 'open';

-- 4. JSONB / 배열 / 전문 검색을 위한 GIN
CREATE INDEX idx_users_profile ON users USING GIN (profile);
CREATE INDEX idx_users_tags    ON users USING GIN (tags);

EXPLAIN — 쿼리 플랜 읽기

explain analyze
EXPLAIN (ANALYZE, BUFFERS) SELECT * FROM orders WHERE user_id = '...';
--  Index Scan using idx_orders_user on orders  (cost=0.42..8.44 rows=1 width=...)
--    Index Cond: (user_id = '...'::uuid)
--  Planning Time: 0.12 ms
--  Execution Time: 0.04 ms
EXPLAIN 읽기: 위에서 아래로 실행, 아래에서 위로 비용. 큰 테이블에서 Index Scan을 기대했는데 Seq Scan이 나오면 인덱스가 없거나 안 쓰는 겁니다. 디스크 읽기 vs 캐시 히트를 보려면 BUFFERS 옵션을 쓰세요.

UPSERT, CTE, 윈도우 함수 — 좋은 것들

upsert (INSERT … ON CONFLICT)
INSERT INTO users (id, email)
VALUES ('a-uuid', 'a@x')
ON CONFLICT (id) DO UPDATE SET email = EXCLUDED.email;
CTE + 윈도우 함수
WITH ranked AS (
    SELECT user_id, total,
           ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY total DESC) AS rn
    FROM orders
)
SELECT * FROM ranked WHERE rn = 1;   -- 사용자별 최대 주문

유용한 한 줄 쿼리

프로덕션 DBA가 새벽 3시에 실제로 붙여 넣는 쿼리들. 저장해두세요.

상위 5개 큰 테이블

크기 탐정
SELECT
    schemaname || '.' || relname           AS table,
    pg_size_pretty(pg_total_relation_size(relid))  AS total_size,
    pg_size_pretty(pg_relation_size(relid))        AS data_size,
    pg_size_pretty(pg_total_relation_size(relid) - pg_relation_size(relid)) AS idx_size
FROM pg_catalog.pg_statio_user_tables
ORDER BY pg_total_relation_size(relid) DESC
LIMIT 5;

실행 중 쿼리 — 누가 지금 뭘 하고 있나

pg_stat_activity
SELECT pid, usename, application_name, state,
       now() - query_start AS runtime,
       left(query, 100) AS q
FROM pg_stat_activity
WHERE state <> 'idle'
ORDER BY runtime DESC;

폭주 쿼리 취소 / 종료

cancel vs terminate
-- 정중함: 쿼리에 멈추라고 요청 (연결은 유지)
SELECT pg_cancel_backend(12345);

-- 핵폭탄: 연결 전체를 죽임
SELECT pg_terminate_backend(12345);

롤 및 GRANT

읽기 전용 사용자 만들기
CREATE ROLE readonly LOGIN PASSWORD 'change-me';
GRANT CONNECT ON DATABASE shop TO readonly;
GRANT USAGE ON SCHEMA public TO readonly;
GRANT SELECT ON ALL TABLES IN SCHEMA public TO readonly;
ALTER DEFAULT PRIVILEGES IN SCHEMA public
    GRANT SELECT ON TABLES TO readonly;

알아두면 좋은 확장

pg_stat_statements
모든 쿼리의 런타임 + 플랜 통계를 추적. 진지한 DBA가 가장 먼저 설치하는 것. 
CREATE EXTENSION IF NOT EXISTS pg_stat_statements;
pgcrypto
gen_random_uuid(), crypt(), bcrypt 비밀번호 해싱. 
CREATE EXTENSION IF NOT EXISTS pgcrypto;
uuid-ossp
오래된 UUID 생성기 (v1, v3, v4, v5). 최신 PG에서는 pgcrypto의 gen_random_uuid()를 쓰는 것이 좋습니다. 
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
pg_trgm
퍼지 / 유사도 텍스트 검색. GIN 인덱스와 결합하면 LIKE '%foo%' 성능이 폭발적. 
CREATE EXTENSION IF NOT EXISTS pg_trgm;
pgvector
벡터 임베딩 저장; <-> = L2, <=> = 코사인. 네이티브 AI/RAG 지원. 
CREATE EXTENSION IF NOT EXISTS vector;

백업 및 복원 — pg_dump / pg_restore

바이너리 두 개, 플래그 네 개. 마스터하면 데이터가 안전합니다.

네 가지 포맷

플래그포맷언제 쓰나
-Fpplain SQL작은 DB, 사람이 읽기 쉬움, psql -f로 재생
-Fccustom (binary)운영 기본 — 압축됨, 병렬 복원, 테이블 필터
-Fttar비압축 디렉토리 tarball — 드묾
-Fddirectory-j N으로 병렬 덤프

레시피

백업
# 단일 DB, custom 포맷 (권장)
pg_dump -h host -U alice -Fc shop > shop.dump

# 압축된 plain SQL
pg_dump -h host -U alice shop | gzip > shop.sql.gz

# 스키마만 / 데이터만
pg_dump -Fc --schema-only shop > shop_schema.dump
pg_dump -Fc --data-only   shop > shop_data.dump

# 병렬 디렉토리 덤프 (4 작업)
pg_dump -Fd -j 4 -f shop_dir shop

# 클러스터의 모든 데이터베이스 (롤 포함)
pg_dumpall -U postgres > cluster.sql
복원
# custom 포맷 → pg_restore 사용
createdb shop_new
pg_restore -h host -U alice -d shop_new -j 4 shop.dump

# plain SQL → psql -f 사용
psql -h host -U alice -d shop_new -f shop.sql

# custom 덤프 내용 목록 (실제 복원 안 함)
pg_restore -l shop.dump | head

# 한 테이블만 복원
pg_restore -d shop_new -t users shop.dump

유지보수 — PG를 행복하게

vacuum / analyze / reindex
-- 죽은 행 회수 + 플래너 통계 갱신
VACUUM ANALYZE;
VACUUM ANALYZE orders;            -- 한 테이블

-- 공격적: 테이블 락, 다시 씀 (거의 필요 없음)
VACUUM FULL orders;

-- 비대해진 인덱스 재구축
REINDEX TABLE CONCURRENTLY orders;
REINDEX INDEX idx_orders_user;

# 1. 데이터베이스 생성 (셸에서)
createdb -U alice shop

# 2. 접속
psql -U alice -d shop

# 3. 스키마
shop=> CREATE EXTENSION IF NOT EXISTS ;

shop=> CREATE TABLE users (
    id     PRIMARY KEY DEFAULT gen_random_uuid(),
    email TEXT NOT NULL UNIQUE,
    profile  NOT NULL DEFAULT '{}'
);

shop=> CREATE TABLE orders (
    id      BIGSERIAL PRIMARY KEY,
    user_id UUID NOT NULL  users(id) ON DELETE CASCADE,
    total   NUMERIC(10,2) NOT NULL
);

# 4. 삽입 (UPSERT로 멱등하게)
shop=> INSERT INTO users (email) VALUES ('a@x'), ('b@x')
       ON  (email) DO NOTHING;

# 5. 쿼리: 사용자별 최대 주문
shop=> SELECT user_id, total,
              ROW_NUMBER()  (PARTITION BY user_id ORDER BY total DESC) AS rn
       FROM orders;

shop=> \

# 7. 백업 (다시 셸에서)

1. tmux란 무엇인가?

tmux는 터미널 멀티플렉서입니다: 하나의 터미널 윈도우가 여러 독립 셸을 나란히, 탭으로, 그리고 (가장 중요하게) SSH 연결과 분리된 상태로 보유합니다.

세 가지 슈퍼파워:

  • 지속성 — 노트북을 닫고, 내일 다시 연결하면, 두고 간 그대로의 셸을 찾을 수 있습니다
  • 레이아웃 — 윈도우를 페인으로 분할; vim, tail -f, htop을 함께 실행
  • 스크립트 가능 — 한 명령으로 프로젝트 레이아웃 시작, 팀원과 세션 공유, 설정 자동화

2. 심상 모델

tmux는 클라이언트/서버 프로그램입니다. 터미널은 장기 실행 tmux 서버와 통신합니다. 서버가 소유하는 것:

server
└── session "work"
    ├── window 0: editor
    │   ├── pane 0: vim
    │   └── pane 1: shell
    └── window 1: logs
        └── pane 0: tail -f app.log

attach = 터미널이 세션에 연결됩니다. detach = 터미널이 분리되지만 세션은 서버에서 계속 실행됩니다. SSH 연결을 끊는 것 ≠ 세션을 죽이는 것.

3. 프리픽스 키

세션 안의 거의 모든 tmux 명령은 프리픽스 키로 시작합니다. 기본값은 Ctrl+b. 프리픽스를 누르고 떼고 다음 키를 누릅니다.

표기법: C-b "는 Ctrl+b 다음 "(즉 Shift+')를 의미합니다. 많은 사람들이 손쉽게 닿도록 프리픽스를 Ctrl+a로 리매핑합니다.

아래 시뮬레이터를 시도해 보세요 — Ctrl+b를 누르면 표시등이 뜨겁게 빛납니다.

5. 윈도우

윈도우는 탭과 같습니다. 각 세션에는 여러 윈도우가 있고, 한 번에 하나만 활성 상태입니다.

동작바인딩
윈도우 생성C-b c
윈도우 이름 변경C-b ,
다음 윈도우C-b n
이전 윈도우C-b p
마지막(가장 최근) 윈도우C-b l
윈도우 N으로 이동C-b 0..9
윈도우 종료C-b &
대화형 윈도우 목록C-b w

6. 페인

페인은 윈도우를 직사각형으로 분할합니다. tmux 표기법은 악명 높게 헷갈립니다:

  • C-b "는 가로로 분할(위/아래 — 가로 구분선). 기억법: "는 두 개의 점이 위아래로 쌓인 모양.
  • C-b %는 세로로 분할(왼쪽/오른쪽 — 세로 구분선). 기억법: %는 두 박스 사이의 슬래시처럼 보임.
동작바인딩
가로 분할 (위/아래)C-b "
세로 분할 (왼쪽/오른쪽)C-b %
페인 사이 이동C-b ↑/↓/←/→
다음 페인으로 순환C-b o
현재 페인 크기 조정 (누른 채)C-b C-↑/↓/←/→
줌 토글 (전체 화면 페인)C-b z
페인 교환C-b { / C-b }
현재 페인 종료C-b x
페인을 새 윈도우로 분리C-b !
페인 번호 표시C-b q

7. 복사 모드 & 스크롤백

마우스로도 터미널을 스크롤할 수 있지만, tmux의 진짜 방법은 복사 모드입니다. C-b [로 진입하세요. 이제 vi 키로 이동하고, 검색하고, 텍스트를 가져올 수 있습니다.

동작키 (복사 모드 안에서)
복사 모드 진입C-b [
이동h j k l 또는 화살표
앞/뒤로 검색/ / ?
선택 시작Space (vi) 또는 mode-keys vi 상태에서 v
선택 복사Enter (기본) 또는 vi 모드에서 y
붙여넣기C-b ]
복사 모드 종료q

~/.tmux.conf에서 setw -g mode-keys vi로 vi 스타일 설정.

8. 설정 — ~/.tmux.conf

tmux 안에서 C-b :를 누른 뒤 source-file ~/.tmux.conf로 설정을 다시 불러오거나, 단축키로 바인딩하세요.

# 프리픽스를 Ctrl-a로 리매핑
unbind C-b
set -g prefix C-a
bind C-a send-prefix

# 윈도우/페인 인덱스를 1부터 시작 (1..9 키에 더 가까움)
set -g base-index 1
setw -g pane-base-index 1
set -g renumber-windows on

# vim 스타일 페인 이동
bind h select-pane -L
bind j select-pane -D
bind k select-pane -U
bind l select-pane -R

# 현재 경로를 유지하는 합리적인 분할
bind | split-window -h -c "#{pane_current_path}"
bind - split-window -v -c "#{pane_current_path}"

# 마우스 + vi 복사 모드
set -g mouse on
setw -g mode-keys vi
bind r source-file ~/.tmux.conf \; display "config reloaded"

# 더 큰 스크롤백
set -g history-limit 50000

9. 플러그인 (TPM)

TPM = Tmux Plugin Manager. ~/.tmux.conf에 추가:

set -g @plugin 'tmux-plugins/tpm'
set -g @plugin 'tmux-plugins/tmux-sensible'
set -g @plugin 'tmux-plugins/tmux-resurrect'   # 세션 저장/복원
set -g @plugin 'tmux-plugins/tmux-continuum'   # 부팅 시 자동 복원
run '~/.tmux/plugins/tpm/tpm'

tmux 안에서: C-b I로 설치, C-b U로 업데이트. tmux-resurrect는 전체 세션 트리를 체크포인트할 수 있고, tmux-continuum은 15분마다 자동 저장하고 부팅 시 다시 불러옵니다 — 노트북 재부팅 = 잃은 작업 없음.

시뮬레이터 실시간 키 코드

터미널을 클릭하고 실제 바인딩을 시도하세요. 먼저 Ctrl+b를 누르면 프리픽스 표시등이 빛납니다. 그 다음 키를 누르세요. 트리거한 고유한 코드마다 챕터 표에 ✓가 찍히고 XP를 받습니다.

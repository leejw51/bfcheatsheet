Chapter 01 · 모든 트랜스포머의 혀

문자열 입력,
정수 출력.

모델은 문자를 읽지 않습니다. 단어도 읽지 않습니다. 모델이 읽는 것은 정수입니다. 토크나이저는 원시 텍스트를 트랜스포머가 실제로 소비하는 정수 언어로 번역하고, 모델이 생성한 정수를 다시 사람이 읽을 수 있는 텍스트로 되돌리는 작은 프로그램입니다. 이게 없으면 GPT, BERT, LLaMA, Claude — 전부 침묵합니다.

한 문장으로
토크나이저는 텍스트를 고정된 어휘(vocabulary) 안의 토큰으로 쪼개고 그 정수 ID를 내보내는 함수 $T : \text{string} \to \mathbb{Z}^n$ 입니다. 디토크나이저(detokenizer) $T^{-1}$ 는 그 반대를 합니다. 
"hello world"
       │
       ▼  encode
[ 31373, 995 ]      # 두 개의 토큰, GPT-2 vocab
       │
       ▼  decode
"hello world"
왜 그냥 단어를 쓰면 안 될까?
  • 어휘 폭발. 영어만 해도 단어 형태가 100만 개 이상 — 임베딩 테이블 행이 100만 줄 필요해집니다.
  • 미등록 단어. 오타나 신조어는 <UNK>가 되고 모델은 눈을 감습니다.
  • 형태론. "running" / "runner" / "runs"는 어근을 공유하지만 모델은 그걸 보지 못합니다.
  • 언어들. 중국어는 띄어쓰기가 없습니다. 터키어는 형태소를 붙입니다. 공백 분리는 실패합니다.
서브워드 토크나이저는 이 네 가지를 모두 해결합니다 — 약 3만~20만 개의 작은 조각으로 어떤 문자열이든 조합으로 덮을 수 있습니다.
만나게 될 세 가지 계열

BPE Byte-Pair Encoding — 가장 빈번한 인접 쌍을 반복해서 병합합니다. GPT-2/3/4, LLaMA, Mistral.

WordPiece BPE와 비슷하지만 코퍼스 우도(likelihood)를 최대화하는 병합을 고릅니다. BERT, DistilBERT, ELECTRA.

SentencePiece 공백을 일반 문자(▁)처럼 다룹니다. 언어 비종속적이고 Unigram 또는 BPE를 감싸는 형태. T5, ALBERT, LLaMA.

다음에 오는 것
토큰은 토큰 ID가 되고, 그게 임베딩이 되고, 결국 [batch, seq_len, d_model] 모양의 텐서가 됩니다. 트랜스포머의 나머지 모든 부분 — 어텐션, FFN, 출력 프로젝션 — 은 단지 그 텐서의 변환일 뿐입니다. 토크나이즈만 제대로 하면, 나머지는 행렬 곱셈입니다.

Chapter 02 · 모든 토크나이저가 거치는 다섯 단계

토크나이저는
실제로 어떻게 작동할까.

현대의 모든 토크나이저는 결국 같은 다섯 가지 일을 어떤 순서로든 합니다: 정규화(normalize), 사전 분할(pre-tokenize), 학습된 병합 테이블로 서브워드로 쪼개기, 각 조각을 ID로 매핑하기, 특수 토큰 추가. 각 단계를 평이한 한국어로 풀어봅니다.

1

정규화 (Normalize)

소문자화 / NFC 유니코드 정규화 / 악센트 제거 — 모델마다 다릅니다. GPT-2는 거의 아무것도 하지 않습니다. BERT-uncased는 공격적으로 소문자화합니다.

"Hèllo!" → "hello!" (BERT-uncased)
2

사전 분할 (Pre-tokenize)

병합 단계가 단어 경계를 넘지 않도록 공백과 구두점에서 먼저 자릅니다. GPT-2는 정규식을 사용합니다. SentencePiece는 사전 분할을 하지 않고 공백을 포함한 원시 바이트 위에서 동작합니다.

"hello world!" → ["hello", " world", "!"]
3

병합 / vocab 테이블 적용

학습 시점에 (느린 단계로) 서브워드 조각의 어휘를 학습합니다. 추론 시에는 각 사전 토큰을 알고 있는 가장 긴 조각으로 쪼갭니다.

"tokenization" → ["token", "ization"] (BPE)
4

조각 → 정수 ID 매핑

어휘는 str → int 사전입니다. 각 조각은 임베딩 테이블의 행 인덱스가 됩니다.

["token", "ization"] → [30001, 1634]
5

특수 토큰 추가 & 패딩

BERT는 앞에 [CLS]를, 끝에 [SEP]를 붙입니다. GPT 계열은 보통 아무것도 안 붙입니다. 배치 입력은 <pad> id로 공통 길이까지 패딩되고, 어텐션 마스크가 어떤 위치가 실제 토큰인지 기록합니다.

[101, 30001, 1634, 102, 0, 0] mask: [1, 1, 1, 1, 0, 0]

02 60초 만에 BPE — 병합 단계

Byte-Pair Encoding은 학습 중에 코퍼스에서 가장 빈번한 인접 심볼 쌍을 반복적으로 찾아서 그 둘을 이어 붙인 것을 새 심볼로 추가하는 방식으로 어휘를 만듭니다. $V$번의 병합을 돌리면 ~초기 바이트 + $V$ 크기의 어휘가 나옵니다.

아주 작은 예제
코퍼스 "low low low lower newer"로 시작해서 단어 경계 마커 ▁와 함께 문자 단위로 쪼갭니다: 
# step 0 — 바이트
▁ l o w   ▁ l o w   ▁ l o w   ▁ l o w e r   ▁ n e w e r

# 가장 빈번한 쌍 = (l, o)  →  "lo"로 병합
▁ lo w   ▁ lo w   ▁ lo w   ▁ lo w e r   ▁ n e w e r

# 가장 빈번한 쌍 = (lo, w)  →  "low"로 병합
▁ low   ▁ low   ▁ low   ▁ low e r   ▁ n e w e r

# 가장 빈번한 쌍 = (e, r)  →  "er"로 병합
▁ low   ▁ low   ▁ low   ▁ low er   ▁ n e w er

# 최종 vocab에 포함: ▁, l, o, w, e, r, n, lo, low, er, ...
추론은 같은 병합 리스트를 순서대로 따라가며 여전히 적용 가능한 병합을 모두 적용합니다. 그래서 BPE는 결정적이고 무손실입니다 — 어떤 바이트 시퀀스에도 고유한 토큰화가 존재합니다.

03 토크나이즈를 머릿속으로 읽기

GPT-2 출력을 한참 보다 보면 보이는 신호 몇 가지:

  • 앞 공백이 중요합니다. 문자열 맨 앞의 "hello"는 문장 중간의 " hello"와 다른 토큰입니다. GPT-2는 바이트 레벨 vocab에서 앞 공백을 Ġ로 표시하기까지 합니다.
  • 흔한 단어 → 토큰 1개. "the", "and", "is"는 단일 토큰입니다.
  • 길거나 희귀한 단어 → 여러 토큰. "tokenization"은 쪼개지고, "antidisestablishmentarianism"은 진짜로 잘게 쪼개집니다.
  • 숫자와 이모지는 잘게 잘립니다 — "2024"는 토크나이저에 따라 1, 2, 또는 4개 토큰이 될 수 있습니다.
  • 비라틴 문자는 byte-level BPE에서 바이트 토큰이 됩니다: 3바이트짜리 UTF-8 코드포인트는 토큰 3개입니다. SentencePiece를 해당 언어로 학습시키면 이 문제를 피할 수 있습니다.

Chapter 04 · 텐서가 어디서 오는가

문자열에서 로짓까지
여덟 개의 모양으로.

같은 문장 — "Hello, world!" — 을 가지고 문자열에서 로짓까지 정확히 어떤 모양들을 거치는지 추적해 봅시다. B = batch, T = sequence length, D = hidden size, V = vocab size.

1

원시 문자열

"Hello, world!"

shape: 스칼라 (아직 텐서 아님)
2

토크나이즈 → ID 리스트

[15496, 11, 995, 0] (GPT-2 BPE, 4 토큰)

shape: [T=4] dtype: int64
3

배치 + 패딩

두 문장을 배치로 쌓고, ID 0으로 T=6까지 패딩:
[[15496, 11, 995, 0, 0, 0],
 [15496, 11, 5440, 16085, 0, 0]]

input_ids: [B=2, T=6] attention_mask: [B=2, T=6]
4

임베딩 룩업

x = embed[input_ids] — [V × D] 테이블에서 ID마다 한 행씩.

x: [B=2, T=6, D=768] embed: [V=50257, D=768]
5

+ 위치 임베딩

[T × D] 테이블을 더합니다 (RoPE는 대신 Q/K를 회전시킵니다). 이제 위치가 의미를 갖습니다.

x: [B=2, T=6, D=768] pos: [T=6, D=768]
6

L개의 트랜스포머 블록

각 블록은 모양을 유지합니다. 내부에서 어텐션이 잠시 scores: [B, H, T, T]를 만듭니다.

x 유지: [B=2, T=6, D=768]
7

출력 헤드 → 로짓

logits = x @ embed.T — 모든 위치에서 vocab 토큰마다 점수 한 개씩.

logits: [B=2, T=6, V=50257]
8

softmax → 다음 토큰 ID

argmax(logits[:, -1, :], dim=-1)가 다음 ID를 고르고, 토크나이저가 그걸 다시 텍스트로 바꿉니다.

next_id: [B=2] T⁻¹로 "decoded text"

04 같은 흐름을, PyTorch로

HuggingFace로 처음부터 끝까지
from transformers import AutoTokenizer, AutoModelForCausalLM
import torch

tok   = AutoTokenizer.from_pretrained("gpt2")
model = AutoModelForCausalLM.from_pretrained("gpt2")

# 1. encode → int64 텐서 dict
batch = tok(["Hello, world!", "Hello, friends, again"],
            padding=True, return_tensors="pt")
print(batch.input_ids.shape)        # torch.Size([2, 6])
print(batch.attention_mask.shape)   # torch.Size([2, 6])

# 2. forward — 임베딩 + 트랜스포머 블록
out = model(**batch)
print(out.logits.shape)            # torch.Size([2, 6, 50257])

# 3. greedy 다음 토큰 = vocab 축에서 argmax
next_id = out.logits[:, -1, :].argmax(dim=-1)
print(tok.batch_decode(next_id.unsqueeze(-1)))
대략적인 메모리 견적. 8k 컨텍스트, 배치 1, fp16 추론은 레이어당 활성값에 대략 $T \times D \times 2$ 바이트가 필요하고, 거기에 $2 \times L \times T \times D \times 2$ 바이트의 KV 캐시가 추가됩니다. LLaMA-7B는 T=8192에서 KV 캐시만 ~512 MB — 토크나이저의 압축률이 직접적으로 결정하는 예산입니다.
Chapter 05 · 서브워드의 세 가족

BPE · WordPiece · SentencePiece.

셋 다 서브워드 토큰을 만들고, 셋 다 빈도 통계에서 출발하지만, 병합 규칙과 공백을 다루는 방식은 다릅니다. 문자열을 입력해서 각각이 어떻게 쪼개는지 보세요.

BPE — 빈도 기반 병합

가장 빈번한 인접 쌍을 골라 병합, 반복. 추론은 같은 병합을 순서대로 적용합니다. 탐욕적이고 무손실입니다.

사용 모델: GPT-2/3/4, RoBERTa, LLaMA, Mistral, Qwen.

특징: 앞 공백이 토큰의 일부입니다 (Ġ 또는 ▁로 출력).

WordPiece — 우도 기반 병합

BPE와 비슷하지만, 병합 규칙이 단순 빈도 대신 코퍼스 로그 우도를 최대화합니다. 이어지는 조각은 앞에 ##가 붙습니다.

사용 모델: BERT, DistilBERT, ELECTRA, MobileBERT.

특징: "playing" → ["play", "##ing"]

SentencePiece — 바이트는 신성하다

공백을 진짜 문자(▁ = U+2581)로 다루고, 사전 분할 없이 원시 텍스트 위에서 직접 학습합니다. Unigram-LM이나 BPE를 감쌀 수 있습니다.

사용 모델: T5, ALBERT, XLNet, mT5, LLaMA.

특징: ▁hello, ▁world 같은 토큰.

05 잘못 골랐을 때 무엇이 달라지나?

  • 토크나이저 불일치는 조용하고 치명적입니다. BPE로 학습하고 WordPiece로 추론하면 — 모든 임베딩 룩업이 잘못됩니다. 모델은 그래도 돌아가고 출력은 헛소리입니다.
  • 런타임에 토큰 추가는 가능합니다 (예: 채팅 템플릿용). 하지만 임베딩 테이블 크기를 맞춰야 합니다. 그 단계를 까먹으면 새 ID에서 크래시 납니다.
  • 언어가 다르면, 청구서도 다릅니다. 한국어를 GPT-2 BPE로 처리하면 영어 대비 약 3배의 토큰이 듭니다. 한국어 학습 SentencePiece에서는 약 1배. 같은 모델, 다른 비용.

A 핵심 아이디어

1 토크나이저는 문자열을 정수 ID 리스트로 매핑합니다. 이 ID들을 토큰 …(이)라고 부릅니다.
encode("hi") → []
2 토크나이저가 내보낼 수 있는 모든 알려진 조각의 집합을 …(이)라고 부릅니다.
len(tokenizer.)   # → 예: GPT-2의 경우 50257
3 반대 방향 — ID를 다시 텍스트로 — 을 … (이)라고 합니다.
tokenizer.([15496, 11, 995])  # → "Hello, world"

B 특수 토큰 & 마스크

4 BERT는 모든 입력의 맨 앞에 특수 토큰 하나를 붙이고, 이를 문장 단위 요약으로 사용합니다.
[] tokens... [SEP]
5 패딩된 위치는 0으로 된 …(을)를 통해 어텐션이 무시합니다.
attention_ = [1, 1, 1, 0, 0]   # 마지막 둘은 패딩

C 모양 & 흐름

6 두 문장을 길이 6으로 패딩해 인코딩하면 input_ids 텐서의 shape는 …
input_ids.shape == (, )
7 [B, T] 모양의 ID 텐서로 임베딩을 룩업하면 [B, T, …] 모양의 텐서가 나옵니다.
x = embed[input_ids]              # x.shape == (B, T, )
8 출력 프로젝션은 hidden state를 모든 어휘 토큰에 대한 점수로 변환합니다.
logits = x @ embed.T              # logits.shape == (B, T, )

D BPE와 친구들

9 BPE 학습은 가장 … 인접 심볼 쌍을 반복적으로 병합합니다.
while merges < V:
    pair = argmax_(adjacent_pairs(corpus))
    corpus = merge(pair, corpus)
10 WordPiece는 "이것은 이전 조각의 연속"임을 접두사로 표시합니다.
"playing" → ["play", ""]
11 SentencePiece는 공백을 단일 가시 문자, U+2581 … (으)로 인코딩합니다.
"hello world" → ["", "▁world"]
12 빠른 HuggingFace 왕복: encode 후 decode.
tok = AutoTokenizer.from_pretrained("gpt2")
ids = tok.("hi there")
text = tok.(ids)
열두 개 다 풀었나요? 이제 토크나이즈된 출력을 읽고, 모양을 예측하고, 같은 문자열이 모델마다 왜 다른 토큰 수를 갖는지 설명할 수 있습니다. 다음 정거장 — 상자들 안에서 일어나는 일을 보려면 transformers.html로.

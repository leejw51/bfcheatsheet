레벨 01 // 동기

§01왜 트랜스포머인가?

2017년 이전에는 RNN과 LSTM이 주류 시퀀스 모델이었습니다. 그들은 토큰을 하나씩 처리하며 히든 상태를 앞으로 전달했습니다. 이는 세 가지 치명적인 문제가 있었습니다:

  • 순차적 병목. 시간 차원에 걸쳐 병렬화할 수 없습니다 — 토큰 t는 토큰 t-1이 필요합니다.
  • 거리에 따른 그래디언트 소실. 토큰 1의 정보는 토큰 100에 도달하기 전 모든 단계를 거쳐 압축됩니다.
  • 고정 크기 히든 상태. 전체 과거가 하나의 벡터에 들어가야 합니다. 장거리 메모리는 취약합니다.
핵심 통찰

Attention is all you need

Vaswani 외는 물었습니다: 모든 토큰이 한 번의 병렬 단계에서 다른 모든 토큰을 직접 볼 수 있다면 어떨까? 재귀도 없고, 컨볼루션도 없고, 그저 각 위치가 학습된 가중치로 다른 모든 위치에 어텐드할 수 있게 하는 학습된 메커니즘.

결과: 하드웨어에 따라 선형적으로 확장되는 학습, O(1) 홉으로 흐르는 그래디언트, 그리고 원칙적으로 단락의 첫 단어를 이웃만큼 쉽게 마지막 단어와 관련시킬 수 있는 모델.

레벨 02 // 입력

§02토큰화 & 임베딩

트랜스포머는 텍스트를 직접 읽지 못합니다. 정수 ID를 읽습니다. 토큰화는 문자열을 고정된 어휘집(보통 30k–200k 항목)에서 가져온 토큰 ID 시퀀스로 바꾸는 손실 압축 방식입니다. 현대 토큰화기(BPE, WordPiece, Unigram, Tiktoken)는 서브워드 조각으로 분할하여, 희귀 단어는 미지의 토큰 대신 흔한 조각들의 시퀀스가 됩니다.

각 정수 ID는 그 다음 임베딩 행렬 $E \in \mathbb{R}^{V \times d_{model}}$ — 여기서 $V$는 어휘 크기 — 을 통해 밀집 벡터로 매핑됩니다. 룩업 = 행렬의 한 행. 결과: $L$개 토큰의 시퀀스에 대해 [L, d_model] 형태의 텐서를 얻습니다.

라이브 데모

문장을 입력하세요 — 토큰을 보세요

참고: 이것은 공백+구두점 기반의 토이 토큰화기입니다. 실제 BPE라면 tokenization을 token + ization 같은 식으로 분할할 것입니다.

레벨 03 // 순서

§03위치 인코딩

셀프 어텐션은 순열 불변입니다: 입력 토큰 순서를 섞어도 (위치 정보가 없다면) 같은 출력이 나옵니다. 언어에는 큰 문제죠. 해결책: 스택에 들어가기 전, 각 토큰 임베딩에 위치 의존 벡터를 더합니다.

원본 논문은 기하학적으로 증가하는 파장의 사인 곡선을 사용합니다:

$$PE_{(pos,\,2i)} = \sin\!\Big(\tfrac{pos}{10000^{2i/d_{model}}}\Big), \qquad PE_{(pos,\,2i+1)} = \cos\!\Big(\tfrac{pos}{10000^{2i/d_{model}}}\Big)$$

시각화

사인 곡선 위치 파동

각 행은 차원 i입니다. 낮은 차원은 빠르게 진동하고(위치마다 매우 달라 보임), 높은 차원은 매끄럽습니다(대규모 순서).

영리한 성질: $PE_{pos+k}$는 $PE_{pos}$의 고정된 선형 함수이므로, 모델이 상대적인 오프셋을 쉽게 학습할 수 있습니다. 현대 변형(RoPE, ALiBi, 학습된 절대 위치)은 서로 다른 트레이드오프를 제공하지만 목표는 같습니다 — 순서를 주입하는 것.

레벨 04 // 핵심

§04셀프 어텐션 (Q · K · V)

모든 입력 토큰은 세 개의 학습된 행렬 $W^Q, W^K, W^V \in \mathbb{R}^{d_{model} \times d_k}$를 통해 세 개의 벡터로 투영됩니다:

  • Query (Q) — "내가 무엇을 찾고 있는가?"
  • Key (K) — "내가 무엇을 담고 있는가?"
  • Value (V) — "내가 어텐드된다면 무엇을 기여하는가?"

스케일드 닷-프로덕트 어텐션은 다음과 같습니다:

$$\mathrm{Attention}(Q,K,V) = \mathrm{softmax}\!\Big(\tfrac{QK^\top}{\sqrt{d_k}}\Big)\,V$$

시퀀스 길이 $L$, 헤드 차원 $d_k$일 때의 형태: Q: [L, d_k] K: [L, d_k] V: [L, d_k]. 곱 $QK^\top$는 [L, L] — (쿼리, 키) 쌍 하나당 유사도 점수 하나. 행을 따라 softmax를 취하면 어텐션 가중치가 되고, 그 가중치로 $V$를 다시 가중합합니다.

레벨 05 // 앙상블

§05멀티헤드 어텐션

하나의 어텐션 패턴만으로는 부족할 때가 많습니다. 멀티헤드 어텐션은 $h$개의 어텐션 연산을 병렬로 실행하며, 각각 자체 $W_i^Q, W_i^K, W_i^V$ 투영을 가지고, 결과를 연결(concatenate)합니다:

$$\mathrm{MHA}(X) = \mathrm{Concat}(\mathrm{head}_1, \ldots, \mathrm{head}_h)\,W^O$$

각 헤드는 서로 다른 관계 패턴에 어텐드하도록 학습됩니다 — 어떤 헤드는 구문 의존성을, 다른 헤드는 상호 참조를, 또 다른 헤드는 단순한 위치 오프셋을 추적할 수 있습니다. 이 분할은 총 연산량을 단일 헤드와 거의 같게 유지합니다: $h=8$개 헤드라면 $d_k = d_{model}/h$로 잡습니다.

레벨 06 // 접착제

§06피드포워드 · 잔차 · 정규화

어텐션 이후, 각 위치는 위치별 피드포워드 네트워크를 통과합니다 — 비선형 활성화 함수를 사이에 둔 두 개의 선형 계층(원래는 ReLU, 오늘날엔 보통 GELU 또는 SwiGLU):

$$\mathrm{FFN}(x) = \max(0,\, xW_1+b_1)\,W_2 + b_2$$

은닉 너비는 보통 4·d_model입니다. 모델 파라미터의 대부분이 이곳에 자리하며 — 이 계층이 사실적 지식을 저장하는 키-값 메모리 역할을 한다는 가설이 인기 있습니다.

잔차 연결

어텐션 서브계층과 FFN 서브계층 모두 잔차 연결 + 레이어 정규화로 감싸집니다:

$$x \leftarrow \mathrm{LayerNorm}(x + \mathrm{Sublayer}(x))$$

왜? 잔차는 손실에서 입력으로 되돌아가는 그래디언트 고속도로를 만들어, 수백 개의 계층이 안정적으로 학습되도록 합니다. LayerNorm은 특징 차원에 걸쳐 활성을 중심화하고 스케일을 맞춰, 각 계층의 최적화를 다른 계층과 분리합니다.

현대 변형은 pre-norm을 선호합니다: $x \leftarrow x + \mathrm{Sublayer}(\mathrm{LayerNorm}(x))$ — 깊이가 깊어질 때 더 안정적인 학습을 제공합니다.

레벨 07 // 토폴로지

§07인코더 · 디코더 · 마스킹

오늘날 세 가지 아키텍처 계열이 지배적입니다:

  • 인코더 전용 (BERT) — 양방향 셀프 어텐션. 모든 토큰이 다른 모든 토큰을 봅니다. 이해 과제(분류, 검색)에 좋습니다.
  • 디코더 전용 (GPT) — 인과적/마스크된 셀프 어텐션. 토큰 t는 $\le t$인 토큰에만 어텐드할 수 있습니다. 생성에 좋습니다. 대부분의 현대 LLM이 사용하는 방식입니다.
  • 인코더-디코더 (T5, 원본 NMT) — 인코더가 소스를 읽고, 디코더는 인코더 출력에 교차 어텐드하면서 타겟을 생성합니다.

인과적 마스킹 — 어떻게?

softmax 이전에 점수 행렬의 상삼각 항목을 $-\infty$로 설정합니다. $-\infty$의 softmax는 0이므로 — 모델은 수학적으로 미래를 들여다볼 수 없습니다:

레벨 08 // 목표

§08학습: 다음 토큰 예측

디코더 전용 LLM (GPT 계열)의 경우, 전체 학습 목표는 충격적일 만큼 단순합니다:

토큰 1..t가 주어지면, 토큰 t+1을 예측하라.

모델은 모든 위치에서 어휘에 대한 확률 분포를 출력합니다. 손실은 실제 다음 토큰에 대한 평균 교차 엔트로피입니다:

$$\mathcal{L} = -\frac{1}{N}\sum_{t=1}^{N} \log P_\theta(x_{t+1} \mid x_1, \ldots, x_t)$$

인과적 마스킹이 이를 효율적으로 만듭니다: 길이 $N$인 시퀀스에 대해 단일 순전파가 위치당 하나씩 $N$개의 예측을 병렬로 산출합니다. 라벨을 한 칸 시프트하는 것 외에 별도의 teacher forcing 트릭이 필요 없습니다.

그 외 모든 것 — 인스트럭션 튜닝, RLHF, 도구 사용 — 은 이 목표 위에 올라탑니다. 사전학습은 인터넷 규모의 데이터를 흡수하고 모델이 "모든 것"을 배우는 부분입니다. 파인튜닝은 모델의 행동을 다듬는 부분입니다.

요약

전체 파이프라인

  • 1. 토큰 → 정수 ID → 임베딩 (+ 위치 인코딩)
  • 2. N개 트랜스포머 블록 스택: MHA → Add&Norm → FFN → Add&Norm
  • 3. 어휘 크기로 최종 선형 투영, softmax → 다음 토큰 분포
  • 4. 정답 다음 토큰에 대한 교차 엔트로피 손실, 위치 전체에 걸쳐 합산
  • 5. 모든 가중치를 통한 역전파. 약 10²² 회 반복. 끝.

자, 이제 드릴해 보세요.

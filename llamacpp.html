llama.cpp — run LLMs locally, in C/C++

llama.cpp is a high-performance inference engine for LLaMA-family models (Llama 1/2/3, Mistral, Qwen, Gemma, Phi, DeepSeek, ...) written in plain C/C++ with zero Python at runtime. It runs on CPU, Apple Silicon (Metal), NVIDIA (CUDA), AMD (ROCm), and any GPU with Vulkan. Models live in a single .gguf file. By the end of this page you'll have typed the commands to build it, quantize, run, and serve it.

Why llama.cpp?
Zero-dependency C++. Boots in under a second. Quantized models (Q4_K_M) shrink a 7B model from ~14 GB (fp16) to ~4 GB. Runs on a laptop, a Raspberry Pi, an M-series Mac, or a 24 GB GPU — same code path. The reference implementation everyone else copies.
What's GGUF?
GGUF (GPT-Generated Unified Format) is a single-file binary that bundles weights + tokenizer + chat-template + metadata. Memory-mappable, versioned, and the de-facto exchange format for local LLMs. Replaces the older GGML format.

cli llama-cli — interactive chat / one-shot generation

# 1. clone & build (Mac with Metal)
git clone https://github.com/ggml-org/llama.cpp
cd llama.cpp && cmake -B build -DGGML_METAL=ON && cmake --build build -j

From source — the canonical path

build from source
# clone
git clone https://github.com/ggml-org/llama.cpp
cd llama.cpp

# cpu-only (any platform)
cmake -B build
cmake --build build --config Release -j

# Apple Silicon — Metal
cmake -B build -DGGML_METAL=ON
cmake --build build --config Release -j

# NVIDIA — CUDA (needs CUDA toolkit installed)
cmake -B build -DGGML_CUDA=ON
cmake --build build --config Release -j

# AMD — HIP/ROCm
cmake -B build -DGGML_HIP=ON -DAMDGPU_TARGETS=gfx1100

# Vulkan — cross-vendor GPU
cmake -B build -DGGML_VULKAN=ON
The classic make path still works but is deprecated. Stick with cmake. Binaries land in build/bin/.

GGUF & Quantization

Convert & quantize your own model

HuggingFace → fp16 GGUF → Q4_K_M
# 1. convert HF safetensors → fp16 gguf
python convert_hf_to_gguf.py /path/to/Meta-Llama-3-8B \
    --outfile llama3-8b-f16.gguf --outtype f16

# 2. quantize fp16 → Q4_K_M
./build/bin/llama-quantize llama3-8b-f16.gguf llama3-8b-q4km.gguf Q4_K_M

# 3. evaluate perplexity (lower = better)
./build/bin/llama-perplexity -m llama3-8b-q4km.gguf -f wiki.test.raw

llama-cli — one-shot & chat

The frontend for direct generation. Pass a prompt, get tokens. Add -cnv for conversation mode, or pipe via stdin. All flags below also work on llama-server.

Most-used flags

cheat sheet of flags
-m path to .gguf model
-p prompt string
-f read prompt from file
-n n_predict — max new tokens (-1 = ∞)
-c context size (e.g. 4096, 8192)
-b batch size for prompt (default 2048)
-cnv conversation / chat mode
-sys system prompt
-ngl # layers offloaded to GPU (99 = all)
-t CPU threads
--temp sampling temperature
--top-k top-k sampling
--top-p nucleus sampling
--repeat-penalty against repetition
--mlock lock weights in RAM
--seed RNG seed (-1 = random)
canonical generation
llama-cli \
    -m models/Meta-Llama-3.1-8B-Instruct-Q4_K_M.gguf \
    -p "Explain backprop in 2 sentences." \
    -n 256 \
    -c 4096 \
    -ngl 99 \
    --temp 0.7 --top-p 0.9
interactive chat (Llama-3 chat template auto-detected)
llama-cli -m model.gguf -cnv \
    -sys "You are a terse assistant." \
    -ngl 99 -c 8192

llama-server — OpenAI-compatible HTTP

llama-server exposes the same model over HTTP, speaking a subset of OpenAI's API. Drop-in replacement for many SDKs. Built-in WebUI at /.

launch the server
llama-server \
    -m models/Meta-Llama-3.1-8B-Instruct-Q4_K_M.gguf \
    --host 0.0.0.0 --port 8080 \
    -c 8192 -ngl 99 \
    --chat-template llama3 \
    --api-key sk-local # optional auth

Endpoints (OpenAI-compatible)

chat — /v1/chat/completions
Accepts { messages: [...] }. Server applies the chat template baked into the GGUF. Supports streaming via "stream": true.
completion — /v1/completions
Raw text completion. { prompt: "..." }. Skip the chat template entirely.
embeddings — /v1/embeddings
For embedding models — start with --embedding. Returns vectors compatible with OpenAI clients.
native — /completion, /tokenize, /health
llama.cpp-native endpoints with extras: grammar, JSON schema, slot management. Visit http://localhost:8080 for the WebUI.
curl example — chat completion
curl http://localhost:8080/v1/chat/completions \
    -H "Content-Type: application/json" \
    -d '{
        "messages": [
            {"role": "system", "content": "You are concise."},
            {"role": "user",   "content": "What is GGUF?"}
        ],
        "temperature": 0.7,
        "max_tokens": 200
    }'
python (openai SDK)
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:8080/v1",
    api_key="sk-local",        # anything works unless --api-key set
)

resp = client.chat.completions.create(
    model="local",             # ignored, only one model loaded
    messages=[
        {"role": "system", "content": "You are terse."},
        {"role": "user",   "content": "Why is the sky blue?"},
    ],
    stream=True,
)
for chunk in resp:
    print(chunk.choices[0].delta.content or "", end="", flush=True)
python — llama-cpp-python (in-process)
# pip install llama-cpp-python
from llama_cpp import Llama

llm = Llama(
    model_path="model-q4_k_m.gguf",
    n_ctx=8192,
    n_gpu_layers=99,         # offload everything

Sampling parameters

After the model produces logits, samplers narrow them to a single token. The defaults are sensible; the knobs below cover 99% of tweaking.

temperature — --temp
Divides logits by T. 0 = greedy / deterministic; 0.7 ≈ creative chat; 1.0+ = wild. Code generation: 0.0–0.3. Story: 0.7–1.0.
top-k — --top-k
Keep only the K highest-probability tokens. 40 is the classic default; 0 disables.
top-p (nucleus) — --top-p
Keep the smallest set of tokens whose cumulative probability ≥ p. Modern default: 0.9–0.95. Set to 1.0 to disable.
repeat-penalty — --repeat-penalty
Punishes already-seen tokens. 1.0 = off. 1.1–1.2 cures most loops. Too high → unnatural prose.
min-p — --min-p
Drop tokens below min_p × max_prob. Newer than top-p; some users prefer it.
seed — --seed
RNG seed. -1 = random. Pin a seed for reproducible runs.

Advanced — grammar & speculative decoding

force structured output (GBNF grammar)
# constrain output to valid JSON
llama-cli -m model.gguf -p "return JSON: {name, age}" \
    --grammar-file grammars/json.gbnf

# or inline grammar
llama-cli -m model.gguf -p "yes or no?" \
    --grammar 'root ::= "yes" | "no"'
speculative decoding (small draft + big verifier)
llama-cli -m big-70b.gguf \
    --draft-model small-3b.gguf \
    --draft 5 \
    -p "explain quicksort" -n 512
14 Constrain output to valid JSON via grammar file:
llama-cli -m m.gguf --grammar-file grammars/.gbnf
15 Activate speculative decoding by passing a small draft model:
llama-cli -m big.gguf ---model small.gguf --draft 5
16 Greedy / deterministic decoding (no randomness) — set temperature to:
llama-cli -m m.gguf -p "hi" --temp

The Perceptron — one neuron, two ideas

The perceptron is the simplest neural network: a single neuron that takes a weighted sum of inputs, adds a bias, and fires 0 or 1 depending on the sign. Frank Rosenblatt introduced it in 1958, and every modern deep network is a stack of variants of this one idea.

the neuron
Inputs $x_1, x_2, \dots, x_n$ are multiplied by weights $w_1, w_2, \dots, w_n$, summed, and offset by a bias $b$. An activation function (here: a step) turns the result into a prediction.
$\hat{y} = \mathrm{step}(w_1 x_1 + w_2 x_2 + \cdots + w_n x_n + b)$
the step activation
$\mathrm{step}(z) = \begin{cases} 1 & z \ge 0 \\ 0 & z < 0 \end{cases}$

Also called the Heaviside or sign function. It's the on/off switch that makes the neuron "fire". Modern nets swap it for ReLU/sigmoid because the step's derivative is 0 almost everywhere — bad for gradient descent.

training
Show one labeled example at a time. Compute the prediction. If it's wrong, nudge the weights so next time it'll be a little less wrong:
$w_i \leftarrow w_i + \eta\,(y - \hat{y})\,x_i$
$b \leftarrow b + \eta\,(y - \hat{y})$
$\eta$ (eta) is the learning rate. Repeat until no example is misclassified.
inference
Once weights are fixed, prediction is a single dot product plus a step function. No gradients, no loops — just $\hat{y} = \mathrm{step}(\mathbf{w}\cdot\mathbf{x} + b)$. That's all an inference call is.

The forward equation, in code

pure Python — no libraries
def step(z):
    return 1 if z >= 0 else 0

def predict(w, b, x):
    z = sum(wi * xi for wi, xi in zip(w, x)) + b
    return step(z)

# example: AND gate after training
w = [1.0, 1.0]
b = -1.5
for x in [(0,0), (0,1), (1,0), (1,1)]:
    print(x, predict(w, b, x))   # 0, 0, 0, 1

The math, written out

For a 2-input perceptron:

$z = w_1 x_1 + w_2 x_2 + b \quad\Longrightarrow\quad \hat{y} = \mathrm{step}(z)$

The boundary line $w_1 x_1 + w_2 x_2 + b = 0$ separates the plane into two half-spaces. Anything with a positive $z$ is class 1; anything negative is class 0.

The Perceptron Learning Rule

Rosenblatt's algorithm is delightfully simple: if you misclassify a point, push the weight vector slightly toward (or away from) it. Do that for every example, repeatedly, and — if the data are linearly separable — you'll converge.

the update — for one example $(x, y)$
$\hat{y} = \mathrm{step}(\mathbf{w}\cdot\mathbf{x} + b)$
$\mathbf{w} \leftarrow \mathbf{w} + \eta\,(y - \hat{y})\,\mathbf{x}$
$b \leftarrow b + \eta\,(y - \hat{y})$

Three cases:

  • correct$y - \hat{y} = 0$ — no update.
  • missed a 1$y - \hat{y} = +1$ — push $\mathbf{w}$ toward $\mathbf{x}$.
  • false alarm$y - \hat{y} = -1$ — push $\mathbf{w}$ away from $\mathbf{x}$.

Walkthrough — the five steps

1pick example

Grab one labeled point $(x, y)$ from the dataset.

2predict

z = w·x + b
ŷ = step(z)

3compute error

err = y − ŷ
(one of −1, 0, +1)

4update weights

w += η · err · x
b += η · err

5repeat

Loop over all examples for several epochs until no errors remain.

Reference implementation

perceptron training — pure NumPy
import numpy as np

def step(z):
    return np.where(z >= 0, 1, 0)

def train_perceptron(X, y, lr=0.1, epochs=20):
    n, d = X.shape
    w = np.zeros(d)
    b = 0.0
    for epoch in range(epochs):
        for i in range(n):
            z    = np.dot(w, X[i]) + b
            yhat = step(z)
            err  = y[i] - yhat
            w   += lr * err * X[i]   # weight update
            b   += lr * err          # bias update
    return w, b

# AND dataset
X = np.array([[0,0],[0,1],[1,0],[1,1]])
y = np.array([ 0,         0,         0,         1      ])

w, b = train_perceptron(X, y)
print(w, b)   # something like [0.2, 0.1] -0.2
Convergence: the perceptron learning rule is guaranteed to find a separating hyperplane in finite steps — but only if one exists. If the classes overlap or are XOR-shaped, it'll oscillate forever. We'll see why in tab 6.

Inference — using the trained perceptron

Once weights are frozen, prediction is a single line of math: dot product, plus bias, plus step. No loops, no gradients, no state. This is the simplest "neural inference" that exists.

scalar form
$\hat{y} = \mathrm{step}\!\left(\sum_{i=1}^{n} w_i x_i + b\right)$
vector form
$\hat{y} = \mathrm{step}(\mathbf{w}^{\!\top}\mathbf{x} + b)$
inference, vectorized for a whole batch
import numpy as np

# trained weights (e.g. for AND)
w = np.array([1.0, 1.0])
b = -1.5

# a batch of inputs, shape (N, 2)
X_new = np.array([[0,0], [0,1], [1,0], [1,1]])

logits = X_new @ w + b           # shape (N,)
preds  = np.where(logits >= 0, 1, 0)

print(preds)                     # [0 0 0 1]
Why it's so fast: inference is one matrix-vector product per layer. A real LLM is just billions of perceptrons stacked — but each individual neuron's inference is identical to this.

The XOR Problem — why one neuron isn't enough

A single perceptron can only draw a straight line. AND and OR are linearly separable — one line splits the 1s from the 0s. XOR isn't. Try drawing one line that separates $(0,1)$ and $(1,0)$ from $(0,0)$ and $(1,1)$. You can't.

XOR truth table

000
011

110

The 1s are on the diagonal corners — no line through the unit square separates them from the 0s.

XOR — uncuttable by a line

Two minuses, two pluses, no straight line cuts them apart.

The fix: stack perceptrons → MLP

Add one hidden layer of perceptrons and you can build any boolean function. XOR = (A OR B) AND NOT (A AND B), which is two neurons feeding a third.

Historical note: Minsky & Papert's 1969 book Perceptrons showed XOR couldn't be solved by a single layer. The "AI winter" followed. The fix — backpropagation through a hidden layer — wasn't widely adopted until the 1980s. That multi-layer perceptron (MLP) is the direct ancestor of every modern deep network.

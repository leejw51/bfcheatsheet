Order · the missing signal

Teaching attention about order

Self-attention computes a weighted average over all tokens at once — it has no built-in notion of order. Permute the input and the outputs just permute the same way (it is permutation-equivariant), so "the cat sat" and "sat cat the" look identical: a pure bag of words. Positional encoding fixes this by adding a position-dependent signal to each token embedding before attention ever runs.

01 The problem & the fix

the problem

Order is invisible

Attention scores depend only on which tokens are present, not where they sit. Reorder the words and every attention weight follows along unchanged — the model can't tell a sentence from its shuffle.

the fix

A position fingerprint

Build a matrix $PE \in \mathbb{R}^{T \times D}$ where row $pos$ is a unique signature for that position, and add it to the token embeddings. Identical words at different positions now get different vectors.

RNNs get order for free — they read tokens one at a time, so step number is position. Transformers process every position in parallel for speed, so order is lost and must be injected explicitly. Positional encoding is the price of parallelism.

01·b Where it sits in the pipeline

shapes · PE broadcasts over the batch
token ids [B, T] → embed [B, T, D] + PE[:T] [T, D] → X [B, T, D]

$B$ = batch, $T$ = sequence length, $D$ = $d_{model}$ (embedding size). The encoding is added once, right at the input, then attention and the rest of the stack run as usual.

The sinusoidal formula

The original Transformer uses a fixed (non-learned) encoding built from sines and cosines of geometrically-spaced frequencies. Each position gets a vector of length $d_{model}$; each dimension pair oscillates at its own rate.

$$PE_{(pos,\,2i)} = \sin\!\left(\frac{pos}{10000^{\,2i/d_{model}}}\right), \qquad PE_{(pos,\,2i+1)} = \cos\!\left(\frac{pos}{10000^{\,2i/d_{model}}}\right)$$
what each symbol means
  • pos  token position — the row of the matrix (0, 1, 2, …).
  • i  index of the dimension pair, $i = 0 \dots d_{model}/2 - 1$.
  • d_model  embedding size $D$ — the number of columns.
  • 10000  the base constant that sets the slowest wavelength.

02 Why sin & cos, paired by frequency

Even dimensions use $\sin$, odd dimensions use $\cos$. Each pair $(2i,\,2i+1)$ shares a single angular frequency $\omega_i = 1/10000^{\,2i/d_{model}}$. The wavelengths form a geometric progression, from $2\pi$ at $i=0$ (fast wiggle) up to about $2\pi \cdot 10000$ at the last pair (barely moves). Low dimensions are the second hand of a clock; high dimensions are the hour hand — together they pinpoint a position like a binary counter.

building PE in PyTorch 
pos = torch.arange(seq_len).unsqueeze(1)                  # [T, 1]
i2  = torch.arange(0, d_model, 2)                          # [D/2]
div = torch.exp(-math.log(10000.0) * i2 / d_model)         # 1 / 10000^(2i/D)
pe = torch.zeros(seq_len, d_model)                         # [T, D]
pe[:, 0::2] = torch.sin(pos * div)                         # even dims
pe[:, 1::2] = torch.cos(pos * div)                         # odd dims
The exp / log trick: computing $10000^{2i/d}$ directly overflows for large exponents, so the code rewrites it as $\exp(-\log(10000)\cdot 2i/d)$ — the same number, computed stably. The result div holds the $D/2$ frequencies $\omega_i$.

Element-wise, not concatenated

Add, don't concatenate

The input to the first layer is X = TokenEmbedding(ids) + PE[:T]. That plus sign is a literal element-wise addition, not a concatenation: same dimensionality $D$, zero extra parameters — and in high-dimensional space the network easily learns to separate the "content + position" sum back into its parts. A dropout layer usually follows.

shapes
emb [B, T, D] + pe[:T] [T, D] → broadcast over B → X [B, T, D]

Concatenation would double the width to $2D$ and demand new weights everywhere downstream. Addition keeps the width fixed and free.

the forward step, in code 
x = self.embed(ids) * math.sqrt(d_model)   # [B, T, D]
x = x + self.pe[:, :T]                       # add positional encoding
x = self.dropout(x)
Why scale by $\sqrt{d_{model}}$? It boosts the token embedding so its magnitude is comparable to the unit-scale sinusoids — otherwise the position signal would drown out the content.

The reason it's sinusoids

Relative position

Here is the headline property. For any fixed offset $k$, the vector $PE_{pos+k}$ is a fixed linear (rotation) function of $PE_{pos}$ — each frequency pair rotates by a constant angle $\omega_i k$ that does not depend on $pos$. So attention can learn one position-independent transform meaning "look $k$ steps back," capturing relative position — and extrapolating to sequences longer than any seen in training.

$$\begin{bmatrix} PE_{pos+k,\,2i} \\ PE_{pos+k,\,2i+1} \end{bmatrix} = \begin{bmatrix} \cos\omega_i k & \sin\omega_i k \\ -\sin\omega_i k & \cos\omega_i k \end{bmatrix} \begin{bmatrix} PE_{pos,\,2i} \\ PE_{pos,\,2i+1} \end{bmatrix}$$

Shift-invariance: the curve peaks at $p = q$ and decays on both sides, and its shape depends only on the offset $(p - q)$, not on where $q$ sits. That offset-only dependence is the relative-position signal attention can exploit.
Beyond the 2017 sinusoid

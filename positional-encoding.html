Positional Encoding — teaching attention about order

Self-attention computes a weighted average over all tokens at once. It has no built-in notion of order: permute the input tokens and the outputs just permute the same way (it is permutation-equivariant). So "the cat sat" and "sat cat the" look identical to raw attention — a pure bag of words. Positional encoding fixes this by adding a position-dependent signal to each token embedding before attention ever runs.

the problem
Attention scores depend only on which tokens are present, not where they sit. Reorder the words and every attention weight follows along unchanged. Without extra information the model literally cannot tell a sentence from its shuffle.
the fix
Build a matrix $PE \in \mathbb{R}^{T \times D}$ where row $pos$ is a unique fingerprint for position $pos$. Add it to the token embeddings. Now identical words at different positions get different vectors, and attention can learn to use order.
RNNs get order for free — they read tokens one at a time, so step number is position. Transformers process every position in parallel for speed, so order is lost and must be injected explicitly. Positional encoding is the price of parallelism.

Where it sits in the pipeline

Shapes move like this (broadcast PE over the batch):

token ids [B, T] → embed [B, T, D] + PE[:T] [T, D] → X [B, T, D]

$B$ = batch, $T$ = sequence length, $D$ = $d_{model}$ (embedding size). The encoding is added once, right at the input, then attention and the rest of the stack run as usual.

The Sinusoidal Formula — Vaswani et al. 2017

The original Transformer uses a fixed (non-learned) encoding built from sines and cosines of geometrically-spaced frequencies. Each position gets a vector of length $d_{model}$; each dimension pair oscillates at its own rate.

$$PE_{(pos,\,2i)} = \sin\!\left(\frac{pos}{10000^{\,2i/d_{model}}}\right), \qquad PE_{(pos,\,2i+1)} = \cos\!\left(\frac{pos}{10000^{\,2i/d_{model}}}\right)$$
what each symbol means
  • pos token position — the row of the matrix (0, 1, 2, …).
  • i index of the dimension pair, $i = 0 \dots d_{model}/2 - 1$.
  • d_model embedding size $D$ — the number of columns.
  • 10000 the base constant that sets the slowest wavelength.

Why sin and cos, paired by frequency

Even dimensions use $\sin$, odd dimensions use $\cos$. Each pair $(2i,\,2i+1)$ shares a single angular frequency $\omega_i = 1/10000^{\,2i/d_{model}}$. The wavelengths form a geometric progression, from $2\pi$ at $i=0$ (fast wiggle) up to about $2\pi \cdot 10000$ at the last pair (barely moves). Low dimensions are like the second hand of a clock; high dimensions are like the hour hand — together they pinpoint a position the way a binary counter uses bits of different significance.

building PE in PyTorch
pos = torch.arange(seq_len).unsqueeze(1)                  # [T, 1]
i2  = torch.arange(0, d_model, 2)                          # [D/2]
div = torch.exp(-math.log(10000.0) * i2 / d_model)         # 1 / 10000^(2i/D)
pe = torch.zeros(seq_len, d_model)                         # [T, D]
pe[:, 0::2] = torch.sin(pos * div)                         # even dims
pe[:, 1::2] = torch.cos(pos * div)                         # odd dims
The exp / log trick: computing $10000^{2i/d}$ directly overflows for large exponents, so the code rewrites it as $\exp(-\log(10000)\cdot 2i/d)$ — the same number, computed stably. The result div holds the $D/2$ frequencies $\omega_i$.

Add, Don't Concatenate

The input to the first layer is X = TokenEmbedding(ids) + PE[:T]. That plus sign is a literal element-wise addition, not a concatenation. Same dimensionality $D$, zero extra parameters — and in high-dimensional space the network easily learns to separate the "content + position" sum back into its parts. A dropout layer usually follows.

emb [B, T, D] + pe[:T] [T, D] → broadcast over B → X [B, T, D]

Concatenation would double the width to $2D$ and demand new weights everywhere downstream. Addition keeps the width fixed and free.

the forward step, in code
x = self.embed(ids) * math.sqrt(d_model)   # [B, T, D]
x = x + self.pe[:, :T]                       # add positional encoding
x = self.dropout(x)
Why scale by $\sqrt{d_{model}}$? It boosts the token embedding so its magnitude is comparable to the unit-scale sinusoids — otherwise the position signal would drown out the content.

Relative Position — why sinusoids?

Here is the headline property. For any fixed offset $k$, the vector $PE_{pos+k}$ is a fixed linear (rotation) function of $PE_{pos}$ — each frequency pair just rotates by a constant angle $\omega_i k$ that does not depend on $pos$. So attention can learn a single, position-independent transform that means "look $k$ steps back," capturing relative position. It also lets the model extrapolate to sequences longer than any it saw in training.

$\begin{bmatrix} PE_{pos+k,\,2i} \\ PE_{pos+k,\,2i+1} \end{bmatrix} = \begin{bmatrix} \cos\omega_i k & \sin\omega_i k \\ -\sin\omega_i k & \cos\omega_i k \end{bmatrix} \begin{bmatrix} PE_{pos,\,2i} \\ PE_{pos,\,2i+1} \end{bmatrix}$

cosine similarity of $PE_{q}$ to every position · $D = 64$, base 10000

Shift-invariance: the curve always peaks at $p = q$ and decays smoothly on both sides. Crucially its shape depends only on the offset $(p - q)$, not on where $q$ sits in the sequence — slide $q$ and the bump simply translates. That offset-only dependence is the relative-position signal attention can exploit.

Variants — Beyond Sinusoidal

The 2017 sinusoid is one of several ways to inject order. The big modern families differ on where they act (added to the embedding vs. inside attention) and whether they encode absolute or relative position.

method how it works used by
Sinusoidal (fixed) Sin/cos table added to the embedding. No parameters, extrapolates to longer sequences. original Transformer
Learned absolute nn.Embedding(max_len, d_model) — a trainable row per position, added to the embedding. Capped at max_len; learns task-specific positions. BERT, GPT-2
RoPE (rotary) Rotates the query and key vectors by position-dependent angles inside attention (multiplicative, applied to Q/K — not the embedding). Excellent relative behavior. LLaMA, GPT-NeoX
ALiBi Adds a distance-proportional bias straight onto the attention scores. No position vectors at all; strong length extrapolation. BLOOM, MPT
The key axis: absolute methods (sinusoidal, learned) add a vector to the embedding before attention. Relative / rotary methods (RoPE, ALiBi) act inside attention — on the Q/K vectors or directly on the scores — so they encode the distance between tokens rather than their absolute index.

+added to embedding

Sinusoidal & learned absolute. One vector per position, summed in at the input.

×rotates Q / K

RoPE. Position becomes a rotation applied inside each attention head.

Δbiases scores

ALiBi. A penalty proportional to token distance, added to the raw attention logits.

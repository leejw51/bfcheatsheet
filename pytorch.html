Tensors

N-dimensional arrays with GPU support and autograd. The atom of PyTorch.

🧪 Shape Playground

Pick a creation function and shape — see what tensor you'd get.

Pitfall  torch.tensor(x) always copies; torch.as_tensor(x) avoids a copy when possible. And torch.from_numpy(arr) shares memory — modifying one mutates the other.

Autograd

Automatic differentiation. The reason PyTorch exists.

🔄 Gradient Flow

Forward pass → computes outputs. Backward pass ← propagates ∂L/∂x by chain rule.

forward activation gradient flowing back
Pitfall  Gradients accumulate across backward() calls. Always call optimizer.zero_grad() (or pass set_to_none=True) before each step, or your update will mix gradients from different batches.
requires_grad & backward
Tag inputs you want gradients for, then call .backward() on a scalar loss.
x = torch.tensor([1., 2.], requires_grad=True)
y = (x ** 2).sum()
y.backward()
print(x.grad)   # tensor([2., 4.])
no_grad / detach
Skip autograd to save memory at inference.
with torch.no_grad():
    preds = model(x)

z = y.detach()       # strip from graph

# Or as a decorator
@torch.no_grad()
def predict(x): ...
zero_grad
Grads accumulate by default — reset before each step.
optimizer.zero_grad(set_to_none=True)
loss.backward()
optimizer.step()
gradient clipping
Stabilize training by capping gradient norm.
torch.nn.utils.clip_grad_norm_(
    model.parameters(),
    max_norm=1.0,
)
custom Function
Define your own forward/backward for niche ops.
class MyReLU(torch.autograd.Function):
    @staticmethod
    def forward(ctx, x):
        ctx.save_for_backward(x)
        return x.clamp(min=0)

    @staticmethod
    def backward(ctx, grad_out):
        (x,) = ctx.saved_tensors
        return grad_out * (x > 0)
torch.autograd.grad
Get gradients without filling .grad — useful for higher-order.
grads = torch.autograd.grad(
    outputs=loss,
    inputs=[x, w],
    create_graph=True,    # for double-backward
    retain_graph=True,
)

torch.func (vmap, grad)
Functional autograd — batches and per-sample gradients.
from torch.func import vmap, grad, jacrev, hessian

def loss_fn(w, x, y): return ((w @ x - y) ** 2).sum()

g = grad(loss_fn)(w, x, y)
batched_g = vmap(grad(loss_fn), in_dims=(None, 0, 0))
J = jacrev(model)(x)
H = hessian(loss_fn)(w, x, y)
jvp / vjp / jacobian
Forward-mode (jvp) and reverse-mode (vjp) products.
from torch.func import jvp, vjp, jacfwd, jacrev

# JVP: forward-mode
out, jvp_out = jvp(f, (x,), (v,))

# VJP: reverse-mode
out, vjp_fn = vjp(f, x)
g = vjp_fn(grad_out)

J_fwd = jacfwd(f)(x)   # fat outputs
J_rev = jacrev(f)(x)   # tall outputs
functional_call
Run a Module with provided parameters (stateless).
from torch.func import functional_call

params = {n: p.detach().clone() for n, p in model.named_parameters()}

inference_mode
Stricter than no_grad — faster, blocks autograd-related state entirely.

with torch.set_grad_enabled(training):
    out = model(x)

with torch.enable_grad():
    # override an outer no_grad

nn.Module subclass
The canonical pattern.
class MLP(nn.Module):
    def __init__(self, d_in, d_hidden, d_out):
        super().__init__()
        self.fc1 = nn.Linear(d_in, d_hidden)
        self.fc2 = nn.Linear(d_hidden, d_out)

        x = F.relu(self.fc1(x))

canonical loop
The five magic lines.
model.train()
for epoch in range(num_epochs):
    for x, y in train_loader:
        x, y = x.to(device), y.to(device)

        optimizer.zero_grad()
        out = model(x)
        loss = loss_fn(out, y)
        loss.backward()
        optimizer.step()

LR schedulers
Adjust learning rate over time. Call .step() per epoch (or per step for OneCycleLR).

    CosineAnnealingLR, OneCycleLR, ReduceLROnPlateau,
    StepLR, MultiStepLR, ExponentialLR, LinearLR,

OneCycleLR
Per-step super-convergence schedule.

    optimizer, max_lr=0.1,
    total_steps=len(loader) * num_epochs,
    pct_start=0.3, anneal_strategy="cos",
)
# Call .step() every BATCH, not every epoch.
eval loop
Always wrap in no_grad and switch to eval() mode.
model.eval()
correct = 0
with torch.no_grad():
    for x, y in val_loader:
        x, y = x.to(device), y.to(device)
        preds = model(x).argmax(dim=1)
        correct += (preds == y).sum().item()
acc = correct / len(val_loader.dataset)
mixed precision (AMP)
Faster training and less memory with fp16/bf16.
scaler = torch.amp.GradScaler("cuda")

for x, y in loader:
    optimizer.zero_grad()
    with torch.autocast("cuda", dtype=torch.bfloat16):
        out = model(x)
        loss = loss_fn(out, y)
    scaler.scale(loss).backward()
    scaler.step(optimizer)
    scaler.update()

gradient accumulation

        optimizer.zero_grad()

torch.compile (PyTorch 2.x)

torch.compile basics
One-line JIT — TorchDynamo traces, Inductor codegens fused kernels. Often 1.5–2× on modern GPUs.
model = torch.compile(model)         # default
model = torch.compile(model, mode="reduce-overhead")   # CUDA Graphs
model = torch.compile(model, mode="max-autotune")        # slow first run
model = torch.compile(model, fullgraph=True)             # error on graph break
model = torch.compile(model, dynamic=True)               # dynamic shapes
@torch.compile
Compile any function — not just nn.Module.
@torch.compile
def step(x, w, b):
    return torch.relu(x @ w + b)

# Per-call options
@torch.compile(mode="reduce-overhead", fullgraph=True)
def loss_fn(x, y):
    return ((x - y) ** 2).mean()
debugging compile
Graph breaks kill speedup — surface them.
import torch._dynamo as dynamo

# Print every graph break with reason
dynamo.explain(model, sample)

# Force errors instead
dynamo.config.suppress_errors = False

Mixed Precision (AMP)

autocast contexts
Run matmul/conv in low precision automatically.
# bfloat16 — recommended on Ampere+ / TPU (no scaler needed)
with torch.autocast("cuda", dtype=torch.bfloat16):
    out = model(x)
    loss = loss_fn(out, y)

# float16 — needs GradScaler to avoid underflow
with torch.autocast("cuda", dtype=torch.float16):
    out = model(x)
GradScaler (fp16)
Scales the loss so tiny gradients survive fp16 → unscales before step().
scaler = torch.amp.GradScaler("cuda")

for x, y in loader:
    optimizer.zero_grad(set_to_none=True)
    with torch.autocast("cuda", dtype=torch.float16):
        loss = loss_fn(model(x), y)
    scaler.scale(loss).backward()
    scaler.unscale_(optimizer)             # for grad clipping
    torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
    scaler.step(optimizer)
    scaler.update()
bf16 vs fp16
Same memory; very different numerics.
# bfloat16: fp32 range, fp16 precision
# → no GradScaler needed, less likely to NaN
# → Ampere (A100, RTX 30xx)+ only on CUDA

# float16: smaller range, finer precision
# → ALWAYS use GradScaler
# → Volta+ supported, more memory-bandwidth wins

# Check support:
torch.cuda.is_bf16_supported()

Distributed: DDP vs FSDP

DDP — replicate everything
Each rank holds full model + optimizer state. Gradients all-reduced. Best when model fits.
import torch.distributed as dist
from torch.nn.parallel import DistributedDataParallel as DDP

dist.init_process_group("nccl")
local_rank = int(os.environ["LOCAL_RANK"])
torch.cuda.set_device(local_rank)

model = DDP(model.to(local_rank), device_ids=[local_rank])
sampler = torch.utils.data.distributed.DistributedSampler(ds)
# Launch:  torchrun --nproc_per_node=8 train.py
FSDP — shard everything
Shards params, grads, and optimizer state across ranks (ZeRO-3). For models too big for one GPU.
from torch.distributed.fsdp import (
    FullyShardedDataParallel as FSDP, MixedPrecision,
)
from torch.distributed.fsdp.wrap import size_based_auto_wrap_policy

mp = MixedPrecision(
    param_dtype=torch.bfloat16,
    reduce_dtype=torch.float32,
)
model = FSDP(
    model,
    auto_wrap_policy=size_based_auto_wrap_policy,
    mixed_precision=mp,
    device_id=torch.cuda.current_device(),
)
when to pick which
Quick decision rules.
# DDP — model fits on 1 GPU.
#   + simplest, fastest per-step
#   + every rank has the full model (easy eval/save)
#   - memory: O(model) per rank

# FSDP — model too big for 1 GPU.
#   + memory: O(model / world_size)
#   + composes with bf16 / activation checkpointing
#   - extra all-gather → slightly more comm

# Tensor-parallel + FSDP for multi-billion-param models.

torch.jit.script / trace
Freeze a model into a static graph.

m = torch.jit.freeze(m)
m = torch.jit.optimize_for_inference(m)

Common Gotchas

Forgot zero_grad: gradients accumulate every backward() call. Always reset.
view vs reshape: after transpose / permute the tensor isn't contiguous — view will error. Use reshape or .contiguous() first.
CrossEntropyLoss takes logits, not softmax outputs. Applying softmax first gives wrong gradients.
BatchNorm in eval(): running stats are used, not batch stats. Forgetting model.eval() at inference makes outputs depend on batch composition.
tensor.item() only works for 1-element tensors and forces CPU sync — avoid in hot loops.

$\nabla_{\!Z}\,L = P - Y$ — the identity that powers transformer training

Every classifier head in every transformer ends with a softmax and a cross-entropy loss. When you push gradients backward through that pair, almost everything magically cancels and you are left with one of the most beautiful equations in deep learning:

$\dfrac{\partial L}{\partial Z} \;=\; P - Y$

where $Z$ are the pre-softmax logits, $P=\mathrm{softmax}(Z)$ is the predicted distribution, and $Y$ is the one-hot target. The gradient on each logit is just "predicted probability minus target probability". No quotient rule in sight, no $\log$, no division. This single identity is why GPT, BERT, ViT and friends train at all.

why it matters
  • Fast. One subtraction per logit, no log/div in the backward pass.
  • Bounded. Each component lies in $[-1, 1]$ — gradients can't explode here.
  • Numerically stable. Skipping the explicit $\log p_c$ avoids the classic softmax-then-NLL overflow.
  • Universal. The same formula falls out for binary sigmoid/BCE, scaled-dot-product attention's softmax row, and any other softmax + cross-entropy stack.
where you'll see it
  • Language-model heads — "next-token logits minus one-hot true token".
  • Image classifiers — logits − one_hot(label).
  • RLHF / preference learning — same trick, different $Y$.
  • Even a knowledge-distillation soft target is just $P_{\!stud} - P_{\!teacher}$.

The whole story in five lines

PyTorch — what really happens behind F.cross_entropy
# Z: (B, V) logits   ·   y: (B,) integer labels
P  = softmax(Z, dim=-1)            # forward
L  = -log(P[arange(B), y]).mean()  # cross-entropy on the true class

# ↓ backward — what F.cross_entropy actually computes ↓
Y  = one_hot(y, V).float()       # (B, V)
dZ = (P - Y) / B                  # dL/dZ — that's it

Softmax — the probability machine

Softmax takes a vector of real numbers (logits) and turns it into a probability distribution: every component is in $(0, 1)$ and they all sum to $1$. Big logits dominate (because of the exponential), small logits get squashed.

the formula
$p_i \;=\; \dfrac{e^{z_i}}{\sum_{k=1}^{K} e^{z_k}}$

$K$ = number of classes (vocab size for a language model). The denominator $\sum_k e^{z_k}$ is the partition function — the normaliser that guarantees $\sum_i p_i = 1$.

Three properties that matter for the gradient proof

1sums to 1

$\sum_i p_i = 1$. So the gradient $P-Y$ has components that also sum to zero — you can sanity-check any backward pass.

2shift-invariant

$\mathrm{softmax}(Z) = \mathrm{softmax}(Z + c)$. Subtract the max before exponentiating to avoid overflow.

3self-derivative

$\dfrac{\partial p_i}{\partial z_j} = p_i(\delta_{ij} - p_j)$ — the heart of the derivation. We'll prove it in tab 4.

Numerically-stable softmax

the max-subtract trick — what every framework actually runs
def softmax(z):
    z = z - z.max()              # shift-invariance — avoid overflow
    e = np.exp(z)
    return e / e.sum()

Cross-Entropy — surprise on the true class

Cross-entropy measures how surprised the model is by the truth. Given a predicted distribution $P$ and a target distribution $Y$, it is:

$L = -\sum_{i=1}^{K} y_i \log p_i$

The one-hot collapse

For classification the target $Y$ is one-hot: $y_c = 1$ for the correct class $c$ and $y_i = 0$ everywhere else. Every term in the sum dies except $i = c$:

$L = -\sum_i y_i \log p_i \;=\; -\,1 \cdot \log p_c \;-\; 0 \cdot \log p_{\neq c} \;=\; -\log p_c$

Cross-entropy on a one-hot target is exactly negative log-likelihood of the true class. The whole batch loss reduces to averaging $-\log p_c$ over examples.

Pain curve — why $-\log$?

surprise grows without bound as $p_c \to 0$
$-\log 1.0 = 0$ — perfectly confident, zero loss.
$-\log 0.5 \approx 0.69$ — coin-flip on the right answer.
$-\log 0.1 \approx 2.30$ — bad, but recoverable.
$-\log 10^{-6} \approx 13.8$ — disaster: the gradient will be huge.

−log pc

The curve explodes as the model assigns the truth zero probability. That's the feedback signal that drives learning.

Soft targets, briefly

Sometimes $Y$ isn't one-hot — label smoothing, knowledge distillation, mixup. The derivation still gives $\partial L / \partial Z = P - Y$ verbatim; only the shape of $Y$ changes. Everything in this page works for any target distribution that sums to 1.

The derivation — every cancellation made visible

Six steps. Tightening each one is worth it: this is the proof you'll see again in attention backward passes, in mixture-of-experts routers, in any softmax-driven loss.

1

Start with the loss

Cross-entropy on a one-hot target collapses to negative log-likelihood:

$L \;=\; -\sum_{i=1}^{K} y_i \log p_i \;=\; -\log p_c$

where $c$ is the index of the correct class ($y_c = 1$, all others $0$).

2

Outer chain rule

$L$ depends on every $z_j$ through every $p_i$, so we need the full sum:

$\dfrac{\partial L}{\partial z_j} \;=\; \sum_{i=1}^{K} \dfrac{\partial L}{\partial p_i}\,\dfrac{\partial p_i}{\partial z_j}$

The first factor is straightforward: $\dfrac{\partial L}{\partial p_i} = -\dfrac{y_i}{p_i}$.

3

Differentiate softmax — the quotient rule

Recall $p_i = e^{z_i} / \sum_k e^{z_k}$. Let $S = \sum_k e^{z_k}$. Then:

$\dfrac{\partial p_i}{\partial z_j} \;=\; \dfrac{\partial}{\partial z_j}\!\left[\dfrac{e^{z_i}}{S}\right] \;=\; \dfrac{(\partial e^{z_i}/\partial z_j)\,S - e^{z_i}(\partial S/\partial z_j)}{S^2}$

$\partial e^{z_i}/\partial z_j = e^{z_j}$ if $i = j$, else $0$. And $\partial S/\partial z_j = e^{z_j}$. Plug in and simplify:

$\dfrac{\partial p_i}{\partial z_j} \;=\; p_i\,(\delta_{ij} - p_j)$

$\delta_{ij}$ is $1$ when $i = j$ and $0$ otherwise (Kronecker delta). The two cases: i = j $\;p_i(1 - p_i)$ i ≠ j $\;-p_i\,p_j$.

4

Plug into the chain rule

Substitute $\partial L/\partial p_i = -y_i/p_i$ and the softmax derivative:

$\dfrac{\partial L}{\partial z_j} \;=\; \sum_{i=1}^{K} \!\left(-\dfrac{y_i}{p_i}\right) p_i\,(\delta_{ij} - p_j) \;=\; -\sum_{i=1}^{K} y_i\,(\delta_{ij} - p_j)$

Notice $p_i$ in numerator and denominator cancel — that's the magic. No more division, no more risk of $\log 0$.

5

Distribute and simplify

Split the sum into the two pieces:

$\dfrac{\partial L}{\partial z_j} \;=\; -\sum_i y_i\,\delta_{ij} + p_j\sum_i y_i \;=\; -y_j + p_j \cdot 1 \;=\; p_j - y_j$

Two facts collapsed the sum: (a) $\sum_i y_i\,\delta_{ij} = y_j$ — only the $i = j$ term survives. (b) $\sum_i y_i = 1$ because targets are a probability distribution.

6

Vectorise

Collect every $j$ into one tidy vector / matrix equation:

$\dfrac{\partial L}{\partial Z} \;=\; P - Y$

For a batch, divide by $B$ if $L$ is the mean. That's the entire backward pass for the softmax + cross-entropy block — one subtraction, no exp, no log.

The two cases, explicitly

Walking through $j = c$ and $j \neq c$ separately makes the cancellation visceral.

j = c the correct class
$\dfrac{\partial L}{\partial z_c} = -\dfrac{1}{p_c} \cdot p_c(1 - p_c) = p_c - 1$

Always negative (since $p_c \in (0,1)$). Gradient descent increases $z_c$ — push the right logit up. Magnitude $1 - p_c$ is the model's "uncertainty" about the truth.

j ≠ c any wrong class
$\dfrac{\partial L}{\partial z_j} = -\dfrac{1}{p_c} \cdot (-p_c\,p_j) = p_j - 0 = p_j$

Always positive. Gradient descent decreases $z_j$ — push wrong logits down. Magnitude is exactly the probability mass the model misallocated.

Total magnitude check: $\sum_j (p_j - y_j) = \sum_j p_j - \sum_j y_j = 1 - 1 = 0$. The gradient vector always sums to zero. That's the geometric reason it lives on the "probability simplex's tangent plane".

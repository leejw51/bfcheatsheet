Lexicon · the model's first layer

How text becomes numbers

A neural network can't read letters — it does arithmetic on integers. The tokenizer is the bridge: it chops raw text into tokens (usually sub-word pieces), then maps each one to an integer ID. Everything the model knows about language starts here.

01 Three ideas to hold onto

idea 01

Integers, not text

The model sees [15496, 11, 995], never the string "Hello, world". Token IDs index into the embedding table.

idea 02

Sub-words, not words

Vocabularies are finite, language isn't. "tokenization" becomes token + ization — rare words are built from common pieces.

idea 03

A reversible map

encode turns text → IDs; decode turns IDs → text. The mapping is fixed once the vocab is trained.

Why not just one ID per word? English has millions of word forms (run, runs, running, runner…). A whole-word vocab would be huge and still miss new words. Sub-word tokenization keeps the vocabulary small (~30k–100k) while never hitting an "unknown" — worst case, it falls back to single bytes.

The pipeline

Modern tokenizers (HuggingFace tokenizers) run text through four stages, then hand the model a tensor of IDs.

02·b The shapes that come out

"Hello, world" → input_ids [B, T] attention_mask [B, T] → embeddings [B, T, D]

B = batch, T = sequence length (number of tokens), D = model width. The IDs index row-by-row into an embedding table of shape [vocab, D] — that lookup is the very first thing the transformer does.

huggingface · in code
from transformers import AutoTokenizer
tok = AutoTokenizer.from_pretrained("gpt2")

enc = tok("Hello, world", return_tensors="pt")
enc.input_ids        # tensor([[15496,    11,   995]])  shape [1, 3]
enc.attention_mask   # tensor([[1, 1, 1]])

tok.decode([15496, 11, 995])   # 'Hello, world'

Learning the vocabulary

Byte-Pair merges

BPE starts with every word as a string of single characters, then repeatedly merges the most frequent adjacent pair into a new symbol. Each merge becomes a permanent vocabulary entry. Step through it on a tiny corpus.

Three ways to split

BPE · WordPiece · Unigram

Every major model picks one of three sub-word algorithms. They differ in how they choose the vocabulary and how they mark word boundaries.

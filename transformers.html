level 01 // motivation

§01Why transformers?

Before 2017, the dominant sequence models were RNNs and LSTMs. They processed tokens one-at-a-time, carrying a hidden state forward. This had three crippling problems:

  • Sequential bottleneck. You can't parallelize across the time dimension — token t needs token t-1.
  • Vanishing gradients over distance. Information from token 1 is squeezed through every step before reaching token 100.
  • Fixed-size hidden state. The whole past has to fit in one vector. Long-range memory is fragile.
key insight

Attention is all you need

Vaswani et al. asked: what if every token could look at every other token directly, in one parallel step? No recurrence. No convolution. Just a learned mechanism that lets each position attend to every other position with a learned weight.

The result: training that scales linearly with hardware, gradients that flow in O(1) hops, and a model that can in principle relate the first word of a paragraph to the last as easily as to its neighbour.

level 02 // input

§02Tokenization & embeddings

A transformer can't read text. It reads integer IDs. Tokenization is the lossy compression scheme that turns a string of characters into a sequence of token IDs from a fixed vocabulary (typically 30k–200k entries). Modern tokenizers (BPE, WordPiece, Unigram, Tiktoken) split on subword pieces so rare words become a sequence of common chunks rather than an unknown token.

Each integer ID is then mapped to a dense vector via an embedding matrix $E \in \mathbb{R}^{V \times d_{model}}$ where $V$ is the vocab size. Lookup = one matrix row. The result: for a sequence of $L$ tokens you get a tensor of shape [L, d_model].

level 03 // order

§03Positional encoding

Self-attention is permutation-invariant: shuffle the input tokens and (without positional info) you get the same output. That's a problem for language. The fix: add a position-dependent vector to each token embedding before feeding the stack.

The original paper uses sinusoids of geometrically-increasing wavelength:

$$PE_{(pos,\,2i)} = \sin\!\Big(\tfrac{pos}{10000^{2i/d_{model}}}\Big), \qquad PE_{(pos,\,2i+1)} = \cos\!\Big(\tfrac{pos}{10000^{2i/d_{model}}}\Big)$$

visualizer

Sinusoidal position waves

Each row is a dimension i. Lower dims oscillate fast (positions look very different); higher dims are smooth (large-scale order).

The clever property: $PE_{pos+k}$ is a fixed linear function of $PE_{pos}$, so the model can learn relative offsets easily. Modern variants (RoPE, ALiBi, learned absolute) give different tradeoffs but the goal is the same — inject order.

level 04 // the core

§04Self-attention (Q · K · V)

Every input token gets projected into three vectors via three learned matrices $W^Q, W^K, W^V \in \mathbb{R}^{d_{model} \times d_k}$:

  • Query (Q) — "what am I looking for?"
  • Key (K) — "what do I contain?"
  • Value (V) — "what do I contribute, if attended to?"

The scaled dot-product attention is then:

$$\mathrm{Attention}(Q,K,V) = \mathrm{softmax}\!\Big(\tfrac{QK^\top}{\sqrt{d_k}}\Big)\,V$$

Shapes for sequence length $L$ and head dim $d_k$: Q: [L, d_k] K: [L, d_k] V: [L, d_k]. The product $QK^\top$ is [L, L] — one similarity score per (query, key) pair. Softmax along the rows turns it into attention weights, then we re-weight $V$.

level 05 // ensemble

§05Multi-head attention

One attention pattern is rarely enough. Multi-head attention runs $h$ attention operations in parallel, each with its own $W_i^Q, W_i^K, W_i^V$ projections, and concatenates the results:

$$\mathrm{MHA}(X) = \mathrm{Concat}(\mathrm{head}_1, \ldots, \mathrm{head}_h)\,W^O$$

Each head learns to attend to a different relational pattern — one head might track syntactic dependencies, another coreference, another simple position offsets. The split keeps the total compute roughly equal to single-head: with $h=8$ heads we pick $d_k = d_{model}/h$.

level 06 // glue

§06Feed-forward · residual · norm

After attention, each position passes through a position-wise feed-forward network — two linear layers with a non-linearity in between (originally ReLU, today usually GELU or SwiGLU):

$$\mathrm{FFN}(x) = \max(0,\, xW_1+b_1)\,W_2 + b_2$$

The hidden width is typically 4·d_model. This is where most of the model's parameters live — and a popular hypothesis is that this layer acts as a key-value memory storing factual knowledge.

Residual connections

Both the attention sublayer and the FFN sublayer are wrapped in residual connections + layer norm:

$$x \leftarrow \mathrm{LayerNorm}(x + \mathrm{Sublayer}(x))$$

Why? Residuals create a gradient highway from the loss back to the input, letting hundreds of layers train stably. LayerNorm keeps activations centred and scaled across the feature dimension, decoupling each layer's optimisation from the others.

Modern variants prefer pre-norm: $x \leftarrow x + \mathrm{Sublayer}(\mathrm{LayerNorm}(x))$ — gives more stable training at depth.

level 07 // topology

§07Encoder · decoder · masking

Three architecture flavours dominate today:

  • Encoder-only (BERT) — bidirectional self-attention. Every token sees every other token. Great for understanding tasks (classification, retrieval).
  • Decoder-only (GPT) — causal/masked self-attention. Token t can only attend to tokens $\le t$. Great for generation. This is what most modern LLMs use.
  • Encoder-decoder (T5, original NMT) — encoder reads source, decoder generates target while cross-attending to the encoder's output.

Causal masking — how?

Before softmax, set the upper-triangular entries of the score matrix to $-\infty$. Softmax of $-\infty$ is 0 — so the model is mathematically incapable of looking at the future:

level 08 // objective

§08Training: next-token prediction

For decoder-only LLMs (the GPT family), the entire training objective is shockingly simple:

Given tokens 1..t, predict token t+1.

The model outputs a probability distribution over the vocabulary at every position. The loss is the average cross-entropy against the actual next token:

$$\mathcal{L} = -\frac{1}{N}\sum_{t=1}^{N} \log P_\theta(x_{t+1} \mid x_1, \ldots, x_t)$$

Causal masking makes this efficient: a single forward pass on a length-$N$ sequence yields $N$ predictions in parallel, one per position. No teacher forcing tricks needed beyond shifting the labels by one.

Everything else — instruction tuning, RLHF, tool use — sits on top of this objective. Pretraining is the part that absorbs internet-scale data and where the model learns "everything." Fine-tuning is the part that shapes its behaviour.

recap

The full pipeline

  • 1. Tokens → integer IDs → embeddings (+ positional encoding)
  • 2. Stack of N transformer blocks: MHA → Add&Norm → FFN → Add&Norm
  • 3. Final linear projection to vocab size, softmax → next-token distribution
  • 4. Cross-entropy loss against ground-truth next token, summed over positions
  • 5. Backprop through every weight. Repeat ~10²² times. Done.
